# IDE от JetBrains

> Используйте Claude Code с IDE от JetBrains, включая IntelliJ, PyCharm, WebStorm и другие

Claude Code интегрируется с IDE от JetBrains через специальный плагин, предоставляя такие возможности, как интерактивный просмотр диффов, передача контекста выделения и многое другое.

## Поддерживаемые IDE

Плагин Claude Code работает с большинством IDE от JetBrains, включая:

* IntelliJ IDEA
* PyCharm
* Android Studio
* WebStorm
* PhpStorm
* GoLand

## Возможности

* **Быстрый запуск**: используйте `Cmd+Esc` (Mac) или `Ctrl+Esc` (Windows/Linux), чтобы открыть Claude Code прямо из редактора, или нажмите кнопку Claude Code в интерфейсе
* **Просмотр диффов**: изменения кода могут отображаться прямо в средстве просмотра диффов IDE, а не в терминале
* **Контекст выделения**: текущее выделение или вкладка в IDE автоматически передаётся в Claude Code. [Правила запрета `Read`](https://code.claude.com/docs/en/permissions#read-and-edit) блокируют эту передачу для подходящих под них файлов
* **Быстрые ссылки на файлы**: используйте `Cmd+Option+K` (Mac) или `Alt+Ctrl+K` (Linux/Windows), чтобы вставить ссылки на файлы, например `@src/auth.ts#L1-99`
* **Передача диагностики**: диагностические ошибки из IDE, такие как ошибки линтера и синтаксиса, автоматически передаются Claude по мере вашей работы

## Установка

Плагин запускает команду `claude` во встроенном терминале вашей IDE и подключается к ней. Он не содержит собственной копии CLI, поэтому установите обе части:

1. **Установите Claude Code CLI**

   Следуйте [краткому руководству](https://code.claude.com/docs/en/quickstart), чтобы установить CLI, если вы ещё этого не сделали. Плагин показывает уведомление «Cannot launch Claude Code», когда `claude` отсутствует в вашем PATH.

2. **Установите плагин для JetBrains**

   Установите [плагин Claude Code](https://plugins.jetbrains.com/plugin/27310-claude-code-beta-) из JetBrains Marketplace и перезапустите вашу IDE.

Если `claude` установлен там, где ваша IDE не может его найти, укажите полный путь в [настройке команды Claude](#general-settings) плагина.

Claude Code работает с любой платной подпиской Claude (Pro, Max, Team или Enterprise) либо с учётной записью Claude Console, при этом ключ API не требуется. При первом запуске `claude` вам будет предложено [войти в систему](https://code.claude.com/docs/en/authentication#log-in-to-claude-code).

> **Примечание.** После установки плагина вам может потребоваться полностью перезапустить IDE, чтобы он вступил в силу.

## Использование

### Из вашей IDE

Запустите `claude` из встроенного терминала вашей IDE, и все возможности интеграции будут активны.

### Из внешних терминалов

Используйте команду `/ide` в любом внешнем терминале, чтобы подключить Claude Code к вашей IDE от JetBrains и активировать все возможности:

```bash
claude
```

```text
/ide
```

Если вы хотите, чтобы Claude имел доступ к тем же файлам, что и ваша IDE, запускайте Claude Code из того же каталога, что и корень проекта вашей IDE.

## Конфигурация

### Настройки Claude Code

Настройте интеграцию с IDE через настройки Claude Code:

1. Запустите `claude`
2. Введите команду `/config`
3. Установите инструмент просмотра диффов в значение `auto`, чтобы показывать диффы в IDE, или `terminal`, чтобы оставить их в терминале

### Настройки плагина

Настройте плагин Claude Code, перейдя в **Settings → Tools → Claude Code \[Beta]**:

#### Общие настройки

* **Claude command**: укажите пользовательскую команду для запуска Claude, например `claude`, `/usr/local/bin/claude` или `npx @anthropic-ai/claude-code`
* **Suppress notification for Claude command not found**: пропускать уведомления о том, что команда Claude не найдена
* **Enable using Option+Enter for multi-line prompts**: только для macOS. Когда включено, Option+Enter вставляет новые строки в промптах Claude Code. Отключите, если клавиша Option перехватывается непредвиденным образом. Требует перезапуска терминала.
* **Enable automatic updates**: автоматически проверять и устанавливать обновления плагина, применяемые при перезапуске

> **Совет.** Для пользователей WSL: установите `wsl -d Ubuntu -- bash -lic "claude"` в качестве вашей команды Claude (замените `Ubuntu` на имя вашего дистрибутива WSL)

#### Настройка клавиши ESC

Если клавиша ESC не прерывает операции Claude Code в терминалах JetBrains:

1. Перейдите в **Settings → Tools → Terminal**
2. Затем либо:
   * снимите флажок «Move focus to the editor with Escape», либо
   * нажмите «Configure terminal keybindings» и удалите сочетание клавиш «Switch focus to Editor»
3. Примените изменения

Это позволит клавише ESC корректно прерывать операции Claude Code.

## Особые конфигурации

### Удалённая разработка

> **Предупреждение.** При использовании удалённой разработки JetBrains (Remote Development) вы должны установить плагин на удалённом хосте через **Settings → Plugin (Host)**.

Плагин должен быть установлен на удалённом хосте, а не на вашей локальной клиентской машине.

### Конфигурация WSL

Если вы используете Claude Code в WSL2 с IDE от JetBrains и видите «No available IDEs detected», причина обычно кроется в сетевом режиме NAT в WSL2 или в брандмауэре Windows, блокирующем соединение между WSL2 и IDE, работающей на хосте Windows. WSL1 напрямую использует сеть хоста и не подвержен этой проблеме.

#### Разрешите трафик WSL2 через брандмауэр Windows

Это рекомендуемое решение, поскольку оно сохраняет ваш текущий сетевой режим WSL2.

1. **Найдите свой IP-адрес WSL2**

   Изнутри вашей оболочки WSL выполните:

   ```bash
   hostname -I
   ```

   Обратите внимание на подсеть, например `172.21.123.45` находится в `172.21.0.0/16`.

2. **Создайте правило брандмауэра**

   Откройте PowerShell от имени администратора и выполните следующее, скорректировав диапазон IP-адресов в соответствии с вашей подсетью:

   ```powershell
   New-NetFirewallRule -DisplayName "Allow WSL2 Internal Traffic" -Direction Inbound -Protocol TCP -Action Allow -RemoteAddress 172.21.0.0/16 -LocalAddress 172.21.0.0/16
   ```

3. **Перезапустите вашу IDE и Claude Code**

   Закройте и снова откройте оба, чтобы новое правило вступило в силу.

#### Переключите WSL2 на зеркальный сетевой режим

Зеркальный сетевой режим (mirrored networking) требует Windows 11 22H2 или новее. Если у вас Windows 10, используйте вместо этого правило брандмауэра, описанное выше.

Добавьте это в `.wslconfig` в вашем пользовательском каталоге Windows:

```ini
[wsl2]
networkingMode=mirrored
```

Затем перезапустите WSL с помощью `wsl --shutdown` из PowerShell.

## Устранение неполадок

### Плагин не работает

Если плагин установлен, но возможности Claude Code не появляются в вашей IDE:

* Убедитесь, что вы запускаете Claude Code из корневого каталога проекта
* Проверьте, что плагин JetBrains включён в настройках IDE
* Полностью перезапустите IDE (возможно, придётся сделать это несколько раз)
* Для удалённой разработки убедитесь, что плагин установлен на удалённом хосте

### IDE не обнаружена

Если при запуске `claude` отображается «No available IDEs detected»:

* Убедитесь, что плагин установлен и включён
* Полностью перезапустите IDE
* Проверьте, что вы запускаете Claude Code из встроенного терминала
* Пользователям WSL см. раздел [Конфигурация WSL](#wsl-configuration) выше

### Команда не найдена

Если при нажатии на значок Claude отображается «command not found»:

1. Проверьте, что Claude Code установлен, выполнив `claude --version` в терминале
2. Настройте путь к команде Claude в настройках плагина
3. Пользователям WSL используйте формат команды WSL, упомянутый в разделе конфигурации

## Соображения безопасности

Когда Claude Code работает в IDE от JetBrains в [режиме разрешений `acceptEdits`](https://code.claude.com/docs/en/permission-modes#auto-approve-file-edits-with-acceptedits-mode), он может иметь возможность изменять конфигурационные файлы IDE, которые могут автоматически выполняться вашей IDE. Это может увеличить риск запуска Claude Code в режиме `acceptEdits` и позволить обойти запросы разрешений Claude Code на выполнение bash.

При работе в IDE от JetBrains учитывайте следующее:

* Используйте режим ручного подтверждения правок
* Проявляйте дополнительную осторожность, чтобы Claude использовался только с доверенными промптами
* Знайте, к каким файлам Claude Code имеет доступ для изменения

По вопросам установки или входа в Claude Code вне IDE см. [Устранение неполадок с установкой и входом](https://code.claude.com/docs/en/troubleshoot-install).

# Использование компьютера (превью)

> Включите использование компьютера в CLI Claude Code, чтобы Claude мог открывать приложения, кликать, печатать и видеть ваш экран на macOS. Тестируйте нативные приложения, отлаживайте визуальные проблемы и автоматизируйте инструменты, доступные только через графический интерфейс, не покидая терминал.

> **Примечание.** Использование компьютера — это исследовательское превью на macOS, для которого требуется план Pro или Max. Оно недоступно на планах Team и Enterprise. Для него требуется Claude Code версии 2.1.85 или новее и интерактивная сессия, поэтому оно недоступно в неинтерактивном режиме с флагом `-p`.

Использование компьютера позволяет Claude открывать приложения, управлять вашим экраном и работать на вашей машине так же, как это делали бы вы. Из CLI Claude может скомпилировать приложение на Swift, запустить его, прокликать каждую кнопку и сделать скриншот результата — всё в той же беседе, где он написал код.

Эта страница описывает, как использование компьютера работает в CLI. О настольном приложении на macOS или Windows см. [использование компьютера в Desktop](https://code.claude.com/docs/en/desktop#let-claude-use-your-computer).

## Что можно делать с использованием компьютера

Использование компьютера решает задачи, требующие графического интерфейса: всё то, что вам обычно пришлось бы делать вручную, выйдя из терминала.

* **Сборка и проверка нативных приложений**: попросите Claude собрать приложение для строки меню macOS. Claude напишет Swift, скомпилирует его, запустит и прокликает каждый элемент управления, чтобы убедиться, что всё работает, ещё до того как вы его откроете.
* **Сквозное тестирование интерфейса**: укажите Claude на локальное приложение Electron и скажите «протестируй процесс онбординга». Claude откроет приложение, прокликает регистрацию и сделает скриншот каждого шага. Никакой конфигурации Playwright, никакого тестового стенда.
* **Отладка проблем с отображением и вёрсткой**: скажите Claude «модальное окно обрезается на маленьких окнах». Claude изменит размер окна, воспроизведёт баг, сделает его скриншот, поправит CSS и проверит исправление. Claude видит то же, что и вы.
* **Управление инструментами только с графическим интерфейсом**: взаимодействуйте с инструментами дизайна, панелями управления оборудованием, iOS Simulator или проприетарными приложениями, у которых нет CLI или API.

## Когда применяется использование компьютера

У Claude есть несколько способов взаимодействия с приложением или сервисом. Использование компьютера — самый широкий и самый медленный, поэтому Claude сначала пробует наиболее точный инструмент:

* Если у вас есть [MCP-сервер](https://code.claude.com/docs/en/mcp) для сервиса, Claude использует его.
* Если задача — это команда оболочки, Claude использует Bash.
* Если задача связана с работой в браузере и у вас настроен [Claude в Chrome](https://code.claude.com/docs/en/chrome), Claude использует его.
* Если ничего из этого не подходит, Claude использует использование компьютера.

Управление экраном зарезервировано для того, до чего не может добраться ничто другое: нативные приложения, симуляторы и инструменты без API.

## Включение использования компьютера

Использование компьютера доступно как встроенный MCP-сервер под названием `computer-use`. По умолчанию он выключен, пока вы его не включите.

1. **Откройте меню MCP**

   В интерактивной сессии Claude Code выполните:

   ```text
   /mcp
   ```

   Найдите `computer-use` в списке серверов. Он отображается как отключённый.

2. **Включите сервер**

   Выберите `computer-use` и нажмите **Enable**. Настройка сохраняется для каждого проекта, поэтому вы делаете это лишь один раз для каждого проекта, где хотите использовать использование компьютера.

3. **Предоставьте разрешения macOS**

   Когда Claude впервые попытается использовать ваш компьютер, вы увидите запрос на предоставление двух разрешений macOS:

   * **Accessibility (Универсальный доступ)**: позволяет Claude кликать, печатать и прокручивать
   * **Screen Recording (Запись экрана)**: позволяет Claude видеть, что находится на вашем экране

   Запрос включает ссылки для открытия соответствующей панели системных настроек. Предоставьте оба разрешения, затем нажмите **Try again** в запросе. macOS может потребовать перезапустить Claude Code после предоставления разрешения на запись экрана.

После настройки попросите Claude сделать что-то, что требует графического интерфейса:

```text
Build the app target, launch it, and click through each tab to make
sure nothing crashes. Screenshot any error states you find.
```

## Одобрение приложений на время сессии

Включение сервера `computer-use` не даёт Claude доступ к каждому приложению на вашей машине. Когда Claude впервые в рамках сессии понадобится конкретное приложение, в вашем терминале появится запрос, показывающий:

* Какими приложениями Claude хочет управлять
* Любые дополнительно запрашиваемые разрешения, например доступ к буферу обмена
* Сколько других приложений будет скрыто, пока Claude работает

Выберите **Allow for this session** или **Deny**. Одобрения действуют в течение текущей сессии. Вы можете одобрить несколько приложений сразу, когда Claude запрашивает их вместе.

Приложения с широким охватом показывают в запросе дополнительное предупреждение, чтобы вы понимали, что даёт их одобрение:

| Предупреждение                          | Относится к                                                        |
| :-------------------------------------- | :---------------------------------------------------------------- |
| Эквивалентно доступу к оболочке         | Terminal, iTerm, VS Code, Warp и другие терминалы и IDE           |
| Может читать или записывать любой файл   | Finder                                                            |
| Может изменять системные настройки       | System Settings                                                   |

Эти приложения не блокируются. Предупреждение позволяет вам решить, оправдывает ли задача такой уровень доступа.

Уровень контроля Claude также различается в зависимости от категории приложения: браузеры и торговые платформы доступны только для просмотра, терминалы и IDE — только для кликов, а всё остальное получает полный контроль. Полную разбивку по уровням см. в разделе [разрешения приложений в Desktop](https://code.claude.com/docs/en/desktop#app-permissions).

## Как Claude работает на вашем экране

Понимание процесса помогает предвидеть, что будет делать Claude и как вмешаться.

### Одна сессия за раз

Использование компьютера удерживает общесистемную блокировку с момента первого действия по использованию компьютера и до тех пор, пока не завершится сессия, которая её взяла. Начиная с версии 2.1.195 завершение задачи не снимает блокировку; её снимает только выход из сессии. Если другая сессия Claude Code уже использует ваш компьютер, новые попытки завершаются неудачей с сообщением, указывающим, какая сессия удерживает блокировку. Сначала завершите ту сессию.

### Приложения скрываются, пока Claude работает

Когда Claude начинает управлять вашим экраном, другие видимые приложения скрываются, чтобы Claude взаимодействовал только с одобренными приложениями. Ваше окно терминала остаётся видимым и исключается из скриншотов, поэтому вы можете наблюдать за сессией, а Claude никогда не видит собственный вывод.

Когда Claude завершает ход, скрытые приложения восстанавливаются автоматически.

### Скриншоты автоматически уменьшаются

Claude Code уменьшает масштаб каждого скриншота перед отправкой его модели. Вам не нужно понижать разрешение дисплея или изменять размеры окон на дисплеях Retina или других дисплеях высокого разрешения. 16-дюймовый MacBook Pro в нативном разрешении Retina снимает при 3456×2234 и уменьшает примерно до 1372×887, сохраняя соотношение сторон.

Настройки для изменения целевого размера нет. Если текст или элементы управления на экране слишком малы, чтобы Claude мог их прочитать после уменьшения, увеличьте их размер в самом приложении, а не меняйте разрешение дисплея.

### Остановка в любой момент

Когда Claude получает блокировку, появляется уведомление macOS: «Claude is using your computer · press Esc to stop» («Claude использует ваш компьютер · нажмите Esc, чтобы остановить»). Нажмите `Esc` где угодно, чтобы немедленно прервать текущее действие, или нажмите `Ctrl+C` в терминале. В любом случае Claude останавливается, снова показывает ваши приложения и возвращает вам управление. Сессия удерживает [блокировку использования компьютера](#one-session-at-a-time) до своего завершения.

Второе уведомление появляется, когда Claude завершил работу.

## Безопасность и граница доверия

> **Предупреждение.** В отличие от [изолированного (sandboxed) инструмента Bash](https://code.claude.com/docs/en/sandboxing), использование компьютера работает на вашем реальном рабочем столе с доступом к приложениям, которые вы одобрили. Claude проверяет каждое действие и помечает потенциальные инъекции промптов из отображаемого на экране содержимого, но граница доверия здесь иная. Рекомендации по безопасности см. в [руководстве по безопасности использования компьютера](https://support.claude.com/en/articles/14128542).

Встроенные защитные механизмы снижают риск без необходимости настройки:

* **Одобрение по каждому приложению**: Claude может управлять только теми приложениями, которые вы одобрили в текущей сессии.
* **Предупреждения-«часовые»**: приложения, дающие доступ к оболочке, файловой системе или системным настройкам, помечаются до того, как вы их одобрите.
* **Терминал исключён из скриншотов**: Claude никогда не видит окно вашего терминала, поэтому отображаемые на экране запросы в вашей сессии не могут вернуться обратно в модель.
* **Глобальный escape**: клавиша `Esc` прерывает использование компьютера откуда угодно, а нажатие клавиши поглощается, поэтому инъекция промпта не может использовать её для закрытия диалогов.
* **Файл блокировки**: только одна сессия может управлять вашей машиной за раз.

## Примеры рабочих процессов

Эти примеры показывают распространённые способы сочетать использование компьютера с задачами по написанию кода.

### Проверка нативной сборки

После внесения изменений в приложение для macOS или iOS попросите Claude скомпилировать и проверить всё за один проход:

```text
Build the MenuBarStats target, launch it, open the preferences window,
and verify the interval slider updates the label. Screenshot the
preferences window when you're done.
```

Claude запускает `xcodebuild`, запускает приложение, взаимодействует с интерфейсом и сообщает, что обнаружил.

### Воспроизведение бага вёрстки

Когда визуальный баг появляется только при определённых размерах окна, дайте Claude найти его:

```text
The settings modal clips its footer on narrow windows. Resize the app
window down until you can reproduce it, screenshot the clipped state,
then check the CSS for the modal container.
```

Claude изменяет размер окна, фиксирует «сломанное» состояние и читает соответствующие таблицы стилей.

### Тестирование процесса в симуляторе

Управляйте iOS Simulator без написания XCTest:

```text
Open the iOS Simulator, launch the app, tap through the onboarding
screens, and tell me if any screen takes more than a second to load.
```

Claude управляет симулятором так же, как это делали бы вы с помощью мыши.

## Отличия от настольного приложения

Поверхности CLI и Desktop используют один и тот же движок использования компьютера, с несколькими отличиями:

| Возможность                | Desktop                                                     | CLI                                     |
| :------------------------- | :--------------------------------------------------------- | :-------------------------------------- |
| Платформы                  | macOS и Windows                                             | Только macOS                            |
| Включение                  | Переключатель в **Settings > General** (в разделе **Desktop app**) | Включение `computer-use` в `/mcp` |
| Список запрещённых приложений | Настраивается в Settings                                 | Пока недоступно                         |
| Переключатель авто-возврата | Опционально                                               | Всегда включено                         |
| Интеграция с Dispatch      | Сессии, порождённые Dispatch, могут использовать использование компьютера | Неприменимо                 |

## Устранение неполадок

### «Computer use is in use by another Claude session»

Другая сессия Claude Code удерживает блокировку, которую она сохраняет до своего завершения. Завершите ту сессию. Если другая сессия аварийно завершилась, блокировка снимается автоматически, когда Claude обнаруживает, что процесс больше не запущен.

### Запрос разрешений macOS продолжает появляться

macOS иногда требует перезапуска запрашивающего процесса после того, как вы предоставили разрешение на запись экрана. Полностью закройте Claude Code и запустите новую сессию. Если запрос не исчезает, откройте **System Settings > Privacy & Security > Screen Recording** и убедитесь, что ваше терминальное приложение есть в списке и включено.

### `computer-use` не появляется в `/mcp`

Сервер появляется только в подходящих конфигурациях. Проверьте, что:

* Вы на macOS. Использование компьютера в CLI недоступно на Linux или Windows. На Windows используйте [использование компьютера в Desktop](https://code.claude.com/docs/en/desktop#let-claude-use-your-computer).
* У вас Claude Code версии 2.1.85 или новее. Выполните `claude --version` для проверки.
* У вас план Pro или Max. Выполните `/status`, чтобы подтвердить свою подписку.
* Вы аутентифицированы через claude.ai. Использование компьютера недоступно со сторонними провайдерами, такими как Amazon Bedrock, Google Cloud Vertex AI или Microsoft Foundry. Если вы получаете доступ к Claude исключительно через стороннего провайдера, для использования этой функции вам нужна отдельная учётная запись claude.ai.
* Вы в интерактивной сессии. Использование компьютера недоступно в неинтерактивном режиме с флагом `-p`.

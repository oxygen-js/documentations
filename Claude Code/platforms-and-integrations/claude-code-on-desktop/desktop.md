# Настольное приложение

> Получите больше от Claude Code Desktop: параллельные сессии с изоляцией через Git, компоновка панелей перетаскиванием, встроенный терминал и редактор файлов, боковые чаты, управление компьютером, сессии Dispatch с телефона, визуальный просмотр диффов, предпросмотр приложений, мониторинг PR, коннекторы и корпоративная конфигурация.

Настольное приложение Claude Desktop имеет три вкладки: **Chat** для разговоров, **Cowork** для [Dispatch и более длительной агентной работы](https://claude.com/product/cowork) и **Code** для разработки ПО. Эта страница — справочник по вкладке Code.

* [Скачать для macOS](https://claude.ai/api/desktop/darwin/universal/dmg/latest/redirect?utm_source=claude_code&utm_medium=docs) — универсальная сборка для Intel и Apple Silicon
* [Скачать для Windows](https://claude.ai/api/desktop/win32/x64/setup/latest/redirect?utm_source=claude_code&utm_medium=docs) — для процессоров x64
* [Получить Claude для Linux (бета)](https://code.claude.com/docs/en/desktop-linux) — apt или .deb для Ubuntu и Debian

Для Windows ARM64 скачайте [установщик ARM64](https://claude.ai/api/desktop/win32/arm64/setup/latest/redirect?utm_source=claude_code&utm_medium=docs). В Linux устанавливайте через apt; см. [Claude Desktop в Linux](https://code.claude.com/docs/en/desktop-linux).

После установки запустите Claude, войдите в систему и нажмите вкладку **Code**. При первом открытии на Windows требуется установленный [Git for Windows](https://git-scm.com/downloads/win); перезапустите приложение после его установки. Пошаговое описание вашей первой сессии см. в [руководстве по началу работы](https://code.claude.com/docs/en/desktop-quickstart).

На вкладке Code каждый разговор — это **сессия**: у неё своя история чата, папка проекта и изменения кода, независимые от любой другой сессии. Боковая панель показывает список ваших сессий и позволяет запускать несколько параллельно. Внутри сессии вы можете:

* [Просматривать и комментировать диффы](#review-changes-with-diff-view), затем [следить за итоговым PR через CI](#monitor-pull-request-status)
* [Предпросматривать запущенное приложение](#preview-your-app) во встроенном браузере, пока Claude проверяет собственные изменения
* [Располагать панели](#arrange-your-workspace) для чата, диффа, предпросмотра, терминала и редактора файлов рядом
* Задать [побочный вопрос](#ask-a-side-question-without-derailing-the-session), который использует контекст сессии, не сбивая её с курса
* [Подключать внешние инструменты](#connect-external-tools), такие как GitHub, Slack и Linear
* Позволить Claude [открывать приложения и управлять вашим экраном](#let-claude-use-your-computer)
* Запускать на вашей машине, в [облаке](#run-long-running-tasks-remotely) или по [SSH](#ssh-sessions)

О [запланированной повторяющейся работе](https://code.claude.com/docs/en/desktop-scheduled-tasks), [сочетаниях клавиш](#keyboard-shortcuts) или [отправке задач с телефона](#sessions-from-dispatch) см. связанные страницы и разделы. Если вы уже используете CLI на основе терминала, см. [сравнение с CLI](#coming-from-the-cli), чтобы понять, что переносится.

## Начать сессию

Перед отправкой первого сообщения настройте четыре вещи в области промпта:

* **Окружение (Environment)**: выберите, где выполняется Claude. Выберите **Local** для вашей машины, **Remote** для облачных сессий на инфраструктуре Anthropic или [**SSH-подключение**](#ssh-sessions) для удалённой машины, которой вы управляете. См. [конфигурацию окружения](#environment-configuration).
* **Папка проекта (Project folder)**: выберите папку или репозиторий, в котором работает Claude. Для облачных сессий можно добавить [несколько репозиториев](#run-long-running-tasks-remotely).
* **Модель (Model)**: выберите [модель](https://code.claude.com/docs/en/model-config#available-models) из выпадающего списка рядом с кнопкой отправки. Вы можете изменить её в ходе сессии.
* **Режим разрешений (Permission mode)**: выберите, сколько автономии у Claude, с помощью [селектора режимов](#choose-a-permission-mode). Вы можете изменить его в ходе сессии.

Введите свою задачу и нажмите **Enter**, чтобы начать. Каждая сессия отслеживает свой контекст и изменения независимо.

## Работа с кодом

Дайте Claude нужный контекст, контролируйте, сколько он делает самостоятельно, и просматривайте, что он изменил.

### Используйте поле промпта

Введите то, что вы хотите, чтобы Claude сделал, и нажмите **Enter** для отправки. Claude читает файлы вашего проекта, вносит изменения и запускает команды в соответствии с вашим [режимом разрешений](#choose-a-permission-mode). Вы можете перенаправить Claude в любой момент: нажмите кнопку остановки, чтобы немедленно прервать, или введите поправку и нажмите **Enter**, чтобы отправить её, не останавливая выполняемое действие. Claude прочитает поправку, как только завершится текущее действие, и скорректируется перед следующим шагом.

Кнопка **+** рядом с полем промпта даёт доступ к вложениям файлов, [навыкам (skills)](#use-skills), [коннекторам](#connect-external-tools) и [плагинам](#install-plugins).

### Добавление файлов и контекста в промпты

Поле промпта поддерживает два способа добавления внешнего контекста:

* **@-упоминание файлов**: введите `@`, а затем имя файла, чтобы добавить файл в контекст разговора. После этого Claude сможет читать и ссылаться на этот файл. @-упоминание недоступно в облачных сессиях.
* **Прикрепление файлов**: прикрепляйте изображения, PDF и другие файлы к вашему промпту с помощью кнопки вложения или перетаскивайте файлы прямо в поле промпта. Это удобно для передачи скриншотов багов, макетов дизайна или справочных документов.

### Выбор режима разрешений

Режимы разрешений контролируют, сколько автономии у Claude во время сессии: спрашивает ли он перед редактированием файлов, запуском команд или тем и другим. Вы можете переключать режимы в любой момент с помощью селектора режимов рядом с кнопкой отправки. Начните с режима Ask permissions, чтобы точно видеть, что делает Claude, затем переходите к Auto accept edits или Plan mode по мере привыкания.

| Режим | Ключ настройки | Поведение |
| --- | --- | --- |
| **Ask permissions** | `default` | Claude спрашивает перед редактированием файлов или запуском команд. Вы видите дифф и можете принять или отклонить каждое изменение. Рекомендуется для новых пользователей. |
| **Auto accept edits** | `acceptEdits` | Claude автоматически принимает правки файлов и распространённые команды файловой системы, такие как `mkdir`, `touch` и `mv`, но всё равно спрашивает перед запуском других команд терминала. Используйте это, когда доверяете изменениям файлов и хотите более быстрой итерации. |
| **Plan mode** | `plan` | Claude читает файлы и запускает команды для исследования, затем предлагает план, не редактируя ваш исходный код. Хорошо подходит для сложных задач, где вы хотите сначала рассмотреть подход. |
| **Auto** | `auto` | Claude выполняет все действия с фоновыми проверками безопасности, которые подтверждают соответствие вашему запросу. Уменьшает количество запросов на разрешение при сохранении контроля. Включите в Settings → Claude Code. См. [требования к доступности](#auto-mode-availability) ниже. |
| **Bypass permissions** | `bypassPermissions` | Claude работает без запросов на разрешение, кроме тех, что принудительно заданы явными [правилами ask](https://code.claude.com/docs/en/permissions#manage-permissions); эквивалент `--dangerously-skip-permissions` в CLI. Включите в Settings → Claude Code в разделе «Allow bypass permissions mode». Используйте это только в изолированных контейнерах или ВМ. Корпоративные администраторы могут отключить эту опцию. |

Режим разрешений `dontAsk` доступен только в [CLI](https://code.claude.com/docs/en/permission-modes#allow-only-pre-approved-tools-with-dontask-mode).

Режим Auto — это исследовательский предпросмотр (research preview), доступный всем пользователям на Anthropic API, и он требует Claude Opus 4.6 или новее либо Sonnet 4.6 или новее. В корпоративных развёртываниях, которые направляют Desktop на Google Cloud Vertex AI, режим Auto отключён, пока вы не [зададите `CLAUDE_CODE_ENABLE_AUTO_MODE`](https://code.claude.com/docs/en/permission-modes#enable-auto-mode-on-bedrock-vertex-ai-or-foundry), и там поддерживаются только Claude Sonnet 5, Opus 4.7 и Opus 4.8.

> **Совет.** Лучшая практика: начинайте сложные задачи в режиме Plan, чтобы Claude наметил подход перед внесением изменений. После утверждения плана переключитесь на Auto accept edits или Ask permissions для его выполнения. Подробнее об этом рабочем процессе см. [сначала исследуйте, затем планируйте, затем пишите код](https://code.claude.com/docs/en/best-practices#explore-first-then-plan-then-code).

Облачные сессии поддерживают Accept edits, Plan mode и Auto mode. Accept edits соответствует режиму `default`: облачные сессии заранее одобряют правки файлов, поэтому селектор показывает Accept edits вместо Ask permissions. Bypass permissions недоступен, потому что облачное окружение уже изолировано.

Корпоративные администраторы могут ограничить, какие режимы разрешений доступны. Подробнее см. [корпоративную конфигурацию](#enterprise-configuration).

### Предпросмотр вашего приложения

Claude может запустить dev-сервер и открыть встроенный браузер, чтобы проверить свои изменения. Это работает как для фронтенд-веб-приложений, так и для бэкенд-серверов: Claude может тестировать API-эндпоинты, просматривать логи сервера и итеративно устранять найденные проблемы. В большинстве случаев Claude запускает сервер автоматически после редактирования файлов проекта. Вы также можете попросить Claude выполнить предпросмотр в любой момент. По умолчанию Claude [автоматически проверяет](#auto-verify-changes) изменения после каждой правки.

Панель предпросмотра также может открывать статические HTML-файлы, PDF, изображения и видео из вашего проекта. Нажмите на путь к HTML, PDF, изображению или видео в чате, чтобы открыть его в предпросмотре.

Из панели предпросмотра вы можете:

* Взаимодействовать с запущенным приложением прямо во встроенном браузере
* Наблюдать, как Claude автоматически проверяет собственные изменения: он делает скриншоты, инспектирует DOM, кликает по элементам, заполняет формы и исправляет найденные проблемы
* Запускать или останавливать серверы из выпадающего меню **Preview** на панели инструментов сессии
* Сохранять cookie и локальное хранилище между перезапусками сервера, выбрав **Persist sessions** в выпадающем меню, чтобы не приходилось повторно входить в систему во время разработки
* Редактировать конфигурацию сервера или останавливать все серверы сразу

Claude создаёт начальную конфигурацию сервера на основе вашего проекта. Если ваше приложение использует пользовательскую команду dev, отредактируйте `.claude/launch.json`, чтобы она соответствовала вашей настройке. Полный справочник см. в разделе [Настройка серверов предпросмотра](#configure-preview-servers).

Чтобы очистить сохранённые данные сессий, отключите **Persist preview sessions** в Settings → Claude Code. Чтобы полностью отключить предпросмотр, отключите **Preview** в Settings → Claude Code.

### Просмотр изменений в режиме диффа

После того как Claude вносит изменения в ваш код, режим диффа позволяет просмотреть модификации файл за файлом перед созданием pull request.

Когда Claude изменяет файлы, появляется индикатор статистики диффа, показывающий количество добавленных и удалённых строк, например `+12 -1`. Нажмите на этот индикатор, чтобы открыть просмотрщик диффов, который показывает список файлов слева и изменения для каждого файла справа.

Чтобы прокомментировать конкретные строки, нажмите на любую строку в диффе, чтобы открыть поле комментария. Введите свой отзыв и нажмите **Enter**, чтобы добавить комментарий. После добавления комментариев к нескольким строкам отправьте все комментарии сразу:

* **macOS**: нажмите **Cmd+Enter**
* **Windows**: нажмите **Ctrl+Enter**

Claude читает ваши комментарии и вносит запрошенные изменения, которые появляются как новый дифф, доступный для просмотра.

### Проверка вашего кода

В режиме диффа нажмите **Review code** на панели инструментов в правом верхнем углу, чтобы попросить Claude оценить изменения перед коммитом. Claude изучает текущие диффы и оставляет комментарии прямо в режиме диффа. Вы можете ответить на любой комментарий или попросить Claude внести правки.

Проверка сосредоточена на значимых проблемах: ошибках компиляции, явных логических ошибках, уязвимостях безопасности и очевидных багах. Она не отмечает стиль, форматирование, ранее существовавшие проблемы или что-либо, что поймал бы линтер.

### Мониторинг статуса pull request

После того как вы открываете pull request, в сессии появляется панель статуса CI. Claude Code использует GitHub CLI для опроса результатов проверок и отображения сбоев.

* **Auto-fix**: когда включено, Claude автоматически пытается исправить падающие проверки CI, читая вывод об ошибке и итеративно устраняя её.
* **Auto-merge**: когда включено, Claude сливает PR после прохождения всех проверок. Метод слияния — squash. Для работы этого auto-merge должен быть [включён в настройках вашего репозитория GitHub](https://docs.github.com/en/repositories/configuring-branches-and-merges-in-your-repository/configuring-pull-request-merges/managing-auto-merge-for-pull-requests-in-your-repository).

Используйте переключатели **Auto-fix** и **Auto-merge** на панели статуса CI, чтобы включить любую из опций. Claude Code также отправляет уведомление на рабочий стол, когда CI завершается. Чтобы автоматически архивировать сессию после слияния или закрытия PR, включите [автоархивацию](#work-in-parallel-with-sessions) в Settings → Claude Code.

> **Примечание.** Мониторинг PR требует, чтобы на вашей машине был установлен и аутентифицирован [GitHub CLI (`gh`)](https://cli.github.com/). Если `gh` не установлен, Desktop предложит установить его при первой попытке создать PR.

## Организация рабочего пространства

Вкладка Code построена вокруг панелей, которые можно располагать в любой компоновке: чат, дифф, предпросмотр, терминал, файл, план, задачи и субагент. Перетащите панель за её заголовок, чтобы переместить, или потяните за край панели, чтобы изменить размер. Нажмите **Cmd+\\** на macOS или **Ctrl+\\** на Windows, чтобы закрыть панель в фокусе. Открывайте дополнительные панели из меню **Views** на панели инструментов сессии.

> **Примечание.** Компоновка панелей, терминал, редактор файлов и режимы отображения из этого раздела требуют Claude Desktop v1.2581.0 или новее. Откройте **Claude → Check for Updates** на macOS или **Help → Check for Updates** на Windows, чтобы обновиться.

### Запуск команд в терминале

Встроенный терминал позволяет запускать команды рядом с вашей сессией, не переключаясь на другое приложение. Откройте его из меню **Views** или нажмите **Ctrl+`** на macOS или Windows. Терминал открывается в рабочем каталоге вашей сессии и использует то же окружение, что и Claude, поэтому команды вроде `npm test` или `git status` видят те же файлы, что редактирует Claude. Чтобы открыть вторую вкладку терминала, нажмите **+** в заголовке панели терминала или щёлкните правой кнопкой по папке в чате и выберите **Open in terminal**. Терминал доступен только в локальных сессиях.

### Открытие и редактирование файлов

Нажмите на путь к файлу в чате или просмотрщике диффов, чтобы открыть его в панели файла. Пути к HTML, PDF, изображениям и видео открываются вместо этого в [панели предпросмотра](#preview-your-app). Вносите точечные правки и нажимайте **Save**, чтобы записать их обратно. Если файл изменился на диске с момента его открытия, панель предупредит вас и позволит перезаписать или отменить. Нажмите **Discard**, чтобы отменить свои правки, или нажмите на путь в заголовке панели, чтобы скопировать абсолютный путь.

Панель файла доступна в локальных и SSH-сессиях. Для облачных сессий попросите Claude внести изменение.

### Открытие файлов в других приложениях

Щёлкните правой кнопкой по любому пути к файлу в чате, просмотрщике диффов или панели файла, чтобы открыть контекстное меню:

* **Attach as context**: добавить файл к вашему следующему промпту
* **Open in**: открыть файл в установленном редакторе, таком как VS Code, Cursor или Zed
* **Show in Finder** на macOS, **Show in Explorer** на Windows: открыть содержащую папку
* **Copy path**: скопировать абсолютный путь в буфер обмена

### Переключение режимов отображения

Режимы отображения контролируют, сколько деталей появляется в транскрипте чата. Переключайте режимы из выпадающего списка **Transcript view** рядом с кнопкой отправки или нажмите **Ctrl+O** на macOS или Windows, чтобы циклически переключаться между ними.

| Режим | Что он показывает |
| --- | --- |
| **Normal** | Вызовы инструментов свёрнуты в сводки, с полными текстовыми ответами |
| **Verbose** | Каждый вызов инструмента, чтение файла и промежуточный шаг, который делает Claude |
| **Summary** | Только финальные ответы Claude и внесённые им изменения |

Используйте Verbose при отладке того, почему Claude предпринял конкретное действие. Используйте Summary, когда запускаете несколько сессий и хотите быстро просматривать результаты.

### Сочетания клавиш

Нажмите **Cmd+/** на macOS или **Ctrl+/** на Windows, чтобы увидеть все сочетания, доступные на вкладке Code. На Windows используйте **Ctrl** вместо **Cmd** для приведённых ниже сочетаний. Циклическое переключение сессий, переключение терминала и переключение режима отображения используют **Ctrl** на всех платформах.

| Сочетание | Действие |
| --- | --- |
| `Cmd` `/` | Показать сочетания клавиш |
| `Cmd` `N` | Новая сессия |
| `Cmd` `W` | Закрыть сессию |
| `Ctrl` `Tab` / `Ctrl` `Shift` `Tab` | Следующая или предыдущая сессия |
| `Cmd` `Shift` `]` / `Cmd` `Shift` `[` | Следующая или предыдущая сессия |
| `Esc` | Остановить ответ Claude |
| `Cmd` `Shift` `D` | Переключить панель диффа |
| `Cmd` `Shift` `P` | Переключить панель предпросмотра |
| `Cmd` `Shift` `S` | Выбрать элемент в предпросмотре |
| `Ctrl` `` ` `` | Переключить панель терминала |
| `Cmd` `\` | Закрыть панель в фокусе |
| `Cmd` `;` | Открыть боковой чат |
| `Ctrl` `O` | Циклически переключать режимы отображения |
| `Cmd` `Shift` `M` | Открыть меню режима разрешений |

Эти сочетания применяются только к вкладке Code. [Сочетания клавиш интерактивного режима](https://code.claude.com/docs/en/interactive-mode#keyboard-shortcuts) на основе терминала, такие как `Shift+Tab` для циклического переключения режимов, не применяются в Desktop.

### Проверка использования

Нажмите на кольцо использования рядом с выбором модели, чтобы увидеть текущее использование окна контекста и использование вашего плана за период. Использование контекста — на уровне сессии; использование плана делится между всеми вашими поверхностями Claude Code.

## Позвольте Claude использовать ваш компьютер

Управление компьютером позволяет Claude открывать ваши приложения, управлять вашим экраном и работать прямо на вашей машине так, как это делали бы вы. Попросите Claude протестировать нативное приложение в мобильном симуляторе, взаимодействовать с настольным инструментом, у которого нет CLI, или автоматизировать что-то, что работает только через GUI.

> **Примечание.** Управление компьютером — это исследовательский предпросмотр на macOS и Windows, который требует плана Pro или Max. Он недоступен на планах Team или Enterprise. Настольное приложение Claude Desktop должно быть запущено.

Управление компьютером по умолчанию отключено. [Включите его в настройках](#enable-computer-use), прежде чем Claude сможет управлять вашим экраном. На macOS также необходимо предоставить разрешения на «Универсальный доступ» (Accessibility) и «Запись экрана» (Screen Recording).

> **Предупреждение.** В отличие от [изолированного инструмента Bash](https://code.claude.com/docs/en/sandboxing), управление компьютером выполняется на вашем реальном рабочем столе с доступом ко всему, что вы одобрите. Claude проверяет каждое действие и отмечает потенциальные внедрения промптов (prompt injection) из отображаемого на экране содержимого, но граница доверия здесь иная. Рекомендации см. в [руководстве по безопасности управления компьютером](https://support.claude.com/en/articles/14128542).

### Когда применяется управление компьютером

У Claude есть несколько способов взаимодействия с приложением или сервисом, и управление компьютером — самый широкий и самый медленный. Он пробует наиболее точный инструмент первым:

* Если у вас есть [коннектор](#connect-external-tools) для сервиса, Claude использует коннектор.
* Если задача — это shell-команда, Claude использует Bash.
* Если задача — работа в браузере, и у вас настроен [Claude в Chrome](https://code.claude.com/docs/en/chrome), Claude использует это.
* Если ничего из этого не подходит, Claude использует управление компьютером.

[Уровни доступа для каждого приложения](#app-permissions) усиливают это: браузеры ограничены только просмотром, а терминалы и IDE — только кликами, что направляет Claude к специализированному инструменту, даже когда управление компьютером активно. Управление экраном зарезервировано для того, до чего ничто другое не может дотянуться, — нативных приложений, аппаратных панелей управления, мобильных симуляторов или проприетарных инструментов без API.

### Включение управления компьютером

Управление компьютером по умолчанию отключено. Если вы попросите Claude сделать что-то, что этого требует, пока оно отключено, Claude сообщит вам, что мог бы выполнить задачу, если вы включите управление компьютером в настройках.

1. **Обновите настольное приложение.** Убедитесь, что у вас последняя версия Claude Desktop. Скачайте или обновите на [claude.com/download](https://claude.com/download), затем перезапустите приложение.

2. **Включите переключатель.** В настольном приложении перейдите в **Settings > General** (в разделе **Desktop app**). Найдите переключатель **Computer use** и включите его. На Windows переключатель вступает в силу немедленно, и настройка завершена. На macOS перейдите к следующему шагу.

   Если вы не видите переключатель, убедитесь, что вы на macOS или Windows с планом Pro или Max, затем обновите и перезапустите приложение.

3. **Предоставьте разрешения macOS.** На macOS предоставьте два системных разрешения, прежде чем переключатель вступит в силу:

   * **Accessibility** (Универсальный доступ): позволяет Claude кликать, печатать и прокручивать
   * **Screen Recording** (Запись экрана): позволяет Claude видеть, что на вашем экране

   Страница настроек показывает текущий статус каждого разрешения. Если какое-либо из них отклонено, нажмите на значок, чтобы открыть соответствующую панель System Settings.

### Разрешения приложений

Когда Claude впервые нужно использовать приложение, в вашей сессии появляется запрос. Нажмите **Allow for this session** или **Deny**. Одобрения действуют на текущую сессию или 30 минут в [сессиях, порождённых Dispatch](#sessions-from-dispatch).

Запрос также показывает, какой уровень контроля Claude получает для этого приложения. Эти уровни фиксированы по категории приложения и не могут быть изменены:

| Уровень | Что может Claude | Применяется к |
| :-- | :-- | :-- |
| Только просмотр | Видеть приложение на скриншотах | Браузеры, торговые платформы |
| Только клики | Кликать и прокручивать, но не печатать и не использовать сочетания клавиш | Терминалы, IDE |
| Полный контроль | Кликать, печатать, перетаскивать и использовать сочетания клавиш | Всё остальное |

Приложения с широким охватом, такие как терминалы, Finder или File Explorer, и System Settings или Settings, показывают в запросе дополнительное предупреждение, чтобы вы знали, что даёт их одобрение.

Вы можете настроить два параметра в **Settings > General** (в разделе **Desktop app**):

* **Denied apps**: добавьте сюда приложения, чтобы отклонять их без запроса. Claude всё равно может косвенно повлиять на отклонённое приложение через действия в разрешённом приложении, но не может взаимодействовать с отклонённым приложением напрямую.
* **Unhide apps when Claude finishes**: пока Claude работает, ваши другие окна скрыты, чтобы он взаимодействовал только с одобренным приложением. Когда Claude заканчивает, скрытые окна восстанавливаются, если только вы не отключите этот параметр.

## Управление сессиями

Каждая сессия — это независимый разговор со своим контекстом и изменениями. Вы можете запускать несколько сессий параллельно, ответвлять боковые чаты, отправлять работу в облако или позволить Dispatch запускать сессии за вас с телефона.

### Параллельная работа с сессиями

Нажмите **+ New session** на боковой панели или нажмите **Cmd+N** на macOS или **Ctrl+N** на Windows, чтобы работать над несколькими задачами параллельно. Нажимайте **Ctrl+Tab** и **Ctrl+Shift+Tab**, чтобы циклически переключаться между сессиями на боковой панели. Для репозиториев Git каждая сессия получает свою изолированную копию проекта с помощью [рабочих деревьев Git (Git worktrees)](https://code.claude.com/docs/en/worktrees), так что изменения в одной сессии не затрагивают другие сессии, пока вы не сделаете коммит.

Чтобы просмотреть две сессии одновременно, удерживайте **Cmd** на macOS или **Ctrl** на Windows и нажмите на сессию в боковой панели. Сессия открывается во второй панели рядом с уже открытой. Пока разделение активно, нажатие на другую сессию в боковой панели заменяет ту панель, что в фокусе. Нажмите **Cmd+\\** на macOS или **Ctrl+\\** на Windows, чтобы закрыть панель в фокусе и вернуться к одной сессии.

Рабочие деревья по умолчанию хранятся в `<project-root>/.claude/worktrees/`. Вы можете изменить это на пользовательский каталог в Settings → Claude Code в разделе «Worktree location». Вы также можете задать префикс ветки, который добавляется к имени ветки каждого рабочего дерева, что полезно для упорядочивания веток, созданных Claude. Чтобы удалить рабочее дерево, когда закончите, наведите курсор на сессию в боковой панели и нажмите значок архивирования. Чтобы сессии архивировались сами при слиянии или закрытии их pull request, включите **Auto-archive after PR merge or close** в Settings → Claude Code. Автоархивация применяется только к локальным сессиям, которые завершили выполнение.

Чтобы включать файлы, игнорируемые gitignore, такие как `.env`, в новые рабочие деревья, создайте [файл `.worktreeinclude`](https://code.claude.com/docs/en/worktrees#copy-gitignored-files-into-worktrees) в корне вашего проекта.

> **Примечание.** Изоляция сессий требует [Git](https://git-scm.com/downloads). Большинство Mac включают Git по умолчанию. Запустите `git --version` в Terminal, чтобы проверить. На Windows Git необходим для работы вкладки Code: [скачайте Git for Windows](https://git-scm.com/downloads/win), установите его и перезапустите приложение. Если вы столкнётесь с ошибками Git, попросите Claude на [вкладке Cowork](https://claude.com/product/cowork) помочь устранить проблему с вашей настройкой.

Используйте элементы управления в верхней части боковой панели, чтобы фильтровать сессии по статусу, проекту или окружению и группировать сессии по проекту. Чтобы переименовать сессию, нажмите на заголовок сессии на панели инструментов в верхней части активной сессии. Чтобы проверить использование контекста, см. [Проверка использования](#check-usage). Когда контекст заполняется, Claude автоматически резюмирует разговор и продолжает работу. Вы также можете ввести `/compact`, чтобы запустить резюмирование раньше и освободить место в контексте. Подробнее о том, как работает уплотнение (compaction), см. [окно контекста](https://code.claude.com/docs/en/how-claude-code-works#the-context-window).

Настольное приложение отправляет уведомление ОС, когда сессия Code завершает задачу, а вы в этот момент не просматриваете эту сессию.

### Задайте побочный вопрос, не сбивая сессию с курса

Боковой чат позволяет задать Claude вопрос, который использует контекст вашей сессии, но ничего не добавляет обратно в основной разговор. Используйте его, когда хотите разобраться в фрагменте кода, проверить предположение или изучить идею, не сбивая сессию с курса.

Нажмите **Cmd+;** на macOS или **Ctrl+;** на Windows, чтобы открыть боковой чат, или введите `/btw` в поле промпта. Боковой чат может читать всё в основном треде вплоть до этого момента. Когда закончите, закройте боковой чат и продолжите основную сессию с того места, где остановились. Боковые чаты доступны в локальных и SSH-сессиях.

### Наблюдение за фоновыми задачами

Панель задач показывает фоновую работу, выполняемую внутри текущей сессии: субагентов, фоновые shell-команды и [динамические рабочие процессы](https://code.claude.com/docs/en/workflows). Откройте её из меню **Views** или перетащите в вашу компоновку.

Нажмите на любую запись, чтобы увидеть её вывод в панели субагента или остановить её. Чтобы увидеть, что делают другие сессии, используйте [боковую панель](#work-in-parallel-with-sessions).

### Запуск длительных задач удалённо

Для крупных рефакторингов, наборов тестов, миграций или других длительных задач выбирайте **Remote** вместо **Local** при запуске сессии. Облачные сессии выполняются на облачной инфраструктуре Anthropic и продолжаются, даже если вы закроете приложение или выключите компьютер. Возвращайтесь в любое время, чтобы увидеть прогресс или направить Claude в другую сторону. Вы также можете отслеживать облачные сессии с [claude.ai/code](https://claude.ai/code) или из приложения Claude для iOS.

Облачные сессии также поддерживают несколько репозиториев. После выбора облачного окружения нажмите кнопку **+** рядом с пилюлей репозитория, чтобы добавить в сессию дополнительные репозитории. Каждый репозиторий получает свой селектор ветки. Это удобно для задач, охватывающих несколько кодовых баз, например обновления общей библиотеки и её потребителей.

Подробнее о том, как работают облачные сессии, см. [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web).

### Продолжение на другой поверхности

Меню **Continue in**, доступное из значка VS Code в правом нижнем углу панели инструментов сессии, позволяет перенести вашу сессию на другую поверхность:

* **Claude Code on the Web**: отправляет вашу локальную сессию для продолжения удалённого выполнения. Desktop отправляет (push) вашу ветку, генерирует сводку разговора и создаёт новую облачную сессию с полным контекстом. Затем вы можете либо архивировать локальную сессию, либо оставить её. Это требует чистого рабочего дерева и недоступно для SSH-сессий.
* **Your IDE**: открывает ваш проект в поддерживаемой IDE в текущем рабочем каталоге.

### Сессии из Dispatch

[Dispatch](https://support.claude.com/en/articles/13947068) — это постоянный разговор с Claude, который живёт на вкладке [Cowork](https://claude.com/product/cowork#dispatch-and-computer-use). Вы отправляете Dispatch задачу, и он решает, как её обработать.

Задача может превратиться в сессию Code двумя способами: вы просите об этом напрямую, например «open a Claude Code session and fix the login bug», или Dispatch решает, что задача — это работа по разработке, и запускает сессию сам. К задачам, которые обычно направляются в Code, относятся исправление багов, обновление зависимостей, запуск тестов или открытие pull request. Исследования, редактирование документов и работа с таблицами остаются в Cowork.

В любом случае сессия Code появляется в боковой панели вкладки Code со значком **Dispatch**. Вы получаете push-уведомление на телефон, когда она завершается или ей нужно ваше одобрение.

Если у вас включено [управление компьютером](#let-claude-use-your-computer), сессии Code, порождённые Dispatch, тоже могут его использовать. Одобрения приложений в таких сессиях истекают через 30 минут и запрашиваются повторно, а не действуют всю сессию, как в обычных сессиях Code.

Инструкции по настройке, сопряжению и параметрам Dispatch см. в [справочной статье о Dispatch](https://support.claude.com/en/articles/13947068). Dispatch требует плана Pro или Max и недоступен на планах Team или Enterprise.

Dispatch — один из нескольких способов работать с Claude, когда вы вдали от терминала. См. [Платформы и интеграции](https://code.claude.com/docs/en/platforms#work-when-you-are-away-from-your-terminal), чтобы сравнить его с Remote Control, Channels, Slack и запланированными задачами.

## Расширение Claude Code

Подключайте внешние сервисы, добавляйте переиспользуемые рабочие процессы, настраивайте поведение Claude и конфигурируйте серверы предпросмотра. Чтобы управлять коннекторами, навыками и плагинами в одном месте, нажмите **Customize** на боковой панели.

### Подключение внешних инструментов

Для локальных и [SSH](#ssh-sessions)-сессий нажмите кнопку **+** рядом с полем промпта и выберите **Connectors**, чтобы добавить интеграции, такие как Google Calendar, Slack, GitHub, Linear, Notion и другие. Вы можете добавлять коннекторы до или во время сессии. Кнопка **+** недоступна в облачных сессиях, но [routines](https://code.claude.com/docs/en/routines) настраивают коннекторы во время создания routine.

Чтобы управлять коннекторами или отключить их, перейдите в Settings → Connectors в настольном приложении или выберите **Manage connectors** в меню Connectors в поле промпта.

После подключения Claude может читать ваш календарь, отправлять сообщения, создавать задачи (issues) и напрямую взаимодействовать с вашими инструментами. Вы можете спросить Claude, какие коннекторы настроены в вашей сессии.

Коннекторы — это [MCP-серверы](https://code.claude.com/docs/en/mcp) с графическим процессом настройки. Используйте их для быстрой интеграции с поддерживаемыми сервисами. Для интеграций, не перечисленных в Connectors, добавляйте MCP-серверы вручную через [файлы настроек](https://code.claude.com/docs/en/mcp#installing-mcp-servers). Вы также можете [создавать пользовательские коннекторы](https://support.claude.com/en/articles/11175166-getting-started-with-custom-connectors-using-remote-mcp).

### Использование навыков

[Навыки (skills)](https://code.claude.com/docs/en/skills) расширяют возможности Claude. Claude загружает их автоматически, когда они уместны, или вы можете вызвать один напрямую: введите `/` в поле промпта или нажмите кнопку **+** и выберите **Slash commands**, чтобы просмотреть доступное. Сюда входят [встроенные команды](https://code.claude.com/docs/en/commands), ваши [пользовательские навыки](https://code.claude.com/docs/en/skills#create-your-first-skill), проектные навыки из вашей кодовой базы и навыки из любых [установленных плагинов](https://code.claude.com/docs/en/plugins). Выберите один, и он появится выделенным в поле ввода. Введите свою задачу после него и отправьте как обычно.

### Установка плагинов

[Плагины](https://code.claude.com/docs/en/plugins) — это переиспользуемые пакеты, которые добавляют навыки, агентов, хуки, MCP-серверы и конфигурации LSP в Claude Code. Вы можете устанавливать плагины из настольного приложения, не используя терминал.

Для локальных и [SSH](#ssh-sessions)-сессий нажмите кнопку **+** рядом с полем промпта и выберите **Plugins**, чтобы увидеть установленные плагины и их навыки. Чтобы добавить плагин, выберите **Add plugin** из подменю, чтобы открыть браузер плагинов, который показывает доступные плагины из ваших настроенных [маркетплейсов](https://code.claude.com/docs/en/plugin-marketplaces), включая официальный маркетплейс Anthropic. Выберите **Manage plugins**, чтобы включить, отключить или удалить плагины.

Плагины могут быть привязаны к вашей учётной записи пользователя, конкретному проекту или только к локальному использованию. Если ваша организация управляет плагинами централизованно, эти плагины доступны в настольных сессиях так же, как и в CLI. Плагины недоступны для облачных сессий. Полный справочник по плагинам, включая создание собственных плагинов, см. в [плагинах](https://code.claude.com/docs/en/plugins).

### Настройка серверов предпросмотра

Claude автоматически определяет настройку вашего dev-сервера и сохраняет конфигурацию в `.claude/launch.json` в корне папки, которую вы выбрали при запуске сессии. Предпросмотр использует эту папку как рабочий каталог, поэтому, если вы выбрали родительскую папку, подпапки со своими собственными dev-серверами не будут обнаружены автоматически. Чтобы работать с сервером в подпапке, либо запустите сессию непосредственно в этой папке, либо добавьте конфигурацию вручную.

Чтобы настроить, как запускается ваш сервер, например использовать `yarn dev` вместо `npm run dev` или изменить порт, отредактируйте файл вручную или нажмите **Edit configuration** в выпадающем меню Preview, чтобы открыть его в вашем редакторе кода. Файл поддерживает JSON с комментариями.

```json
{
  "version": "0.0.1",
  "configurations": [
    {
      "name": "my-app",
      "runtimeExecutable": "npm",
      "runtimeArgs": ["run", "dev"],
      "port": 3000
    }
  ]
}
```

Вы можете определить несколько конфигураций, чтобы запускать разные серверы из одного проекта, например фронтенд и API. См. [примеры](#examples) ниже.

#### Автоматическая проверка изменений

Когда включён `autoVerify`, Claude автоматически проверяет изменения кода после редактирования файлов. Он делает скриншоты, проверяет наличие ошибок и подтверждает, что изменения работают, прежде чем завершить свой ответ.

Автопроверка включена по умолчанию. Отключите её для конкретного проекта, добавив `"autoVerify": false` в `.claude/launch.json`, или переключите её из выпадающего меню **Preview**.

```json
{
  "version": "0.0.1",
  "autoVerify": false,
  "configurations": [...]
}
```

Когда автопроверка отключена, инструменты предпросмотра всё равно доступны, и вы можете попросить Claude выполнить проверку в любой момент. Автопроверка делает это автоматически после каждой правки.

#### Поля конфигурации

Каждая запись в массиве `configurations` принимает следующие поля:

| Поле | Тип | Описание |
| --- | --- | --- |
| `name` | string | Уникальный идентификатор этого сервера |
| `runtimeExecutable` | string | Команда для запуска, например `npm`, `yarn` или `node` |
| `runtimeArgs` | string\[] | Аргументы, передаваемые в `runtimeExecutable`, например `["run", "dev"]` |
| `port` | number | Порт, который слушает ваш сервер. По умолчанию 3000 |
| `cwd` | string | Рабочий каталог относительно корня вашего проекта. По умолчанию корень проекта. Используйте `${workspaceFolder}`, чтобы явно сослаться на корень проекта |
| `env` | object | Дополнительные переменные окружения в виде пар ключ-значение, например `{ "NODE_ENV": "development" }`. Не помещайте сюда секреты, так как этот файл коммитится в ваш репозиторий. Чтобы передать секреты вашему dev-серверу, задайте их вместо этого в [редакторе локального окружения](#local-sessions). |
| `autoPort` | boolean | Как обрабатывать конфликты портов. См. ниже |
| `program` | string | Скрипт для запуска с `node`. См. [когда использовать `program` вместо `runtimeExecutable`](#when-to-use-program-vs-runtimeexecutable) |
| `args` | string\[] | Аргументы, передаваемые в `program`. Используются только когда задан `program` |

##### Когда использовать `program` вместо `runtimeExecutable`

Используйте `runtimeExecutable` с `runtimeArgs`, чтобы запустить dev-сервер через менеджер пакетов. Например, `"runtimeExecutable": "npm"` с `"runtimeArgs": ["run", "dev"]` запускает `npm run dev`.

Используйте `program`, когда у вас есть отдельный скрипт, который вы хотите запустить напрямую через `node`. Например, `"program": "server.js"` запускает `node server.js`. Передавайте дополнительные флаги через `args`.

#### Конфликты портов

Поле `autoPort` управляет тем, что происходит, когда предпочитаемый вами порт уже используется:

* **`true`**: Claude автоматически находит и использует свободный порт. Подходит для большинства dev-серверов.
* **`false`**: Claude завершается с ошибкой. Используйте это, когда ваш сервер должен использовать конкретный порт, например для OAuth-колбэков или списков разрешений CORS.
* **Не задано (по умолчанию)**: Claude спрашивает, нужен ли серверу именно этот порт, затем сохраняет ваш ответ.

Когда Claude выбирает другой порт, он передаёт назначенный порт вашему серверу через переменную окружения `PORT`.

#### Примеры

Эти конфигурации показывают распространённые настройки для разных типов проектов:

##### Next.js

Эта конфигурация запускает приложение Next.js с помощью Yarn на порту 3000:

```json
{
  "version": "0.0.1",
  "configurations": [
    {
      "name": "web",
      "runtimeExecutable": "yarn",
      "runtimeArgs": ["dev"],
      "port": 3000
    }
  ]
}
```

##### Несколько серверов

Для монорепозитория с фронтендом и API-сервером определите несколько конфигураций. Фронтенд использует `autoPort: true`, поэтому он выбирает свободный порт, если 3000 занят, тогда как API-серверу требуется именно порт 8080:

```json
{
  "version": "0.0.1",
  "configurations": [
    {
      "name": "frontend",
      "runtimeExecutable": "npm",
      "runtimeArgs": ["run", "dev"],
      "cwd": "apps/web",
      "port": 3000,
      "autoPort": true
    },
    {
      "name": "api",
      "runtimeExecutable": "npm",
      "runtimeArgs": ["run", "start"],
      "cwd": "server",
      "port": 8080,
      "env": { "NODE_ENV": "development" },
      "autoPort": false
    }
  ]
}
```

##### Скрипт Node.js

Чтобы запустить скрипт Node.js напрямую, а не через команду менеджера пакетов, используйте поле `program`:

```json
{
  "version": "0.0.1",
  "configurations": [
    {
      "name": "server",
      "program": "server.js",
      "args": ["--verbose"],
      "port": 4000
    }
  ]
}
```

## Конфигурация окружения

Окружение, которое вы выбираете при [запуске сессии](#start-a-session), определяет, где выполняется Claude и как вы подключаетесь:

* **Local**: выполняется на вашей машине с прямым доступом к вашим файлам
* **Remote**: выполняется на облачной инфраструктуре Anthropic. Сессии продолжаются, даже если вы закроете приложение.
* **SSH**: выполняется на удалённой машине, к которой вы подключаетесь по SSH, например на ваших собственных серверах, облачных ВМ или dev-контейнерах

### Локальные сессии

Настольное приложение не всегда наследует ваше полное окружение оболочки. На macOS, когда вы запускаете приложение из Dock или Finder, оно читает профиль вашей оболочки, такой как `~/.zshrc` или `~/.bashrc`, чтобы извлечь `PATH` и фиксированный набор переменных Claude Code, но другие переменные, которые вы там экспортируете, не подхватываются. На Windows приложение наследует пользовательские и системные переменные окружения, но не читает профили PowerShell.

Чтобы задать переменные окружения для локальных сессий и dev-серверов на любой платформе, откройте выпадающее меню окружения в поле промпта, наведите курсор на **Local** и нажмите значок шестерёнки, чтобы открыть редактор локального окружения. Переменные, которые вы сохраняете здесь, хранятся зашифрованными на вашей машине и применяются к каждой локальной сессии и серверу предпросмотра, который вы запускаете. Вы также можете добавить переменные в ключ `env` в вашем файле `~/.claude/settings.json`, хотя они достигают только сессий Claude, но не dev-серверов. Полный список поддерживаемых переменных см. в [переменных окружения](https://code.claude.com/docs/en/env-vars).

[Расширенное мышление](https://code.claude.com/docs/en/model-config#extended-thinking) включено по умолчанию, что улучшает производительность на сложных задачах рассуждения, но использует дополнительные токены. Чтобы отключить мышление, задайте `MAX_THINKING_TOKENS` равным `0` в редакторе локального окружения; это не влияет на Fable 5, который всегда использует расширенное мышление. На [сторонних провайдерах](https://code.claude.com/docs/en/third-party-integrations) `0` вместо этого опускает параметр `thinking`, и модели с адаптивным рассуждением всё равно могут мыслить. На моделях с [адаптивным рассуждением](https://code.claude.com/docs/en/model-config#adjust-effort-level) любое другое значение `MAX_THINKING_TOKENS` игнорируется, потому что адаптивное рассуждение вместо этого контролирует глубину мышления. На Opus 4.6 и Sonnet 4.6 задайте `CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING` равным `1`, чтобы использовать фиксированный бюджет мышления; Fable 5, Sonnet 5 и Opus 4.7 и новее всегда используют адаптивное рассуждение и не имеют режима фиксированного бюджета.

### Облачные сессии

Облачные сессии продолжаются в фоне, даже если вы закроете приложение. Использование засчитывается в [лимиты вашего плана подписки](https://code.claude.com/docs/en/costs) без отдельной платы за вычисления.

Вы можете создавать пользовательские облачные окружения с разными уровнями сетевого доступа и переменными окружения. Выберите выпадающее меню окружения при запуске облачной сессии и выберите **Add environment**. Подробнее о настройке сетевого доступа и переменных окружения см. [облачное окружение](https://code.claude.com/docs/en/claude-code-on-the-web#the-cloud-environment).

### SSH-сессии

SSH-сессии позволяют запускать Claude Code на удалённой машине, используя настольное приложение как интерфейс. Это удобно для работы с кодовыми базами, которые находятся на облачных ВМ, в dev-контейнерах или на серверах с особым оборудованием или зависимостями.

Чтобы добавить SSH-подключение, нажмите выпадающее меню окружения перед запуском сессии и выберите **+ Add SSH connection**. Диалог запрашивает:

* **Name**: понятная метка для этого подключения
* **SSH Host**: `user@hostname` или хост, определённый в `~/.ssh/config`
* **SSH Port**: по умолчанию 22, если оставить пустым, либо используется порт из вашей конфигурации SSH
* **Identity File**: путь к вашему приватному ключу, например `~/.ssh/id_rsa`. Оставьте пустым, чтобы использовать ключ по умолчанию или вашу конфигурацию SSH.

После добавления подключение появляется в выпадающем меню окружения. Выберите его, чтобы запустить сессию на этой машине. Claude выполняется на удалённой машине с доступом к её файлам и инструментам.

Удалённая машина должна работать под управлением Linux или macOS. Desktop автоматически устанавливает Claude Code на удалённую машину при первом подключении. После подключения SSH-сессии поддерживают режимы разрешений, коннекторы, плагины и MCP-серверы.

#### Предварительная настройка SSH-подключений для вашей команды

Администраторы могут распространять SSH-подключения среди членов команды, добавляя `sshConfigs` в файл [управляемых настроек](https://code.claude.com/docs/en/settings#settings-precedence). Определённые таким образом подключения автоматически появляются в выпадающем меню окружения каждого пользователя и отображаются как управляемые, поэтому пользователи могут их выбирать, но не могут редактировать или удалять в приложении.

Следующий пример предварительно настраивает одно подключение, которое открывается в `~/projects` на удалённом хосте:

```json
{
  "sshConfigs": [
    {
      "id": "shared-dev-vm",
      "name": "Shared Dev VM",
      "sshHost": "user@dev.example.com",
      "sshPort": 22,
      "sshIdentityFile": "~/.ssh/id_ed25519",
      "startDirectory": "~/projects"
    }
  ]
}
```

Каждая запись требует `id`, `name` и `sshHost`. Поля `sshPort`, `sshIdentityFile` и `startDirectory` необязательны. Пользователи также могут добавлять `sshConfigs` в свой собственный `~/.claude/settings.json` — именно там хранятся подключения, добавленные через диалог.

#### Ограничение хостов SSH, к которым могут подключаться пользователи

Администраторы могут ограничить SSH-сессии Desktop одобренным набором хостов, добавив `sshHostAllowlist` в файл [управляемых настроек](https://code.claude.com/docs/en/settings#settings-precedence). Когда это задано, пользователи могут подключаться только к хостам, чьё разрешённое имя хоста соответствует одному из шаблонов. Задайте пустой массив, чтобы полностью отключить SSH-сессии.

Следующий пример разрешает подключения к любому хосту под `devboxes.example.com` и к одному именованному бастион-хосту:

```json
{
  "sshHostAllowlist": ["*.devboxes.example.com", "bastion.example.com"]
}
```

Шаблоны нечувствительны к регистру. `*` соответствует любому хосту, а `*.example.com` соответствует `example.com` и любому поддомену. Всё остальное — точное совпадение. Проверка выполняется против имени хоста после разрешения `~/.ssh/config` через `ssh -G`, поэтому псевдонимы `Host` и записи `ProxyCommand`/`ProxyJump` разрешены, пока разрешённое `HostName` совпадает.

`sshHostAllowlist` читается только из управляемых настроек; значения в пользовательских или проектных настройках игнорируются. Только настольное приложение Claude Desktop учитывает этот параметр; Claude Code CLI и расширения для IDE его не читают, и он не ограничивает команды `ssh`, запущенные через инструмент Bash. Он управляет тем, к каким хостам подключается приложение Desktop, а не сетевым исходящим трафиком, поэтому сочетайте его с сетевыми или zero-trust-контролями вашей организации, если вам нужна жёсткая граница.

## Корпоративная конфигурация

Организации на планах Team или Enterprise могут управлять поведением настольного приложения через элементы управления консоли администратора, файлы управляемых настроек и политики управления устройствами.

### Элементы управления консоли администратора

Эти настройки конфигурируются через [консоль настроек администратора](https://claude.ai/admin-settings/claude-code):

* **Code in the desktop**: контролировать, могут ли пользователи в вашей организации получать доступ к Claude Code в настольном приложении
* **Code in the web**: включить или отключить [веб-сессии](https://code.claude.com/docs/en/claude-code-on-the-web) для вашей организации
* **Remote Control**: включить или отключить [Remote Control](https://code.claude.com/docs/en/remote-control) для вашей организации
* **Disable Bypass permissions mode**: не позволять пользователям в вашей организации включать режим bypass permissions

### Управляемые настройки

Управляемые настройки переопределяют проектные и пользовательские настройки и применяются к сессиям Claude Code в Desktop. Вы можете задать эти ключи в файле [управляемых настроек](https://code.claude.com/docs/en/settings#settings-precedence) вашей организации или отправить их удалённо через консоль администратора.

| Ключ | Описание |
| --- | --- |
| `permissions.disableBypassPermissionsMode` | задайте `"disable"`, чтобы не позволять пользователям включать режим Bypass permissions. |
| `disableAutoMode` | задайте `"disable"`, чтобы не позволять пользователям включать режим [Auto](https://code.claude.com/docs/en/permission-modes#eliminate-prompts-with-auto-mode). Убирает Auto из селектора режимов. Также принимается внутри `permissions`. |
| `autoMode` | настройте, чему доверяет и что блокирует классификатор режима auto по всей вашей организации. См. [Настройка режима auto](https://code.claude.com/docs/en/auto-mode-config). |
| `sshConfigs` | предварительно настройте [SSH-подключения](#pre-configure-ssh-connections-for-your-team), которые появляются в выпадающем меню окружения. Пользователи не могут редактировать или удалять управляемые подключения. |
| `sshHostAllowlist` | ограничьте [SSH-сессии](#restrict-which-ssh-hosts-users-can-connect-to) хостами, чьё разрешённое имя хоста соответствует одному из этих шаблонов. Пустой массив отключает SSH-сессии. Читается только из управляемых настроек. |
| `managedMcpServers` | отправьте конфигурации MCP-серверов всем пользователям в стороннем развёртывании. Каждая запись задаёт транспорт `"http"`, `"sse"` или `"stdio"`, детали подключения и, при необходимости, карту `toolPolicy`, которая ограничивает, какие инструменты этого сервера пользователи могут вызывать. Доступно только в сторонних (3P) развёртываниях Desktop. Доставляйте этот ключ через файл управляемых настроек или MDM, так как сторонние развёртывания не получают настройки из консоли администратора. |

То, какие управляемые настройки достигают сессии Desktop, зависит от того, где эта сессия выполняется. Ограничения моделей, такие как [`availableModels`](https://code.claude.com/docs/en/model-config#restrict-model-selection), применяются в сессиях Claude Code в Desktop так же, как и в терминальном CLI; см. [охват поверхностей](https://code.claude.com/docs/en/model-config#surface-coverage).

* **Локальные сессии на этой машине**: применяется файл управляемых настроек, развёрнутый на диск. Управляемые настройки, отправленные удалённо через консоль администратора, также достигают этих сессий на Anthropic API, когда сессия аутентифицируется через вход в организацию или напрямую настроенный API-ключ, следуя тому же [приоритету настроек](https://code.claude.com/docs/en/settings#settings-precedence), что и терминальный CLI.
* **[Облачные сессии](#cloud-sessions)**: выполняются на управляемых Anthropic ВМ и получают только [настройки, управляемые сервером](https://code.claude.com/docs/en/server-managed-settings).
* **[SSH-сессии](#ssh-sessions)**: сессия читает файл управляемых настроек с удалённого хоста. Сам Desktop читает `sshConfigs` и `sshHostAllowlist` из управляемых настроек локальной машины при создании подключения.

`permissions.disableBypassPermissionsMode` и `disableAutoMode` также работают в пользовательских и проектных настройках, но размещение их в управляемых настройках не позволяет пользователям их переопределять. `autoMode` читается из пользовательских настроек, `.claude/settings.local.json` и управляемых настроек, но не из закоммиченного `.claude/settings.json`: клонированный репозиторий не может внедрить свои собственные правила классификатора. Полный список настроек только для управляемого режима, включая `allowManagedPermissionRulesOnly` и `allowManagedHooksOnly`, см. в [настройках только для управляемого режима](https://code.claude.com/docs/en/permissions#managed-only-settings).

### Политики управления устройствами

ИТ-команды могут управлять настольным приложением через MDM на macOS или групповую политику на Windows. Доступные политики включают включение или отключение функции Claude Code, управление автообновлениями и установку пользовательского URL развёртывания.

* **macOS**: настраивается через домен предпочтений `com.anthropic.claudefordesktop` с помощью инструментов вроде Jamf или Kandji
* **Windows**: настраивается через реестр по пути `SOFTWARE\Policies\Claude`

### Аутентификация и SSO

Корпоративные организации могут требовать SSO для всех пользователей. Подробности на уровне планов см. в [аутентификации](https://code.claude.com/docs/en/authentication), а конфигурацию SAML и OIDC см. в [настройке SSO](https://support.claude.com/en/articles/13132885-setting-up-single-sign-on-sso).

### Обработка данных

Claude Code обрабатывает ваш код локально в локальных сессиях или на облачной инфраструктуре Anthropic в облачных сессиях. Разговоры и контекст кода отправляются в Anthropic API для обработки. Подробности о хранении данных, конфиденциальности и соответствии требованиям см. в [обработке данных](https://code.claude.com/docs/en/data-usage).

### Развёртывание

Desktop можно распространять через корпоративные инструменты развёртывания:

* **macOS**: распространяйте через MDM, такой как Jamf или Kandji, с помощью установщика `.dmg`
* **Windows**: развёртывайте через пакет MSIX или установщик `.exe`. Варианты корпоративного развёртывания, включая тихую установку, см. в [Развёртывание Claude Desktop для Windows](https://support.claude.com/en/articles/12622703-deploy-claude-desktop-for-windows)

Сетевую конфигурацию, такую как настройки прокси, добавление в списки разрешений брандмауэра и LLM-шлюзы, см. в [сетевой конфигурации](https://code.claude.com/docs/en/network-config).

Полный справочник по корпоративной конфигурации см. в [руководстве по корпоративной конфигурации](https://support.claude.com/en/articles/12622667-enterprise-configuration).

## Переходите с CLI?

Если вы уже используете Claude Code CLI, Desktop запускает тот же базовый движок с графическим интерфейсом. Вы можете запускать оба одновременно на одной машине, даже на одном проекте. Каждый ведёт отдельную историю сессий, но они делят конфигурацию и память проекта через файлы CLAUDE.md.

Чтобы перенести сессию CLI в Desktop, запустите `/desktop` в терминале. Claude сохраняет вашу сессию и открывает её в настольном приложении, затем закрывает CLI. Эта команда доступна на macOS и Windows, когда вы вошли с подпиской Claude. Она недоступна при аутентификации через API-ключ или на Bedrock, Vertex или Foundry.

> **Совет.** Когда использовать Desktop, а когда CLI: используйте Desktop, когда хотите управлять параллельными сессиями в одном окне, располагать панели рядом или просматривать изменения визуально. Используйте CLI, когда вам нужны скриптинг, автоматизация или вы предпочитаете терминальный рабочий процесс.

### Эквиваленты флагов CLI

Эта таблица показывает эквивалент в настольном приложении для распространённых флагов CLI. Флаги, не перечисленные здесь, не имеют настольного эквивалента, потому что предназначены для скриптинга или автоматизации.

| CLI | Эквивалент в Desktop |
| --- | --- |
| `--model sonnet` | Выпадающий список модели рядом с кнопкой отправки |
| `--resume`, `--continue` | Нажатие на сессию в боковой панели |
| `--permission-mode` | Селектор режима рядом с кнопкой отправки |
| `--dangerously-skip-permissions` | Режим Bypass permissions. Включите в Settings → Claude Code → «Allow bypass permissions mode». Корпоративные администраторы могут отключить эту настройку. |
| `--add-dir` | Добавление нескольких репозиториев кнопкой **+** в облачных сессиях |
| `--allowedTools`, `--disallowedTools` | Нет эквивалента на уровне сессии. Правила разрешений в [файлах настроек](https://code.claude.com/docs/en/settings) всё равно применяются. |
| `--verbose` | [Режим отображения Verbose](#switch-view-modes) в выпадающем списке Transcript view |
| `--print`, `--output-format` | Недоступно. Desktop только интерактивный. |
| Переменная окружения `ANTHROPIC_MODEL` | Выпадающий список модели рядом с кнопкой отправки |
| Переменная окружения `MAX_THINKING_TOKENS` | Задаётся в редакторе локального окружения. См. [конфигурацию окружения](#environment-configuration). |

### Общая конфигурация

Desktop и CLI читают одни и те же файлы конфигурации, поэтому ваша настройка переносится:

* Файлы **[CLAUDE.md](https://code.claude.com/docs/en/memory)** и `CLAUDE.local.md` в вашем проекте используются обоими
* **[MCP-серверы](https://code.claude.com/docs/en/mcp)**, настроенные в `~/.claude.json` или `.mcp.json`, работают в обоих
* **[Хуки](https://code.claude.com/docs/en/hooks)** и **[навыки](https://code.claude.com/docs/en/skills)**, определённые в настройках, применяются к обоим
* **[Настройки](https://code.claude.com/docs/en/settings)** в `~/.claude.json` и `~/.claude/settings.json` являются общими. Правила разрешений, разрешённые инструменты и другие настройки в `settings.json` применяются к сессиям Desktop.
* **Модели**: одни и те же [модели](https://code.claude.com/docs/en/model-config#available-models) доступны в обоих. В Desktop выберите модель из выпадающего списка рядом с кнопкой отправки. Вы можете сменить модель в середине сессии из того же списка.

> **Примечание.** **MCP-серверы из чат-приложения Claude Desktop**: приложение Desktop загружает MCP-серверы из `claude_desktop_config.json` в сессии вкладки Code, наряду с серверами из `~/.claude.json` и `.mcp.json`. Сервер, определённый в `claude_desktop_config.json`, доступен как на поверхности чата Desktop, так и на вкладке Code.
>
> Отдельный CLI не читает `claude_desktop_config.json`. На macOS и WSL запустите `claude mcp add-from-claude-desktop`, чтобы скопировать эти серверы в `~/.claude.json`. Процесс импорта и параметры области видимости см. в [Импорт MCP-серверов из Claude Desktop](https://code.claude.com/docs/en/mcp#import-mcp-servers-from-claude-desktop).

### Что недоступно в Desktop

Следующие функции доступны только в CLI или расширении VS Code, за исключением отмеченных случаев:

* **Сторонние провайдеры**: Desktop по умолчанию подключается к Anthropic API. Корпоративные развёртывания могут настроить Vertex AI и провайдеров-шлюзы через [управляемые настройки](https://support.claude.com/en/articles/12622667-enterprise-configuration). Для Bedrock или Foundry в CLI см. [быстрый старт](https://code.claude.com/docs/en/quickstart). В качестве исключения из раздела выше [исследовательский предпросмотр Cowork on 3P](https://claude.com/docs/cowork/3p/overview) запускает вкладку Code на Bedrock, Vertex AI, Foundry или самостоятельно размещённом LLM-шлюзе.
* **Linux (бета)**: управление компьютером пока недоступно в настольном приложении для Linux. См. [Claude Desktop в Linux](https://code.claude.com/docs/en/desktop-linux).
* **Встроенные подсказки кода**: Desktop не предоставляет подсказки в стиле автодополнения. Он работает через разговорные промпты и явные изменения кода.
* **Команды агентов (Agent teams)**: параллельные сессии Claude Code, которые обмениваются сообщениями друг с другом, доступны в [CLI](https://code.claude.com/docs/en/agent-teams), но не в Desktop. Для многоагентной работы внутри одной сессии используйте [динамические рабочие процессы](https://code.claude.com/docs/en/workflows), которые выполняются в Desktop.
* **Команды с терминальным диалогом**: встроенные команды, которые открывают интерактивную панель в терминале, такие как `/permissions`, `/config`, `/agents` и `/doctor`, недоступны на вкладке Code и отвечают `isn't available in this environment`. Редактируйте [файлы настроек](https://code.claude.com/docs/en/settings) напрямую, чтобы управлять правилами разрешений и конфигурацией, или запускайте команду из отдельного CLI.

## Устранение неполадок

Разделы ниже охватывают проблемы, специфичные для настольного приложения. Об ошибках выполнения API, которые появляются в чате, таких как `API Error: 500`, `529 Overloaded`, `429` или `Prompt is too long`, см. в [справочнике по ошибкам](https://code.claude.com/docs/en/errors). Эти ошибки и их исправления одинаковы для CLI, desktop и веба.

### Проверьте вашу версию

Чтобы узнать, какую версию настольного приложения вы используете:

* **macOS**: нажмите **Claude** в строке меню, затем **About Claude**
* **Windows**: нажмите **Help**, затем **About**

Нажмите на номер версии, чтобы скопировать его в буфер обмена.

### Ошибки 403 или аутентификации на вкладке Code

Если вы видите `Error 403: Forbidden` или другие сбои аутентификации при использовании вкладки Code:

1. Выйдите и снова войдите из меню приложения. Это самое распространённое исправление.
2. Убедитесь, что у вас есть активная платная подписка: Pro, Max, Team или Enterprise.
3. Если CLI работает, а Desktop нет, полностью закройте настольное приложение, а не просто окно, затем снова откройте и войдите заново.
4. Проверьте ваше интернет-соединение и настройки прокси.

### Пустой или зависший экран при запуске

Если приложение открывается, но показывает пустой или не отвечающий экран:

1. Перезапустите приложение.
2. Проверьте наличие ожидающих обновлений. На macOS и Windows приложение автоматически обновляется при запуске; на Linux обновляйтесь через apt, как описано в [Claude Desktop в Linux](https://code.claude.com/docs/en/desktop-linux).
3. На Windows проверьте Event Viewer на наличие логов сбоев в разделе **Windows Logs → Application**.

### «Failed to load session»

Если вы видите `Failed to load session`, выбранная папка может больше не существовать, репозиторию Git может требоваться Git LFS, который не установлен, или права доступа к файлам могут препятствовать доступу. Попробуйте выбрать другую папку или перезапустить приложение.

### Сессия не находит установленные инструменты

Если Claude не может найти инструменты вроде `npm`, `node` или другие команды CLI, убедитесь, что инструменты работают в вашем обычном терминале, проверьте, что профиль вашей оболочки правильно настраивает PATH, и перезапустите настольное приложение, чтобы перезагрузить переменные окружения.

### Ошибки Git и Git LFS

На Windows Git необходим для запуска локальных сессий на вкладке Code. Если вы видите «Git is required», установите [Git for Windows](https://git-scm.com/downloads/win) и перезапустите приложение.

Если вы видите «Git LFS is required by this repository but is not installed», установите Git LFS с [git-lfs.com](https://git-lfs.com/), запустите `git lfs install` и перезапустите приложение.

### MCP-серверы не работают на Windows

Если переключатели MCP-серверов не реагируют или серверы не подключаются на Windows, проверьте, что сервер правильно настроен в ваших настройках, перезапустите приложение, убедитесь, что процесс сервера запущен в Task Manager, и просмотрите логи сервера на наличие ошибок подключения.

### Приложение не закрывается

* **macOS**: нажмите Cmd+Q. Если приложение не отвечает, используйте Force Quit с Cmd+Option+Esc, выберите Claude и нажмите Force Quit.
* **Windows**: используйте Task Manager с Ctrl+Shift+Esc, чтобы завершить процесс Claude.

### Проблемы, специфичные для Windows

* **PATH не обновляется после установки**: откройте новое окно терминала. Обновления PATH применяются только к новым сессиям терминала.
* **Ошибка одновременной установки**: если вы видите ошибку о том, что выполняется другая установка, но её нет, попробуйте запустить установщик от имени администратора.

### «Branch doesn't exist yet» при открытии в CLI

Облачные сессии могут создавать ветки, которых нет на вашей локальной машине. Нажмите на имя ветки на панели инструментов сессии, чтобы скопировать его, затем получите её локально:

```bash
git fetch origin <branch-name>
git checkout <branch-name>
```

### Всё ещё не получается?

* Откройте Help → Get Support в настольном приложении или посетите [центр поддержки Claude](https://support.claude.com/) напрямую
* О проблемах, которые также воспроизводятся в отдельном CLI `claude`, ищите или заводите баг на [GitHub Issues](https://github.com/anthropics/claude-code/issues)

При сообщении о проблеме укажите версию вашего настольного приложения, вашу операционную систему, точное сообщение об ошибке и соответствующие логи. На macOS проверьте Console.app. На Windows проверьте Event Viewer → Windows Logs → Application.

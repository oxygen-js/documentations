# Запланированные задачи в настольном приложении

> Настройте запланированные задачи в Claude Code Desktop, чтобы Claude запускался автоматически по расписанию для ежедневных ревью кода, аудита зависимостей или утренних сводок.

Запланированные задачи автоматически запускают новую сессию в выбранное вами время и с выбранной периодичностью. Используйте их для повторяющейся работы: ежедневных ревью кода, проверок обновлений зависимостей или утренних сводок, которые собирают данные из вашего календаря и почты.

Страница **Routines** в настольном приложении позволяет создавать как локальные запланированные задачи, так и удалённые [рутины](https://code.claude.com/docs/en/routines). Локальная задача выполняется на вашей машине с прямым доступом к вашим файлам и инструментам, но срабатывает только пока приложение открыто, а компьютер не спит. Удалённая рутина выполняется на облачной инфраструктуре, управляемой Anthropic, даже когда ваш компьютер выключен, и может также срабатывать по вызовам API или событиям GitHub. Эта страница посвящена локальным запланированным задачам; про удалённые рутины и варианты их триггеров см. [Рутины](https://code.claude.com/docs/en/routines).

## Сравнение вариантов планирования

Claude Code предлагает три способа планировать повторяющуюся или разовую работу:

|                                    | [Облако](https://code.claude.com/docs/en/routines) | [Десктоп](https://code.claude.com/docs/en/desktop-scheduled-tasks) | [`/loop`](https://code.claude.com/docs/en/scheduled-tasks) |
| :--------------------------------- | :----------------------------- | :------------------------------------- | :---------------------------------- |
| Выполняется на                     | Облаке Anthropic               | Вашей машине                           | Вашей машине                        |
| Требует включённой машины          | Нет                            | Да                                     | Да                                  |
| Требует открытой сессии            | Нет                            | Нет                                    | Да                                  |
| Сохраняется между перезапусками    | Да                             | Да                                     | Восстанавливается по `--resume`, если не истекла |
| Доступ к локальным файлам          | Нет (свежий клон)              | Да                                     | Да                                  |
| MCP-серверы                        | Коннекторы настраиваются для каждой задачи | [Конфигурационные файлы](https://code.claude.com/docs/en/mcp) и коннекторы | Наследуются из сессии |
| Запросы разрешений                 | Нет (выполняется автономно)    | Настраиваются для каждой задачи        | Наследуются из сессии               |
| Настраиваемое расписание           | Через `/schedule` в CLI        | Да                                     | Да                                  |
| Минимальный интервал               | 1 час                          | 1 минута                               | 1 минута                            |

> **Совет.** Используйте **облачные задачи** для работы, которая должна надёжно выполняться без вашей машины. Используйте **задачи Desktop**, когда вам нужен доступ к локальным файлам и инструментам. Используйте **`/loop`** для быстрого опроса во время сессии.

> **Примечание.** По умолчанию запланированные задачи выполняются на основе того состояния, в котором находится ваш рабочий каталог, включая незакоммиченные изменения. Включите переключатель worktree при создании задачи, чтобы каждый запуск получал собственный изолированный Git worktree — так же, как работают [параллельные сессии](https://code.claude.com/docs/en/desktop#work-in-parallel-with-sessions).

## Создание запланированной задачи

Нажмите **Routines** в боковой панели, затем нажмите **New routine** и выберите **Local**. Настройте следующие поля:

| Поле         | Описание                                                                                                                                                                                                                                                                       |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Name         | Идентификатор задачи. Преобразуется в нижний регистр в стиле kebab-case и используется как имя папки на диске. Должен быть уникальным среди ваших задач.                                                                                                                        |
| Description  | Краткое описание, отображаемое в списке задач.                                                                                                                                                                                                                                 |
| Instructions | Что Claude должен делать при запуске задачи. Пишите так же, как вы бы писали любое сообщение в поле промпта. Поле инструкций включает селекторы режима разрешений и модели, а под ним вы выбираете рабочую папку и то, запускать ли задачу в изолированном worktree.            |
| Schedule     | Как часто выполняется задача. См. [варианты расписания](#варианты-расписания) ниже.                                                                                                                                                                                             |

Прежде чем сохранить задачу, необходимо указать папку. Если вы ещё не доверяете этой папке, Desktop предложит вам подтвердить доверие к ней перед сохранением.

Вы также можете создать задачу, описав желаемое в любой сессии. Например, «set up a daily code review that runs every morning at 9am» создаёт повторяющуюся задачу, а «remind me at 3pm tomorrow to check the deploy» создаёт разовую задачу, которая отключается после срабатывания.

## Варианты расписания

Выберите пресет в элементе управления Schedule:

* **Manual**: без расписания, выполняется только когда вы нажимаете **Run now**. Полезно для сохранения промпта, который вы запускаете по требованию
* **Hourly**: выполняется каждый час
* **Daily**: показывает выбор времени, по умолчанию 9:00 по местному времени
* **Weekdays**: то же, что Daily, но пропускает субботу и воскресенье
* **Weekly**: показывает выбор времени и выбор дня

Для интервалов, которые не предлагает селектор, — например, каждые 15 минут, первое число каждого месяца или единичный запуск в определённое время в будущем — попросите Claude в любой сессии Desktop настроить расписание. Используйте обычный язык; например, «schedule a task to run all the tests every 6 hours».

## Как выполняются запланированные задачи

Запланированные задачи выполняются на вашей машине. Desktop проверяет расписание каждую минуту, пока приложение открыто, и запускает новую сессию, когда наступает время задачи, независимо от любых открытых у вас вручную сессий. Каждая задача получает небольшую задержку в несколько минут после запланированного времени, чтобы распределить нагрузку на API. Задержка детерминирована: одна и та же задача всегда стартует с одним и тем же смещением.

Когда задача срабатывает, вы получаете уведомление на рабочем столе, и в боковой панели под разделом **Scheduled** появляется новая сессия. Откройте её, чтобы посмотреть, что сделал Claude, проверить изменения или ответить на запросы разрешений. Сессия работает как любая другая: Claude может редактировать файлы, выполнять команды, создавать коммиты и открывать pull request.

Задачи выполняются только пока настольное приложение запущено, а компьютер не спит. Если компьютер проспит запланированное время, запуск пропускается. Чтобы предотвратить переход в спящий режим при простое, включите **Keep computer awake** в Settings в разделе **Desktop app → General**. Закрытие крышки ноутбука всё равно переводит его в спящий режим. Для задач, которые должны выполняться даже когда компьютер выключен, или которые должны срабатывать по вызову API или событию GitHub, вместо этого создайте удалённую [рутину](https://code.claude.com/docs/en/routines).

## Пропущенные запуски

Когда приложение запускается или компьютер выходит из спящего режима, Desktop проверяет, не пропустила ли каждая задача какие-либо запуски за последние семь дней. Если пропустила, Desktop запускает ровно один навёрстывающий запуск для самого недавнего пропущенного времени и отбрасывает всё более старое. Ежедневная задача, пропустившая шесть дней, при пробуждении выполнится один раз. Desktop показывает уведомление, когда начинается навёрстывающий запуск.

Учитывайте это при написании промптов. Задача, запланированная на 9 утра, может выполниться в 11 вечера, если ваш компьютер спал весь день. Если время имеет значение, добавьте ограничения в сам промпт, например: «Only review today's commits. If it's after 5pm, skip the review and just post a summary of what was missed.»

## Разрешения для запланированных задач

У каждой задачи есть собственный режим разрешений, который вы задаёте при создании или редактировании задачи. Разрешающие правила из `~/.claude/settings.json` также применяются к сессиям запланированных задач. Если задача выполняется в режиме Ask и ей нужно запустить инструмент, на который у неё нет разрешения, запуск приостанавливается, пока вы его не одобрите. Сессия остаётся открытой в боковой панели, чтобы вы могли ответить позже.

Чтобы избежать зависаний, нажмите **Run now** после создания задачи, следите за запросами разрешений и выбирайте «always allow» для каждого из них. Будущие запуски этой задачи автоматически одобрят те же инструменты без запросов. Вы можете просмотреть и отозвать эти одобрения на странице деталей задачи.

## Управление запланированными задачами

Нажмите на задачу в списке **Routines**, чтобы открыть её страницу деталей. Отсюда вы можете:

* **Run now**: запустить задачу немедленно, не дожидаясь следующего запланированного времени
* **Status**: переключаться между Active и Paused, чтобы приостановить или возобновить запланированные запуски без удаления задачи
* **Edit**: изменить инструкции, расписание, папку или другие настройки
* **Review history**: посмотреть каждый прошлый запуск, включая пропущенные. Наведите курсор на пропущенную запись, чтобы узнать причину: ваш компьютер спал, предыдущий запуск ещё выполнялся или уже выполнялись другие запланированные задачи. Нажмите **Show more**, чтобы загрузить более старые записи.
* **Review allowed permissions**: посмотреть и отозвать сохранённые одобрения инструментов для этой задачи в панели **Always allowed**
* **Delete**: удалить задачу и архивировать все созданные ею сессии. В диалоге подтверждения появляется флажок **Also delete files on disk**; отметьте его, чтобы также удалить файл `SKILL.md` задачи и связанные данные из `~/.claude/scheduled-tasks/`.

Вы также можете просматривать, создавать, редактировать и приостанавливать задачи, попросив Claude в любой сессии Desktop. Например, «pause my dependency-audit task» или «show me my scheduled tasks». Чтобы удалить задачу, используйте кнопку **Delete** на её странице деталей.

Запланированная задача также может изменять собственное расписание или промпт изнутри выполняющейся сессии с помощью MCP-инструмента `update_scheduled_task`. Это позволяет задаче перепланировать саму себя в зависимости от того, что она обнаружит, — например, перепланировать ревью кода на более раннее время, когда она обнаруживает, что создана ветка релиза.

Чтобы отредактировать промпт задачи на диске, откройте `~/.claude/scheduled-tasks/<task-name>/SKILL.md` (или в каталоге [`CLAUDE_CONFIG_DIR`](https://code.claude.com/docs/en/env-vars), если он задан). Файл использует YAML-фронтматтер для `name` и `description`, а промпт находится в теле. Изменения вступают в силу при следующем запуске. Расписание, папка, модель и состояние включённости не находятся в этом файле: изменяйте их через форму Edit или попросите Claude.

## Связанные ресурсы

* [Рутины](https://code.claude.com/docs/en/routines): выполнение задач на инфраструктуре, управляемой Anthropic, по расписанию, по вызову API или в ответ на события GitHub, даже когда ваш компьютер выключен
* [Запуск промптов по расписанию](https://code.claude.com/docs/en/scheduled-tasks): планирование в рамках сессии с помощью `/loop` в CLI
* [Claude Code GitHub Actions](https://code.claude.com/docs/en/github-actions): запуск Claude по расписанию в CI вместо вашей машины
* [Использование Claude Code Desktop](https://code.claude.com/docs/en/desktop): полное руководство по настольному приложению

# Быстрый старт для настольного приложения

> Установите Claude Code на десктоп и начните свою первую сессию работы с кодом

Настольное приложение даёт вам Claude Code с графическим интерфейсом, созданным для запуска нескольких сессий бок о бок: боковая панель для управления параллельной работой, макет с перетаскиванием, встроенным терминалом и редактором файлов, визуальный просмотр различий (diff), предпросмотр приложения в реальном времени, мониторинг PR на GitHub с автоматическим слиянием и запланированные задачи. Терминал не требуется.

* [Скачать для macOS](https://claude.ai/api/desktop/darwin/universal/dmg/latest/redirect?utm_source=claude_code&utm_medium=docs) — универсальная сборка для Intel и Apple Silicon
* [Скачать для Windows](https://claude.ai/api/desktop/win32/x64/setup/latest/redirect?utm_source=claude_code&utm_medium=docs) — для процессоров x64
* [Получить Claude для Linux (бета)](https://code.claude.com/docs/en/desktop-linux) — apt или .deb для Ubuntu и Debian

Для Windows ARM64 скачайте [установщик ARM64](https://claude.ai/api/desktop/win32/arm64/setup/latest/redirect?utm_source=claude_code&utm_medium=docs). В Linux устанавливайте через apt; см. [Claude Desktop в Linux](https://code.claude.com/docs/en/desktop-linux).

> **Примечание.** Для Claude Code требуется [подписка Pro, Max, Team или Enterprise](https://claude.com/pricing?utm_source=claude_code&utm_medium=docs&utm_content=desktop_quickstart_pricing).

Эта страница проведёт вас через установку приложения и запуск первой сессии. Если вы уже всё настроили, см. [Использование Claude Code Desktop](https://code.claude.com/docs/en/desktop) для полного справочника.

В настольном приложении три вкладки:

* **Chat**: обычный разговор без доступа к файлам, подобно claude.ai.
* **Cowork**: автономный фоновый агент, который работает над задачами в облачной виртуальной машине с собственным окружением. Он может выполняться независимо, пока вы занимаетесь другой работой.
* **Code**: интерактивный помощник по коду с прямым доступом к вашим локальным файлам. Вы просматриваете и утверждаете каждое изменение в реальном времени.

Chat и Cowork описаны в [статьях поддержки Claude Desktop](https://support.claude.com/en/collections/16163169-claude-desktop). Эта страница сфокусирована на вкладке **Code**.

## Установка

1. **Установка и вход в систему**

   В macOS и Windows скачайте установщик по ссылкам выше и запустите его. В Linux следуйте шагам установки из [Claude Desktop в Linux](https://code.claude.com/docs/en/desktop-linux). Запустите Claude из папки «Программы» (Applications) в macOS, из меню «Пуск» в Windows или из вашего лаунчера приложений в Linux, затем войдите в систему с помощью вашей учётной записи Anthropic.

2. **Откройте вкладку Code**

   Нажмите на вкладку **Code** вверху по центру. Если при нажатии на Code вам предлагается выполнить обновление, сначала нужно [оформить платный план](https://claude.com/pricing?utm_source=claude_code&utm_medium=docs&utm_content=desktop_quickstart_upgrade). Если предлагается войти онлайн, завершите вход и перезапустите приложение. Если вы видите ошибку 403, см. [устранение неполадок аутентификации](https://code.claude.com/docs/en/desktop#403-or-authentication-errors-in-the-code-tab).

Настольное приложение включает Claude Code. Вам не нужно устанавливать Node.js или CLI отдельно. Чтобы использовать `claude` из терминала, установите CLI отдельно. См. [Быстрый старт с CLI](https://code.claude.com/docs/en/quickstart).

## Запустите свою первую сессию

Открыв вкладку Code, выберите проект и дайте Claude какое-нибудь задание.

1. **Выберите окружение и папку**

   Выберите **Local**, чтобы запустить Claude на вашей машине, используя ваши файлы напрямую. Нажмите **Select folder** и выберите каталог вашего проекта.

   > **Совет.** Начните с небольшого проекта, который вы хорошо знаете. Это самый быстрый способ увидеть, на что способен Claude Code. В Windows для работы локальных сессий должен быть установлен [Git](https://git-scm.com/downloads/win). На большинстве компьютеров Mac Git установлен по умолчанию.

   Вы также можете выбрать:

   * **Remote**: запуск сессий в облачной инфраструктуре Anthropic, которые продолжаются, даже если вы закроете приложение. Облачные сессии используют ту же инфраструктуру, что и [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web).
   * **SSH**: подключение к удалённой машине по SSH, например к вашим собственным серверам, облачным виртуальным машинам или dev-контейнерам. Desktop автоматически устанавливает Claude Code на удалённой машине при первом подключении.

2. **Выберите модель**

   Выберите модель из выпадающего списка рядом с кнопкой отправки. См. [модели](https://code.claude.com/docs/en/model-config#available-models) для сравнения доступных моделей. Позже вы можете изменить модель из того же выпадающего списка.

3. **Скажите Claude, что делать**

   Введите то, что вы хотите, чтобы Claude сделал:

   * `Find a TODO comment and fix it`
   * `Add tests for the main function`
   * `Create a CLAUDE.md with instructions for this codebase`

   [Сессия](https://code.claude.com/docs/en/desktop#work-in-parallel-with-sessions) — это разговор с Claude о вашем коде. Каждая сессия отслеживает свой собственный контекст и изменения, поэтому вы можете работать над несколькими задачами, не создавая помех друг для друга.

4. **Просмотрите и примите изменения**

   По умолчанию вкладка Code запускается в [режиме разрешений Ask](https://code.claude.com/docs/en/desktop#choose-a-permission-mode), в котором Claude предлагает изменения и ждёт вашего одобрения перед их применением. Вы увидите:

   1. [Просмотр различий (diff)](https://code.claude.com/docs/en/desktop#review-changes-with-diff-view), показывающий, что именно изменится в каждом файле
   2. Кнопки Accept/Reject для одобрения или отклонения каждого изменения
   3. Обновления в реальном времени по мере того, как Claude выполняет ваш запрос

   Если вы отклоните изменение, Claude спросит, как вы хотели бы поступить иначе. Ваши файлы не изменяются, пока вы не примете изменения.

## Что теперь?

Вы внесли своё первое изменение. Полный справочник по всему, что умеет Desktop, см. в разделе [Использование Claude Code Desktop](https://code.claude.com/docs/en/desktop). Вот несколько вещей, которые стоит попробовать дальше.

**Прерывайте и направляйте.** Вы можете перенаправить Claude в любой момент. Нажмите кнопку остановки, чтобы прервать немедленно, или введите поправку и нажмите **Enter**, чтобы отправить её, не останавливая выполняемое действие. В любом случае вам не нужно ждать завершения или начинать заново.

**Давайте Claude больше контекста.** Введите `@filename` в поле промпта, чтобы подтянуть конкретный файл в разговор, прикрепите изображения и PDF-файлы с помощью кнопки вложений или перетащите файлы прямо в поле промпта. Чем больше контекста у Claude, тем лучше результаты. См. [Добавление файлов и контекста](https://code.claude.com/docs/en/desktop#add-files-and-context-to-prompts).

**Используйте навыки (skills) для повторяющихся задач.** Введите `/` или нажмите **+** → **Slash commands**, чтобы просмотреть [встроенные команды](https://code.claude.com/docs/en/commands), [пользовательские навыки](https://code.claude.com/docs/en/skills) и навыки плагинов. Навыки — это переиспользуемые промпты, которые вы можете вызывать всякий раз, когда они вам нужны, например чек-листы для ревью кода или шаги развёртывания.

**Просматривайте изменения перед коммитом.** После того как Claude отредактирует файлы, появится индикатор `+12 -1`. Нажмите на него, чтобы открыть [просмотр различий (diff)](https://code.claude.com/docs/en/desktop#review-changes-with-diff-view), просмотреть изменения файл за файлом и оставить комментарии к конкретным строкам. Claude читает ваши комментарии и вносит правки. Нажмите **Review code**, чтобы Claude сам оценил различия и оставил встроенные предложения.

**Настраивайте степень контроля.** Ваш [режим разрешений](https://code.claude.com/docs/en/desktop#choose-a-permission-mode) управляет балансом. Ask permissions (по умолчанию) требует одобрения перед каждым изменением. Auto accept edits автоматически принимает правки файлов для более быстрой итерации. Plan mode позволяет Claude наметить подход, не касаясь никаких файлов, что полезно перед крупным рефакторингом.

**Добавляйте плагины для расширения возможностей.** Нажмите кнопку **+** рядом с полем промпта и выберите **Plugins**, чтобы просмотреть и установить [плагины](https://code.claude.com/docs/en/desktop#install-plugins), которые добавляют навыки, агентов, серверы MCP и многое другое.

**Организуйте своё рабочее пространство.** Перетаскивайте панели чата, различий, терминала, файлов и предпросмотра в любой удобный вам макет. Откройте терминал с помощью **Ctrl+`**, чтобы выполнять команды рядом с вашей сессией, или нажмите на путь к файлу, чтобы открыть его в панели файлов. См. [Организация рабочего пространства](https://code.claude.com/docs/en/desktop#arrange-your-workspace).

**Просматривайте предпросмотр вашего приложения.** Нажмите на выпадающий список **Preview**, чтобы запустить ваш dev-сервер прямо в десктопе. Claude может просматривать работающее приложение, тестировать эндпоинты, инспектировать логи и итеративно улучшать то, что видит. См. [Предпросмотр вашего приложения](https://code.claude.com/docs/en/desktop#preview-your-app).

**Отслеживайте ваш pull request.** После открытия PR Claude Code отслеживает результаты CI-проверок и может автоматически исправлять сбои или сливать PR, как только все проверки пройдут. См. [Мониторинг статуса pull request](https://code.claude.com/docs/en/desktop#monitor-pull-request-status).

**Поставьте Claude на расписание.** Настройте [запланированные задачи](https://code.claude.com/docs/en/desktop-scheduled-tasks), чтобы запускать Claude автоматически на регулярной основе: ежедневное ревью кода каждое утро, еженедельный аудит зависимостей или сводку, которая подтягивает данные из ваших подключённых инструментов.

**Масштабируйтесь, когда будете готовы.** Открывайте [параллельные сессии](https://code.claude.com/docs/en/desktop#work-in-parallel-with-sessions) из боковой панели, чтобы работать над несколькими задачами одновременно, каждая в своём Git-worktree, и открывайте [панель задач](https://code.claude.com/docs/en/desktop#watch-background-tasks), чтобы следить за субагентами и фоновыми командами, которые выполняет сессия. Откройте [боковой чат](https://code.claude.com/docs/en/desktop#ask-a-side-question-without-derailing-the-session), чтобы задать вопрос, не сбивая с курса основной поток. Отправляйте [длительную работу в облако](https://code.claude.com/docs/en/desktop#run-long-running-tasks-remotely), чтобы она продолжалась, даже если вы закроете приложение, или [продолжайте сессию в вебе или в вашей IDE](https://code.claude.com/docs/en/desktop#continue-in-another-surface), если задача занимает больше времени, чем ожидалось. [Подключайте внешние инструменты](https://code.claude.com/docs/en/desktop#extend-claude-code), такие как GitHub, Slack и Linear, чтобы объединить ваш рабочий процесс воедино.

## Переходите с CLI?

Desktop работает на том же движке, что и CLI, но с графическим интерфейсом. Вы можете запускать оба одновременно на одном проекте, и они разделяют конфигурацию (файлы CLAUDE.md, серверы MCP, хуки, навыки и настройки). Полное сравнение функций, эквивалентов флагов и того, что недоступно в Desktop, см. в разделе [Сравнение с CLI](https://code.claude.com/docs/en/desktop#coming-from-the-cli).

# Ultrareview

> Запускайте глубокое многоагентное код-ревью в облаке командой `/code-review ultra`, чтобы находить и проверять баги до слияния.

> **Примечание.** Ultrareview — это функция в статусе исследовательского превью, доступная в Claude Code версии v2.1.86 и новее. Функция, её стоимость и доступность могут измениться на основе обратной связи. Команда теперь вызывается как `/code-review ultra`, а `/ultrareview` остаётся псевдонимом.

Ultrareview — это глубокое код-ревью, которое выполняется на инфраструктуре Claude Code в вебе. Когда вы запускаете `/code-review ultra`, Claude Code разворачивает флотилию агентов-рецензентов в удалённой песочнице, чтобы найти баги в вашей ветке или pull request.

По сравнению с локальными `/code-review` или `/review`, ultrareview предлагает:

* **Более высокий сигнал**: каждая обнаруженная находка независимо воспроизводится и проверяется, поэтому результаты сосредоточены на реальных багах, а не на замечаниях по стилю
* **Более широкий охват**: более крупная флотилия агентов-рецензентов исследует изменения параллельно, что выявляет проблемы, которые может упустить локальное ревью со средним уровнем усилий
* **Отсутствие использования локальных ресурсов**: ревью выполняется полностью в удалённой песочнице, поэтому во время его работы ваш терминал остаётся свободным для других задач

Ultrareview требует аутентификации через аккаунт Claude.ai, поскольку выполняется на инфраструктуре Claude Code в вебе. Если вы вошли только по API-ключу, выполните `/login` и сначала пройдите аутентификацию через Claude.ai. Ultrareview недоступен при использовании Claude Code с Amazon Bedrock, Google Cloud Vertex AI или Microsoft Foundry, а также недоступен для организаций, включивших Zero Data Retention.

## Запуск ultrareview из CLI

Начните ревью из любого git-репозитория в Claude Code CLI.

```text
/code-review ultra
```

Без аргументов ultrareview проверяет разницу между вашей текущей веткой и веткой по умолчанию, включая любые незакоммиченные и проиндексированные изменения в вашем рабочем дереве. Claude Code упаковывает состояние репозитория и загружает его в удалённую песочницу для ревью.

Чтобы вместо этого проверить GitHub pull request, передайте номер PR.

```text
/code-review ultra 1234
```

В режиме PR удалённая песочница клонирует pull request непосредственно с хоста, а не упаковывает ваше локальное рабочее дерево. Режим PR работает с репозиториями на `github.com` и на экземплярах [GitHub Enterprise Server](https://code.claude.com/docs/en/github-enterprise-server), которые владелец (Owner) подключил к Claude Code.

> **Совет.** Если ваш репозиторий слишком велик для упаковки, Claude Code предложит вам использовать режим PR. Отправьте свою ветку и откройте черновой (draft) PR, затем выполните `/code-review ultra <PR-number>`.
>
> Если разница pull request слишком велика, Claude Code откажется от ревью с подсказкой об области охвата ещё до начала какой-либо работы по ревью.

Перед запуском Claude Code показывает диалог подтверждения с областью ревью (включая количество файлов и строк при ревью ветки), количеством оставшихся у вас бесплатных запусков и оценочной стоимостью. После подтверждения ревью продолжается в фоновом режиме, и вы можете продолжать пользоваться своей сессией. Команда выполняется только тогда, когда вы вызываете её через `/code-review ultra`; Claude не запускает ultrareview самостоятельно.

## Стоимость и бесплатные запуски

Ultrareview — это премиальная функция, которая оплачивается из кредитов на использование (usage credits), а не из включённого в ваш тарифный план объёма.

| Тарифный план       | Включённые бесплатные запуски | После бесплатных запусков                                                                                     |
| ------------------- | ------------------ | ------------------------------------------------------------------------------------------------------------ |
| Pro                 | 3 бесплатных запуска | оплачивается как [кредиты на использование](https://support.claude.com/en/articles/12429409-extra-usage-for-paid-claude-plans) |
| Max                 | 3 бесплатных запуска | оплачивается как [кредиты на использование](https://support.claude.com/en/articles/12429409-extra-usage-for-paid-claude-plans) |
| Team и Enterprise   | нет                | оплачивается как [кредиты на использование](https://support.claude.com/en/articles/12429409-extra-usage-for-paid-claude-plans) |

Подписчики Pro и Max получают три бесплатных запуска ultrareview, чтобы опробовать функцию. Эти три запуска — единоразовое выделение на аккаунт, и они не обновляются. После того как вы используете все три или после окончания периода бесплатных запусков, каждое ревью оплачивается из кредитов на использование и обычно стоит от \$5 до \$20 в зависимости от размера изменения. Запуск засчитывается, как только начинается облачная сессия, поэтому ревью, которое вы остановили досрочно или которое не завершилось, всё равно расходует бесплатный запуск. Для платного ревью кредиты на использование списываются только за ту часть, которая была выполнена.

Поскольку ultrareview всегда оплачивается как кредиты на использование за пределами бесплатных запусков, на вашем аккаунте или в организации должны быть включены кредиты на использование, прежде чем вы сможете запустить платное ревью. Если кредиты на использование не включены, Claude Code блокирует запуск и направляет вас в настройки биллинга, где вы можете их включить. Вы также можете выполнить `/usage-credits`, чтобы проверить или изменить свою текущую настройку.

## Отслеживание выполняющегося ревью

Ревью обычно занимает от 5 до 10 минут. Ревью выполняется как фоновая задача, поэтому вы можете продолжать работать в своей сессии, запускать другие команды или полностью закрыть терминал.

Используйте `/tasks`, чтобы увидеть выполняющиеся и завершённые ревью, открыть подробный вид ревью или остановить ревью, которое находится в процессе. Остановка ревью архивирует облачную сессию, а частичные находки не возвращаются. Когда ревью завершается, проверенные находки появляются в виде уведомления в вашей сессии. Каждая находка включает расположение в файле и объяснение проблемы, чтобы вы могли попросить Claude исправить её напрямую.

## Запуск ultrareview в неинтерактивном режиме

Используйте подкоманду `claude ultrareview`, чтобы запустить ultrareview из CI или скрипта без интерактивной сессии. Подкоманда запускает то же ревью, что и `/code-review ultra`, блокируется до завершения удалённого ревью, выводит находки в stdout и завершается с кодом 0 при успехе или 1 при сбое.

```bash
claude ultrareview
claude ultrareview 1234
claude ultrareview origin/main
```

Без аргументов подкоманда проверяет разницу между вашей текущей веткой и веткой по умолчанию. Передайте номер PR, чтобы проверить pull request, или передайте базовую ветку, чтобы вместо этого проверить разницу относительно этой ветки. Вызов подкоманды считается согласием с приглашением по биллингу и условиям, которое показывает интерактивная команда.

Сообщения о ходе выполнения и URL живой сессии направляются в stderr, чтобы stdout оставался пригодным для разбора. Используйте эти флаги для управления выводом и таймаутом:

| Флаг                  | Описание                                                            |
| --------------------- | ------------------------------------------------------------------- |
| `--json`              | Вывести необработанную полезную нагрузку `bugs.json` вместо форматированных находок |
| `--timeout <minutes>` | Максимальное количество минут ожидания завершения ревью. По умолчанию 30 |

Для запуска `claude ultrareview` требуется та же аутентификация и настройка кредитов на использование, что и для `/code-review ultra`. Подкоманда завершается с кодом 0, когда ревью завершается с находками или без них, кодом 1, когда ревью не удаётся запустить, облачная сессия завершается ошибкой или истекает таймаут, и кодом 130 при прерывании через Ctrl-C. Удалённое ревью продолжает выполняться, если вы прерываете подкоманду; перейдите по URL сессии, выведенному в stderr, чтобы наблюдать за ним в браузере.

Для автоматических ревью GitHub pull request функция [Code Review](https://code.claude.com/docs/en/code-review) интегрируется с вашим репозиторием напрямую и публикует находки как встроенные комментарии в PR без шага в CLI.

## Как ultrareview соотносится с /code-review и /review

Все три команды выполняют ревью кода, но нацелены на разные этапы вашего рабочего процесса.

|          | `/code-review`                  | `/review <pr>`                             | `/code-review ultra`                                            |
| -------- | ------------------------------- | ------------------------------------------ | --------------------------------------------------------------- |
| Цель     | ваша рабочая разница            | GitHub pull request                        | ваша рабочая разница или pull request                           |
| Где выполняется | локально в вашей сессии   | локально в вашей сессии                    | удалённо в облачной песочнице                                   |
| Глубина  | масштабируется с аргументом уровня усилий | движок `/code-review` среднего уровня | многоагентная флотилия с независимой проверкой                  |
| Длительность | от секунд до нескольких минут | несколько минут                          | примерно от 5 до 10 минут                                      |
| Стоимость | засчитывается в обычное использование | засчитывается в обычное использование | бесплатные запуски, затем примерно от \$5 до \$20 за ревью как кредиты на использование |
| Лучше всего для | быстрой обратной связи по ходу работы | ревью PR коллеги перед одобрением | уверенности перед слиянием при существенных изменениях          |

Используйте `/code-review` для быстрой обратной связи по ходу работы. Используйте `/review <pr>`, чтобы просмотреть pull request так же, как вы делали бы это перед одобрением. Используйте `/code-review ultra` перед слиянием существенного изменения, когда вам нужен более глубокий проход, выявляющий проблемы, которые локальное ревью может упустить.

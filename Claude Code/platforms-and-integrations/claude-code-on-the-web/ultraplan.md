# Ultraplan

> Начните планирование из CLI, составьте план в Claude Code в вебе, а затем выполните его удалённо или снова в терминале

> **Примечание.** Ultraplan находится в стадии исследовательского превью (research preview) и требует Claude Code версии v2.1.91 или новее. Поведение и возможности могут меняться на основе обратной связи.

Ultraplan передаёт задачу планирования из вашего локального CLI в сессию [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web), работающую в [режиме планирования](https://code.claude.com/docs/en/permission-modes#analyze-before-you-edit-with-plan-mode). Claude составляет план в облаке, пока вы продолжаете работать в терминале. Когда план готов, вы открываете его в браузере, чтобы комментировать конкретные разделы, запрашивать правки и выбирать, где его выполнить.

Это удобно, когда вам нужна более богатая поверхность для проверки, чем предлагает терминал:

* **Точечная обратная связь**: комментируйте отдельные разделы плана вместо ответа на него целиком
* **Составление без участия рук**: план генерируется удалённо, поэтому ваш терминал остаётся свободным для другой работы
* **Гибкое выполнение**: одобрите план для запуска в вебе и открытия pull request или отправьте его обратно в терминал

Ultraplan требует учётной записи [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web) и репозитория GitHub. Поскольку он работает на облачной инфраструктуре Anthropic, он недоступен при использовании Amazon Bedrock, Google Cloud Vertex AI или Microsoft Foundry. Облачная сессия запускается в облачном окружении по умолчанию для вашей учётной записи ([cloud environment](https://code.claude.com/docs/en/claude-code-on-the-web#the-cloud-environment)). Если у вас ещё нет облачного окружения, ultraplan создаст его автоматически при первом запуске.

## Запуск ultraplan из CLI

Из локальной сессии CLI вы можете запустить ultraplan тремя способами:

* **Команда**: выполните `/ultraplan`, а затем ваш промпт
* **Ключевое слово**: включите слово `ultraplan` в любое место обычного промпта
* **Из локального плана**: когда Claude завершает локальный план и показывает диалог одобрения, выберите **No, refine with Ultraplan on Claude Code on the web**, чтобы отправить черновик в облако для дальнейшей итерации

Например, чтобы спланировать миграцию сервиса командой:

```
/ultraplan migrate the auth service from sessions to JWTs
```

Пути через команду и ключевое слово открывают диалог подтверждения перед запуском. Путь через локальный план пропускает этот диалог, потому что этот выбор уже служит подтверждением. Если активен [Remote Control](https://code.claude.com/docs/en/remote-control), он отключается при запуске ultraplan, потому что обе функции занимают интерфейс claude.ai/code и одновременно может быть подключена только одна.

После запуска облачной сессии поле ввода промпта в вашем CLI показывает индикатор состояния, пока облачная сессия работает:

| Состояние                      | Значение                                                                          |
| :----------------------------- | :-------------------------------------------------------------------------------- |
| `◇ ultraplan`                  | Claude исследует ваш кодовый базис и составляет план                               |
| `◇ ultraplan needs your input` | У Claude есть уточняющий вопрос; откройте ссылку сессии, чтобы ответить            |
| `◆ ultraplan ready`            | План готов к проверке в вашем браузере                                             |

Выполните `/tasks` и выберите запись ultraplan, чтобы открыть детальный вид со ссылкой сессии, активностью агента и действием **Stop ultraplan**. Остановка архивирует облачную сессию и очищает индикатор; в ваш терминал ничего не сохраняется.

## Проверка и правка плана в браузере

Когда состояние меняется на `◆ ultraplan ready`, откройте ссылку сессии, чтобы просмотреть план на claude.ai. План появляется в специальном виде для проверки:

* **Встроенные комментарии**: выделите любой фрагмент и оставьте комментарий, чтобы Claude его учёл
* **Реакции-эмодзи**: реагируйте на раздел, чтобы обозначить одобрение или беспокойство, не написав полный комментарий
* **Боковая панель оглавления**: переходите между разделами плана

Когда вы просите Claude учесть ваши комментарии, он правит план и представляет обновлённый черновик. Вы можете итерировать столько раз, сколько нужно, прежде чем выбрать, где выполнить план.

## Выбор места выполнения

Когда план выглядит верным, вы выбираете в браузере, будет ли Claude реализовывать его в той же облачной сессии или отправит его обратно в ваш ожидающий терминал.

### Выполнение в вебе

Выберите **Approve Claude's plan and start coding** в браузере, чтобы Claude реализовал план в той же сессии Claude Code в вебе. Ваш терминал показывает подтверждение, индикатор состояния очищается, и работа продолжается в облаке. Когда реализация завершается, [проверьте изменения (diff)](https://code.claude.com/docs/en/claude-code-on-the-web#review-changes) и создайте pull request через веб-интерфейс.

### Отправка плана обратно в терминал

Выберите **Approve plan and teleport back to terminal** в браузере, чтобы реализовать план локально с полным доступом к вашему окружению. Этот вариант появляется, когда сессия была запущена из вашего CLI и терминал всё ещё опрашивает состояние. Веб-сессия архивируется, чтобы она не продолжала работать параллельно.

Ваш терминал показывает план в диалоге под названием **Ultraplan approved** с тремя вариантами:

* **Implement here**: внедрить план в текущий разговор и продолжить с того места, где вы остановились
* **Start new session**: очистить текущий разговор и начать заново, имея в качестве контекста только план
* **Cancel**: сохранить план в файл без выполнения; Claude выводит путь к файлу, чтобы вы могли вернуться к нему позже

Если вы начинаете новую сессию, Claude выводит вверху команду `claude --resume`, чтобы вы могли вернуться к предыдущему разговору позже.

## Связанные ресурсы

* [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web): облачная инфраструктура, на которой работает ultraplan
* [Режим планирования](https://code.claude.com/docs/en/permission-modes#analyze-before-you-edit-with-plan-mode): как работает планирование в локальной сессии
* [Поиск багов с помощью ultrareview](https://code.claude.com/docs/en/ultrareview): аналог проверки кода для ultraplan, позволяющий выявлять проблемы до слияния
* [Remote Control](https://code.claude.com/docs/en/remote-control): использование интерфейса claude.ai/code с сессией, работающей на вашей собственной машине

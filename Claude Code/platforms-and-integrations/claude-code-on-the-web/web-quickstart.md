# Быстрый старт в вебе

> Запускайте Claude Code в облаке из браузера или с телефона. Подключите репозиторий GitHub, поставьте задачу и проверьте PR без локальной настройки.

> **Примечание.** Claude Code в вебе доступен в режиме исследовательского превью для пользователей Pro, Max и Team, а также для корпоративных (Enterprise) пользователей с премиум-местами или местами Chat + Claude Code.

Claude Code в вебе работает на управляемой Anthropic облачной инфраструктуре, а не на вашей машине. Ставьте задачи с [claude.ai/code](https://claude.ai/code) в браузере или в мобильном приложении Claude.

Для [начала работы](#connect-github-and-create-an-environment) вам понадобится репозиторий GitHub. Claude клонирует его в изолированную виртуальную машину, вносит изменения и отправляет ветку, чтобы вы могли её проверить. Сессии сохраняются между устройствами, поэтому задачу, начатую на ноутбуке, можно позже проверить с телефона.

Claude Code в вебе хорошо подходит для:

* **Параллельных задач**: запускайте несколько независимых задач одновременно, каждую в собственной сессии и ветке, без необходимости управлять несколькими рабочими деревьями (worktrees)
* **Репозиториев, которых нет локально**: Claude заново клонирует репозиторий в каждой сессии, поэтому его не нужно держать выгруженным локально
* **Задач, не требующих частого управления**: поставьте чётко определённую задачу, займитесь чем-то другим и проверьте результат, когда Claude закончит
* **Вопросов о коде и исследования**: разберитесь в кодовой базе или проследите, как реализована функция, без локальной выгрузки

Для работы, которой нужны ваша локальная конфигурация, инструменты или окружение, лучше подойдёт запуск Claude Code локально или использование [Remote Control](https://code.claude.com/docs/en/remote-control).

## Как выполняются сессии

Когда вы ставите задачу:

1. **Клонирование и подготовка**: ваш репозиторий клонируется на управляемую Anthropic виртуальную машину, и, если настроен, запускается ваш [скрипт настройки](https://code.claude.com/docs/en/claude-code-on-the-web#setup-scripts).
2. **Настройка сети**: доступ в интернет устанавливается на основе [уровня доступа](https://code.claude.com/docs/en/claude-code-on-the-web#access-levels) вашего окружения.
3. **Работа**: Claude анализирует код, вносит изменения, запускает тесты и проверяет свою работу. Вы можете наблюдать и направлять его на протяжении всего процесса или отойти и вернуться, когда он закончит.
4. **Отправка ветки**: когда Claude достигает точки остановки, он отправляет свою ветку в GitHub. Вы проверяете дифф, оставляете встроенные комментарии, создаёте PR или отправляете новое сообщение, чтобы продолжить.

Сессия не закрывается после отправки ветки. Создание PR и дальнейшие правки происходят в рамках того же разговора.

## Сравнение способов запуска Claude Code

Claude Code ведёт себя одинаково везде. Меняется лишь то, где выполняется код и доступна ли ваша локальная конфигурация. Приложение Desktop предлагает как локальные, так и облачные сессии, поэтому его ответы ниже зависят от того, какую вы выберете:

|                                              | В вебе                                                                                                            | Remote Control               | Терминальный CLI       | Приложение Desktop           |
| :------------------------------------------- | :-------------------------------------------------------------------------------------------------------------- | :--------------------------- | :--------------------- | :-------------------------- |
| **Код выполняется на**                       | Облачной VM Anthropic                                                                                            | Вашей машине                 | Вашей машине           | Вашей машине или облачной VM |
| **Вы общаетесь из**                          | claude.ai или мобильного приложения                                                                              | claude.ai или мобильного приложения | Вашего терминала  | Интерфейса Desktop           |
| **Использует вашу локальную конфигурацию**   | Нет, только репозиторий                                                                                          | Да                           | Да                     | Да для локальных, нет для облачных |
| **Требует GitHub**                           | Да, или [упакуйте локальный репозиторий](https://code.claude.com/docs/en/claude-code-on-the-web#send-local-repositories-without-github) через `--remote` | Нет | Нет | Только для облачных сессий |
| **Продолжает работу при отключении**         | Да                                                                                                              | Пока терминал открыт         | Нет                    | Зависит от типа сессии       |
| **[Режимы разрешений](https://code.claude.com/docs/en/permission-modes)** | Accept edits, Plan, Auto                                                                    | Ask, Auto accept edits, Plan | Все режимы             | Зависит от типа сессии       |
| **Доступ к сети**                            | Настраивается для каждого окружения                                                                              | Сеть вашей машины            | Сеть вашей машины      | Зависит от типа сессии       |

Смотрите документацию по [быстрому старту в терминале](https://code.claude.com/docs/en/quickstart), [приложению Desktop](https://code.claude.com/docs/en/desktop) или [Remote Control](https://code.claude.com/docs/en/remote-control), чтобы их настроить.

## Подключение GitHub и создание окружения

Настройка — это разовый процесс. Если вы уже используете GitHub CLI, вы можете [сделать это из терминала](#connect-from-your-terminal) вместо браузера.

1. ### Перейдите на claude.ai/code

   Зайдите на [claude.ai/code](https://claude.ai/code) и войдите со своей учётной записью Anthropic.

2. ### Установите приложение Claude GitHub App

   После входа claude.ai/code предложит подключить GitHub. Следуйте подсказке, чтобы установить приложение Claude GitHub App и предоставить ему доступ к вашим репозиториям. Облачные сессии работают с существующими репозиториями GitHub, поэтому, чтобы начать новый проект, сначала [создайте пустой репозиторий на GitHub](https://github.com/new).

3. ### Создайте своё окружение

   После подключения GitHub вам будет предложено создать облачное окружение. Окружение определяет, какой сетевой доступ есть у Claude во время сессий и что запускается при создании новой сессии. Смотрите [Установленные инструменты](https://code.claude.com/docs/en/claude-code-on-the-web#installed-tools), чтобы узнать, что доступно без какой-либо настройки.

   В форме есть следующие поля:

   * **Name**: отображаемая метка. Полезна, когда у вас несколько окружений для разных проектов или уровней доступа.
   * **Network access**: определяет, к чему сессия может обращаться в интернете. Значение по умолчанию, `Trusted`, разрешает подключения к [распространённым реестрам пакетов](https://code.claude.com/docs/en/claude-code-on-the-web#default-allowed-domains), таким как npm, PyPI и RubyGems, при этом блокируя общий доступ в интернет.
   * **Environment variables**: необязательные переменные, доступные в каждой сессии, в формате `.env`. Не заключайте значения в кавычки, поскольку кавычки сохраняются как часть значения. Они видны всем, кто может редактировать это окружение.
   * **Setup script**: необязательный Bash-скрипт, который запускается перед стартом Claude Code. Используйте его для установки системных инструментов, которых нет в облачной VM, например `apt install -y gh`. Результат [кэшируется](https://code.claude.com/docs/en/claude-code-on-the-web#environment-caching), поэтому скрипт не запускается заново в каждой сессии. Примеры и советы по отладке смотрите в разделе [Скрипты настройки](https://code.claude.com/docs/en/claude-code-on-the-web#setup-scripts).

   Для первого проекта оставьте значения по умолчанию и нажмите **Create environment**. Позже вы сможете [отредактировать его или создать дополнительные окружения](https://code.claude.com/docs/en/claude-code-on-the-web#configure-your-environment) для разных проектов.

### Подключение из терминала

Если вы уже используете GitHub CLI (`gh`), вы можете настроить Claude Code в вебе, не открывая браузер. Для этого требуется [Claude Code CLI](https://code.claude.com/docs/en/quickstart). `/web-setup` считывает ваш локальный токен `gh`, привязывает его к вашей учётной записи Claude и создаёт облачное окружение по умолчанию, если у вас его ещё нет.

> **Примечание.** Организации с включённым [Zero Data Retention](https://code.claude.com/docs/en/zero-data-retention) не могут использовать `/web-setup` или другие функции облачных сессий. Если GitHub CLI не установлен или не авторизован, `/web-setup` вместо этого открывает процесс онбординга в браузере.

1. ### Авторизуйтесь в GitHub CLI

   В своей оболочке авторизуйтесь в GitHub CLI, если вы ещё этого не сделали:

   ```bash
   gh auth login
   ```

2. ### Войдите в Claude

   В Claude Code CLI выполните `/login`, чтобы войти со своей учётной записью claude.ai. Пропустите этот шаг, если вы уже вошли.

3. ### Запустите /web-setup

   В Claude Code CLI выполните:

   ```text
   /web-setup
   ```

   Это синхронизирует ваш токен `gh` с вашей учётной записью Claude. Если у вас ещё нет облачного окружения, `/web-setup` создаёт его с сетевым доступом Trusted и без скрипта настройки. Впоследствии вы можете [отредактировать окружение или добавить переменные](https://code.claude.com/docs/en/claude-code-on-the-web#configure-your-environment). После завершения `/web-setup` вы можете запускать облачные сессии из терминала с помощью [`--remote`](https://code.claude.com/docs/en/claude-code-on-the-web#from-terminal-to-web) или настраивать повторяющиеся задачи с помощью [`/schedule`](https://code.claude.com/docs/en/routines).

## Постановка задачи

Когда GitHub подключён и окружение создано, вы готовы ставить задачи.

1. ### Выберите репозиторий и ветку

   На [claude.ai/code](https://claude.ai/code) или во вкладке Code в мобильном приложении Claude нажмите на селектор репозитория под полем ввода и выберите репозиторий, в котором будет работать Claude. Для каждого репозитория показывается селектор ветки. Измените его, чтобы Claude начал с функциональной ветки, а не с ветки по умолчанию. Вы можете добавить несколько репозиториев, чтобы работать с ними в одной сессии.

2. ### Выберите режим разрешений

   Выпадающий список режимов рядом с полем ввода по умолчанию установлен на **Accept edits**, при котором Claude вносит изменения и отправляет ветку, не останавливаясь для получения одобрения. Переключитесь на **Plan mode**, если хотите, чтобы Claude предложил подход и дождался вашего согласия, прежде чем редактировать файлы. Облачные сессии не предлагают разрешения Ask или Bypass. Полный список смотрите в разделе [Режимы разрешений](https://code.claude.com/docs/en/permission-modes).

3. ### Опишите задачу и отправьте

   Введите описание того, что вам нужно, и нажмите Enter. Будьте конкретны:

   * Называйте файл или функцию: «Add a README with setup instructions» или «Fix the failing auth test in `tests/test_auth.py`» лучше, чем «fix tests»
   * Вставляйте вывод ошибки, если он у вас есть
   * Описывайте ожидаемое поведение, а не только симптом

   Claude клонирует репозитории, запускает ваш скрипт настройки, если он настроен, и начинает работу. Каждая задача получает собственную сессию и собственную ветку, поэтому вам не нужно ждать завершения одной, чтобы начать другую.

## Предзаполнение сессий

Вы можете предзаполнить промпт, репозитории и окружение для новой сессии, добавив параметры запроса к URL [claude.ai/code](https://claude.ai/code). Используйте это для создания интеграций, таких как кнопка в вашем трекере задач, которая открывает Claude Code с описанием задачи в качестве промпта.

| Параметр       | Описание                                                                                                                                                        |
| :------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `prompt`       | Текст промпта для предзаполнения в поле ввода. Также принимается псевдоним `q`.                                                                                 |
| `prompt_url`   | URL, из которого нужно получить текст промпта, для промптов, слишком длинных для встраивания в строку запроса. URL должен разрешать кросс-доменные запросы. Игнорируется, когда также задан `prompt`. |
| `repositories` | Список слагов `owner/repo` через запятую для предвыбора. Также принимается псевдоним `repo`.                                                                    |
| `environment`  | Имя или ID [окружения](#connect-github-and-create-an-environment) для предвыбора.                                                                               |

Кодируйте каждое значение в URL-формат. Пример ниже открывает форму с уже выбранными промптом и репозиторием:

```text
https://claude.ai/code?prompt=Fix%20the%20login%20bug&repositories=acme/webapp
```

## Проверка и итерации

Когда Claude закончит, проверьте изменения, оставьте отзыв к конкретным строкам и продолжайте, пока дифф не будет выглядеть правильно.

1. ### Откройте представление диффа

   Индикатор диффа показывает добавленные и удалённые строки за сессию, например `+42 -18`. Выберите его, чтобы открыть представление диффа со списком файлов слева и изменениями справа.

2. ### Оставляйте встроенные комментарии

   Выберите любую строку в диффе, введите свой отзыв и нажмите Enter. Комментарии накапливаются в очереди, пока вы не отправите следующее сообщение, после чего объединяются с ним. Claude видит «at `src/auth.ts:47`, don't catch the error here» рядом с вашей основной инструкцией, поэтому вам не нужно описывать, где находится проблема.

3. ### Создайте pull request

   Когда дифф выглядит правильно, выберите **Create PR** в верхней части представления диффа. Вы можете открыть его как полноценный PR, черновик или перейти на страницу создания GitHub с сгенерированными заголовком и описанием.

4. ### Продолжайте итерации после PR

   Сессия остаётся активной после создания PR. Вставьте вывод сбоя CI или комментарии рецензента в чат и попросите Claude их устранить. Чтобы Claude отслеживал PR автоматически, смотрите [Автоисправление pull request'ов](https://code.claude.com/docs/en/claude-code-on-the-web#auto-fix-pull-requests).

## Устранение проблем настройки

### После подключения GitHub не появляются репозитории

Облачная сессия может использовать любой репозиторий, видимый подключённой учётной записи GitHub, независимо от того, на каких репозиториях установлено приложение Claude GitHub App. Если репозиторий отсутствует, убедитесь, что подключённая учётная запись GitHub имеет к нему доступ на GitHub. Если вам также нужно [Автоисправление](https://code.claude.com/docs/en/claude-code-on-the-web#auto-fix-pull-requests) для репозитория, установите на него приложение: на github.com откройте **Settings → Applications → Claude → Configure** и убедитесь, что репозиторий указан в разделе **Repository access**. Приватным репозиториям нужна такая же авторизация, как и публичным.

### На странице отображается только кнопка входа в GitHub

Облачные сессии требуют подключённой учётной записи GitHub. Подключитесь через процесс в браузере, описанный выше, или выполните `/web-setup` из терминала, если вы используете GitHub CLI. Если вы предпочитаете вообще не подключать GitHub, смотрите [Remote Control](https://code.claude.com/docs/en/remote-control), чтобы запускать Claude Code на собственной машине и отслеживать его из веба.

### «Not available for the selected organization»

Корпоративным (Enterprise) организациям может потребоваться, чтобы владелец (Owner) включил Claude Code в вебе. Обратитесь к вашей команде по работе с клиентами Anthropic.

### `/web-setup` показывает «No commands match» или «Unknown command»

`/web-setup` выполняется внутри Claude Code CLI, а не в вашей оболочке. Сначала запустите `claude`, затем введите `/web-setup` в приглашении.

Если вы ввели команду внутри Claude Code, а меню команд показывает `No commands match "/web-setup"`, или при отправке возвращается `Unknown command: /web-setup`, команда скрыта, потому что не выполнено какое-то требование. Обычно причина в том, что ваш CLI старше v2.1.80 или вы авторизованы через API-ключ или сторонний провайдер, а не через подписку claude.ai. Выполните `claude update`, затем `/login`, чтобы войти со своей учётной записью claude.ai.

### «Could not create a cloud environment» или «No cloud environment available» при использовании `--remote` или ultraplan

Функции удалённых сессий автоматически создают облачное окружение по умолчанию, если у вас его нет. Если вы видите «Could not create a cloud environment», автоматическое создание не удалось. Если вы видите «No cloud environment available», ваш CLI появился раньше автоматического создания. В любом случае выполните `/web-setup` в Claude Code CLI, чтобы создать его вручную, или зайдите на [claude.ai/code](https://claude.ai/code) и выполните описанный выше шаг **Create your environment**.

### Скрипт настройки завершился с ошибкой

Скрипт настройки завершился с ненулевым статусом, что блокирует запуск сессии. Распространённые причины:

* Установка пакета не удалась, потому что реестр не входит в ваш [уровень сетевого доступа](https://code.claude.com/docs/en/claude-code-on-the-web#access-levels). `Trusted` охватывает большинство менеджеров пакетов; `None` блокирует их все.
* Скрипт ссылается на файл или путь, который не существует в свежем клоне.
* Команда, работающая локально, требует другого вызова на Ubuntu.

Для отладки добавьте `set -x` в начало скрипта, чтобы увидеть, какая команда завершилась с ошибкой. Для некритичных команд добавьте `|| true`, чтобы они не блокировали запуск сессии.

### Новые сессии зависают или прерываются по тайм-ауту во время настройки

Если новые сессии застревают на шаге скрипта настройки или завершаются с общей ошибкой контейнера до завершения скрипта, скорее всего, скрипт превышает бюджет времени примерно в пять минут на сборку [кэша окружения](https://code.claude.com/docs/en/claude-code-on-the-web#environment-caching). Тяжёлые шаги, такие как загрузка больших Docker-образов, синхронизация полных деревьев зависимостей или скачивание весов модели, часто выводят общее время за предел, особенно когда они выполняются один за другим.

Чтобы это исправить, сократите скрипт так, чтобы он надёжно завершался менее чем за пять минут:

* Запускайте независимые установки параллельно с `&` и финальным `wait` вместо последовательного выполнения.
* Вынесите самые большие загрузки из скрипта настройки в [хук SessionStart](https://code.claude.com/docs/en/claude-code-on-the-web#setup-scripts-vs-sessionstart-hooks), который запускает их в фоне, чтобы сессия стала пригодной к использованию, пока они завершаются.
* Уберите из скрипта настройки длинные паузы повторных попыток (retry sleeps), поскольку зависший цикл повторов засчитывается в бюджет.

### Сессия продолжает работать после закрытия вкладки

Так и задумано. Закрытие вкладки или переход на другую страницу не останавливает сессию. Она продолжает работать в фоне, пока Claude не завершит текущую задачу, а затем простаивает. На боковой панели вы можете [заархивировать сессию](https://code.claude.com/docs/en/claude-code-on-the-web#archive-sessions), чтобы скрыть её из списка, или [удалить её](https://code.claude.com/docs/en/claude-code-on-the-web#delete-sessions), чтобы убрать навсегда.

## Дальнейшие шаги

Теперь, когда вы умеете ставить и проверять задачи, эти страницы охватывают то, что будет дальше: запуск облачных сессий из терминала, планирование повторяющейся работы и предоставление Claude постоянных инструкций.

* [Использование Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web): полный справочник, включая телепортацию сессий в ваш терминал, скрипты настройки, переменные окружения и настройку сети
* [Routines](https://code.claude.com/docs/en/routines): автоматизируйте работу по расписанию, через вызов API или в ответ на события GitHub
* [CLAUDE.md](https://code.claude.com/docs/en/memory): предоставьте Claude постоянные инструкции и контекст, которые загружаются в начале каждой сессии
* Установите мобильное приложение Claude для [iOS](https://apps.apple.com/us/app/claude-by-anthropic/id6473753684) или [Android](https://play.google.com/store/apps/details?id=com.anthropic.claude), чтобы отслеживать сессии с телефона. Из Claude Code CLI команда `/mobile` показывает QR-код.

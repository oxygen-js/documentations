# Удалённое управление (Remote Control)

> Продолжайте локальную сессию Claude Code с телефона, планшета или любого браузера с помощью Remote Control. Работает с claude.ai/code и мобильным приложением Claude.

> **Примечание.** Remote Control находится в стадии исследовательского превью и доступно на всех тарифах. На тарифах Team и Enterprise функция по умолчанию выключена, пока владелец (Owner) не включит переключатель Remote Control в [настройках администратора Claude Code](https://claude.ai/admin-settings/claude-code).

Remote Control подключает [claude.ai/code](https://claude.ai/code) или приложение Claude для [iOS](https://apps.apple.com/us/app/claude-by-anthropic/id6473753684) и [Android](https://play.google.com/store/apps/details?id=com.anthropic.claude) к сессии Claude Code, запущенной на вашей машине. Начните задачу за рабочим столом, а затем продолжите её с телефона на диване или в браузере на другом компьютере.

Когда вы запускаете сессию Remote Control на своей машине, Claude всё время продолжает работать локально, поэтому ничего не переносится в облако. С помощью Remote Control вы можете:

* **Использовать полное локальное окружение удалённо**: ваша файловая система, [MCP-серверы](https://code.claude.com/docs/en/mcp), инструменты и конфигурация проекта остаются доступными, а при вводе `@` автодополняются пути к файлам из вашего локального проекта
* **Работать сразу с обеих сторон**: разговор синхронизируется между всеми подключёнными устройствами, поэтому вы можете отправлять сообщения из терминала, браузера и с телефона взаимозаменяемо
* **Переживать прерывания**: если ваш ноутбук уходит в сон или пропадает сеть, сессия автоматически переподключается, когда ваша машина снова выходит в онлайн

В отличие от [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web), который работает на облачной инфраструктуре, сессии Remote Control выполняются непосредственно на вашей машине и взаимодействуют с вашей локальной файловой системой. Веб- и мобильный интерфейсы — это лишь окно в ту самую локальную сессию.

> **Примечание.** Remote Control требует Claude Code v2.1.51 или новее. Проверьте свою версию командой `claude --version`.

Эта страница охватывает настройку, запуск сессий и подключение к ним, а также сравнение Remote Control с Claude Code в вебе.

## Требования

Перед использованием Remote Control убедитесь, что ваше окружение удовлетворяет следующим условиям:

* **Подписка**: доступно на тарифах Pro, Max, Team и Enterprise. API-ключи не поддерживаются. На тарифах Team и Enterprise владелец (Owner) должен сначала включить переключатель Remote Control в [настройках администратора Claude Code](https://claude.ai/admin-settings/claude-code).
* **Аутентификация**: запустите `claude` и используйте `/login`, чтобы войти через claude.ai, если вы этого ещё не сделали.
* **API-эндпоинт**: недоступно на Amazon Bedrock, Google Vertex AI или Microsoft Foundry. Начиная с v2.1.196, Remote Control также отключается, когда [`ANTHROPIC_BASE_URL`](https://code.claude.com/docs/en/env-vars) указывает на хост, отличный от `api.anthropic.com`, например на [LLM-шлюз](https://code.claude.com/docs/en/llm-gateway) или прокси. Сбросьте эту переменную, чтобы использовать Remote Control.
* **Доверие к рабочему пространству**: запустите `claude` в каталоге вашего проекта хотя бы один раз, чтобы принять диалог доверия к рабочему пространству.

## Запуск сессии Remote Control

Вы можете запустить сессию Remote Control из CLI или из расширения VS Code. CLI предлагает три режима вызова; VS Code использует команду `/remote-control`.

### Режим сервера

Перейдите в каталог вашего проекта и выполните:

```bash
claude remote-control
```

Процесс продолжает работать в вашем терминале в режиме сервера, ожидая удалённых подключений. Он отображает URL сессии, который можно использовать, чтобы [подключиться с другого устройства](#connect-from-another-device), а нажатием пробела можно показать QR-код для быстрого доступа с телефона. Пока удалённая сессия активна, терминал показывает статус подключения и активность инструментов.

Доступные флаги:

| Флаг                                            | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ----------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--name "My Project"`                           | Задать пользовательский заголовок сессии, видимый в списке сессий на claude.ai/code.                                                                                                                                                                                                                                                                                                                                                                                                   |
| `--remote-control-session-name-prefix <prefix>` | Префикс для автоматически генерируемых имён сессий, когда явное имя не задано. ### Интерактивная сессия

Чтобы запустить обычную интерактивную сессию Claude Code с включённым Remote Control, используйте флаг `--remote-control` (или `--rc`):

```bash
claude --remote-control
```

При желании передайте имя для сессии:

```bash
claude --remote-control "My Project"
```

Это даёт вам полноценную интерактивную сессию в терминале, которой вы также можете управлять из claude.ai или приложения Claude. В отличие от `claude remote-control` (режим сервера), здесь вы можете вводить сообщения локально, пока сессия одновременно доступна удалённо.

### Из существующей сессии

Если вы уже находитесь в сессии Claude Code и хотите продолжить её удалённо, используйте команду `/remote-control` (или `/rc`):

```text
/remote-control
```

Передайте имя в качестве аргумента, чтобы задать пользовательский заголовок сессии:

```text
/remote-control My Project
```

Это запускает сессию Remote Control, которая переносит вашу текущую историю разговора.

Флаги `--verbose`, `--sandbox` и `--no-sandbox` недоступны с этой командой.

### VS Code

В [расширении Claude Code для VS Code](https://code.claude.com/docs/en/vs-code) введите `/remote-control` или `/rc` в поле ввода, либо откройте меню команд с помощью `/` и выберите её. Требуется Claude Code v2.1.79 или новее.

```text
/remote-control
```

Над полем ввода появляется баннер, показывающий статус подключения. После подключения нажмите **Open in browser** в баннере, чтобы сразу перейти к сессии, или найдите её в списке сессий на [claude.ai/code](https://claude.ai/code). URL сессии также публикуется в разговоре.

Чтобы отключиться, нажмите значок закрытия на баннере или снова выполните `/remote-control`.

В отличие от CLI, команда VS Code не принимает аргумент с именем и не отображает QR-код. Заголовок сессии выводится из вашей истории разговора или первого промпта.

### Проверка статуса подключения

В интерактивной терминальной сессии, пока соединение активно, в подвале под полем ввода находится индикатор `/rc active`, который скрывается, если терминал слишком узок, чтобы его вместить. Текст индикатора является ссылкой на сессию на claude.ai. Выберите его клавишей «стрелка вниз» и нажмите Enter, либо снова выполните `/remote-control`, чтобы открыть панель статуса с URL сессии и QR-кодом, который можно использовать, чтобы [подключиться с другого устройства](#connect-from-another-device).

Если подключение не удаётся, появляется уведомление с причиной сбоя, а индикатор исчезает из подвала. Снова выполните `/remote-control`, чтобы повторить попытку.

### Подключение с другого устройства

Как только сессия Remote Control активна, у вас есть несколько способов подключиться с другого устройства:

* **Откройте URL сессии** в любом браузере, чтобы сразу перейти к сессии на [claude.ai/code](https://claude.ai/code).
* **Отсканируйте QR-код**, показанный рядом с URL сессии, чтобы открыть её напрямую в приложении Claude. С `claude remote-control` нажмите пробел, чтобы переключить отображение QR-кода.
* **Откройте [claude.ai/code](https://claude.ai/code) или приложение Claude** и найдите сессию по имени в списке сессий. В мобильном приложении Claude нажмите **Code** в навигации, чтобы перейти к списку сессий. Сессии Remote Control показываются со значком компьютера и зелёной точкой статуса, когда они онлайн.

Заголовок удалённой сессии выбирается в следующем порядке:

1. Имя, которое вы передали в `--name`, `--remote-control` или `/remote-control`
2. Заголовок, который вы задали командой `/rename`
3. Последнее осмысленное сообщение в существующей истории разговора
4. Автоматически сгенерированное имя вида `myhost-graceful-unicorn`, где `myhost` — это имя хоста вашей машины или префикс, который вы задали с помощью `--remote-control-session-name-prefix`

Если вы не задали явное имя, заголовок обновляется, отражая ваш промпт, как только вы его отправите. Начиная с Claude Code v2.1.176, автоматически генерируемые заголовки соответствуют языку вашего разговора или настройке [`language`](https://code.claude.com/docs/en/settings#available-settings), если она сконфигурирована. Переименование сессии из claude.ai или приложения Claude также обновляет локальный заголовок, отображаемый в `claude --resume`.

Если в окружении уже есть активная сессия, вас спросят, продолжить её или начать новую.

Если у вас ещё нет приложения Claude, используйте команду `/mobile` внутри Claude Code, чтобы показать QR-код для загрузки для [iOS](https://apps.apple.com/us/app/claude-by-anthropic/id6473753684) или [Android](https://play.google.com/store/apps/details?id=com.anthropic.claude).

### Включение Remote Control для всех сессий

Remote Control активируется только тогда, когда вы явно выполняете `claude remote-control`, `claude --remote-control` или `/remote-control`, если только не включено автоподключение. Чтобы включать его автоматически для каждой интерактивной сессии, выполните `/config` внутри Claude Code и установите **Enable Remote Control for all sessions** в `true`. Установите `false`, чтобы никогда не подключаться автоматически, или оставьте значение незаданным, чтобы следовать умолчанию вашей организации. В приложении Desktop это также можно переключить в **Settings → Claude Code → Enable remote control by default**.

Когда эта настройка включена, каждый интерактивный процесс Claude Code регистрирует одну удалённую сессию. Если вы запускаете несколько экземпляров, каждый из них получает своё собственное окружение и сессию. Чтобы запускать несколько одновременных сессий из одного процесса, используйте вместо этого [режим сервера](#start-a-remote-control-session).

## Подключение и безопасность

Ваша локальная сессия Claude Code выполняет только исходящие HTTPS-запросы и никогда не открывает входящие порты на вашей машине. Когда вы запускаете Remote Control, она регистрируется в Anthropic API и опрашивает наличие работы. Когда вы подключаетесь с другого устройства, сервер маршрутизирует сообщения между веб- или мобильным клиентом и вашей локальной сессией по потоковому соединению.

Весь трафик проходит через Anthropic API по TLS — той же транспортной безопасности, что и у любой сессии Claude Code. Соединение использует несколько кратковременных учётных данных, каждое из которых ограничено единственным назначением и истекает независимо.

## Доверенные устройства (Trusted Devices)

> **Примечание.** Доверенные устройства (Trusted Devices) в настоящее время находятся в стадии бета-тестирования. Возможности и функциональность могут развиваться по мере доработки этого опыта.
>
> Доверенные устройства доступны на тарифах Team и Enterprise. Функция по умолчанию выключена, пока администратор её не включит.

Доверенные устройства — это настройка уровня всей организации, которая требует от участников подтвердить своё устройство, прежде чем они смогут просматривать сессии Remote Control или управлять ими из claude.ai, мобильных приложений Claude или Claude Desktop. Она привязывает доступ к Remote Control к известному устройству и недавней аутентификации, а не просто к аккаунту, в который выполнен вход.

Когда настройка включена, взаимодействие с сессией Remote Control требует одновременно обоих следующих условий:

* **Зарегистрированное устройство**: каждый браузер, телефон или десктоп-приложение, которое участник использует для Remote Control, регистрирует своё собственное учётное данное. Регистрация предлагается только вскоре после полноценного входа, так что устройство попадает в список доверенных как часть настоящей аутентификации, а не тихо в фоновом режиме.
* **Недавний вход**: с момента входа участника должно пройти не более 18 часов. Вместо того чтобы входить заново каждый день, участники подтверждают своё присутствие с помощью Face ID, Touch ID, Windows Hello или passkey. Этот биометрический шаг усиления мгновенно обновляет сессию.

Биометрические проверки выполняются на устройстве через операционную систему или браузер — тем же механизмом, что и вход по passkey. Anthropic никогда не получает и не хранит отпечатки пальцев, данные лица или любую другую биометрическую информацию. Хранятся только открытый ключ устройства и базовые метаданные, такие как отображаемое имя, платформа и время регистрации.

Настройка применяется только к Remote Control. Обычный чат Claude, Claude Code в терминале и использование API не затрагиваются.

### Включение доверенных устройств для вашей организации

Администраторы включают настройку из консоли администратора Claude Code.

1. **Откройте настройки администратора Claude Code**

   Перейдите на [claude.ai/admin-settings/claude-code](https://claude.ai/admin-settings/claude-code). Переключатель **Require trusted devices** появляется под настройкой Remote Control.

2. **Включите Require trusted devices**

   Настройка применяется к каждому участнику организации и к сессиям Remote Control, запущенным после того, как вы её включите. Сессии, которые уже выполнялись до включения переключателя, ретроспективно не защищаются и продолжают работать без требования к устройству, пока не завершатся. Определение области действия по командам или по проектам недоступно.

3. **Расскажите участникам, чего ожидать**

   В первый раз, когда участник просматривает новую сессию Remote Control или управляет ею из браузера, телефона или десктоп-приложения после включения настройки, ему предлагается зарегистрировать это устройство. Предупреждение об этом заранее избавляет от путаницы.

### Что видят участники

Регистрация — это одноразовый шаг для каждого устройства. После этого единственное видимое изменение — это периодический биометрический запрос.

* **Первое использование на каждом устройстве**: участнику предлагается пройти регистрацию. Если его вход не является недавним, он сначала входит через ваш обычный процесс, включая SSO, если он настроен, а затем подтверждает регистрацию.
* **Повседневно**: участники с зарегистрированным устройством и недавним входом не видят никаких запросов. Когда с момента входа проходит более 18 часов, при следующем взаимодействии с Remote Control показывается единственный запрос Face ID, Touch ID, Windows Hello или passkey.
* **Незарегистрированные устройства**: сессии Remote Control нельзя просматривать или управлять ими, пока устройство не зарегистрировано. Обычный чат Claude на этом устройстве не затрагивается.
* **Нет платформенного аутентификатора**: участники на машине без Face ID, Touch ID или Windows Hello могут использовать аппаратный ключ безопасности или войти заново вместо усиления.
* **В терминале**: машина, на которой работает Claude Code, автоматически получает своё собственное учётное данное, когда разработчик входит в CLI. Отдельного шага регистрации в терминале нет.

### Управление зарегистрированными устройствами

Участники могут просматривать и отзывать свои собственные устройства из настроек аккаунта.

Откройте [claude.ai/settings/account](https://claude.ai/settings/account#trusted-devices) и найдите раздел **Trusted devices**, чтобы увидеть каждое зарегистрированное устройство с его именем, платформой и датой регистрации. Удаление устройства немедленно отзывает его учётное данное, и устройство можно зарегистрировать снова позже после нового входа. Учётные данные также истекают сами по себе, если их не обновлять, поэтому неиспользуемое устройство автоматически выпадает из списка доверенных.

В случае потерянного или украденного устройства участник удаляет его с этой страницы. Если участник не может войти, администратор может использовать **Sign out everywhere** в консоли администратора, чтобы отозвать каждую сессию и каждое зарегистрированное устройство этого участника, после чего участник заново регистрирует устройства, которые у него ещё остались.

## Remote Control против Claude Code в вебе

Remote Control и [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web) оба используют интерфейс claude.ai/code. Ключевое отличие — где выполняется сессия: Remote Control выполняется на вашей машине, поэтому ваши локальные MCP-серверы, инструменты и конфигурация проекта остаются доступными. Claude Code в вебе выполняется в облачной инфраструктуре, управляемой Anthropic.

Используйте Remote Control, когда вы находитесь в разгаре локальной работы и хотите продолжить её с другого устройства. Используйте Claude Code в вебе, когда хотите запустить задачу без какой-либо локальной настройки, поработать над репозиторием, который у вас не склонирован, или запустить несколько задач параллельно.

## Мобильные push-уведомления

Когда Remote Control активно, Claude может отправлять push-уведомления на ваш телефон.

Claude решает, когда отправлять push. Обычно он отправляет его, когда завершается длительная задача или когда ему нужно ваше решение, чтобы продолжить. Вы также можете запросить push в своём промпте, например `notify me when the tests finish`. Помимо двух переключателей вкл/выкл ниже, конфигурации по отдельным событиям нет.

> **Примечание.** Мобильные push-уведомления требуют Claude Code v2.1.110 или новее.

Чтобы настроить мобильные push-уведомления:

1. **Установите мобильное приложение Claude**

   Загрузите приложение Claude для [iOS](https://apps.apple.com/us/app/claude-by-anthropic/id6473753684) или [Android](https://play.google.com/store/apps/details?id=com.anthropic.claude).

2. **Войдите со своим аккаунтом Claude Code**

   Используйте тот же аккаунт и организацию, что и для Claude Code в терминале.

3. **Разрешите уведомления**

   Примите запрос разрешения на уведомления от операционной системы.

4. **Включите push в Claude Code**

   В своём терминале выполните `/config` и включите **Push when Claude decides** для проактивных уведомлений, **Push when actions required** для запросов разрешений и вопросов, либо оба.

Если уведомления не приходят:

* Если `/config` показывает **No mobile registered**, откройте приложение Claude на телефоне, чтобы оно обновило свой push-токен. Предупреждение исчезнет при следующем подключении Remote Control.
* На iOS режимы Focus и сводки уведомлений могут подавлять или задерживать push-уведомления. Проверьте Settings → Notifications → Claude.
* На Android агрессивная оптимизация батареи может задерживать доставку. Исключите приложение Claude из оптимизации батареи в системных настройках.

Claude Code пропускает мобильные push-уведомления, пока вы вводите текст в подключённом терминале или он находится в фокусе. Начиная с v2.1.181, вы можете задать [`CLAUDE_CLIENT_PRESENCE_FILE`](https://code.claude.com/docs/en/env-vars) в качестве пути к файлу-маркеру, чтобы распространить это на любое время, когда вы находитесь за машиной, даже в другом окне: уведомления пропускаются, пока этот файл существует. Настройте слушатель блокировки экрана или подобный инструмент, чтобы создавать файл, когда ваш экран разблокируется, и удалять его, когда экран блокируется.

## Ограничения

* **Одна удалённая сессия на интерактивный процесс**: вне режима сервера каждый экземпляр Claude Code поддерживает одну удалённую сессию за раз. Используйте [режим сервера](#start-a-remote-control-session), чтобы запускать несколько одновременных сессий из одного процесса.
* **Локальный процесс должен продолжать работать**: Remote Control работает как локальный процесс. Если вы закроете терминал, выйдете из VS Code или иным образом остановите процесс `claude`, сессия завершится.
* **Длительный сетевой сбой**: если ваша машина активна, но не может дотянуться до сети более чем примерно 10 минут, сессия завершается по тайм-ауту, а процесс выходит. Снова выполните `claude remote-control`, чтобы начать новую сессию.
* **Ultraplan отключает Remote Control**: запуск сессии [ultraplan](https://code.claude.com/docs/en/ultraplan) отключает любую активную сессию Remote Control, потому что обе функции занимают интерфейс claude.ai/code, и одновременно может быть подключена только одна.
* **Некоторые команды доступны только локально**: команды, открывающие интерактивный выбор в терминале, такие как `/plugin` или `/resume`, работают только из локального CLI. Следующие работают с мобильного и из веба:
  * Команды с текстовым выводом: `/compact`, `/clear`, `/context`, `/usage`, `/exit`, `/usage-credits`, `/recap`, `/reload-plugins`
  * `/mcp`, начиная с v2.1.166: возвращает текстовую сводку статуса серверов вместо открытия окна выбора и принимает [подкоманды](https://code.claude.com/docs/en/commands#all-commands) `reconnect`, `enable` и `disable`. В отличие от локального CLI, `/mcp reconnect` без имени сервера переподключает каждый сервер, который дал сбой или требует аутентификации.
  * `/config`, начиная с v2.1.181: передайте `key=value`, чтобы установить настройку, или запустите без аргумента, чтобы вывести список ключей, которые вы можете установить.

## Устранение неполадок

### «Remote Control requires a claude.ai subscription»

Вы не аутентифицированы с аккаунтом claude.ai. Выполните `claude auth login` и выберите вариант claude.ai. Если в вашем окружении установлена `ANTHROPIC_API_KEY`, сначала сбросьте её.

### «Remote Control requires a full-scope login token»

Вы аутентифицированы с долгоживущим токеном из `claude setup-token` или переменной окружения `CLAUDE_CODE_OAUTH_TOKEN`. Эти токены ограничены только выводом (inference-only) и не могут устанавливать сессии Remote Control. Выполните `claude auth login`, чтобы аутентифицироваться с сессионным токеном полного объёма вместо этого.

### «Unable to determine your organization for Remote Control eligibility»

Ваша кэшированная информация об аккаунте устарела или неполна. Выполните `claude auth login`, чтобы обновить её.

### «Remote Control is not yet enabled for your account»

Развёртывание Remote Control ещё не дошло до вашего аккаунта, или ваши кэшированные права доступа устарели. Если вы недавно сменили тариф, выполните `claude auth logout`, затем `claude auth login`, чтобы обновить их. Выполните `claude doctor`, чтобы увидеть, какая конкретная проверка права доступа не прошла. Конфликты переменных окружения, недоступные проверки и политика организации каждый производят своё собственное сообщение, так что эта ошибка означает именно шлюз развёртывания.

### «Couldn't verify Remote Control eligibility»

Claude Code не смог дотянуться до сервиса флагов функций, чтобы проверить, включён ли Remote Control для вашего аккаунта, обычно из-за того, что вы офлайн или прокси блокирует запрос. Повторите попытку, когда получите доступ к сети, или выполните `claude doctor` для подробностей. Связанное сообщение «Couldn't verify your organization's Remote Control policy» имеет ту же причину и то же решение. Оба сообщения были добавлены в v2.1.178.

### «Remote Control is only available when using Claude via api.anthropic.com»

Сессия не общается с Anthropic API напрямую, поэтому нет бэкенда claude.ai, с которым можно было бы сопрячься. Это происходит на Amazon Bedrock, Google Vertex AI и Microsoft Foundry. Начиная с v2.1.196, это также происходит, когда [`ANTHROPIC_BASE_URL`](https://code.claude.com/docs/en/env-vars) указывает на хост, отличный от `api.anthropic.com`, например на [LLM-шлюз](https://code.claude.com/docs/en/llm-gateway) или прокси, даже если вы входите через claude.ai. Сбросьте `ANTHROPIC_BASE_URL` и перезапустите сессию, чтобы использовать Remote Control.

### «Remote Control is disabled by your organization's policy»

У этой ошибки четыре разные причины. Сначала выполните `/status`, чтобы увидеть, какой метод входа и подписку вы используете.

* **Вы аутентифицированы с API-ключом или аккаунтом Console**: Remote Control требует OAuth через claude.ai. Выполните `/login` и выберите вариант claude.ai. Если в вашем окружении установлена `ANTHROPIC_API_KEY`, сбросьте её.
* **Владелец (Owner) не включил её для вашей организации**: Remote Control по умолчанию выключено на тарифах Team и Enterprise. Владелец может включить его на [claude.ai/admin-settings/claude-code](https://claude.ai/admin-settings/claude-code), включив переключатель **Remote Control**. Этот переключатель — серверная настройка организации.
* **Переключатель администратора неактивен (grayed out)**: у вашей организации есть конфигурация хранения данных или соответствия требованиям, которая несовместима с Remote Control. Это нельзя изменить из панели администратора. Свяжитесь со службой поддержки Anthropic, чтобы обсудить варианты.
* **В ошибке упоминается `disableRemoteControl`**: ваш IT-администратор отключил Remote Control на этом устройстве через [управляемые настройки](https://code.claude.com/docs/en/settings#settings-files), независимо от переключателя уровня всей организации.

### «Remote credentials fetch failed»

Claude Code не смог получить кратковременное учётное данное от Anthropic API, чтобы установить соединение. Перезапустите с `--verbose`, чтобы увидеть полную ошибку:

```bash
claude remote-control --verbose
```

Частые причины:

* Не выполнен вход: запустите `claude` и используйте `/login`, чтобы аутентифицироваться со своим аккаунтом claude.ai. Аутентификация по API-ключу для Remote Control не поддерживается.
* Проблема с сетью или прокси: файрвол или прокси могут блокировать исходящий HTTPS-запрос. Remote Control требует доступа к Anthropic API по порту 443.
* Сбой создания сессии: если вы также видите `Session creation failed — see debug log`, сбой произошёл раньше в процессе настройки. Проверьте, что ваша подписка активна.

### «Your organization requires Trusted Devices for Remote Control, but this device is not enrolled»

У вашей организации включены [доверенные устройства](#trusted-devices), а эта машина ещё не зарегистрирована. Выполните `/login` в Claude Code. Регистрация происходит как часть входа, и отдельной команды регистрации нет.

### «session expired for trusted-device check»

С момента вашего входа прошло более 18 часов. Выполните `/login` в Claude Code или подтвердите с помощью Face ID, Touch ID, Windows Hello или passkey, когда claude.ai или мобильное приложение это запросит. См. [доверенные устройства](#trusted-devices).

## Связанные ресурсы

* [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web): запускайте сессии в облачных окружениях, управляемых Anthropic, вместо своей машины
* [Ultraplan](https://code.claude.com/docs/en/ultraplan): запустите облачную сессию планирования из своего терминала и просмотрите план в браузере

* [Dispatch](https://code.claude.com/docs/en/desktop#sessions-from-dispatch): отправьте задачу сообщением с телефона, и она может породить сессию Desktop для её обработки
* [Аутентификация](https://code.claude.com/docs/en/authentication): настройте `/login` и управляйте учётными данными для claude.ai
* [Справочник по CLI](https://code.claude.com/docs/en/cli-reference): полный список флагов и команд, включая `claude remote-control`

* [Использование данных](https://code.claude.com/docs/en/data-usage): какие данные проходят через Anthropic API во время локальных и удалённых сессий

# Visual Studio Code

> Установите и настройте расширение Claude Code для VS Code. Получите помощь ИИ в написании кода с встроенными диффами, @-упоминаниями, проверкой планов и сочетаниями клавиш.

Расширение для VS Code предоставляет нативный графический интерфейс для Claude Code, интегрированный прямо в вашу IDE. Это рекомендуемый способ использования Claude Code в VS Code.

С этим расширением вы можете просматривать и редактировать планы Claude перед их принятием, автоматически принимать правки по мере их внесения, @-упоминать файлы с определёнными диапазонами строк из вашего выделения, обращаться к истории диалогов и открывать несколько диалогов в отдельных вкладках или окнах.

## Требования

Перед установкой убедитесь, что у вас есть:

* VS Code 1.98.0 или выше
* Учётная запись Anthropic: подойдёт любая платная подписка Claude (Pro, Max, Team или Enterprise) либо учётная запись Claude Console, при этом ключ API не требуется. Вы [войдёте](https://code.claude.com/docs/en/authentication#log-in-to-claude-code) с этой учётной записью при первом открытии расширения. Если вы обращаетесь к Claude через стороннего провайдера, например Amazon Bedrock или Google Vertex AI, см. инструкции по настройке в разделе [Использование сторонних провайдеров](#use-third-party-providers).

> **Совет.** Расширение включает собственную копию CLI (интерфейса командной строки) для панели чата. Чтобы запускать `claude` во встроенном терминале VS Code, вам также нужна [отдельная установка CLI](https://code.claude.com/docs/en/setup). Подробнее см. [Расширение для VS Code и CLI Claude Code](#vs-code-extension-vs-claude-code-cli).

## Установка расширения

Нажмите ссылку для вашей IDE, чтобы установить напрямую:

* [Установить для VS Code](vscode:extension/anthropic.claude-code)
* [Установить для Cursor](cursor:extension/anthropic.claude-code)

Либо в VS Code нажмите `Cmd+Shift+X` (Mac) или `Ctrl+Shift+X` (Windows/Linux), чтобы открыть представление Extensions, найдите «Claude Code» и нажмите **Install**.

Расширение также устанавливается в другие форки VS Code, такие как Devin Desktop или Kiro. Найдите «Claude Code» в представлении Extensions вашего редактора или установите из [реестра Open VSX](https://open-vsx.org/extension/Anthropic/claude-code). Если ваш редактор не может установить расширение, [установите CLI](https://code.claude.com/docs/en/quickstart) и запустите `claude` в его встроенном терминале. CLI работает в любом терминале.

> **Примечание.** Если расширение не появилось после установки, перезапустите VS Code или выполните «Developer: Reload Window» из палитры команд.

## Начало работы

После установки вы можете начать использовать Claude Code через интерфейс VS Code:

1. **Откройте панель Claude Code**

   По всему VS Code значок Spark обозначает Claude Code: <img src="https://mintcdn.com/claude-code/c5r9_6tjPMzFdDDT/images/vs-code-spark-icon.svg?fit=max&auto=format&n=c5r9_6tjPMzFdDDT&q=85&s=3ca45e00deadec8c8f4b4f807da94505" alt="Значок Spark" style={{display: "inline", height: "0.85em", verticalAlign: "middle"}} width="16" height="16" data-path="images/vs-code-spark-icon.svg" />

   Самый быстрый способ открыть Claude — нажать значок Spark в **панели инструментов редактора** (Editor Toolbar, правый верхний угол редактора). Значок появляется только когда у вас открыт файл.

   <img src="https://mintcdn.com/claude-code/mfM-EyoZGnQv8JTc/images/vs-code-editor-icon.png?fit=max&auto=format&n=mfM-EyoZGnQv8JTc&q=85&s=eb4540325d94664c51776dbbfec4cf02" alt="Редактор VS Code со значком Spark в панели инструментов редактора" width="2796" height="734" data-path="images/vs-code-editor-icon.png" />

   Другие способы открыть Claude Code:

   * **Панель действий (Activity Bar)**: нажмите значок Spark на левой боковой панели, чтобы открыть список сессий. Нажмите любую сессию, чтобы открыть её как полноценную вкладку редактора, или начните новую. Этот значок всегда виден в панели действий.
   * **Палитра команд**: `Cmd+Shift+P` (Mac) или `Ctrl+Shift+P` (Windows/Linux), введите «Claude Code» и выберите вариант, например «Open in New Tab»
   * **Строка состояния (Status Bar)**: нажмите **✱ Claude Code** в правом нижнем углу окна. Это работает даже когда ни один файл не открыт.

   Вы можете перетащить панель Claude, чтобы разместить её в любом месте VS Code. Подробнее см. [Настройка рабочего процесса](#customize-your-workflow).

2. **Войдите в систему**

   При первом открытии панели появляется экран входа. Нажмите **Sign in** и завершите авторизацию в браузере.

   Если позже вы увидите **Not logged in · Please run /login**, расширение автоматически снова откроет экран входа. Если он не появляется, перезагрузите окно из палитры команд с помощью **Developer: Reload Window**.

   Если у вас в оболочке задана переменная `ANTHROPIC_API_KEY`, но вы всё равно видите приглашение войти, возможно, VS Code не унаследовал окружение вашей оболочки. Запустите VS Code из терминала командой `code .`, чтобы он унаследовал ваши переменные окружения, либо войдите со своей учётной записью Claude.

   После входа появляется контрольный список **Learn Claude Code**. Пройдите каждый пункт, нажимая **Show me**, или закройте его крестиком (X). Чтобы снова открыть его позже, снимите флажок **Hide Onboarding** в настройках VS Code в разделе Extensions → Claude Code.

3. **Отправьте промпт**

   Попросите Claude помочь с вашим кодом или файлами — будь то объяснение того, как что-то работает, отладка проблемы или внесение изменений.

   > **Совет.** Claude автоматически видит выделенный вами текст. Нажмите `Option+K` (Mac) / `Alt+K` (Windows/Linux), чтобы также вставить в свой промпт ссылку-@-упоминание (вроде `@file.ts#5-10`).

   Вот пример вопроса о конкретной строке в файле:

   <img src="https://mintcdn.com/claude-code/FVYz38sRY-VuoGHA/images/vs-code-send-prompt.png?fit=max&auto=format&n=FVYz38sRY-VuoGHA&q=85&s=ede3ed8d8d5f940e01c5de636d009cfd" alt="Редактор VS Code с выделенными строками 2-3 в файле Python и панелью Claude Code, где показан вопрос об этих строках со ссылкой-@-упоминанием" width="3288" height="1876" data-path="images/vs-code-send-prompt.png" />

4. **Просмотрите изменения**

   Когда Claude хочет отредактировать файл, он показывает сравнение оригинала и предлагаемых изменений бок о бок, а затем запрашивает разрешение. Вы можете принять, отклонить или сказать Claude, что делать вместо этого. Если вы отредактируете предлагаемое содержимое прямо в представлении диффа перед принятием, Claude сообщается, что вы его изменили, чтобы он не предполагал, что файл соответствует его исходному предложению.

   <img src="https://mintcdn.com/claude-code/FVYz38sRY-VuoGHA/images/vs-code-edits.png?fit=max&auto=format&n=FVYz38sRY-VuoGHA&q=85&s=e005f9b41c541c5c7c59c082f7c4841c" alt="VS Code показывает дифф предлагаемых Claude изменений с приглашением-разрешением, спрашивающим, вносить ли правку" width="3292" height="1876" data-path="images/vs-code-edits.png" />

Больше идей о том, что можно делать с Claude Code, см. в разделе [Типовые рабочие процессы](https://code.claude.com/docs/en/common-workflows).

> **Совет.** Выполните «Claude Code: Open Walkthrough» из палитры команд для наглядного знакомства с основами.

## Использование поля ввода промпта

Поле ввода промпта поддерживает несколько возможностей:

* **Режимы разрешений**: нажмите индикатор режима внизу поля ввода промпта, чтобы переключить режимы. В обычном режиме Claude запрашивает разрешение перед каждым действием. В режиме планирования (Plan mode) Claude описывает, что он собирается сделать, и ждёт одобрения перед внесением изменений. VS Code автоматически открывает план как полноценный markdown-документ, где вы можете добавлять встроенные комментарии, чтобы дать обратную связь до того, как Claude начнёт. В режиме автопринятия (auto-accept) Claude вносит правки без запроса. Задайте значение по умолчанию в настройках VS Code через `claudeCode.initialPermissionMode`.
* **Меню команд**: нажмите `/` или введите `/`, чтобы открыть меню команд. Среди вариантов — прикрепление файлов, переключение моделей, включение расширенного мышления, просмотр использования плана (`/usage`) и запуск сессии [Remote Control](https://code.claude.com/docs/en/remote-control) (`/remote-control`). Раздел Customize предоставляет доступ к серверам MCP, хукам, памяти, разрешениям и плагинам. Пункты со значком терминала открываются во встроенном терминале.
* **Индикатор контекста**: поле ввода промпта показывает, какую часть окна контекста Claude вы используете. Claude автоматически уплотняет контекст, когда это необходимо, либо вы можете выполнить `/compact` вручную.
* **Расширенное мышление**: позволяет Claude уделять больше времени рассуждениям над сложными задачами. Включите его через меню команд (`/`). Рассуждения Claude появляются в диалоге в виде свёрнутых блоков: нажмите на блок, чтобы прочитать его, или нажмите `Ctrl+O`, чтобы развернуть или свернуть все блоки мышления в сессии. Подробнее см. [Расширенное мышление](https://code.claude.com/docs/en/model-config#extended-thinking).
* **Многострочный ввод**: нажмите `Shift+Enter`, чтобы добавить новую строку без отправки. Это также работает в свободном текстовом поле «Other» в диалогах с вопросами.

### Ссылки на файлы и папки

Используйте @-упоминания, чтобы дать Claude контекст о конкретных файлах или папках. Когда вы вводите `@`, за которым следует имя файла или папки, Claude читает это содержимое и может отвечать на вопросы о нём или вносить в него изменения. Claude Code поддерживает нечёткое сопоставление, поэтому вы можете вводить частичные имена, чтобы найти нужное:

```text
> Explain the logic in @auth (fuzzy matches auth.js, AuthService.ts, etc.)
> What's in @src/components/ (include a trailing slash for folders)
```

Для больших PDF-файлов вы можете попросить Claude прочитать конкретные страницы вместо всего файла: одну страницу, диапазон вроде страниц 1-10 или открытый диапазон, например с 3-й страницы и далее.

Когда вы выделяете текст в редакторе, Claude может автоматически видеть ваш выделенный код. В нижней части поля ввода промпта показывается, сколько строк выделено. Нажмите `Option+K` (Mac) / `Alt+K` (Windows/Linux), чтобы вставить @-упоминание с путём к файлу и номерами строк (например, `@app.ts#5-10`). Нажмите индикатор выделения, чтобы переключить, может ли Claude видеть ваш выделенный текст — значок перечёркнутого глаза означает, что выделение скрыто от Claude.

Вы также можете, удерживая `Shift`, перетаскивать файлы в поле ввода промпта, чтобы добавить их как вложения. Нажмите крестик (X) на любом вложении, чтобы удалить его из контекста.

### Возобновление прошлых диалогов

Нажмите кнопку **Session history** в верхней части панели Claude Code, чтобы получить доступ к истории диалогов. Вы можете искать по ключевому слову или просматривать по времени (Today, Yesterday, Last 7 days и т. д.). Нажмите любой диалог, чтобы возобновить его с полной историей сообщений. Новые сессии получают сгенерированные ИИ заголовки на основе вашего первого сообщения. Наведите курсор на сессию, чтобы показать действия переименования и удаления: переименуйте, чтобы дать ей описательный заголовок, или удалите, чтобы убрать из списка. Подробнее о возобновлении сессий см. [Управление сессиями](https://code.claude.com/docs/en/sessions).

### Возобновление облачных сессий из Claude.ai

Если вы используете [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web), вы можете возобновлять эти облачные сессии прямо в VS Code. Для этого требуется вход через **Claude.ai Subscription**, а не Anthropic Console.

1. **Откройте историю сессий**

   Нажмите кнопку **Session history** в верхней части панели Claude Code.

2. **Выберите вкладку Remote**

   В диалоге показаны две вкладки: Local и Remote. Нажмите **Remote**, чтобы увидеть сессии с claude.ai.

3. **Выберите сессию для возобновления**

   Просматривайте или ищите свои облачные сессии. Нажмите любую сессию, чтобы скачать её и продолжить диалог локально.

> **Примечание.** Во вкладке Remote появляются только веб-сессии, начатые с репозиторием GitHub. Возобновление загружает историю диалога локально; изменения не синхронизируются обратно в claude.ai.

### Проверка учётной записи и использования

Выполните `/usage` из меню команд, чтобы открыть диалог Account & usage. В нём показаны учётная запись, под которой вы вошли, ваш план, а также полосы использования для текущей сессии и недели с указанием, сколько осталось до сброса каждого лимита.

Диалог также разбивает по составляющим, что вносит вклад в лимиты вашего плана. Он выделяет поведение, которое составляет 10% или более недавнего использования, например промахи кэша, длинный контекст, а также сессии с интенсивным использованием субагентов или высокой параллельностью, каждое с советом по его сокращению. Таблицы атрибуции показывают, сколько использования пришлось на каждый навык, субагента, плагин и сервер MCP. Требуется Claude Code v2.1.174 или новее.

Используйте переключатель Day и Week, чтобы переключаться между последними 24 часами и последними 7 днями. Показатели приблизительны и вычисляются из локальных сессий на этой машине, поэтому использование с других устройств или с claude.ai не учитывается. Подробнее об отслеживании и сокращении использования см. [Отслеживание затрат](https://code.claude.com/docs/en/costs#track-your-costs).

## Настройка рабочего процесса

Когда всё запущено и работает, вы можете переместить панель Claude, запускать несколько сессий или переключаться в терминальный режим.

### Выберите, где будет находиться Claude

Вы можете перетащить панель Claude, чтобы разместить её в любом месте VS Code. Возьмите вкладку панели или её заголовок и перетащите в:

* **Вторичную боковую панель (Secondary sidebar)**: правая сторона окна. Держит Claude на виду, пока вы пишете код.
* **Основную боковую панель (Primary sidebar)**: левая боковая панель со значками Explorer, Search и т. д.
* **Область редактора (Editor area)**: открывает Claude как вкладку рядом с вашими файлами. Удобно для побочных задач.

> **Совет.** Используйте боковую панель для вашей основной сессии Claude и открывайте дополнительные вкладки для побочных задач. Claude запоминает предпочитаемое вами расположение. Значок списка сессий в панели действий отделён от панели Claude: список сессий всегда виден в панели действий, тогда как значок панели Claude появляется там только когда панель закреплена в левой боковой панели.

### Запуск нескольких диалогов

Используйте **Open in New Tab** или **Open in New Window** из палитры команд, чтобы начать дополнительные диалоги. Каждый диалог сохраняет собственную историю и контекст, что позволяет вам работать над разными задачами параллельно.

При использовании вкладок небольшая цветная точка на значке Spark указывает статус: синяя означает, что ожидается запрос разрешения, оранжевая — что Claude завершил работу, пока вкладка была скрыта.

### Переключение в терминальный режим

По умолчанию расширение открывает графическую панель чата. Если вы предпочитаете интерфейс в стиле CLI, откройте [настройку Use Terminal](vscode://settings/claudeCode.useTerminal) и установите флажок.

Вы также можете открыть настройки VS Code (`Cmd+,` на Mac или `Ctrl+,` на Windows/Linux), перейти в Extensions → Claude Code и установить флажок **Use Terminal**.

## Управление плагинами

Расширение для VS Code включает графический интерфейс для установки и управления [плагинами](https://code.claude.com/docs/en/plugins). Введите `/plugins` в поле ввода промпта, чтобы открыть интерфейс **Manage plugins**.

### Установка плагинов

Диалог плагинов показывает две вкладки: **Plugins** и **Marketplaces**.

Во вкладке Plugins:

* **Установленные плагины** отображаются вверху с переключателями для включения или отключения
* **Доступные плагины** из ваших настроенных маркетплейсов отображаются ниже
* Используйте поиск, чтобы отфильтровать плагины по имени или описанию
* Нажмите **Install** на любом доступном плагине

Когда вы устанавливаете плагин, выберите область установки:

* **Install for you**: доступен во всех ваших проектах (пользовательская область)
* **Install for this project**: разделяется с участниками проекта (область проекта)
* **Install locally**: только для вас, только в этом репозитории (локальная область)

### Управление маркетплейсами

Переключитесь на вкладку **Marketplaces**, чтобы добавить или удалить источники плагинов:

* Введите репозиторий GitHub, URL или локальный путь, чтобы добавить новый маркетплейс
* Нажмите значок обновления, чтобы обновить список плагинов маркетплейса
* Нажмите значок корзины, чтобы удалить маркетплейс

После внесения изменений баннер предложит вам перезапустить Claude Code, чтобы применить обновления.

> **Примечание.** Управление плагинами в VS Code использует те же команды CLI под капотом. Плагины и маркетплейсы, которые вы настраиваете в расширении, также доступны в CLI, и наоборот.

Подробнее о системе плагинов см. [Плагины](https://code.claude.com/docs/en/plugins) и [Маркетплейсы плагинов](https://code.claude.com/docs/en/plugin-marketplaces).

## Автоматизация задач в браузере с Chrome

Подключите Claude к вашему браузеру Chrome, чтобы тестировать веб-приложения, отлаживать с помощью логов консоли и автоматизировать браузерные рабочие процессы, не покидая VS Code. Для этого требуется [расширение Claude in Chrome](https://chromewebstore.google.com/detail/claude/fcoeoabgfenejglbffodgkkbkcdhcgfn) версии 1.0.36 или выше.

Введите `@browser` в поле ввода промпта, а затем то, что вы хотите, чтобы Claude сделал:

```text
@browser go to localhost:3000 and check the console for errors
```

Вы также можете открыть меню вложений, чтобы выбрать конкретные браузерные инструменты, например открыть новую вкладку или прочитать содержимое страницы.

Claude открывает новые вкладки для браузерных задач и разделяет состояние входа вашего браузера, поэтому он может получить доступ к любому сайту, в который вы уже вошли.

Инструкции по настройке, полный список возможностей и устранение неполадок см. в разделе [Использование Claude Code с Chrome](https://code.claude.com/docs/en/chrome).

### Запуск вкладки VS Code из других инструментов

Расширение регистрирует обработчик URI по адресу `vscode://anthropic.claude-code/open`. Используйте его, чтобы открыть новую вкладку Claude Code из вашего собственного инструментария: псевдонима оболочки, букмарклета браузера или любого скрипта, способного открыть URL. Если VS Code ещё не запущен, открытие URL сначала запустит его. Если VS Code уже запущен, URL откроется в том окне, которое сейчас в фокусе.

Вызовите обработчик с помощью средства открытия URL вашей операционной системы.

### macOS

```bash
open "vscode://anthropic.claude-code/open"
```

### Linux

```bash
xdg-open "vscode://anthropic.claude-code/open"
```

### Windows

В PowerShell:

```powershell
Start-Process "vscode://anthropic.claude-code/open"
```

В `cmd.exe` команда `start` трактует свой первый аргумент в кавычках как заголовок окна, поэтому передайте пустой заголовок перед URL:

```cmd
start "" "vscode://anthropic.claude-code/open"
```

Обработчик принимает два необязательных параметра запроса:

| Параметр  | Описание                                                                                                                                                                                                                                                                                                                                                                       |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `prompt`  | Текст для предварительного заполнения поля ввода промпта. Должен быть закодирован для URL. Промпт заполняется заранее, но не отправляется автоматически.                                                                                                                                                                                                                          |
| `session` | Идентификатор сессии для возобновления вместо начала нового диалога. Сессия должна принадлежать рабочей области, открытой в данный момент в VS Code. Если сессия не найдена, вместо неё начинается новый диалог. Если сессия уже открыта во вкладке, эта вкладка фокусируется. Как захватить идентификатор сессии программно, см. [Продолжение диалогов](https://code.claude.com/docs/en/headless#continue-conversations). |

Например, чтобы открыть вкладку, предварительно заполненную текстом «review my changes»:

```text
vscode://anthropic.claude-code/open?prompt=review%20my%20changes
```

Чтобы запустить терминальную сессию вместо вкладки VS Code, используйте обработчик CLI `claude-cli://`. См. [Запуск сессий по ссылкам](https://code.claude.com/docs/en/deep-links).

## Настройка параметров

* **Настройки расширения** в VS Code: управляют поведением расширения внутри VS Code. Откройте с помощью `Cmd+,` (Mac) или `Ctrl+,` (Windows/Linux), затем перейдите в Extensions → Claude Code. Вы также можете ввести `/` и выбрать **General Config**, чтобы открыть настройки.

## Расширение для VS Code и CLI Claude Code

Claude Code доступен и как расширение для VS Code (графическая панель), и как CLI (интерфейс командной строки в терминале). Некоторые возможности доступны только в CLI. Если вам нужна возможность, доступная только в CLI, запустите `claude` во встроенном терминале VS Code. Для этого требуется [отдельная установка CLI](https://code.claude.com/docs/en/setup): расширение не добавляет `claude` в ваш PATH. См. [Запуск CLI в VS Code](#run-cli-in-vs-code).

| Возможность          | CLI                 | Расширение для VS Code                                                                |
| ------------------- | ------------------- | ------------------------------------------------------------------------------------ |
| Команды и навыки    | [Все](https://code.claude.com/docs/en/commands) | Подмножество (введите `/`, чтобы увидеть доступные)                       |
| Настройка сервера MCP | Да                | Частично (добавляйте серверы через CLI; управляйте существующими серверами через `/mcp` в панели чата) |
| Контрольные точки   | Да                  | Да                                                                                   |
| Быстрая команда `!` bash | Да             | Нет                                                                                  |
| Автодополнение по Tab | Да                | Нет                                                                                  |

### Откат назад с контрольными точками

Расширение для VS Code поддерживает контрольные точки, которые отслеживают правки Claude в файлах и позволяют откатиться к предыдущему состоянию. Наведите курсор на любое сообщение, чтобы показать кнопку отката, затем выберите один из трёх вариантов:

* **Fork conversation from here**: начать новую ветвь диалога от этого сообщения, сохранив все изменения кода в целости
* **Rewind code to here**: откатить изменения файлов обратно к этой точке в диалоге, сохранив полную историю диалога
* **Fork conversation and rewind code**: начать новую ветвь диалога и откатить изменения файлов к этой точке

Полные сведения о том, как работают контрольные точки, и об их ограничениях см. в разделе [Контрольные точки](https://code.claude.com/docs/en/checkpointing).

### Запуск CLI в VS Code

Чтобы использовать CLI, оставаясь в VS Code, откройте встроенный терминал (`` Ctrl+` `` на Windows/Linux или `` Cmd+` `` на Mac) и запустите `claude`. CLI автоматически интегрируется с вашей IDE для таких функций, как просмотр диффов и обмен диагностикой.

Установка расширения не помещает `claude` в PATH вашей оболочки. Расширение включает приватную копию CLI для своей панели чата, но чтобы набирать `claude` в терминале, требуется [отдельная установка CLI](https://code.claude.com/docs/en/setup). Выполните установку один раз, и команды с этой страницы, включая `claude mcp add` и `claude --resume`, будут работать в любом терминале. Если `claude` по-прежнему не найден после установки, [проверьте ваш PATH](https://code.claude.com/docs/en/troubleshoot-install#verify-your-path).

Если вы используете внешний терминал, выполните `/ide` внутри Claude Code, чтобы подключить его к VS Code.

### Переключение между расширением и CLI

Расширение и CLI разделяют одну и ту же историю диалогов. Чтобы продолжить диалог расширения в CLI, выполните `claude --resume` в терминале. Это откроет интерактивный выбор, где вы можете найти и выбрать свой диалог.

### Включение вывода терминала в промпты

Ссылайтесь на вывод терминала в своих промптах через `@terminal:name`, где `name` — заголовок терминала. Это позволяет Claude видеть вывод команд, сообщения об ошибках или логи без копирования-вставки.

### Мониторинг фоновых процессов

Когда Claude запускает длительные команды, расширение показывает прогресс в строке состояния. Однако видимость фоновых задач ограничена по сравнению с CLI. Для лучшей видимости попросите Claude вывести команду, чтобы вы могли запустить её во встроенном терминале VS Code.

### Подключение к внешним инструментам через MCP

Серверы MCP (Model Context Protocol) дают Claude доступ к внешним инструментам, базам данных и API.

Чтобы добавить сервер MCP, откройте встроенный терминал (`` Ctrl+` `` или `` Cmd+` ``) и выполните `claude mcp add`. В примере ниже добавляется удалённый сервер MCP от GitHub, который аутентифицируется с помощью [персонального токена доступа](https://github.com/settings/personal-access-tokens), передаваемого как заголовок:

```bash
claude mcp add --transport http github https://api.githubcopilot.com/mcp/ \
  --header "Authorization: Bearer YOUR_GITHUB_PAT"
```

После настройки попросите Claude использовать эти инструменты (например, «Review PR #456»).

Чтобы управлять серверами MCP, не покидая VS Code, введите `/mcp` в панели чата. Диалог управления MCP позволяет включать или отключать серверы, переподключаться к серверу и управлять аутентификацией OAuth. Доступные серверы см. в [документации по MCP](https://code.claude.com/docs/en/mcp).

## Работа с git

Claude Code интегрируется с git, чтобы помогать с рабочими процессами контроля версий прямо в VS Code. Попросите Claude закоммитить изменения, создать pull request'ы или работать с несколькими ветками.

### Создание коммитов и pull request'ов

Claude может индексировать изменения, писать сообщения коммитов и создавать pull request'ы на основе вашей работы:

```text
> commit my changes with a descriptive message
> create a pr for this feature
> summarize the changes I've made to the auth module
```

При создании pull request'ов Claude генерирует описания на основе реальных изменений кода и может добавить контекст о тестировании или решениях по реализации.

### Использование git worktree для параллельных задач

Используйте флаг `--worktree` (`-w`), чтобы запустить Claude в изолированном worktree с собственными файлами и веткой:

```bash
claude --worktree feature-auth
```

Каждый worktree сохраняет независимое состояние файлов, разделяя при этом историю git. Это не даёт экземплярам Claude мешать друг другу при работе над разными задачами. Подробнее см. [Запуск параллельных сессий с помощью Git worktree](https://code.claude.com/docs/en/worktrees).

## Использование сторонних провайдеров

По умолчанию Claude Code подключается напрямую к API Anthropic. Если ваша организация использует Amazon Bedrock, Google Vertex AI или Microsoft Foundry для доступа к Claude, настройте расширение на использование вашего провайдера:

1. **Отключите приглашение входа**

   Откройте [настройку Disable Login Prompt](vscode://settings/claudeCode.disableLoginPrompt) и установите флажок.

   Вы также можете открыть настройки VS Code (`Cmd+,` на Mac или `Ctrl+,` на Windows/Linux), найти «Claude Code login» и установить флажок **Disable Login Prompt**.

2. **Настройте своего провайдера**

   Следуйте руководству по настройке для вашего провайдера:

   * [Claude Code на Amazon Bedrock](https://code.claude.com/docs/en/amazon-bedrock)
   * [Claude Code на Google Vertex AI](https://code.claude.com/docs/en/google-vertex-ai)
   * [Claude Code на Microsoft Foundry](https://code.claude.com/docs/en/microsoft-foundry)

   Эти руководства охватывают настройку вашего провайдера в `~/.claude/settings.json`, что гарантирует, что ваши настройки разделяются между расширением для VS Code и CLI.

## Безопасность и конфиденциальность

Ваш код остаётся приватным. Claude Code обрабатывает ваш код, чтобы оказывать помощь, но не использует его для обучения моделей. Подробности об обращении с данными и о том, как отказаться от логирования, см. в разделе [Данные и конфиденциальность](https://code.claude.com/docs/en/data-usage).

При включённых разрешениях на автоматическое редактирование Claude Code может изменять конфигурационные файлы VS Code (такие как `settings.json` или `tasks.json`), которые VS Code может выполнять автоматически. Чтобы снизить риск при работе с недоверенным кодом:

* Включите [ограниченный режим VS Code (Restricted Mode)](https://code.visualstudio.com/docs/editor/workspace-trust#_restricted-mode) для недоверенных рабочих областей
* Используйте режим ручного одобрения вместо автопринятия правок
* Тщательно проверяйте изменения перед их принятием

### Встроенный сервер IDE MCP

Когда расширение активно, оно запускает локальный сервер MCP, к которому CLI подключается автоматически. Именно так CLI открывает диффы в нативном средстве просмотра диффов VS Code, читает ваше текущее выделение для `@`-упоминаний и — когда вы работаете в блокноте Jupyter — просит VS Code выполнить ячейки.

Сервер называется `ide` и скрыт из `/mcp`, потому что в нём нечего настраивать. Однако если ваша организация использует хук `PreToolUse` для формирования белого списка инструментов MCP, вам нужно знать, что он существует.

**Контекст выделения и открытого файла.** Пока сервер подключён, CLI включает ваше текущее выделение в редакторе и путь к активному файлу как контекст в каждый отправляемый вами промпт. Транскрипт показывает строку `⧉ Selected N lines from <file>`, когда это происходит. Чтобы исключить чувствительный файл, например `.env`, добавьте [правило запрета `Read`](https://code.claude.com/docs/en/permissions#read-and-edit) для его пути. Совпадающее правило запрета не даёт ни выделенному тексту, ни уведомлению об открытом файле для этого файла попасть к Claude.

**Транспорт и аутентификация.** Сервер привязывается к `127.0.0.1` на случайном высоком порту и недоступен с других машин. Каждая активация расширения генерирует новый случайный токен аутентификации, который CLI должен предъявить для подключения. Токен записывается в файл блокировки в `~/.claude/ide/` с правами `0600` в каталоге с правами `0700`, поэтому прочитать его может только пользователь, запустивший VS Code.

**Инструменты, доступные модели.** Сервер размещает дюжину инструментов, но только два видны модели. Остальные — это внутренний RPC, который CLI использует для собственного интерфейса — открытие диффов, чтение выделений, сохранение файлов — и они отфильтровываются до того, как список инструментов достигнет Claude.

| Имя инструмента (как видно хукам) | Что он делает                                                                                                            | Пишет? |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------- | ------- |
| `mcp__ide__getDiagnostics`   | Возвращает диагностику языкового сервера — ошибки и предупреждения из панели Problems VS Code. Опционально ограничивается одним файлом. | Нет     |
| `mcp__ide__executeCode`      | Запускает код Python в ядре активного блокнота Jupyter. См. процесс подтверждения ниже.                                    | Да      |

**Выполнение в Jupyter всегда сначала спрашивает.** `mcp__ide__executeCode` не может ничего выполнить незаметно. При каждом вызове код вставляется как новая ячейка в конец активного блокнота, VS Code прокручивает её в поле зрения, а нативный Quick Pick просит вас выбрать **Execute** или **Cancel**. Отмена — или закрытие выбора клавишей `Esc` — возвращает Claude ошибку, и ничего не запускается. Инструмент также напрямую отказывает, когда нет активного блокнота, когда расширение Jupyter (`ms-toolsai.jupyter`) не установлено или когда ядро не Python.

> **Примечание.** Подтверждение Quick Pick отделено от хуков `PreToolUse`. Запись в белом списке для `mcp__ide__executeCode` позволяет Claude *предложить* запуск ячейки; Quick Pick внутри VS Code — это то, что позволяет ему *реально* запустить её.

## Устранение распространённых проблем

### Расширение не устанавливается

* Убедитесь, что у вас совместимая версия VS Code (1.98.0 или новее)
* Проверьте, что у VS Code есть разрешение на установку расширений
* Попробуйте установить напрямую из [VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=anthropic.claude-code)

### Значок Spark не виден

Значок Spark появляется в **панели инструментов редактора** (Editor Toolbar, правый верхний угол редактора), когда у вас открыт файл. Если вы его не видите:

1. **Откройте файл**: значку требуется открытый файл. Открытой только папки недостаточно.
2. **Проверьте версию VS Code**: требуется 1.98.0 или выше (Help → About)
3. **Перезапустите VS Code**: выполните «Developer: Reload Window» из палитры команд
4. **Отключите конфликтующие расширения**: временно отключите другие ИИ-расширения (Cline, Continue и т. д.)
5. **Проверьте доверие к рабочей области**: расширение не работает в ограниченном режиме (Restricted Mode)

Как вариант, нажмите «✱ Claude Code» в **строке состояния** (правый нижний угол). Это работает даже без открытого файла. Вы также можете использовать **палитру команд** (`Cmd+Shift+P` / `Ctrl+Shift+P`) и ввести «Claude Code».

### Cmd+Esc ничего не делает в macOS

В macOS Tahoe и новее системное сочетание Game Overlay по умолчанию привязано к `Cmd+Esc` и перехватывает нажатие клавиши до того, как оно достигнет VS Code. Чтобы освободить сочетание:

1. Откройте System Settings
2. Перейдите в Keyboard, затем Keyboard Shortcuts, затем Game Controllers
3. Снимите флажок Game Overlay

Как вариант, переназначьте расширение на другую клавишу: откройте [редактор сочетаний клавиш VS Code](https://code.visualstudio.com/docs/configure/keybindings) (`Cmd+K Cmd+S`), найдите `Claude Code: Focus input` и назначьте новую привязку.

### Claude Code никогда не отвечает

Если Claude Code не отвечает на ваши промпты:

1. **Проверьте ваше интернет-соединение**: убедитесь, что у вас стабильное интернет-соединение
2. **Начните новый диалог**: попробуйте начать свежий диалог, чтобы увидеть, сохраняется ли проблема
3. **Попробуйте CLI**: запустите `claude` из терминала, чтобы увидеть, получите ли вы более подробные сообщения об ошибках

Если проблемы сохраняются, [создайте issue на GitHub](https://github.com/anthropics/claude-code/issues) с подробностями об ошибке.

## Удаление расширения

Чтобы удалить расширение Claude Code:

1. Откройте представление Extensions (`Cmd+Shift+X` на Mac или `Ctrl+Shift+X` на Windows/Linux)
2. Найдите «Claude Code»
3. Нажмите **Uninstall**

Запуск `claude` во встроенном терминале VS Code переустанавливает расширение автоматически. Чтобы оно оставалось удалённым, отключите **Auto-install IDE extension** в `/config` или установите [`autoInstallIdeExtension`](https://code.claude.com/docs/en/settings#global-config-settings) в `false`. Вы также можете установить переменную окружения [`CLAUDE_CODE_IDE_SKIP_AUTO_INSTALL`](https://code.claude.com/docs/en/env-vars) в `1`.

Чтобы также удалить данные расширения и сбросить все настройки, удалите каталог хранилища расширения для вашей платформы.

На macOS:

```bash
rm -rf ~/Library/"Application Support"/Code/User/globalStorage/anthropic.claude-code
```

На Linux:

```bash
rm -rf ~/.config/Code/User/globalStorage/anthropic.claude-code
```

На Windows, в PowerShell:

```powershell
Remove-Item -Recurse -Force "$env:APPDATA\Code\User\globalStorage\anthropic.claude-code"
```

За дополнительной помощью обратитесь к [руководству по устранению неполадок](https://code.claude.com/docs/en/troubleshooting).

## Дальнейшие шаги

Теперь, когда у вас настроен Claude Code в VS Code:

* [Изучите типовые рабочие процессы](https://code.claude.com/docs/en/common-workflows), чтобы получить максимум от Claude Code
* [Настройте серверы MCP](https://code.claude.com/docs/en/mcp), чтобы расширить возможности Claude внешними инструментами. Добавляйте серверы с помощью CLI, затем управляйте ими через `/mcp` в панели чата.
* [Настройте параметры Claude Code](https://code.claude.com/docs/en/settings), чтобы кастомизировать разрешённые команды, хуки и многое другое. Эти настройки разделяются между расширением и CLI.

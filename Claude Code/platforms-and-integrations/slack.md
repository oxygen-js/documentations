# Claude Code в Slack

> Делегируйте задачи по написанию кода прямо из вашего рабочего пространства Slack

> **Примечание.** Claude Code в Slack заменяется на [Claude Tag](https://claude.com/docs/claude-tag/overview) для рабочих пространств Team и Enterprise. Claude Tag запускает @Claude как общую идентичность вашей организации с настраиваемым администратором доступом, в рамках того же приложения Slack, поэтому переустанавливать ничего не нужно, а существующие настройки продолжают работать в период перехода. Чтобы переключить рабочее пространство, см. [Миграция с более ранней версии Claude в Slack](https://claude.com/docs/claude-tag/admins/migrate-from-earlier).

Claude Code в Slack привносит всю мощь Claude Code прямо в ваше рабочее пространство Slack. Когда вы упоминаете `@Claude` с задачей по написанию кода, Claude автоматически распознаёт намерение и создаёт сессию Claude Code в вебе, позволяя вам делегировать работу по разработке, не покидая командные обсуждения.

Эта интеграция построена на существующем приложении Claude for Slack, но добавляет интеллектуальную маршрутизацию к Claude Code в вебе для запросов, связанных с написанием кода. Каждая сессия выполняется под вашей собственной учётной записью Claude, используя ваши подключённые репозитории и лимиты вашего тарифного плана.

## Сценарии использования

* **Исследование и исправление багов**: попросите Claude исследовать и исправить баги сразу же, как только о них сообщают в каналах Slack.
* **Быстрые обзоры кода и модификации**: поручите Claude реализовать небольшие функции или отрефакторить код на основе обратной связи от команды.
* **Совместная отладка**: когда командные обсуждения предоставляют критически важный контекст (например, воспроизведение ошибок или сообщения пользователей), Claude может использовать эту информацию для формирования своего подхода к отладке.
* **Параллельное выполнение задач**: запускайте задачи по написанию кода в Slack, продолжая при этом другую работу, и получайте уведомления по завершении.

## Предварительные требования

Прежде чем использовать Claude Code в Slack, убедитесь, что у вас есть следующее:

| Требование             | Подробности                                                                                        |
| :--------------------- | :------------------------------------------------------------------------------------------------ |
| Тарифный план Claude   | Pro, Max, Team или Enterprise с доступом к Claude Code (премиум-места или места Chat + Claude Code) |
| Claude Code в вебе     | Доступ к [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web) должен быть включён |
| Учётная запись GitHub  | Подключена к Claude Code в вебе с как минимум одним аутентифицированным репозиторием               |
| Аутентификация Slack   | Ваша учётная запись Slack связана с вашей учётной записью Claude через приложение Claude            |

## Настройка Claude Code в Slack

1. **Установите приложение Claude в Slack**

   Администратор рабочего пространства должен установить приложение Claude из Slack App Marketplace. Перейдите в [Slack App Marketplace](https://slack.com/marketplace/A08SF47R6P4) и нажмите «Add to Slack», чтобы начать процесс установки.

2. **Подключите свою учётную запись Claude**

   После установки приложения аутентифицируйте свою личную учётную запись Claude:

   1. Откройте приложение Claude в Slack, нажав «Claude» в разделе Apps
   2. Перейдите на вкладку App Home
   3. Нажмите «Connect», чтобы связать свою учётную запись Slack с учётной записью Claude
   4. Завершите процесс аутентификации в браузере

3. **Настройте Claude Code в вебе**

   Убедитесь, что ваш Claude Code в вебе правильно настроен:

   * Перейдите на [claude.ai/code](https://claude.ai/code) и войдите в ту же учётную запись, которую вы подключили к Slack
   * Подключите свою учётную запись GitHub, если она ещё не подключена
   * Аутентифицируйте как минимум один репозиторий, с которым вы хотите, чтобы Claude работал

4. **Выберите режим маршрутизации**

   После подключения учётных записей настройте, как Claude обрабатывает ваши сообщения в Slack. Перейдите в Claude App Home в Slack, чтобы найти настройку **Routing Mode**.

   | Режим            | Поведение                                                                                                                                                                                                                                 |
   | :--------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Code only**    | Claude направляет все @упоминания в сессии Claude Code. Лучший вариант для команд, использующих Claude в Slack исключительно для задач по разработке.                                                                                        |
   | **Code + Chat**  | Claude анализирует каждое сообщение и интеллектуально направляет его между Claude Code (для задач по написанию кода) и Claude Chat (для написания текстов, анализа и общих вопросов). Лучший вариант для команд, которым нужна единая точка входа @Claude для всех типов работы. |

   > **Примечание.** В режиме Code + Chat, если Claude направил сообщение в Chat, а вам была нужна сессия по написанию кода, вы можете нажать «Retry as Code», чтобы вместо этого создать сессию Claude Code. Аналогично, если сообщение было направлено в Code, а вам была нужна сессия Chat, вы можете выбрать этот вариант в том же треде.

5. **Добавьте Claude в каналы**

   После установки Claude не добавляется автоматически ни в один канал. Чтобы использовать Claude в канале, пригласите его, введя `/invite @Claude` в этом канале. Claude может отвечать на @упоминания только в тех каналах, куда он был добавлен.

## Как это работает

### Автоматическое распознавание

Когда вы упоминаете @Claude в канале или треде Slack, Claude автоматически анализирует ваше сообщение, чтобы определить, является ли это задачей по написанию кода. Если Claude обнаруживает намерение написать код, он направит ваш запрос в Claude Code в вебе вместо того, чтобы отвечать как обычный чат-ассистент.

Вы также можете явно указать Claude обработать запрос как задачу по написанию кода, даже если он не распознал это автоматически.

> **Примечание.** Claude Code в Slack работает только в каналах (публичных или приватных). Он не работает в личных сообщениях (DM).

### Сбор контекста

**Из тредов**: когда вы @упоминаете Claude в треде, он собирает контекст из всех сообщений в этом треде, чтобы понять весь разговор.

**Из каналов**: при упоминании непосредственно в канале Claude просматривает недавние сообщения канала для получения релевантного контекста.

Этот контекст помогает Claude понять проблему, выбрать подходящий репозиторий и сформировать свой подход к задаче.

> **Предупреждение.** Когда @Claude вызывается в Slack, Claude получает доступ к контексту разговора, чтобы лучше понять ваш запрос. Claude может следовать указаниям из других сообщений в контексте, поэтому пользователям следует использовать Claude только в доверенных разговорах Slack.

### Ход сессии

1. **Инициация**: вы @упоминаете Claude с запросом на написание кода
2. **Распознавание**: Claude анализирует ваше сообщение и обнаруживает намерение написать код
3. **Создание сессии**: новая сессия Claude Code создаётся на claude.ai/code
4. **Обновления о ходе работы**: Claude публикует обновления статуса в вашем треде Slack по мере выполнения работы
5. **Завершение**: по окончании Claude @упоминает вас со сводкой и кнопками действий
6. **Обзор**: нажмите «View Session», чтобы увидеть полную стенограмму, или «Create PR», чтобы открыть pull request

## Элементы пользовательского интерфейса

### App Home

Вкладка App Home показывает статус вашего подключения и позволяет подключить или отключить вашу учётную запись Claude от Slack.

### Действия с сообщениями

* **View Session**: открывает полную сессию Claude Code в вашем браузере, где вы можете увидеть всю выполненную работу, продолжить сессию или отправить дополнительные запросы.
* **Create PR**: создаёт pull request непосредственно из изменений сессии.
* **Retry as Code**: если Claude изначально ответил как чат-ассистент, а вам была нужна сессия по написанию кода, нажмите эту кнопку, чтобы повторить запрос как задачу Claude Code.
* **Change Repo**: позволяет выбрать другой репозиторий, если Claude выбрал неправильно.

### Выбор репозитория

Claude автоматически выбирает репозиторий на основе контекста из вашего разговора в Slack. Если подходить могут несколько репозиториев, Claude может отобразить выпадающий список, позволяющий выбрать правильный.

## Доступ и разрешения

### Доступ на уровне пользователя

| Тип доступа              | Требование                                                          |
| :----------------------- | :----------------------------------------------------------------- |
| Сессии Claude Code       | Каждый пользователь выполняет сессии под своей учётной записью Claude |
| Использование и лимиты   | Сессии учитываются в лимитах тарифного плана конкретного пользователя |
| Доступ к репозиториям    | Пользователи могут получить доступ только к тем репозиториям, которые они подключили лично |
| История сессий           | Сессии отображаются в вашей истории Claude Code на claude.ai/code   |

### Доступ на уровне рабочего пространства

Администраторы рабочего пространства Slack контролируют, доступно ли приложение Claude в их рабочем пространстве:

| Элемент управления              | Описание                                                                                                          |
| :------------------------------ | :--------------------------------------------------------------------------------------------------------------- |
| Установка приложения            | Администраторы рабочего пространства решают, устанавливать ли приложение Claude из Slack App Marketplace           |
| Распространение в Enterprise Grid | Для организаций Enterprise Grid администраторы организации могут контролировать, какие рабочие пространства имеют доступ к приложению Claude |
| Удаление приложения             | Удаление приложения из рабочего пространства немедленно отзывает доступ для всех пользователей этого рабочего пространства |

### Контроль доступа на основе каналов

После установки Claude не добавляется автоматически ни в один канал. Пользователи должны явно приглашать Claude в каналы, где они хотят его использовать:

* **Требуется приглашение**: введите `/invite @Claude` в любом канале, чтобы добавить Claude в этот канал
* **Членство в канале контролирует доступ**: Claude может отвечать на @упоминания только в тех каналах, куда он был добавлен
* **Ограничение доступа через каналы**: администраторы могут контролировать, кто использует Claude Code, управляя тем, в какие каналы приглашён Claude и у кого есть доступ к этим каналам
* **Поддержка приватных каналов**: Claude работает как в публичных, так и в приватных каналах, что даёт командам гибкость в контроле видимости

Эта модель на основе каналов позволяет командам ограничить использование Claude Code определёнными каналами, обеспечивая дополнительный уровень контроля доступа помимо разрешений на уровне рабочего пространства.

## Что и где доступно

**В Slack**: вы увидите обновления статуса, итоговые сводки о завершении и кнопки действий. Полная стенограмма сохраняется и всегда доступна.

**В вебе**: полная сессия Claude Code с полной историей разговора, всеми изменениями кода, операциями с файлами и возможностью продолжить сессию или создать pull request.

Для учётных записей Enterprise и Team сессии, созданные из Claude в Slack, автоматически видны организации. Подробнее см. [Совместный доступ к сессиям Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web#share-sessions).

## Рекомендации

### Как писать эффективные запросы

* **Будьте конкретны**: включайте имена файлов, имена функций или сообщения об ошибках, когда это уместно.
* **Предоставляйте контекст**: упоминайте репозиторий или проект, если это неясно из разговора.
* **Определите успех**: объясните, как выглядит «готово» — должен ли Claude написать тесты? Обновить документацию? Создать PR?
* **Используйте треды**: отвечайте в тредах при обсуждении багов или функций, чтобы Claude мог собрать полный контекст.

### Когда использовать Slack, а когда веб

**Используйте Slack, когда**: контекст уже существует в обсуждении в Slack, вы хотите запустить задачу асинхронно или вы сотрудничаете с коллегами, которым нужна видимость.

**Используйте веб напрямую, когда**: вам нужно загрузить файлы, вам нужно взаимодействие в реальном времени во время разработки или вы работаете над более длинными и сложными задачами.

## Устранение неполадок

### «Claude Code is not enabled for your account»

Эта ошибка означает, что у вашей учётной записи Claude пока нет облачного окружения, а не то, что администратору нужно что-то включить. Войдите на [claude.ai/code](https://claude.ai/code) один раз с той же учётной записью, которую вы подключили к Slack. Первое посещение создаёт ваше облачное окружение по умолчанию, и ошибка исчезнет при вашем следующем упоминании. Каждый пользователь должен сделать это индивидуально.

### Сессии не запускаются

1. Убедитесь, что ваша учётная запись Claude подключена в Claude App Home
2. Проверьте, что у вас включён доступ к Claude Code в вебе
3. Убедитесь, что у вас есть как минимум один репозиторий GitHub, подключённый к Claude Code

### Репозиторий не отображается

1. Подключите репозиторий в Claude Code в вебе на [claude.ai/code](https://claude.ai/code)
2. Проверьте свои разрешения GitHub для этого репозитория
3. Попробуйте отключить и снова подключить свою учётную запись GitHub

### Выбран неправильный репозиторий

1. Нажмите кнопку «Change Repo», чтобы выбрать другой репозиторий
2. Включите имя репозитория в свой запрос для более точного выбора

### Ошибки аутентификации

1. Отключите и снова подключите свою учётную запись Claude в App Home
2. Убедитесь, что вы вошли в правильную учётную запись Claude в своём браузере
3. Проверьте, что ваш тарифный план Claude включает доступ к Claude Code

### Истечение срока действия сессии

1. Сессии остаются доступными в вашей истории Claude Code в вебе
2. Вы можете продолжить или обратиться к прошлым сессиям с [claude.ai/code](https://claude.ai/code)

## Текущие ограничения

* **Только GitHub**: в настоящее время поддерживаются репозитории на GitHub.
* **Один PR за раз**: каждая сессия может создать один pull request.
* **Применяются лимиты**: сессии используют лимиты вашего индивидуального тарифного плана Claude.
* **Требуется доступ к вебу**: у пользователей должен быть доступ к Claude Code в вебе; те, у кого его нет, будут получать только стандартные ответы чата Claude.

# Расширение для Chrome (бета)

> Подключите Claude Code к браузеру Chrome, чтобы тестировать веб-приложения, отлаживать по логам консоли, автоматизировать заполнение форм и извлекать данные с веб-страниц.

Claude Code интегрируется с [расширением Claude in Chrome для браузера](https://chromewebstore.google.com/detail/claude/fcoeoabgfenejglbffodgkkbkcdhcgfn), чтобы предоставить вам возможности автоматизации браузера из CLI или из [расширения VS Code](https://code.claude.com/docs/en/vs-code#automate-browser-tasks-with-chrome). Собирайте свой код, а затем тестируйте и отлаживайте его в браузере, не переключая контекст.

Claude открывает новые вкладки для задач в браузере и использует состояние вашего входа в браузере, поэтому он может получить доступ к любому сайту, на котором вы уже авторизованы. Действия в браузере выполняются в видимом окне Chrome в реальном времени. Когда Claude сталкивается со страницей входа или CAPTCHA, он приостанавливается и просит вас обработать это вручную.

> **Примечание.** Интеграция с Chrome находится в стадии бета-тестирования и в настоящее время работает с Google Chrome и Microsoft Edge. Она пока не поддерживается в Brave, Arc и других браузерах на базе Chromium. WSL (Windows Subsystem for Linux) также не поддерживается.

## Возможности

С подключённым Chrome вы можете объединять действия в браузере с задачами по написанию кода в едином рабочем процессе:

* **Отладка вживую**: читайте ошибки консоли и состояние DOM напрямую, а затем исправляйте код, который их вызвал
* **Проверка дизайна**: соберите UI по макету из Figma, а затем откройте его в браузере, чтобы убедиться в соответствии
* **Тестирование веб-приложений**: тестируйте валидацию форм, проверяйте визуальные регрессии или верифицируйте пользовательские сценарии
* **Веб-приложения с авторизацией**: взаимодействуйте с Google Docs, Gmail, Notion или любым приложением, в которое вы вошли, без коннекторов через API
* **Извлечение данных**: извлекайте структурированную информацию с веб-страниц и сохраняйте её локально
* **Автоматизация задач**: автоматизируйте повторяющиеся задачи в браузере, такие как ввод данных, заполнение форм или рабочие процессы, охватывающие несколько сайтов
* **Запись сессии**: записывайте взаимодействия с браузером в виде GIF-файлов, чтобы задокументировать или поделиться тем, что произошло

## Предварительные требования

Прежде чем использовать Claude Code с Chrome, вам понадобятся:

* Браузер [Google Chrome](https://www.google.com/chrome/) или [Microsoft Edge](https://www.microsoft.com/edge)
* [Расширение Claude in Chrome](https://chromewebstore.google.com/detail/claude/fcoeoabgfenejglbffodgkkbkcdhcgfn) версии 1.0.36 или выше, доступное в Chrome Web Store для обоих браузеров
* [Claude Code](https://code.claude.com/docs/en/quickstart#step-1-install-claude-code) версии 2.0.73 или выше
* Прямой план Anthropic (Pro, Max, Team или Enterprise)

> **Примечание.** Интеграция с Chrome недоступна через сторонних провайдеров, таких как Amazon Bedrock, Google Cloud Vertex AI или Microsoft Foundry. Если вы получаете доступ к Claude исключительно через стороннего провайдера, для использования этой функции вам понадобится отдельная учётная запись claude.ai.

## Начало работы в CLI

1. **Запустите Claude Code с Chrome**

   Запустите Claude Code с флагом `--chrome`:

   ```bash
   claude --chrome
   ```

   Вы также можете включить Chrome изнутри существующей сессии, выполнив команду `/chrome`.

2. **Попросите Claude использовать браузер**

   Этот пример переходит на страницу, взаимодействует с ней и сообщает о том, что обнаруживает, — и всё это прямо из вашего терминала или редактора:

   ```text
   Go to code.claude.com/docs, click on the search box,
   type "hooks", and tell me what results appear
   ```

Выполните `/chrome` в любой момент, чтобы проверить статус подключения, управлять разрешениями, переподключить расширение или выбрать, какой из подключённых браузеров использовать. Если при запуске действия в браузере подключено более одного браузера, Claude предложит вам выбрать один из них.

Для VS Code смотрите [автоматизацию браузера в VS Code](https://code.claude.com/docs/en/vs-code#automate-browser-tasks-with-chrome).

### Включить Chrome по умолчанию

Чтобы не передавать `--chrome` в каждой сессии, выполните `/chrome` и выберите «Enabled by default».

В [расширении VS Code](https://code.claude.com/docs/en/vs-code#automate-browser-tasks-with-chrome) Chrome доступен всякий раз, когда установлено расширение Chrome. Дополнительный флаг не нужен.

> **Примечание.** Включение Chrome по умолчанию в CLI увеличивает потребление контекста, поскольку инструменты браузера загружаются всегда. Если вы заметили увеличенное потребление контекста, отключите эту настройку и используйте `--chrome` только по необходимости.

### Управление разрешениями для сайтов

Разрешения на уровне сайтов наследуются от расширения Chrome. Управляйте разрешениями в настройках расширения Chrome, чтобы контролировать, на каких сайтах Claude может просматривать, кликать и вводить текст.

## Примеры рабочих процессов

Эти примеры показывают распространённые способы сочетания действий в браузере с задачами по написанию кода. Выполните `/mcp` и выберите `claude-in-chrome`, чтобы увидеть полный список доступных инструментов браузера.

### Тестирование локального веб-приложения

При разработке веб-приложения попросите Claude убедиться, что ваши изменения работают правильно:

```text
I just updated the login form validation. Can you open localhost:3000,
try submitting the form with invalid data, and check if the error
messages appear correctly?
```

Claude переходит на ваш локальный сервер, взаимодействует с формой и сообщает о том, что наблюдает.

### Отладка по логам консоли

Claude может читать вывод консоли, чтобы помочь диагностировать проблемы. Скажите Claude, какие паттерны искать, вместо того чтобы запрашивать весь вывод консоли, поскольку логи могут быть многословными:

```text
Open the dashboard page and check the console for any errors when
the page loads.
```

Claude читает сообщения консоли и может фильтровать их по определённым паттернам или типам ошибок.

### Автоматизация заполнения форм

Ускорьте повторяющиеся задачи по вводу данных:

```text
I have a spreadsheet of customer contacts in contacts.csv. For each row,
go to the CRM at crm.example.com, click "Add Contact", and fill in the
name, email, and phone fields.
```

Claude читает ваш локальный файл, ориентируется в веб-интерфейсе и вводит данные для каждой записи.

### Составление контента в Google Docs

Используйте Claude, чтобы писать прямо в ваших документах без настройки API:

```text
Draft a project update based on the recent commits and add it to my
Google Doc at docs.google.com/document/d/abc123
```

Claude открывает документ, кликает по редактору и печатает содержимое. Это работает с любым веб-приложением, в которое вы вошли: Gmail, Notion, Sheets и другими.

### Извлечение данных с веб-страниц

Извлекайте структурированную информацию с веб-сайтов:

```text
Go to the product listings page and extract the name, price, and
availability for each item. Save the results as a CSV file.
```

Claude переходит на страницу, читает содержимое и компилирует данные в структурированный формат.

### Запуск рабочих процессов на нескольких сайтах

Координируйте задачи между несколькими веб-сайтами:

```text
Check my calendar for meetings tomorrow, then for each meeting with
an external attendee, look up their company website and add a note
about what they do.
```

Claude работает через вкладки, чтобы собрать информацию и выполнить рабочий процесс.

### Запись демонстрационного GIF

Создавайте записи взаимодействий с браузером, которыми можно поделиться:

```text
Record a GIF showing how to complete the checkout flow, from adding
an item to the cart through to the confirmation page.
```

Claude записывает последовательность взаимодействий и сохраняет её в виде GIF-файла.

## Устранение неполадок

### Расширение не обнаружено

Если Claude Code не может обнаружить расширение Chrome:

1. Убедитесь, что расширение Chrome установлено и включено в `chrome://extensions`
2. Убедитесь, что Claude Code обновлён, выполнив `claude --version`
3. Проверьте, что Chrome запущен
4. Выполните `/chrome` и выберите «Reconnect extension», чтобы восстановить подключение
5. Если проблема сохраняется, перезапустите и Claude Code, и Chrome

При первом включении интеграции с Chrome Claude Code устанавливает конфигурационный файл нативного узла обмена сообщениями (native messaging host). Chrome читает этот файл при запуске, поэтому, если расширение не обнаружено с первой попытки, перезапустите Chrome, чтобы применить новую конфигурацию.

Если подключение по-прежнему не удаётся, убедитесь, что конфигурационный файл узла существует по пути:

Для Chrome:

* **macOS**: `~/Library/Application Support/Google/Chrome/NativeMessagingHosts/com.anthropic.claude_code_browser_extension.json`
* **Linux**: `~/.config/google-chrome/NativeMessagingHosts/com.anthropic.claude_code_browser_extension.json`
* **Windows**: проверьте `HKCU\Software\Google\Chrome\NativeMessagingHosts\` в реестре Windows

Для Edge:

* **macOS**: `~/Library/Application Support/Microsoft Edge/NativeMessagingHosts/com.anthropic.claude_code_browser_extension.json`
* **Linux**: `~/.config/microsoft-edge/NativeMessagingHosts/com.anthropic.claude_code_browser_extension.json`
* **Windows**: проверьте `HKCU\Software\Microsoft\Edge\NativeMessagingHosts\` в реестре Windows

### Браузер не отвечает

Если команды Claude для браузера перестают работать:

1. Проверьте, не блокирует ли страницу модальное диалоговое окно (alert, confirm, prompt). Диалоговые окна JavaScript блокируют события браузера и не дают Claude получать команды. Закройте диалоговое окно вручную, затем скажите Claude продолжить.
2. Попросите Claude создать новую вкладку и попробовать снова
3. Перезапустите расширение Chrome, отключив и снова включив его в `chrome://extensions`

### Подключение обрывается во время длинных сессий

Сервис-воркер (service worker) расширения Chrome может перейти в спящий режим во время продолжительных сессий, что разрывает подключение. Если инструменты браузера перестают работать после периода бездействия, выполните `/chrome` и выберите «Reconnect extension».

### Проблемы, специфичные для Windows

В Windows вы можете столкнуться со следующим:

* **Конфликты именованных каналов (EADDRINUSE)**: если другой процесс использует тот же именованный канал, перезапустите Claude Code. Закройте любые другие сессии Claude Code, которые могут использовать Chrome.
* **Ошибки нативного узла обмена сообщениями**: если нативный узел обмена сообщениями аварийно завершается при запуске, попробуйте переустановить Claude Code, чтобы перегенерировать конфигурацию узла.

### Распространённые сообщения об ошибках

Это наиболее часто встречающиеся ошибки и способы их устранения:

| Ошибка                               | Причина                                            | Решение                                                             |
| ------------------------------------ | ------------------------------------------------ | --------------------------------------------------------------- |
| «Browser extension is not connected» | Нативный узел обмена сообщениями не может достучаться до расширения | Перезапустите Chrome и Claude Code, затем выполните `/chrome` для переподключения |
| «Extension not detected»             | Расширение Chrome не установлено или отключено | Установите или включите расширение в `chrome://extensions`        |
| «No tab available»                   | Claude попытался выполнить действие до того, как вкладка была готова | Попросите Claude создать новую вкладку и повторить попытку                       |
| «Receiving end does not exist»       | Сервис-воркер расширения перешёл в спящий режим | Выполните `/chrome` и выберите «Reconnect extension»                  |

## Рецепты промптов

Это шаблоны промптов для повседневных задач: изучения незнакомого кода, отладки, рефакторинга, написания тестов и создания PR. Каждый из них работает в любой среде Claude Code; адаптируйте формулировки под свой проект.

### Понимание новых кодовых баз

О настройке Claude Code в монорепозитории или большой кодовой базе см. [Монорепозитории и большие репозитории](https://code.claude.com/docs/en/large-codebases).

#### Получить быстрый обзор кодовой базы

Предположим, вы только что присоединились к новому проекту и вам нужно быстро разобраться в его структуре.

1. Перейдите в корневой каталог проекта

   ```bash
   cd /path/to/project 
   ```

2. Запустите Claude Code

   ```bash
   claude 
   ```

3. Попросите высокоуровневый обзор

   ```text
   give me an overview of this codebase
   ```

4. Углубитесь в конкретные компоненты

   ```text
   explain the main architecture patterns used here
   ```

   ```text
   what are the key data models?
   ```

   ```text
   how is authentication handled?
   ```

> **Совет.** Советы:
>
> * Начинайте с широких вопросов, затем сужайте до конкретных областей
> * Спрашивайте о соглашениях по коду и паттернах, используемых в проекте
> * Запросите глоссарий специфичных для проекта терминов

#### Найти нужный код

Предположим, вам нужно найти код, относящийся к определённой функции или возможности.

1. Попросите Claude найти нужные файлы

   ```text
   find the files that handle user authentication
   ```

2. Получите контекст о том, как взаимодействуют компоненты

   ```text
   how do these authentication files work together?
   ```

3. Разберитесь в потоке выполнения

   ```text
   trace the login process from front-end to database
   ```

> **Совет.** Советы:
>
> * Будьте конкретны в том, что вы ищете
> * Используйте предметную терминологию проекта
> * Установите [плагин интеллектуального анализа кода](https://code.claude.com/docs/en/discover-plugins#code-intelligence) для вашего языка, чтобы дать Claude точную навигацию «перейти к определению» и «найти ссылки»

### Эффективное исправление ошибок

Предположим, вы столкнулись с сообщением об ошибке и вам нужно найти и устранить её источник.

1. Поделитесь ошибкой с Claude

   ```text
   I'm seeing an error when I run npm test
   ```

2. Попросите рекомендации по исправлению

   ```text
   suggest a few ways to fix the @ts-ignore in user.ts
   ```

3. Примените исправление

   ```text
   update user.ts to add the null check you suggested
   ```

> **Совет.** Советы:
>
> * Сообщите Claude команду для воспроизведения проблемы и получите трассировку стека
> * Упомяните любые шаги для воспроизведения ошибки
> * Дайте Claude знать, является ли ошибка периодической или постоянной

### Рефакторинг кода

Предположим, вам нужно обновить старый код, чтобы использовать современные паттерны и практики.

1. Определите устаревший код для рефакторинга

   ```text
   find deprecated API usage in our codebase
   ```

2. Получите рекомендации по рефакторингу

   ```text
   suggest how to refactor utils.js to use modern JavaScript features
   ```

3. Примените изменения безопасно

   ```text
   refactor utils.js to use ES2024 features while maintaining the same behavior
   ```

4. Проверьте рефакторинг

   ```text
   run tests for the refactored code
   ```

> **Совет.** Советы:
>
> * Попросите Claude объяснить преимущества современного подхода
> * Требуйте, чтобы изменения сохраняли обратную совместимость, когда это необходимо
> * Выполняйте рефакторинг небольшими, поддающимися тестированию шагами

### Работа с тестами

Предположим, вам нужно добавить тесты для непокрытого кода.

1. Определите непротестированный код

   ```text
   find functions in NotificationsService.swift that are not covered by tests
   ```

2. Сгенерируйте каркас тестов

   ```text
   add tests for the notification service
   ```

3. Добавьте содержательные тестовые случаи

   ```text
   add test cases for edge conditions in the notification service
   ```

4. Запустите и проверьте тесты

   ```text
   run the new tests and fix any failures
   ```

Claude может генерировать тесты, следующие существующим паттернам и соглашениям вашего проекта. Запрашивая тесты, будьте конкретны в том, какое поведение вы хотите проверить. Claude изучает ваши существующие тестовые файлы, чтобы соответствовать уже используемым стилю, фреймворкам и паттернам утверждений.

Для всестороннего покрытия попросите Claude выявить граничные случаи, которые вы могли упустить. Claude может проанализировать пути выполнения вашего кода и предложить тесты для условий ошибок, граничных значений и неожиданных входных данных, которые легко упустить из виду.

### Создание pull request

Вы можете создавать pull request, попросив Claude напрямую («create a pr for my changes»), или провести Claude через это пошагово:

1. Обобщите ваши изменения

   ```text
   summarize the changes I've made to the authentication module
   ```

2. Сгенерируйте pull request

   ```text
   create a pr
   ```

3. Проверьте и доработайте

   ```text
   enhance the PR description with more context about the security improvements
   ```

Когда вы создаёте PR с помощью `gh pr create`, сессия автоматически связывается с этим PR. Чтобы вернуться к ней позже, запустите `claude --from-pr <number>` или вставьте URL PR в поиск [окна выбора `/resume`](https://code.claude.com/docs/en/sessions#use-the-session-picker).

> **Совет.** Проверьте сгенерированный Claude PR перед отправкой и попросите Claude указать на потенциальные риски или моменты, требующие внимания.

### Работа с документацией

Предположим, вам нужно добавить или обновить документацию для вашего кода.

1. Определите недокументированный код

   ```text
   find functions without proper JSDoc comments in the auth module
   ```

2. Сгенерируйте документацию

   ```text
   add JSDoc comments to the undocumented functions in auth.js
   ```

3. Проверьте и улучшите

   ```text
   improve the generated documentation with more context and examples
   ```

4. Проверьте документацию

   ```text
   check if the documentation follows our project standards
   ```

> **Совет.** Советы:
>
> * Укажите желаемый стиль документации (JSDoc, docstrings и т. д.)
> * Попросите включить примеры в документацию
> * Запросите документацию для публичных API, интерфейсов и сложной логики

### Работа в заметках и папках без кода

Claude Code работает в любом каталоге. Запустите его внутри хранилища заметок, папки с документацией или любой коллекции markdown-файлов, чтобы искать, редактировать и реорганизовывать содержимое так же, как вы это делаете с кодом.

Каталог `.claude/` и `CLAUDE.md` располагаются рядом с каталогами конфигурации других инструментов без конфликтов. Claude читает файлы заново при каждом вызове инструмента, поэтому он видит правки, которые вы вносите в другом приложении, в следующий раз, когда читает этот файл.

### Работа с изображениями

Предположим, вам нужно работать с изображениями в вашей кодовой базе, и вы хотите, чтобы Claude помог проанализировать содержимое изображения.

1. Добавьте изображение в разговор

   Вы можете использовать любой из этих способов:

   1. Перетащите изображение в окно Claude Code
   2. Скопируйте изображение и вставьте его в CLI с помощью ctrl+v (не используйте cmd+v)
   3. Укажите Claude путь к изображению. Например, «Analyze this image: /path/to/your/image.png»

2. Попросите Claude проанализировать изображение

   ```text
   What does this image show?
   ```

   ```text
   Describe the UI elements in this screenshot
   ```

   ```text
   Are there any problematic elements in this diagram?
   ```

3. Используйте изображения для контекста

   ```text
   Here's a screenshot of the error. What's causing it?
   ```

   ```text
   This is our current database schema. How should we modify it for the new feature?
   ```

4. Получите предложения по коду на основе визуального содержимого

   ```text
   Generate CSS to match this design mockup
   ```

   ```text
   What HTML structure would recreate this component?
   ```

> **Совет.** Советы:
>
> * Используйте изображения, когда текстовые описания были бы неясными или громоздкими
> * Включайте скриншоты ошибок, дизайнов UI или диаграмм для лучшего контекста
> * Вы можете работать с несколькими изображениями в одном разговоре
> * Анализ изображений работает с диаграммами, скриншотами, макетами и не только
> * Когда Claude ссылается на изображения (например, `[Image #1]`), нажмите `Cmd+Click` (Mac) или `Ctrl+Click` (Windows/Linux) по ссылке, чтобы открыть изображение в вашем средстве просмотра по умолчанию

### Ссылки на файлы и каталоги

Используйте @, чтобы быстро включить файлы или каталоги, не дожидаясь, пока Claude их прочитает.

1. Сослаться на один файл

   ```text
   Explain the logic in @src/utils/auth.js
   ```

   Это включает полное содержимое файла в разговор.

2. Сослаться на каталог

   ```text
   What's the structure of @src/components?
   ```

   Это предоставляет список каталога с информацией о файлах.

3. Сослаться на ресурсы MCP

   ```text
   Show me the data from @github:repos/owner/repo/issues
   ```

   Это извлекает данные из подключённых серверов MCP, используя формат @server:resource. Подробности см. в [Ресурсы MCP](https://code.claude.com/docs/en/mcp#use-mcp-resources).

> **Совет.** Советы:
>
> * Пути к файлам могут быть относительными или абсолютными
> * Ссылки на файлы через @ добавляют в контекст `CLAUDE.md` из каталога файла и родительских каталогов
> * Ссылки на каталоги показывают списки файлов, а не содержимое
> * Вы можете ссылаться на несколько файлов в одном сообщении (например, «@file1.js and @file2.js»)

### Запуск Claude по расписанию

Предположим, вы хотите, чтобы Claude автоматически выполнял задачу на регулярной основе, например проверял открытые PR каждое утро, еженедельно проводил аудит зависимостей или проверял сбои CI по ночам.

Выберите вариант планирования в зависимости от того, где вы хотите запускать задачу:

| Вариант | Где выполняется | Лучше всего подходит для |
| :------ | :-------------- | :----------------------- |
| [Routines](https://code.claude.com/docs/en/routines) | Инфраструктура под управлением Anthropic | Задачи, которые должны выполняться, даже когда ваш компьютер выключен. Могут также запускаться по вызовам API или событиям GitHub в дополнение к расписанию. Настраивается на [claude.ai/code/routines](https://claude.ai/code/routines). |
| [Запланированные задачи на рабочем столе](https://code.claude.com/docs/en/desktop-scheduled-tasks) | Ваша машина, через приложение для рабочего стола | Задачи, которым нужен прямой доступ к локальным файлам, инструментам или незакоммиченным изменениям. |
| [GitHub Actions](https://code.claude.com/docs/en/github-actions) | Ваш конвейер CI | Задачи, привязанные к событиям репозитория, таким как открытые PR, или расписания cron, которые должны находиться рядом с конфигурацией вашего рабочего процесса. |
| [`/loop`](https://code.claude.com/docs/en/scheduled-tasks) | Текущая сессия CLI | Быстрый опрос, пока сессия открыта. Задачи останавливаются, когда вы начинаете новый разговор; `--resume` и `--continue` восстанавливают неистёкшие. |

> **Совет.** При написании промптов для запланированных задач чётко указывайте, как выглядит успех и что делать с результатами. Задача выполняется автономно, поэтому она не может задавать уточняющие вопросы. Например: «Review open PRs labeled `needs-review`, leave inline comments on any issues, and post a summary in the `#eng-reviews` Slack channel».

### Спросите Claude о его возможностях

У Claude есть встроенный доступ к своей документации, и он может отвечать на вопросы о собственных функциях и ограничениях.

#### Примеры вопросов

```text
can Claude Code create pull requests?
```

```text
how does Claude Code handle permissions?
```

```text
what skills are available?
```

```text
how do I use MCP with Claude Code?
```

```text
how do I configure Claude Code for Amazon Bedrock?
```

```text
what are the limitations of Claude Code?
```

> **Примечание.** Claude предоставляет ответы на эти вопросы на основе документации. Для практических демонстраций запустите `/powerup`, чтобы получить интерактивные уроки с анимированными демонстрациями, или обратитесь к конкретным разделам о рабочих процессах выше.

> **Совет.** Советы:
>
> * У Claude всегда есть доступ к последней документации Claude Code, независимо от используемой вами версии
> * Задавайте конкретные вопросы, чтобы получить подробные ответы
> * Claude может объяснить сложные функции, такие как интеграция MCP, корпоративные конфигурации и продвинутые рабочие процессы

## Возобновление предыдущих разговоров

Когда задача охватывает несколько сеансов работы, продолжайте с того места, где остановились, вместо повторного объяснения контекста. Claude Code сохраняет каждый разговор локально.

```bash
claude --continue
```

Это возобновляет самую последнюю сессию в текущем каталоге; если её ещё нет, выводится `No conversation found to continue` и выполняется выход. Используйте `claude --resume`, чтобы выбрать из списка, или `/resume` изнутри работающей сессии. См. [Управление сессиями](https://code.claude.com/docs/en/sessions) для именования, ветвления и полного справочника по окну выбора.

## Запуск параллельных сессий с worktree

Работайте над функцией в одном терминале, пока Claude исправляет ошибку в другом, без конфликта правок. Каждый worktree — это отдельная выгрузка на своей собственной ветке.

```bash
claude --worktree feature-auth
```

Запустите ту же команду с другим именем во втором терминале, чтобы начать изолированную параллельную сессию. См. [Worktrees](https://code.claude.com/docs/en/worktrees) для очистки, `.worktreeinclude` и поддержки не-git систем контроля версий. Чтобы отслеживать параллельные сессии с одного экрана, а не в отдельных терминалах, см. [фоновые агенты](https://code.claude.com/docs/en/agent-view).

## Планирование перед редактированием

Для изменений, которые вы хотите проверить до их записи на диск, переключитесь в режим планирования. Claude читает файлы и предлагает план, но не вносит правок, пока вы не одобрите.

```bash
claude --permission-mode plan
```

Вы также можете нажать `Shift+Tab` в середине сессии, чтобы переключиться в режим планирования. См. [Режим планирования](https://code.claude.com/docs/en/permission-modes#analyze-before-you-edit-with-plan-mode) для процесса одобрения и редактирования плана в вашем текстовом редакторе.

## Делегирование исследований субагентам

Изучение большой кодовой базы заполняет ваш контекст чтением файлов. Делегируйте изучение, чтобы обратно возвращались только выводы.

```text
use a subagent to investigate how our auth system handles token refresh
```

Субагент читает файлы в своём собственном окне контекста и сообщает сводку. См. [Субагенты](https://code.claude.com/docs/en/sub-agents) для определения пользовательских агентов с их собственными инструментами и промптами.

## Передача Claude в скрипты через конвейер

Запускайте Claude неинтерактивно для CI, хуков перед коммитом или пакетной обработки. Stdin и stdout работают как в любом Unix-инструменте.

```bash
git log --oneline -20 | claude -p "summarize these recent commits"
```

См. [Неинтерактивный режим](https://code.claude.com/docs/en/headless) для форматов вывода, флагов разрешений и паттернов веерного распределения.

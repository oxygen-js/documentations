# Управление сессиями

> Именуйте, возобновляйте, ответвляйте и переключайтесь между разговорами Claude Code. Охватывает `--continue`, `--resume`, `--from-pr`, выбор сессии через `/resume`, именование сессий, экспорт транскриптов и место хранения транскриптов.

Сессия — это сохранённый разговор, привязанный к каталогу проекта. Claude Code сохраняет её локально по мере работы, чтобы вы могли возобновить с того места, где остановились, ответвиться и попробовать другой подход или переключаться между задачами.

[Настольное приложение](https://code.claude.com/docs/en/desktop#work-in-parallel-with-sessions), [Claude Code в вебе](https://code.claude.com/docs/en/claude-code-on-the-web) и [расширение для VS Code](https://code.claude.com/docs/en/vs-code#resume-past-conversations) — каждое ведёт собственную историю сессий. Эта страница посвящена CLI.

## Возобновление сессии

Сессии непрерывно сохраняются в [локальные файлы транскриптов](#export-and-locate-session-data) по мере работы, поэтому вы можете вернуться к сессии после выхода или выполнения `/clear`. Используйте следующие точки входа:

| Команда                     | Что она делает                                                     |
| :-------------------------- | :---------------------------------------------------------------- |
| `claude --continue`         | Возобновляет самую последнюю сессию в текущем каталоге             |
| `claude --resume`           | Открывает [окно выбора сессии](#use-the-session-picker)            |
| `claude --resume <name>`    | Возобновляет именованную сессию напрямую                           |
| `claude --from-pr <number>` | Возобновляет сессию, связанную с этим pull request                 |
| `/resume`                   | Переключается на другой разговор изнутри активной сессии           |

Сессии, созданные с помощью [`claude -p`](https://code.claude.com/docs/en/headless) или [Agent SDK](https://code.claude.com/docs/en/agent-sdk/overview), не появляются в окне выбора сессии, но вы всё равно можете возобновить такую сессию, передав её идентификатор в `claude --resume <session-id>`. Запускайте это из того каталога, в котором была начата сессия: поиск по идентификатору сессии ограничен текущим каталогом проекта и его git-worktree, поэтому сессия, созданная в другом месте, сообщит `No conversation found with session ID: <session-id>`.

### Где ищет окно выбора сессии

Сессии хранятся отдельно для каждого каталога проекта. По умолчанию окно выбора сессии показывает интерактивные сессии из текущего worktree, а также сессии, начатые в другом месте, которые добавили текущий каталог с помощью `/add-dir`. Используйте `Ctrl+W`, чтобы расширить область до всех worktree репозитория, или `Ctrl+A`, чтобы расширить её до каждого проекта на этой машине.

Начиная с версии v2.1.169, перемещение сессии с помощью [`/cd`](https://code.claude.com/docs/en/commands) переносит её в хранилище проекта нового каталога, поэтому впоследствии она появляется в окне выбора этого каталога. Начиная с версии v2.1.196, перемещённая сессия больше не появляется в окне выбора старого каталога даже после сбоя или принудительного выхода. В более ранних версиях она также могла снова появляться в списке старого каталога после выхода, который не был чистым, если старый путь содержал специальные символы, такие как подчёркивания.

Выбор сессии из другого worktree того же репозитория возобновляет её на месте. Выбор сессии из несвязанного проекта вместо этого копирует в буфер обмена команду `cd` и команду возобновления.

Возобновление по имени работает в рамках текущего репозитория и его worktree. Обе формы ищут точное совпадение и возобновляют сессию напрямую, даже если она находится в другом worktree:

| Команда                  | Точное совпадение      | Неоднозначное имя                                                              |
| :----------------------- | :--------------------- | :---------------------------------------------------------------------------- |
| `claude --resume <name>` | Возобновляет напрямую  | Открывает окно выбора сессии с предзаполненным именем в качестве поискового запроса |
| `/resume <name>`         | Возобновляет напрямую  | Сообщает об ошибке; запустите `/resume` без аргумента, чтобы открыть окно выбора сессии |

## Именуйте свои сессии

Давайте сессиям описательные имена, чтобы их можно было найти в окне выбора сессии и возобновить по имени. Это особенно важно, когда вы работаете над несколькими задачами параллельно.

| Когда                      | Как задать имя                                                                                                                                                          |
| :------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| При запуске                | `claude -n auth-refactor`                                                                                                                                             |
| Во время сессии            | `/rename auth-refactor`. Имя также появляется на панели промпта                                                                                                      |
| Из окна выбора сессии      | Выделите сессию и нажмите `Ctrl+R`                                                                                                                                    |
| При принятии плана         | Принятие плана в [режиме планирования](https://code.claude.com/docs/en/permission-modes#analyze-before-you-edit-with-plan-mode) именует сессию по содержимому плана, если вы ещё не задали имя |

После того как сессия названа, вернитесь к ней с помощью `claude --resume <name>` или `/resume <name>`. О том, как разрешение имён ведёт себя в разных worktree, см. [Возобновление сессии](#resume-a-session).

Начиная с версии v2.1.196, интерактивные сессии, которым вы никогда не давали имя, всё равно получают отображаемое имя по умолчанию при запуске. Требуется Claude Code v2.1.196 или новее. Имя по умолчанию объединяет имя рабочего каталога с двухсимвольным суффиксом, например `my-app-3f`, и идентифицирует сессию в списках работающих сессий, таких как [представление агентов](https://code.claude.com/docs/en/agent-view) и вывод `claude agents --json`.

Имя по умолчанию не является дескриптором для возобновления: `claude --resume <name>`, `/resume <name>` и окно выбора сессии сопоставляют только имена, заданные вами. Именование сессии заменяет имя по умолчанию.

## Использование окна выбора сессии

Запустите `/resume` внутри сессии или `claude --resume` без аргументов, чтобы открыть интерактивное окно выбора сессии. Используйте эти сочетания клавиш для навигации, поиска и расширения списка:

| Сочетание клавиш                                    | Действие                                                                                                                                                     |
| :-------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `↑` / `↓`                                           | Переход между сессиями                                                                                                                                      |
| `→` / `←`                                           | Развернуть или свернуть сгруппированные сессии                                                                                                             |
| `Enter`                                             | Возобновить выделенную сессию                                                                                                                              |
| `Space`                                             | Предпросмотр содержимого сессии. `Ctrl+V` также работает в терминалах, которые не перехватывают его как вставку                                            |
| `Ctrl+R`                                            | Переименовать выделенную сессию                                                                                                                            |
| `/` или любой печатаемый символ, кроме `Space`      | Войти в режим поиска и отфильтровать сессии. Вставьте URL pull request или merge request из GitHub, GitHub Enterprise, GitLab или Bitbucket, чтобы найти создавшую его сессию |
| `Ctrl+A`                                            | Показать сессии из всех проектов на этой машине. Нажмите ещё раз, чтобы вернуться к текущему репозиторию                                                    |
| `Ctrl+W`                                            | Показать сессии из всех worktree текущего репозитория. Нажмите ещё раз, чтобы вернуться к текущему worktree. Показывается только в репозиториях с несколькими worktree |
| `Ctrl+B`                                            | Отфильтровать до сессий из текущей git-ветки. Нажмите ещё раз, чтобы показать все ветки                                                                     |
| `Esc`                                               | Выйти из окна выбора сессии или режима поиска                                                                                                              |

Каждая строка показывает имя сессии, если оно задано, иначе — сводку разговора или первый промпт, а также время с момента последней активности, количество сообщений и git-ветку. Путь проекта появляется после того, как вы расширите список до всех проектов с помощью `Ctrl+A`.

Ответвлённые сессии, созданные с помощью `/branch`, `/rewind` или `--fork-session`, группируются под своей корневой сессией. Нажмите `→`, чтобы развернуть группу.

## Ответвление сессии

Ответвление создаёт копию разговора на текущий момент и переключает вас в неё, оставляя оригинал нетронутым. Используйте его, чтобы попробовать другой подход, не теряя тот путь, на котором вы были.

Изнутри сессии запустите `/branch` с необязательным именем:

```text
/branch try-streaming-approach
```

Из командной строки сочетайте `--continue` или `--resume` с `--fork-session`:

```bash
claude --continue --fork-session
```

Оригинальная сессия остаётся без изменений и по-прежнему доступна в окне выбора сессии. Подтверждение `/branch` печатает два идентификатора сессий: новую ветку, в которой вы теперь находитесь, и оригинал. Чтобы вернуться к оригиналу, передайте его идентификатор в `/resume`, используйте окно выбора сессии или запустите `/resume <original-name>`. Разрешения, которые вы одобрили с помощью «allow for this session» (разрешить для этой сессии), не переносятся в новую ветку. Если вы возобновите одну и ту же сессию в двух терминалах без ответвления, сообщения из обоих будут чередоваться в одном транскрипте.

О перемотке на основе контрольных точек в пределах одной сессии см. [Контрольные точки](https://code.claude.com/docs/en/checkpointing).

## Управление контекстом в пределах сессии

Эти команды управляют тем, что находится в окне контекста, не покидая сессию:

* **`/clear`**: начать заново с пустым контекстом. Предыдущий разговор сохраняется и может быть возобновлён
* **`/compact [instructions]`**: заменить историю сводкой, при желании сфокусированной на том, что вы укажете
* **`/context`**: показать, что в данный момент потребляет контекст

О том, как уплотнение (compaction) взаимодействует с CLAUDE.md, навыками и правилами, см. [руководство по окну контекста](https://code.claude.com/docs/en/context-window). О стратегиях того, когда очищать, а когда уплотнять, см. [Лучшие практики](https://code.claude.com/docs/en/best-practices#manage-your-session).

## Экспорт и поиск данных сессии

Запустите `/export`, чтобы открыть меню, которое позволяет скопировать текущий разговор в буфер обмена или сохранить его как файл в виде простого текста, где сообщения и вывод инструментов отображаются как читаемый текст. Передайте имя файла, чтобы пропустить меню и записать данные напрямую в этот файл.

### Доступ к разговорам из скриптов

`/export` создаёт отрисованный транскрипт для чтения человеком. Приведённые ниже интерфейсы создают структурированные данные для разбора скриптом: JSON-результат запуска, путь к файлу транскрипта сессии или живой поток событий. Выбирайте по тому, что запускает скрипт:

* **Запустить Claude один раз и захватить результат**: вызовите `claude -p` с [`--output-format json` или `stream-json`](https://code.claude.com/docs/en/headless#get-structured-output), чтобы захватить результат, идентификатор сессии, потребление и стоимость неинтерактивного запуска в виде структурированного JSON.
* **Задать вопрос существующей сессии**: передайте идентификатор сессии в [`claude -p --resume`](https://code.claude.com/docs/en/headless#continue-conversations), чтобы отправить дополнительный промпт, например запрос сводки, и захватить структурированный ответ.
* **Реагировать на события сессии**: читайте поле `transcript_path`, которое [хуки](https://code.claude.com/docs/en/hooks#common-input-fields) и [команды строки состояния](https://code.claude.com/docs/en/statusline#available-data) получают на вход. Хук `SessionEnd` может архивировать транскрипт при завершении сессии.
* **Встроить Claude в приложение на TypeScript или Python**: используйте [Agent SDK](https://code.claude.com/docs/en/agent-sdk/overview), чтобы получать каждое сообщение программно.

Приведённый ниже пример использует второй интерфейс. Он отправляет дополнительный промпт существующей сессии и читает ответ с помощью `jq`:

```bash
claude -p --resume <session-id> --output-format json "summarize what we changed" | jq -r '.result'
```

### Где хранятся транскрипты

По умолчанию транскрипты хранятся в формате JSONL по пути `~/.claude/projects/<project>/<session-id>.jsonl`, где `<project>` — это путь к вашему рабочему каталогу, в котором неалфавитно-цифровые символы заменены на `-`. Каждая строка — это JSON-объект для сообщения, использования инструмента или записи метаданных. Формат записи является внутренним для Claude Code и меняется между версиями, поэтому скрипты, которые разбирают эти файлы напрямую, могут сломаться в любом релизе. Чтобы строить работу на основе данных сессии, используйте вместо этого `/export` или [интерфейсы для скриптов](#access-conversations-from-scripts).

Расположение, срок хранения и поведение записи настраиваются:

| Чтобы                                          | Задайте                                                | Где                       |
| ---------------------------------------------- | ------------------------------------------------------ | ------------------------- |
| Перенести хранилище за пределы `~/.claude`     | [`CLAUDE_CONFIG_DIR`](https://code.claude.com/docs/en/env-vars) | Переменная окружения      |
| Изменить 30-дневный срок хранения              | [`cleanupPeriodDays`](https://code.claude.com/docs/en/settings#available-settings) | `settings.json`           |
| Подавить запись транскриптов во всех режимах    | [`CLAUDE_CODE_SKIP_PROMPT_HISTORY`](https://code.claude.com/docs/en/env-vars) | Переменная окружения      |
| Подавить запись для одного неинтерактивного запуска | [`--no-session-persistence`](https://code.claude.com/docs/en/cli-reference) | Флаг CLI с `claude -p`    |

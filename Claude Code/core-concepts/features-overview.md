# Расширение Claude Code

> Разберитесь, когда использовать CLAUDE.md, навыки, субагентов, хуки, MCP и плагины.

Claude Code сочетает модель, которая рассуждает о вашем коде, со [встроенными инструментами](https://code.claude.com/docs/en/how-claude-code-works#tools) для операций с файлами, поиска, выполнения и доступа к вебу. Встроенные инструменты покрывают большинство задач по программированию. Это руководство посвящено слою расширений: функциям, которые вы добавляете, чтобы настроить то, что знает Claude, подключить его к внешним сервисам и автоматизировать рабочие процессы.

> **Примечание.** О том, как работает основной агентный цикл, см. [Как работает Claude Code](https://code.claude.com/docs/en/how-claude-code-works).

**Впервые в Claude Code?** Начните с [CLAUDE.md](https://code.claude.com/docs/en/memory) для соглашений проекта, а затем добавляйте другие расширения [по мере появления конкретных триггеров](#build-your-setup-over-time).

## Обзор

Расширения подключаются к разным частям агентного цикла:

* **[CLAUDE.md](https://code.claude.com/docs/en/memory)** добавляет постоянный контекст, который Claude видит в каждой сессии
* **[Навыки (skills)](https://code.claude.com/docs/en/skills)** добавляют переиспользуемые знания и вызываемые рабочие процессы
* **[Интеллектуальный анализ кода](https://code.claude.com/docs/en/tools-reference#lsp-tool-behavior)** подключает Claude к языковому серверу для навигации на уровне символов и отображения ошибок типов в реальном времени
* **[MCP](https://code.claude.com/docs/en/mcp)** подключает Claude к внешним сервисам и инструментам
* **[Субагенты](https://code.claude.com/docs/en/sub-agents)** выполняют собственные циклы в изолированном контексте, возвращая сводки
* **[Команды агентов](https://code.claude.com/docs/en/agent-teams)** координируют несколько независимых сессий с общими задачами и обменом сообщениями между участниками
* **[Хуки](https://code.claude.com/docs/en/hooks-guide)** срабатывают на событиях жизненного цикла и могут запускать скрипт, HTTP-запрос, промпт или субагента
* **[Плагины](https://code.claude.com/docs/en/plugins)** и **[маркетплейсы](https://code.claude.com/docs/en/plugin-marketplaces)** упаковывают и распространяют эти функции

[Навыки](https://code.claude.com/docs/en/skills) — самое гибкое расширение. Навык — это markdown-файл, содержащий знания, рабочие процессы или инструкции. Вы можете вызывать навыки командой вроде `/deploy`, либо Claude может загружать их автоматически, когда это уместно. Навыки могут выполняться в вашем текущем разговоре или в изолированном контексте через субагентов.

## Подбирайте функции под свою цель

Функции варьируются от всегда доступного контекста, который Claude видит в каждой сессии, до возможностей по запросу, которые можете вызвать вы или Claude, и до фоновой автоматизации, запускаемой на определённых событиях. В таблице ниже показано, что доступно и когда каждая из функций имеет смысл.

| Функция                                                        | Что она делает                                                  | Когда использовать                                                                  | Пример                                                                         |
| -------------------------------------------------------------- | ------------------------------------------------------------- | ------------------------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| **CLAUDE.md**                                                  | Постоянный контекст, загружаемый в каждом разговоре                  | Соглашения проекта, правила «всегда делай X»                                        | «Используй pnpm, а не npm. Запускай тесты перед коммитом.»                               |
| **Навык**                                                      | Инструкции, знания и рабочие процессы, которые Claude может использовать         | Переиспользуемое содержимое, справочная документация, повторяемые задачи                              | `/deploy` запускает ваш чек-лист развёртывания; навык с документацией API и шаблонами эндпоинтов |
| **Субагент**                                                   | Изолированный контекст выполнения, возвращающий обобщённые результаты    | Изоляция контекста, параллельные задачи, специализированные исполнители                          | Исследовательская задача, которая читает множество файлов, но возвращает только ключевые выводы               |
| **[Команды агентов](https://code.claude.com/docs/en/agent-teams)**                             | Координация нескольких независимых сессий Claude Code          | Параллельное исследование, разработка новых функций, отладка с конкурирующими гипотезами | Запуск ревьюеров для одновременной проверки безопасности, производительности и тестов        |
| **[Интеллектуальный анализ кода](https://code.claude.com/docs/en/tools-reference#lsp-tool-behavior)** | Навигация и диагностика на основе языкового сервера                    | Типизированные языки, крупные кодовые базы, где grep медленный или неточный                | Переход к определению символа вместо чтения всего файла                 |
| **MCP**                                                        | Подключение к внешним сервисам                                  | Внешние данные или действия                                                        | Запросы к вашей базе данных, публикация в Slack, управление браузером                    |
| **Хук**                                                        | Скрипт, HTTP-запрос, промпт или субагент, запускаемый событиями | Автоматизация, которая должна выполняться на каждом подходящем событии                                | Запуск ESLint после каждого редактирования файла                                                 |
| **[Артефакт](https://code.claude.com/docs/en/artifacts)**                                  | Публикация результата сессии как приватной интерактивной веб-страницы     | Результат, который вы хотите видеть или делиться визуально, а не в виде текста в терминале           | Хронология инцидента, обновляющаяся по мере того, как Claude ведёт расследование                    |

**[Плагины](https://code.claude.com/docs/en/plugins)** — это слой упаковки. Плагин объединяет навыки, хуки, субагентов и MCP-серверы в единый устанавливаемый блок. Навыки плагинов имеют пространства имён (например, `/my-plugin:review`), поэтому несколько плагинов могут сосуществовать. Используйте плагины, когда хотите переиспользовать одну и ту же настройку в нескольких репозиториях или распространять её другим через **[маркетплейс](https://code.claude.com/docs/en/plugin-marketplaces)**.

### Выстраивайте свою настройку со временем

Вам не нужно настраивать всё заранее. У каждой функции есть узнаваемый триггер, и большинство команд добавляют их примерно в таком порядке:

| Триггер                                                                          | Добавьте                                                                                            |
| :------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------- |
| Claude дважды неправильно применяет соглашение или команду                                  | Добавьте это в [CLAUDE.md](https://code.claude.com/docs/en/memory)                                                              |
| Вы постоянно набираете один и тот же промпт, чтобы начать задачу                                  | Сохраните его как вызываемый пользователем [навык](https://code.claude.com/docs/en/skills)                                                |
| Вы в третий раз вставляете в чат один и тот же плейбук или многошаговую процедуру | Зафиксируйте это как [навык](https://code.claude.com/docs/en/skills)                                                            |
| Вы постоянно копируете данные из вкладки браузера, которую Claude не видит                        | Подключите эту систему как [MCP-сервер](https://code.claude.com/docs/en/mcp)                                                |
| Claude читает множество файлов, чтобы найти, где определён или используется символ                | Установите [плагин интеллектуального анализа кода](https://code.claude.com/docs/en/discover-plugins#code-intelligence) для вашего языка |
| Побочная задача заваливает ваш разговор выводом, к которому вы больше не обратитесь               | Направьте её через [субагента](https://code.claude.com/docs/en/sub-agents)                                                  |
| Вы хотите, чтобы что-то происходило каждый раз без запроса                                   | Напишите [хук](https://code.claude.com/docs/en/hooks-guide)                                                                |
| Второму репозиторию нужна та же настройка                                         | Упакуйте это как [плагин](https://code.claude.com/docs/en/plugins)                                                            |

Те же самые триггеры подсказывают, когда обновлять то, что у вас уже есть. Повторяющаяся ошибка или регулярно всплывающий комментарий на ревью — это повод отредактировать CLAUDE.md, а не разовое исправление в чате. Рабочий процесс, который вы постоянно правите вручную, — это навык, которому нужна очередная доработка.

### Сравните похожие функции

Некоторые функции могут казаться похожими. Вот как их различать.

### Навык против субагента

Навыки и субагенты решают разные задачи:

* **Навыки** — это переиспользуемое содержимое, которое можно загрузить в любой контекст
* **Субагенты** — это изолированные исполнители, работающие отдельно от вашего основного разговора

| Аспект                                          | Навык                                          | Субагент                                                         |
| ----------------------------------------------- | ---------------------------------------------- | ---------------------------------------------------------------- |
| **Что это**                                  | Переиспользуемые инструкции, знания или рабочие процессы | Изолированный исполнитель с собственным контекстом                             |
| **Ключевое преимущество**                                 | Совместное использование содержимого между контекстами                  | Изоляция контекста. Работа происходит отдельно, возвращается только сводка |
| **Влияние на [окно контекста](https://code.claude.com/docs/en/context-window)** | Добавляется к вашему основному окну                       | Использует отдельное окно с собственными входными и выходными токенами      |
| **Лучше всего для**                                    | Справочных материалов, вызываемых рабочих процессов        | Задач, читающих множество файлов, параллельной работы, специализированных исполнителей   |

**Навыки могут быть справочными или деятельными.** Справочные навыки предоставляют знания, которые Claude использует на протяжении всей сессии (например, ваш стайл-гайд по API). Деятельные навыки говорят Claude сделать что-то конкретное (например, `/deploy`, который запускает ваш рабочий процесс развёртывания).

**Используйте субагента**, когда вам нужна изоляция контекста или когда ваше окно контекста заполняется. Субагент может прочитать десятки файлов или выполнить обширный поиск, но ваш основной разговор получит только сводку. Поскольку работа субагента не расходует ваш основной контекст, это также полезно, когда промежуточная работа не должна оставаться видимой. Пользовательские субагенты могут иметь собственные инструкции и предзагружать навыки.

**Их можно комбинировать.** Субагент может предзагрузить определённые навыки (поле `skills:`). Навык может выполняться в изолированном контексте с помощью `context: fork`. Подробнее см. [Навыки](https://code.claude.com/docs/en/skills).

### CLAUDE.md против навыка

Оба хранят инструкции, но загружаются по-разному и служат разным целям.

| Аспект                    | CLAUDE.md                    | Навык                                   |
| ------------------------- | ---------------------------- | --------------------------------------- |
| **Загружается**                 | Каждую сессию, автоматически | По запросу                               |
| **Может включать файлы**     | Да, через импорты `@path`    | Да, через импорты `@path`               |
| **Может запускать рабочие процессы** | Нет                           | Да, через `/<name>`                     |
| **Лучше всего для**              | Правил «всегда делай X»          | Справочных материалов, вызываемых рабочих процессов |

**Поместите это в CLAUDE.md**, если Claude должен всегда это знать: соглашения по коду, команды сборки, структура проекта, правила «никогда не делай X».

**Поместите это в навык**, если это справочный материал, который иногда нужен Claude (документация API, стайл-гайды), или рабочий процесс, который вы запускаете через `/<name>` (deploy, review, release).

**Практическое правило:** держите CLAUDE.md короче 200 строк. Если он разрастается, перенесите справочное содержимое в навыки или разбейте его на файлы [`.claude/rules/`](https://code.claude.com/docs/en/memory#organize-rules-with-claude/rules/).

### CLAUDE.md против правил против навыков

Все три хранят инструкции, но загружаются по-разному:

| Аспект       | CLAUDE.md                           | `.claude/rules/`                                   | Навык                                    |
| ------------ | ----------------------------------- | -------------------------------------------------- | ---------------------------------------- |
| **Загружается**    | Каждую сессию                       | Каждую сессию или при открытии подходящих файлов   | По запросу, при вызове или когда уместно      |
| **Область**    | Весь проект                       | Может быть ограничена путями к файлам                        | Специфична для задачи                            |
| **Лучше всего для** | Основных соглашений и команд сборки | Указаний, специфичных для языка или каталога | Справочных материалов, повторяемых рабочих процессов |

**Используйте CLAUDE.md** для инструкций, нужных в каждой сессии: команды сборки, соглашения по тестированию, архитектура проекта.

**Используйте правила**, чтобы CLAUDE.md оставался сфокусированным. Правила с [frontmatter `paths`](https://code.claude.com/docs/en/memory#path-specific-rules) загружаются только тогда, когда Claude работает с подходящими файлами, экономя контекст.

**Используйте навыки** для содержимого, которое нужно Claude лишь иногда, например документации API или чек-листа развёртывания, который вы запускаете через `/<name>`.

### Субагент против команды агентов

Оба распараллеливают работу, но архитектурно различаются:

* **Субагенты** выполняются внутри вашей сессии и передают результаты обратно в ваш основной контекст
* **Команды агентов** — это независимые сессии Claude Code, которые общаются друг с другом

| Аспект            | Субагент                                         | Команда агентов                                          |
| ----------------- | ------------------------------------------------ | --------------------------------------------------- |
| **Контекст**       | Собственное окно контекста; результаты возвращаются вызывающему | Собственное окно контекста; полностью независима               |
| **Коммуникация** | Передаёт результаты только основному агенту      | Участники обмениваются сообщениями напрямую               |
| **Координация**  | Основной агент управляет всей работой                      | Общий список задач с самокоординацией             |
| **Лучше всего для**      | Сфокусированных задач, где важен только результат      | Сложной работы, требующей обсуждения и совместной работы |
| **Стоимость в токенах**    | Ниже: результаты обобщаются обратно в основной контекст   | Выше: каждый участник — отдельный экземпляр Claude |

**Используйте субагента**, когда вам нужен быстрый, сфокусированный исполнитель: исследовать вопрос, проверить утверждение, отревьюить файл. Субагент выполняет работу и возвращает сводку. Ваш основной разговор остаётся чистым.

**Используйте команду агентов**, когда участникам нужно делиться находками, оспаривать друг друга и координироваться независимо. Команды агентов лучше всего подходят для исследования с конкурирующими гипотезами, параллельного ревью кода и разработки новых функций, где каждый участник отвечает за отдельную часть.

**Точка перехода:** если вы запускаете параллельных субагентов, но упираетесь в ограничения контекста, или если вашим субагентам нужно общаться друг с другом, команды агентов — естественный следующий шаг.

> **Примечание.** Команды агентов экспериментальны и отключены по умолчанию. О настройке и текущих ограничениях см. [команды агентов](https://code.claude.com/docs/en/agent-teams).

### MCP против навыка

MCP подключает Claude к внешним сервисам. Навыки расширяют то, что знает Claude, включая то, как эффективно использовать эти сервисы.

| Аспект         | MCP                                                  | Навык                                                   |
| -------------- | ---------------------------------------------------- | ------------------------------------------------------- |
| **Что это** | Протокол для подключения к внешним сервисам         | Знания, рабочие процессы и справочные материалы            |
| **Предоставляет**   | Инструменты и доступ к данным                                | Знания, рабочие процессы, справочные материалы            |
| **Примеры**   | Интеграция со Slack, запросы к базе данных, управление браузером | Чек-лист ревью кода, рабочий процесс развёртывания, стайл-гайд по API |

Они решают разные задачи и хорошо работают вместе:

**MCP** даёт Claude специально созданные инструменты для внешней системы, при этом подключение и аутентификация обрабатываются сервером.

**Навыки** дают Claude знания о том, как эффективно использовать эти инструменты, плюс рабочие процессы, которые вы можете запускать через `/<name>`. Навык может включать схему базы данных вашей команды и шаблоны запросов или рабочий процесс `/post-to-slack` с правилами форматирования сообщений вашей команды.

Пример: MCP-сервер подключает Claude к вашей базе данных. Навык обучает Claude вашей модели данных, распространённым шаблонам запросов и тому, какие таблицы использовать для разных задач.

### Хук против навыка

Хук срабатывает на событии жизненного цикла; навык загружается в контекст, чтобы Claude его применил.

| Аспект           | Хук                                                                              | Навык                                                                 |
| ---------------- | --------------------------------------------------------------------------------- | --------------------------------------------------------------------- |
| **Выполняет**         | Команду оболочки, HTTP-запрос, промпт для LLM или субагента                            | Инструкции, которые Claude читает и которым следует                                 |
| **Запускается** | [Событиями жизненного цикла](https://code.claude.com/docs/en/hooks#hook-events), такими как `PostToolUse` или `SessionStart` | Тем, что вы набираете `/<name>`, или Claude сопоставляет описание с вашей задачей |
| **Детерминизм**  | Всегда срабатывает на своём событии; триггер гарантирован                                | Claude интерпретирует инструкции; результат может варьироваться                  |
| **Стоимость в контексте** | Нулевая, если хук не возвращает вывод                                                | Описание загружается каждую сессию; полное содержимое — при использовании          |
| **Лучше всего для**     | Линтинга после редактирования, блокировки небезопасных команд, логирования, уведомлений             | Рабочих процессов, требующих рассуждения, справочных материалов, многошаговых задач   |

**Используйте хук**, когда действие должно происходить одинаково каждый раз и не требует, чтобы Claude думал. Например: форматирование при сохранении, отклонение `rm -rf /`, публикация сообщения в Slack при завершении сессии.

**Используйте навык**, когда Claude должен решать, как применять шаги, или когда содержимое — это знание, а не скрипт. Например: чек-лист `/release`, ваш стайл-гайд по API, плейбук отладки.

**Помещайте ограничители в хуки.** Инструкция вроде «никогда не редактируй `.env`» в CLAUDE.md или навыке — это просьба, а не гарантия. Хук `PreToolUse`, который блокирует редактирование, — это принуждение. Если правило должно соблюдаться каждый раз, сделайте его хуком, а не инструкцией в промпте.

**Вывод хука попадает в контекст.** Хук `PostToolUse`, который запускает ваш линтер, передаёт результаты обратно в виде текста, который Claude читает; навык `/fix-lint` говорит Claude, как их устранить.

### Разберитесь, как функции наслаиваются

Функции можно определять на нескольких уровнях: для всего пользователя, для отдельного проекта, через плагины или через управляемые политики. Вы также можете вкладывать файлы CLAUDE.md в подкаталоги или размещать навыки в конкретных пакетах монорепозитория. Когда одна и та же функция существует на нескольких уровнях, вот как они наслаиваются:

* **Файлы CLAUDE.md** аддитивны: все уровни одновременно вносят содержимое в контекст Claude. Файлы из вашего рабочего каталога и выше загружаются при запуске; подкаталоги загружаются по мере того, как вы в них работаете. Когда инструкции конфликтуют, Claude использует суждение для их согласования, при этом более конкретные инструкции обычно имеют приоритет. См. [как загружаются файлы CLAUDE.md](https://code.claude.com/docs/en/memory#how-claude-md-files-load).
* **Навыки и субагенты** переопределяются по имени: когда одно и то же имя существует на нескольких уровнях, побеждает одно определение на основе приоритета (управляемый > пользователь > проект для навыков; управляемый > флаг CLI > проект > пользователь > плагин для субагентов). Навыки плагинов имеют [пространства имён](https://code.claude.com/docs/en/plugins#add-skills-to-your-plugin) во избежание конфликтов. См. [обнаружение навыков](https://code.claude.com/docs/en/skills#where-skills-live) и [область субагентов](https://code.claude.com/docs/en/sub-agents#choose-the-subagent-scope).
* **MCP-серверы** переопределяются по имени: локальный > проект > пользователь. См. [область MCP](https://code.claude.com/docs/en/mcp#scope-hierarchy-and-precedence).
* **Хуки** объединяются: все зарегистрированные хуки срабатывают на своих подходящих событиях независимо от источника. См. [хуки](https://code.claude.com/docs/en/hooks).

### Комбинируйте функции

Каждое расширение решает свою задачу: CLAUDE.md обрабатывает всегда доступный контекст, навыки обрабатывают знания и рабочие процессы по запросу, MCP обрабатывает внешние подключения, субагенты обрабатывают изоляцию, а хуки обрабатывают автоматизацию. Реальные настройки комбинируют их в зависимости от вашего рабочего процесса.

Например, вы можете использовать CLAUDE.md для соглашений проекта, навык для рабочего процесса развёртывания, MCP для подключения к базе данных и хук для запуска линтинга после каждого редактирования. Каждая функция обрабатывает то, в чём она сильнее всего.

| Шаблон                | Как это работает                                                                     | Пример                                                                                           |
| ---------------------- | -------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| **Навык + MCP**        | MCP обеспечивает подключение; навык обучает Claude, как хорошо им пользоваться           | MCP подключается к вашей базе данных, навык документирует вашу схему и шаблоны запросов                   |
| **Навык + субагент**   | Навык порождает субагентов для параллельной работы                                       | Навык `/audit` запускает субагентов по безопасности, производительности и стилю, работающих в изолированном контексте |
| **CLAUDE.md + навыки** | CLAUDE.md хранит всегда действующие правила; навыки хранят справочный материал, загружаемый по запросу | CLAUDE.md говорит «следуй нашим соглашениям по API», навык содержит полный стайл-гайд по API            |
| **Хук + MCP**         | Хук запускает внешние действия через MCP                                     | Хук после редактирования отправляет уведомление в Slack, когда Claude изменяет критические файлы                     |

## Разберитесь в затратах контекста

Каждая функция, которую вы добавляете, потребляет часть контекста Claude. Слишком много может заполнить ваше окно контекста, но также может добавить шум, делающий Claude менее эффективным; навыки могут срабатывать некорректно, или Claude может терять из виду ваши соглашения. Понимание этих компромиссов помогает выстроить эффективную настройку. Для интерактивного представления того, как эти функции сочетаются в работающей сессии, см. [Исследуйте окно контекста](https://code.claude.com/docs/en/context-window).

### Затраты контекста по функциям

У каждой функции своя стратегия загрузки и своя стоимость в контексте:

| Функция               | Когда загружается                  | Что загружается                                          | Стоимость в контексте                                 |
| --------------------- | ------------------------------ | --------------------------------------------------- | -------------------------------------------- |
| **CLAUDE.md**         | Начало сессии                  | Полное содержимое                                        | Каждый запрос                                |
| **Навыки**            | Начало сессии + при использовании      | Описания в начале, полное содержимое при использовании       | Низкая (описания в каждом запросе)\*           |
| **MCP-серверы**       | Начало сессии                  | Имена инструментов; полные схемы по запросу                  | Низкая, пока инструмент не используется                 |
| **Интеллектуальный анализ кода** | После редактирования файлов и по запросу | Диагностика после редактирований; расположение символов при поиске | Низкая; сокращает чтение файлов в других местах            |
| **Субагенты**         | При порождении                   | Свежий контекст с указанными навыками                 | Изолирована от основной сессии                   |
| **Хуки**             | При срабатывании                     | Ничего (выполняются внешне)                           | Нулевая, если хук не возвращает дополнительный контекст |

\*По умолчанию описания навыков загружаются в начале сессии, чтобы Claude мог решать, когда их использовать. Установите `disable-model-invocation: true` во frontmatter навыка, чтобы полностью скрыть его от Claude, пока вы не вызовете его вручную. Это снижает стоимость в контексте до нуля для навыков, которые вы запускаете только сами. Для навыка, который написали не вы, установите [`skillOverrides`](https://code.claude.com/docs/en/skills#override-skill-visibility-from-settings) в настройках, чтобы добиться того же без редактирования его файла.

### Разберитесь, как загружаются функции

Каждая функция загружается в разные моменты вашей сессии. Вкладки ниже объясняют, когда загружается каждая из них и что попадает в контекст.

![Загрузка контекста: CLAUDE.md загружается в начале сессии и остаётся в каждом запросе. Имена инструментов MCP загружаются в начале, а полные схемы откладываются до использования. Навыки загружают описания в начале, полное содержимое — при вызове. Субагенты получают изолированный контекст. Хуки выполняются внешне.](https://mintcdn.com/claude-code/ikqp3_70mqIahteV/images/context-loading.svg?fit=max&auto=format&n=ikqp3_70mqIahteV&q=85&s=aab139e750494a237ae2e0c8f9139b0a)

### CLAUDE.md

**Когда:** начало сессии

**Что загружается:** полное содержимое всех файлов CLAUDE.md (управляемого, пользовательского и проектного уровней).

**Наследование:** Claude читает файлы CLAUDE.md от вашего рабочего каталога вверх до корня и обнаруживает вложенные в подкаталогах по мере обращения к этим файлам. Подробнее см. [Как загружаются файлы CLAUDE.md](https://code.claude.com/docs/en/memory#how-claude-md-files-load).

> **Совет.** Держите CLAUDE.md короче 200 строк. Перенесите справочный материал в навыки, которые загружаются по запросу.

### Навыки

Навыки — это дополнительные возможности в инструментарии Claude. Они могут быть справочным материалом (например, стайл-гайдом по API) или вызываемыми рабочими процессами, которые вы запускаете через `/<name>` (например, `/deploy`). Claude Code включает [встроенные навыки](https://code.claude.com/docs/en/commands) вроде `/code-review`, `/batch` и `/debug`, которые работают из коробки. Вы также можете создавать свои. Claude использует навыки, когда это уместно, или вы можете вызвать один напрямую.

**Когда:** зависит от конфигурации навыка. По умолчанию описания загружаются в начале сессии, а полное содержимое — при использовании. Для навыков только для пользователя (`disable-model-invocation: true`) ничего не загружается, пока вы их не вызовете.

**Что загружается:** для навыков, вызываемых моделью, Claude видит имена и описания в каждом запросе. Когда вы вызываете навык через `/<name>` или Claude загружает его автоматически, полное содержимое загружается в ваш разговор.

**Как Claude выбирает навыки:** Claude сопоставляет вашу задачу с описаниями навыков, чтобы решить, какие из них релевантны. Если описания расплывчаты или пересекаются, Claude может загрузить не тот навык или пропустить тот, что был бы полезен. Чтобы указать Claude использовать конкретный навык, вызовите его через `/<name>`. Навыки с `disable-model-invocation: true` невидимы для Claude, пока вы их не вызовете.

**Стоимость в контексте:** низкая до использования. Навыки только для пользователя имеют нулевую стоимость до вызова.

**В субагентах:** в субагентах навыки работают иначе. Вместо загрузки по запросу навыки, перечисленные в поле `skills` субагента, полностью предзагружаются в его контекст при запуске. Субагенты всё ещё могут обнаруживать и вызывать неперечисленные проектные, пользовательские и плагинные навыки через инструмент Skill.

> **Совет.** Используйте `disable-model-invocation: true` для навыков с побочными эффектами. Это экономит контекст и гарантирует, что запускаете их только вы.

### MCP-серверы

**Когда:** начало сессии.

**Что загружается:** имена инструментов с подключённых серверов. Полные JSON-схемы остаются отложенными, пока Claude не понадобится конкретный инструмент.

**Стоимость в контексте:** [поиск по инструментам](https://code.claude.com/docs/en/mcp#scale-with-mcp-tool-search) включён по умолчанию, поэтому простаивающие MCP-инструменты потребляют минимальный контекст.

> **Совет.** Запустите `/mcp`, чтобы увидеть статус подключения и затраты токенов по каждому серверу. Claude Code [переподключается к удалённым серверам автоматически](https://code.claude.com/docs/en/mcp#automatic-reconnection), если они отваливаются, и вы можете отключать серверы, которые не используете активно.

### Интеллектуальный анализ кода

**Когда:** после редактирования файлов и по запросу, когда Claude перемещается по коду.

**Что загружается:** ошибки типов и предупреждения после каждого редактирования файла. Информация об определении, ссылках и типе, когда Claude ищет символ.

**Стоимость в контексте:** низкая. Поиск символов часто заменяет широкое чтение файлов, поэтому чистое использование контекста может снизиться.

> **Совет.** Инструмент LSP неактивен, пока вы не установите [плагин интеллектуального анализа кода](https://code.claude.com/docs/en/discover-plugins#code-intelligence) для вашего языка.

### Субагенты

**Когда:** по запросу, когда вы или Claude порождаете одного для задачи.

**Что загружается:** свежий изолированный контекст, содержащий:

* Собственный системный промпт агента, а не полный системный промпт Claude Code
* Полное содержимое навыков, перечисленных в поле `skills:` агента
* CLAUDE.md и статус git, за исключением встроенных агентов Explore и Plan, которые [опускают оба](https://code.claude.com/docs/en/sub-agents#what-loads-at-startup)
* Любой контекст, который ведущий агент передаёт в промпте

**Стоимость в контексте:** изолирована от основной сессии. Субагенты не наследуют историю вашего разговора или вызванные навыки.

> **Совет.** Используйте субагентов для работы, которой не нужен полный контекст вашего разговора. Их изоляция предотвращает разбухание вашей основной сессии.

### Хуки

**Когда:** при срабатывании. Хуки срабатывают на определённых событиях жизненного цикла, таких как выполнение инструмента, границы сессии, отправка промпта, запросы разрешений и уплотнение. Полный список см. в [Хуки](https://code.claude.com/docs/en/hooks).

**Что загружается:** ничего по умолчанию. Хуки выполняются вне основного разговора.

**Стоимость в контексте:** нулевая, если хук не возвращает вывод, который добавляется в виде сообщений в ваш разговор.

> **Совет.** Хуки идеальны для побочных эффектов (линтинг, логирование), которые не должны влиять на контекст Claude.

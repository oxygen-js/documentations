# Как работает Claude Code

> Разберитесь в агентном цикле, встроенных инструментах и в том, как Claude Code взаимодействует с вашим проектом.

Claude Code — это агентный помощник, который работает в вашем терминале. Он отлично справляется с программированием, но также может помочь с чем угодно, что вы можете сделать из командной строки: написание документации, запуск сборок, поиск файлов, изучение тем и многое другое.

Это руководство охватывает базовую архитектуру, встроенные возможности и [советы по эффективной работе](#work-effectively-with-claude-code). Пошаговые разборы см. в разделе [Типовые рабочие процессы](https://code.claude.com/docs/en/common-workflows). Возможности расширения, такие как навыки, MCP и хуки, описаны в разделе [Расширение Claude Code](https://code.claude.com/docs/en/features-overview).

## Агентный цикл

Когда вы даёте Claude задачу, он проходит через три фазы: **сбор контекста**, **выполнение действий** и **проверка результатов**. Эти фазы переплетаются между собой. Claude использует инструменты на всём протяжении работы — будь то поиск файлов для понимания вашего кода, редактирование для внесения изменений или запуск тестов для проверки своей работы.

![Диаграмма агентного цикла: ваш промпт приводит к тому, что Claude собирает контекст, выполняет действия, проверяет результаты и повторяет это до завершения задачи. Вы можете прервать процесс в любой момент.](https://mintcdn.com/claude-code/ikqp3_70mqIahteV/images/agentic-loop.svg?fit=max&auto=format&n=ikqp3_70mqIahteV&q=85&s=4a30fb7ce2815012a9f27c955e2c6bb0)

Цикл подстраивается под то, о чём вы просите. Вопрос о вашей кодовой базе может потребовать лишь сбора контекста. Исправление бага циклически проходит через все три фазы неоднократно. Рефакторинг может потребовать обширной проверки. Claude решает, что нужно на каждом шаге, исходя из того, что он узнал на предыдущем шаге, связывая воедино десятки действий и корректируя курс по ходу дела.

Вы тоже являетесь частью этого цикла. Вы можете вмешаться в любой момент, чтобы направить Claude в другую сторону, предоставить дополнительный контекст или попросить его попробовать иной подход. Claude работает автономно, но остаётся отзывчивым к вашим указаниям.

Агентный цикл работает за счёт двух компонентов: [моделей](#models), которые рассуждают, и [инструментов](#tools), которые действуют. Claude Code служит **агентной обвязкой (harness)** вокруг Claude: он предоставляет инструменты, управление контекстом и среду выполнения, которые превращают языковую модель в способного к программированию агента.

### Модели

Claude Code использует модели Claude, чтобы понимать ваш код и рассуждать о задачах. Claude может читать код на любом языке, понимать, как связаны компоненты, и выяснять, что нужно изменить для достижения вашей цели. Для сложных задач он разбивает работу на шаги, выполняет их и корректируется на основе того, что узнаёт.

Доступно [несколько моделей](https://code.claude.com/docs/en/model-config) с разными компромиссами. Sonnet хорошо справляется с большинством задач программирования. Opus обеспечивает более сильное рассуждение для сложных архитектурных решений. Переключайтесь с помощью `/model` во время сессии или запускайте с `claude --model <name>`.

Когда в этом руководстве говорится «Claude выбирает» или «Claude решает», рассуждение выполняет именно модель.

### Инструменты

Инструменты — это то, что делает Claude Code агентным. Без инструментов Claude может лишь отвечать текстом. С инструментами Claude может действовать: читать ваш код, редактировать файлы, выполнять команды, искать в интернете и взаимодействовать с внешними сервисами. Каждое использование инструмента возвращает информацию, которая снова поступает в цикл, определяя следующее решение Claude.

Встроенные инструменты в целом делятся на пять категорий, каждая из которых представляет свой вид агентности.

| Категория              | Что может делать Claude                                                                                                                                            |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Операции с файлами**   | Читать файлы, редактировать код, создавать новые файлы, переименовывать и реорганизовывать                                                                                                |
| **Поиск**            | Находить файлы по шаблону, искать содержимое с помощью регулярных выражений, исследовать кодовые базы                                                                                          |
| **Выполнение**         | Выполнять команды оболочки, запускать серверы, запускать тесты, использовать git                                                                                                    |
| **Интернет**               | Искать в интернете, получать документацию, искать сообщения об ошибках                                                                                                   |
| **Понимание кода (code intelligence)** | Видеть ошибки типов и предупреждения после редактирования, переходить к определениям, находить ссылки (требуются [плагины понимания кода](https://code.claude.com/docs/en/discover-plugins#code-intelligence)) |

Это основные возможности. У Claude также есть инструменты для порождения субагентов, для того чтобы задавать вам вопросы, и для других задач оркестрации. Полный список см. в разделе [Инструменты, доступные Claude](https://code.claude.com/docs/en/tools-reference).

Claude выбирает, какие инструменты использовать, исходя из вашего промпта и того, что он узнаёт по ходу дела. Когда вы говорите «исправь падающие тесты», Claude может:

1. Запустить набор тестов, чтобы увидеть, что не проходит
2. Прочитать вывод с ошибками
3. Найти соответствующие исходные файлы
4. Прочитать эти файлы, чтобы понять код
5. Отредактировать файлы, чтобы исправить проблему
6. Снова запустить тесты для проверки

Каждое использование инструмента даёт Claude новую информацию, которая определяет следующий шаг. Это и есть агентный цикл в действии.

**Расширение базовых возможностей.** Встроенные инструменты — это фундамент. Вы можете расширить знания Claude с помощью [навыков (skills)](https://code.claude.com/docs/en/skills), подключиться к внешним сервисам с помощью [MCP](https://code.claude.com/docs/en/mcp), автоматизировать рабочие процессы с помощью [хуков](https://code.claude.com/docs/en/hooks) и делегировать задачи [субагентам](https://code.claude.com/docs/en/sub-agents). Эти расширения образуют слой поверх базового агентного цикла. Рекомендации по выбору подходящего расширения под ваши нужды см. в разделе [Расширение Claude Code](https://code.claude.com/docs/en/features-overview).

## К чему Claude имеет доступ

Это руководство сосредоточено на терминале. Claude Code также работает в [VS Code](https://code.claude.com/docs/en/vs-code), [IDE от JetBrains](https://code.claude.com/docs/en/jetbrains) и других средах.

Когда вы запускаете `claude` в каталоге, Claude Code получает доступ к:

* **Вашему проекту.** Файлы в вашем каталоге и подкаталогах, а также файлы в других местах с вашего разрешения.
* **Вашему терминалу.** Любая команда, которую вы могли бы выполнить: инструменты сборки, git, менеджеры пакетов, системные утилиты, скрипты. Если вы можете сделать это из командной строки, то и Claude может.
* **Состоянию вашего git.** Текущая ветка, незакоммиченные изменения и недавняя история коммитов.
* **Вашему [CLAUDE.md](https://code.claude.com/docs/en/memory).** Markdown-файл, в котором вы храните специфичные для проекта инструкции, соглашения и контекст, которые Claude должен знать в каждой сессии.
* **[Автопамяти](https://code.claude.com/docs/en/memory#auto-memory).** Знания, которые Claude сохраняет автоматически по ходу вашей работы, например паттерны проекта и ваши предпочтения. Первые 200 строк или 25 КБ файла MEMORY.md, в зависимости от того, что наступит раньше, загружаются в начале каждой сессии.
* **Настроенным вами расширениям.** [Серверы MCP](https://code.claude.com/docs/en/mcp) для внешних сервисов, [навыки](https://code.claude.com/docs/en/skills) для рабочих процессов, [субагенты](https://code.claude.com/docs/en/sub-agents) для делегированной работы и [Claude в Chrome](https://code.claude.com/docs/en/chrome) для взаимодействия с браузером.

Поскольку Claude видит весь ваш проект, он может работать по всему проекту. Когда вы просите Claude «исправить баг с аутентификацией», он ищет соответствующие файлы, читает несколько файлов, чтобы понять контекст, вносит согласованные правки в них, запускает тесты для проверки исправления и коммитит изменения, если вы попросите. Это отличается от встроенных помощников по коду, которые видят только текущий файл.

## Среды и интерфейсы

Агентный цикл, инструменты и возможности, описанные выше, одинаковы везде, где бы вы ни использовали Claude Code. Меняется лишь то, где выполняется код и как вы с ним взаимодействуете.

### Среды выполнения

Claude Code работает в трёх средах, каждая из которых имеет разные компромиссы в отношении того, где выполняется ваш код.

| Среда        | Где выполняется код                         | Сценарий использования                                                   |
| ------------------ | --------------------------------------- | ---------------------------------------------------------- |
| **Локально (Local)**          | На вашей машине                            | По умолчанию. Полный доступ к вашим файлам, инструментам и среде |
| **Облако (Cloud)**          | На виртуальных машинах под управлением Anthropic                   | Делегирование задач, работа с репозиториями, которых у вас нет локально        |
| **Удалённое управление (Remote Control)** | На вашей машине, управление из браузера | Использование веб-интерфейса при сохранении всего локально              |

### Интерфейсы

Вы можете получить доступ к Claude Code через терминал, [десктопное приложение](https://code.claude.com/docs/en/desktop), [расширения IDE](https://code.claude.com/docs/en/vs-code), [claude.ai/code](https://claude.ai/code), [Удалённое управление](https://code.claude.com/docs/en/remote-control), [Slack](https://code.claude.com/docs/en/slack) и [конвейеры CI/CD](https://code.claude.com/docs/en/github-actions). Интерфейс определяет, как вы видите Claude и взаимодействуете с ним, но лежащий в основе агентный цикл идентичен. Полный список см. в разделе [Используйте Claude Code повсюду](https://code.claude.com/docs/en/overview#use-claude-code-everywhere).

## Работа с сессиями

Claude Code сохраняет вашу беседу локально по ходу работы. Каждое сообщение, использование инструмента и результат записываются в текстовый файл JSONL в каталоге `~/.claude/projects/`, что делает возможными [перемотку](#undo-changes-with-checkpoints), [возобновление и ответвление](#resume-or-fork-sessions) сессий. Прежде чем Claude вносит изменения в код, он также делает снимок затронутых файлов, чтобы вы могли откатиться при необходимости. Пути, сроки хранения и информацию о том, как очистить эти данные, см. в разделе [данные приложения в `~/.claude`](https://code.claude.com/docs/en/claude-directory#application-data).

**Сессии независимы.** Каждая новая сессия начинается со свежим окном контекста, без истории беседы из предыдущих сессий. Claude может сохранять знания между сессиями с помощью [автопамяти](https://code.claude.com/docs/en/memory#auto-memory), а вы можете добавлять собственные постоянные инструкции в [CLAUDE.md](https://code.claude.com/docs/en/memory).

### Работа между ветками

Каждая беседа в Claude Code — это сессия, привязанная к вашему текущему каталогу. Выбор `/resume` по умолчанию показывает сессии из текущего рабочего дерева (worktree), с сочетаниями клавиш для расширения списка на другие рабочие деревья или проекты. Полный список сочетаний клавиш для выбора и информацию о том, как работает разрешение имён, см. в разделе [Управление сессиями](https://code.claude.com/docs/en/sessions#use-the-session-picker).

Claude видит файлы вашей текущей ветки. Когда вы переключаете ветки, Claude видит файлы новой ветки, но ваша история беседы остаётся прежней. Claude помнит, что вы обсуждали, даже после переключения.

Поскольку сессии привязаны к каталогам, вы можете запускать параллельные сессии Claude, используя [рабочие деревья git (worktrees)](https://code.claude.com/docs/en/worktrees), которые создают отдельные каталоги для отдельных веток.

### Возобновление или ответвление сессий

Возобновление сессии с помощью `claude --continue` или `claude --resume` заново открывает её под тем же идентификатором сессии и добавляет новые сообщения к существующей беседе. Ответвление с помощью `--fork-session` или `/branch` копирует историю в новый идентификатор сессии, оставляя оригинал без изменений.

![Диаграмма непрерывности сессии: возобновление продолжает ту же сессию, ответвление создаёт новую ветвь с новым идентификатором.](https://mintcdn.com/claude-code/ikqp3_70mqIahteV/images/session-continuity.svg?fit=max&auto=format&n=ikqp3_70mqIahteV&q=85&s=04ed0984a58e4127e05b3640265241a3)

Информацию о флагах возобновления, выборе `/resume`, именовании и о том, что происходит, когда одна и та же сессия открыта в двух терминалах, см. в разделе [Управление сессиями](https://code.claude.com/docs/en/sessions).

### Окно контекста

Окно контекста Claude содержит историю вашей беседы, содержимое файлов, вывод команд, [CLAUDE.md](https://code.claude.com/docs/en/memory), [автопамять](https://code.claude.com/docs/en/memory#auto-memory), загруженные навыки и системные инструкции. По ходу работы контекст заполняется. Claude сжимает его автоматически, но инструкции из ранней части беседы могут потеряться. Помещайте постоянные правила в CLAUDE.md и запускайте `/context`, чтобы увидеть, что занимает место.

Интерактивный разбор того, что загружается и когда, см. в разделе [Исследуйте окно контекста](https://code.claude.com/docs/en/context-window).

#### Когда контекст заполняется

Claude Code управляет контекстом автоматически по мере приближения к лимиту. Сначала он очищает более старые выводы инструментов, затем при необходимости резюмирует беседу. Ваши запросы и ключевые фрагменты кода сохраняются; подробные инструкции из ранней части беседы могут быть утрачены. Помещайте постоянные правила в CLAUDE.md, а не полагайтесь на историю беседы.

Чтобы управлять тем, что сохраняется при сжатии, добавьте в CLAUDE.md раздел «Compact Instructions» или запустите `/compact` с фокусом (например, `/compact focus on the API changes`).

Если отдельный файл или вывод инструмента настолько велик, что контекст заполняется заново сразу после каждого резюме, Claude Code прекращает автосжатие после нескольких попыток и показывает ошибку вместо зацикливания. Действия по восстановлению см. в разделе [Автосжатие останавливается с ошибкой пробуксовки](https://code.claude.com/docs/en/troubleshooting#auto-compaction-stops-with-a-thrashing-error).

Запустите `/context`, чтобы увидеть, что занимает место. Определения инструментов MCP по умолчанию откладываются и загружаются по требованию через [поиск инструментов](https://code.claude.com/docs/en/mcp#scale-with-mcp-tool-search), поэтому до тех пор, пока Claude не использует конкретный инструмент, контекст расходуют лишь имена инструментов. Запустите `/mcp`, чтобы проверить затраты по каждому серверу.

#### Управление контекстом с помощью навыков и субагентов

Помимо сжатия, вы можете использовать другие возможности для управления тем, что загружается в контекст.

[Навыки](https://code.claude.com/docs/en/skills) загружаются по требованию. Claude видит описания навыков в начале сессии, но полное содержимое загружается только тогда, когда навык используется. Для навыков, которые вы вызываете вручную, установите `disable-model-invocation: true`, чтобы описания не попадали в контекст, пока они вам не понадобятся. Для навыков, которые написали не вы, используйте [`skillOverrides`](https://code.claude.com/docs/en/skills#override-skill-visibility-from-settings), чтобы сделать то же самое из настроек.

[Субагенты](https://code.claude.com/docs/en/sub-agents) получают собственный свежий контекст, полностью отдельный от вашей основной беседы. Их работа не раздувает ваш контекст. По завершении они возвращают резюме. Именно эта изоляция делает субагентов полезными для длинных сессий.

Информацию о том, чего стоит каждая возможность, см. в разделе [затраты контекста](https://code.claude.com/docs/en/features-overview#understand-context-costs), а советы по управлению контекстом — в разделе [сокращение расхода токенов](https://code.claude.com/docs/en/costs#reduce-token-usage).

## Оставайтесь в безопасности с помощью контрольных точек и разрешений

У Claude есть два механизма безопасности: контрольные точки позволяют отменять изменения файлов, а разрешения контролируют, что Claude может делать без спроса.

### Отмена изменений с помощью контрольных точек

**Каждая правка файла обратима.** Прежде чем Claude отредактирует какой-либо файл, он делает снимок текущего содержимого. Если что-то пойдёт не так, дважды нажмите `Esc`, чтобы перемотать к предыдущему состоянию, или попросите Claude отменить изменения.

Контрольные точки локальны для вашей сессии и отделены от git. Они охватывают только изменения файлов. Действия, затрагивающие удалённые системы (базы данных, API, развёртывания), нельзя зафиксировать контрольной точкой — именно поэтому Claude спрашивает, прежде чем выполнять команды с внешними побочными эффектами.

### Контроль того, что может делать Claude

Нажмите `Shift+Tab`, чтобы переключаться между режимами разрешений:

* **Default (По умолчанию)**: Claude спрашивает перед правками файлов и командами оболочки
* **Auto-accept edits (Автопринятие правок)**: Claude редактирует файлы и выполняет распространённые команды файловой системы, такие как `mkdir` и `mv`, без спроса, но по-прежнему спрашивает для других команд
* **Plan mode (Режим планирования)**: Claude исследует и предлагает план, не редактируя ваши исходные файлы; запросы разрешений применяются так же, как в режиме по умолчанию
* **Auto mode (Автоматический режим)**: Claude оценивает все действия с помощью фоновых проверок безопасности. В настоящее время это исследовательская предварительная версия (research preview)

Вы также можете разрешить определённые команды в `.claude/settings.json`, чтобы Claude не спрашивал каждый раз. Это полезно для доверенных команд, таких как `npm test` или `git status`. Настройки можно применять на разных уровнях — от общеорганизационных политик до личных предпочтений. Подробности см. в разделе [Разрешения](https://code.claude.com/docs/en/permissions).

## Эффективная работа с Claude Code

Эти советы помогут вам получать лучшие результаты от Claude Code.

### Просите помощи у Claude Code

Claude Code может научить вас пользоваться им. Задавайте вопросы вроде «как настроить хуки?» или «как лучше всего структурировать мой CLAUDE.md?», и Claude объяснит.

Встроенные команды также проводят вас через настройку:

* `/init` проводит вас через создание CLAUDE.md для вашего проекта
* `/agents` помогает настроить пользовательских субагентов
* `/doctor` диагностирует распространённые проблемы с вашей установкой

### Это беседа

Claude Code — диалоговый. Вам не нужны идеальные промпты. Начните с того, что вы хотите, а затем уточняйте:

```text
Fix the login bug
```

\[Claude исследует, что-то пробует]

```text
That's not quite right. The issue is in the session handling.
```

\[Claude корректирует подход]

Когда первая попытка неудачна, вы не начинаете заново. Вы итерируете.

#### Прерывайте и направляйте

Вы можете перенаправить Claude в любой момент, не дожидаясь окончания хода и не начиная заново:

* **Нажмите `Esc`**, чтобы немедленно остановить Claude. Выполняемый вызов инструмента отменяется, и Claude ждёт вашей следующей инструкции.
* **Введите поправку и нажмите `Enter`**, чтобы отправить её, не останавливая выполняемый инструмент. Claude прочитает её, как только завершится текущее действие, и скорректируется, прежде чем решать, каким будет следующий шаг.

### Будьте конкретны с самого начала

Чем точнее ваш первоначальный промпт, тем меньше поправок вам понадобится. Ссылайтесь на конкретные файлы, упоминайте ограничения и указывайте на примеры паттернов.

```text
The checkout flow is broken for users with expired cards.
Check src/payments/ for the issue, especially token refresh.
Write a failing test first, then fix it.
```

Расплывчатые промпты работают, но вы потратите больше времени на управление. Конкретные промпты, как приведённый выше, часто успешны с первой попытки.

### Дайте Claude то, с чем можно сверяться

Claude работает лучше, когда может проверить свою собственную работу. Включайте тестовые случаи, вставляйте скриншоты ожидаемого интерфейса или определяйте желаемый вывод.

```text
Implement validateEmail. Test cases: 'user@example.com' → true,
'invalid' → false, 'user@.com' → false. Run the tests after.
```

Для визуальной работы вставьте скриншот дизайна и попросите Claude сравнить свою реализацию с ним.

### Исследуйте, прежде чем реализовывать

Для сложных задач отделяйте исследование от программирования. Используйте режим планирования (`Shift+Tab` дважды), чтобы сначала проанализировать кодовую базу:

```text
Read src/auth/ and understand how we handle sessions.
Then create a plan for adding OAuth support.
```

Просмотрите план, доработайте его через беседу, а затем позвольте Claude реализовать его. Такой двухфазный подход даёт лучшие результаты, чем переход сразу к коду.

### Делегируйте, а не диктуйте

Думайте об этом как о делегировании способному коллеге. Дайте контекст и направление, а затем доверьте Claude разобраться в деталях:

```text
The checkout flow is broken for users with expired cards.
The relevant code is in src/payments/. Can you investigate and fix it?
```

Вам не нужно указывать, какие файлы читать или какие команды выполнять. Claude выясняет это сам.

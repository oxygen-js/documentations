# Начинаем преобразовывать потоки с помощью map, pluck и mapTo

При работе с observable один из самых частых сценариев, с которыми вы столкнётесь, —
необходимость преобразовать поток значений одного типа в поток значений
другого типа. Например, у вас может быть observable событий клика, который вы
хотите преобразовать в observable объектов, содержащих только координаты
`clientX` и `clientY`. Или, может быть, вам нужно извлечь значение из
потока событий ввода, чтобы выполнить вычисление или инициировать запрос. Или,
возможно, вы просто хотите извлечь из объекта одно свойство, например код
клавиши, чтобы выполнить дальнейшее действие по (pipe)-цепочке. Сценарии
преобразования потоков бесконечны.

В этой статье мы познакомимся с самым распространённым оператором для
преобразования потоков — оператором `map`. Начнём с того, что рассмотрим
`Array.map`, чтобы получить общее представление об операции `map`. Затем мы
изучим, как применить этот подход к observable с помощью оператора RxJS
`map`. Наконец, мы разберём несколько вспомогательных операторов, которые можно
использовать вместо `map`, если ситуация подходящая, попутно рассматривая
типичные сценарии применения. Приступим!

## Знакомство с `map`

Если вы уже поработали с массивами JavaScript, то, возможно, знакомы
с `Array.map`. При работе с массивами метод `map` позволяет
преобразовать массив, применяя предоставленную функцию (её часто называют функцией
«проекции») к каждому элементу массива. Например, допустим, у нас
есть массив чисел `1-5`:

```js
const numbers = [1, 2, 3, 4, 5];
```

Если бы мы захотели превратить его в массив, где каждое число умножено на десять,
мы могли бы использовать метод `map`. Для этого мы вызываем `map` на нашем массиве чисел,
передавая ему функцию, которая будет вызвана с каждым значением исходного массива
и вернёт число, умноженное на десять:

```js
const numbers = [1, 2, 3, 4, 5];
const numbersTimesTen = numbers.map(number => number * 10);

// [10,20,30,40,50]
console.log(numbersTimesTen);
```

Метод `map` не изменяет существующий массив, а вместо этого возвращает новый
массив. Например, если мы выведем массив `numbers` после вызова `map`,
мы увидим, что он не изменился:

```js
const numbers = [1, 2, 3, 4, 5];
const numbersTimesTen = numbers.map(number => number * 10);

// [10,20,30,40,50]
console.log(numbersTimesTen);

// [1,2,3,4,5]
console.log(numbers);
```

Чтобы лучше это понять, давайте пройдёмся по тому, как могла бы выглядеть наивная
реализация `Array.map`.

1. Мы создаём новый массив.
2. Для каждого элемента исходного массива мы применяем предоставленную функцию.
3. Затем мы добавляем результат этой функции во временный массив `resultArray`.
4. Проделав это для каждого элемента, мы возвращаем новый массив.

```js
Array.prototype.map = function(projectFn) {
  let resultArray = [];
  // проходим по каждому элементу
  this.forEach(item => {
    // применяем предоставленную функцию проекции
    let result = projectFn(item);
    // добавляем результат в наш новый массив
    resultArray.push(result);
  });
  // возвращаем массив с преобразованными значениями
  return resultArray;
};
```

Хотя
[настоящая реализация](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/map)
`Array.map` включает такие возможности, как отслеживание индекса и корректную
обработку ошибок, это даёт нам общее представление о том, как всё работает под капотом.

{% hint style="info" %}

RxJS также предлагает observable-варианты других популярных методов массивов, таких как
[`filter`](<../02. Operators/06. Filtering/08. filter.md>),
[`reduce`](<../02. Operators/07. Transformation/17. reduce.md>) и
[`find`](<../02. Operators/06. Filtering/09. find.md>)!

{% endhint %}

Итак, в каких ещё типичных сценариях мы могли бы применить метод `map`?
С помощью `Array.map` мы также можем преобразовывать объекты. Например,
предположим, у нас есть массив объектов со свойствами имени и фамилии, и мы
хотим добавить к каждому объекту свойство полного имени. Мы могли бы сделать это,
предоставив функцию, которая принимает каждый объект и _отображает_ его на новый объект,
включающий все текущие свойства плюс новое свойство `fullName`. В этом
примере мы используем
[синтаксис расширения объектов (object spread)](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Spread_syntax)
и
[шаблонные литералы](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Template_literals),
но вы могли бы также явно переписать свойства:

```js
const people = [
  { firstName: 'Brian', lastName: 'Troncone' },
  { firstName: 'Todd', lastName: 'Motto' }
];
const peopleWithFullName = people.map(person => ({
  ...person,
  fullName: `${person.firstName} ${person.lastName}`
}));

// [{ firstName: 'Brian', lastName: 'Troncone', fullName: 'Brian Troncone' }, {firstName: 'Todd', lastName: 'Motto', fullName: 'Todd Motto' }]
console.log(peopleWithFullName);
```

Ещё один типичный сценарий использования `map` — извлечение одного свойства из
объекта. Например, для приведённого выше образца предположим, что мы решили, что нам
_на самом деле_ нужно только свойство фамилии для отображения. Вместо того чтобы наша функция возвращала новый
объект, мы можем вернуть лишь нужное нам свойство этого объекта:

```js
const people = [
  { firstName: 'Brian', lastName: 'Troncone' },
  { firstName: 'Todd', lastName: 'Motto' }
];
const lastNames = people.map(person => person.lastName);

// [ 'Troncone', 'Motto' ]
console.log(lastNames);
```

На этом этапе мы преобразуем массив объектов людей в массив
строк-фамилий.

Как видите, метод `map` чрезвычайно гибок и имеет огромное разнообразие
сценариев применения, но как это переносится на `map` в RxJS и когда вы стали бы
применять его к observable?

## Знакомство с оператором RxJS `map`

![map](https://drive.google.com/uc?export=view&id=1fbxzA5p0FFFUTo0dOAanoq6s1LBip3Ga)

Оператор `map` в RxJS преобразует значения, испускаемые исходным observable,
на основе предоставленной функции проекции. Это похоже на `Array.map`, за исключением того, что
мы работаем с каждым значением, испускаемым observable по мере его появления, а не
с каждым значением, содержащимся в массиве.

Например, начнём с нашего исходного примера, но вместо преобразования
массива чисел давайте преобразуем observable чисел. Для этого мы
используем оператор создания [`from`](<../02. Operators/03. Creation/05. from.md>), чтобы сначала
преобразовать наш массив чисел в observable:

```js
import { from } from 'rxjs';

const numbers = [1, 2, 3, 4, 5];
const number$ = from(numbers);
```

Получив массив, оператор создания `from` будет проходить по нему
(синхронно), испуская каждый элемент по очереди. При подписке мы видим,
как каждое значение печатается в консоль:

```js
import { from } from 'rxjs';

const numbers = [1, 2, 3, 4, 5];
const number$ = from(numbers);

/*
 * 1
 * 2
 * 3
 * 4
 * 5
 */
number$.subscribe(console.log);
```

**Совет:** _Если вы хотите увидеть, как `from` обрабатывает каждый тип значения под
капотом, вы можете изучить
[`subscribeTo`](https://github.com/ReactiveX/rxjs/blob/e17df333fec66ea3d79e3f70565064f757c3a4fe/src/internal/util/subscribeTo.ts#L14-L29)
и связанные с ним вспомогательные функции. В данном случае используется
[subscribeToArray](https://github.com/ReactiveX/rxjs/blob/e17df333fec66ea3d79e3f70565064f757c3a4fe/src/internal/util/subscribeToArray.ts#L7-L12).
Эта же вспомогательная функция также используется для обработки не-observable
возвращаемых значений разворачивающих (flattening) операторов, таких как
[mergeMap](<../02. Operators/07. Transformation/13. mergeMap - flatMap.md>)_

Если бы затем мы захотели преобразовать этот observable в испускаемые значения,
умноженные на десять, мы могли бы использовать оператор `map`. Как и `Array.map`, оператор
`map` принимает функцию проекции, которая описывает, как каждое значение из
источника будет преобразовано. В данном случае мы предоставим функцию, которая
принимает значение, испущенное исходным observable, и возвращает это значение
умноженным:

```js
import { from } from 'rxjs';

const numbers = [1, 2, 3, 4, 5];
const number$ = from(numbers);
const numbersMultipliedByTen$ = number$.pipe(map(number => number * 10));

/*
 * 10
 * 20
 * 30
 * 40
 * 50
 */
numbersMultipliedByTen$.subscribe(console.log);
```

Вместо того чтобы функция применялась к каждому элементу массива перед тем, как
будет возвращён новый массив, с observable функция проекции применяется, и
результат испускается в реальном времени по мере того, как значения проносятся через ваши потоки. Мы можем убедиться
в этом в исходном коде RxJS, увидев, что предоставленная нами функция вызывается, а
результат передаётся подписчику (получателю, destination):

[(Исходный код)](https://github.com/ReactiveX/rxjs/blob/e17df333fec66ea3d79e3f70565064f757c3a4fe/packages/rxjs/src/internal/operators/map.ts#L81-L91)

```ts
protected _next(value: T) {
    let result: any;
    try {
      // project — это функция, которую мы передаём оператору map
      result = this.project.call(this.thisArg, value, this.count++);
    } catch (err) {
    // пробрасываем любые возникшие ошибки
      this.destination.error(err);
      return;
    }
    // испускаем результат вызова нашей функции проекции подписчику
    this.destination.next(result);
  }
```

Подобно нашему примеру с массивом объектов, мы также можем захотеть преобразовать
observable объектов с помощью оператора `map`. Например, предположим, у нас есть
observable событий `click`, который мы хотим преобразовать в observable
объектов, содержащих только координаты `clientX` и `clientY` этих событий.
Для этого мы могли бы применить оператор `map`, предоставив функцию, которая возвращает
объект только с этими свойствами:

```js
import { fromEvent } from 'rxjs';
import { map } from 'rxjs/operators';

const click$ = fromEvent(document, 'click');

click$
  .pipe(
    map(event => ({
      x: event.clientX,
      y: event.clientY
    }))
    // { x: 12, y: 45 }, { x: 23, y: 132 }
  )
  .subscribe(console.log);
```

Также могут быть случаи, когда мы хотим извлечь из объекта одно свойство с помощью
`map`. Например, у нас может быть сценарий, где нужен observable только свойства `code`
из событий `keyup`, чтобы мы могли предпринять действие, когда пользователь нажимает
определённый символ или клавишу. Для этого мы можем применить оператор `map`,
возвращая только интересующее нас свойство:

```js
import { fromEvent } from 'rxjs';
import { map } from 'rxjs/operators';

const keyup$ = fromEvent(document, 'keyup');

keyup$
  .pipe(map(event => event.code))
  // 'Space', 'Enter'
  .subscribe(console.log);
```

Хотя `map` прекрасно справляется в этих ситуациях, RxJS также предоставляет вспомогательные
операторы для случаев, когда вы просто хотите _отобразить_ (map) на одно свойство или когда
вы _всегда_ хотите отображать на одно и то же значение при любом событии. Сначала давайте рассмотрим
сценарий с одним свойством.

## Извлечение одного свойства с помощью `pluck`

![pluck](https://drive.google.com/uc?export=view&id=1-TdTqWb-qoif4FJojY3sC0oS81EkB65z)

RxJS содержит множество операторов, которые представляют собой просто сокращения для других операторов.
Например, всякий раз, когда мы хотим просто взять одно свойство из испущенного значения,
вместо `map` мы могли бы использовать `pluck`. Оператор `pluck` принимает список
значений, описывающих свойство, которое вы хотите взять из испущенного элемента.
Например, используя наш пример с кодом события выше, мы могли бы использовать `pluck`
вместо `map`, чтобы извлечь свойство `code` из объекта `event`:

```js
import { fromEvent } from 'rxjs';
import { pluck } from 'rxjs/operators';

const keyup$ = fromEvent(document, 'keyup');

keyup$
  .pipe(pluck('code'))
  // 'Space', 'Enter'
  .subscribe(console.log);
```

Мы также можем передать `pluck` несколько значений, чтобы взять вложенное свойство внутри
объекта. Например, если бы мы хотели взять `nodeName` из элемента `target`
при клике, мы могли бы передать оба этих свойства оператору `pluck` по порядку:

```js
import { fromEvent } from 'rxjs';
import { pluck } from 'rxjs/operators';

const click$ = fromEvent(document, 'click');

click$
  .pipe(pluck('target', 'nodeName'))
  // 'DIV', 'MAIN'
  .subscribe(console.log);
```

Как и многие другие вспомогательные операторы в RxJS, под капотом `pluck` просто
повторно использует оператор `map`, передавая ему функцию для взятия нужного
свойства:

[(Исходный код)](https://github.com/ReactiveX/rxjs/blob/e17df333fec66ea3d79e3f70565064f757c3a4fe/packages/rxjs/src/internal/operators/pluck.ts#L48-L54)

```ts
export function pluck<T, R>(...properties: string[]): OperatorFunction<T, R> {
  const length = properties.length;
  if (length === 0) {
    throw new Error('list of properties cannot be empty.');
  }
  return (source: Observable<T>) =>
    map(plucker(properties, length))(source as any);
}
```

Функционально `map` и `pluck` в этих сценариях работают одинаково, я
советовал бы использовать тот, который вам удобнее читать с первого взгляда.

Наконец, могут быть случаи, когда вы **всегда** хотите отобразить на одно
значение, независимо от входных данных. Для таких ситуаций вы можете использовать оператор
`mapTo`.

## Отображение на постоянное значение с помощью `mapTo`

![mapTo](https://drive.google.com/uc?export=view&id=1329klWDEvjgjh3JVJkzHoZDc7CpcLZvO)

Для ситуаций, когда вы обнаруживаете, что всегда хотите отобразить на конкретное
значение, один из способов справиться с этим — просто использовать `map`, игнорируя
входное значение:

```js
import { fromEvent } from 'rxjs';
import { map } from 'rxjs/operators';

const click$ = fromEvent(document, 'click');

click$
  .pipe(map(() => 'You clicked!'))
  // 'You clicked!', 'You clicked!'
  .subscribe(console.log);
```

Хотя это работает, оборачивающая функция не нужна, поскольку мы игнорируем
полученное значение. Для таких сценариев вы можете заменить `map` на `mapTo` и
просто предоставить значение, которое хотите возвращать при всех испусканиях:

```js
import { fromEvent } from 'rxjs';
import { mapTo } from 'rxjs/operators';

const click$ = fromEvent(document, 'click');

click$
  .pipe(mapTo('You clicked!'))
  // 'You clicked!', 'You clicked!'
  .subscribe(console.log);
```

Как и `pluck`, `mapTo` не даёт реального функционального преимущества перед возвратом
постоянного значения с помощью `map`, но синтаксически он может оказаться чуть проще
для восприятия и чтения с первого взгляда.

## Заключение

В заключение: `map` — это универсальный оператор, который позволяет преобразовать поток
с помощью предоставленной функции проекции. Будь то отображение на код клавиши, обновления
значения из поля ввода или изменение формы объекта, `map` будет одним из самых
используемых операторов в вашем повседневном наборе инструментов RxJS. Для сценариев, где вам просто
нужно отобразить на одно свойство или всегда хочется отобразить на постоянное значение, вы
также можете обратить внимание на вспомогательные операторы [`pluck`](<../02. Operators/07. Transformation/16. pluck.md>) и
[`mapTo`](<../02. Operators/07. Transformation/12. mapTo.md>).

Полный список преобразующих операторов с примерами, включая операторы,
которые управляют отображением на более сложные значения, такие как другие observable, смотрите
в [разделе преобразующих операторов](<../02. Operators/07. Transformation/00. Transformation.md>). Мы
подробно рассмотрим эти темы в будущих статьях!

# ReplaySubject

## «Воспроизводит» или испускает старые значения новым подписчикам

### Зачем нужен `ReplaySubject`?

Эта разновидность subject блистает, когда вы хотите, чтобы у подписчиков был
доступ к ранее испущенным значениям, даже если они подписались уже после того,
как эти значения были отправлены. Представьте его как Tivo или видеорекордер для
observable. Пропустили начало передачи? Не беда! С `ReplaySubject` вы всё ещё
можете наверстать упущенное.

В качестве примера из реальной жизни рассмотрите чат-приложение. Когда
пользователь заходит в чат, он, возможно, захочет увидеть несколько последних
сообщений, а не только новые. `ReplaySubject` может удерживать заданное число
самых свежих сообщений и показывать их пользователю при входе.

Итак, как он соотносится с `BehaviorSubject` или обычным `Subject`?

- **[`BehaviorSubject`](<02. BehaviorSubject.md>)** всегда требует начального
  значения и хранит только самое свежее значение. Подписчики получат это
  последнее значение при подписке. Это как войти в кинозал и застать лишь самую
  последнюю сцену.

- **[`Subject`](<04. Subject.md>)**, напротив, не выдаёт новым подписчикам никаких
  ранее испущенных значений. Это классическая живая трансляция. Опоздали —
  пропустили.

- **`ReplaySubject`** может помнить больше, чем только последнюю сцену. В
  зависимости от того, как он настроен, он может воспроизвести весь фильм целиком
  или лишь несколько последних сцен.

Однако будьте осторожны. `ReplaySubject` потенциально может использовать больше
памяти, поскольку хранит несколько значений. Если вас беспокоит расход памяти,
обязательно укажите ограниченный размер буфера.

В заключение: если вам нужна возможность воспроизводить новым подписчикам серию
предыдущих испусканий — ваш выбор `ReplaySubject`. Если вас интересует только
самое свежее испускание (с начальным значением), выбирайте `BehaviorSubject`. А
если вам нужен простой, без излишеств механизм вещания, при котором поздние
подписчики пропускают предыдущие испускания, остановитесь на обычном `Subject`.



### Примеры

##### Пример 1: простой ReplaySubject

(
[Stackblitz](https://stackblitz.com/edit/rxjs-replaysubject-simple-example?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { ReplaySubject } from 'rxjs';

const sub = new ReplaySubject(3);

sub.next(1);
sub.next(2);
sub.subscribe(console.log); // ВЫВОД => 1,2
sub.next(3); // ВЫВОД => 3
sub.next(4); // ВЫВОД => 4
sub.subscribe(console.log); // ВЫВОД => 2,3,4 (лог последних 3 значений для нового подписчика)
sub.next(5); // ВЫВОД => 5,5 (лог от обоих подписчиков)
```

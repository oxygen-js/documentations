# BehaviorSubject

## Требует начального значения и испускает текущее значение новым подписчикам

---

💡 Если вам нужны последнее значение (или несколько последних значений) при
подписке, но не нужно задавать начальное затравочное значение, обратите внимание
на [ReplaySubject](<03. ReplaySubject.md>)!

---

### Зачем нужен `BehaviorSubject`?

Этот специализированный subject идеален, когда вы хотите поддерживать и
предоставлять подписчикам «текущее значение». Представьте его как табло в
баскетбольном матче. Даже если вы начали смотреть в середине игры, вы всё равно
увидите текущий счёт. Точно так же, когда новый наблюдатель подписывается на
`BehaviorSubject`, он немедленно получает текущее значение (или последнее
испущенное значение).

Важно помнить, что **`BehaviorSubject` требует начального значения при создании**.
Именно этим он отличается от обычного [`Subject`](<04. Subject.md>), у которого
начального значения нет. Вообразите только что установленное табло: с
`BehaviorSubject` вы задаёте стартовый счёт, скажем 0-0. С обычным
[`Subject`](<04. Subject.md>) табло остаётся пустым, пока не будет забито очко.
Подписчики (ранние или поздние) обычного [`Subject`](<04. Subject.md>) не будут
получать испускаемые значения, пока `Subject` не испустит значение.

В отличие от [`ReplaySubject`](<03. ReplaySubject.md>): хотя оба предоставляют
исторические значения, [`ReplaySubject`](<03. ReplaySubject.md>) может передавать
несколько предыдущих значений, а не только последнее. Если бы баскетбольное табло
могло показывать последние пять результатов в порядке их появления в матче — это
было бы похоже на [`ReplaySubject`](<03. ReplaySubject.md>).
[`ReplaySubject`](<03. ReplaySubject.md>) также не получает начального
затравочного значения.

В заключение: если вам нужно гарантировать, что подписчики при подписке всегда
получают самое свежее значение, или у вас есть начальное затравочное значение —
ваш выбор `BehaviorSubject`. Если вам нужно больше исторических испусканий,
рассмотрите [`ReplaySubject`](<03. ReplaySubject.md>). А если история вам вовсе не
нужна, возможно, вам подойдёт простой [`Subject`](<04. Subject.md>).



### Примеры

##### Пример 1: Простой BehaviorSubject

(
[Stackblitz](https://stackblitz.com/edit/rxjs-behaviorsubject-simpleexample?file=index.ts?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { BehaviorSubject } from 'rxjs';

const subject = new BehaviorSubject(123);

// два новых подписчика получат начальное значение => вывод: 123, 123
subject.subscribe(console.log);
subject.subscribe(console.log);

// два подписчика получат новое значение => вывод: 456, 456
subject.next(456);

// новый подписчик получит последнее значение (456) => вывод: 456
subject.subscribe(console.log);

// все три подписчика получат новое значение => вывод: 789, 789, 789
subject.next(789);

// вывод: 123, 123, 456, 456, 456, 789, 789, 789
```

##### Пример 2: BehaviorSubject с новыми подписчиками, создаваемыми по кликам мыши

(
[Stackblitz](https://stackblitz.com/edit/rxjs-behaviorsubject-mouseclicks?file=index.ts)
)

```js
// RxJS v6+
import { BehaviorSubject, fromEvent, interval, merge } from 'rxjs';
import { map, tap, mergeMap } from 'rxjs/operators';

const setElementText = (elemId, text) =>
  (document.getElementById(elemId).innerText = text.toString());
const addHtmlElement = coords =>
  (document.body.innerHTML += `
  <div 
    id=${coords.id}
    style="
      position: absolute;
      height: 30px;
      width: 30px;
      text-align: center;
      top: ${coords.y}px;
      left: ${coords.x}px;
      background: silver;
      border-radius: 80%;"
    >
  </div>`);

const subject = new BehaviorSubject(0);

const click$ = fromEvent(document, 'click').pipe(
  map((e: MouseEvent) => ({
    x: e.clientX,
    y: e.clientY,
    id: Math.random()
  })),
  tap(addHtmlElement),
  mergeMap(coords => subject.pipe(tap(v => setElementText(coords.id, v))))
);

const interval$ = interval(1000).pipe(
  tap(v => subject.next(v)),
  tap(v => setElementText('intervalValue', v))
);

merge(click$, interval$).subscribe();
```

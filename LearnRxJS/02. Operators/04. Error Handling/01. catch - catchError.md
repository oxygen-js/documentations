# catchError

## сигнатура: `catchError(project: (err: any, caught: Observable<T>) => ObservableInput<any>): Observable`

### Изящно обрабатывайте ошибки в последовательности-observable.

---

💡 Нужно повторить неудавшуюся операцию? Загляните в [**retry**](<02. retry.md>) или [**retryWhen**](<03. retryWhen.md>)!

💡 Для очистки ресурсов независимо от ошибки используйте [**finalize**](<../08. Utility/05. finalize - finally.md>)!

⚠ Не забудьте вернуть observable из функции catchError!

---

## Зачем нужен catchError?

Представьте `catchError` как страховочную сетку для вашего observable. Когда в потоке возникает ошибка — из-за неудавшегося HTTP-запроса, неожиданных данных или любой другой проблемы, — `catchError` даёт вам шанс изящно восстановиться, вместо того чтобы позволить всей последовательности-observable рухнуть и сгореть. Это разница между тем, показывает ли ваше приложение дружелюбное сообщение «Что-то пошло не так» или пустой экран.

Особую ценность `catchError` придаёт гибкость в том, как именно вы реагируете на ошибки. Вы можете предоставить [резервные данные из кеша](#пример-1-перехват-ошибки-из-observable), можете [преобразовать ошибки в понятные пользователю сообщения](#пример-2-перехват-отклонённого-промиса) или даже решить повторить операцию (хотя для этого обычно лучше подходит [`retry`](<02. retry.md>)). Ключевой момент — понимать, *где* именно вы размещаете `catchError` в цепочке операторов. Поставьте его на внешнем уровне — и весь поток завершится при возникновении ошибки; но разместите его внутри операторов вроде `switchMap`, и [сбой затронет только эту внутреннюю операцию, а поток продолжит работу](#пример-3-сравнение-перехвата-ошибок-при-использовании-switchmapmergemapconcatmapexhaustmap).

По сути, `catchError` делает ваши реактивные приложения устойчивыми, гарантируя, что ошибки обрабатываются на ваших условиях, а не оставляются на волю случая, чтобы обрушить пользовательский опыт.

## Примеры

### Пример 1: Перехват ошибки из observable

( [StackBlitz](https://stackblitz.com/edit/typescript-auc2u2?file=index.ts&devtoolsheight=100) )

```js
// RxJS v6+
import { throwError, of } from 'rxjs';
import { catchError } from 'rxjs/operators';

// испускаем ошибку
const source = throwError('This is an error!');

// изящно обрабатываем ошибку, возвращая observable с сообщением об ошибке
const example = source.pipe(
  catchError(val => of(`I caught: ${val}`))
);

// вывод: 'I caught: This is an error'
const subscribe = example.subscribe(val => console.log(val));
```

### Пример 2: Перехват отклонённого промиса

( [StackBlitz](https://stackblitz.com/edit/typescript-nte3xs?file=index.ts&devtoolsheight=100) )

```js
// RxJS v6+
import { timer, from, of } from 'rxjs';
import { mergeMap, catchError } from 'rxjs/operators';

// создаём промис, который немедленно отклоняется
const myBadPromise = () =>
  new Promise((resolve, reject) => reject('Rejected!'));

// испускаем единственное значение через 1 секунду
const source = timer(1000);

// перехватываем отклонённый промис, возвращая observable с сообщением об ошибке
const example = source.pipe(
  mergeMap(_ =>
    from(myBadPromise()).pipe(
      catchError(error => of(`Bad Promise: ${error}`))
    )
  )
);

// вывод: 'Bad Promise: Rejected'
const subscribe = example.subscribe(val => console.log(val));
```

### Пример 3: Сравнение перехвата ошибок при использовании switchMap/mergeMap/concatMap/exhaustMap

( [StackBlitz](https://stackblitz.com/edit/rxjs-catcherror-withmapoperators?file=index.ts&devtoolsheight=80) )

```js
// RxJS v6+
import { throwError, fromEvent, of } from 'rxjs';
import {
  catchError,
  tap,
  switchMap,
  mergeMap,
  concatMap,
  exhaustMap
} from 'rxjs/operators';

// имитируем вызов API, который выбрасывает ошибку
const fakeRequest$ = of().pipe(
  tap(_ => console.log('fakeRequest')),
  throwError
);

/*
 * Размещение catchError ИМЕЕТ ЗНАЧЕНИЕ!
 *
 * Когда catchError помещён ВНУТРИ switchMap, ошибка возникает только
 * во внутреннем observable, а внешний поток продолжает работу.
 * Вы можете продолжать кликать и делать новые запросы.
 */
const iWillContinueListening$ = fromEvent(
  document.getElementById('continued'),
  'click'
).pipe(
  switchMap(_ => 
    fakeRequest$.pipe(
      catchError(_ => of('keep on clicking!!!'))
    )
  )
);

/*
 * Когда catchError помещён на ВНЕШНЕМ уровне, после switchMap,
 * весь поток завершается с ошибкой и прекращается.
 * После первого клика поток мёртв и больше не реагирует на клики.
 */
const iWillStopListening$ = fromEvent(
  document.getElementById('stopped'),
  'click'
).pipe(
  switchMap(_ => fakeRequest$),
  catchError(_ => of('no more requests!!!'))
);

iWillContinueListening$.subscribe(console.log);
iWillStopListening$.subscribe(console.log);
```

### Пример 4: Предоставление резервных данных при ошибке HTTP

( [StackBlitz](https://stackblitz.com/edit/typescript-n9llq6e3?file=index.ts) )

```js
// RxJS v6+
import { ajax } from 'rxjs/ajax';
import { of } from 'rxjs';
import { catchError, map } from 'rxjs/operators';

// пытаемся получить данные пользователя из API
const userData$ = ajax.getJSON('https://api.example.com/user/123').pipe(
  map(response => response.data),
  // если запрос не удался, предоставляем кешированные данные или данные по умолчанию
  catchError(error => {
    console.error('Failed to fetch user, using cached data', error);
    return of({ 
      id: 123, 
      name: 'Cached User', 
      status: 'offline' 
    });
  })
);

// пользователь всегда получает данные, даже если API недоступен
userData$.subscribe(user => console.log('User:', user));
```

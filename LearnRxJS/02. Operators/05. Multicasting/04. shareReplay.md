# shareReplay

#### сигнатура: `shareReplay(bufferSize?: number, windowTime?: number, scheduler?I IScheduler): Observable`

## Разделить источник и воспроизвести заданное число испущенных значений при подписке.

### Зачем нужен `shareReplay`?

Обычно вы хотите использовать `shareReplay`, когда у вас есть побочные эффекты или
затратные вычисления, которые вы не хотите выполнять для каждого из нескольких
подписчиков. Он также может быть полезен в ситуациях, когда вы знаете, что у потока
появятся поздние подписчики, которым нужен доступ к ранее испущенным значениям. Эта
способность _воспроизводить_ значения при подписке и отличает
[`share`](<03. share.md>) от `shareReplay`.

Например, предположим, у вас есть observable, который испускает последний посещённый url.
В первом примере мы воспользуемся [`share`](<03. share.md>):

```js
// имитируем изменение url с помощью subject
const routeEnd = new Subject<{data: any, url: string}>();

// берём url и разделяем с подписчиками
const lastUrl = routeEnd.pipe(
  pluck('url'),
  share()
);

// требуется начальный подписчик
const initialSubscriber = lastUrl.subscribe(console.log);

// имитируем смену маршрута
routeEnd.next({data: {}, url: 'my-path'});

// ничего не выводится
const lateSubscriber = lastUrl.subscribe(console.log);
```

В приведённом выше примере ничего не выводится, поскольку `lateSubscriber` подписывается на
источник. Теперь предположим, что вместо этого мы хотели бы дать доступ к последнему испущенному значению
при подписке — этого можно добиться с помощью `shareReplay`:

```js
import { Subject } from 'rxjs/Subject';
import { ReplaySubject } from 'rxjs/ReplaySubject';
import { pluck, share, shareReplay, tap } from 'rxjs/operators';

// имитируем изменение url с помощью subject
const routeEnd = new Subject<{data: any, url: string}>();

// берём url и разделяем с подписчиками
const lastUrl = routeEnd.pipe(
  tap(_ => console.log('executed')),
  pluck('url'),
  // по умолчанию хранятся все значения, поэтому задаём хранить и воспроизводить только последнее
  shareReplay(1)
);

// требует начальной подписки
const initialSubscriber = lastUrl.subscribe(console.log);

// имитируем смену маршрута
// выведено: 'executed', 'my-path'
routeEnd.next({data: {}, url: 'my-path'});

// выведено: 'my-path'
const lateSubscriber = lastUrl.subscribe(console.log);
```

Обратите внимание, что это поведение похоже на то, что вы увидели бы, если бы подписали
`ReplaySubject` на поток `lastUrl`, а затем подписались бы на этот `Subject`:

```js
// имитируем изменение url с помощью subject
const routeEnd = new Subject<{data: any, url: string}>();

// вместо shareReplay используем ReplaySubject
const shareWithReplay = new ReplaySubject();

// берём url и разделяем с подписчиками
const lastUrl = routeEnd.pipe(
  pluck('url')
)
.subscribe(val => shareWithReplay.next(val));

// имитируем смену маршрута
routeEnd.next({data: {}, url: 'my-path'});

// подписываемся на ReplaySubject вместо этого
// выведено: 'my path'
shareWithReplay.subscribe(console.log);
```

Более того, если мы заглянем в исходный код, то увидим, что применяется очень похожая
техника. Когда происходит подписка, `shareReplay` подписывается на
источник, отправляя значения через внутренний `ReplaySubject`:

(
[source](https://github.com/ReactiveX/rxjs/blob/b25db9f369b07f26cf2fc11714ec1990b78a4536/packages/rxjs/src/internal/operators/shareReplay.ts#L26-L37)
)

```js
  return function shareReplayOperation(this: Subscriber<T>, source: Observable<T>) {
    refCount++;
    if (!subject || hasError) {
      hasError = false;
      subject = new ReplaySubject<T>(bufferSize, windowTime, scheduler);
      subscription = source.subscribe({
        next(value) { subject.next(value); },
        error(err) {
          hasError = true;
          subject.error(err);
        },
        complete() {
          isComplete = true;
          subject.complete();
        },
      });
    }


    const innerSub = subject.subscribe(this);


    return () => {
      refCount--;
      innerSub.unsubscribe();
      if (subscription && refCount === 0 && isComplete) {
        subscription.unsubscribe();
      }
    };
  };
}
```

##### Пример 1: несколько подписчиков разделяют источник

```js
// RxJS v6+
import { Subject, ReplaySubject } from 'rxjs';
import { pluck, share, shareReplay, tap } from 'rxjs/operators';

const routeEnd = new Subject<{data: any, url: string}>();
// берём url и разделяем с подписчиками
const lastUrl = routeEnd.pipe(
  tap(_ => console.log('executed')),
  pluck('url'),
  // по умолчанию хранятся все значения, поэтому задаём хранить и воспроизводить только последнее

);
// требует начальной подписки
const initialSubscriber = lastUrl.subscribe(console.log)
// имитируем смену маршрута
// выведено: 'executed', 'my-path'
routeEnd.next({data: {}, url: 'my-path'});
// выведено: 'my-path'
const lateSubscriber = lastUrl.subscribe(console.log);
```

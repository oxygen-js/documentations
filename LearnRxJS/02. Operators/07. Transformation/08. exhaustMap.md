# exhaustMap

#### сигнатура: `exhaustMap(project: function, resultSelector: function): Observable`

## Отображает во внутренний observable, игнорируя остальные значения, пока тот не завершится.

### Зачем нужен `exhaustMap`?

Представьте, что вы в кофейне, где каждому посетителю разрешено делать только один заказ за раз, и он должен дождаться, пока заказ будет полностью готов, прежде чем сделать следующий. Если он попытается заказать снова, пока его кофе ещё готовится, бариста просто проигнорирует его. В этом и суть `exhaustMap`.

Этот оператор идеально подходит для обработки событий, которые могут срабатывать несколько раз подряд в быстрой последовательности, но реагировать нужно только на первое срабатывание, пока оно не завершится. Отличный пример — игнорирование кликов, когда пользователь нетерпеливо несколько раз нажимает кнопку «Submit». Вместо того чтобы отправлять несколько сетевых запросов, в идеале вы хотите, чтобы действие запускал только первый клик, а последующие клики игнорировались до завершения запроса.

Обратите внимание: **`exhaustMap` будет игнорировать значения источника, пока предыдущий внутренний observable ещё активен**. Это значит, что если внутренний observable не завершился, любые новые значения, испущенные источником, будут отброшены без какого-либо отображения.

Если нужно обрабатывать каждое событие, даже если предыдущие ещё не завершились, лучше подойдут другие операторы, такие как [`switchMap`](<19. switchMap.md>) или [`mergeMap`](<13. mergeMap - flatMap.md>). Но когда важно, чтобы одна задача была полностью завершена перед переходом к следующей, `exhaustMap` — ваш выбор.



### Примеры

##### Пример 1: exhaustMap с interval

(
[Stackblitz](https://stackblitz.com/edit/typescript-3qydhn?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/woposeqobo/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/9ovzapp9/) )

```js
// RxJS v6+
import { interval, merge, of } from 'rxjs';
import { delay, take, exhaustMap } from 'rxjs/operators';

const sourceInterval = interval(1000);
const delayedInterval = sourceInterval.pipe(delay(10), take(4));

const exhaustSub = merge(
  // задержка 10 мс, затем запускаем interval, испускающий 4 значения
  delayedInterval,
  // испускаем немедленно
  of(true)
)
  .pipe(exhaustMap(_ => sourceInterval.pipe(take(5))))
  /*
   *  Первое испущенное значение (of(true)) будет отображено
   *  в observable interval, испускающий по 1 значению каждую
   *  секунду и завершающийся после 5.
   *  Поскольку испускания из задержанного interval
   *  приходятся на время, когда этот observable ещё активен, они будут проигнорированы.
   *
   *  Сравните это с concatMap, который поставил бы их в очередь,
   *  switchMap, который переключался бы на новый внутренний observable при каждом испускании,
   *  и mergeMap, который поддерживал бы новую подписку для каждого испущенного значения.
   */
  // вывод: 0, 1, 2, 3, 4
  .subscribe(val => console.log(val));
```

##### Пример 2: Ещё один exhaustMap с interval

(
[Stackblitz](https://stackblitz.com/edit/typescript-vxussb?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/fizuduzuti/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/5ck8yg5k/3/) )

```js
// RxJS v6+
import { interval } from 'rxjs';
import { exhaustMap, tap, take } from 'rxjs/operators';

const firstInterval = interval(1000).pipe(take(10));
const secondInterval = interval(1000).pipe(take(2));

const exhaustSub = firstInterval
  .pipe(
    exhaustMap(f => {
      console.log(`Emission Corrected of first interval: ${f}`);
      return secondInterval;
    })
  )
  /*
                Когда мы подписались на первый interval, он начинает испускать значения (начиная с 0).
                Это значение отображается во второй interval, который затем начинает испускать (начиная с 0).  
                Пока второй interval активен, значения из первого interval игнорируются.
                Мы видим это, когда firstInterval испускает числа 3, 6 и так далее...

                  Вывод:
                  Emission of first interval: 0
                  0
                  1
                  Emission of first interval: 3
                  0
                  1
                  Emission of first interval: 6
                  0
                  1
                  Emission of first interval: 9
                  0
                  1
              */
  .subscribe(s => console.log(s));
```

### Внешние примеры

##### `exhaustMap` для эффекта логина в [примере приложения @ngrx](https://github.com/ngrx/platform/tree/a9e522953832b09bb329bac4524637bc608c450a/example-app)

(
[Источник](https://github.com/ngrx/platform/blob/a9e522953832b09bb329bac4524637bc608c450a/example-app/app/auth/effects/auth.effects.ts#L18-L30)
)

```js
@Effect()
  login$ = this.actions$.pipe(
    ofType(AuthActionTypes.Login),
    map((action: Login) => action.payload),
    exhaustMap((auth: Authenticate) =>
      this.authService
        .login(auth)
        .pipe(
          map(user => new LoginSuccess({ user })),
          catchError(error => of(new LoginFailure(error)))
        )
    )
  );
```

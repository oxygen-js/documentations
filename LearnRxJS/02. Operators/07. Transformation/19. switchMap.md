# switchMap

#### сигнатура: `switchMap(project: function: Observable, resultSelector: function(outerValue, innerValue, outerIndex, innerIndex): any): Observable`

## Отображает в observable, завершает предыдущий внутренний observable и испускает значения.

---

💡 Если вы хотите, чтобы поддерживалось более одной внутренней подписки,
попробуйте [`mergeMap`](<13. mergeMap - flatMap.md>)!

💡 Этот оператор обычно считается более безопасным вариантом по умолчанию, чем
[`mergeMap`](<13. mergeMap - flatMap.md>)!

💡 Этот оператор способен отменять сетевые запросы, находящиеся в процессе выполнения!

---

### Зачем нужен `switchMap`?

Главное отличие `switchMap` от других уплощающих операторов — эффект отмены. При
каждом испускании предыдущий внутренний observable (результат переданной вами
функции) отменяется, и происходит подписка на новый observable. Запомнить это
можно по фразе **переключиться на новый observable**.

Это идеально подходит для сценариев вроде
[поиска с подсказками (typeahead)](https://angular-2-training-book.rangle.io/handout/http/search_with_switchmap.html),
когда ответ на предыдущий запрос вас больше не интересует, как только приходит
новый ввод. Это также безопасный вариант в ситуациях, когда долгоживущий
внутренний observable мог бы привести к утечкам памяти, — например, если бы вы
использовали [mergeMap](<13. mergeMap - flatMap.md>) с интервалом и забыли
корректно освободить внутренние подписки. Помните: `switchMap` поддерживает
только одну внутреннюю подписку в каждый момент времени — это ясно видно в
[первом примере](#example-1-restart-interval-every-5-seconds).

Однако будьте осторожны: вам, скорее всего, стоит избегать `switchMap` в
сценариях, где каждый запрос должен завершиться, — вспомните записи в базу
данных. `switchMap` может отменить запрос, если источник испускает значения
достаточно быстро. В таких сценариях правильным выбором будет
[mergeMap](<13. mergeMap - flatMap.md>).



### Примеры

##### Пример 1: Перезапуск интервала при каждом клике

(
[StackBlitz](https://stackblitz.com/edit/typescript-s4pvix?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { interval, fromEvent } from 'rxjs';
import { switchMap } from 'rxjs/operators';

fromEvent(document, 'click')
  .pipe(
    // перезапускаем счётчик при каждом клике
    switchMap(() => interval(1000))
  )
  .subscribe(console.log);
```

##### Пример 2: Таймер обратного отсчёта с паузой и возобновлением

( [StackBlitz](https://stackblitz.com/edit/typescript-ivdebg?file=index.ts) )

```js
// RxJS v6+
import { interval, fromEvent, merge, empty } from 'rxjs';
import { switchMap, scan, takeWhile, startWith, mapTo } from 'rxjs/operators';

const COUNTDOWN_SECONDS = 10;

// ссылки на элементы
const remainingLabel = document.getElementById('remaining');
const pauseButton = document.getElementById('pause');
const resumeButton = document.getElementById('resume');

// потоки
const interval$ = interval(1000).pipe(mapTo(-1));
const pause$ = fromEvent(pauseButton, 'click').pipe(mapTo(false));
const resume$ = fromEvent(resumeButton, 'click').pipe(mapTo(true));

const timer$ = merge(pause$, resume$)
  .pipe(
    startWith(true),
    switchMap(val => (val ? interval$ : empty())),
    scan((acc, curr) => (curr ? curr + acc : acc), COUNTDOWN_SECONDS),
    takeWhile(v => v >= 0)
  )
  .subscribe((val: any) => (remainingLabel.innerHTML = val));
```

##### Пример 3: Использование функции `resultSelector`

(
[StackBlitz](https://stackblitz.com/edit/typescript-bmibzi?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { timer, interval } from 'rxjs';
import { switchMap } from 'rxjs/operators';

// переключаемся на новый внутренний observable, когда источник испускает значение, и испускаем результат функции проекции
timer(0, 5000)
  .pipe(
    switchMap(
      _ => interval(2000),
      (outerValue, innerValue, outerIndex, innerIndex) => ({
        outerValue,
        innerValue,
        outerIndex,
        innerIndex
      })
    )
  )
  /*
	Вывод:
	{outerValue: 0, innerValue: 0, outerIndex: 0, innerIndex: 0}
	{outerValue: 0, innerValue: 1, outerIndex: 0, innerIndex: 1}
	{outerValue: 1, innerValue: 0, outerIndex: 1, innerIndex: 0}
	{outerValue: 1, innerValue: 1, outerIndex: 1, innerIndex: 1}
*/
  .subscribe(console.log);
```

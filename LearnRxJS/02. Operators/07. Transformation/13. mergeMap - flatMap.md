# mergeMap

#### сигнатура: `mergeMap(project: function: Observable, resultSelector: function: any, concurrent: number): Observable`

## Отобразить в observable, испустить значения.

---

💡 flatMap — это псевдоним для mergeMap!

💡 Если в один момент времени должна быть активна только одна внутренняя подписка, попробуйте
[`switchMap`](<19. switchMap.md>)!

💡 Если важен порядок испускания и подписки на внутренние observable,
попробуйте [`concatMap`](<06. concatMap.md>)!

---

### Зачем нужен `mergeMap`?

Этот оператор лучше всего использовать, когда вы хотите уплощить внутренний observable, но хотите
вручную контролировать число внутренних подписок.

Например, при использовании [`switchMap`](<19. switchMap.md>) каждая внутренняя подписка
завершается, когда источник испускает значение, что позволяет держать активной лишь одну внутреннюю подписку. В
отличие от этого, `mergeMap` допускает несколько одновременно активных внутренних подписок. Именно
поэтому один из наиболее распространённых сценариев использования `mergeMap` —
запросы, которые не должны отменяться, — вспомните операции записи, а не чтения. Обратите внимание, что
если необходимо сохранять порядок, [`concatMap`](<06. concatMap.md>) — лучший вариант.

Имейте в виду, что, поскольку `mergeMap` поддерживает несколько активных внутренних подписок
одновременно, есть возможность создать утечку памяти из-за долгоживущих внутренних
подписок. Простой пример — если бы вы отображали в observable
с внутренним таймером или потоком dom-событий. В этих случаях, если вы всё же
хотите воспользоваться `mergeMap`, вам, возможно, стоит задействовать другой оператор для
управления завершением внутренней подписки — вспомните
[`take`](<../06. Filtering/18. take.md>) или [`takeUntil`](<../06. Filtering/20. takeUntil.md>). Вы
также можете ограничить число одновременно активных внутренних подписок с помощью
параметра `concurrent`, как показано в
[примере 5](#пример-5-mergemap-со-значением-concurrent).



### Примеры

##### Пример 1: mergeMap, имитирующий сохранение координат кликов

(
[StackBlitz](https://stackblitz.com/edit/rxjs-xfwdnl?file=index.ts&devtoolsheight=60)
)

```js
// RxJS v6+
import { fromEvent, of } from 'rxjs';
import { mergeMap, delay } from 'rxjs/operators';

// имитируем сетевой запрос на сохранение
const saveLocation = location => {
  return of(location).pipe(delay(500));
};
// потоки
const click$ = fromEvent(document, 'click');

click$
  .pipe(
    mergeMap((e: MouseEvent) => {
      return saveLocation({
        x: e.clientX,
        y: e.clientY,
        timestamp: Date.now()
      });
    })
  )
  // Saved! {x: 98, y: 170, ...}
  .subscribe(r => console.log('Saved!', r));
```

##### Пример 2: mergeMap с ajax-observable

(
[StackBlitz](https://stackblitz.com/edit/rxjs-wixf2a?file=index.ts&devtoolsheight=60)
)

```js
// RxJS v6+
import { fromEvent } from 'rxjs';
import { ajax } from 'rxjs/ajax';
import { mergeMap } from 'rxjs/operators';

// url бесплатного api
const API_URL = 'https://jsonplaceholder.typicode.com/todos/1';

// потоки
const click$ = fromEvent(document, 'click');

click$
  .pipe(
    /*
     * Здесь для примера используется mergeMap, но в общем случае для GET-запросов
     * вы предпочтёте switchMap.
     * Также, если вам не нужен параметр, как
     * ниже, можно использовать mergeMapTo.
     * напр. mergeMapTo(ajax.getJSON(API_URL))
     */
    mergeMap(() => ajax.getJSON(API_URL))
  )
  // { userId: 1, id: 1, ...}
  .subscribe(console.log);
```

##### Пример 3: mergeMap с промисом (можно также использовать [from](<../03. Creation/05. from.md>) для преобразования в observable)

(
[StackBlitz](https://stackblitz.com/edit/typescript-pnnsrq?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { of } from 'rxjs';
import { mergeMap } from 'rxjs/operators';

// вспомогательная функция для создания промиса
const myPromise = val =>
  new Promise(resolve => resolve(`${val} World From Promise!`));

// испускаем 'Hello'
const source$ = of('Hello');

// отображаем в промис и испускаем результат
source$
  .pipe(mergeMap(val => myPromise(val)))
  // вывод: 'Hello World From Promise'
  .subscribe(val => console.log(val));
```

##### Пример 4: mergeMap с `resultSelector`

(
[StackBlitz](https://stackblitz.com/edit/typescript-9p6ws7?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { of } from 'rxjs';
import { mergeMap } from 'rxjs/operators';

// вспомогательная функция для создания промиса
const myPromise = val =>
  new Promise(resolve => resolve(`${val} World From Promise!`));

// испускаем 'Hello'
const source$ = of('Hello');

source$
  .pipe(
    mergeMap(
      val => myPromise(val),
      /*
      можно также передать второй аргумент, который получает значение из источника и испущенное
      значение внутреннего observable или промиса
    */
      (valueFromSource, valueFromPromise) => {
        return `Source: ${valueFromSource}, Promise: ${valueFromPromise}`;
      }
    )
  )
  // вывод: "Source: Hello, Promise: Hello World From Promise!"
  .subscribe(val => console.log(val));
```

##### Пример 5: mergeMap со значением concurrent

(
[StackBlitz](https://stackblitz.com/edit/typescript-r3gcr4?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { interval } from 'rxjs';
import { mergeMap, take } from 'rxjs/operators';

// испускаем значение каждую 1с
const source$ = interval(1000);

source$
  .pipe(
    mergeMap(
      // project
      val => interval(5000).pipe(take(2)),
      // resultSelector
      (oVal, iVal, oIndex, iIndex) => [oIndex, oVal, iIndex, iVal],
      // concurrent
      2
    )
  )
  /*
		Вывод:
		[0, 0, 0, 0] <--1-й внутренний observable
		[1, 1, 0, 0] <--2-й внутренний observable
		[0, 0, 1, 1] <--1-й внутренний observable
		[1, 1, 1, 1] <--2-й внутренний observable
		[2, 2, 0, 0] <--3-й внутренний observable
		[3, 3, 0, 0] <--4-й внутренний observable
*/
  .subscribe(val => console.log(val));
```

# concatMap

#### сигнатура: `concatMap(project: function, resultSelector: function): Observable`

## Отобразить значения во внутренний observable, подписаться и испускать по порядку.

### Зачем нужен `concatMap`?

Этот оператор — ваш выбор, когда у вас есть observable, испускающий значения, и для каждого из этих значений вы хотите выполнить другую последовательность observable, гарантируя, что они обрабатываются по порядку, а не параллельно. Представьте это как ожидание в очереди в булочной: даже если несколько покупателей приходят одновременно, их обслуживают по одному. Так, например, если у вас есть поток событий кликов пользователя и для каждого клика вы хотите инициировать HTTP-запрос, но вам нужно, чтобы эти запросы выполнялись последовательно (один завершается прежде, чем начинается следующий), — `concatMap` именно то, что вам нужно.

Практический сценарий — упорядочивание сохранений последовательно, без перегрузки сервера одновременными запросами.

Держите в уме, что **если внутреннему observable требуется значительное время для завершения, это может привести к накоплению очереди внешних значений, ожидающих обработки**. В таких случаях может показаться, будто ваше приложение тормозит или зависло, тогда как в действительности `concatMap` старательно обрабатывает каждое значение по порядку. Для сценариев, где вы предпочли бы обрабатывать самое свежее значение и отбрасывать предыдущие, рассмотрите использование [`switchMap`](<19. switchMap.md>).

Наконец, если вас не заботит порядок обработки и вы просто хотите, чтобы всё выполнялось по мере поступления, [`mergeMap`](<13. mergeMap - flatMap.md>) может оказаться лучшим выбором.



### Примеры

##### Пример 1: Демонстрация разницы между `concatMap` и [`mergeMap`](<13. mergeMap - flatMap.md>)

(
[StackBlitz](https://stackblitz.com/edit/typescript-pkyxa1?file=index.ts&devtoolsheight=100)
)

💡 Обратите внимание на разницу между `concatMap` и [`mergeMap`](<13. mergeMap - flatMap.md>).
Поскольку `concatMap` не подписывается на следующий observable, пока предыдущий не
завершится, значение от источника, задержанное на 2000мс, будет испущено первым.
Сравните это с [`mergeMap`](<13. mergeMap - flatMap.md>), который подписывается сразу на
внутренние observable: observable с меньшей задержкой (1000мс) испустит,
а следом — observable, которому требуется 2000мс для завершения.

```js
// RxJS v6+
import { of } from 'rxjs';
import { concatMap, delay, mergeMap } from 'rxjs/operators';

// испустить значение задержки
const source = of(2000, 1000);
// отобразить значение от источника во внутренний observable, по завершении испустить результат и перейти к следующему
const example = source.pipe(
  concatMap(val => of(`Delayed by: ${val}ms`).pipe(delay(val)))
);
// вывод: With concatMap: Delayed by: 2000ms, With concatMap: Delayed by: 1000ms
const subscribe = example.subscribe(val =>
  console.log(`With concatMap: ${val}`)
);

// показ разницы между concatMap и mergeMap
const mergeMapExample = source
  .pipe(
    // просто чтобы мы могли залогировать это после того, как отработает первый пример
    delay(5000),
    mergeMap(val => of(`Delayed by: ${val}ms`).pipe(delay(val)))
  )
  .subscribe(val => console.log(`With mergeMap: ${val}`));
```

```js
// RxJS v6+
import { of } from 'rxjs';
import { concatMap } from 'rxjs/operators';

// испустить 'Hello' и 'Goodbye'
const source = of('Hello', 'Goodbye');
// пример с промисом
const examplePromise = val => new Promise(resolve => resolve(`${val} World!`));

  console.log('Example w/ Promise:', val)
);
```

##### Пример 3: Передача функции проекции

(
[StackBlitz](https://stackblitz.com/edit/typescript-2elzt7?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/vihacewozo/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/5sr5zzgy/) )

```js
// RxJS v6+
import { of } from 'rxjs';
import { concatMap } from 'rxjs/operators';

// испустить 'Hello' и 'Goodbye'
const source = of('Hello', 'Goodbye');
// пример с промисом
const examplePromise = val => new Promise(resolve => resolve(`${val} World!`));
// результат первого параметра передаётся во второй параметр — функцию-селектор — перед тем как быть возвращённым
const example = source.pipe(
  concatMap(
    val => examplePromise(val),
    result => `${result} w/ selector!`
  )
);
// вывод: 'Example w/ Selector: 'Hello w/ Selector', Example w/ Selector: 'Goodbye w/ Selector'
const subscribe = example.subscribe(val =>
  console.log('Example w/ Selector:', val)
);
```

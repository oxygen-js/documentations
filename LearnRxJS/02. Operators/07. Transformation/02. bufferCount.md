# bufferCount

#### сигнатура: `bufferCount(bufferSize: number, startBufferEvery: number = null): Observable`

## Накапливать испущенные значения, пока не наберётся заданное число, затем испустить их массивом.

### Зачем нужен `bufferCount`?
Ключевое отличие bufferCount от других буферизующих операторов заключается в его подходе к буферизации, основанном на количестве. Достигнув указанного числа испусканий, bufferCount группирует значения и испускает их массивом. Представьте это как накопление элементов группами заданного размера.

Этот оператор оказывается полезным в сценариях, где обрабатывать данные порциями эффективнее, — например, при массовых обновлениях или пакетной обработке. В отличие от него, оператор buffer полагается на замыкающий уведомитель для определения окна буферизации, что подходит не для всех случаев.

Однако имейте в виду, что bufferCount может быть не лучшим выбором, когда стратегия буферизации требует окон, основанных на времени или событиях. В таких случаях рассмотрите использование [buffer](<01. buffer.md>) или [buffertime](<03. bufferTime.md>). Помните, что bufferCount организует значения на основе количества испусканий, как наглядно показано в первом примере.

Проявляйте осторожность в ситуациях, где стратегия буферизации играет критическую роль в желаемом результате, поскольку выбор неподходящего оператора может привести к неожиданному поведению. Ознакомьтесь с различными буферизующими операторами, чтобы принимать обоснованные решения исходя из ваших конкретных требований.



### Примеры

##### Пример 1: Накопить буфер и испустить после заданного числа значений

(
[StackBlitz](https://stackblitz.com/edit/typescript-osryhu?file=index.ts&devtoolsheight=50)
| [jsBin](http://jsbin.com/suveqaromu/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/ky9myc5b/) )

```js
// RxJS v6+
import { interval } from 'rxjs';
import { bufferCount } from 'rxjs/operators';

//Создаём observable, испускающий значение каждую секунду
const source = interval(1000);
//После испускания трёх значений передаём их дальше массивом буферизованных значений
const bufferThree = source.pipe(bufferCount(3));
//Выводим значения в консоль
//например, вывод [0,1,2]...[3,4,5]
const subscribe = bufferThree.subscribe(val =>
  console.log('Buffered Values:', val)
);
```

##### Пример 2: Перекрывающиеся буферы

(
[StackBlitz](https://stackblitz.com/edit/typescript-vvccar?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/kiloxiraya/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/3c67qcz1/) )

```js
// RxJS v6+
import { interval } from 'rxjs';
import { bufferCount } from 'rxjs/operators';

//Создаём observable, испускающий значение каждую секунду
const source = interval(1000);
/*
bufferCount также принимает второй аргумент — когда начинать следующий буфер.
например, если у нас bufferCount равен 3, но второй аргумент (startBufferEvery) равен 1:
1-е значение интервала:
буфер 1: [0]
2-е значение интервала:
буфер 1: [0,1]
буфер 2: [1]
3-е значение интервала:
буфер 1: [0,1,2] Буфер из 3, испускаем буфер
буфер 2: [1,2]
буфер 3: [2]
4-е значение интервала:
буфер 2: [1,2,3] Буфер из 3, испускаем буфер
буфер 3: [2, 3]
буфер 4: [3]
*/
const bufferEveryOne = source.pipe(bufferCount(3, 1));
//Выводим значения в консоль
const subscribe = bufferEveryOne.subscribe(val =>
  console.log('Start Buffer Every 1:', val)
);
```

##### Пример 3: Отслеживание последних n нажатий клавиш

(
[StackBlitz](https://stackblitz.com/edit/rxjs-buffecount-keypresses-tracking?file=index.ts&devtoolsheight=50)
)

```js
// RxJS v6+
import { fromEvent, of } from 'rxjs';
import { bufferCount, map, mergeMap, tap } from 'rxjs/operators';

const fakeKeyPressesPost = keypresses =>
  of(201).pipe(
    tap(_ => {
      console.log(`received key presses are: ${keypresses}`);
      document.getElementById('output').innerText = keypresses;
    })
  );

fromEvent(document, 'keydown')
  .pipe(
    map((e: KeyboardEvent) => e.key),
    bufferCount(5),
    mergeMap(fakeKeyPressesPost)
  )
  .subscribe();
```

# scan

#### сигнатура: `scan(accumulator: function, seed: any): Observable`

## Свёртка (reduce) во времени.

---

💡 С помощью scan можно организовать управление состоянием в стиле [Redux](http://redux.js.org)!

---

### Зачем нужен `scan`?

Ключевое отличие оператора scan от других операторов свёртки — его особенность непрерывного накопления. С каждым испущенным значением применяется функция-аккумулятор, и накопленный результат испускается мгновенно. Запомнить это можно фразой «накапливай и испускай на ходу».

Оператор scan крайне полезен в сценариях, требующих мониторинга и обработки в реальном времени, например при подсчёте очков в игре, когда вы хотите отображать обновлённый счёт каждый раз, когда добавляются очки. Однако будьте осторожны, применяя scan в случаях, где важен *только* итоговый накопленный результат. В таких ситуациях более подходящим может оказаться оператор [`reduce`](<17. reduce.md>), поскольку он испускает лишь итоговое значение после завершения источника.

В итоге оператор scan предоставляет мощное и гибкое средство для обработки непрерывного накопления и испускания значений, что может быть особенно полезно в задачах мониторинга и обработки в реальном времени.



### Примеры

##### Пример 1: Сумма во времени

(
[StackBlitz](https://stackblitz.com/edit/typescript-ltcl9d?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { of } from 'rxjs';
import { scan } from 'rxjs/operators';

const source = of(1, 2, 3);
// базовый пример scan, сумма во времени начиная с нуля
const example = source.pipe(scan((acc, curr) => acc + curr, 0));
// логируем накопленные значения
// вывод: 1,3,6
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 2: Накопление объекта

(
[StackBlitz](https://stackblitz.com/edit/typescript-vu63kz?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/fusunoguqu/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/36rbu38b/) )

```js
// RxJS v6+
import { Subject } from 'rxjs';
import { scan } from 'rxjs/operators';

const subject = new Subject();
//пример scan, собирающий объект во времени
const example = subject.pipe(
  scan((acc, curr) => Object.assign({}, acc, curr), {})
);
//логируем накопленные значения
const subscribe = example.subscribe(val =>
  console.log('Accumulated object:', val)
);
//отправляем значения в subject, добавляя свойства к объекту
// {name: 'Joe'}
subject.next({ name: 'Joe' });
// {name: 'Joe', age: 30}
subject.next({ age: 30 });
// {name: 'Joe', age: 30, favoriteLanguage: 'JavaScript'}
subject.next({ favoriteLanguage: 'JavaScript' });
```

##### Пример 3: Испускание случайных значений из накопленного массива.

(
[StackBlitz](https://stackblitz.com/edit/typescript-lb8aw9?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { interval } from 'rxjs';
import { scan, map, distinctUntilChanged } from 'rxjs/operators';

// Накапливаем значения в массиве, испускаем случайные значения из этого массива.
const scanObs = interval(1000)
  .pipe(
    scan((a, c) => [...a, c], []),
    map(r => r[Math.floor(Math.random() * r.length)]),
    distinctUntilChanged()
  )
  .subscribe(console.log);
```

##### Пример 4: Накопление http-ответов во времени

(
[StackBlitz](https://stackblitz.com/edit/rxjs-scan-accumulate-request-responses?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { interval, of } from 'rxjs';
import { scan, delay, repeat, mergeMap } from 'rxjs/operators';

const fakeRequest = of('response').pipe(delay(2000));

// вывод:
// ['response'],
// ['response','response'],
// ['response','response','response'],
// и т.д...

interval(1000)
  .pipe(
    mergeMap(_ => fakeRequest),
    scan < string > ((all, current) => [...all, current], [])
  )
  .subscribe(console.log);
```

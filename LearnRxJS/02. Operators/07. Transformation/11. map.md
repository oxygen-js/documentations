# map

## сигнатура: `map(project: (value: T, index: number) => R): Observable<R>`

---

💡 Подобно хорошо известному Array.prototype.map, это оператор, который вы будете использовать чаще всего для преобразования значений в ваших потоках!

💡 Нужно извлечь единственное свойство из объекта? До RxJS v7 вы могли бы использовать `pluck`, но теперь рекомендуемый подход — `map` с деструктуризацией или доступом к свойству.

💡 Если ваша функция преобразования возвращает Observable (например, HTTP-вызов), вам не нужен `map` — вам нужен один из уплощающих операторов: `mergeMap`, `switchMap`, `concatMap` или `exhaustMap`.

💡 Оператор `map` применяется к каждому значению по отдельности. Если вам нужно преобразовать или объединить несколько значений вместе, посмотрите на операторы вроде `scan`, `reduce` или `combineLatest`.

---

### Зачем нужен map?

`map` — ваш основной оператор для преобразования данных по мере их прохождения через ваши observable-потоки. Представьте его как заводскую сборочную линию, где каждый предмет модифицируется, прежде чем двинуться дальше. Подобно тому как штамповочный станок оттискивает логотип на каждом проходящем изделии, `map` применяет вашу функцию преобразования к каждому значению, что спускается вниз по потоку.

Этот оператор по-настоящему раскрывается, когда нужно изменить форму данных под нужды вашего приложения. Представьте, что вы создаёте функцию поиска для магазина. Ваш API возвращает сложные объекты товаров с десятками полей — остатки на складе, местоположения складов, внутренние идентификаторы и многое другое, — но вашему UI нужны лишь название товара, цена и URL изображения. Вместо того чтобы таскать все эти лишние данные по всему приложению, вы используете `map` сразу после HTTP-вызова, чтобы [извлечь только то, что вам нужно](#пример-3-отображение-ответа-api-в-ui-модель). Это сохраняет ваш поток данных чистым, а компоненты — сфокусированными.

Один из самых распространённых паттернов, с которыми вы столкнётесь, — использование `map` для [извлечения свойств из объектов](#пример-2-отображение-в-единственное-свойство) или ответов. Будь то захват свойства `data` из обёртки ответа API, извлечение идентификаторов пользователей из коллекции объектов пользователей или преобразование значений формы в форму, которую ожидает ваш бэкенд, — `map` справляется с этими повседневными задачами преобразования. Он также идеален для [простых вычислений](#пример-5-вычисления-и-форматирование) — добавления налога к ценам, преобразования временных меток в форматированные даты или преобразования координат для картографической библиотеки.

Что делает `map` особенно мощным — это его предсказуемость и простота. В отличие от операторов, управляющих множественными испусканиями или вопросами тайминга, у `map` прямолинейное отношение один к одному: одно входное значение производит одно выходное значение, синхронно. Это упрощает рассуждение о нём и отладку. Если значение входит — преобразованное значение выходит немедленно, без каких-либо подписок, без асинхронной сложности.

`map` одинаково полезен для преобразования событий в данные, которые вас действительно интересуют. Работая с DOM-событиями, вы редко нуждаетесь в целом объекте события — лишь в конкретных свойствах, таких как [координаты мыши](#пример-4-преобразование-dom-событий) или значения клавиатурного ввода. `map` позволяет вам аккуратно извлечь ровно то, что важно.

Критически важный момент, о котором стоит помнить: `map` предназначен для *синхронных* преобразований. Если вашей функции преобразования нужно вызвать API, обратиться к базе данных или выполнить любую асинхронную операцию, возвращающую Observable, — `map` неверный выбор. Когда вы отображаете значение в Observable, вы создаёте «Observable высшего порядка» (Observable из Observable), а это не то, что вам нужно. Для таких сценариев обращайтесь к `mergeMap`, `switchMap`, `concatMap` или `exhaustMap` — уплощающим операторам, которые обрабатывают внутренние Observable.

По сути, всякий раз, когда вы ловите себя на мысли «мне нужно преобразовать каждое значение в этом потоке», `map` должен быть вашим первым порывом. Это швейцарский нож RxJS — простой, надёжный и идеальный для подавляющего большинства задач преобразования, с которыми вы столкнётесь в реактивном программировании.

## Примеры

### Пример 1: Прибавить 10 к каждому числу

( [StackBlitz](https://stackblitz.com/edit/typescript-a7bnxb?file=index.ts&devtoolsheight=100) )

```javascript
// RxJS v6+
import { from } from 'rxjs';
import { map } from 'rxjs/operators';

// испустить (1,2,3,4,5)
const source = from([1, 2, 3, 4, 5]);
// прибавить 10 к каждому значению
const example = source.pipe(map(val => val + 10));
// вывод: 11,12,13,14,15
const subscribe = example.subscribe(val => console.log(val));
```

### Пример 2: Отображение в единственное свойство

( [StackBlitz](https://stackblitz.com/edit/typescript-qgpnju?file=index.ts&devtoolsheight=100) )

```javascript
// RxJS v6+
import { from } from 'rxjs';
import { map } from 'rxjs/operators';

// испустить ({name: 'Joe', age: 30}, {name: 'Frank', age: 20},{name: 'Ryan', age: 50})
const source = from([
  { name: 'Joe', age: 30 },
  { name: 'Frank', age: 20 },
  { name: 'Ryan', age: 50 }
]);
// извлечь имя каждого человека с помощью деструктуризации
const example = source.pipe(map(({ name }) => name));
// вывод: "Joe","Frank","Ryan"
const subscribe = example.subscribe(val => console.log(val));
```

### Пример 3: Отображение ответа API в UI-модель

( [StackBlitz](https://stackblitz.com/edit/typescript-kus9nthn?file=index.ts) )

```javascript
// RxJS v6+
import { of } from 'rxjs';
import { map } from 'rxjs/operators';

// Имитация ответа API с дополнительными метаданными
const apiResponse = of({
  status: 'success',
  timestamp: 1634567890,
  data: {
    id: 123,
    firstName: 'Jane',
    lastName: 'Doe',
    email: 'jane@example.com',
    internalFields: { /* ... */ }
  }
});

// Извлечь и переформатировать только то, что нужно UI
const uiModel = apiResponse.pipe(
  map(response => ({
    fullName: `${response.data.firstName} ${response.data.lastName}`,
    email: response.data.email,
    id: response.data.id
  }))
);

// вывод: { fullName: 'Jane Doe', email: 'jane@example.com', id: 123 }
const subscribe = uiModel.subscribe(val => console.log(val));
```

### Пример 4: Преобразование DOM-событий

( [StackBlitz](https://stackblitz.com/edit/typescript-a8rwwcaj?file=index.ts) )

```javascript
// RxJS v6+
import { fromEvent } from 'rxjs';
import { map } from 'rxjs/operators';

// Отслеживать позицию мыши по событиям клика
const clicks = fromEvent(document, 'click');

const positions = clicks.pipe(
  map((event: MouseEvent) => ({
    x: event.clientX,
    y: event.clientY,
    timestamp: Date.now()
  }))
);

// вывод: { x: 234, y: 567, timestamp: 1634567891234 }
positions.subscribe(pos => console.log('Click position:', pos));
```

### Пример 5: Вычисления и форматирование

( [StackBlitz](https://stackblitz.com/edit/typescript-rfbasqua?file=index.ts) )

```javascript
// RxJS v6+
import { from } from 'rxjs';
import { map } from 'rxjs/operators';

// Товары в корзине
const cartItems = from([
  { name: 'Laptop', price: 999.99, quantity: 1 },
  { name: 'Mouse', price: 29.99, quantity: 2 },
  { name: 'Keyboard', price: 79.99, quantity: 1 }
]);

// Вычислить итог и отформатировать для отображения
const itemsWithTotal = cartItems.pipe(
  map(item => ({
    ...item,
    total: item.price * item.quantity,
    displayPrice: `$${(item.price * item.quantity).toFixed(2)}`
  }))
);

/* вывод: 
  { name: 'Laptop', price: 999.99, quantity: 1, total: 999.99, displayPrice: '$999.99' }
  { name: 'Mouse', price: 29.99, quantity: 2, total: 59.98, displayPrice: '$59.98' }
  { name: 'Keyboard', price: 79.99, quantity: 1, total: 79.99, displayPrice: '$79.99' }
*/
const subscribe = itemsWithTotal.subscribe(item => console.log(item));
```

# buffer

#### сигнатура: `buffer(closingNotifier: Observable): Observable`

## Собирает выходные значения, пока не испустит предоставленный observable, и испускает их массивом.

### Зачем нужен `buffer`?
Оператор buffer в RxJS выделяется способностью накапливать испускаемые значения в массив до тех пор, пока не испустит заданный notifier. Представьте его как механизм «собери и выпусти». Это хорошо подходит для случаев, когда вы хотите группировать события по определённому условию, например по времени или действиям пользователя.

Buffer блистает в сценариях вроде объединения нескольких событий в пакет перед обработкой, что позволяет оптимизировать производительность и снизить нагрузку. Например, при отслеживании кликов на веб-странице buffer может собрать серию кликов перед отправкой их в аналитику, вместо того чтобы отправлять каждый по отдельности.

Однако имейте в виду, что buffer может оказаться не лучшим выбором, когда вам требуется немедленная обработка каждого испущенного значения. Один из способов это осмыслить — «buffer нужен для пакетирования». Помните: buffer целиком про накопление и выпуск значений по определённым триггерам, что делает его мощным инструментом для управления сгруппированными событиями.



### Примеры

##### Пример 1: Использование buffer для распознавания двойных кликов

(
[StackBlitz](https://stackblitz.com/edit/typescript-x5zyn5?file=index.ts&devtoolsheight=50))

```js
// RxJS v6+
import { fromEvent } from 'rxjs';
import { buffer, filter, throttleTime } from 'rxjs/operators';

// потоки
const clicks$ = fromEvent(document, 'click');

/*
Собираем произошедшие клики, через 250мс испускаем массив кликов
*/
clicks$
  .pipe(
    buffer(clicks$.pipe(throttleTime(250))),
    // если массив больше 1, значит произошёл двойной клик
    filter(clickArray => clickArray.length > 1)
  )
  .subscribe(() => console.log('Double Click!'));
```

##### Пример 2: Буферизация до клика по документу

(
[StackBlitz](https://stackblitz.com/edit/typescript-nwp2cl?file=index.ts&devtoolsheight=50)
| [jsBin](http://jsbin.com/fazimarajo/edit?js,console,output) |
[jsFiddle](https://jsfiddle.net/btroncone/7451s67k/) )

```js
// RxJS v6+
import { interval, fromEvent } from 'rxjs';
import { buffer } from 'rxjs/operators';

// Создаём observable, испускающий значение каждую секунду
const myInterval = interval(1000);
// Создаём observable, испускающий значение при каждом клике по документу
const bufferBy = fromEvent(document, 'click');
/*
Собираем все значения, испущенные нашим observable interval, пока не кликнем по документу. Это заставит observable bufferBy испустить значение, удовлетворяя buffer. Он передаст нам все собранные с прошлого буфера значения в виде массива.
*/
const myBufferedInterval = myInterval.pipe(buffer(bufferBy));
// Печатаем значения в консоль
// пример вывода: [1,2,3] ... [4,5,6,7,8]
const subscribe = myBufferedInterval.subscribe(val =>
  console.log(' Buffered Values:', val)
);
```

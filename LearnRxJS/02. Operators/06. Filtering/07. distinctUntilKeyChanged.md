# distinctUntilKeyChanged

#### сигнатура: `distinctUntilKeyChanged(key, compare?: fn): Observable`

## Испускает значение только тогда, когда меняется значение указанного ключа

💡 Если вы сравниваете значения целиком, используйте вместо этого [distinctUntilChanged](<06. distinctUntilChanged.md>)!  
💡 Для действительно уникальных значений по всему потоку (а не только для идущих подряд) обратите внимание на [distinct](<05. distinct.md>)!

### Зачем нужен distinctUntilKeyChanged?

Когда вы работаете с потоками объектов — будь то профили пользователей, товарные позиции или ответы API, — вас часто интересуют изменения конкретного свойства, а не всего объекта целиком. Возможно, вы отслеживаете пользователей по их ID, фильтруете товары по SKU или следите за обновлениями статуса. Вот где `distinctUntilKeyChanged` раскрывается во всей красе.

Представьте его как сообразительного швейцара в клубе, который узнаёт людей по их клубным картам. Даже если кто-то сменил наряд или причёску (другие свойства объекта), швейцара интересует только номер карты. Если вы попытаетесь войти дважды подряд с одной и той же картой, на второй раз он вас остановит: «Вы уже внутри!». Но если появится кто-то другой с иной картой, его пропустят. Именно это и делает `distinctUntilKeyChanged` — он проверяет конкретную «карту» (ключ) на каждом объекте и не пропускает идущие подряд дубликаты.

Это невероятно полезно в сценариях вроде живых потоков данных, где объекты могут повторно испускаться очень часто, но вы хотите реагировать только тогда, когда действительно меняется конкретное свойство. Например, вы можете получать объекты пользователей из WebSocket-соединения, но хотите обновлять свой UI только тогда, когда поле статуса пользователя меняется с 'online' на 'offline', а не при каждой повторной рассылке объекта. Используя `distinctUntilKeyChanged('status')`, вы отсеиваете шум и [реагируете только на значимые изменения](#пример-1-сравнение-по-ключу).

### Примеры

##### Пример 1: Сравнение по ключу

(
[StackBlitz](https://stackblitz.com/edit/typescript-bpl1gwyk?file=index.ts)
)

```js
// RxJS v6+
import { from } from 'rxjs';
import { distinctUntilKeyChanged } from 'rxjs/operators';

// выводим только различающиеся значения по ключу 'name'
const source$ = from([
  { name: 'Brian' },
  { name: 'Joe' },
  { name: 'Joe' },
  { name: 'Sue' }
]);

source$
  // испускаем только когда меняется свойство name
  .pipe(distinctUntilKeyChanged('name'))
  // вывод: { name: 'Brian' }, { name: 'Joe' }, { name: 'Sue' }
  .subscribe(console.log);
```

##### Пример 2: События клавиатуры

(
[StackBlitz](https://stackblitz.com/edit/rxjs-distinctuntilkeychanged?file=index.ts&devtoolsheight=50)
)

```js
// RxJS v6+
import { fromEvent } from 'rxjs';
import { distinctUntilKeyChanged, map } from 'rxjs/operators';

// слушаем события клавиатуры
const keys$ = fromEvent(document, 'keyup').pipe(
  // испускаем только когда меняется свойство 'code' (игнорируем повторные удержания клавиши)
  distinctUntilKeyChanged<KeyboardEvent>('code'),
  // извлекаем само значение клавиши для отображения
  map(e => e?.key)
);

keys$.subscribe(console.log);
```

##### Пример 3: Пользовательская функция сравнения

(
[StackBlitz](https://stackblitz.com/edit/typescript-exygmrr6?file=index.ts)
)

```js
// RxJS v6+
import { of } from 'rxjs';
import { distinctUntilKeyChanged } from 'rxjs/operators';

interface Person {
  age: number;
  name: string;
}

// поток объектов person
const people$ = of<Person>(
  { age: 4, name: 'Foo1' },
  { age: 7, name: 'Bar' },
  { age: 5, name: 'Foo2' },
  { age: 6, name: 'Foo3' }
);

people$
  .pipe(
    // испускаем только когда меняются первые 3 буквы имени
    distinctUntilKeyChanged(
      'name',
      (x: string, y: string) => x.substring(0, 3) === y.substring(0, 3)
    )
  )
  // вывод: { age: 4, name: 'Foo1' }, { age: 7, name: 'Bar' }, { age: 5, name: 'Foo2' }
  .subscribe(console.log);
```

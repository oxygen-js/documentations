# auditTime

#### сигнатура: `auditTime(duration: number, scheduler?: Scheduler): Observable`

## Игнорировать в течение заданного времени, затем испустить самое свежее значение

### Зачем нужен `auditTime`

Когда вам нужно игнорировать источник-observable в течение заданного промежутка времени, вы можете использовать `auditTime`. Возможный сценарий применения — испускать определённые события (например, клики мышью) не чаще некоторой максимальной частоты в секунду. После того как указанная длительность истекла, таймер отключается, и самое свежее значение источника испускается в выходной Observable, после чего этот процесс повторяется для следующего значения источника.

---

💡 Если вы хотите, чтобы таймер сбрасывался всякий раз, когда в источнике-observable происходит новое событие, вы можете использовать [debounceTime](<04. debounceTime.md>)

---

### Примеры

##### Пример 1: Испускать клики с частотой не более одного клика в секунду

( [stackBlitz](https://stackblitz.com/edit/typescript-skykxw) )

```js
import { fromEvent } from 'rxjs';
import { auditTime } from 'rxjs/operators';

// Создаём observable, испускающий события кликов
const source = fromEvent(document, 'click');
// Испускаем клики с частотой не более одного клика в секунду
const example = source.pipe(auditTime(1000))
// Вывод (пример): '(1s) --- Clicked --- (1s) --- Clicked' 
const subscribe = example.subscribe(val => console.log('Clicked'));
```

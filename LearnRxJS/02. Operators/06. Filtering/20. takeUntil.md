# takeUntil

#### сигнатура: `takeUntil(notifier: Observable): Observable`

## Испускает значения до тех пор, пока переданный observable не испустит значение.

---

💡 Если вам нужно только определённое количество значений, попробуйте [take](<18. take.md>)!

---

### Зачем нужен `takeUntil`?

Представьте себе рабочий день: вы ожидаете важное письмо, но решили, что как только часы пробьют 5 вечера, вы уходите с работы вне зависимости от того, получили вы это письмо или нет. В этой аналогии с RxJS ожидание письма — это один observable, а время окончания рабочего дня в 5 вечера — другой. Оператор `takeUntil` следит за возможным поступлением письма, но в тот момент, когда наступает 5 вечера, вы прекращаете проверку (отписываетесь).

В реальных приложениях представьте сценарий, где вы отслеживаете ответы сервера на панели мониторинга. Однако вы хотите, чтобы это отслеживание прекратилось, как только будет нажата определённая кнопка «Остановить мониторинг». Вот тут-то `takeUntil` и проявляет себя во всей красе.

В контексте Angular `takeUntil` особенно удобен для автоматической отписки от observable при уничтожении компонента. Это достигается за счёт использования хука жизненного цикла `ngOnDestroy`. Обычно вы создаёте `Subject`, часто называемый `destroy$`, и используете его вместе с `takeUntil`:

```typescript
private destroy$ = new Subject<void>();

observable$
  .pipe(takeUntil(this.destroy$))
  .subscribe(data => console.log(data));

ngOnDestroy() {
  this.destroy$.next();
  this.destroy$.complete();
}
```

При такой настройке, как только вызывается метод `ngOnDestroy` (когда компонент вот-вот будет уничтожен), observable, использующие `takeUntil` с subject-ом `destroy$`, автоматически отпишутся, гарантируя, что не возникнет нежелательных утечек памяти или неожиданного поведения.



### Примеры

##### Пример 1: Брать значения, пока таймер не испустит значение

(
[StackBlitz](https://stackblitz.com/edit/typescript-ujwjbg?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/yevuhukeja/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/zbe9dzb9/) )

```js
// RxJS v6+
import { interval, timer } from 'rxjs';
import { takeUntil } from 'rxjs/operators';

//испускать значение каждую 1s
const source = interval(1000);
//через 5 секунд испустить значение
const timer$ = timer(5000);
//когда таймер испустит значение через 5s, завершить источник
const example = source.pipe(takeUntil(timer$));
//вывод: 0,1,2,3
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 2: Взять первые 5 чётных чисел

(
[StackBlitz](https://stackblitz.com/edit/typescript-djhv7s?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/doquqecara/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/0dLeksLe/) )

```js
// RxJS v6+
import { interval } from 'rxjs/observable/interval';
import { takeUntil, filter, scan, map, withLatestFrom } from 'rxjs/operators';

//испускать значение каждую 1s
const source = interval(1000);
//является ли число чётным?
const isEven = val => val % 2 === 0;
//пропускать только те значения, которые чётные
const evenSource = source.pipe(filter(isEven));
//вести текущий подсчёт количества выданных чётных чисел
const evenNumberCount = evenSource.pipe(scan((acc, _) => acc + 1, 0));
//не испускать, пока не будет испущено 5 чётных чисел
const fiveEvenNumbers = evenNumberCount.pipe(filter(val => val > 5));

const example = evenSource.pipe(
  //также передай мне текущий счётчик чётных чисел для отображения
  withLatestFrom(evenNumberCount),
  map(([val, count]) => `Even number (${count}) : ${val}`),
  //когда пять чётных чисел будут испущены, завершить исходный observable
  takeUntil(fiveEvenNumbers)
);
/*
	Even number (1) : 0,
  Even number (2) : 2
	Even number (3) : 4
	Even number (4) : 6
	Even number (5) : 8
*/
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 3: Брать события мыши при нажатии кнопки до её отпускания

(
[StackBlitz](https://stackblitz.com/edit/rxjs-ug2ezf?file=index.ts&devtoolsheight=50)
)

```js
// RxJS v6+
import { fromEvent } from 'rxjs';
import { takeUntil, mergeMap, map } from 'rxjs/operators';

const mousedown$ = fromEvent(document, 'mousedown');
const mouseup$ = fromEvent(document, 'mouseup');
const mousemove$ = fromEvent(document, 'mousemove');

// после нажатия кнопки мыши брать позицию до её отпускания
mousedown$
  .pipe(
    mergeMap(_ => {
      return mousemove$.pipe(
        map((e: any) => ({
          x: e.clientX,
          y: e.clientY
        })),
        // завершить внутренний observable по событию отпускания кнопки мыши
        takeUntil(mouseup$)
      );
    })
  )
  .subscribe(console.log);
```

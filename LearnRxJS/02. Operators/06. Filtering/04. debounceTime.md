# debounceTime

#### сигнатура: `debounceTime(dueTime: number, scheduler: Scheduler): Observable`

## Отбрасывает испускаемые значения, между которыми проходит меньше заданного времени.

---

💡 Этот оператор популярен в сценариях вроде автодополнения при вводе, где частоту
пользовательского ввода необходимо контролировать!

---

### Зачем нужен `debounceTime`?

Представьте `debounceTime` как паузу в разговоре, чтобы дать собеседнику закончить мысль. Этот оператор невероятно удобен, когда вы имеете дело с быстрыми последовательностями событий и вас интересует только реакция на последнее событие по прошествии заданного времени.

Классическое применение в реальном мире — поля форм, особенно поисковые строки. Представьте, что вы печатаете в поисковом поле. Вместо того чтобы отправлять запрос к API при каждом нажатии клавиши (что перегружает систему и неэффективно), лучше немного подождать после того, как пользователь перестанет печатать, чтобы получить данные по его завершённой мысли. Вот это «немного подождать» и есть тот случай, где `debounceTime` блистает. Например, задав `debounceTime(300)`, система будет ждать 300 миллисекунд после последнего нажатия клавиши, прежде чем продолжить.

В Angular при работе с реактивными формами `debounceTime` — настоящее спасение. Добавив этот оператор к observable изменений значения элемента управления формы, вы можете эффективно обрабатывать значения только после того, как пользователь завершит ввод. Взгляните на пример ниже:
```typescript
this.myFormControl.valueChanges.pipe(
  debounceTime(300)
).subscribe(value => {
  // обрабатываем значение после 300 мс бездействия
});
```




### Примеры

##### Пример 1: Debounce на основе времени между вводом

(
[StackBlitz](https://stackblitz.com/edit/typescript-adheqt?file=index.ts&devtoolsheight=50)
)

```js
// RxJS v6+
import { fromEvent } from 'rxjs';
import { debounceTime, map } from 'rxjs/operators';

// ссылка на элемент
const searchBox = document.getElementById('search');

// потоки
const keyup$ = fromEvent(searchBox, 'keyup');

// ждём 0.5с между нажатиями клавиш, чтобы испустить текущее значение
keyup$
  .pipe(
    map((i: any) => i.currentTarget.value),
    debounceTime(500)
  )
  .subscribe(console.log);
```

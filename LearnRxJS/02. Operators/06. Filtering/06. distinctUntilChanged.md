# distinctUntilChanged

#### сигнатура: `distinctUntilChanged(compare: function): Observable`

## Испускать только тогда, когда текущее значение отличается от предыдущего.

---

💡 distinctUntilChanged по умолчанию использует сравнение через `===`, ссылки на объекты
должны совпадать!

💡 Если вы хотите сравнивать по свойству объекта, вместо этого можно использовать
[`distinctUntilKeyChanged`](<07. distinctUntilKeyChanged.md>)!

---

### Зачем нужен `distinctUntilChanged`?

Этот оператор стоит на страже, следя за тем, чтобы вас не заваливали повторяющейся информацией. Представьте, что ваш лучший друг снова и снова рассказывает вам одну и ту же историю при каждой встрече. Это быстро надоест, верно? Оператор `distinctUntilChanged` делает как раз это — он предотвращает последующие идентичные испускания из observable.

Подумайте о строке поиска на веб-сайте. Пока пользователи вводят свои запросы, вы не хотите отправлять на сервер запрос с одним и тем же значением несколько раз подряд — это было бы избыточно и неэффективно. Вот где может блеснуть `distinctUntilChanged` в сочетании с [`debounceTime`](<04. debounceTime.md>). Представьте, что пользователь набирает поисковый термин, а затем слегка медлит, прежде чем добавить ещё одну букву. Возможно, вам захочется немного подождать (это [`debounceTime`](<04. debounceTime.md>) творит свою магию), а затем, перед отправкой запроса, убедиться, что термин действительно отличается от предыдущего, — именно здесь вы можете задействовать `distinctUntilChanged`.

Например, если пользователь ищет «apple» и набирает «app» -> ждёт немного -> «appl» -> возвращается к «app» -> снова набирает «appl», то без `distinctUntilChanged` вы можете в итоге отправить избыточные запросы. Но с ним, как только «appl» будет распознан как уже обработанный, повторный поисковый запрос больше не отправится.

Важно помнить, что **`distinctUntilChanged` сравнивает текущее значение с последним испущенным значением**. Он не хранит длинную историю. Так что если observable испустил значения 1, 2, 2, 3, 3, 2 — вы получите в ответ 1, 2, 3, 2. По умолчанию он также использует простое равенство для сравнения значений. Если вы работаете с объектами или массивами, вам может понадобиться предоставить собственную функцию сравнения, чтобы определять, различаются ли значения.

По сути, когда вам нужно отфильтровать подряд идущие дубликаты испусканий из ваших observable, думайте о `distinctUntilChanged` как о вашем основном варианте.



### Примеры

##### Пример 1: distinctUntilChanged с базовыми значениями

(
[StackBlitz](https://stackblitz.com/edit/typescript-bsb8mw?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { from } from 'rxjs';
import { distinctUntilChanged } from 'rxjs/operators';

// выводим только различающиеся значения, основываясь на последнем испущенном значении
const source$ = from([1, 1, 2, 2, 3, 3]);

source$
  .pipe(distinctUntilChanged())
  // вывод: 1,2,3
  .subscribe(console.log);
```

##### Пример 2: distinctUntilChanged с объектами

(
[StackBlitz](https://stackblitz.com/edit/typescript-moe7mh?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { from } from 'rxjs';
import { distinctUntilChanged } from 'rxjs/operators';

const sampleObject = { name: 'Test' };

//Объекты должны быть одной и той же ссылкой
const source$ = from([sampleObject, sampleObject, sampleObject]);

// испускаем только различающиеся объекты, основываясь на последнем испущенном значении
source$
  .pipe(distinctUntilChanged())
  // вывод: {name: 'Test'}
  .subscribe(console.log);
```

##### Пример 3: Использование собственной функции сравнения

(
[StackBlitz](https://stackblitz.com/edit/typescript-hzta27?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { from } from 'rxjs';
import { distinctUntilChanged } from 'rxjs/operators';

// выводим только различающиеся значения, основываясь на последнем испущенном значении
const source$ = from([
  { name: 'Brian' },
  { name: 'Joe' },
  { name: 'Joe' },
  { name: 'Sue' }
]);

source$
  // собственное сравнение по имени
  .pipe(distinctUntilChanged((prev, curr) => prev.name === curr.name))
  // вывод: { name: 'Brian }, { name: 'Joe' }, { name: 'Sue' }
  .subscribe(console.log);
```

# delay

#### сигнатура: `delay(delay: number | Date, scheduler: Scheduler): Observable`

## Задерживает испускаемые значения на заданное время.

### Зачем нужен `delay`?

Этот оператор — то, что нужно, когда вы моделируете реальные сценарии, такие как сетевая задержка, или вводите паузу перед испусканием значения. Оператор `delay` позволяет придержать значения на заданный промежуток времени, прежде чем они будут переданы подписчикам. 

Имейте в виду, что **`delay` не помешает исходному observable испускать значения**. Он лишь откладывает их доставку подписчикам. Это _подводный камень_, поскольку может показаться, что ваши данные запаздывают или рассинхронизированы с источником, особенно когда задействовано несколько observable.



### Примеры

##### Пример 1: Задержка для распознавания долгого нажатия

( [StackBlitz](https://stackblitz.com/edit/rxjs-bru5fi?devtoolsheight=60) )

```js
import { fromEvent, of } from 'rxjs';
import { mergeMap, delay, takeUntil } from 'rxjs/operators';

const mousedown$ = fromEvent(document, 'mousedown');
const mouseup$ = fromEvent(document, 'mouseup');

mousedown$
  .pipe(mergeMap(event => of(event).pipe(delay(700), takeUntil(mouseup$))))
  .subscribe(event => console.log('Long Press!', event));
```

##### Пример 2: Задержка на возрастающую длительность

(
[StackBlitz](https://stackblitz.com/edit/typescript-twjn8r?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { of, merge } from 'rxjs';
import { mapTo, delay } from 'rxjs/operators';

//испускает один элемент
const example = of(null);
//задерживаем вывод каждого на дополнительную секунду
const message = merge(
  example.pipe(mapTo('Hello')),
  example.pipe(mapTo('World!'), delay(1000)),
  example.pipe(mapTo('Goodbye'), delay(2000)),
  example.pipe(mapTo('World!'), delay(3000))
);
//вывод: 'Hello'...'World!'...'Goodbye'...'World!'
const subscribe = message.subscribe(val => console.log(val));
```

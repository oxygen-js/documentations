# let

#### сигнатура: `let(function): Observable`

## Даёт мне доступ ко всему observable целиком.

---

⚠ `let` больше недоступен, да и не нужен при использовании пайпируемых операторов! (RxJS
5.5+)

---



### Примеры

##### Пример 1: Переиспользование логики обработки ошибок с let

( [jsBin](http://jsbin.com/rosuborara/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/qtq1h8vw/) )

```js
// своя логика обработки ошибок
const retryThreeTimes = obs =>
  obs.retry(3).catch(_ => Rx.Observable.of('ERROR!'));
const examplePromise = val =>
  new Promise(resolve => resolve(`Complete: ${val}`));

//имитируем запрос
const subscribe = Rx.Observable.of('some_url')
  .mergeMap(url => examplePromise(url))
  // с помощью let можно переиспользовать логику обработки ошибок в разных местах
  .let(retryThreeTimes)
  //вывод: Complete: some_url
  .subscribe(result => console.log(result));

const customizableRetry = retryTimes => obs =>
  obs.retry(retryTimes).catch(_ => Rx.Observable.of('ERROR!'));

//имитируем запрос
const secondSubscribe = Rx.Observable.of('some_url')
  .mergeMap(url => examplePromise(url))
  // с помощью let можно переиспользовать логику обработки ошибок в разных местах
  .let(customizableRetry(3))
  //вывод: Complete: some_url
  .subscribe(result => console.log(result));
```

##### Пример 2: Применение map с let

( [jsBin](http://jsbin.com/jiyupaxomo/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/6n7w3b22/) )

```js
//испускаем массив как последовательность
const source = Rx.Observable.from([1, 2, 3, 4, 5]);
//демонстрируем отличие let от других операторов
const test = source
  .map(val => val + 1)
  /*
    	это не сработает, let ведёт себя иначе, чем большинство операторов
    	val в данном случае — это observable
    */
  //.let(val => val + 2)
  .subscribe(val => console.log('VALUE FROM ARRAY: ', val));

const subscribe = source
  .map(val => val + 1)
  //'let' даёт мне доступ ко всему observable целиком
  .let(obs => obs.map(val => val + 2))
  //вывод: 4,5,6,7,8
  .subscribe(val => console.log('VALUE FROM ARRAY WITH let: ', val));
```

##### Пример 3: Применение нескольких операторов с let

( [jsBin](http://jsbin.com/zamizapaho/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/gxsq1woc/) )

```js
//испускаем массив как последовательность
const source = Rx.Observable.from([1, 2, 3, 4, 5]);

//let даёт гибкость: можно добавить несколько операторов к исходному observable, а затем вернуть его
const subscribeTwo = source
  .map(val => val + 1)
  .let(obs =>
    obs
      .map(val => val + 2)
      //а также вернуть только чётные
      .filter(val => val % 2 === 0)
  )
  //вывод: 4,6,8
  .subscribe(val => console.log('let WITH MULTIPLE OPERATORS: ', val));
```

##### Пример 4: Применение операторов через функцию

( [jsBin](http://jsbin.com/vojelelamu/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/ah09dL9e/) )

```js
//испускаем массив как последовательность
const source = Rx.Observable.from([1, 2, 3, 4, 5]);

//передаём собственную функцию для добавления операторов к observable
const obsArrayPlusYourOperators = yourAppliedOperators => {
  return source.map(val => val + 1).let(yourAppliedOperators);
};
const addTenThenTwenty = obs => obs.map(val => val + 10).map(val => val + 20);
const subscribe = obsArrayPlusYourOperators(addTenThenTwenty)
  //вывод: 32, 33, 34, 35, 36
  .subscribe(val => console.log('let FROM FUNCTION:', val));
```

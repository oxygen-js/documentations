# create

## Устарело: используйте `new Observable()` вместо этого

> ⚠️ **Важно**: `Observable.create()` был объявлен устаревшим в RxJS v6.4.0. Используйте вместо него конструктор `new Observable()`.

**сигнатура**: `new Observable(subscribe: (observer: Observer) => TeardownLogic)`

```typescript
new Observable(subscribe: (observer: Observer) => TeardownLogic)
```

Создайте пользовательский observable, определив поведение подписки.

## 💡 Советы

- **Сначала рассмотрите операторы создания**: прежде чем создавать пользовательский observable, проверьте, не решают ли уже вашу задачу операторы вроде [defer](<03. defer.md>), [from](<05. from.md>), [fromEvent](<06. fromEvent.md>) или [interval](<08. interval.md>)
- **Используйте для связывания нереактивных API**: пользовательские observable незаменимы, когда нужно обернуть коллбэки, эмиттеры событий, WebSocket-ы или другие нереактивные источники данных
- **Всегда возвращайте логику очистки**: возвращайте функцию очистки, чтобы предотвратить утечки памяти по завершении подписок

## Зачем нужен пользовательский observable?

Пользовательские observable — это ваш мост между нереактивным миром и RxJS. Думайте о них как об адаптерах: когда у вас есть источник данных, который не говорит на языке «Observable» (например, WebSocket-соединение, сторонняя библиотека с коллбэками или браузерный API вроде Geolocation), создание пользовательского observable позволяет обернуть его и заставить дружелюбно взаимодействовать с остальным вашим реактивным кодом.

Вот в чём суть: **вам редко придётся создавать пользовательские observable**. RxJS уже предоставляет операторы создания для большинства распространённых сценариев. Таймеры, события, промисы, массивы и многое другое. Но когда вы сталкиваетесь с API, который не подходит ни под один существующий оператор, пользовательские observable дают вам детальный контроль. Вы сами решаете, когда именно испускать значения (observer.next()), как обрабатывать ошибки (observer.error()) и какая очистка должна произойти, когда кто-то отписывается.

Один из способов взглянуть на это — как на написание переводчика. Нереактивный API говорит на одном языке, ваши потоки Observable — на другом, а ваш пользовательский observable находится посередине и следит, чтобы они понимали друг друга. Реализуя [функцию очистки](#пример-2-observable-с-корректной-очисткой), помните, что это ваш шанс быть добропорядочным гражданином. Закрывайте соединения, отменяйте таймеры, удаляйте слушателей. Это как выключить свет, покидая комнату.

По сути, пользовательские observable мощны, но использовать их следует умеренно. Если существующий оператор создания может справиться с задачей, используйте его. Но когда вам нужен этот дополнительный контроль — когда вы интегрируетесь с устаревшим кодом, сторонними библиотеками или необычными источниками данных, это ваш инструмент.

## Примеры

### Пример 1: простой пользовательский observable, испускающий несколько значений

([StackBlitz](https://stackblitz.com/edit/typescript-meebpsdr?file=index.ts))

```typescript
// RxJS v6+
import { Observable } from 'rxjs';

/*
  Создаём observable, который при подписке испускает 'Hello' и 'World'.
  Обратите внимание: используется современный синтаксис конструктора, а не устаревший Observable.create()
*/
const hello$ = new Observable(observer => {
  observer.next('Hello');
  observer.next('World');
  observer.complete();
});

// Вывод: 'Hello'...'World'
const subscription = hello$.subscribe(val => console.log(val));
```

### Пример 2: Observable с корректной очисткой

([StackBlitz](https://stackblitz.com/edit/typescript-wbwrxn67?file=index.ts))

```typescript
// RxJS v6+
import { Observable } from 'rxjs';

/*
  Создаём observable, который каждую секунду испускает чётные числа.
  Демонстрирует важность возврата функции очистки (teardown).
*/
const evenNumbers$ = new Observable(observer => {
  let value = 0;
  
  const intervalId = setInterval(() => {
    if (value % 2 === 0) {
      observer.next(value);
    }
    value++;
  }, 1000);
  
  // Возвращаем функцию очистки — вызывается при обращении к unsubscribe()
  return () => {
    console.log('Cleaning up interval');
    clearInterval(intervalId);
  };
});

// Вывод: 0...2...4...6...8
const subscription = evenNumbers$.subscribe(val => console.log(val));

// Отписываемся через 10 секунд — запускает очистку
setTimeout(() => {
  subscription.unsubscribe();
}, 10000);
```

### Пример 3: обёртывание API на основе коллбэков

([StackBlitz](https://stackblitz.com/edit/typescript-ury4kzws?file=index.ts))

```typescript
// RxJS v6+
import { Observable } from 'rxjs';

/*
  Оборачиваем браузерный API Geolocation в observable.
  Это демонстрирует, как пользовательские observable связывают нереактивные API.
*/
function getCurrentPosition(): Observable<GeolocationPosition> {
  return new Observable(observer => {
    // Проверяем, доступна ли геолокация
    if (!navigator.geolocation) {
      observer.error(new Error('Geolocation not supported'));
      return;
    }
    
    // Получаем позицию и испускаем её
    const watchId = navigator.geolocation.watchPosition(
      position => observer.next(position),
      error => observer.error(error),
      { enableHighAccuracy: true }
    );
    
    // Возвращаем очистку, чтобы прекратить отслеживание позиции при отписке
    return () => {
      navigator.geolocation.clearWatch(watchId);
    };
  });
}

// Использование
const position$ = getCurrentPosition();
const subscription = position$.subscribe({
  next: position => {
    console.log('Latitude:', position.coords.latitude);
    console.log('Longitude:', position.coords.longitude);
  },
  error: err => console.error('Error getting position:', err)
});

// Прекращаем отслеживание через 30 секунд
setTimeout(() => subscription.unsubscribe(), 30000);
```

### Пример 4: создание observable из WebSocket


```typescript
// RxJS v6+
import { Observable } from 'rxjs';

/*
  Оборачиваем WebSocket-соединение в observable.
  Демонстрирует управление сложными асинхронными ресурсами с корректной очисткой.
*/
function createWebSocketObservable(url: string): Observable<MessageEvent> {
  return new Observable(observer => {
    const socket = new WebSocket(url);
    
    socket.onopen = () => {
      console.log('WebSocket connected');
    };
    
    socket.onmessage = (event) => {
      observer.next(event);
    };
    
    socket.onerror = (error) => {
      observer.error(error);
    };
    
    socket.onclose = () => {
      observer.complete();
    };
    
    // Очистка: закрываем сокет при отписке
    return () => {
      if (socket.readyState === WebSocket.OPEN) {
        socket.close();
      }
    };
  });
}

// Использование
const messages$ = createWebSocketObservable('wss://example.com/socket');
const subscription = messages$.subscribe({
  next: event => console.log('Message received:', event.data),
  error: err => console.error('WebSocket error:', err),
  complete: () => console.log('WebSocket closed')
});

// Закрываем соединение через 60 секунд
setTimeout(() => subscription.unsubscribe(), 60000);
```

# forkJoin

#### сигнатура: `forkJoin(...args, selector : function): Observable`

## Когда все observable завершатся, испускает последнее значение, испущенное каждым из них.

---

💡 Если вам нужны соответствующие друг другу испускания из нескольких observable по мере их появления,
попробуйте [zip](<13. zip.md>)!

⚠ Если внутренний observable не завершается, `forkJoin` никогда не испустит значение!

---

### Зачем нужен `forkJoin`?

Этот оператор лучше всего применять, когда у вас есть группа observable и вас
интересует только итоговое значение, испущенное каждым из них. Один из
распространённых сценариев использования — если вы хотите выполнить несколько
запросов при загрузке страницы (или по какому-то другому событию) и хотите
предпринять действие только после того, как будет получен ответ на все запросы.
В этом смысле он похож на то, как вы могли бы использовать
[`Promise.all`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise/all).

Учтите, что если какой-либо из внутренних observable, переданных `forkJoin`,
завершится с ошибкой, вы потеряете значения всех остальных observable, которые
завершились бы или уже завершились, если вы не
[перехватите (`catch`)](<../04. Error Handling/01. catch - catchError.md>)
[ошибку правильно на внутреннем observable](#example-4-getting-successful-results-when-one-innner-observable-errors).
Если вас заботит только успешное завершение всех внутренних observable, вы можете
[перехватить ошибку снаружи](#example-3-handling-errors-on-outside).

Также стоит отметить, что если у вас есть observable, испускающий более одного
элемента, и вас интересуют предыдущие испускания, то `forkJoin` — не тот выбор.
В таких случаях вам, возможно, лучше подойдёт оператор вроде
[combineLatest](<02. combineLatest.md>) или [zip](<13. zip.md>).



### Примеры

##### Пример 1: Использование словаря источников для выполнения AJAX-запроса

(
[StackBlitz](https://stackblitz.com/edit/typescript-u5pzuf?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6.5+
import { ajax } from 'rxjs/ajax';
import { forkJoin } from 'rxjs';

/*
  когда все observable завершатся, предоставляем последнее
  испущенное каждым из них значение в виде словаря
*/
forkJoin(
  // начиная с RxJS 6.5+ мы можем использовать словарь источников
  {
    google: ajax.getJSON('https://api.github.com/users/google'),
    microsoft: ajax.getJSON('https://api.github.com/users/microsoft'),
    users: ajax.getJSON('https://api.github.com/users')
  }
)
  // { google: object, microsoft: object, users: array }
  .subscribe(console.log);
```

##### Пример 2: Observable, завершающиеся спустя разное время

(
[StackBlitz](https://stackblitz.com/edit/typescript-c3f62b?file=index.ts&devtoolsheight=100) )

```js
// RxJS v6+
import { interval, forkJoin, of } from 'rxjs';
import { delay, take } from 'rxjs/operators';

const myPromise = val =>
  new Promise(resolve =>
    setTimeout(() => resolve(`Promise Resolved: ${val}`), 5000)
  );

/*
  когда все observable завершатся, отдаём последнее
  испущенное каждым из них значение в виде массива
*/
const example = forkJoin({
  //испускаем 'Hello' немедленно
  sourceOne: of('Hello'),
  //испускаем 'World' через 1 секунду
  sourceTwo: of('World').pipe(delay(1000)),
  //испускаем 0 через 1 секунду
  sourceThree: interval(1000).pipe(take(1)),
  //испускаем 0...1 с интервалом в 1 секунду
  sourceFour: interval(1000).pipe(take(2)),
  //промис, который разрешается в 'Promise Resolved' через 5 секунд
  sourceFive: myPromise('RESULT')
});
/*
 * Вывод:
 * { 
 *   sourceOne: "Hello", 
 *   sourceTwo: "World", 
 *   sourceThree: 0,
 *   sourceFour: 1,
 *   sourceFive: "Promise Resolved: RESULT"
 * }
 */
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 3: Выполнение переменного количества запросов (используется устаревший API)

(
[StackBlitz](https://stackblitz.com/edit/typescript-3mbbjw?file=index.ts&devtoolsheight=100) )

```js
// RxJS v6+
import { mergeMap } from 'rxjs/operators';
import { forkJoin, of } from 'rxjs';

const myPromise = val =>
  new Promise(resolve =>
    setTimeout(() => resolve(`Promise Resolved: ${val}`), 5000)
  );

const source = of([1, 2, 3, 4, 5]);
//испускаем массив из всех 5 результатов
const example = source.pipe(mergeMap(q => forkJoin(...q.map(myPromise))));
/*
  вывод:
  [
   "Promise Resolved: 1",
   "Promise Resolved: 2",
   "Promise Resolved: 3",
   "Promise Resolved: 4",
   "Promise Resolved: 5"
  ]
*/
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 4: Обработка ошибок снаружи

(
[StackBlitz](https://stackblitz.com/edit/typescript-petcwk?file=index.ts&devtoolsheight=100) )

```js
// RxJS v6+
import { delay, catchError } from 'rxjs/operators';
import { forkJoin, of, throwError } from 'rxjs';

/*
  Если какой-либо из внутренних observable завершится с ошибкой,
  результат с ошибкой будет испущен через catchError.
*/
const example = forkJoin({
  // испускаем 'Hello' немедленно
  sourceOne: of('Hello'),
  // испускаем 'World' через 1 секунду
  sourceTwo: of('World').pipe(delay(1000)),
  // выбрасываем ошибку
  sourceThree: throwError('This will error')
}).pipe(catchError(error => of(error)));

// вывод: 'This will Error'
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 5: Получение успешных результатов, когда один из внутренних observable завершается с ошибкой

(
[StackBlitz](https://stackblitz.com/edit/typescript-7qcyvz?file=index.ts&devtoolsheight=100) )

```js
// RxJS v6+
import { delay, catchError } from 'rxjs/operators';
import { forkJoin, of, throwError } from 'rxjs';

/*
  Испускаем значения из успешно завершившихся
  внутренних observable.
*/
const example = forkJoin({
  // испускаем 'Hello' немедленно
  sourceOne: of('Hello'),
  // испускаем 'World' через 1 секунду
  sourceTwo: of('World').pipe(delay(1000)),
  // выбрасываем ошибку
  sourceThree: throwError('This will error').pipe(catchError(error => of(error)))
});

/*
 * Вывод:
 * {
 *   sourceOne: "Hello",
 *   sourceTwo: "World",
 *   sourceThree: "This will error"
 * }
 */
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 6: forkJoin в Angular


```js
@Injectable()
export class MyService {
  makeRequest(value: string, delayDuration: number) {
    // имитируем http-запрос
    return of(`Complete: ${value}`).pipe(
      delay(delayDuration)
    );
  }
}

@Component({
  selector: 'my-app',
  template: `
    <div>
      <h2>forkJoin Example</h2>
      <ul>
        <li> {{propOne}} </li>
        <li> {{propTwo}} </li>
        <li> {{propThree}} </li>
      </ul>
    </div>
  `,
})
export class App {
  public propOne: string;
  public propTwo: string;
  public propThree: string;
  constructor(private _myService: MyService) {}

  ngOnInit() {
    // имитируем 3 запроса с разными задержками
    forkJoin({
      requestOne: this._myService.makeRequest('Request One', 2000),
      requestTwo: this._myService.makeRequest('Request Two', 1000),
      requestThree: this._myService.makeRequest('Request Three', 3000)
    })
    .subscribe(({requestOne, requestTwo, requestThree}) => {
      this.propOne = requestOne;
      this.propTwo = requestTwo;
      this.propThree = requestThree;
    });
  }
}
```

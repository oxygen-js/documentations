# concatAll

#### сигнатура: `concatAll(): Observable`

## Собирает observable и подписывается на следующий, когда предыдущий завершается.

---

⚠ Остерегайтесь
[обратного давления (backpressure)](https://github.com/Reactive-Extensions/RxJS/blob/master/doc/gettingstarted/backpressure.md),
когда источник испускает значения быстрее, чем успевают завершаться внутренние observable!

💡 Во многих случаях вместо этого можно использовать один-единственный оператор [concatMap](<../07. Transformation/06. concatMap.md>)!

---

### Зачем нужен `concatAll`?
Этот оператор лучше всего применять, когда у вас есть несколько observable, которые нужно обрабатывать последовательно, гарантируя, что каждый observable завершится, прежде чем начнётся обработка следующего. Реальные примеры такого поведения можно увидеть в сценариях вроде загрузки нескольких файлов на сервер по очереди или показа последовательности анимаций по порядку.

Имейте в виду, что `concatAll` начнёт обрабатывать следующий observable только после завершения текущего. Это важное соображение, если у вас есть observable, которые испускают значения бесконечно или завершаются долго, поскольку это может задержать обработку последующих observable.

Кроме того, если вы работаете с observable, которые могут испускать значения параллельно и которым не нужно ждать завершения одного, прежде чем обрабатывать другой, более подходящим выбором может оказаться [mergeAll](<08. mergeAll.md>). Аналогично, если вам нужно лишь скомбинировать значения нескольких observable в тот момент, когда все они завершатся, лучшим вариантом может стать [forkJoin](<06. forkJoin.md>).

```js
// RxJS v6+
import { map, concatAll } from 'rxjs/operators';
import { of, interval } from 'rxjs';

//испускаем значение каждые 2 секунды
const source = interval(2000);
const example = source.pipe(
  //для наглядности прибавляем 10 и возвращаем как observable
  map(val => of(val + 10)),
  //объединяем значения из внутреннего observable
  concatAll()
);
//вывод: 'Example with Basic Observable 10', 'Example with Basic Observable 11'...
const subscribe = example.subscribe(val =>
  console.log('Example with Basic Observable:', val)
);
```

##### Пример 2: concatAll с промисом

(
[StackBlitz](https://stackblitz.com/edit/typescript-3w4px3?file=index.ts&devtoolsheight=100)

```js
// RxJS v6+
import { map, concatAll } from 'rxjs/operators';
import { interval } from 'rxjs';

//создаём и разрешаем простой промис
const samplePromise = val => new Promise(resolve => resolve(val));
//испускаем значение каждые 2 секунды
const source = interval(2000);

const example = source.pipe(
  map(val => samplePromise(val)),
  //объединяем значения из разрешённого промиса
  concatAll()
);
//вывод: 'Example with Promise 0', 'Example with Promise 1'...
const subscribe = example.subscribe(val =>
  console.log('Example with Promise:', val)
);
```

##### Пример 3: Задержка на время завершения внутренних observable

(
[StackBlitz](https://stackblitz.com/edit/typescript-ft3rbf?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/pojolatile/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/8230ucbg/) )

```js
// RxJS v6+
import { take, concatAll } from 'rxjs/operators';
import { interval, of } from 'rxjs';

const obs1 = interval(1000).pipe(take(5));
const obs2 = interval(500).pipe(take(2));
const obs3 = interval(2000).pipe(take(1));
//испускаем три observable
const source = of(obs1, obs2, obs3);
//подписываемся на каждый внутренний observable по порядку, когда предыдущий завершается
const example = source.pipe(concatAll());
/*
  вывод: 0,1,2,3,4,0,1,0
  Как это работает...
  Подписывается на каждый внутренний observable и испускает значения, а при завершении подписывается на следующий
  obs1: 0,1,2,3,4 (завершён)
  obs2: 0,1 (завершён)
  obs3: 0 (завершён)
*/

const subscribe = example.subscribe(val => console.log(val));
```

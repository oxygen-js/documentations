# zip

#### сигнатура: `zip(observables: *): Observable`

## После того как все observable испустят значения, испускает их в виде массива

---

💡 В сочетании с [interval](<../03. Creation/08. interval.md>) или
[timer](<../03. Creation/12. timer.md>) zip можно использовать, чтобы синхронизировать по времени вывод из другого источника!

---

### Зачем нужен zip?
Этот оператор идеален, когда нужно попарно объединить значения из нескольких observable, словно смыкая зубцы застёжки-молнии. Представьте два observable: один испускает значения вроде «hot», «warm», «cold», а другой — «coffee», «tea», «lemonade». С помощью оператора zip вы соедините их в пары «hot coffee», «warm tea» и «cold lemonade».

Повседневный пример — когда нужно сопоставить информацию из двух источников, например связать имена игроков с соответствующими им очками в игре. Вообразите observable, испускающий имена игроков, и другой, испускающий их очки. С помощью zip вы легко создадите пары `[player, score]`, гарантируя, что каждый игрок связан с правильным счётом.

Учтите, что zip испустит значение только тогда, когда все входные observable испустят соответствующее значение. Это означает, что если один observable испустил больше значений, чем другой, несопоставленные значения будут придержаны, пока другой observable не испустит своё следующее значение. В некоторых случаях это может привести к неспаренным значениям, поэтому важно следить за тем, чтобы ваши observable были синхронизированы.



### Примеры

##### Пример 1: zip нескольких observable, испускающих значения с чередующимися интервалами

(
[StackBlitz](https://stackblitz.com/edit/typescript-5az27c?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/lireyisira/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/ton462sg/) )

```js
// RxJS v6+
import { delay } from 'rxjs/operators';
import { of, zip } from 'rxjs';

const sourceOne = of('Hello');
const sourceTwo = of('World!');
const sourceThree = of('Goodbye');
const sourceFour = of('World!');
//ждём, пока все observable испустят значение, затем испускаем все как массив
const example = zip(
  sourceOne,
  sourceTwo.pipe(delay(1000)),
  sourceThree.pipe(delay(2000)),
  sourceFour.pipe(delay(3000))
);
//вывод: ["Hello", "World!", "Goodbye", "World!"]
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 2: zip, когда 1 observable завершается

(
[StackBlitz](https://stackblitz.com/edit/typescript-f4qgry?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/fisitatesa/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/oamyk3xr/) )

```js
// RxJS v6+
import { take } from 'rxjs/operators';
import { interval, zip } from 'rxjs';

//испускаем каждую 1с
const source = interval(1000);
//когда один observable завершается, значения больше не испускаются
const example = zip(source, source.pipe(take(2)));
//вывод: [0,0]...[1,1]
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 3: получение координат X/Y начала/конца перетаскивания (нажатие/отпускание кнопки мыши)

(
[StackBlitz](https://stackblitz.com/edit/rxjs-zip-mousedownup-coordinates?file=index.ts&devtoolsheight=50)
)

```js
// RxJS v6+
import { fromEvent, zip } from 'rxjs';
import { map } from 'rxjs/operators';

const documentEvent = eventName =>
  fromEvent(document, eventName).pipe(
    map((e: MouseEvent) => ({ x: e.clientX, y: e.clientY }))
  );

zip(documentEvent('mousedown'), documentEvent('mouseup')).subscribe(e =>
  console.log(JSON.stringify(e))
);
```

##### Пример 4: длительность щелчка мыши

(
[StackBlitz](https://stackblitz.com/edit/rxjs-zip-mouseclickduration?file=index.ts&devtoolsheight=50)
)

```js
// RxJS v6+
import { fromEvent, zip } from 'rxjs';
import { map } from 'rxjs/operators';

const eventTime = eventName =>
  fromEvent(document, eventName).pipe(map(() => new Date()));

const mouseClickDuration = zip(
  eventTime('mousedown'),
  eventTime('mouseup')
).pipe(map(([start, end]) => Math.abs(start.getTime() - end.getTime())));

mouseClickDuration.subscribe(console.log);
```

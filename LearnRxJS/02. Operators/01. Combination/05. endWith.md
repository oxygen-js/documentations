# endWith

#### сигнатура: `endWith(an: Values): Observable`

## Испускает заданное значение (или значения) при завершении.

---

💡 Если вы хотите вместо этого начать с некоторого значения, взгляните на
[`startWith`](<11. startWith.md>)!

💡 Если вы хотите выполнить действие при завершении, но не хотите испускать
значение, взгляните на [`finalize`](<../08. Utility/05. finalize - finally.md>)!

---

### Зачем нужен `endWith`?
Оператор `endWith` особенно удобен, когда вам нужно гарантировать, что определённое значение будет испущено после того, как исходный observable завершится. Считайте, что это финальные титры фильма, сигнализирующие о том, что история подошла к своему завершению. Примеры из реальной жизни для endWith можно найти в сценариях, где вы хотите добавить определённое сообщение или обновление статуса после серии событий: например, загрузка файла, которая заканчивается уведомлением «Загрузка завершена», или таймер обратного отсчёта, завершающийся сигналом «Время вышло!».

Имейте в виду, что endWith испускает указанное значение только тогда, когда исходный observable завершается. Это означает, что если ваш исходный observable не завершается, то значение, переданное в `endWith`, не будет испущено. Чтобы избежать неожиданностей, убедитесь, что ваш исходный observable рассчитан на то, чтобы в какой-то момент завершиться.

В случаях, когда вы хотите добавить значение в начало последовательности observable, а не в её конец, рассмотрите использование оператора [startWith](<11. startWith.md>).



### Примеры

##### Пример 1: Базовый пример `endWith`

(
[StackBlitz](https://stackblitz.com/edit/typescript-gexe9u?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { endWith } from 'rxjs/operators';
import { of } from 'rxjs';

const source$ = of('Hello', 'Friend', 'Goodbye');

source$
  // испустить при завершении
  .pipe(endWith('Friend'))
  // 'Hello', 'Friend', 'Goodbye', 'Friend'
  .subscribe(console.log(val));
```

##### Пример 2: endWith с несколькими значениями

(
[StackBlitz](https://stackblitz.com/edit/typescript-dyed7x?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { endWith } from 'rxjs/operators';
import { of } from 'rxjs';

const source$ = of('Hello', 'Friend');

source$
  // испустить при завершении
  .pipe(endWith('Goodbye', 'Friend'))
  // 'Hello', 'Friend', 'Goodbye', 'Friend'
  .subscribe(console.log(val));
```

##### Пример 3: Сравнение с [`finalize`](<../08. Utility/05. finalize - finally.md>)

(
[StackBlitz](https://stackblitz.com/edit/typescript-lkk1pj?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { endWith, finalize } from 'rxjs/operators';
import { of } from 'rxjs';

const source$ = of('Hello', 'Friend');

source$
  // испустить при завершении
  .pipe(
    endWith('Goodbye', 'Friend'),
    // эта функция вызывается при вызове методов отписки
    finalize(() => console.log('Finally'))
  )
  // 'Hello', 'Friend', 'Goodbye', 'Friend'
  .subscribe(val => console.log(val));
// 'Finally'
```

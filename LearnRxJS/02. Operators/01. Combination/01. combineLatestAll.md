# combineLatestAll

## Сигнатура

```typescript
combineLatestAll<T>(): OperatorFunction<ObservableInput<T>, T[]>
```

Уплощает observable высшего порядка, применяя [combineLatest](<02. combineLatest.md>) в момент, когда внешний observable завершается.

---

💡 `combineLatestAll` лучше всего подходит, когда вы работаете с **observable высшего порядка** (observable, который испускает другие observable) и вам нужно отслеживать самые свежие значения от каждого внутреннего observable
💡 В отличие от [`mergeAll`](<08. mergeAll.md>), который испускает значения, как только любой внутренний observable что-то испустит, `combineLatestAll` дожидается, пока **все внутренние observable испустят хотя бы по одному значению**, прежде чем выдать результат
💡 Если вам нужны только итоговые значения после завершения всех observable (а не непрерывные обновления), рассмотрите вместо этого [`forkJoin`](<06. forkJoin.md>)

---

## Зачем нужен combineLatestAll?

Представьте `combineLatestAll` как организацию панельной дискуссии, где докладчики присоединяются в разное время. Вы не можете начать трансляцию, пока не подключатся все, но после этого, всякий раз, когда кто-нибудь снова заговорит, вы транслируете самое свежее высказывание от каждого участника панели. Именно так `combineLatestAll` управляет вашими потоками observable.

Этот оператор незаменим, когда вы имеете дело с **динамическими коллекциями observable** или ситуациями, где вы заранее не знаете, сколько observable у вас будет, но они испускаются исходным потоком. Практический пример — когда вы [отображаете поток событий в observable-интервалы](#пример-1-отображение-во-внутренний-observable-интервал) и хотите одновременно наблюдать за самым свежим значением каждого из них. Это особенно полезно для сценариев вроде отслеживания нескольких пользовательских сессий, мониторинга виджетов дашборда в реальном времени, которые добавляются динамически, или объединения результатов от переменного числа вызовов API.

Имейте в виду, что `combineLatestAll` не выдаст ни одного значения, пока не будут выполнены два условия: внешний observable должен **завершиться** (сигнализируя, что больше внутренних observable не появится), и каждый внутренний observable должен **испустить хотя бы одно значение**. Это поведение «ждём всех» может стать подводным камнем, если один из ваших внутренних observable так и не испустит значение или никогда не завершится — ваш поток останется безмолвным.

По сути, `combineLatestAll` преобразует поток потоков в единый поток, который держит вас в курсе последнего снимка состояния от каждого участника вашей динамической коллекции.

## Примеры

### Пример 1: отображение во внутренний observable-интервал

([StackBlitz](https://stackblitz.com/edit/typescript-bzwkrl?file=index.ts))

```typescript
import { interval } from 'rxjs';
import { take, map, combineLatestAll } from 'rxjs/operators';

// Испускаем значение каждую 1с, берём 2
const source$ = interval(1000).pipe(take(2));

// Отображаем каждое значение из source в observable-интервал, который берёт 5 значений
const example$ = source$.pipe(
  map(val =>
    interval(1000).pipe(
      map(i => `Result (${val}): ${i}`),
      take(5)
    )
  )
);

/*
  2 значения из source будут отображены в 2 (внутренних) observable-интервала, испускающих каждую 1с.
  combineLatestAll использует стратегию combineLatest, испуская последнее значение от каждого
  всякий раз, когда любой из observable испускает значение
*/
example$
  .pipe(combineLatestAll())
  /*
    вывод:
    ["Result (0): 0", "Result (1): 0"]
    ["Result (0): 1", "Result (1): 0"]
    ["Result (0): 1", "Result (1): 1"]
    ["Result (0): 2", "Result (1): 1"]
    ["Result (0): 2", "Result (1): 2"]
    ["Result (0): 3", "Result (1): 2"]
    ["Result (0): 3", "Result (1): 3"]
    ["Result (0): 4", "Result (1): 3"]
    ["Result (0): 4", "Result (1): 4"]
  */
  .subscribe(console.log);
```

# concat

#### сигнатура: `concat(observables: ...*): Observable`

## Подписывается на observable по очереди — по мере завершения предыдущего

---

💡 Представляйте `concat` как очередь к банкомату: следующая операция
(подписка) не может начаться, пока не завершится предыдущая!

💡 Если для вас важнее пропускная способность, а не порядок, попробуйте вместо него [merge](<07. merge.md>)!

---

### Зачем нужен `concat`?
Оператор `concat` лучше всего подходит, когда нужно объединить несколько observable, но при этом требуется, чтобы их значения испускались в определённом порядке, один за другим. Это как собирать пазл, части которого нужно складывать последовательно, чтобы получить полную картину. Пример из реального мира — загрузка и отображение нескольких изображений в правильном порядке, когда вы не хотите, чтобы следующее изображение начинало загружаться, пока текущее не загрузилось полностью.

Помните, что `concat` начнёт испускать значения из следующего observable только после того, как завершится предыдущий. Это означает, что если один из ваших observable никогда не завершается, то последующие observable вообще не испустят ни одного значения. Такое поведение может стать _подводным камнем_: вывода не будет, ошибки тоже, но при этом один (или несколько) ваших внутренних observable могут работать не так, как задумано, либо подписка настроена неверно.

И наоборот, если вам нужно объединить observable, которые испускают значения одновременно, или вам требуются актуальные значения из нескольких observable при каждом новом испускании любого из них, более подходящими вариантами могут оказаться [combineLatest](<02. combineLatest.md>) или [withLatestFrom](<12. withLatestFrom.md>).



### Примеры

##### Пример 1: Базовое использование concat с тремя observable

(
[StackBlitz](https://stackblitz.com/edit/typescript-ks8chl?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { of, concat } from 'rxjs';

concat(
  of(1, 2, 3),
  // подписка происходит после завершения первого
  of(4, 5, 6),
  // подписка происходит после завершения второго
  of(7, 8, 9)
)
  // log: 1, 2, 3, 4, 5, 6, 7, 8, 9
  .subscribe(console.log);
```

##### Пример 2: Вывод сообщения с помощью concat и observable с задержкой

( [StackBlitz](https://stackblitz.com/edit/typescript-jtzuaa?file=index.ts) )

![Пример 2](https://drive.google.com/uc?export=view&id=1fKsYUKXkSWEDLdii-5rmOAgqy6sUGNjl)

```js
// RxJS v6+
import { concat, empty } from 'rxjs';
import { delay, startWith } from 'rxjs/operators';

// элементы
const userMessage = document.getElementById('message');
// вспомогательная функция
const delayedMessage = (message, delayedTime = 1000) => {
  return empty().pipe(startWith(message), delay(delayedTime));
};

concat(
  delayedMessage('Get Ready!'),
  delayedMessage(3),
  delayedMessage(2),
  delayedMessage(1),
  delayedMessage('Go!'),
  delayedMessage('', 2000)
).subscribe((message: any) => (userMessage.innerHTML = message));
```

##### Пример 3: (Внимание!) concat с источником, который не завершается

(
[StackBlitz](https://stackblitz.com/edit/typescript-njc2jw?file=index.ts&devtoolsheight=100)
)

```js
// RxJS v6+
import { interval, of, concat } from 'rxjs';

// когда источник никогда не завершается, все последующие observable никогда не запускаются
concat(interval(1000), of('This', 'Never', 'Runs'))
  // log: 1,2,3,4.....
  .subscribe(console.log);
```

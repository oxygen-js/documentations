# mergeAll

#### сигнатура: `mergeAll(concurrent: number): Observable`

## Собрать все observable и подписаться на них.

---

💡 Во многих случаях вместо этого можно использовать [mergeMap](<../07. Transformation/13. mergeMap - flatMap.md>) как
единый оператор!

---

### Зачем нужен mergeAll?
Этот оператор лучше всего подходит, когда у вас есть несколько короткоживущих observable, которые производят значения независимо друг от друга, и вы хотите уплотнить их в единый выходной поток. Реальные примеры этого можно увидеть в сценариях вроде одновременной обработки нескольких запросов, где каждый запрос — это observable, а ответы нужно объединить в единый поток.

Представьте себе веб-страницу, отображающую живые новостные обновления из разных источников. Каждый источник производит обновления в виде отдельных observable, и вы хотите объединить их все вместе, чтобы показать единую ленту новостных обновлений. В этом случае mergeAll приходит на помощь.

Важно отметить, что `mergeAll` начнёт испускать значения, как только любой из внутренних observable испустит значение. Это отличается от combineLatest, который ждёт, пока каждый observable испустит хотя бы одно значение, прежде чем производить вывод.

Выбирая между mergeAll и другими операторами, держите в уме следующее:

- Если вам нужно объединить несколько observable, зависящих друг от друга при вычислениях или принятии решений, [`combineLatest`](<02. combineLatest.md>) может подойти лучше.
- Если вы работаете с observable, которые испускают только одно значение, или вам нужно лишь последнее значение каждого из них перед завершением, [`forkJoin`](<06. forkJoin.md>), вероятно, будет лучшим выбором.
- Если вам нужно объединить observable, которые производят значения независимо и короткоживущи, `mergeAll` — тот оператор, к которому стоит обратиться.

##### Пример 1: mergeAll с промисами

(
[StackBlitz](https://stackblitz.com/edit/typescript-y4ncvc?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/worecuhiba/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/0sc4nsxa/) )

```js
// RxJS v6+
import { map, mergeAll } from 'rxjs/operators';
import { of } from 'rxjs';

const myPromise = val =>
  new Promise(resolve => setTimeout(() => resolve(`Result: ${val}`), 2000));
// испустить 1,2,3
const source = of(1, 2, 3);

const example = source.pipe(
  // отобразить каждое значение в промис
  map(val => myPromise(val)),
  // испустить результат от источника
  mergeAll()
);

/*
  вывод:
  "Result: 1"
  "Result: 2"
  "Result: 3"
*/
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 2: mergeAll с параметром _concurrent_

(
[StackBlitz](https://stackblitz.com/edit/typescript-xpaqjh?file=index.ts&devtoolsheight=100)
| [jsFiddle](https://jsfiddle.net/zra3zxhs/) )

```js
// RxJS v6+
import { take, map, delay, mergeAll } from 'rxjs/operators';
import { interval } from 'rxjs';

const source = interval(500).pipe(take(5));

/*
  interval испускает значение каждые 0.5с.  Затем это значение отображается в interval,
  который задержан на 1.0с.  Оператор mergeAll принимает необязательный аргумент, определяющий, на сколько
  внутренних observable подписываться одновременно.  Остальные observable хранятся
  в очереди в ожидании подписки.
*/
const example = source
  .pipe(
    map(val => source.pipe(delay(1000), take(3))),
    mergeAll(2)
  )
  .subscribe(val => console.log(val));
/*
  Подписка завершается, как только оператор испустит все значения.
*/
```

# withLatestFrom

#### сигнатура: `withLatestFrom(other: Observable, project: Function): Observable`

## Дополнительно предоставляет последнее значение из другого observable.

---

💡 Если вам нужно последнее значение при каждом испускании переменного числа
observable, попробуйте [combinelatest](<02. combineLatest.md>)!

---

### Зачем нужен withLatestFrom?
Оператор `withLatestFrom` — ваш лучший друг, когда у вас есть один основной observable, испускания которого зависят от последних значений одного или нескольких других observable. Представьте это как однонаправленный поток данных, где главный observable ведёт за собой, а остальные observable добавляют свои самые свежие значения.

Классический пример для запоминания `withLatestFrom` — приложение для чата, которому нужно отправить сообщение с текущим местоположением пользователя. Событие отправки сообщения (основной observable) комбинируется с последними данными о местоположении (другой observable), формируя итоговый объект сообщения.

Имейте в виду, что `withLatestFrom` испускает значение только тогда, когда испускает основной observable, и после того, как каждый из дополнительных observable испустил хотя бы раз. Это может застать вас врасплох, ведь вы можете не увидеть ни вывода, ни ошибок, пока один из observable ведёт себя не так, как ожидалось, или подписка задерживается.

Если вам нужно комбинировать значения из нескольких observable, которые испускают более одного раза и взаимозависимы, лучше используйте [`combineLatest`](<02. combineLatest.md>). А для сценариев, где observable испускают только один раз или вам нужны лишь их последние значения, более подходящим выбором может оказаться [`forkJoin`](<06. forkJoin.md>).



### Примеры

##### Пример 1: Последнее значение из более быстрого второго источника

(
[StackBlitz](https://stackblitz.com/edit/typescript-tznzbj?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/fitekeseru/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/9c3pfgpk/) )

```js
// RxJS v6+
import { withLatestFrom, map } from 'rxjs/operators';
import { interval } from 'rxjs';

//испускает каждые 5 секунд
const source = interval(5000);
//испускает каждую секунду
const secondSource = interval(1000);
const example = source.pipe(
  withLatestFrom(secondSource),
  map(([first, second]) => {
    return `First Source (5s): ${first} Second Source (1s): ${second}`;
  })
);
/*
  "First Source (5s): 0 Second Source (1s): 4"
  "First Source (5s): 1 Second Source (1s): 9"
  "First Source (5s): 2 Second Source (1s): 14"
  ...
*/
const subscribe = example.subscribe(val => console.log(val));
```

##### Пример 2: Более медленный второй источник

(
[StackBlitz](https://stackblitz.com/edit/typescript-gigsdv?file=index.ts&devtoolsheight=100)
| [jsBin](http://jsbin.com/vujekucuxa/1/edit?js,console) |
[jsFiddle](https://jsfiddle.net/btroncone/bywLL579/) )

```js
// RxJS v6+
import { withLatestFrom, map } from 'rxjs/operators';
import { interval } from 'rxjs';

//испускает каждые 5 секунд
const source = interval(5000);
//испускает каждую секунду
const secondSource = interval(1000);
//withLatestFrom медленнее источника
const example = secondSource.pipe(
  //оба источника должны испустить хотя бы 1 значение (5 секунд), прежде чем произойдёт испускание
  withLatestFrom(source),
  map(([first, second]) => {
    return `Source (1s): ${first} Latest From (5s): ${second}`;
  })
);
/*
  "Source (1s): 4 Latest From (5s): 0"
  "Source (1s): 5 Latest From (5s): 0"
  "Source (1s): 6 Latest From (5s): 0"
  ...
*/
const subscribe = example.subscribe(val => console.log(val));
```

# Знакомство с воркерами

В этой заключительной статье нашего модуля «Асинхронный JavaScript» мы познакомимся с _воркерами (workers)_, которые позволяют выполнять некоторые задачи в отдельном [потоке](https://developer.mozilla.org/en-US/docs/Glossary/Thread) выполнения.

| | |
|---|---|
| **Предварительные требования:** | Уверенное понимание [основ JavaScript](https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/Scripting) и асинхронных концепций, рассмотренных в предыдущих уроках этого модуля. |
| **Результаты обучения:** | <ul><li>Как использовать выделенные веб-воркеры и зачем.</li><li>Понять назначение других типов веб-воркеров — таких как общие (shared) и сервис-воркеры.</li></ul> |

В первой статье этого модуля мы видели, что происходит, когда в программе есть долго выполняющаяся синхронная задача, — всё окно становится полностью неотзывчивым. По сути, причина в том, что программа _однопоточная_. _Поток (thread)_ — это последовательность инструкций, которым следует программа. Поскольку программа состоит из одного потока, она может делать только одну вещь за раз: так что, если она ждёт возврата из нашего долго выполняющегося синхронного вызова, она не может делать ничего другого.

Воркеры дают вам возможность выполнять некоторые задачи в другом потоке, так что вы можете запустить задачу, а затем продолжить другую обработку (например, обработку действий пользователя).

Одна из проблем при всём этом состоит в том, что если несколько потоков могут иметь доступ к одним и тем же общим данным, то они могут изменять их независимо и непредсказуемо (по отношению друг к другу). Это может приводить к ошибкам, которые трудно найти.

Чтобы избежать этих проблем в вебе, ваш основной код и код воркера никогда не получают прямого доступа к переменным друг друга и могут по-настоящему «делиться» данными лишь в очень специфических случаях. Воркеры и основной код работают в совершенно раздельных мирах и взаимодействуют, только отправляя друг другу сообщения. В частности, это означает, что воркеры не могут обращаться к DOM (окну, документу, элементам страницы и так далее).

Существует три разных вида воркеров:

- выделенные воркеры (dedicated workers)
- общие воркеры (shared workers)
- сервис-воркеры (service workers)

В этой статье мы разберём пример первого вида воркеров, а затем вкратце обсудим два других.

## Использование веб-воркеров

Помните первую статью, где у нас была страница, вычисляющая простые числа? Мы собираемся использовать воркер для выполнения вычисления простых чисел, чтобы наша страница оставалась отзывчивой к действиям пользователя.

### Синхронный генератор простых чисел

Давайте сначала ещё раз взглянем на JavaScript из нашего предыдущего примера:

```js
function generatePrimes(quota) {
  function isPrime(n) {
    for (let c = 2; c <= Math.sqrt(n); ++c) {
      if (n % c === 0) {
        return false;
      }
    }
    return true;
  }

  const primes = [];
  const maximum = 1000000;

  while (primes.length < quota) {
    const candidate = Math.floor(Math.random() * (maximum + 1));
    if (isPrime(candidate)) {
      primes.push(candidate);
    }
  }

  return primes;
}

document.querySelector("#generate").addEventListener("click", () => {
  const quota = document.querySelector("#quota").value;
  const primes = generatePrimes(quota);
  document.querySelector("#output").textContent =
    `Finished generating ${quota} primes!`;
});

document.querySelector("#reload").addEventListener("click", () => {
  document.querySelector("#user-input").value =
    'Try typing in here immediately after pressing "Generate primes"';
  document.location.reload();
});
```

В этой программе после того, как мы вызываем `generatePrimes()`, программа становится полностью неотзывчивой.

### Генерация простых чисел с помощью воркера

Для этого примера начните с создания локальной копии файлов по адресу <https://github.com/mdn/learning-area/tree/main/javascript/asynchronous/workers/start>. В этом каталоге четыре файла:

- index.html
- style.css
- main.js
- generate.js

Файлы «index.html» и «style.css» уже готовы:

```html
<!doctype html>
<html lang="en-US">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width" />
    <title>Prime numbers</title>
    <script src="main.js" defer></script>
    <link href="style.css" rel="stylesheet" />
  </head>

  <body>
    <label for="quota">Number of primes:</label>
    <input type="text" id="quota" name="quota" value="1000000" />

    <button id="generate">Generate primes</button>
    <button id="reload">Reload</button>

    <textarea id="user-input" rows="5" cols="62">
Try typing in here immediately after pressing "Generate primes"
    </textarea>

    <div id="output"></div>
  </body>
</html>
```

```css
textarea {
  display: block;
  margin: 1rem 0;
}
```

Файлы «main.js» и «generate.js» пусты. Мы собираемся добавить основной код в «main.js», а код воркера — в «generate.js».

Итак, во-первых, мы видим, что код воркера хранится в отдельном скрипте, отдельно от основного кода. Мы также видим, глядя на «index.html» выше, что в элемент `<script>` включён только основной код.

Теперь скопируйте следующий код в «main.js»:

```js
// Создаём новый воркер, передавая ему код из "generate.js"
const worker = new Worker("./generate.js");

// Когда пользователь нажимает "Generate primes", отправляем сообщение воркеру.
// Команда сообщения — "generate", а также сообщение содержит "quota" —
// количество простых чисел, которые нужно сгенерировать.
document.querySelector("#generate").addEventListener("click", () => {
  const quota = document.querySelector("#quota").value;
  worker.postMessage({
    command: "generate",
    quota,
  });
});

// Когда воркер отправляет сообщение обратно в основной поток,
// обновляем поле вывода сообщением для пользователя, включающим количество
// сгенерированных простых чисел, взятое из данных сообщения.
worker.addEventListener("message", (message) => {
  document.querySelector("#output").textContent =
    `Finished generating ${message.data} primes!`;
});

document.querySelector("#reload").addEventListener("click", () => {
  document.querySelector("#user-input").value =
    'Try typing in here immediately after pressing "Generate primes"';
  document.location.reload();
});
```

- Сначала мы создаём воркер с помощью конструктора [`Worker()`](https://developer.mozilla.org/en-US/docs/Web/API/Worker/Worker). Мы передаём ему URL, указывающий на скрипт воркера. Как только воркер создан, скрипт воркера выполняется.

- Затем, как и в синхронной версии, мы добавляем обработчик события `click` к кнопке «Generate primes». Но теперь, вместо того чтобы вызывать функцию `generatePrimes()`, мы отправляем сообщение воркеру с помощью [`worker.postMessage()`](https://developer.mozilla.org/en-US/docs/Web/API/Worker/postMessage). Это сообщение может принимать аргумент, и в данном случае мы передаём JSON-объект с двумя свойствами:
  - `command`: строка, идентифицирующая то, что мы хотим, чтобы воркер сделал (на случай, если наш воркер мог бы делать больше одной вещи);
  - `quota`: количество простых чисел для генерации.

- Затем мы добавляем обработчик события `message` к воркеру. Это нужно, чтобы воркер мог сообщить нам, когда он закончит, и передать нам любые полученные данные. Наш обработчик берёт данные из свойства `data` сообщения и записывает их в элемент вывода (данные в точности совпадают с `quota`, так что это немного бессмысленно, но демонстрирует сам принцип).

- Наконец, мы реализуем обработчик события `click` для кнопки «Reload». Он в точности такой же, как в синхронной версии.

Теперь код воркера. Скопируйте следующий код в «generate.js»:

```js
// Слушаем сообщения от основного потока.
// Если команда сообщения — "generate", вызываем `generatePrimes()`
addEventListener("message", (message) => {
  if (message.data.command === "generate") {
    generatePrimes(message.data.quota);
  }
});

// Генерируем простые числа (очень неэффективно)
function generatePrimes(quota) {
  function isPrime(n) {
    for (let c = 2; c <= Math.sqrt(n); ++c) {
      if (n % c === 0) {
        return false;
      }
    }
    return true;
  }

  const primes = [];
  const maximum = 1000000;

  while (primes.length < quota) {
    const candidate = Math.floor(Math.random() * (maximum + 1));
    if (isPrime(candidate)) {
      primes.push(candidate);
    }
  }

  // Когда закончили, отправляем сообщение в основной поток,
  // включая количество сгенерированных нами простых чисел.
  postMessage(primes.length);
}
```

Помните, что этот код выполняется сразу же, как только основной скрипт создаёт воркер.

Первое, что делает воркер, — начинает слушать сообщения от основного скрипта. Он делает это с помощью `addEventListener()`, которая является глобальной функцией внутри воркера. Внутри обработчика события `message` свойство `data` события содержит копию аргумента, переданного из основного скрипта. Если основной скрипт передал команду `generate`, мы вызываем `generatePrimes()`, передавая значение `quota` из события сообщения.

Функция `generatePrimes()` точно такая же, как в синхронной версии, только вместо того, чтобы возвращать значение, мы отправляем сообщение в основной скрипт, когда закончили. Для этого мы используем функцию [`postMessage()`](https://developer.mozilla.org/en-US/docs/Web/API/DedicatedWorkerGlobalScope/postMessage), которая, как и `addEventListener()`, является глобальной функцией внутри воркера. Как мы уже видели, основной скрипт слушает это сообщение и обновит DOM, когда сообщение будет получено.

> [!NOTE]
> Чтобы запустить этот сайт, вам придётся запустить локальный веб-сервер, потому что URL вида `file://` не имеют права загружать воркеры. О том, как это сделать, читайте в статье [«Как настроить локальный тестовый сервер?»](https://developer.mozilla.org/en-US/docs/Learn_web_development/Howto/Tools_and_setup/set_up_a_local_testing_server). Сделав это, вы должны получить возможность нажать «Generate primes» так, чтобы ваша основная страница оставалась отзывчивой.
>
> Если у вас возникнут какие-либо проблемы с созданием или запуском примера, вы можете свериться с [готовой версией](https://github.com/mdn/learning-area/tree/main/javascript/asynchronous/workers/finished) и попробовать её [вживую](https://mdn.github.io/learning-area/javascript/asynchronous/workers/finished/).

## Другие типы воркеров

Воркер, который мы только что создали, — это так называемый _выделенный воркер (dedicated worker)_. Это значит, что он используется одним-единственным экземпляром скрипта.

Однако есть и другие типы воркеров:

- [_Общие воркеры (shared workers)_](https://developer.mozilla.org/en-US/docs/Web/API/SharedWorker) могут использоваться совместно несколькими разными скриптами, работающими в разных окнах.
- [_Сервис-воркеры (service workers)_](https://developer.mozilla.org/en-US/docs/Web/API/Service_Worker_API) действуют как прокси-серверы, кэшируя ресурсы, чтобы веб-приложения могли работать, когда пользователь находится офлайн. Они являются ключевым компонентом [прогрессивных веб-приложений (Progressive Web Apps)](https://developer.mozilla.org/en-US/docs/Web/Progressive_web_apps).

## Итоги

В этой статье мы познакомились с веб-воркерами, которые позволяют веб-приложению перекладывать задачи в отдельный поток. Основной поток и воркер напрямую не разделяют никаких переменных, а взаимодействуют, отправляя сообщения, которые принимаются другой стороной как события `message`.

Воркеры могут быть эффективным способом сохранять отзывчивость основного приложения, хотя они не могут обращаться ко всем тем API, к которым может обращаться основное приложение, и, в частности, не могут обращаться к DOM.

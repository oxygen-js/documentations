# Разбор таблицы стилей

Таблица стилей Sass разбирается из последовательности юникод-символов (code
points). Она разбирается напрямую, без предварительного преобразования в
поток токенов.

## Кодировка входных данных

**Совместимость:** Dart Sass: ✗ · LibSass: ✓ · Ruby Sass: ✓

Dart Sass в настоящее время поддерживает *только* кодировку UTF-8. Поэтому
  безопаснее всего кодировать все таблицы стилей Sass в UTF-8.


Часто документ изначально доступен только как последовательность байтов,
которую нужно декодировать в юникод. Sass выполняет это декодирование
следующим образом:

* Если последовательность байтов начинается с кодировки UTF-8 или UTF-16
  символа U+FEFF BYTE ORDER MARK, используется соответствующая кодировка.

* Если последовательность байтов начинается с простой ASCII-строки
  `@charset`, Sass определяет кодировку с помощью шага 2 алгоритма CSS для
  [определения запасной кодировки][determining the fallback encoding].

  [determining the fallback encoding]: https://drafts.csswg.org/css-syntax-3/#input-byte-stream

* В противном случае используется UTF-8.

## Ошибки разбора

Когда Sass обнаруживает недопустимый синтаксис в таблице стилей, разбор
завершается неудачей, и пользователю показывается ошибка с информацией о
месте недопустимого синтаксиса и причине, по которой он недопустим.

Обратите внимание, что это отличается от CSS, который определяет, как
восстанавливаться после большинства ошибок, вместо того чтобы сразу
завершаться неудачей. Это один из немногих случаев, когда SCSS не является
*строго* надмножеством CSS. Однако для пользователей Sass гораздо полезнее
видеть ошибки сразу, а не получать их в неизменном виде в результирующем
CSS.

Место возникновения ошибок разбора можно получить через специфичные для
реализации API. Например, в Dart Sass доступно свойство
[`SassException.span`][], а в JS API Node Sass и Dart Sass — свойства
[`file`, `line` и `column`][js error].

[`SassException.span`]: https://pub.dartlang.org/documentation/sass/latest/sass/SassException/span.html
[js error]: https://github.com/sass/node-sass#error-object

# Встроенные модули

Sass предоставляет множество встроенных модулей, которые содержат полезные
функции (а иногда и миксины). Эти модули можно загрузить с помощью [`@use`-правила](https://sass-lang.com/documentation/at-rules/use), как и любую пользовательскую таблицу стилей, а их
функции можно вызывать [как и любой другой член модуля](https://sass-lang.com/documentation/at-rules/use#loading-members). URL-адреса всех встроенных модулей
начинаются с `sass:`, указывая на то, что они являются частью самого Sass.

> ⚠️ **Внимание**
>
> До появления системы модулей Sass все функции Sass были глобально доступны
> в любой момент. У многих функций до сих пор есть глобальные псевдонимы (они
> указаны в их документации). Команда Sass не рекомендует их использовать и
> со временем объявит их устаревшими, но пока что они остаются доступны для
> совместимости со старыми версиями Sass и с LibSass (который пока не
> поддерживает систему модулей).
>
> [Несколько функций][] доступны глобально *только* в новой системе модулей,
> потому что они добавляют дополнительное поведение поверх встроенных функций
> CSS.

[несколько функций]: #global-functions
[`rgb()`]: #rgb
[`hsl()`]: #hsl

**SCSS**

```scss
@use "sass:color";

.button {
  $primary-color: #6b717f;
  color: $primary-color;
  border: 1px solid color.scale($primary-color, $lightness: 20%);
}
```

**Sass**

```sass
@use "sass:color"

.button
  $primary-color: #6b717f
  color: $primary-color
  border: 1px solid color.scale($primary-color, $lightness: 20%)
```

**Результат CSS**

```css
.button {
  color: #6b717f;
  border: 1px solid rgb(135.1641025641, 140.8256410256, 154.0358974359);
}
```

Sass предоставляет следующие встроенные модули:

* [Модуль `sass:math`][] предоставляет функции для работы с [числами][].

* [Модуль `sass:string`][] упрощает объединение, поиск и разбиение [строк][].

* [Модуль `sass:color`][] создаёт новые [цвета][] на основе существующих,
  что упрощает построение цветовых тем.

* [Модуль `sass:list`][] позволяет обращаться к значениям в [списках][] и
  изменять их.

* [Модуль `sass:map`][] позволяет искать значение по ключу в [карте][] (map)
  и многое другое.

* [Модуль `sass:selector`][] даёт доступ к мощному движку селекторов Sass.

* [Модуль `sass:meta`][] раскрывает подробности внутреннего устройства Sass.

## Глобальные функции

> 💡 **Занятный факт**
>
> Вы можете передавать [специальные функции] вроде `calc()` или `var()`
> вместо любого аргумента глобального конструктора цвета. Вы даже можете
> использовать `var()` вместо нескольких аргументов, поскольку она может быть
> заменена несколькими значениями! Когда функция цвета вызывается таким
> образом, она возвращает строку без кавычек с той же сигнатурой, с которой
> была вызвана.
>
> **SCSS**
>
> ```scss
> @debug rgb(0 51 102 / var(--opacity)); // rgb(0 51 102 / var(--opacity))
> @debug color(display-p3 var(--peach)); // color(display-p3 var(--peach))
> ```
>
> **Sass**
>
> ```sass
> @debug rgb(0 51 102 / var(--opacity))  // rgb(0 51 102 / var(--opacity))
> @debug color(display-p3 var(--peach))  // color(display-p3 var(--peach))
> ```

[специальные функции]: https://sass-lang.com/documentation/syntax/special-functions

**`color($space $channel1 $channel2 $channel3)`**

**`color($space $channel1 $channel2 $channel3 / $alpha)`**

*Возвращает: `color`*

  **Совместимость:** Dart Sass: начиная с 1.78.0 · LibSass: ✗ · Ruby Sass: ✗


  Возвращает цвет в заданном цветовом пространстве с заданными значениями
  каналов.

  Поддерживаются цветовые пространства `srgb`, `srgb-linear`, `display-p3`,
  `a98-rgb`, `prophoto-rgb`, `rec2020`, `xyz` и `xyz-d50`, а также `xyz-d65`,
  который является псевдонимом для `xyz`. Для всех пространств каналы — это
  числа от 0 до 1 (включительно) или проценты от `0%` до `100%` (включительно).

  Если значение какого-либо цветового канала выходит за пределы диапазона от
  0 до 1, это означает цвет за пределами стандартного охвата данного
  цветового пространства.

  **SCSS**

```scss
@debug color(srgb 0.1 0.6 1); // color(srgb 0.1 0.6 1)
@debug color(xyz 30% 0% 90% / 50%); // color(xyz 0.3 0 0.9 / 50%)
```

**Sass**

```sass
@debug color(srgb 0.1 0.6 1)  // color(srgb 0.1 0.6 1)
@debug color(xyz 30% 0% 90% / 50%)  // color(xyz 0.3 0 0.9 / 50%)
```

**`hsl($hue $saturation $lightness)`**

**`hsl($hue $saturation $lightness / $alpha)`**

**`hsl($hue, $saturation, $lightness, $alpha: 1)`**

**`hsla($hue $saturation $lightness)`**

**`hsla($hue $saturation $lightness / $alpha)`**

**`hsla($hue, $saturation, $lightness, $alpha: 1)`**

*Возвращает: `color`*

  **Совместимость (синтаксис уровня 4):** Dart Sass: начиная с 1.15.0 · LibSass: ✗ · Ruby Sass: ✗

LibSass и Ruby Sass поддерживают только следующие сигнатуры:

    * `hsl($hue, $saturation, $lightness)`
    * `hsla($hue, $saturation, $lightness, $alpha)`

    Обратите внимание, что для этих реализаций аргумент `$alpha` *обязателен*,
    если используется имя функции `hsla()`, и *запрещён*, если используется
    имя функции `hsl()`.


  **Совместимость (проценты для альфа-канала):** Dart Sass: ✓ · LibSass: ✗ · Ruby Sass: начиная с 3.7.0

LibSass и более старые версии Ruby Sass не поддерживают указание значений альфа-канала в процентах.


  Возвращает цвет с заданными [оттенком, насыщенностью и светлотой][] и
  заданным альфа-каналом.

  [оттенком, насыщенностью и светлотой]: https://en.wikipedia.org/wiki/HSL_and_HSV

  Оттенок — это число от `0deg` до `360deg` (включительно), которое может
  быть безразмерным. Насыщенность и светлота обычно представляют собой числа
  от `0%` до `100%` (включительно) и *не могут* быть безразмерными.
  Альфа-канал можно задать либо безразмерным числом от 0 до 1 (включительно),
  либо процентом от `0%` до `100%` (включительно).

  Оттенок за пределами `0deg` и `360deg` эквивалентен `$hue % 360deg`.
  Насыщенность меньше `0%` ограничивается значением `0%`. Насыщенность выше
  `100%` или светлота за пределами `0%` и `100%` допустимы и означают цвета
  за пределами стандартного охвата RGB.

  > ⚠️ **Внимание**
  >
  > [Особые правила разбора][] Sass для значений, разделённых слэшем,
  > затрудняют передачу переменных для `$lightness` или `$alpha` при
  > использовании сигнатуры `hsl($hue $saturation $lightness / $alpha)`.
  > Вместо этого рассмотрите использование
  > `hsl($hue, $saturation, $lightness, $alpha)`.

  [особые правила разбора]: https://sass-lang.com/documentation/operators/numeric#slash-separated-values

  **SCSS**

```scss
@debug hsl(210deg 100% 20%); // #036
@debug hsl(210deg 100% 20% / 50%); // rgba(0, 51, 102, 0.5)
@debug hsla(34, 35%, 92%, 0.2); // rgba(241.74, 235.552, 227.46, 0.2)
```

**Sass**

```sass
@debug hsl(210deg 100% 20%) // #036
@debug hsl(210deg 100% 20% / 50%)  // rgba(0, 51, 102, 0.5)
@debug hsla(34, 35%, 92%, 0.2)  // rgba(241.74, 235.552, 227.46, 0.2)
```

**`hwb($hue $whiteness $blackness)`**

**`hwb($hue $whiteness $blackness / $alpha)`**

**`color.hwb($hue $whiteness $blackness)`**

**`color.hwb($hue $whiteness $blackness / $alpha)`**

**`color.hwb($hue, $whiteness, $blackness, $alpha: 1)`**

*Возвращает: `color`*

  **Совместимость:** Dart Sass: начиная с 1.78.0 · LibSass: ✗ · Ruby Sass: ✗


  Возвращает цвет с заданными [оттенком, белизной и чернотой] и заданным
  альфа-каналом.

  [оттенком, белизной и чернотой]: https://en.wikipedia.org/wiki/HWB_color_model

  Оттенок — это число от `0deg` до `360deg` (включительно), которое может
  быть безразмерным. Белизна и чернота — это числа, обычно от `0%` до `100%`
  (включительно), которые *не могут* быть безразмерными. Альфа-канал можно
  задать либо безразмерным числом от 0 до 1 (включительно), либо процентом
  от `0%` до `100%` (включительно).

  Оттенок за пределами `0deg` и `360deg` эквивалентен `$hue % 360deg`. Если
  `$whiteness + $blackness > 100%`, оба значения масштабируются так, чтобы
  их сумма составляла `100%`. Если `$whiteness`, `$blackness` или оба значения
  меньше `0%`, это означает цвет за пределами стандартного охвата RGB.

  > ⚠️ **Внимание**
  >
  > Варианты `color.hwb()` являются устаревшими. В новом коде Sass вместо
  > них следует использовать глобальную функцию `hwb()`.

  **SCSS**

```scss
@debug hwb(210deg 0% 60%); // #036
@debug hwb(210 0% 60% / 0.5); // rgba(0, 51, 102, 0.5)
```

**Sass**

```sass
@debug hwb(210deg 0% 60%)  // #036
@debug hwb(210 0% 60% / 0.5)  // rgba(0, 51, 102, 0.5)
```

  > затрудняют передачу переменных для `$blue` или `$alpha` при
  > использовании сигнатуры `rgb($red $green $blue / $alpha)`. Вместо этого
  > рассмотрите использование `rgb($red, $green, $blue, $alpha)`.

  [особые правила разбора]: https://sass-lang.com/documentation/operators/numeric#slash-separated-values

  **SCSS**

```scss
@debug rgb(0 51 102); // #036
@debug rgb(95%, 92.5%, 89.5%); // #f2ece4
@debug rgb(0 51 102 / 50%); // rgba(0, 51, 102, 0.5)
@debug rgba(95%, 92.5%, 89.5%, 0.2); // rgba(242, 236, 228, 0.2)
```

**Sass**

```sass
@debug rgb(0 51 102)  // #036
@debug rgb(95%, 92.5%, 89.5%)  // #f2ece4
@debug rgb(0 51 102 / 50%)  // rgba(0, 51, 102, 0.5)
@debug rgba(95%, 92.5%, 89.5%, 0.2)  // rgba(242, 236, 228, 0.2)
```

  ---

  Если переданы `$color` и `$alpha`, возвращается `$color` с заданным
  каналом `$alpha` вместо исходного альфа-канала.

  **SCSS**

```scss
@debug rgb(#f2ece4, 50%); // rgba(242, 236, 228, 0.5);
@debug rgba(rgba(0, 51, 102, 0.5), 1); // #003366
```

**Sass**

```sass
@debug rgb(#f2ece4, 50%)  // rgba(242, 236, 228, 0.5)
@debug rgba(rgba(0, 51, 102, 0.5), 1)  // #003366
```


## Устаревшие функции


---


**`if($condition, $if-true, $if-false)`**

  Возвращает `$if-true`, если `$condition` [истинно (truthy)][], и
  `$if-false` в противном случае.

  Особенность этой функции в том, что она даже не вычисляет тот аргумент,
  который не возвращается, поэтому её безопасно вызывать, даже если
  неиспользуемый аргумент привёл бы к ошибке.

  [истинно (truthy)]: https://sass-lang.com/documentation/at-rules/control/if#truthiness-and-falsiness

  > ⚠️ **Внимание**
  >
  > Теперь, когда CSS поддерживает собственный [синтаксис функции `if()`],
  > Sass переходит на использование этого синтаксиса. Старая функция `if()`
  > считается устаревшей, хотя она продолжит поддерживаться в Dart Sass
  > вплоть до версии 3.0.0. Подробнее см. [/d/if-function].

  [синтаксис функции `if()`]: https://sass-lang.com/documentation/syntax/special-functions#if
  [/d/if-function]: https://sass-lang.com/documentation/breaking-changes/if-function

  **SCSS**

```scss
@debug if(true, 10px, 15px); // 10px
@debug if(false, 10px, 15px); // 15px
@debug if(variable-defined($var), $var, null); // null
```

**Sass**

```sass
@debug if(true, 10px, 15px)  // 10px
@debug if(false, 10px, 15px)  // 15px
@debug if(variable-defined($var), $var, null)  // null
```

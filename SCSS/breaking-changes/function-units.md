# Ломающее изменение: строгие единицы измерения в функциях

Различные встроенные функции станут строже в том, какие единицы измерения они допускают, и будут обрабатывать эти единицы более последовательно. Это делает Sass более совместимым со спецификацией CSS и помогает быстрее находить ошибки.

## Оттенок

**Совместимость:** Dart Sass: начиная с 1.32.0 · LibSass: ✗ · Ruby Sass: ✗


При указании оттенка (hue) цвета CSS допускает любую [единицу измерения угла][] (`deg`, `grad`, `rad` или `turn`). Также допускается число без единицы измерения, которое трактуется как `deg`. Исторически Sass допускал *любую* единицу измерения и интерпретировал её как `deg`. Это особенно проблематично, поскольку означало, что корректное CSS-выражение `hsl(0.5turn, 100%, 50%)` было бы допустимо в Sass, но интерпретировалось бы совершенно неверно.

[единица измерения угла]: https://drafts.csswg.org/css-values-4/#angles

Чтобы исправить эту проблему и привести Sass в соответствие со спецификацией CSS, мы вносим изменения в несколько этапов:

### Этап 1

**Совместимость:** Dart Sass: начиная с 1.32.0 · LibSass: ✗ · Ruby Sass: ✗


Сначала Sass просто выводил предупреждение об устаревании, если в качестве оттенка в какую-либо функцию передавалось число с единицей измерения, отличной от `deg`. Передача числа без единицы измерения по-прежнему допускается.

### Этап 2

**Совместимость:** Dart Sass: начиная с 1.52.1 · LibSass: ✗ · Ruby Sass: ✗


Затем мы изменили способ обработки единиц измерения угла для параметров оттенка в соответствии со спецификацией CSS. Это означает, что числа с единицами измерения `grad`, `rad` или `turn` будут преобразовываться в `deg`: `0.5turn` будет преобразовано в `180deg`, `100grad` — в `90deg`, и так далее.

Поскольку это изменение необходимо для сохранения совместимости с CSS, согласно [политике совместимости Dart Sass] оно было внесено с повышением лишь минорной версии. При этом оно меняет поведение минимально возможным образом, чтобы гарантировать, что Sass интерпретирует весь допустимый CSS в соответствии со спецификацией CSS.

[политике совместимости Dart Sass]: https://github.com/sass/dart-sass#compatibility-policy

### Этап 3

**Совместимость:** Dart Sass: ✗ · LibSass: ✗ · Ruby Sass: ✗


Наконец, в Dart Sass 2.0.0 функции работы с цветом будут выбрасывать ошибки, если им передан параметр оттенка с единицей измерения, не являющейся единицей измерения угла. Оттенки без единицы измерения по-прежнему будут допустимы.

## Насыщенность и светлота

При указании насыщенности и светлоты цвета HSL CSS допускает только единицу измерения `%`. Даже числа без единицы измерения не допускаются (в отличие от оттенка). Исторически Sass допускал *любую* единицу измерения и интерпретировал её как `%`. Можно было даже написать `hsl(0, 100px, 50s)`, и Sass вернул бы цвет `red`.

Чтобы исправить эту проблему и привести Sass в соответствие со спецификацией CSS, мы вносим изменения в два этапа:

### Этап 1

**Совместимость:** Dart Sass: начиная с 1.32.0 · LibSass: ✗ · Ruby Sass: ✗


В настоящее время Sass просто выводит предупреждение об устаревании, если в качестве светлоты или насыщенности в какую-либо функцию передаётся число без единицы измерения или с единицей измерения, отличной от `%`.

### Этап 2

**Совместимость:** Dart Sass: ✗ · LibSass: ✗ · Ruby Sass: ✗


В Dart Sass 2.0.0 функции работы с цветом будут выбрасывать ошибки, если им передан параметр насыщенности или светлоты без единицы измерения либо с единицей измерения, отличной от `%`.

## Альфа-канал

При указании альфа-значения цвета CSS (начиная с [Colors Level 4]) допускает либо числа без единицы измерения в диапазоне от 0 до 1, либо значения в `%` в диапазоне от `0%` до `100%`. В большинстве случаев Sass следует этому поведению, но функции `color.adjust()` и `color.change()` исторически допускали *любую* единицу измерения и интерпретировали её так, будто единицы измерения нет. Можно было даже написать `color.change(red, $alpha: 1%)`, и Sass вернул бы непрозрачный цвет `red`.

[Colors Level 4]: https://www.w3.org/TR/css-color-4/#typedef-alpha-value

Чтобы исправить эту проблему и привести Sass в соответствие со спецификацией CSS, мы вносим изменения в три этапа:

### Этап 1

**Совместимость:** Dart Sass: начиная с 1.56.0 · LibSass: ✗ · Ruby Sass: ✗


В настоящее время Sass просто выводит предупреждение об устаревании, если в `color.change()` или `color.adjust()` в качестве альфа-значения передаётся число с любой единицей измерения, включая `%`.

### Этап 2

**Совместимость:** Dart Sass: ✗ · LibSass: ✗ · Ruby Sass: ✗


Затем мы изменим способ обработки единицы измерения `%` для аргумента альфа-канала в `color.change()` и `color.adjust()`. Значения альфа-канала с единицей измерения `%` будут делиться на `100%`, преобразуясь в числа без единицы измерения в диапазоне от 0 до 1.

Поскольку это изменение является исправлением ошибки, улучшающим согласованность с другими функциями Sass, оно будет внесено с повышением лишь минорной версии. Оно будет внесено не раньше чем через три месяца после выпуска этапа 1, чтобы дать пользователям время адаптировать свой код и избежать этой ошибки.

[политике совместимости Dart Sass]: https://github.com/sass/dart-sass#compatibility-policy

### Этап 3

**Совместимость:** Dart Sass: ✗ · LibSass: ✗ · Ruby Sass: ✗


Наконец, в Dart Sass 2.0.0 функции `color.change()` и `color.adjust()` будут выбрасывать ошибки, если им передан параметр альфа-канала с единицей измерения, отличной от `%`. Значения альфа-канала без единицы измерения по-прежнему будут допустимы.

## `math.random()`

[Функция `math.random()`] исторически игнорировала единицы измерения в `$limit` и возвращала значение без единицы измерения. Например, `math.random(100px)` отбрасывала «px» и возвращала значение вроде `42`.

Будущая версия Sass перестанет игнорировать единицы измерения для аргумента `$limit` и будет возвращать случайное целое число с теми же единицами измерения.

[Функция `math.random()`]: https://sass-lang.com/documentation/modules/math#random

**SCSS**

```scss
@use "sass:math";

// Future Sass, doesn't work yet!
@debug math.random(100px); // 42px
```

**Sass**

```sass
@use "sass:math"

// Future Sass, doesn't work yet!
@debug math.random(100px)  // 42px
```

### Этап 1

**Совместимость:** Dart Sass: начиная с 1.54.5 · LibSass: ✗ · Ruby Sass: ✗


В настоящее время Sass выводит предупреждение об устаревании, если в `math.random()` передаётся `$limit` с единицей измерения.

### Этап 2

**Совместимость:** Dart Sass: ✗ · LibSass: ✗ · Ruby Sass: ✗


В Dart Sass 2.0.0 передача числа `$limit` с единицей измерения будет приводить к ошибке.

### Этап 3

**Совместимость:** Dart Sass: ✗ · LibSass: ✗ · Ruby Sass: ✗


В одном из минорных релизов после Dart Sass 2.0.0 передача числа `$limit` с единицей измерения в функцию `math.random()` снова будет разрешена. Она будет возвращать случайное целое число с теми же единицами измерения, что и `$limit`, вместо числа без единицы измерения.

[`color.mix()`]: https://sass-lang.com/documentation/modules/color#mix
[`color.invert()`]: https://sass-lang.com/documentation/modules/color#invert

### Этап 1

**Совместимость:** Dart Sass: начиная с 1.56.0 · LibSass: ✗ · Ruby Sass: ✗


В настоящее время Sass выводит предупреждение об устаревании, если в `color.mix()` или `color.invert()` передаётся `$weight` без единицы измерения или с единицей измерения, отличной от `%`.

### Этап 2

**Совместимость:** Dart Sass: ✗ · LibSass: ✗ · Ruby Sass: ✗

[`list.nth()`]: https://sass-lang.com/documentation/modules/list#nth
[`list.set-nth()`]: https://sass-lang.com/documentation/modules/list#set-nth

### Этап 2

**Совместимость:** Dart Sass: ✗ · LibSass: ✗ · Ruby Sass: ✗

## Как отключить предупреждения?

Sass предоставляет широкий набор опций для управления тем, какие предупреждения об устаревании вы видите и когда.

### Краткий и подробный режим

По умолчанию Sass работает в кратком режиме (terse mode), в котором каждый вид предупреждения об устаревании выводится не более пяти раз, после чего дальнейшие предупреждения подавляются. Это помогает пользователям узнавать о предстоящих ломающих изменениях, не создавая при этом излишний шум в консоли.

Если же запустить Sass в подробном режиме (verbose mode), он будет выводить *каждое* встреченное предупреждение об устаревании. Это может быть полезно для отслеживания того, сколько работы осталось для устранения устареваний. Включить подробный режим можно с помощью

[флага `--verbose`] в командной строке, или

[опции `verbose`] в JavaScript API.

[флага `--verbose`]: https://sass-lang.com/documentation/cli/dart-sass/#verbose
[опции `verbose`]: https://sass-lang.com/documentation/js-api/interfaces/Options/#verbose

> ⚠️ **Внимание**
>
> При запуске из [JS API] Sass не сохраняет никакой информации между отдельными компиляциями, поэтому по умолчанию для *каждой* компилируемой таблицы стилей будет выведено до пяти предупреждений. Однако это можно исправить, написав (или попросив автора вашего любимого фреймворкового плагина Sass написать) [пользовательский `Logger`], который выводит не более пяти ошибок на каждое устаревание и может использоваться совместно для нескольких компиляций.

[JS API]: https://sass-lang.com/documentation/js-api/
[пользовательский `Logger`]: https://sass-lang.com/documentation/js-api/interfaces/Logger-1/

### Отключение предупреждений в зависимостях

Иногда в ваших зависимостях появляются предупреждения об устаревании, с которыми вы ничего не можете поделать. Вы можете отключить предупреждения об устаревании из зависимостей, продолжая при этом выводить их для вашего приложения, с помощью

[флага `--quiet-deps`] в командной строке, или

[опции `quietDeps`] в JavaScript API.

[флага `--quiet-deps`]: https://sass-lang.com/documentation/cli/dart-sass/#quiet-deps
[опции `quietDeps`]: https://sass-lang.com/documentation/js-api/interfaces/Options/#quietDeps

Для целей этого флага «зависимостью» считается любая таблица стилей, которая не является просто цепочкой относительных загрузок из входной таблицы стилей. Это означает всё, что приходит из пути загрузки, а также большинство таблиц стилей, загружаемых через пользовательские импортеры (custom importers).

### Отключение отдельных предупреждений

Если вы знаете, что конкретное устаревание не представляет для вас проблемы, вы можете отключить предупреждения именно для него с помощью

[флага `--silence-deprecation`] в командной строке, или

[опции `silenceDeprecations`] в JavaScript API.

[флага `--silence-deprecation`]: https://sass-lang.com/documentation/cli/dart-sass/#silence-deprecation
[опции `silenceDeprecations`]: https://sass-lang.com/documentation/js-api/interfaces/Options/#silenceDeprecations

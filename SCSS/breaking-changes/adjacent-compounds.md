# Ломающее изменение: смежные составные селекторы

В дальнейшем будет считаться ошибкой наличие двух
[составных селекторов](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Selectors/Selector_structure#compound_selector),
не разделённых пробельными символами, например `[id]a`.

Селекторы, записанные таким образом, не допускаются в CSS, и неясно, задуманы ли они как
эквивалент, например, `a[id]` или `[id] a`. Исторически Sass допускал их и разбирал так,
как будто между ними был пробел, но в будущем это станет ошибкой.

**Совместимость:** Dart Sass: начиная с 2.0.0 · LibSass: ✗ · Ruby Sass: ✗


Начиная с Dart Sass 2.0.0, между несколькими составными селекторами требуется пробельный
символ (или явный комбинатор, например `+` или `~`).

## Переходный период

**Совместимость:** Dart Sass: начиная с 1.100.0 · LibSass: ✗ · Ruby Sass: ✗


До выхода Dart Sass 2.0.0 смежные составные селекторы лишь порождают предупреждение об
устаревании.

## Можно ли отключить предупреждения?

Sass предоставляет мощный набор опций для управления тем, какие предупреждения об
устаревании вы видите и когда.

### Краткий и подробный режимы

По умолчанию Sass работает в кратком режиме, в котором каждый тип предупреждения об
устаревании выводится не более пяти раз, после чего дополнительные предупреждения этого
типа подавляются. Это помогает гарантировать, что пользователи узнают о необходимости
обратить внимание на предстоящее ломающее изменение, не создавая при этом чрезмерного
шума в консоли.

Если вместо этого вы запустите Sass в подробном режиме, он будет выводить *каждое*
встреченное предупреждение об устаревании. Это может быть полезно для отслеживания
оставшейся работы по исправлению устареваний. Включить подробный режим можно с помощью

флага [`--verbose`][`--verbose` flag] в командной строке, или

опции [`verbose`][`verbose` option] в JavaScript API.

[`--verbose` flag]: https://sass-lang.com/documentation/cli/dart-sass/#verbose
[`verbose` option]: https://sass-lang.com/documentation/js-api/interfaces/Options/#verbose

> ⚠️ **Внимание**
>
> При запуске из [JS API][JS API] Sass не передаёт информацию между компиляциями, поэтому
> по умолчанию будет выводить пять предупреждений для *каждой таблицы стилей*, которая
> компилируется. Однако вы можете это исправить, написав (или попросив автора вашего
> любимого фреймворкового плагина Sass написать) [пользовательский `Logger`][custom `Logger`],
> который выводит только пять ошибок на каждое устаревание и может использоваться
> совместно для нескольких компиляций.
>
> [JS API]: https://sass-lang.com/documentation/js-api/
> [custom `Logger`]: https://sass-lang.com/documentation/js-api/interfaces/Logger-1/

### Отключение устареваний в зависимостях

Иногда в ваших зависимостях есть предупреждения об устаревании, с которыми вы ничего
не можете поделать. Вы можете отключить предупреждения об устаревании из зависимостей,
продолжая выводить их для вашего приложения, с помощью

флага [`--quiet-deps`][`--quiet-deps` flag] в командной строке, или

опции [`quietDeps`][`quietDeps` option] в JavaScript API.

[`--quiet-deps` flag]: https://sass-lang.com/documentation/cli/dart-sass/#quiet-deps
[`quietDeps` option]: https://sass-lang.com/documentation/js-api/interfaces/Options/#quietDeps

Для целей этого флага «зависимостью» считается любая таблица стилей, которая не
загружается просто через цепочку относительных загрузок из точки входа. Это значит —
всё, что приходит из пути загрузки, и большинство таблиц стилей, загружаемых через
пользовательские импортеры.

### Отключение конкретных устареваний

Если вы знаете, что конкретное устаревание не является для вас проблемой, вы можете
отключить предупреждения об этом конкретном устаревании с помощью

флага [`--silence-deprecation`][`--silence-deprecation` flag] в командной строке, или

опции [`silenceDeprecations`][`silenceDeprecations` option] в JavaScript API.

[`--silence-deprecation` flag]: https://sass-lang.com/documentation/cli/dart-sass/#silence-deprecation
[`silenceDeprecations` option]: https://sass-lang.com/documentation/js-api/interfaces/Options/#silenceDeprecations

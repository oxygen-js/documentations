# Ломающее изменение: -moz-document

Раньше в Firefox существовало правило @-moz-document, требовавшее особого разбора. Поскольку поддержка этого правила убирается из Firefox, Sass также постепенно убирает поддержку его особого разбора.

Sass исторически поддерживал особый разбор правила `@-moz-document`. Поскольку [Firefox прекратил его поддержку], Sass тоже откажется от особого разбора и будет трактовать это правило как неизвестное at-правило.

[Firefox прекратил его поддержку]: https://web.archive.org/web/20200528221656/https://www.fxsitecompat.dev/en-CA/docs/2018/moz-document-support-has-been-dropped-except-for-empty-url-prefix/

**Есть одно исключение**: функция с пустым url-префиксом по-прежнему разрешена, поскольку она используется в хаке, нацеленном на Firefox.

**SCSS**

```scss
@-moz-document url-prefix() {
  .error {
    color: red;
  }
}
```

**Sass**

```sass
@-moz-document url-prefix()
  .error
    color: red
```

**Результат CSS**

```css
@-moz-document url-prefix() {
  .error {
    color: red;
  }
}
```

## Переходный период

**Совместимость:** Dart Sass: начиная с 1.7.2 · LibSass: ✗ · Ruby Sass: ✗

Сначала мы будем выводить предупреждения об устаревании для всех случаев использования `@-moz-document`, кроме хака с пустым url-префиксом.

В Dart Sass 2.0 правило `@-moz-document` будет трактоваться как неизвестное at-правило.

## Могу ли я отключить предупреждения?

Sass предоставляет мощный набор опций для управления тем, какие предупреждения об устаревании вы видите и когда.

### Краткий и подробный режим

По умолчанию Sass работает в кратком режиме, в котором каждый тип предупреждения об устаревании выводится не более пяти раз, после чего дальнейшие предупреждения такого типа подавляются. Это помогает пользователям узнавать о предстоящем ломающем изменении, не заваливая консоль избыточным шумом.

Если же запустить Sass в подробном режиме, он выведет *каждое* встреченное предупреждение об устаревании. Это может быть полезно для отслеживания оставшейся работы по исправлению устареваний. Включить подробный режим можно с помощью

[флага `--verbose`] в командной строке, либо

[опции `verbose`] в JavaScript API.

[флага `--verbose`]: https://sass-lang.com/documentation/cli/dart-sass/#verbose
[опции `verbose`]: https://sass-lang.com/documentation/js-api/interfaces/Options/#verbose

> ⚠️ **Внимание**
>
> При запуске через [JS API] Sass не делится никакой информацией между компиляциями, поэтому по умолчанию он выводит по пять предупреждений для *каждой* компилируемой таблицы стилей. Однако это можно исправить, написав (или попросив автора вашего любимого фреймворкового Sass-плагина написать) [пользовательский `Logger`], который выводит только пять ошибок на каждое устаревание и может использоваться совместно в нескольких компиляциях.
>
> [JS API]: https://sass-lang.com/documentation/js-api/
> [пользовательский `Logger`]: https://sass-lang.com/documentation/js-api/interfaces/Logger-1/

### Отключение предупреждений для зависимостей

Иногда предупреждения об устаревании возникают в ваших зависимостях, и вы никак не можете на это повлиять. Вы можете отключить предупреждения об устаревании из зависимостей, при этом по-прежнему выводя их для вашего приложения, с помощью

[флага `--quiet-deps`] в командной строке, либо

[опции `quietDeps`] в JavaScript API.

[флага `--quiet-deps`]: https://sass-lang.com/documentation/cli/dart-sass/#quiet-deps
[опции `quietDeps`]: https://sass-lang.com/documentation/js-api/interfaces/Options/#quietDeps

Для целей этого флага «зависимостью» считается любая таблица стилей, загрузка которой не является просто цепочкой относительных загрузок из точки входа. Это означает всё, что загружается через путь загрузки, и большинство таблиц стилей, загружаемых через пользовательские импортеры.

### Отключение конкретных устареваний

Если вы знаете, что конкретное устаревание не создаёт для вас проблем, вы можете отключить предупреждения именно для него с помощью

[флага `--silence-deprecation`] в командной строке, либо

[опции `silenceDeprecations`] в JavaScript API.

[флага `--silence-deprecation`]: https://sass-lang.com/documentation/cli/dart-sass/#silence-deprecation
[опции `silenceDeprecations`]: https://sass-lang.com/documentation/js-api/interfaces/Options/#silenceDeprecations

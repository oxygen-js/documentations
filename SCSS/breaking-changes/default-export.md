# Ломающее изменение: экспорт по умолчанию

По умолчанию Node.js позволяет загружать [модули CommonJS] из модулей
ECMAScript с помощью синтаксиса `import sass from 'sass'`. Теперь это
объявлено устаревшим; пользователям ESM следует вместо этого использовать
`import * as sass from 'sass'`.

Исторически Dart Sass был доступен только в виде модуля CommonJS. Это означало,
что любой, кто использовал его в проекте с встроенной поддержкой модулей
ECMAScript в Node.js, мог загружать его так, будто он предоставляет [экспорт по
умолчанию]:

[экспорт по умолчанию]: https://developer.mozilla.org/en-US/docs/web/javascript/reference/statements/export#using_the_default_export

```js
import sass from 'sass'; // Don't do this anymore
```

Команда Sass никогда не закладывала такое поведение, и оно не соответствовало
объявлениям типов, поставляемым вместе с пакетом, но при этом оно
_действительно_ работало. Мы решили убрать эту возможность в Dart Sass 2.0.0 и
требовать, чтобы пользователи модулей ECMAScript использовали только именованные
экспорты пакета:

```js
import * as sass from 'sass'; // Do this
```

## Переходный период

**Совместимость:** Dart Sass: начиная с 1.54.0 · LibSass: ✗ · Ruby Sass: ✗


До выхода Dart Sass 2.0.0 мы продолжим поддерживать загрузку пользователями
экспорта Sass по умолчанию. При первом обращении к любым свойствам экспорта по
умолчанию будет выдано предупреждение об устаревании в `console.error()`. Чтобы
избежать этой ошибки, используйте вместо этого `import * as sass from 'sass'`.

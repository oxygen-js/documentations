# Числа

Числа в Sass состоят из двух частей: собственно числа и единицы измерения.
Например, в `16px` число — это `16`, а единица измерения — `px`. Числа могут быть
без единиц измерения, а могут иметь составные единицы измерения. Подробнее см.
раздел [Единицы измерения](#units) ниже.

**SCSS**

```scss
@debug 100; // 100
@debug 0.8; // 0.8
@debug 16px; // 16px
@debug 5px * 2px; // 10px*px (read "square pixels")
```

**Sass**

```sass
@debug 100  // 100
@debug 0.8  // 0.8
@debug 16px  // 16px
@debug 5px * 2px  // 10px*px (read "square pixels")
```

Числа в Sass поддерживают те же форматы, что и числа CSS, включая [научную
нотацию][] — запись с буквой `e` между числом и степенью числа 10. Поскольку
поддержка научной нотации в браузерах исторически была нестабильной, Sass всегда
компилирует такие числа в полностью развёрнутом виде.

[научную нотацию]: https://en.wikipedia.org/wiki/Scientific_notation

**SCSS**

```scss
@debug 5.2e3; // 5200
@debug 6e-2; // 0.06
```

**Sass**

```sass
@debug 5.2e3  // 5200
@debug 6e-2  // 0.06
```

> ⚠️ **Внимание**
>
> Sass не различает целые числа и десятичные дроби, поэтому, например,
> `math.div(5, 2)` возвращает `2.5`, а не `2`. Это то же поведение, что и в
> JavaScript, но отличается от многих других языков программирования.

## Единицы измерения

Sass обладает мощной поддержкой работы с единицами измерения, основанной на том,
как устроены [реальные вычисления с единицами измерения][]. При умножении двух
чисел их единицы измерения также перемножаются. При делении одного числа на другое
результат получает единицы измерения числителя от первого числа, а единицы
измерения знаменателя — от второго. Число может иметь любое количество единиц
измерения в числителе и/или знаменателе.

[реальные вычисления с единицами измерения]: https://en.wikipedia.org/wiki/Unit_of_measurement#Calculations_with_units_of_measurement

**SCSS**

```scss
@use 'sass:math';

@debug 4px * 6px; // calc(24px * 1px) (read "square pixels")
@debug math.div(5px, 2s); // calc(2.5px / 1s) (read "pixels per second")

// calc(3.125px * 1deg / 1s / 1em) (read "pixel-degrees per second-em")
@debug 5px * math.div(math.div(30deg, 2s), 24em);

$degrees-per-second: math.div(20deg, 1s);
@debug $degrees-per-second; // calc(20deg / 1s)
@debug math.div(1, $degrees-per-second); // calc(0.05s / 1deg)
```

**Sass**

```sass
@use 'sass:math'

@debug 4px * 6px  // calc(24px * 1px) (read "square pixels")
@debug math.div(5px, 2s)  // calc(2.5px / 1s) (read "pixels per second")

// calc(3.125px * 1deg / 1s / 1em) (read "pixel-degrees per second-em")
@debug 5px * math.div(math.div(30deg, 2s), 24em)

$degrees-per-second: math.div(20deg, 1s)
@debug $degrees-per-second  // calc(20deg / 1s)
@debug math.div(1, $degrees-per-second)  // calc(0.05s / 1deg)
```

> ⚠️ **Внимание**
>
> В CSS числа со составными единицами измерения могут быть представлены только в
> `calc()` и других математических выражениях, поэтому Sass выводит такие числа в
> виде выражений `calc()`. Обычные свойства CSS не принимают составные единицы
> измерения, но они могут быть полезны в [пользовательских свойствах][] или
> [функциях][].
>
> **Совместимость (составные единицы измерения в CSS):** Dart Sass: начиная с
> 1.96.0 · LibSass: ✗ · Ruby Sass: ✗
>
> CSS специфицировал поддержку составных единиц измерения, таких как квадратные
> пиксели, только в [Values and Units Level 4][], и браузеры начали реализовывать
> её лишь в 2025 году. Более ранние версии Sass выдавали ошибки при попытке
> использовать в CSS значение с составными единицами измерения, хотя их всё ещё
> можно было использовать в переменных и функциях Sass.
>
> Начиная с Dart Sass 1.69.6, [`meta.inspect()`][] возвращает корректные выражения
> `calc()` для чисел со составными единицами измерения.

[пользовательских свойствах]: https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Cascading_variables/Using_custom_properties
[функциях]: https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Custom_functions_and_mixins/Using_custom_functions
[Values and Units Level 4]: https://www.w3.org/TR/css-values-4/
[`meta.inspect()`]: https://sass-lang.com/documentation/modules/meta#inspect

Sass автоматически преобразует между совместимыми единицами измерения, хотя то,
какую единицу измерения он выберет для результата, зависит от того, какая
реализация Sass используется. Если попытаться объединить несовместимые единицы
измерения, например `1in + 1em`, Sass выдаст ошибку.

**SCSS**

```scss
// CSS defines one inch as 96 pixels.
@debug 1in + 6px; // 102px or 1.0625in

@debug 1in + 1s;
//     ^^^^^^^^
// Error: Incompatible units s and in.
```

**Sass**

```sass
// CSS defines one inch as 96 pixels.
@debug 1in + 6px  // 102px or 1.0625in

@debug 1in + 1s
//     ^^^^^^^^
// Error: Incompatible units s and in.
```

Как и в реальных вычислениях с единицами измерения, если числитель содержит
единицы измерения, совместимые с единицами измерения в знаменателе (например,
`math.div(96px, 1in)`), они сокращаются. Благодаря этому легко задать
коэффициент, который можно использовать для преобразования между единицами
измерения. В примере ниже мы задаём желаемую скорость как одну секунду на 50
пикселей, а затем умножаем её на количество пикселей, которое покрывает переход,
чтобы получить время, которое он должен занять.

**SCSS**

```scss
@use 'sass:math';

$transition-speed: math.div(1s, 50px);

@mixin move($left-start, $left-stop) {
  position: absolute;
  left: $left-start;
  transition: left ($left-stop - $left-start) * $transition-speed;

  &:hover {
    left: $left-stop;
  }
}

.slider {
  @include move(10px, 120px);
}
```

**Sass**

```sass
@use 'sass:math'

$transition-speed: math.div(1s, 50px)

@mixin move($left-start, $left-stop)
  position: absolute
  left: $left-start
  transition: left ($left-stop - $left-start) * $transition-speed

  &:hover
    left: $left-stop



.slider
  @include move(10px, 120px)
```

**CSS output**

```css
.slider {
  position: absolute;
  left: 10px;
  transition: left 2.2s;
}
.slider:hover {
  left: 120px;
}
```

> ⚠️ **Внимание**
>
> Если в результате арифметики вы получаете неверную единицу измерения, вероятно,
> стоит проверить вычисления. Возможно, вы забыли единицы измерения для величины,
> у которой они должны быть! Соблюдение чистоты единиц измерения позволяет Sass
> выдавать полезные ошибки, когда что-то не так.
>
> Особенно стоит избегать интерполяции вида `#{$number}px`. На самом деле это не
> создаёт число! Это создаёт [строку без кавычек][], которая *выглядит* как число,
> но не будет работать ни с какими [числовыми операциями][], ни с [функциями][].
> Постарайтесь сделать вычисления чистыми по единицам измерения, чтобы `$number`
> уже имело единицу измерения `px`, либо напишите `$number * 1px`.

[строку без кавычек]: https://sass-lang.com/documentation/values/strings#unquoted
[числовыми операциями]: https://sass-lang.com/documentation/operators/numeric
[функциями]: https://sass-lang.com/documentation/modules/math

> ⚠️ **Внимание**
>
> Проценты в Sass работают точно так же, как и любая другая единица измерения. Они
> *не* взаимозаменяемы с десятичными дробями, потому что в CSS десятичные дроби и
> проценты означают разные вещи. Например, `50%` — это число с единицей измерения
> `%`, и Sass считает его отличным от числа `0.5`.
>
> Вы можете преобразовывать десятичные дроби в проценты и обратно с помощью
> арифметики единиц измерения. `math.div($percentage, 100%)` вернёт
> соответствующую десятичную дробь, а `$decimal * 100%` вернёт соответствующий
> процент. Также можно использовать [функцию `math.percentage()`][] как более
> явный способ записи `$decimal * 100%`.

[функцию `math.percentage()`]: https://sass-lang.com/documentation/modules/math#percentage

## Точность

**Совместимость (10 цифр по умолчанию):** Dart Sass: ✓ · LibSass: ✗ · Ruby Sass: начиная с 3.5.0

LibSass и более старые версии Ruby Sass по умолчанию используют точность в 5
цифр, но её можно настроить на другое количество. Рекомендуется настраивать их на
10 цифр для большей точности и совместимости с будущими версиями.

Числа в Sass представлены внутренне как 64-битные значения с плавающей точкой.
Они поддерживают точность до 10 цифр после десятичной точки при сериализации в
CSS и для целей сравнения на равенство. Это означает несколько разных вещей:

* В сгенерированный CSS будут включены только первые десять цифр числа после
  десятичной точки.

* Операции вроде [`==`][] и [`>=`][] будут считать два числа эквивалентными, если
  они совпадают вплоть до десятой цифры после десятичной точки.

* Если число отличается от целого меньше чем на `0.0000000001`, оно считается
  целым для целей таких функций, как [`list.nth()`][], которые требуют
  целочисленные аргументы.

[`==`]: https://sass-lang.com/documentation/operators/equality
[`>=`]: https://sass-lang.com/documentation/operators/relational
[`list.nth()`]: https://sass-lang.com/documentation/modules/list#nth

**SCSS**

```scss
@debug 0.012345678912345; // 0.0123456789
@debug 0.01234567891 == 0.01234567899; // true
@debug 1.00000000009; // 1
@debug 0.99999999991; // 1
```

**Sass**

```sass
@debug 0.012345678912345  // 0.0123456789
@debug 0.01234567891 == 0.01234567899  // true
@debug 1.00000000009  // 1
@debug 0.99999999991  // 1
```

> 💡 **Занятный факт**
>
> Числа округляются до 10 цифр точности *лениво* (lazily) — только тогда, когда
> они используются в месте, где точность имеет значение. Это значит, что
> математические функции работают с полным значением числа внутри, чтобы избежать
> накопления лишних ошибок округления.

# Интерфейс командной строки (CLI)

## Dev-сервер

### `vite`

Запускает dev-сервер Vite в текущем каталоге. `vite dev` и `vite serve` — псевдонимы для `vite`.

#### Использование

```bash
vite [root]
```

#### Опции

| Опции                     |                                                                                                                                                                                                          |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--host [host]`           | Указать имя хоста (`string`)                                                                                                                                                                            |
| `--port <port>`           | Указать порт (`number`)                                                                                                                                                                                 |
| `--open [path]`           | Открывать браузер при запуске (`boolean \| string`)                                                                                                                                                     |
| `--cors`                  | Включить CORS (`boolean`)                                                                                                                                                                                |
| `--strictPort`            | Завершить работу, если указанный порт уже занят (`boolean`)                                                                                                                                             |
| `--force`                 | Заставить оптимизатор игнорировать кэш и выполнить повторную сборку (`boolean`)                                                                                                                         |
| `-c, --config <file>`     | Использовать указанный файл конфигурации (`string`)                                                                                                                                                     |
| `--base <path>`           | Публичный базовый путь (по умолчанию: `/`) (`string`)                                                                                                                                                   |
| `-l, --logLevel <level>`  | info \| warn \| error \| silent (`string`)                                                                                                                                                              |
| `--clearScreen`           | Разрешить/запретить очистку экрана при выводе логов (`boolean`)                                                                                                                                         |
| `--configLoader <loader>` | Использовать `bundle`, чтобы собрать конфигурацию с помощью Rolldown, либо `runner` (экспериментально), чтобы обрабатывать её на лету, либо `native` (экспериментально), чтобы загружать её через нативную среду выполнения (по умолчанию: `bundle`) |
| `--profile`               | Запустить встроенный инспектор Node.js (см. [Узкие места производительности](https://vite.dev/guide/troubleshooting#performance-bottlenecks))                                                            |
| `-d, --debug [feat]`      | Показывать отладочные логи (`string \| boolean`)                                                                                                                                                        |
| `-f, --filter <filter>`   | Фильтровать отладочные логи (`string`)                                                                                                                                                                  |
| `-m, --mode <mode>`       | Задать режим окружения (`string`)                                                                                                                                                                       |
| `-h, --help`              | Показать доступные опции CLI                                                                                                                                                                            |
| `-v, --version`           | Показать номер версии                                                                                                                                                                                   |

## Сборка

### `vite build`

Сборка для продакшна.

#### Использование

```bash
vite build [root]
```

#### Опции

| Опции                          |                                                                                                                                                                                                          |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--target <target>`            | Целевая версия для транспиляции (по умолчанию: `"baseline-widely-available"`) (`string`)                                                                                                                |
| `--outDir <dir>`               | Каталог для вывода (по умолчанию: `dist`) (`string`)                                                                                                                                                    |
| `--assetsDir <dir>`            | Каталог внутри outDir для размещения ресурсов (по умолчанию: `"assets"`) (`string`)                                                                                                                     |
| `--assetsInlineLimit <number>` | Порог встраивания статических ресурсов в base64 в байтах (по умолчанию: `4096`) (`number`)                                                                                                              |
| `--ssr [entry]`                | Собрать указанную точку входа для серверного рендеринга (`string`)                                                                                                                                      |
| `--sourcemap [output]`         | Выводить карты исходников для сборки (по умолчанию: `false`) (`boolean \| "inline" \| "hidden"`)                                                                                                        |
| `--minify [minifier]`          | Включить/отключить минификацию или указать используемый минификатор (по умолчанию: `"oxc"`) (`boolean \| "oxc" \| "terser" \| "esbuild"`)                                                               |
| `--manifest [name]`            | Создать json-манифест сборки (`boolean \| string`)                                                                                                                                                      |
| `--ssrManifest [name]`         | Создать json-манифест ssr (`boolean \| string`)                                                                                                                                                         |
| `--emptyOutDir`                | Принудительно очищать outDir, когда он находится за пределами корня (`boolean`)                                                                                                                         |
| `-w, --watch`                  | Пересобирать при изменении модулей на диске (`boolean`)                                                                                                                                                 |
| `-c, --config <file>`          | Использовать указанный файл конфигурации (`string`)                                                                                                                                                     |
| `--base <path>`                | Публичный базовый путь (по умолчанию: `/`) (`string`)                                                                                                                                                   |
| `-l, --logLevel <level>`       | Info \| warn \| error \| silent (`string`)                                                                                                                                                              |
| `--clearScreen`                | Разрешить/запретить очистку экрана при выводе логов (`boolean`)                                                                                                                                         |
| `--configLoader <loader>`      | Использовать `bundle`, чтобы собрать конфигурацию с помощью Rolldown, либо `runner` (экспериментально), чтобы обрабатывать её на лету, либо `native` (экспериментально), чтобы загружать её через нативную среду выполнения (по умолчанию: `bundle`) |
| `--profile`                    | Запустить встроенный инспектор Node.js (см. [Узкие места производительности](https://vite.dev/guide/troubleshooting#performance-bottlenecks))                                                            |
| `-d, --debug [feat]`           | Показывать отладочные логи (`string \| boolean`)                                                                                                                                                        |
| `-f, --filter <filter>`        | Фильтровать отладочные логи (`string`)                                                                                                                                                                  |
| `-m, --mode <mode>`            | Задать режим окружения (`string`)                                                                                                                                                                       |
| `-h, --help`                   | Показать доступные опции CLI                                                                                                                                                                            |
| `--app`                        | Собрать все окружения, эквивалентно `builder: {}` (`boolean`, экспериментально)                                                                                                                         |

Предварительная сборка зависимостей.

| `--clearScreen`           | Разрешить/запретить очистку экрана при выводе логов (`boolean`)                                                                                                                                         |
| `--configLoader <loader>` | Использовать `bundle`, чтобы собрать конфигурацию с помощью Rolldown, либо `runner` (экспериментально), чтобы обрабатывать её на лету, либо `native` (экспериментально), чтобы загружать её через нативную среду выполнения (по умолчанию: `bundle`) |

### `vite preview`

Локальный предпросмотр продакшн-сборки. Не используйте эту команду в качестве продакшн-сервера — она для этого не предназначена.

Эта команда запускает сервер в каталоге сборки (по умолчанию `dist`). Предварительно выполните `vite build`, чтобы убедиться, что каталог сборки актуален. В зависимости от заданного в проекте значения [`appType`](https://vite.dev/config/shared-options#apptype) она задействует определённые промежуточные обработчики (middleware).

#### Использование

```bash
vite preview [root]
```

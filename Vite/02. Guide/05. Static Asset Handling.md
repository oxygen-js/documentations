# Работа со статическими ресурсами

- Связанное: [Публичный базовый путь](https://vite.dev/guide/build#public-base-path)
- Связанное: [параметр конфигурации `assetsInclude`](https://vite.dev/config/shared-options#assetsinclude)

## Импорт ресурса как URL

Импорт статического ресурса вернёт разрешённый публичный URL, по которому он будет обслуживаться:

```js
import imgUrl from './img.png'
document.getElementById('hero-img').src = imgUrl
```

Например, `imgUrl` будет иметь значение `/src/img.png` во время разработки и превратится в `/assets/img.2d8efhg.png` в продакшн-сборке.

Поведение похоже на `file-loader` из webpack. Разница в том, что импорт может использовать как абсолютные публичные пути (относительно корня проекта во время разработки), так и относительные пути.

- Ссылки `url()` в CSS обрабатываются точно так же.

- При использовании плагина Vue ссылки на ресурсы в шаблонах Vue SFC автоматически преобразуются в импорты.

- Распространённые типы файлов изображений, медиа и шрифтов определяются как ресурсы автоматически. Вы можете расширить внутренний список с помощью [параметра `assetsInclude`](https://vite.dev/config/shared-options#assetsinclude).

- Ресурсы, на которые есть ссылки, включаются в граф ресурсов сборки, получают хешированные имена файлов и могут обрабатываться плагинами для оптимизации.

- Ресурсы, размер которых в байтах меньше значения [параметра `assetsInlineLimit`](https://vite.dev/config/build-options#build-assetsinlinelimit), будут встроены в виде base64 data URL.

- Placeholder-файлы Git LFS автоматически исключаются из встраивания, потому что они не содержат содержимого файла, который представляют. Чтобы получить встраивание, убедитесь, что вы загрузили содержимое файлов через Git LFS перед сборкой.

- TypeScript по умолчанию не распознаёт импорты статических ресурсов как валидные модули. Чтобы это исправить, подключите [`vite/client`](https://vite.dev/guide/features#client-types).

> **Совет: Встраивание SVG через `url()`**
>
> При передаче URL SVG в вручную сконструированный `url()` через JS переменную следует обернуть в двойные кавычки.
>
> ```js
> import imgUrl from './img.svg'
> document.getElementById('hero-img').style.background = `url("${imgUrl}")`
> ```

### Явный импорт URL

Ресурсы, которые не входят во внутренний список или в `assetsInclude`, можно явно импортировать как URL с помощью суффикса `?url`. Это полезно, например, для импорта [Houdini Paint Worklets](https://developer.mozilla.org/en-US/docs/Web/API/CSS/paintWorklet_static).

```js
import workletURL from 'extra-scalloped-border/worklet.js?url'
CSS.paintWorklet.addModule(workletURL)
```

### Явное управление встраиванием

Ресурсы можно явно импортировать с встраиванием или без него, используя суффикс `?inline` или `?no-inline` соответственно.

```js
import imgUrl1 from './img.svg?no-inline'
import imgUrl2 from './img.png?inline'
```

### Импорт ресурса как строки

Ресурсы можно импортировать как строки с помощью суффикса `?raw`.

```js
import shaderString from './shader.glsl?raw'
```

### Импорт скрипта как воркера

Скрипты можно импортировать как веб-воркеры с помощью суффикса `?worker` или `?sharedworker`.

```js
// Отдельный чанк в продакшн-сборке
import Worker from './shader.js?worker'
const worker = new Worker()
```

```js
// sharedworker
import SharedWorker from './shader.js?sharedworker'
const sharedWorker = new SharedWorker()
```

```js
// Встроен как base64-строка
import InlineWorker from './shader.js?worker&inline'
```

Подробности смотрите в [разделе о веб-воркерах](https://vite.dev/guide/features#web-workers).

## Директория `public`

Если у вас есть ресурсы, которые:

- Никогда не упоминаются в исходном коде (например, `robots.txt`)
- Должны сохранять точно такое же имя файла (без хеширования)
- ...или вы просто не хотите сначала импортировать ресурс только для того, чтобы получить его URL

Тогда вы можете разместить ресурс в специальной директории `public` в корне проекта. Ресурсы в этой директории будут обслуживаться по корневому пути `/` во время разработки и копироваться в корень директории dist как есть.

По умолчанию используется директория `<root>/public`, но её можно настроить через [параметр `publicDir`](https://vite.dev/config/shared-options#publicdir).

Обратите внимание, что на ресурсы из `public` всегда следует ссылаться с использованием корневого абсолютного пути — например, на `public/icon.png` в исходном коде нужно ссылаться как на `/icon.png`.

> **Совет: Выбор между импортами и директорией `public`**
>
> В общем случае предпочитайте **импорт ресурсов**, если только вам не нужны именно гарантии, предоставляемые директорией `public`.

## new URL(url, import.meta.url)

[import.meta.url](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/import.meta) — это встроенная возможность ESM, которая предоставляет URL текущего модуля. Комбинируя её с встроенным [конструктором URL](https://developer.mozilla.org/en-US/docs/Web/API/URL), мы можем получить полный, разрешённый URL статического ресурса, используя относительный путь из модуля JavaScript:

```js
const imgUrl = new URL('./img.png', import.meta.url).href

document.getElementById('hero-img').src = imgUrl
```

Это работает нативно в современных браузерах — фактически, Vite вообще не нужно обрабатывать этот код во время разработки!

Этот паттерн также поддерживает динамические URL через шаблонные строки:

```js
function getImageUrl(name) {
  // обратите внимание, что это не включает файлы в поддиректориях
  return new URL(`./dir/${name}.png`, import.meta.url).href
}
```

Во время продакшн-сборки Vite выполнит необходимые преобразования, чтобы URL по-прежнему указывали на правильное расположение даже после бандлинга и хеширования ресурсов. Однако строка URL должна быть статической, чтобы её можно было проанализировать, иначе код будет оставлен как есть, что может привести к ошибкам во время выполнения, если `build.target` не поддерживает `import.meta.url`.

```js
// Vite не преобразует это
const imgUrl = new URL(imagePath, import.meta.url).href
```

<details><summary>Как это работает</summary>

Vite преобразует функцию `getImageUrl` в:

```js
import __img0png from './dir/img0.png'
import __img1png from './dir/img1.png'

function getImageUrl(name) {
  const modules = {
    './dir/img0.png': __img0png,
    './dir/img1.png': __img1png,
  }
  return new URL(modules[`./dir/${name}.png`], import.meta.url).href
}
```

</details>

> **Внимание: Не работает при SSR**
>
> Этот паттерн не работает, если вы используете Vite для серверного рендеринга (SSR), потому что `import.meta.url` имеет разную семантику в браузерах и в Node.js. Кроме того, серверный бандл не может заранее определить URL хоста клиента.

# Развёртывание статического сайта

Приведённые ниже руководства основаны на нескольких общих допущениях:

- Вы используете расположение вывода сборки по умолчанию (`dist`). Это расположение [можно изменить с помощью `build.outDir`](https://vite.dev/config/build-options#build-outdir), и в этом случае вы можете адаптировать инструкции из этих руководств.
- Вы используете npm. Если вы используете Yarn или другие менеджеры пакетов, применяйте эквивалентные команды для запуска скриптов.
- Vite установлен как локальная dev-зависимость в вашем проекте, и вы настроили следующие npm-скрипты:

**`package.json`**

```json
{
  "scripts": {
    "build": "vite build",
    "preview": "vite preview"
  }
}
```

Важно отметить, что `vite preview` предназначен для локального предпросмотра сборки, а не в качестве продакшн-сервера.

> **Примечание**
>
> Эти руководства содержат инструкции по выполнению статического развёртывания вашего сайта на Vite. Vite также поддерживает серверный рендеринг. SSR относится к frontend-фреймворкам, которые поддерживают запуск одного и того же приложения в Node.js, его предварительный рендеринг в HTML и последующую гидратацию на клиенте. Ознакомьтесь с [руководством по SSR](https://vite.dev/guide/ssr), чтобы узнать об этой возможности. С другой стороны, если вам нужна интеграция с традиционными серверными фреймворками, обратитесь к [руководству по интеграции с бэкендом](https://vite.dev/guide/backend-integration).

## Сборка приложения

Чтобы собрать приложение, вы можете выполнить команду `npm run build`.

```bash
$ npm run build
```

По умолчанию результат сборки помещается в `dist`. Вы можете развернуть эту папку `dist` на любой предпочитаемой платформе.

### Локальное тестирование приложения

После сборки приложения вы можете протестировать его локально, выполнив команду `npm run preview`.

```bash
$ npm run preview
```

Команда `vite preview` запустит локальный статический веб-сервер, который обслуживает файлы из `dist` по адресу `http://localhost:4173`. Это простой способ проверить, нормально ли выглядит продакшн-сборка в вашем локальном окружении.

Вы можете настроить порт сервера, передав флаг `--port` в качестве аргумента.

**`package.json`**

```json
{
  "scripts": {
    "preview": "vite preview --port 8080"
  }
}
```

Теперь команда `preview` запустит сервер по адресу `http://localhost:8080`.

## GitHub Pages

1. **Обновите конфигурацию Vite**

   Задайте корректный `base` в `vite.config.js`.

   Если вы развёртываете на `https://<USERNAME>.github.io/` или на собственный домен через GitHub Pages (например, `www.example.com`), задайте `base` равным `'/'`. Также можно удалить `base` из конфигурации, так как по умолчанию он равен `'/'`.

   Если вы развёртываете на `https://<USERNAME>.github.io/<REPO>/` (например, ваш репозиторий находится по адресу `https://github.com/<USERNAME>/<REPO>`), задайте `base` равным `'/<REPO>/'`.

2. **Включите GitHub Pages**

   В своём репозитории перейдите в **Settings → Pages**. В разделе **Build and deployment** откройте выпадающий список **Source** и выберите **GitHub Actions**.

   Теперь GitHub будет развёртывать ваш сайт с помощью [workflow](https://docs.github.com/en/actions/concepts/workflows-and-actions/workflows) GitHub Actions — это необходимо, поскольку для развёртывания Vite требует этап сборки.

3. **Создайте workflow**

   Создайте в репозитории новый файл `.github/workflows/deploy.yml`. Также можно нажать **«create your own»** на предыдущем шаге — это сгенерирует для вас стартовый файл workflow.

   Ниже приведён пример workflow, который устанавливает зависимости через npm, собирает сайт и развёртывает его при каждом push изменений в ветку `main`:

   ```yaml [.github/workflows/deploy.yml]
   # Простой workflow для развёртывания статического контента на GitHub Pages
   name: Deploy static content to Pages

   on:
     # Запускается при push в ветку по умолчанию
     push:
       branches: ['main']

     # Позволяет запускать этот workflow вручную со вкладки Actions
     workflow_dispatch:

   # Задаёт права GITHUB_TOKEN, разрешающие развёртывание на GitHub Pages
   permissions:
     contents: read
     pages: write
     id-token: write

   # Разрешить одно параллельное развёртывание
   concurrency:
     group: 'pages'
     cancel-in-progress: true

   jobs:
     # Единственная задача развёртывания, так как мы только развёртываем
     deploy:
       environment:
         name: github-pages
         url: ${{ steps.deployment.outputs.page_url }}
       runs-on: ubuntu-latest
       steps:
         - name: Checkout
           uses: actions/checkout@9c091bb21b7c1c1d1991bb908d89e4e9dddfe3e0 # v7
         - name: Set up Node
           uses: actions/setup-node@48b55a011bda9f5d6aeb4c2d9c7362e8dae4041e # v6
           with:
             node-version: lts/*
             cache: 'npm'
         - name: Install dependencies
           run: npm ci
         - name: Build
           run: npm run build
         - name: Setup Pages
           uses: actions/configure-pages@45bfe0192ca1faeb007ade9deae92b16b8254a0d # v6
         - name: Upload artifact
           uses: actions/upload-pages-artifact@fc324d3547104276b827a68afc52ff2a11cc49c9 # v5
           with:
             # Загрузить папку dist
             path: './dist'
         - name: Deploy to GitHub Pages
           id: deployment
           uses: actions/deploy-pages@cd2ce8fcbc39b97be8ca5fce6e763baed58fa128 # v5
   ```

## GitLab Pages и GitLab CI

1. Задайте корректный `base` в `vite.config.js`.

   Если вы развёртываете на `https://<USERNAME or GROUP>.gitlab.io/`, `base` можно не указывать, так как по умолчанию он равен `'/'`.

   Если вы развёртываете на `https://<USERNAME or GROUP>.gitlab.io/<REPO>/` (например, ваш репозиторий находится по адресу `https://gitlab.com/<USERNAME>/<REPO>`), задайте `base` равным `'/<REPO>/'`.

2. Создайте в корне проекта файл с именем `.gitlab-ci.yml` со следующим содержимым. Он будет собирать и развёртывать ваш сайт при каждом изменении вашего контента:

   ```yaml [.gitlab-ci.yml]
   image: node:lts
   pages:
     stage: deploy
     cache:
       key:
         files:
           - package-lock.json
         prefix: npm
       paths:
         - node_modules/
     script:
       - npm install
       - npm run build
       - cp -a dist/. public/
     artifacts:
       paths:
         - public
     rules:
       - if: $CI_COMMIT_BRANCH == $CI_DEFAULT_BRANCH
   ```

## Netlify

### Netlify CLI

1. Установите [Netlify CLI](https://docs.netlify.com/api-and-cli-guides/cli-guides/get-started-with-cli/) с помощью `npm install -g netlify-cli`.
2. Создайте новый сайт с помощью `netlify init`.
3. Разверните с помощью `netlify deploy`.

Netlify CLI предоставит вам URL для предпросмотра. Когда будете готовы перейти в продакшн, используйте флаг `prod`: `netlify deploy --prod`.

### Netlify с Git

1. Отправьте (push) свой код в git-репозиторий (GitHub, GitLab, BitBucket, Azure DevOps).
2. [Импортируйте проект](https://app.netlify.com/start) в Netlify.
3. Выберите ветку, каталог вывода и при необходимости настройте переменные окружения.
4. Нажмите **Deploy**.
5. Ваше приложение на Vite развёрнуто!

После того как проект будет импортирован и развёрнут, все последующие push в ветки, отличные от продакшн-ветки, а также pull-запросы будут генерировать [превью-развёртывания (Preview Deployments)](https://docs.netlify.com/deploy/deploy-types/deploy-previews/), а все изменения в продакшн-ветке (обычно «main») будут приводить к [продакшн-развёртыванию (Production Deployment)](https://docs.netlify.com/deploy/deploy-overview/#definitions).

## Cloudflare

### Cloudflare Workers

[Плагин Cloudflare Vite](https://developers.cloudflare.com/workers/vite-plugin/) обеспечивает интеграцию с Cloudflare Workers и использует API окружений Vite для запуска вашего серверного кода в среде выполнения Cloudflare Workers во время разработки.

Чтобы добавить Cloudflare Workers в существующий проект на Vite, установите плагин и добавьте его в конфигурацию:

```bash
$ npm install --save-dev @cloudflare/vite-plugin
```

**`vite.config.js`**

```js
import { defineConfig } from 'vite'
import { cloudflare } from '@cloudflare/vite-plugin'

export default defineConfig({
  plugins: [cloudflare()],
})
```

**`wrangler.jsonc`**

```jsonc
{
  "name": "my-vite-app",
}
```

После выполнения `npm run build` ваше приложение можно развернуть с помощью `npx wrangler deploy`.

Вы также можете легко добавить в своё приложение на Vite бэкенд-API для безопасного взаимодействия с ресурсами Cloudflare. Оно работает в среде выполнения Workers во время разработки и развёртывается вместе с вашим фронтендом. Полное пошаговое руководство смотрите в [руководстве по плагину Cloudflare Vite](https://developers.cloudflare.com/workers/vite-plugin/tutorial/).

### Cloudflare Pages

#### Cloudflare Pages с Git

Cloudflare Pages даёт возможность развёртывать напрямую в Cloudflare без необходимости управлять файлом Wrangler.

1. Отправьте (push) свой код в git-репозиторий (GitHub, GitLab).
2. Войдите в панель управления Cloudflare и выберите свою учётную запись в **Account Home** > **Workers & Pages**.
3. Выберите **Create a new Project** и опцию **Pages**, затем выберите Git.
4. Выберите git-проект, который хотите развернуть, и нажмите **Begin setup**.
5. В настройках сборки выберите соответствующий пресет фреймворка в зависимости от выбранного вами Vite-фреймворка. В противном случае введите команды сборки для вашего проекта и ожидаемый каталог вывода.
6. Затем сохраните и разверните!
7. Ваше приложение развёрнуто! (например, `https://<PROJECTNAME>.pages.dev/`)

После того как проект будет импортирован и развёрнут, все последующие push в ветки будут генерировать [превью-развёртывания (Preview Deployments)](https://developers.cloudflare.com/pages/platform/preview-deployments/), если только это не отключено в [настройках сборки веток](https://developers.cloudflare.com/pages/platform/branch-build-controls/). Все изменения в продакшн-ветке (обычно «main») будут приводить к продакшн-развёртыванию.

Вы также можете добавлять собственные домены и настраивать параметры сборки в Pages. Узнайте больше об [интеграции Cloudflare Pages с Git](https://developers.cloudflare.com/pages/get-started/#manage-your-site).

## Google Firebase

1. Установите [firebase-tools](https://www.npmjs.com/package/firebase-tools) с помощью `npm i -g firebase-tools`.

2. Создайте в корне проекта следующие файлы:

   **firebase.json**

   ```json
   {
     "hosting": {
       "public": "dist",
       "ignore": [],
       "rewrites": [
         {
           "source": "**",
           "destination": "/index.html"
         }
       ]
     }
   }
   ```

   **.firebaserc**

   ```js
   {
     "projects": {
       "default": "<YOUR_FIREBASE_ID>"
     }
   }
   ```

3. После выполнения `npm run build` разверните с помощью команды `firebase deploy`.

## Azure Static Web Apps

Вы можете быстро развернуть своё приложение на Vite с помощью сервиса Microsoft Azure [Static Web Apps](https://aka.ms/staticwebapps). Вам понадобятся:

- Учётная запись Azure и ключ подписки. Вы можете создать [бесплатную учётную запись Azure здесь](https://azure.microsoft.com/free).
- Код вашего приложения, отправленный (push) в [GitHub](https://github.com).
- [Расширение SWA](https://marketplace.visualstudio.com/items?itemName=ms-azuretools.vscode-azurestaticwebapps) в [Visual Studio Code](https://code.visualstudio.com).

Установите расширение в VS Code и перейдите в корневой каталог вашего приложения. Откройте расширение Static Web Apps, войдите в Azure и нажмите значок «+», чтобы создать новое приложение Static Web App. Вам будет предложено указать, какой ключ подписки использовать.

Следуйте инструкциям мастера, запущенного расширением: задайте имя приложению, выберите пресет фреймворка и укажите корневой каталог приложения (обычно `/`) и расположение собранных файлов `/dist`. Мастер выполнится и создаст GitHub action в вашем репозитории в папке `.github`.

Этот action развернёт ваше приложение (следите за его прогрессом на вкладке Actions вашего репозитория), и после успешного завершения вы сможете открыть своё приложение по адресу, указанному в окне прогресса расширения, нажав кнопку «Browse Website», которая появляется после выполнения GitHub action.

## Render

Вы можете развернуть своё приложение на Vite как статический сайт (Static Site) на [Render](https://render.com/).

1. Создайте [учётную запись Render](https://dashboard.render.com/register).

2. В [панели управления (Dashboard)](https://dashboard.render.com/) нажмите кнопку **New** и выберите **Static Site**.

3. Подключите свою учётную запись GitHub/GitLab или используйте публичный репозиторий.

4. Укажите имя проекта и ветку.
   - **Build Command**: `npm install && npm run build`
   - **Publish Directory**: `dist`

5. Нажмите **Create Static Site**. Ваше приложение будет развёрнуто по адресу `https://<PROJECTNAME>.onrender.com/`.

По умолчанию любой новый коммит, отправленный (push) в указанную ветку, автоматически запускает новое развёртывание. [Автоматическое развёртывание (Auto-Deploy)](https://render.com/docs/deploys#toggling-auto-deploy-for-a-service) можно настроить в параметрах проекта.

Вы также можете добавить [собственный домен](https://render.com/docs/custom-domains) к своему проекту.

## Zephyr Cloud

[Zephyr Cloud](https://zephyr-cloud.io) — это платформа развёртывания, которая интегрируется непосредственно в ваш процесс сборки и обеспечивает глобальное edge-распространение для module federation и приложений других видов.

Zephyr использует иной подход, чем другие облачные провайдеры. Он интегрируется напрямую с процессом сборки Vite, поэтому каждый раз, когда вы собираете приложение или запускаете для него dev-сервер, оно будет автоматически развёртываться с помощью Zephyr Cloud.

Чтобы начать, выполните шаги из [руководства по развёртыванию Vite](https://docs.zephyr-cloud.io/bundlers/vite).

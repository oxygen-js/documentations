# Начало работы

## Обзор

Vite (французское слово, означающее «быстрый», произносится как `/viːt/`, «вит») — это инструмент сборки, цель которого — обеспечить более быструю и лёгкую разработку современных веб-проектов. Он состоит из двух основных частей:

- Dev-сервер, который предоставляет [богатые возможности](https://vite.dev/guide/features) поверх [нативных ES-модулей](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Modules), например чрезвычайно быструю [горячую замену модулей (HMR)](https://vite.dev/guide/features#hot-module-replacement).

- Команда сборки, которая собирает ваш код в бандл с помощью [Rolldown](https://rolldown.rs), предварительно настроенного на вывод высокооптимизированных статических ресурсов для продакшена.

Vite имеет чёткую позицию и поставляется с разумными настройками по умолчанию «из коробки». О том, что возможно, читайте в [руководстве по возможностям](https://vite.dev/guide/features). Поддержка фреймворков или интеграция с другими инструментами возможна через [плагины](https://vite.dev/guide/using-plugins). В [разделе о конфигурации](https://vite.dev/config/) объясняется, как при необходимости адаптировать Vite под ваш проект.

Vite также легко расширяем благодаря его [Plugin API](https://vite.dev/guide/api-plugin) и [JavaScript API](https://vite.dev/guide/api-javascript) с полной поддержкой типизации.

Больше об идеях, лежащих в основе проекта, вы можете узнать в разделе [Почему Vite](https://vite.dev/guide/why).

Изучайте Vite через интерактивные уроки на Scrimba

## Поддержка браузеров

Во время разработки Vite предполагает, что используется современный браузер. Это означает, что браузер поддерживает большинство новейших возможностей JavaScript и CSS. По этой причине Vite устанавливает [`esnext` в качестве цели преобразования](https://oxc.rs/docs/guide/usage/transformer/lowering.html#target). Это предотвращает понижение синтаксиса, позволяя Vite отдавать модули как можно ближе к исходному коду. Vite внедряет некоторый код среды выполнения (runtime), чтобы сервер разработки работал. Этот код использует возможности, входящие в статус Newly Available набора [Baseline](https://web-platform-dx.github.io/web-features/) на момент каждого мажорного релиза (2026-01-01 для данного мажорного релиза).

Для продакшн-сборок Vite по умолчанию нацеливается на браузеры со статусом Widely Available набора [Baseline](https://web-platform-dx.github.io/web-features/). Это браузеры, выпущенные не менее 2,5 лет назад. Цель можно понизить через конфигурацию. Кроме того, поддержку устаревших браузеров можно обеспечить с помощью официального плагина [@vitejs/plugin-legacy](https://github.com/vitejs/vite/tree/main/packages/plugin-legacy). Подробнее см. в разделе [Сборка для продакшена](https://vite.dev/guide/build).

## Создание вашего первого проекта Vite

**npm**

```bash
$ npm create vite@latest
```

**Yarn**

```bash
$ yarn create vite
```

**pnpm**

```bash
$ pnpm create vite
```

**Bun**

```bash
$ bun create vite
```

**Deno**

```bash
$ deno init --npm vite
```

Затем следуйте подсказкам!

Посмотрите интерактивный урок на Scrimba

> **Замечание о совместимости**
> Vite требует [Node.js](https://nodejs.org/en/) версии 20.19+, 22.12+. Однако некоторым шаблонам для работы требуется более высокая версия Node.js, пожалуйста, обновитесь, если ваш менеджер пакетов предупреждает об этом.

## Шаблоны сообщества

create-vite — это инструмент для быстрого старта проекта из базового шаблона для популярных фреймворков. Загляните в Awesome Vite за [шаблонами, поддерживаемыми сообществом](https://github.com/vitejs/awesome-vite#templates), которые включают другие инструменты или нацелены на другие фреймворки.

Для шаблона по адресу `https://github.com/user/project` вы можете опробовать его онлайн, используя `https://github.stackblitz.com/user/project` (добавив `.stackblitz` после `github` в URL проекта).

Вы также можете использовать инструмент вроде [tiged](https://github.com/tiged/tiged), чтобы создать свой проект на основе одного из шаблонов. Предполагая, что проект находится на GitHub и использует `main` в качестве ветки по умолчанию, вы можете создать локальную копию с помощью:

```bash
npx tiged user/project my-project
cd my-project

npm install
npm run dev
```

## Ручная установка

В вашем проекте вы можете установить CLI `vite` с помощью:

**npm**

```bash
$ npm install -D vite
```

**Yarn**

```bash
$ yarn add -D vite
```

**pnpm**

```bash
$ pnpm add -D vite
```

**Bun**

```bash
$ bun add -D vite
```

**Deno**

```bash
$ deno add -D npm:vite
```

И создать файл `index.html` вот так:

```html
<p>Hello Vite!</p>
```

Затем выполните соответствующую команду CLI в вашем терминале:

**npm**

```bash
$ npx vite
```

**Yarn**

```bash
$ yarn vite
```

**pnpm**

```bash
$ pnpm vite
```

**Bun**

```bash
$ bunx vite
```

**Deno**

```bash
$ deno run -A npm:vite
```

`index.html` будет доступен по адресу `http://localhost:5173`.

## `index.html` и корень проекта

Возможно, вы заметили, что в проекте Vite файл `index.html` находится на переднем плане и в центре, а не спрятан внутри `public`. Это сделано намеренно: во время разработки Vite — это сервер, а `index.html` — точка входа в ваше приложение.

Vite рассматривает `index.html` как исходный код и часть графа модулей. Он разрешает `<script type="module" src="...">`, который ссылается на ваш исходный код на JavaScript. Даже встроенные `<script type="module">` и CSS, на который ссылаются через `<link href>`, тоже получают специфичные для Vite возможности. Кроме того, URL внутри `index.html` автоматически перебазируются, поэтому нет необходимости в специальных плейсхолдерах вроде `%PUBLIC_URL%`.

Подобно статическим HTTP-серверам, у Vite есть понятие «корневого каталога», из которого отдаются ваши файлы. В остальной части документации вы будете встречать его под обозначением `<root>`. Абсолютные URL в вашем исходном коде будут разрешаться с использованием корня проекта в качестве базы, поэтому вы можете писать код так, как будто работаете с обычным статическим файловым сервером (только гораздо более мощным!). Vite также способен обрабатывать зависимости, которые разрешаются в расположения файловой системы за пределами корня, что делает его пригодным для использования даже в конфигурации на основе монорепозитория.

Vite также поддерживает [многостраничные приложения](https://vite.dev/guide/build#multi-page-app) с несколькими точками входа `.html`.

#### Указание альтернативного корня

Запуск `vite` стартует dev-сервер, используя текущий рабочий каталог в качестве корня. Вы можете указать альтернативный корень с помощью `vite serve some/sub/dir`.
Обратите внимание, что Vite также будет разрешать [свой файл конфигурации (то есть `vite.config.js`)](https://vite.dev/config/#configuring-vite) внутри корня проекта, поэтому вам придётся переместить его, если корень изменён.

## Интерфейс командной строки

В проекте, где установлен Vite, вы можете использовать бинарный файл `vite` в ваших npm-скриптах или запускать его напрямую с помощью `npx vite`. Вот npm-скрипты по умолчанию в созданном проекте Vite:

```json
{
  "scripts": {
    "dev": "vite", // запуск dev-сервера, псевдонимы: `vite dev`, `vite serve`
    "build": "vite build", // сборка для продакшена
    "preview": "vite preview" // локальный предпросмотр продакшн-сборки
  }
}
```

Вы можете указывать дополнительные опции CLI, такие как `--port` или `--open`. Полный список опций CLI можно получить, выполнив `npx vite --help` в вашем проекте.

Подробнее об [интерфейсе командной строки](https://vite.dev/guide/cli)

## Использование невыпущенных коммитов

Если вы не можете дождаться нового релиза, чтобы протестировать новейшие возможности, вы можете установить конкретный коммит Vite с помощью https://pkg.pr.new:

**npm**

```bash
$ npm install -D https://pkg.pr.new/vite@SHA
```

**Yarn**

```bash
$ yarn add -D https://pkg.pr.new/vite@SHA
```

**pnpm**

```bash
$ pnpm add -D https://pkg.pr.new/vite@SHA
```

**Bun**

```bash
$ bun add -D https://pkg.pr.new/vite@SHA
```

Замените `SHA` любым из [SHA коммитов Vite](https://github.com/vitejs/vite/commits/main/). Обратите внимание, что будут работать только коммиты за последний месяц, поскольку релизы из более старых коммитов удаляются.

В качестве альтернативы вы также можете склонировать [репозиторий vite](https://github.com/vitejs/vite) на свою локальную машину, а затем самостоятельно собрать и слинковать его (требуется [pnpm](https://pnpm.io/)):

```bash
git clone https://github.com/vitejs/vite.git
cd vite
pnpm install
cd packages/vite
pnpm run build
pnpm link # используйте предпочитаемый вами менеджер пакетов для этого шага
```

Затем перейдите в ваш проект на основе Vite и выполните `pnpm link vite` (или менеджер пакетов, который вы использовали для глобальной линковки `vite`). Теперь перезапустите сервер разработки, чтобы оказаться на переднем крае разработки!

Чтобы узнать больше о том, как и когда Vite выпускает релизы, ознакомьтесь с документацией [Релизы](https://vite.dev/guide/releases).

> **Зависимости, использующие Vite**
> Чтобы заменить версию Vite, используемую зависимостями транзитивно, вам следует использовать [npm overrides](https://docs.npmjs.com/cli/v11/configuring-npm/package-json#overrides) или [pnpm overrides](https://pnpm.io/settings#overrides).

# Связи «один к одному»

«Один к одному» — это связь, в которой A содержит только один экземпляр B, а B содержит только один экземпляр A.
Возьмём для примера сущности `User` и `Profile`.
У пользователя может быть только один профиль, и один профиль принадлежит только одному пользователю.

```typescript
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm"

@Entity()
export class Profile {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    gender: string

    @Column()
    photo: string
}
```

```typescript
import {
    Entity,
    PrimaryGeneratedColumn,
    Column,
    OneToOne,
    JoinColumn,
} from "typeorm"
import { Profile } from "./Profile"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @OneToOne(() => Profile)
    @JoinColumn()
    profile: Profile
}
```

Здесь мы добавили `@OneToOne` к `user` и указали, что целевым типом связи будет `Profile`.
Мы также добавили `@JoinColumn` — он обязателен и должен быть указан только на одной стороне связи.
Таблица той стороны, на которой вы поставили `@JoinColumn`, будет содержать «id связи» и внешние ключи на таблицу целевой сущности.

Этот пример создаст следующие таблицы:

```text
+-------------+--------------+----------------------------+
|                        profile                          |
+-------------+--------------+----------------------------+
| id          | int          | PRIMARY KEY AUTO_INCREMENT |
| gender      | varchar(255) |                            |
| photo       | varchar(255) |                            |
+-------------+--------------+----------------------------+

+-------------+--------------+----------------------------+
|                          user                           |
+-------------+--------------+----------------------------+
| id          | int          | PRIMARY KEY AUTO_INCREMENT |
| name        | varchar(255) |                            |
| profileId   | int          | FOREIGN KEY                |
+-------------+--------------+----------------------------+
```

Повторим: `@JoinColumn` должен стоять только на одной стороне связи — на той, в таблице которой должен быть внешний ключ.

Пример того, как сохранить такую связь:

```typescript
const profile = new Profile()
profile.gender = "male"
profile.photo = "me.jpg"
await dataSource.manager.save(profile)

const user = new User()
user.name = "Joe Smith"
user.profile = profile
await dataSource.manager.save(user)
```

Если включены [каскадные операции](<01. Relations.md#каскады>), эту связь можно сохранить одним вызовом `save`.

Чтобы загрузить пользователя вместе с профилем, нужно указать связь в `FindOptions`:

```typescript
const users = await dataSource.getRepository(User).find({
    relations: {
        profile: true,
    },
})
```

Либо соединить их с помощью `QueryBuilder`:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.profile", "profile")
    .getMany()
```

Если для связи включена жадная загрузка, указывать связи в команде поиска не нужно — они будут загружаться ВСЕГДА и автоматически. При использовании QueryBuilder жадные связи отключаются, и для загрузки связи придётся применить `leftJoinAndSelect`.

Связи бывают однонаправленными и двунаправленными.
Однонаправленные — это связи с декоратором связи только на одной стороне.
Двунаправленные — это связи с декораторами на обеих сторонах.

Мы только что создали однонаправленную связь. Сделаем её двунаправленной:

```typescript
import { Entity, PrimaryGeneratedColumn, Column, OneToOne } from "typeorm"
import { User } from "./User"

@Entity()
export class Profile {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    gender: string

    @Column()
    photo: string

    @OneToOne(() => User, (user) => user.profile) // указываем обратную сторону вторым параметром
    user: User
}
```

```typescript
import {
    Entity,
    PrimaryGeneratedColumn,
    Column,
    OneToOne,
    JoinColumn,
} from "typeorm"
import { Profile } from "./Profile"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @OneToOne(() => Profile, (profile) => profile.user) // указываем обратную сторону вторым параметром
    @JoinColumn()
    profile: Profile
}
```

Мы только что сделали связь двунаправленной. Обратите внимание: у обратной связи нет `@JoinColumn`.
`@JoinColumn` должен быть только на одной стороне связи — на той таблице, которая будет владеть внешним ключом.

Двунаправленные связи позволяют соединять сущности с обеих сторон с помощью `QueryBuilder`:

```typescript
const profiles = await dataSource
    .getRepository(Profile)
    .createQueryBuilder("profile")
    .leftJoinAndSelect("profile.user", "user")
    .getMany()
```

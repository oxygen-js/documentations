# Частые вопросы о связях

## Как создать связь сущности с самой собой?

Самоссылающиеся связи — это связи, указывающие на ту же самую сущность.
Они полезны, когда вы храните сущности в древовидных структурах.
На самоссылающихся связях также реализуется паттерн «список смежности» (adjacency list).
Например, вы хотите построить в приложении дерево категорий.
Категории могут содержать вложенные категории, вложенные категории — другие категории и т. д.
Здесь самоссылающиеся связи как раз кстати.
По сути это обычные связи, целью которых является сама сущность.
Пример:

```typescript
import {
    Entity,
    PrimaryGeneratedColumn,
    Column,
    ManyToOne,
    OneToMany,
} from "typeorm"

@Entity()
export class Category {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    title: string

    @Column()
    text: string

    @ManyToOne((type) => Category, (category) => category.childCategories)
    parentCategory: Category

    @OneToMany((type) => Category, (category) => category.parentCategory)
    childCategories: Category[]
}
```

## Как использовать id связи, не присоединяя саму связь?

Иногда вам нужно иметь в объекте id связанного объекта, не загружая его целиком.
Например:

```typescript
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm"

@Entity()
export class Profile {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    gender: string

    @Column()
    photo: string
}
```

```typescript
import {
    Entity,
    PrimaryGeneratedColumn,
    Column,
    OneToOne,
    JoinColumn,
} from "typeorm"
import { Profile } from "./Profile"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @OneToOne((type) => Profile)
    @JoinColumn()
    profile: Profile
}
```

Если вы загрузите пользователя без присоединения `profile`, в объекте пользователя не будет никакой информации о профиле,
даже его id:

```javascript
User {
  id: 1,
  name: "Umed"
}
```

Но иногда нужно знать «profile id» этого пользователя, не загружая профиль целиком.
Для этого достаточно добавить в сущность ещё одно свойство с `@Column`,
названное ровно так же, как колонка, созданная вашей связью. Пример:

```typescript
import {
    Entity,
    PrimaryGeneratedColumn,
    Column,
    OneToOne,
    JoinColumn,
} from "typeorm"
import { Profile } from "./Profile"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @Column({ nullable: true })
    profileId: number

    @OneToOne((type) => Profile)
    @JoinColumn()
    profile: Profile
}
```

Вот и всё. В следующий раз загруженный объект пользователя будет содержать id профиля:

```javascript
User {
  id: 1,
  name: "Umed",
  profileId: 1
}
```

## Как загружать связи в сущностях?

Проще всего загрузить связи сущности через опцию `relations` в `FindOptions`:

```typescript
const users = await dataSource.getRepository(User).find({
    relations: {
        profile: true,
        photos: true,
        videos: true,
    },
})
```

Альтернативный и более гибкий способ — использовать `QueryBuilder`:

```typescript
const user = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.profile", "profile")
    .leftJoinAndSelect("user.photos", "photo")
    .leftJoinAndSelect("user.videos", "video")
    .getMany()
```

С `QueryBuilder` вместо `leftJoinAndSelect` можно использовать `innerJoinAndSelect`
(о разнице между `LEFT JOIN` и `INNER JOIN` см. документацию по вашему SQL),
можно присоединять данные связи по условию, задавать сортировку и т. д.

Подробнее о [`QueryBuilder`](<../08. Query Builder/01. Select using Query Builder.md>).

## Избегайте инициализаторов у свойств-связей

Иногда бывает удобно инициализировать свойства-связи, например:

```typescript
import {
    Entity,
    PrimaryGeneratedColumn,
    Column,
    ManyToMany,
    JoinTable,
} from "typeorm"
import { Category } from "./Category"

@Entity()
export class Question {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    title: string

    @Column()
    text: string

    @ManyToMany((type) => Category, (category) => category.questions)
    @JoinTable()
    categories: Category[] = [] // обратите внимание на инициализацию = []
}
```

Однако в сущностях TypeORM это может вызвать проблемы.
Чтобы понять их суть, сначала попробуем загрузить сущность Question БЕЗ инициализатора.
При загрузке вопроса вернётся объект такого вида:

```javascript
Question {
    id: 1,
    title: "Question about ..."
}
```

Теперь, когда вы сохраните этот объект, поле `categories` внутри него не будет затронуто — ведь оно не задано.

Но если инициализатор есть, загруженный объект будет выглядеть так:

```javascript
Question {
    id: 1,
    title: "Question about ...",
    categories: []
}
```

При сохранении объекта TypeORM проверит, есть ли в базе данных категории, привязанные к этому вопросу, —
и отвяжет их все. Почему? Потому что связь, равная `[]`, как и любой набор элементов внутри неё, будет воспринята
так, будто из неё что-то удалили: другого способа определить, был ли объект удалён из сущности, нет.

Поэтому сохранение такого объекта принесёт вам проблемы — оно удалит все ранее заданные категории.

Как избежать такого поведения? Просто не инициализируйте массивы в своих сущностях.
То же правило касается и конструктора — в нём тоже не нужно ничего инициализировать.

## Как не создавать ограничение внешнего ключа

Иногда из соображений производительности вам может понадобиться связь между сущностями без ограничения внешнего ключа.
Указать, создавать ли ограничение внешнего ключа, можно опцией `createForeignKeyConstraints` (по умолчанию: true).

```typescript
import { Entity, PrimaryColumn, Column, ManyToOne } from "typeorm"
import { Person } from "./Person"

@Entity()
export class ActionLog {
    @PrimaryColumn()
    id: number

    @Column()
    date: Date

    @Column()
    action: string

    @ManyToOne((type) => Person, {
        createForeignKeyConstraints: false,
    })
    person: Person
}
```

## Как избежать ошибок циклического импорта

Вот пример на случай, если вы хотите объявить свои сущности так, чтобы они не вызывали ошибок в некоторых окружениях.
В этой ситуации Action.ts и Person.ts импортируют друг друга ради связи «многие ко многим».
Мы используем import type, чтобы задействовать информацию о типах без генерации какого-либо JavaScript-кода.

```typescript
import { Entity, PrimaryColumn, Column, ManytoMany } from "typeorm"
import type { Person } from "./Person"

@Entity()
export class ActionLog {
    @PrimaryColumn()
    id: number

    @Column()
    date: Date

    @Column()
    action: string

    @ManyToMany("Person", (person: Person) => person.id)
    person: Person
}
```

```typescript
import { Entity, PrimaryColumn, ManytoMany } from "typeorm"
import type { ActionLog } from "./Action"

@Entity()
export class Person {
    @PrimaryColumn()
    id: number

    @ManyToMany("ActionLog", (actionLog: ActionLog) => actionLog.id)
    log: ActionLog
}
```

# Связи «многие к одному» / «один ко многим»

«Многие к одному» / «один ко многим» — это связь, при которой A содержит несколько экземпляров B, а B содержит только один экземпляр A.
Возьмём для примера сущности `User` и `Photo`.
У пользователя может быть несколько фотографий, но каждая фотография принадлежит только одному пользователю.

```typescript
import { Entity, PrimaryGeneratedColumn, Column, ManyToOne } from "typeorm"
import { User } from "./User"

@Entity()
export class Photo {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    url: string

    @ManyToOne(() => User, (user) => user.photos)
    user: User
}
```

```typescript
import { Entity, PrimaryGeneratedColumn, Column, OneToMany } from "typeorm"
import { Photo } from "./Photo"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @OneToMany(() => Photo, (photo) => photo.user)
    photos: Photo[]
}
```

Здесь мы добавили `@OneToMany` к свойству `photos` и указали, что целевым типом связи является `Photo`.
В связи `@ManyToOne` / `@OneToMany` декоратор `@JoinColumn` можно опустить.
`@OneToMany` не может существовать без `@ManyToOne`.
Если вы хотите использовать `@OneToMany`, наличие `@ManyToOne` обязательно. Обратное же не требуется: если вам нужна только связь `@ManyToOne`, её можно описать, не добавляя `@OneToMany` в связанную сущность.
Там, где вы указываете `@ManyToOne`, у связанной сущности появятся «идентификатор связи» и внешний ключ.

Этот пример создаст следующие таблицы:

```text
+-------------+--------------+----------------------------+
|                         photo                           |
+-------------+--------------+----------------------------+
| id          | int          | PRIMARY KEY AUTO_INCREMENT |
| url         | varchar(255) |                            |
| userId      | int          | FOREIGN KEY                |
+-------------+--------------+----------------------------+

+-------------+--------------+----------------------------+
|                          user                           |
+-------------+--------------+----------------------------+
| id          | int          | PRIMARY KEY AUTO_INCREMENT |
| name        | varchar(255) |                            |
+-------------+--------------+----------------------------+
```

Пример того, как сохранить такую связь:

```typescript
const photo1 = new Photo()
photo1.url = "me.jpg"
await dataSource.manager.save(photo1)

const photo2 = new Photo()
photo2.url = "me-and-bears.jpg"
await dataSource.manager.save(photo2)

const user = new User()
user.name = "John"
user.photos = [photo1, photo2]
await dataSource.manager.save(user)
```

либо можно поступить так:

```typescript
const user = new User()
user.name = "Leo"
await dataSource.manager.save(user)

const photo1 = new Photo()
photo1.url = "me.jpg"
photo1.user = user
await dataSource.manager.save(photo1)

const photo2 = new Photo()
photo2.url = "me-and-bears.jpg"
photo2.user = user
await dataSource.manager.save(photo2)
```

С включёнными [каскадами](<01. Relations.md#каскады>) эту связь можно сохранить одним вызовом `save`.

Чтобы загрузить пользователя вместе с фотографиями, нужно указать связь в `FindOptions`:

```typescript
const userRepository = dataSource.getRepository(User)
const users = await userRepository.find({
    relations: {
        photos: true,
    },
})

// или с обратной стороны

const photoRepository = dataSource.getRepository(Photo)
const photos = await photoRepository.find({
    relations: {
        user: true,
    },
})
```

Либо их можно соединить с помощью `QueryBuilder`:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.photos", "photo")
    .getMany()

// или с обратной стороны

const photos = await dataSource
    .getRepository(Photo)
    .createQueryBuilder("photo")
    .leftJoinAndSelect("photo.user", "user")
    .getMany()
```

Если для связи включена жадная загрузка, указывать связи в команде find не нужно — она ВСЕГДА будет загружаться автоматически.
При использовании QueryBuilder жадные связи отключаются, и для загрузки связи необходимо применять `leftJoinAndSelect`.

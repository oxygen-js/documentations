# Жадные и ленивые связи

## Жадные связи

Жадные связи загружаются автоматически каждый раз, когда вы загружаете сущности из базы данных.
Например:

```typescript
import { Entity, PrimaryGeneratedColumn, Column, ManyToMany } from "typeorm"
import { Question } from "./Question"

@Entity()
export class Category {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @ManyToMany((type) => Question, (question) => question.categories)
    questions: Question[]
}
```

```typescript
import {
    Entity,
    PrimaryGeneratedColumn,
    Column,
    ManyToMany,
    JoinTable,
} from "typeorm"
import { Category } from "./Category"

@Entity()
export class Question {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    title: string

    @Column()
    text: string

    @ManyToMany((type) => Category, (category) => category.questions, {
        eager: true,
    })
    @JoinTable()
    categories: Category[]
}
```

Теперь при загрузке вопросов вам не нужно делать join или указывать связи, которые вы хотите загрузить.
Они загрузятся автоматически:

```typescript
const questionRepository = dataSource.getRepository(Question)

// вопросы будут загружены вместе со своими категориями
const questions = await questionRepository.find()
```

Жадные связи работают только при использовании методов `find*`.
При использовании `QueryBuilder` жадные связи отключаются, и для загрузки связи придётся применять `leftJoinAndSelect`.
Жадные связи можно задавать только на одной стороне связи —
указывать `eager: true` с обеих сторон запрещено.

По умолчанию жадные связи загружаются через `LEFT JOIN`. Если связь дополнительно помечена как `nullable: false` (и владеет join-колонкой, то есть это `ManyToOne` или владеющая `OneToOne`), TypeORM использует `INNER JOIN`, что может дать более эффективные планы запросов.

### Стратегия загрузки связей

По умолчанию жадные связи загружаются добавлением SQL-джойнов к основному запросу (стратегия `"join"`). Если вложенные джойны тянут слишком много данных, можно переключиться на стратегию `"query"`, при которой связи загружаются отдельными запросами к базе данных:

```typescript
// для отдельного запроса
const questions = await questionRepository.find({
    relationLoadStrategy: "query",
})

// или как значение по умолчанию для всего DataSource
const dataSource = new DataSource({
    // ...
    relationLoadStrategy: "query",
})
```

С помощью `loadEagerRelations` можно также управлять тем, загружаются ли жадные связи вообще:

```typescript
// полностью отключить загрузку жадных связей
const questions = await questionRepository.find({
    loadEagerRelations: false,
})

// загрузить только явно указанные связи, подавив вложенные жадные связи
const questions = await questionRepository.find({
    relations: { categories: true },
    loadEagerRelations: false,
})
```

## Ленивые связи

Сущности в ленивых связях загружаются в момент обращения к ним.
Такие связи должны иметь тип `Promise` — вы храните значение в промисе,
и при загрузке тоже получаете промис. Пример:

```typescript
import { Entity, PrimaryGeneratedColumn, Column, ManyToMany } from "typeorm"
import { Question } from "./Question"

@Entity()
export class Category {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @ManyToMany((type) => Question, (question) => question.categories)
    questions: Promise<Question[]>
}
```

```typescript
import {
    Entity,
    PrimaryGeneratedColumn,
    Column,
    ManyToMany,
    JoinTable,
} from "typeorm"
import { Category } from "./Category"

@Entity()
export class Question {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    title: string

    @Column()
    text: string

    @ManyToMany((type) => Category, (category) => category.questions)
    @JoinTable()
    categories: Promise<Category[]>
}
```

`categories` — это Promise. Значит, связь ленивая и может хранить только промис со значением внутри.
Пример сохранения такой связи:

```typescript
const category1 = new Category()
category1.name = "animals"
await dataSource.manager.save(category1)

const category2 = new Category()
category2.name = "zoo"
await dataSource.manager.save(category2)

const question = new Question()
question.categories = Promise.resolve([category1, category2])
await dataSource.manager.save(question)
```

Пример загрузки объектов внутри ленивых связей:

```typescript
const [question] = await dataSource.getRepository(Question).find()
const categories = await question.categories
// теперь в переменной "categories" будут все категории вопроса
```

Примечание: если вы пришли из других языков (Java, PHP и т. д.) и привыкли использовать ленивые связи повсюду — будьте осторожны.
Эти языки не асинхронные, и ленивая загрузка достигается в них иначе, без промисов.
В JavaScript и Node.JS для ленивой загрузки связей приходится использовать промисы.
Это нестандартный приём, и в TypeORM он считается экспериментальным.

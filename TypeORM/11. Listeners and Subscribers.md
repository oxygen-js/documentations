# Слушатели и подписчики

## Что такое слушатель сущности?

У любой из ваших сущностей могут быть методы с собственной логикой, которые слушают определённые события сущности.
Такие методы нужно помечать специальными декораторами — в зависимости от того, какое событие вы хотите слушать.

**Примечание:** не делайте обращений к базе данных внутри слушателя, используйте для этого [подписчиков](#что-такое-подписчик).

### `@AfterLoad`

Вы можете объявить в сущности метод с любым именем и пометить его `@AfterLoad` —
тогда TypeORM будет вызывать его каждый раз, когда сущность
загружается через `QueryBuilder` или методы поиска репозитория/менеджера.
Пример:

```typescript
@Entity()
export class Post {
    @AfterLoad()
    updateCounters() {
        if (this.likesCount === undefined) this.likesCount = 0
    }
}
```

### `@BeforeInsert`

Вы можете объявить в сущности метод с любым именем и пометить его `@BeforeInsert` —
тогда TypeORM вызовет его перед вставкой сущности через `save` репозитория/менеджера.
Пример:

```typescript
@Entity()
export class Post {
    @BeforeInsert()
    updateDates() {
        this.createdDate = new Date()
    }
}
```

### `@AfterInsert`

Вы можете объявить в сущности метод с любым именем и пометить его `@AfterInsert` —
тогда TypeORM вызовет его после вставки сущности через `save` репозитория/менеджера.
Пример:

```typescript
@Entity()
export class Post {
    @AfterInsert()
    resetCounters() {
        this.counters = 0
    }
}
```

### `@BeforeUpdate`

Вы можете объявить в сущности метод с любым именем и пометить его `@BeforeUpdate` —
тогда TypeORM вызовет его перед обновлением существующей сущности через `save` репозитория/менеджера. Учтите, однако, что это произойдёт только при изменении данных в модели. Если вы вызовете `save`, ничего в модели не поменяв, `@BeforeUpdate` и `@AfterUpdate` не сработают.
Пример:

```typescript
@Entity()
export class Post {
    @BeforeUpdate()
    updateDates() {
        this.updatedDate = new Date()
    }
}
```

### `@AfterUpdate`

Вы можете объявить в сущности метод с любым именем и пометить его `@AfterUpdate` —
тогда TypeORM вызовет его после обновления существующей сущности через `save` репозитория/менеджера.
Пример:

```typescript
@Entity()
export class Post {
    @AfterUpdate()
    updateCounters() {
        this.counter = 0
    }
}
```

### `@BeforeRemove`

Вы можете объявить в сущности метод с любым именем и пометить его `@BeforeRemove` —
тогда TypeORM вызовет его перед удалением сущности через `remove` репозитория/менеджера.
Пример:

```typescript
@Entity()
export class Post {
    @BeforeRemove()
    updateStatus() {
        this.status = "removed"
    }
}
```

### `@AfterRemove`

Вы можете объявить в сущности метод с любым именем и пометить его `@AfterRemove` —
тогда TypeORM вызовет его после удаления сущности через `remove` репозитория/менеджера.
Пример:

```typescript
@Entity()
export class Post {
    @AfterRemove()
    updateStatus() {
        this.status = "removed"
    }

```

### `@BeforeSoftRemove`

Вы можете объявить в сущности метод с любым именем и пометить его `@BeforeSoftRemove` —
тогда TypeORM вызовет его перед мягким удалением сущности через `softRemove` репозитория/менеджера.
Пример:

```typescript

```

### `@AfterSoftRemove`

Вы можете объявить в сущности метод с любым именем и пометить его `@AfterSoftRemove` —
тогда TypeORM вызовет его после мягкого удаления сущности через `softRemove` репозитория/менеджера.
Пример:

Вы можете объявить в сущности метод с любым именем и пометить его `@BeforeRecover` —

## Что такое подписчик?

Помечает класс как подписчик на события, который может слушать события конкретной сущности или события любых сущностей.
События возбуждаются при использовании `QueryBuilder` и методов репозитория/менеджера.
Пример:

```typescript
@EventSubscriber()
export class PostSubscriber implements EntitySubscriberInterface<Post> {
    /**
     * Указывает, что этот подписчик слушает только события Post.
     */
    listenTo() {
        return Post
    }

    /**
     * Вызывается перед вставкой post.
     */
    beforeInsert(event: InsertEvent<Post>) {
        console.log(`BEFORE POST INSERTED: `, event.entity)
    }
}
```

Вы можете реализовать любой метод из `EntitySubscriberInterface`.
Чтобы слушать любые сущности, просто опустите метод `listenTo` и используйте `any`:

```typescript
@EventSubscriber()
export class PostSubscriber implements EntitySubscriberInterface {
    /**
     * Вызывается после загрузки сущности.
     */
    afterLoad(entity: any) {
        console.log(`AFTER ENTITY LOADED: `, entity)
    }

    /**
     * Вызывается перед выполнением запроса.
     */
    beforeQuery(event: BeforeQueryEvent) {
        console.log(`BEFORE QUERY: `, event.query)
    }

    /**
     * Вызывается после выполнения запроса.
     */
    afterQuery(event: AfterQueryEvent) {
        console.log(`AFTER QUERY: `, event.query)
    }

    /**
     * Вызывается перед вставкой сущности.
     */
    beforeInsert(event: InsertEvent<any>) {
        console.log(`BEFORE ENTITY INSERTED: `, event.entity)
    }

    /**
     * Вызывается после вставки сущности.
     */
    afterInsert(event: InsertEvent<any>) {
        console.log(`AFTER ENTITY INSERTED: `, event.entity)
    }

    /**
     * Вызывается перед обновлением сущности.
     */
    beforeUpdate(event: UpdateEvent<any>) {
        console.log(`BEFORE ENTITY UPDATED: `, event.entity)
    }

    /**
     * Вызывается после обновления сущности.
     */
    afterUpdate(event: UpdateEvent<any>) {
        console.log(`AFTER ENTITY UPDATED: `, event.entity)
    }

    /**
     * Вызывается перед удалением сущности.
     */
    beforeRemove(event: RemoveEvent<any>) {
        console.log(
            `BEFORE ENTITY WITH ID ${event.entityId} REMOVED: `,
            event.entity,
        )
    }

    /**
     * Вызывается после удаления сущности.
     */
    afterRemove(event: RemoveEvent<any>) {
        console.log(
            `AFTER ENTITY WITH ID ${event.entityId} REMOVED: `,
            event.entity,
        )
    }

    /**
     * Вызывается перед удалением сущности.
     */
    beforeSoftRemove(event: SoftRemoveEvent<any>) {
        console.log(
            `BEFORE ENTITY WITH ID ${event.entityId} SOFT REMOVED: `,
            event.entity,
        )
    }

    /**
     * Вызывается после удаления сущности.
     */
    afterSoftRemove(event: SoftRemoveEvent<any>) {
        console.log(
            `AFTER ENTITY WITH ID ${event.entityId} SOFT REMOVED: `,
            event.entity,
        )
    }

    /**
     * Вызывается перед восстановлением сущности.
     */
    beforeRecover(event: RecoverEvent<any>) {
        console.log(
            `BEFORE ENTITY WITH ID ${event.entityId} RECOVERED: `,
            event.entity,
        )
    }

    /**
     * Вызывается после восстановления сущности.
     */
    afterRecover(event: RecoverEvent<any>) {
        console.log(
            `AFTER ENTITY WITH ID ${event.entityId} RECOVERED: `,
            event.entity,
        )
    }

    /**
     * Вызывается перед началом транзакции.
     */
    beforeTransactionStart(event: TransactionStartEvent) {
        console.log(`BEFORE TRANSACTION STARTED: `, event)
    }

    /**
     * Вызывается после начала транзакции.
     */
    afterTransactionStart(event: TransactionStartEvent) {
        console.log(`AFTER TRANSACTION STARTED: `, event)
    }

    /**
     * Вызывается перед фиксацией транзакции.
     */
    beforeTransactionCommit(event: TransactionCommitEvent) {
        console.log(`BEFORE TRANSACTION COMMITTED: `, event)
    }

    /**
     * Вызывается после фиксации транзакции.
     */
    afterTransactionCommit(event: TransactionCommitEvent) {
        console.log(`AFTER TRANSACTION COMMITTED: `, event)
    }

    /**
     * Вызывается перед откатом транзакции.
     */
    beforeTransactionRollback(event: TransactionRollbackEvent) {
        console.log(`BEFORE TRANSACTION ROLLBACK: `, event)
    }

    /**
     * Вызывается после отката транзакции.
     */
    afterTransactionRollback(event: TransactionRollbackEvent) {
        console.log(`AFTER TRANSACTION ROLLBACK: `, event)
    }
}
```

Убедитесь, что в ваших [DataSourceOptions](<02. Data Source/02. Data Source Options.md#общие-опции-источника-данных>) задано свойство `subscribers`, иначе TypeORM не загрузит ваш подписчик.

### `Объект события`

За исключением `listenTo`, во все методы `EntitySubscriberInterface` передаётся объект события со следующими базовыми свойствами:

- `dataSource: DataSource` — DataSource, использованный в событии.
- `queryRunner: QueryRunner` — QueryRunner, использованный в транзакции события.
- `manager: EntityManager` — EntityManager, использованный в транзакции события.

Дополнительные свойства смотрите в [интерфейсе каждого события](https://github.com/typeorm/typeorm/tree/master/src/subscriber/event).

Учтите, что `event.entity` не обязательно содержит первичный ключ (или ключи), когда используется `Repository.update()`. Доступны будут только значения, переданные в частичной сущности. Чтобы первичные ключи были доступны в подписчиках, можно явно передать их значения в объектном литерале частичной сущности либо использовать `Repository.save()`, который выполняет повторную выборку.

```typescript
await postRepository.update(post.id, { description: "Bacon ipsum dolor amet cow" })

// post.subscriber.ts
afterUpdate(event: UpdateEvent<Post>) {
  console.log(event.entity) // выводит { description: 'Bacon ipsum dolor amet cow' }
}
```

**Примечание:** все операции с базой данных в слушателях подписанных событий должны выполняться через экземпляр `queryRunner` или `manager` из объекта события.

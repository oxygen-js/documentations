# Примечания к выпуску 1.0

TypeORM 1.0 — крупный выпуск, в котором удалены давно устаревшие API, обновлены требования к платформе и собраны десятки исправлений и новых возможностей, накопившихся за цикл 0.3.x.

## Ломающие изменения

> Подробные инструкции по обновлению см. в [руководстве по обновлению](<02. Upgrading from 0.3.md>).

### Требования к платформе

- **Требуется Node.js 20+** — поддержка Node.js 16 и 18 прекращена, минимальная цель компиляции JavaScript теперь ES2023 ([#11382](https://github.com/typeorm/typeorm/pull/11382), автор [@alumni](https://github.com/alumni))
- **Полифил `Buffer` удалён** — для двоичных данных на платформах вне Node теперь используется `Uint8Array`; `Buffer` из Node.js (который наследует `Uint8Array`) продолжает работать как раньше ([#11935](https://github.com/typeorm/typeorm/pull/11935), автор [@pujux](https://github.com/pujux))
- **Заменена библиотека glob** — `glob` заменён на `tinyglobby`, а `rimraf` удалён, что уменьшило число зависимостей ([#11699](https://github.com/typeorm/typeorm/pull/11699), автор [@alumni](https://github.com/alumni))
- **Хэширование переведено на нативный `crypto`** — пакеты `sha.js` и `uuid` заменены на нативный модуль `crypto` и `crypto.randomUUID()` ([#11864](https://github.com/typeorm/typeorm/pull/11864), автор [@G0maa](https://github.com/G0maa), [#11769](https://github.com/typeorm/typeorm/pull/11769), автор [@mag123c](https://github.com/mag123c))

### Изменения в драйверах

- **MySQL / MariaDB: пакет `mysql` больше не поддерживается** — поддерживается только `mysql2`; опция `connectorPackage` удалена ([#11766](https://github.com/typeorm/typeorm/pull/11766), автор [@pkuczynski](https://github.com/pkuczynski))
- **MySQL: `legacySpatialSupport` теперь по умолчанию `false`** — по умолчанию используются стандартные функции `ST_GeomFromText`/`ST_AsText` ([#12083](https://github.com/typeorm/typeorm/pull/12083), автор [@pkuczynski](https://github.com/pkuczynski))
- **MySQL: опции колонок `width` и `zerofill` удалены** — они были объявлены устаревшими в MySQL 8.0.17 и удалены в MySQL 8.4 ([#12084](https://github.com/typeorm/typeorm/pull/12084), автор [@pkuczynski](https://github.com/pkuczynski))
- **MySQL: `QueryBuilder.useIndex()` принимает имена индексов, а не сырой SQL** — `useIndex()` теперь принимает либо одно имя индекса, либо массив имён и экранирует их как идентификаторы. Ранее аргумент подставлялся как сырой SQL-фрагмент с разделителями-запятыми, что допускало инъекцию идентификаторов ([#12344](https://github.com/typeorm/typeorm/pull/12344), автор [@eddieran](https://github.com/eddieran))
- **SQLite: `sqlite3` больше не поддерживается, по умолчанию используется `better-sqlite3`** — пакет `sqlite3` не поддерживается; опции `flags` и `busyTimeout` удалены ([#11836](https://github.com/typeorm/typeorm/pull/11836), автор [@pkuczynski](https://github.com/pkuczynski))
- **MongoDB: требуется драйвер v7+** — поддержка драйверов MongoDB для Node.js версий v5/v6 прекращена; метод `stats()` удалён; устаревшие опции подключения удалены; внутренние типы больше не экспортируются ([#12208](https://github.com/typeorm/typeorm/pull/12208), автор [@naorpeled](https://github.com/naorpeled), [#12179](https://github.com/typeorm/typeorm/pull/12179), автор [@pkuczynski](https://github.com/pkuczynski), [#12120](https://github.com/typeorm/typeorm/pull/12120), автор [@pkuczynski](https://github.com/pkuczynski), [#12037](https://github.com/typeorm/typeorm/pull/12037), автор [@alumni](https://github.com/alumni))
- **MS SQL Server: опция подключения `domain` удалена** — используйте `authentication` с типом NTLM ([#12135](https://github.com/typeorm/typeorm/pull/12135), автор [@pkuczynski](https://github.com/pkuczynski))
- **MS SQL Server: `options.isolation` переименована в `options.isolationLevel`** — формат значения изменён с `READ_COMMITTED` на `READ COMMITTED` для соответствия типу `IsolationLevel`; добавлен уровень изоляции `SNAPSHOT` ([#12231](https://github.com/typeorm/typeorm/pull/12231), автор [@Cprakhar](https://github.com/Cprakhar))
- **SAP HANA: устаревшие псевдонимы опций подключения удалены** — `hanaClientDriver`, `pool.max`, `pool.requestTimeout`, `pool.idleTimeout` и другие удалены в пользу современных эквивалентов ([#12080](https://github.com/typeorm/typeorm/pull/12080), автор [@gioboa](https://github.com/gioboa))
- **Expo: устаревший драйвер удалён** — прежний драйвер Expo SQLite удалён; используйте Expo SDK v52+ с современным асинхронным API ([#11860](https://github.com/typeorm/typeorm/pull/11860), автор [@G0maa](https://github.com/G0maa))
- **Redis: поддержка устаревшего клиента удалена** — для кэширования результатов запросов поддерживается только современный клиент Redis (v4+) ([#12057](https://github.com/typeorm/typeorm/pull/12057), автор [@G0maa](https://github.com/G0maa))

### Удалённые API

- **`Connection` и `ConnectionOptions` удалены** — используйте вместо них `DataSource` и `DataSourceOptions` ([#12022](https://github.com/typeorm/typeorm/pull/12022), автор [@alumni](https://github.com/alumni))
- **Свойство `.connection` переименовано в `.dataSource`** — свойство `connection` у `Driver`, `QueryRunner`, `EntityManager`, `QueryBuilder`, `EntityMetadata` и всех интерфейсов подписчиков `*Event` переименовано в `dataSource`; для совместимости оставлен устаревший геттер ([#12244](https://github.com/typeorm/typeorm/pull/12244), [#12245](https://github.com/typeorm/typeorm/pull/12245), [#12246](https://github.com/typeorm/typeorm/pull/12246), [#12249](https://github.com/typeorm/typeorm/pull/12249), автор [@pkuczynski](https://github.com/pkuczynski))
- **`ConnectionManager` и глобальные вспомогательные функции удалены** — `createConnection`, `getConnection`, `getManager`, `getRepository`, `createQueryBuilder` и другие глобальные функции удалены ([#12098](https://github.com/typeorm/typeorm/pull/12098), автор [@michaelbromley](https://github.com/michaelbromley))
- **Глобальные `getMongoRepository` и `getMongoManager` удалены** — используйте `dataSource.getMongoRepository()` и `dataSource.mongoManager` ([#12099](https://github.com/typeorm/typeorm/pull/12099), автор [@pkuczynski](https://github.com/pkuczynski))
- **`DataSource.name` удалён** — именованные соединения были объявлены устаревшими в v0.3; `ConnectionOptionsReader.all()` переименован в `get()` ([#12136](https://github.com/typeorm/typeorm/pull/12136), автор [@pkuczynski](https://github.com/pkuczynski))
- **Поддержка переменных окружения `TYPEORM_*` удалена** — `ConnectionOptionsEnvReader`, `ormconfig.env` и автозагрузка `dotenv` удалены ([#12134](https://github.com/typeorm/typeorm/pull/12134), автор [@pkuczynski](https://github.com/pkuczynski))
- **`findByIds` удалён** — используйте `findBy` с оператором `In` ([#12114](https://github.com/typeorm/typeorm/pull/12114), автор [@pkuczynski](https://github.com/pkuczynski))
- **`findOneById` удалён** — используйте `findOneBy` ([#12198](https://github.com/typeorm/typeorm/pull/12198), автор [@pkuczynski](https://github.com/pkuczynski))
- **`Repository.exist()` удалён** — используйте `Repository.exists()` ([#12131](https://github.com/typeorm/typeorm/pull/12131), автор [@pkuczynski](https://github.com/pkuczynski))
- **`AbstractRepository`, `@EntityRepository` и `getCustomRepository` удалены** — используйте `Repository.extend()` ([#12096](https://github.com/typeorm/typeorm/pull/12096), автор [@pkuczynski](https://github.com/pkuczynski))
- **Декоратор `@RelationCount` удалён** — используйте `@VirtualColumn` с подзапросом ([#12181](https://github.com/typeorm/typeorm/pull/12181), автор [@pkuczynski](https://github.com/pkuczynski))
- **Система IoC-контейнера удалена** — `useContainer()`, `getFromContainer()` и связанные типы удалены ([#12180](https://github.com/typeorm/typeorm/pull/12180), автор [@pkuczynski](https://github.com/pkuczynski))
- **Опция колонки `readonly` удалена** — используйте `update: false` ([#12132](https://github.com/typeorm/typeorm/pull/12132), автор [@pkuczynski](https://github.com/pkuczynski))
- **`unsigned` в `ColumnNumericOptions` удалён** — влиял только на типы decimal/float; `unsigned` для целочисленных типов не изменился ([#12133](https://github.com/typeorm/typeorm/pull/12133), автор [@pkuczynski](https://github.com/pkuczynski))
- **QueryBuilder: удалены `onConflict()`, устаревшая перегрузка `orUpdate()` и `setNativeParameters()`** — используйте `orIgnore()`/сигнатуру `orUpdate()` с массивом и `setParameters()` ([#12090](https://github.com/typeorm/typeorm/pull/12090), автор [@pkuczynski](https://github.com/pkuczynski))
- **QueryBuilder: `printSql()` удалён** — он был избыточен, поскольку все выполняемые запросы и так пишутся в настроенный логгер; для просмотра SQL используйте `getSql()` или `getQueryAndParameters()` ([#12151](https://github.com/typeorm/typeorm/pull/12151), автор [@naorpeled](https://github.com/naorpeled), [#12220](https://github.com/typeorm/typeorm/pull/12220), автор [@pkuczynski](https://github.com/pkuczynski))
- **QueryBuilder: псевдоним типа `WhereExpression` удалён** — используйте `WhereExpressionBuilder` ([#12097](https://github.com/typeorm/typeorm/pull/12097), автор [@pkuczynski](https://github.com/pkuczynski))
- **QueryBuilder: `replacePropertyNames()` удалён** — метод ничего не делал ([#12178](https://github.com/typeorm/typeorm/pull/12178), автор [@pkuczynski](https://github.com/pkuczynski))
- **Опция поиска `join` удалена** — используйте `relations` для LEFT JOIN или QueryBuilder для других типов соединений ([#12188](https://github.com/typeorm/typeorm/pull/12188), автор [@pkuczynski](https://github.com/pkuczynski))
- **Строковый вариант `select` удалён** — используйте объектный синтаксис `select: { id: true }` вместо `select: ["id"]` ([#12214](https://github.com/typeorm/typeorm/pull/12214), автор [@pkuczynski](https://github.com/pkuczynski))
- **Строковый вариант `relations` удалён** — используйте объектный синтаксис `relations: { profile: true }` вместо `relations: ["profile"]` ([#12215](https://github.com/typeorm/typeorm/pull/12215), автор [@pkuczynski](https://github.com/pkuczynski))
- **Устаревшие режимы блокировки удалены** — `pessimistic_partial_write` и `pessimistic_write_or_fail` заменены на `pessimistic_write` с опцией `onLocked` ([#12093](https://github.com/typeorm/typeorm/pull/12093), автор [@pkuczynski](https://github.com/pkuczynski))
- **`QueryRunner.loadedTables` и `loadedViews` удалены** — используйте `getTables()` и `getViews()` ([#12183](https://github.com/typeorm/typeorm/pull/12183), автор [@pkuczynski](https://github.com/pkuczynski))
- **`MigrationExecutor.getAllMigrations()` удалён** — используйте `getPendingMigrations()`, `getExecutedMigrations()` или `dataSource.migrations` ([#12142](https://github.com/typeorm/typeorm/pull/12142), автор [@pkuczynski](https://github.com/pkuczynski))
- **Статический метод `EntityMetadata.createPropertyPath()` удалён** — внутренняя утилита без публичной замены ([#12141](https://github.com/typeorm/typeorm/pull/12141), автор [@pkuczynski](https://github.com/pkuczynski))
- **Внутренняя обвязка `nativeParameters` удалена** из драйверов и построителей запросов ([#12104](https://github.com/typeorm/typeorm/pull/12104), автор [@pkuczynski](https://github.com/pkuczynski))
- **Внутренний `broadcastLoadEventsForAll()` удалён** из Broadcaster ([#12137](https://github.com/typeorm/typeorm/pull/12137), автор [@pkuczynski](https://github.com/pkuczynski))
- **Внутренний `DriverUtils.buildColumnAlias()` удалён** — используйте `buildAlias()` ([#12138](https://github.com/typeorm/typeorm/pull/12138), автор [@pkuczynski](https://github.com/pkuczynski))
- **`RdbmsSchemaBuilder.renameTables()` удалён** — пустой метод-заглушка, который никогда не вызывался ([#12284](https://github.com/typeorm/typeorm/pull/12284), автор [@naorpeled](https://github.com/naorpeled))
- **Параметр `options` у `EntityMetadata.getValueMap()` удалён** — опция `skipNulls` никогда не работала; уберите третий аргумент ([#12303](https://github.com/typeorm/typeorm/pull/12303), автор [@naorpeled](https://github.com/naorpeled))
- **`LegacyOracleNamingStrategy` и `NamingStrategyV03` перенесены в `@typeorm/legacy-naming-strategies`** — эти классы больше не экспортируются из ядра `typeorm`; если они всё ещё нужны, установите отдельный пакет `@typeorm/legacy-naming-strategies` и импортируйте их оттуда ([#12416](https://github.com/typeorm/typeorm/pull/12416), автор [@alumni](https://github.com/alumni))

### Изменения в поведении

- **Связи, не допускающие NULL, теперь используют INNER JOIN** — связи `ManyToOne` и владеющие `OneToOne`, помеченные `nullable: false`, теперь используют `INNER JOIN` вместо `LEFT JOIN`, из-за чего строки с «осиротевшими» внешними ключами могут не попадать в выборку ([#12064](https://github.com/typeorm/typeorm/pull/12064), автор [@pkuczynski](https://github.com/pkuczynski))
- **`invalidWhereValuesBehavior` по умолчанию равно `throw`** — передача `null` или `undefined` в условия where теперь выбрасывает ошибку, а не молча игнорирует свойство; для сопоставления с NULL используйте `IsNull()` ([#11710](https://github.com/typeorm/typeorm/pull/11710), автор [@naorpeled](https://github.com/naorpeled))
- **`invalidWhereValuesBehavior` действует только на высокоуровневые API** — методы QueryBuilder `.where()`, `.andWhere()`, `.orWhere()` больше не подчиняются этой настройке ([#11878](https://github.com/typeorm/typeorm/pull/11878), автор [@naorpeled](https://github.com/naorpeled))
- **Пути в `ConnectionOptionsReader` и `FileLogger` разрешаются от `process.cwd()`** — `ConnectionOptionsReader` больше не использует `app-root-path` для поиска файлов `ormconfig`; вместо этого он ищет в `process.cwd()`. Чтобы изменить это, передайте в конструктор `{ root: "/custom/path" }`. `FileLogger.logPath` также разрешается от `process.cwd()` — используйте абсолютный путь, если приложение запускается не из своей корневой папки ([#12257](https://github.com/typeorm/typeorm/pull/12257), автор [@alumni](https://github.com/alumni))

## Новые возможности

### Query Builder

- **`INSERT INTO ... SELECT FROM ...`** — новый метод `valuesFromSelect()` у `InsertQueryBuilder` для запросов миграции и преобразования данных ([#11896](https://github.com/typeorm/typeorm/pull/11896), автор [@Cprakhar](https://github.com/Cprakhar))
- **Опция `returning` для update/upsert** — методы `update()` и `upsert()` репозитория и entity manager теперь поддерживают опцию `returning` в базах данных, где доступно предложение `RETURNING` ([#11782](https://github.com/typeorm/typeorm/pull/11782), автор [@naorpeled](https://github.com/naorpeled))
- **Параметр `ifExists` у всех методов удаления** — `dropColumn`, `dropIndex`, `dropPrimaryKey`, `dropForeignKey`, `dropUniqueConstraint`, `dropCheckConstraint`, `dropExclusionConstraint` и их варианты во множественном числе теперь принимают флаг `ifExists` ([#12121](https://github.com/typeorm/typeorm/pull/12121), автор [@pkuczynski](https://github.com/pkuczynski))
- **Явное управление ресурсами для `QueryRunner`** — поддерживается синтаксис `await using` (TypeScript 5.2+) для автоматической очистки ([#11701](https://github.com/typeorm/typeorm/pull/11701), автор [@alumni](https://github.com/alumni))

### Транзакции

- **Уровень изоляции по умолчанию на уровне DataSource для всех драйверов** — `DataSourceOptions.isolationLevel` теперь учитывается каждым драйвером, поддерживающим транзакции, а не только SQL Server. Транзакции, начатые через DataSource (явно или неявно для DML), по умолчанию используют этот уровень ([#12269](https://github.com/typeorm/typeorm/pull/12269), автор [@pkuczynski](https://github.com/pkuczynski))
- **Aurora Postgres: поддержка уровня изоляции транзакций** — Aurora Postgres теперь учитывает опцию `isolationLevel` у DataSource и при явных вызовах `startTransaction()` ([#12334](https://github.com/typeorm/typeorm/pull/12334), автор [@pkuczynski](https://github.com/pkuczynski))
- **Spanner: поддержка уровня изоляции транзакций** — Google Spanner теперь учитывает опцию `isolationLevel`. Значение `supportedIsolationLevels` для Spanner ограничено уровнями `REPEATABLE READ` (сейчас в статусе предварительной версии) и `SERIALIZABLE` ([#12335](https://github.com/typeorm/typeorm/pull/12335), автор [@pkuczynski](https://github.com/pkuczynski))

### Драйверы

- **PostgreSQL: `ADD VALUE` при изменении перечислений** — при добавлении новых значений enum TypeORM теперь по возможности использует более простой синтаксис `ALTER TYPE ... ADD VALUE` вместо четырёхшагового подхода «переименовать — создать — перенести — удалить» ([#10956](https://github.com/typeorm/typeorm/pull/10956), автор [@janzipek](https://github.com/janzipek))
- **PostgreSQL: дополнительные расширения** — новая опция `installExtensions` для установки дополнительных расширений PostgreSQL при настройке соединения ([#11888](https://github.com/typeorm/typeorm/pull/11888), автор [@Cprakhar](https://github.com/Cprakhar))
- **PostgreSQL: поддержка частичных индексов** — добавлена поддержка частичных индексов PostgreSQL ([#11318](https://github.com/typeorm/typeorm/pull/11318), автор [@freePixel](https://github.com/freePixel))
- **SAP HANA: блокировки в SELECT** — в запросах SAP HANA теперь поддерживаются `FOR UPDATE` и другие режимы блокировки ([#11996](https://github.com/typeorm/typeorm/pull/11996), автор [@alumni](https://github.com/alumni))
- **SAP HANA: комментарии к таблицам** — `@Entity({ comment: "..." })` теперь работает в SAP HANA ([#11939](https://github.com/typeorm/typeorm/pull/11939), автор [@Cprakhar](https://github.com/Cprakhar))
- **SAP HANA: таймаут пула** — новая опция пула `maxWaitTimeoutIfPoolExhausted` ([#11868](https://github.com/typeorm/typeorm/pull/11868), автор [@alumni](https://github.com/alumni))
- **SAP HANA: генерируемые колонки** — `@Column({ generatedType: "STORED", asExpression: "..." })` теперь поддерживается в SAP HANA ([#12393](https://github.com/typeorm/typeorm/pull/12393), автор [@Cprakhar](https://github.com/Cprakhar))
- **SQLite: тип колонки `jsonb`** — SQLite теперь поддерживает тип колонки `jsonb` ([#11933](https://github.com/typeorm/typeorm/pull/11933), автор [@Cprakhar](https://github.com/Cprakhar))
- **MongoDB: объектная проекция `select`** — методы `find*()` теперь принимают тот же объектный синтаксис `select`, что и другие драйверы (`select: { id: true, name: true }`), в дополнение к прежней форме с массивом строк ([#12237](https://github.com/typeorm/typeorm/pull/12237), автор [@pkuczynski](https://github.com/pkuczynski))
- **React Native: ключ шифрования** — новая опция для передачи ключа шифрования для баз данных SQLite в React Native ([#11736](https://github.com/typeorm/typeorm/pull/11736), автор [@HtSpChakradharCholleti](https://github.com/HtSpChakradharCholleti))

### Сохранение данных и upsert

- **Каскадное усечение в `clear()`** — `Repository.clear()` и `EntityManager.clear()` теперь принимают `{ cascade: true }` для выполнения `TRUNCATE ... CASCADE` в PostgreSQL, CockroachDB и Oracle ([#11866](https://github.com/typeorm/typeorm/pull/11866), автор [@Cprakhar](https://github.com/Cprakhar))
- **Улучшенная типизация `increment`/`decrement`** — параметр условий теперь использует корректные типы, учитывающие сущность, вместо `any` ([#11294](https://github.com/typeorm/typeorm/pull/11294), автор [@OSA413](https://github.com/OSA413))

### Типы колонок и декораторы

- **Поддержка deferrable в `@Exclusion`** — повторяет уже имеющуюся поддержку deferrable в `@Unique` и `@Index` ([#11802](https://github.com/typeorm/typeorm/pull/11802), автор [@oGAD31](https://github.com/oGAD31))

### Прочее

## Исправления ошибок

### Формирование запросов

- **Псевдоним колонки корректно экранируется в `orderBy`** — предотвращает ошибки SQL, когда имена псевдонимов совпадают с зарезервированными словами ([#12027](https://github.com/typeorm/typeorm/pull/12027), автор [@Cprakhar](https://github.com/Cprakhar))
- **`addOrderBy` разрешает имена колонок базы данных** — теперь корректно работает использование имени колонки в базе данных (например, `created_at`) вместо имени свойства ([#11904](https://github.com/typeorm/typeorm/pull/11904), автор [@smith-xyz](https://github.com/smith-xyz))
- **Разрешение колонок подзапроса в сортировке** — исправлена ошибка «Cannot get metadata for given alias» при сортировке по колонкам подзапроса ([#11343](https://github.com/typeorm/typeorm/pull/11343), автор [@trannhan0810](https://github.com/trannhan0810))
- **Порядок колонок в `select` сохраняется** — `getQuery()`/`getSql()` теперь возвращают колонки в том порядке, в каком они были добавлены через `select()` и `addSelect()` ([#11902](https://github.com/typeorm/typeorm/pull/11902), автор [@Cprakhar](https://github.com/Cprakhar))
- **Исправлено формирование запроса `.update()`** — устранена генерация некорректного SQL при использовании `.update()` в QueryBuilder ([#11993](https://github.com/typeorm/typeorm/pull/11993), автор [@gioboa](https://github.com/gioboa))
- **Формирование SQL для upsert с псевдонимом таблицы** — исправлены некорректные ссылки на колонки в upsert-запросах при наследовании таблиц и пользовательских схемах ([#11915](https://github.com/typeorm/typeorm/pull/11915), автор [@Cprakhar](https://github.com/Cprakhar))
- **Ограничение выборки при соединениях** — исправлены неверные результаты при постраничной выборке через `skip`/`take` вместе с соединениями ([#11987](https://github.com/typeorm/typeorm/pull/11987), автор [@gioboa](https://github.com/gioboa))
- **Атрибуты соединений внутри скобок** — исправлен разбор соединений, когда условия содержат скобки ([#11218](https://github.com/typeorm/typeorm/pull/11218), автор [@balkrushna](https://github.com/balkrushna))
- **Отключён глобальный `ORDER BY` для агрегатных функций** — `repo.max()`, `repo.min()` и подобные больше не порождают некорректный SQL с предложением `ORDER BY` ([#11925](https://github.com/typeorm/typeorm/pull/11925), автор [@Cprakhar](https://github.com/Cprakhar))
- **Подзапрос постраничной выборки включает первичные ключи присоединённых сущностей** — `leftJoin` вместе с `skip`/`take` теперь корректно загружает связанные сущности ([#11669](https://github.com/typeorm/typeorm/pull/11669), автор [@mag123c](https://github.com/mag123c))
- **Сокращение псевдонимов с camelCase** — метод `shorten` теперь корректно обрабатывает `camelCase_aliases` ([#11283](https://github.com/typeorm/typeorm/pull/11283), автор [@OSA413](https://github.com/OSA413))
- **Типизация сущности для `QueryBuilder.update()`** — сигнатура `update(partialEntity)` теперь использует корректный тип, учитывающий сущность, вместо произвольного объекта ([#11296](https://github.com/typeorm/typeorm/pull/11296), автор [@OSA413](https://github.com/OSA413))
- **Пробелы в логируемых запросах** — начальные и конечные пробелы больше не попадают в записываемые в лог строки запросов ([#12047](https://github.com/typeorm/typeorm/pull/12047), автор [@Cprakhar](https://github.com/Cprakhar))
- **`JsonContains` принимает массивы** — `JsonContains([...])` больше не выбрасывает ошибку и формирует ожидаемый предикат вхождения `@>` в PostgreSQL ([#12420](https://github.com/typeorm/typeorm/pull/12420), автор [@kyungseopk1m](https://github.com/kyungseopk1m))
- **Область соединения для вычисляемых колонок MSSQL** — интроспекция схемы для вычисляемых колонок теперь ограничивается нужной таблицей, когда имена колонок совпадают в нескольких таблицах ([#12288](https://github.com/typeorm/typeorm/pull/12288), автор [@PreAgile](https://github.com/PreAgile))

### Связи и жадная загрузка

- **«Осиротевшие» дочерние записи «один ко многим» с внешним ключом, не допускающим NULL, теперь удаляются** — при сохранении связи «один ко многим» с каскадом и заменой дочерних записей осиротевшие строки с внешним ключом, не допускающим NULL, теперь удаляются, а не приводят к нарушению ограничения; строки с внешним ключом, допускающим NULL, по-прежнему обнуляются ([#11982](https://github.com/typeorm/typeorm/pull/11982), автор [@naorpeled](https://github.com/naorpeled))
- **Жадные связи теперь учитывают `relationLoadStrategy: "query"`** — при выбранной стратегии `"query"` жадные связи загружаются отдельными запросами, а не всегда через JOIN ([#11326](https://github.com/typeorm/typeorm/pull/11326), автор [@SharkSharp](https://github.com/SharkSharp), [#12256](https://github.com/typeorm/typeorm/pull/12256), автор [@pkuczynski](https://github.com/pkuczynski))
- **Конфликт псевдонимов в самоссылающихся связях** — самоссылающиеся связи со стратегией `relationLoadStrategy: "query"` больше не порождают некорректный SQL из-за конфликта псевдонимов ([#11066](https://github.com/typeorm/typeorm/pull/11066), автор [@campmarc](https://github.com/campmarc))
- **Жадные связи больше не присоединяются дважды** — явное указание жадной связи в `relations` больше не приводит к дублирующимся JOIN ([#11991](https://github.com/typeorm/typeorm/pull/11991), автор [@veeceey](https://github.com/veeceey))
- **Сохранение при жадно загруженных связях** — исправлены сбои сохранения, когда у сущности есть жадно загружаемые связи ([#11975](https://github.com/typeorm/typeorm/pull/11975), автор [@gioboa](https://github.com/gioboa))
- **Колонки с `select: false` больше не возвращаются запросами на чтение** — колонки, помеченные `select: false`, теперь корректно исключаются из результатов `find` и query builder; чтобы включить их, используйте `addSelect`. Обратите внимание: `select: false` управляет только тем, что загружается из базы данных при чтении, — сущность в памяти, возвращаемая `repository.save()` (и переданная вами ссылка), не изменяется, поэтому любое заданное вами значение остаётся доступным в ней после сохранения ([#11944](https://github.com/typeorm/typeorm/pull/11944), автор [@gioboa](https://github.com/gioboa), [#12501](https://github.com/typeorm/typeorm/pull/12501), автор [@tada5hi](https://github.com/tada5hi))
- **Подзапрос с методами `joinMapOne`** — исправлено некорректное поведение при использовании методов join map ([#11943](https://github.com/typeorm/typeorm/pull/11943), автор [@gioboa](https://github.com/gioboa))
- **Идентификаторы связей во вложенных встраиваемых сущностях** — исправлена ошибка `TypeError: Cannot set properties of undefined` при отображении идентификаторов связей внутри встраиваемых сущностей ([#11942](https://github.com/typeorm/typeorm/pull/11942), автор [@Cprakhar](https://github.com/Cprakhar))
- **Обработка псевдонимов в `RelationIdLoader`** — используется `DriverUtils.getAlias`, чтобы избежать обрезки псевдонимов в базах данных с коротким пределом длины идентификаторов ([#11228](https://github.com/typeorm/typeorm/pull/11228), автор [@te1](https://github.com/te1))
- **`*-to-many` в `createPropertyPath`** — удалена ошибочная обработка ошибок, мешавшая некоторым конфигурациям связей ([#11119](https://github.com/typeorm/typeorm/pull/11119), автор [@ThbltLmr](https://github.com/ThbltLmr))
- **Каскадное удаление в `OneToMany` с составными первичными ключами** — `cascade: ["remove"]` теперь корректно распространяет удаление на дочерние сущности, если у целевой сущности составной первичный ключ; ранее `CascadesSubjectBuilder` не помечал такие дочерние записи как `mustBeRemoved`, что вызывало нарушения ограничений внешнего ключа ([#12286](https://github.com/typeorm/typeorm/pull/12286), автор [@pkuczynski](https://github.com/pkuczynski))
- **`withDeleted` передаётся в загрузчик идентификаторов связей при восстановлении связей «многие ко многим»** — восстановление мягко удалённой сущности со связями «многие ко многим» больше не пытается заново вставить существующие строки связующей таблицы; `RelationIdLoader` теперь включает мягко удалённые сущности, когда в родительском запросе задан `withDeleted` ([#12287](https://github.com/typeorm/typeorm/pull/12287), автор [@pkuczynski](https://github.com/pkuczynski))

### Сохранение данных

- **Upsert с `update: false` или `generatedType`** — upsert теперь корректно обрабатывает колонки, которые не должны обновляться ([#12030](https://github.com/typeorm/typeorm/pull/12030), автор [@gioboa](https://github.com/gioboa))

- **Мягкое удаление больше не обновляет уже мягко удалённые строки** — `softDelete` и `softRemove` теперь пропускают строки, которые уже мягко удалены ([#10705](https://github.com/typeorm/typeorm/pull/10705), автор [@hassanmehdi98](https://github.com/hassanmehdi98))

- **Сравнение Map и объектов** — исправлено некорректное определение изменений для значений колонок типа Map и простых объектов ([#10990](https://github.com/typeorm/typeorm/pull/10990), автор [@mgohin](https://github.com/mgohin))
- **Определение изменений с преобразователями дат** — исправлены ложные срабатывания определения изменений при использовании преобразователей значений для дат ([#11963](https://github.com/typeorm/typeorm/pull/11963), автор [@gioboa](https://github.com/gioboa))
- **Обновление mpath у дочерних узлов** — mpath древовидной сущности теперь корректно обновляется при смене родителя, в том числе при мягко удалённых родителях ([#10844](https://github.com/typeorm/typeorm/pull/10844), автор [@JoseCToscano](https://github.com/JoseCToscano))
- **Передача схемы/базы данных в связующие таблицы closure** — настройки схемы и базы данных теперь корректно распространяются на связующие таблицы closure ([#12110](https://github.com/typeorm/typeorm/pull/12110), автор [@pkuczynski](https://github.com/pkuczynski))
- **Обработка виртуальных свойств в конструкторе схемы** — конструктор схемы больше не пытается создавать колонки для виртуальных свойств ([#11000](https://github.com/typeorm/typeorm/pull/11000), автор [@skyran1278](https://github.com/skyran1278))
- **Удаление безымянного `TableForeignKey`** — удаление внешнего ключа без явного имени больше не завершается ошибкой ([#10744](https://github.com/typeorm/typeorm/pull/10744), автор [@taichunmin](https://github.com/taichunmin))
- **`getPendingMigrations` больше не создаёт таблицу миграций** — проверка ожидающих миграций больше не имеет побочных эффектов ([#11672](https://github.com/typeorm/typeorm/pull/11672), автор [@pkuczynski](https://github.com/pkuczynski))
- **Внешние ключи `deferrable` для связей «многие ко многим»** — `@ManyToMany` теперь учитывает опцию `deferrable` для внешних ключей связующей таблицы ([#11924](https://github.com/typeorm/typeorm/pull/11924), автор [@smith-xyz](https://github.com/smith-xyz))
- **Порядок колонок в составном внешнем ключе** — конструктор схемы сортирует ссылающиеся колонки в составных внешних ключах в соответствии с порядком индекса первичного ключа, на который они ссылаются, чтобы MySQL / MSSQL / SAP HANA не отклоняли ограничение ([#12280](https://github.com/typeorm/typeorm/pull/12280), автор [@pkuczynski](https://github.com/pkuczynski))
- **Пользовательские общие join-колонки сохраняются в наборе изменений** — явные записи `@JoinColumn`, общие для нескольких связей, больше не теряются при сохранении ([#12354](https://github.com/typeorm/typeorm/pull/12354), автор [@PreAgile](https://github.com/PreAgile))

### Исправления в конкретных драйверах

- **PostgreSQL: сохранение и загрузка `timestamptz`** — колонки `timestamp with time zone` теперь корректно сохраняются и загружаются ([#11774](https://github.com/typeorm/typeorm/pull/11774), автор [@Minishlink](https://github.com/Minishlink))
- **PostgreSQL: повторное сохранение геометрических типов** — значения point/circle теперь нормализуются при сохранении, чтобы избежать ошибок некорректного ввода при повторном сохранении загруженных объектов ([#11857](https://github.com/typeorm/typeorm/pull/11857), автор [@Cprakhar](https://github.com/Cprakhar))
- **PostgreSQL/CockroachDB: таблицы с именами в кавычках** — исправлена обработка таблиц со специальными символами в именах ([#10993](https://github.com/typeorm/typeorm/pull/10993), автор [@iskalyakin](https://github.com/iskalyakin))
- **PostgreSQL: последовательное выполнение запросов** — запросы теперь выполняются последовательно в рамках одного соединения, чтобы избежать предупреждений об устаревании в `pg` 8.19.0; это касается в том числе оставшихся путей загрузки связей и сохранения данных ([#12105](https://github.com/typeorm/typeorm/pull/12105), автор [@pkuczynski](https://github.com/pkuczynski), [#12421](https://github.com/typeorm/typeorm/pull/12421), автор [@kyungseopk1m](https://github.com/kyungseopk1m))

- **MongoDB: преобразование имени свойства `ObjectIdColumn`** — `findOneBy({ id: value })` теперь корректно преобразуется в `_id` в запросах MongoDB ([#12200](https://github.com/typeorm/typeorm/pull/12200), автор [@pkuczynski](https://github.com/pkuczynski))
- **MongoDB: встроенные массивы вложенных документов** — корректно обрабатываются встроенные массивы внутри вложенных структур документов ([#10940](https://github.com/typeorm/typeorm/pull/10940), автор [@mciuchitu](https://github.com/mciuchitu))
- **SQLite: массивы simple-enum** — колонки `simple-enum` с `array: true` больше не приводят к ошибкам CHECK-ограничения ([#11865](https://github.com/typeorm/typeorm/pull/11865), автор [@Cprakhar](https://github.com/Cprakhar))
- **SAP HANA: экранирование параметров `Date`** — значения `Date` из JS теперь передаются как параметры запроса, а не встраиваются в строки SQL ([#11867](https://github.com/typeorm/typeorm/pull/11867), автор [@alumni](https://github.com/alumni))
- **CockroachDB: структурированные результаты запросов при повторе транзакции** — флаг `useStructuredResult` теперь сохраняется при повторном проигрывании транзакции в CockroachDB ([#11861](https://github.com/typeorm/typeorm/pull/11861), автор [@naorpeled](https://github.com/naorpeled))
- **Cordova: количество затронутых строк** — результаты запросов теперь включают число затронутых строк ([#10873](https://github.com/typeorm/typeorm/pull/10873), автор [@jacobg](https://github.com/jacobg))
- **PostgreSQL / CockroachDB: порядок объявления значений enum** — значения перечислений теперь загружаются в порядке их объявления, поэтому генерируемые миграции больше не «дрожат», когда ничего не изменилось ([#12404](https://github.com/typeorm/typeorm/pull/12404), автор [@pkuczynski](https://github.com/pkuczynski))
- **Expo: автозагрузка `expo-sqlite`** — драйвер Expo загружает `expo-sqlite` через `loadDependencies()`, поэтому передавать модуль явно больше не нужно ([#12363](https://github.com/typeorm/typeorm/pull/12363), автор [@pkuczynski](https://github.com/pkuczynski))

## Исправления безопасности

- **Защита от SQL-инъекций** — во всех драйверах для интроспекции схемы и методов DDL теперь используются параметризованные запросы и экранированные идентификаторы, что предотвращает SQL-инъекции через имена баз данных, схем, таблиц и колонок ([#12207](https://github.com/typeorm/typeorm/pull/12207), автор [@pkuczynski](https://github.com/pkuczynski), [#12197](https://github.com/typeorm/typeorm/pull/12197), автор [@pkuczynski](https://github.com/pkuczynski), [#12185](https://github.com/typeorm/typeorm/pull/12185), автор [@pkuczynski](https://github.com/pkuczynski))
- **Проверка условий в orderBy** — `orderBy` и `addOrderBy` в QueryBuilder теперь проверяют значения условий во время выполнения, предотвращая инъекции через выражения сортировки ([#12217](https://github.com/typeorm/typeorm/pull/12217), автор [@pkuczynski](https://github.com/pkuczynski))
- **Проверка `.limit()` в построителях запросов Update/SoftDelete** — нечисловые значения, переданные в `.limit()`, теперь вызывают ошибку в момент вызова, а не подставляются в SQL ([#12436](https://github.com/typeorm/typeorm/pull/12436), автор [@smith-xyz](https://github.com/smith-xyz))

## Улучшения производительности

- **PostgreSQL / CockroachDB: пакетный DROP в `clearDatabase()`** — отдельные операторы DROP объединяются в единые пакетные запросы, что существенно сокращает число обращений к серверу при подготовке тестов ([#12164](https://github.com/typeorm/typeorm/pull/12164), [#12159](https://github.com/typeorm/typeorm/pull/12159), автор [@pkuczynski](https://github.com/pkuczynski))

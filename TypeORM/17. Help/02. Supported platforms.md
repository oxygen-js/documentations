# Поддерживаемые платформы

## NodeJS

TypeORM совместим с Node.js 20+; в настоящее время каждый коммит тестируется на Node.js 20 и 24.

## Браузер

В браузере можно использовать [sql.js](https://sql.js.org).

### Настройка Webpack

В папке `browser` пакет также содержит версию, скомпилированную как модуль ES2015. Если вы хотите использовать другой загрузчик, начинать следует отсюда. До версии TypeORM 0.1.7 пакет был устроен так, что загрузчики вроде webpack автоматически использовали папку `browser`. В 0.1.7 от этого отказались, чтобы поддержать использование Webpack в проектах на Node.js. Это значит, что для браузерных проектов нужно применять `NormalModuleReplacementPlugin`, гарантирующий загрузку правильной версии. Настройка этого плагина в конфигурационном файле webpack выглядит так:

```js
plugins: [
    ..., // любые плагины, которые у вас уже есть
    new webpack.NormalModuleReplacementPlugin(/typeorm$/, function (result) {
        result.request = result.request.replace(/typeorm/, "typeorm/browser");
    }),
    new webpack.ProvidePlugin({
      'window.SQL': 'sql.js/dist/sql-wasm.js'
    })
]
```

и убедитесь, что [файл sql-wasm.wasm](https://github.com/sql-js/sql.js/blob/master/README.md#downloadingusing) доступен по вашему public-пути.

### Пример конфигурации

```typescript
new DataSource({
    type: "sqljs",
    entities: [Photo],
    synchronize: true,
})
```

### Не забудьте подключить reflect-metadata

На главной HTML-странице нужно подключить reflect-metadata:

```html
<script src="./node_modules/reflect-metadata/Reflect.js"></script>
```

## Приложения Cordova / Ionic

TypeORM может работать в приложениях Cordova/Ionic через плагин
[cordova-sqlite-storage](https://github.com/litehelpers/Cordova-sqlite-storage).
Как и в браузерном пакете, у вас есть выбор между загрузчиками модулей.
**Важно**: для использования с Ionic нужен собственный конфигурационный файл webpack. Учтите, что при работе с плагином [cordova-sqlite-storage](https://github.com/litehelpers/Cordova-sqlite-storage) транзакции сейчас не поддерживаются. Подробнее — в разделе [ограничения Cordova SQLite](https://github.com/storesafe/cordova-sqlite-storage#other-limitations).

## Expo

TypeORM может работать в приложениях Expo через [Expo SQLite API](https://docs.expo.io/versions/latest/sdk/sqlite/).

## NativeScript

1. `tns install webpack` (почему нужен webpack — читайте ниже)
2. `tns plugin add nativescript-sqlite`
3. Создайте DataSource в точке входа вашего приложения

    ```typescript
    import driver from "nativescript-sqlite"

    const dataSource = new DataSource({
        database: "test.db",
        type: "nativescript",
        driver,
        entities: [
            Todo, //... любые ваши сущности
        ],
        logging: true,
    })
    ```

Примечание: это работает только с NativeScript 4.x и выше

_При работе с NativeScript **использование webpack обязательно**.
Пакет `typeorm/browser` — это «сырой» код ES7 с `import/export`,
который **НЕ** будет работать как есть. Его нужно собрать в бандл.
Используйте способ `tns run --bundle`_

Пример смотрите [здесь](https://github.com/championswimmer/nativescript-vue-typeorm-sample)!

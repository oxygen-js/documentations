# Древовидные сущности

TypeORM поддерживает паттерны Adjacency list (список смежности) и Closure table (таблица замыканий) для хранения древовидных структур.
Подробнее об иерархических таблицах — в [этой отличной презентации Билла Карвина](https://www.slideshare.net/billkarwin/models-for-hierarchical-data).

## Adjacency list

Adjacency list — это простая модель со ссылкой на саму себя.
Обратите внимание: `TreeRepository` не поддерживает Adjacency list.
Преимущество этого подхода — простота,
недостаток — из-за ограничений на join нельзя загрузить большое дерево целиком.
Подробнее о преимуществах и применении списков смежности — в [этой статье Мэттью Шинкеля](http://schinckel.net/2014/09/13/long-live-adjacency-lists/).
Пример:

```typescript
import {
    Entity,
    Column,
    PrimaryGeneratedColumn,
    ManyToOne,
    OneToMany,
} from "typeorm"

@Entity()
export class Category {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @Column()
    description: string

    @ManyToOne((type) => Category, (category) => category.children)
    parent: Category

    @OneToMany((type) => Category, (category) => category.parent)
    children: Category[]
}
```

## Nested set

Nested set (вложенные множества) — ещё один паттерн хранения древовидных структур в базе данных.
Он очень эффективен при чтении, но плох при записи.
В nested set не может быть нескольких корней.
Пример:

```typescript
import {
    Entity,
    Tree,
    Column,
    PrimaryGeneratedColumn,
    TreeChildren,
    TreeParent,
    TreeLevelColumn,
} from "typeorm"

@Entity()
@Tree("nested-set")
export class Category {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @TreeChildren()
    children: Category[]

    @TreeParent()
    parent: Category
}
```

## Materialized Path (он же Path Enumeration)

Materialized Path (материализованный путь, также называемый Path Enumeration) — ещё один паттерн хранения древовидных структур в базе данных.
Он прост и эффективен.
Пример:

```typescript
import {
    Entity,
    Tree,
    Column,
    PrimaryGeneratedColumn,
    TreeChildren,
    TreeParent,
    TreeLevelColumn,
} from "typeorm"

@Entity()
@Tree("materialized-path")
export class Category {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @TreeChildren()
    children: Category[]

    @TreeParent()
    parent: Category
}
```

## Closure table

Closure table особым образом хранит связи между родителем и потомком в отдельной таблице.
Этот подход эффективен и при чтении, и при записи.
Пример:

```typescript
import {
    Entity,
    Tree,
    Column,
    PrimaryGeneratedColumn,
    TreeChildren,
    TreeParent,
    TreeLevelColumn,
} from "typeorm"

@Entity()
@Tree("closure-table")
export class Category {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @TreeChildren()
    children: Category[]

    @TreeParent()
    parent: Category
}
```

Имя closure-таблицы и/или имена её колонок можно задать через необязательный параметр `options` в `@Tree("closure-table", options)`. `ancestorColumnName` и `descendantColumnName` — это колбэк-функции, которые получают метаданные первичной колонки и возвращают имя колонки. `closureTableName` — аргумент, передаваемый в `closureJunctionTableName` стратегии именования источника данных; по умолчанию используется `tableNameWithoutPrefix` родительской сущности. `closureTableSchema` задаёт имя схемы, в которой должна быть сохранена closure-таблица; по умолчанию — схема родительской сущности (если она указана в опциях декоратора родительской сущности) либо схема источника данных.

```ts
@Tree("closure-table", {
    closureTableName: "category",
    closureTableSchema: "closures_schema",
    ancestorColumnName: (column) => "ancestor_" + column.propertyName,
    descendantColumnName: (column) => "descendant_" + column.propertyName,
})
```

## Работа с древовидными сущностями

Чтобы связать древовидные сущности друг с другом, нужно задать родителя в дочерней сущности и затем сохранить их.
Например:

```typescript
const a1 = new Category()
a1.name = "a1"
await dataSource.manager.save(a1)

const a11 = new Category()
a11.name = "a11"
a11.parent = a1
await dataSource.manager.save(a11)

const a12 = new Category()
a12.name = "a12"
a12.parent = a1
await dataSource.manager.save(a12)

const a111 = new Category()
a111.name = "a111"
a111.parent = a11
await dataSource.manager.save(a111)

const a112 = new Category()
a112.name = "a112"
a112.parent = a11
await dataSource.manager.save(a112)
```

Для загрузки такого дерева используйте `TreeRepository`:

```typescript
const trees = await dataSource.manager.getTreeRepository(Category).findTrees()
```

`trees` будет таким:

```json
[
    {
        "id": 1,
        "name": "a1",
        "children": [
            {
                "id": 2,
                "name": "a11",
                "children": [
                    {
                        "id": 4,
                        "name": "a111"
                    },
                    {
                        "id": 5,
                        "name": "a112"
                    }
                ]
            },
            {
                "id": 3,
                "name": "a12"
            }
        ]
    }
]
```

Есть и другие специальные методы `TreeRepository` для работы с древовидными сущностями:

- `findTrees` — возвращает все деревья из базы данных со всеми их потомками, потомками потомков и т. д.

```typescript
const treeCategories = await dataSource.manager
    .getTreeRepository(Category)
    .findTrees()
// возвращает корневые категории с вложенными подкатегориями

const treeCategoriesWithLimitedDepth = await dataSource.manager
    .getTreeRepository(Category)
    .findTrees({ depth: 2 })
// возвращает корневые категории с вложенными подкатегориями, до глубины 2
```

- `findRoots` — корни — это сущности без предков. Находит их все.
  Листья потомков не загружаются.

```typescript
const rootCategories = await dataSource.manager
    .getTreeRepository(Category)
    .findRoots()
// возвращает корневые категории без вложенных подкатегорий
```

- `findDescendants` — получает всех потомков заданной сущности. Возвращает их плоским массивом.

```typescript
const children = await dataSource.manager
    .getTreeRepository(Category)
    .findDescendants(parentCategory)
// возвращает все непосредственные подкатегории parentCategory (без вложенных в них категорий)
```

- `findDescendantsTree` — получает всех потомков заданной сущности. Возвращает их деревом, вложенными друг в друга.

```typescript
const childrenTree = await repository.findDescendantsTree(parentCategory)
// возвращает все непосредственные подкатегории parentCategory (вместе с вложенными в них категориями)
const childrenTreeWithLimitedDepth = await repository.findDescendantsTree(
    parentCategory,
    { depth: 2 },
)
// возвращает все непосредственные подкатегории parentCategory (вместе с вложенными в них категориями), до глубины 2
```

- `createDescendantsQueryBuilder` — создаёт query builder для получения потомков сущностей в дереве.

```typescript
const children = await repository
    .createDescendantsQueryBuilder(
        "category",
        "categoryClosure",
        parentCategory,
    )
    .andWhere("category.type = 'secondary'")
    .getMany()
```

- `countDescendants` — получает количество потомков сущности.

```typescript
const childrenCount = await dataSource.manager
    .getTreeRepository(Category)
    .countDescendants(parentCategory)
```

- `findAncestors` — получает всех предков заданной сущности. Возвращает их плоским массивом.

```typescript
const parents = await repository.findAncestors(childCategory)
// возвращает все непосредственные родительские категории childCategory (без «родителей родителей»)
```

- `findAncestorsTree` — получает всех предков заданной сущности. Возвращает их деревом, вложенными друг в друга.

```typescript
const parentsTree = await dataSource.manager
    .getTreeRepository(Category)
    .findAncestorsTree(childCategory)
// возвращает все непосредственные родительские категории childCategory (вместе с «родителями родителей»)
```

- `createAncestorsQueryBuilder` — создаёт query builder для получения предков сущностей в дереве.

```typescript
const parents = await repository
    .createAncestorsQueryBuilder("category", "categoryClosure", childCategory)
    .andWhere("category.type = 'secondary'")
    .getMany()
```

- `countAncestors` — получает количество предков сущности.

```typescript
const parentsCount = await dataSource.manager
    .getTreeRepository(Category)
    .countAncestors(childCategory)
```

Следующим методам можно передавать опции:

- findTrees
- findRoots
- findDescendants
- findDescendantsTree
- findAncestors
- findAncestorsTree

Доступны такие опции:

- `relations` — указывает, какие связи сущности должны быть загружены (упрощённая форма left join).

Примеры:

```typescript
const treeCategoriesWithRelations = await dataSource.manager
    .getTreeRepository(Category)
    .findTrees({
        relations: ["sites"],
    })
// автоматически присоединяет связь sites

const parentsWithRelations = await dataSource.manager
    .getTreeRepository(Category)
    .findAncestors(childCategory, {
        relations: ["members"],
    })
// возвращает все непосредственные родительские категории childCategory (без «родителей родителей») и присоединяет связь 'members'
```

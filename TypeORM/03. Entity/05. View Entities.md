# Сущности-представления

## Что такое ViewEntity?

Сущность-представление — это класс, который отображается на представление (view) базы данных.
Создать сущность-представление можно, объявив новый класс и пометив его `@ViewEntity()`:

`@ViewEntity()` принимает следующие опции:

- `name` — имя представления. Если не указано, имя генерируется из имени класса сущности.
- `database` — имя базы данных на выбранном сервере БД.
- `schema` — имя схемы.
- `expression` — определение представления. **Обязательный параметр**.
- `dependsOn` — список других представлений, от которых зависит текущее. Если ваше представление использует в своём определении другое представление, укажите его здесь, чтобы [миграции](<../06. Migrations/01. How migrations work.md>) были сгенерированы в правильном порядке.

`expression` может быть строкой с корректно экранированными колонками и таблицами — в зависимости от используемой базы данных (в примере postgres):

```typescript
@ViewEntity({
    expression: `
        SELECT "post"."id" AS "id", "post"."name" AS "name", "category"."name" AS "categoryName"
        FROM "post" "post"
        LEFT JOIN "category" "category" ON "post"."categoryId" = "category"."id"
    `
})
```

либо экземпляром QueryBuilder

```typescript
@ViewEntity({
    expression: (dataSource: DataSource) => dataSource
        .createQueryBuilder()
        .select("post.id", "id")
        .addSelect("post.name", "name")
        .addSelect("category.name", "categoryName")
        .from(Post, "post")
        .leftJoin(Category, "category", "category.id = post.categoryId")
})
```

**Примечание:** привязка параметров не поддерживается из-за ограничений драйверов. Используйте литеральные значения.

```typescript
@ViewEntity({
    expression: (dataSource: DataSource) => dataSource
        .createQueryBuilder()
        .select("post.id", "id")
        .addSelect("post.name", "name")
        .addSelect("category.name", "categoryName")
        .from(Post, "post")
        .leftJoin(Category, "category", "category.id = post.categoryId")
        .where("category.name = :name", { name: "Cars" })  // <-- так неправильно
        .where("category.name = 'Cars'")                   // <-- а так правильно
})
```

Каждая сущность-представление должна быть зарегистрирована в опциях источника данных:

```typescript
import { DataSource } from "typeorm"
import { UserView } from "./entities/UserView"

const dataSource = new DataSource({
    type: "mysql",
    host: "localhost",
    port: 3306,
    username: "test",
    password: "test",
    database: "test",
    entities: [UserView],
})
```

## Колонки сущности-представления

Чтобы данные из представления попали в нужные колонки сущности, пометьте колонки сущности декоратором `@ViewColumn()`
и укажите эти колонки как псевдонимы в выражении select.

пример с определением через строку:

```typescript
import { ViewEntity, ViewColumn } from "typeorm"

@ViewEntity({
    expression: `
        SELECT "post"."id" AS "id", "post"."name" AS "name", "category"."name" AS "categoryName"
        FROM "post" "post"
        LEFT JOIN "category" "category" ON "post"."categoryId" = "category"."id"
    `,
})
export class PostCategory {
    @ViewColumn()
    id: number

    @ViewColumn()
    name: string

    @ViewColumn()
    categoryName: string
}
```

пример с использованием QueryBuilder:

```typescript
import { ViewEntity, ViewColumn } from "typeorm"

@ViewEntity({
    expression: (dataSource: DataSource) =>
        dataSource
            .createQueryBuilder()
            .select("post.id", "id")
            .addSelect("post.name", "name")
            .addSelect("category.name", "categoryName")
            .from(Post, "post")
            .leftJoin(Category, "category", "category.id = post.categoryId"),
})
export class PostCategory {
    @ViewColumn()
    id: number

    @ViewColumn()
    name: string

    @ViewColumn()
    categoryName: string
}
```

## Опции колонок представления

Опции колонок представления задают дополнительные параметры для колонок вашей сущности-представления — аналогично [опциям колонок](<01. Entities.md#опции-колонок>) обычных сущностей.

Указать опции колонки представления можно в `@ViewColumn`:

```typescript
@ViewColumn({
    name: "postName",
    // ...
})
name: string;
```

Список доступных опций в `ViewColumnOptions`:

- `name: string` — имя колонки в представлении базы данных.
- `transformer: { from(value: DatabaseType): EntityType, to(value: EntityType): DatabaseType }` — используется для преобразования свойств произвольного типа `DatabaseType`, поддерживаемого базой данных, в тип `EntityType`. Массивы трансформеров также поддерживаются и применяются при чтении в обратном порядке. Учтите, что представления базы данных доступны только для чтения, поэтому `transformer.to(value)` никогда не будет вызван.

## Индексы материализованных представлений

Для `PostgreSQL` поддерживается создание индексов для материализованных представлений.

```typescript
@ViewEntity({
    materialized: true,
    expression: (dataSource: DataSource) =>
        dataSource
            .createQueryBuilder()
            .select("post.id", "id")
            .addSelect("post.name", "name")
            .addSelect("category.name", "categoryName")
            .from(Post, "post")
            .leftJoin(Category, "category", "category.id = post.categoryId"),
})
export class PostCategory {
    @ViewColumn()
    id: number

    @Index()
    @ViewColumn()
    name: string

    @Index("catname-idx")
    @ViewColumn()
    categoryName: string
}
```

Однако на данный момент для индексов материализованных представлений поддерживается только опция `unique`. Остальные опции индекса игнорируются.

```typescript
@Index("name-idx", { unique: true })
@ViewColumn()
name: string
```

## Полный пример

Создадим две сущности и представление с агрегированными данными из них:

```typescript
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm"

@Entity()
export class Category {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string
}
```

```typescript
import {
    Entity,
    PrimaryGeneratedColumn,
    Column,
    ManyToOne,
    JoinColumn,
} from "typeorm"
import { Category } from "./Category"

@Entity()
export class Post {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @Column()
    categoryId: number

    @ManyToOne(() => Category)
    @JoinColumn({ name: "categoryId" })
    category: Category
}
```

```typescript
import { ViewEntity, ViewColumn, DataSource } from "typeorm"

@ViewEntity({
    expression: (dataSource: DataSource) =>
        dataSource
            .createQueryBuilder()
            .select("post.id", "id")
            .addSelect("post.name", "name")
            .addSelect("category.name", "categoryName")
            .from(Post, "post")
            .leftJoin(Category, "category", "category.id = post.categoryId"),
})
export class PostCategory {
    @ViewColumn()
    id: number

    @ViewColumn()
    name: string

    @ViewColumn()
    categoryName: string
}
```

затем заполним эти таблицы данными и запросим все данные из представления PostCategory:

```typescript
import { Category } from "./entities/Category"
import { Post } from "./entities/Post"
import { PostCategory } from "./entities/PostCategory"

const category1 = new Category()
category1.name = "Cars"
await dataSource.manager.save(category1)

const category2 = new Category()
category2.name = "Airplanes"
await dataSource.manager.save(category2)

const post1 = new Post()
post1.name = "About BMW"
post1.categoryId = category1.id
await dataSource.manager.save(post1)

const post2 = new Post()
post2.name = "About Boeing"
post2.categoryId = category2.id
await dataSource.manager.save(post2)

const postCategories = await dataSource.manager.find(PostCategory)
const postCategory = await dataSource.manager.findOneBy(PostCategory, { id: 1 })
```

в результате в `postCategories` будет:

```javascript
[ PostCategory { id: 1, name: 'About BMW', categoryName: 'Cars' },
  PostCategory { id: 2, name: 'About Boeing', categoryName: 'Airplanes' } ]
```

а в `postCategory`:

```javascript
PostCategory { id: 1, name: 'About BMW', categoryName: 'Cars' }
```

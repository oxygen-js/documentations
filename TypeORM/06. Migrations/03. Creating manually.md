# Создание вручную

Новую миграцию можно создать через CLI, указав имя и расположение миграции:

```shell
npx typeorm migration:create <path/to/migrations>/<migration-name>
```

Например:

```shell
npx typeorm migration:create src/db/migrations/post-refactoring
```

После выполнения команды в каталоге `src/db/migrations` появится новый файл с именем `{TIMESTAMP}-post-refactoring.ts`, где `{TIMESTAMP}` — текущая метка времени на момент генерации миграции.

Теперь можно открыть файл и добавить в него SQL-запросы миграции. Внутри миграции вы увидите следующее содержимое:

```typescript
import { MigrationInterface, QueryRunner } from "typeorm"

export class PostRefactoringTIMESTAMP implements MigrationInterface {
    async up(queryRunner: QueryRunner): Promise<void> {}

    async down(queryRunner: QueryRunner): Promise<void> {}
}
```

Есть два метода, которые нужно заполнить кодом миграции: `up` и `down`.
`up` должен содержать код, выполняющий миграцию.
`down` должен отменять всё, что изменил `up`.
Метод `down` используется для отката последней миграции.

Внутри и `up`, и `down` вам доступен объект `QueryRunner`.
Все операции с базой данных выполняются через этот объект.
Подробнее о [query runner](<../09. Query Runner.md>).

Посмотрим, как выглядит миграция с нашими изменениями в `Post`:

```typescript
import { MigrationInterface, QueryRunner } from "typeorm"

export class PostRefactoringTIMESTAMP implements MigrationInterface {
    async up(queryRunner: QueryRunner): Promise<void> {
        await queryRunner.query(
            `ALTER TABLE "post" RENAME COLUMN "title" TO "name"`,
        )
    }

    async down(queryRunner: QueryRunner): Promise<void> {
        await queryRunner.query(
            `ALTER TABLE "post" RENAME COLUMN "name" TO "title"`,
        ) // отменяет то, что было сделано в методе "up"
    }
}
```

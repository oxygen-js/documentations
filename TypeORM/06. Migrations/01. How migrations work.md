# Как работают миграции?

Когда дело доходит до продакшена, вам потребуется переносить изменения модели в базу данных.
Как правило, использовать `synchronize: true` для синхронизации схемы в продакшене небезопасно,
если в базе уже есть данные. Здесь на помощь приходят миграции.

Миграция — это просто один файл с SQL-запросами, которые обновляют схему базы данных
и применяют новые изменения к существующей базе.

Допустим, у вас уже есть база данных и сущность `Post`:

```typescript
import { Entity, Column, PrimaryGeneratedColumn } from "typeorm"

@Entity()
export class Post {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    title: string

    @Column()
    text: string
}
```

И ваша сущность месяцами работала в продакшене без изменений.
В базе данных накопились тысячи постов.

Теперь вам нужно выпустить новый релиз и переименовать `title` в `name`.
Что делать?

Нужно создать новую миграцию со следующим SQL-запросом (диалект PostgreSQL):

```sql
ALTER TABLE "post" RENAME COLUMN "title" TO "name";
```

После выполнения этого SQL-запроса схема базы данных будет готова к работе с вашей новой кодовой базой.
TypeORM предоставляет место, где можно писать такие SQL-запросы и запускать их по мере необходимости.
Это место называется «миграции».

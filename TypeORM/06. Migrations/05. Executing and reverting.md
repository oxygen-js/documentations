# Выполнение и откат

Когда миграция готова к запуску на продакшене, её можно выполнить командой CLI:

```shell
typeorm migration:run -- -d path-to-datasource-config
```

**Команды `typeorm migration:create` и `typeorm migration:generate` создают файлы `.ts`, если только вы не указали флаг `o` (подробнее в разделе [Генерация миграций](<04. Generating.md>)). Команды `migration:run` и `migration:revert` работают только с файлами `.js`. Поэтому перед их запуском файлы TypeScript нужно скомпилировать.** Как вариант, можно использовать `ts-node` вместе с `typeorm`, чтобы запускать `.ts`-файлы миграций напрямую.

Пример с `ts-node`:

```shell
npx typeorm-ts-node-commonjs migration:run -- -d path-to-datasource-config
```

Пример с `ts-node` в ESM-проектах:

```shell
npx typeorm-ts-node-esm migration:run -- -d path-to-datasource-config
```

```shell
npx typeorm-ts-node-esm migration:generate ./src/migrations/update-post-table -d ./src/data-source.ts
```

Эта команда выполнит все ожидающие миграции в порядке их временных меток.
То есть будут выполнены все SQL-запросы, написанные в методах `up` созданных вами миграций.
Вот и всё! Теперь схема вашей базы данных актуальна.

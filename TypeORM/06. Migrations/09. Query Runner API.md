# API query runner

Чтобы менять схему базы данных через API, используйте `QueryRunner`.

```ts
import {
    MigrationInterface,
    QueryRunner,
    Table,
    TableIndex,
    TableColumn,
    TableForeignKey,
} from "typeorm"

export class QuestionRefactoringTIMESTAMP implements MigrationInterface {
    async up(queryRunner: QueryRunner): Promise<void> {
        await queryRunner.createTable(
            new Table({
                name: "question",
                columns: [
                    {
                        name: "id",
                        type: "int",
                        isPrimary: true,
                    },
                    {
                        name: "name",
                        type: "varchar",
                    },
                ],
            }),
            true,
        )

        await queryRunner.createIndex(
            "question",
            new TableIndex({
                name: "IDX_QUESTION_NAME",
                columnNames: ["name"],
            }),
        )

        await queryRunner.createTable(
            new Table({
                name: "answer",
                columns: [
                    {
                        name: "id",
                        type: "int",
                        isPrimary: true,
                    },
                    {
                        name: "name",
                        type: "varchar",
                    },
                    {
                        name: "created_at",
                        type: "timestamp",
                        default: "now()",
                    },
                ],
            }),
            true,
        )

        await queryRunner.addColumn(
            "answer",
            new TableColumn({
                name: "questionId",
                type: "int",
            }),
        )

        await queryRunner.createForeignKey(
            "answer",
            new TableForeignKey({
                columnNames: ["questionId"],
                referencedColumnNames: ["id"],
                referencedTableName: "question",
                onDelete: "CASCADE",
            }),
        )
    }

    async down(queryRunner: QueryRunner): Promise<void> {
        const table = await queryRunner.getTable("answer")
        const foreignKey = table.foreignKeys.find(
            (fk) => fk.columnNames.indexOf("questionId") !== -1,
        )
        await queryRunner.dropForeignKey("answer", foreignKey)
        await queryRunner.dropColumn("answer", "questionId")
        await queryRunner.dropTable("answer")
        await queryRunner.dropIndex("question", "IDX_QUESTION_NAME")
        await queryRunner.dropTable("question")
    }
}
```

---

```ts
createSchema(schemaPath: string, ifNotExists?: boolean): Promise<void>
```

- `schemaPath` — имя схемы. Для SqlServer может принимать путь к схеме (например, 'dbName.schemaName').
  Если передан путь к схеме, схема будет создана в указанной базе данных
- `ifNotExists` — при значении `true` молча игнорирует случай, когда схема уже существует; иначе выбрасывает ошибку (по умолчанию)

Создаёт новую схему таблиц.

---

```ts
dropSchema(schemaPath: string, ifExists?: boolean, isCascade?: boolean): Promise<void>
```

- `schemaPath` — имя схемы. Для SqlServer может принимать путь к схеме (например, 'dbName.schemaName').
  Если передан путь к схеме, схема будет удалена в указанной базе данных
- `ifExists` — при значении `true` молча игнорирует случай, когда схемы не существует; иначе выбрасывает ошибку (по умолчанию)
- `isCascade` — при значении `true` автоматически удаляет объекты (таблицы, функции и т. д.), содержащиеся в схеме.
  Используется только в Postgres.

Удаляет схему таблиц.

```ts
createTable(table: Table, ifNotExists?: boolean, createForeignKeys?: boolean, createIndices?: boolean): Promise<void>
```

- `table` — объект Table.
- `ifNotExists` — при значении `true` молча игнорирует случай, когда таблица уже существует; иначе выбрасывает ошибку (по умолчанию)
- `createForeignKeys` — указывает, будут ли внешние ключи созданы вместе с таблицей. По умолчанию `true`
- `createIndices` — указывает, будут ли индексы созданы вместе с таблицей. По умолчанию `true`

Создаёт новую таблицу.

---

```ts
dropTable(table: Table|string, ifExists?: boolean, dropForeignKeys?: boolean, dropIndices?: boolean): Promise<void>
```

- `table` — объект Table или имя удаляемой таблицы
- `ifExists` — при значении `true` молча игнорирует случай, когда таблицы не существует; иначе выбрасывает ошибку (по умолчанию)
- `dropForeignKeys` — указывает, будут ли внешние ключи удалены вместе с таблицей. По умолчанию `true`
- `dropIndices` — указывает, будут ли индексы удалены вместе с таблицей. По умолчанию `true`

Удаляет таблицу.

```ts
clearTable(tableName: string, options?: {cascade?: boolean}): Promise<void>
```

- `tableName` — имя таблицы
- `options` — дополнительные опции
    - `cascade` — указывает, очищать ли строки таблиц, имеющих внешние ключи (поддерживается только PostgreSQL/CockroachDB и Oracle; остальные базы данных выбрасывают ошибку при значении `true`). По умолчанию `false`

> **Примечание** эта операция использует SQL-запрос TRUNCATE, который нельзя откатить в транзакциях.

```ts
enableSqlMemory(): void
```

Включает особый режим query runner, в котором SQL-запросы не выполняются, а вместо этого запоминаются в специальной переменной внутри query runner.
Запомненный SQL можно получить методом `getMemorySql()`.

---

```ts
disableSqlMemory(): void
```

Отключает особый режим query runner, в котором SQL-запросы не выполняются. Ранее запомненный SQL будет сброшен.

---

```ts
clearSqlMemory(): void
```

Сбрасывает все запомненные SQL-выражения.

---

```ts
getMemorySql(): SqlInMemory
```

- возвращает объект `SqlInMemory` с массивами SQL-выражений `upQueries` и `downQueries`

Получает SQL, сохранённый в памяти. Параметры в SQL уже подставлены.

---

```ts
executeMemoryUpSql(): Promise<void>
```

Выполняет запомненные SQL-запросы up.

---

```ts
executeMemoryDownSql(): Promise<void>
```

Выполняет запомненные SQL-запросы down.

# Имитация миграций и откатов

Вы также можете имитировать выполнение миграции с помощью флага `--fake` (сокращённо `-f`). Он добавит миграцию
в таблицу миграций, не выполняя её. Это полезно для миграций, созданных после того, как в базе данных уже были
сделаны изменения вручную, или когда миграции выполнялись извне
(например, другим инструментом или приложением), а вы всё же хотите сохранить целостную историю миграций.

```shell
typeorm migration:run -d path-to-datasource-config --fake
```

То же самое возможно и для откатов.

```shell
typeorm migration:revert -d path-to-datasource-config --fake
```

### Режимы транзакций

По умолчанию TypeORM выполняет все ваши миграции внутри одной охватывающей транзакции.
Это соответствует флагу `--transaction all`.
Если вам нужен более тонкий контроль над транзакциями, используйте флаг `--transaction each`, чтобы обернуть каждую миграцию по отдельности, или флаг `--transaction none`, чтобы вовсе отказаться от оборачивания миграций в транзакции.

Помимо этих флагов, поведение транзакций можно переопределить для отдельной миграции, задав свойство `transaction` у `MigrationInterface` в значение `true` или `false`. Это работает только в режимах транзакций `each` и `none`.

```typescript
import { MigrationInterface, QueryRunner } from "typeorm"

export class AddIndexTIMESTAMP implements MigrationInterface {
    transaction = false

    async up(queryRunner: QueryRunner): Promise<void> {
        await queryRunner.query(
            `CREATE INDEX CONCURRENTLY post_names_idx ON post(name)`,
        )
    }

    async down(queryRunner: QueryRunner): Promise<void> {
        await queryRunner.query(`DROP INDEX CONCURRENTLY post_names_idx`)
    }
}
```

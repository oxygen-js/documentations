# Использование CLI

## Установка CLI

### Если файлы сущностей на JavaScript

Если у вас есть локальная версия typeorm, убедитесь, что она совпадает с глобальной, которую мы собираемся установить.

Установить typeorm глобально можно командой `npm i -g typeorm`.
Если вы предпочитаете обойтись без установки, для каждой команды можно использовать `npx typeorm <params>`.

### Если файлы сущностей на TypeScript

Этот инструмент CLI написан на JavaScript и запускается под node. Если файлы ваших сущностей написаны на TypeScript, перед использованием CLI их нужно скомпилировать в JavaScript. Если вы пользуетесь только JavaScript, этот раздел можно пропустить.

Чтобы упростить себе жизнь, настройте в проекте ts-node следующим образом.

Установите ts-node:

```shell
npm install ts-node --save-dev
```

Добавьте команду typeorm в раздел scripts файла package.json

```json
"scripts": {
    ...
    "typeorm": "typeorm-ts-node-commonjs"
}
```

Для ESM-проектов добавьте вместо этого:

```json
"scripts": {
    ...
    "typeorm": "typeorm-ts-node-esm"
}
```

Если вы хотите подгружать дополнительные модули, например [module-alias](https://github.com/ilearnio/module-alias), можно добавить ещё несколько `--require my-module-supporting-register`

После этого команду можно запускать так:

```shell
npm run typeorm migration:run -- -d path-to-datasource-config
```

### Как читать эту документацию?

Чтобы не загромождать документацию, в разделах ниже используется глобально установленный CLI typeorm. В зависимости от того, как вы установили CLI, замените `typeorm` в начале команды на `npx typeorm` или `npm run typeorm`.

## Инициализация нового проекта TypeORM

Вы можете создать новый проект, в котором уже всё настроено:

```shell
typeorm init
```

Команда создаёт все файлы, необходимые для базового проекта на TypeORM:

- .gitignore
- package.json
- README.md
- tsconfig.json
- src/entity/User.ts
- src/index.ts

Затем можно выполнить `npm install`, чтобы установить все зависимости.
После этого приложение запускается командой `npm start`.

Все файлы создаются в текущем каталоге.
Если вы хотите создать их в отдельном каталоге, используйте `--name`:

```shell
typeorm init --name my-project
```

Чтобы указать конкретную используемую базу данных, воспользуйтесь `--database`:

```shell
typeorm init --database mssql
```

Чтобы сгенерировать базовый ESM-проект, используйте `--module esm`:

```shell
typeorm init --name my-project --module esm
```

Можно также сгенерировать базовый проект с Express:

```shell
typeorm init --name my-project --express
```

Если вы используете docker, файл `docker-compose.yml` можно сгенерировать так:

```shell
typeorm init --docker
```

`typeorm init` — самый простой и быстрый способ настроить проект на TypeORM.

## Создание новой сущности

Создать новую сущность можно через CLI:

```shell
typeorm entity:create path-to-entity-dir/entity
```

Подробнее о [сущностях](<03. Entity/01. Entities.md>).

## Создание нового подписчика

Создать нового подписчика можно через CLI:

```shell
typeorm subscriber:create path-to-subscriber-dir/subscriber
```

Подробнее о [подписчиках](<11. Listeners and Subscribers.md>).

## Управление миграциями

- `typeorm migration:create` — [создать](<06. Migrations/03. Creating manually.md>) пустую миграцию
- `typeorm migration:generate` — [сгенерировать](<06. Migrations/04. Generating.md>) миграцию, сравнив сущности с фактической схемой базы данных
- `typeorm migration:run` — [выполнить](<06. Migrations/05. Executing and reverting.md>) все миграции
- `typeorm migration:revert` — [откатить](<06. Migrations/06. Reverting.md>) последнюю миграцию
- `typeorm migration:show` — [показать список](<06. Migrations/07. Status.md>) всех миграций с их статусом выполнения

Подробнее о [миграциях](<06. Migrations/01. How migrations work.md>).

## Синхронизация схемы базы данных

Чтобы синхронизировать схему базы данных, выполните:

```shell
typeorm schema:sync
```

Будьте осторожны с этой командой в продакшене — при неаккуратном использовании синхронизация схемы может привести к потере данных.
Прежде чем запускать её в продакшене, проверьте, какие SQL-запросы она выполнит.

## Вывод запросов синхронизации схемы без их выполнения

Чтобы посмотреть, какие SQL-запросы собирается выполнить `schema:sync`, используйте:

```shell
typeorm schema:log
```

## Удаление схемы базы данных

Чтобы полностью удалить схему базы данных, выполните:

```shell
typeorm schema:drop -- -d path-to-datasource-config
```

Будьте осторожны с этой командой в продакшене: она полностью удаляет данные из вашей базы.

## Выполнение произвольного SQL-запроса

Выполнить любой SQL-запрос напрямую в базе данных можно так:

```shell
typeorm query "SELECT * FROM USERS"
```

## Очистка кэша

Если вы используете кэширование в `QueryBuilder`, иногда бывает нужно очистить всё, что хранится в кэше.
Сделать это можно следующей командой:

```shell
typeorm cache:clear
```

## Проверка версии

Узнать, какая версия typeorm у вас установлена (локальная и глобальная), можно так:

```shell
typeorm version
```

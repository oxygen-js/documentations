# Индексы

## Индексы по колонке

Чтобы создать индекс в базе данных для конкретной колонки, примените к ней `@Index`.
Индексы можно создавать для любых колонок сущности.
Пример:

```typescript
import { Entity, PrimaryGeneratedColumn, Column, Index } from "typeorm"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Index()
    @Column()
    firstName: string

    @Column()
    @Index()
    lastName: string
}
```

Можно также задать имя индекса:

```typescript
import { Entity, PrimaryGeneratedColumn, Column, Index } from "typeorm"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Index("name1-idx")
    @Column()
    firstName: string

    @Column()
    @Index("name2-idx")
    lastName: string
}
```

## Уникальные индексы

Чтобы создать уникальный индекс, укажите `{ unique: true }` в опциях индекса:

> **Примечание**
>
> CockroachDB хранит уникальные индексы как ограничения `UNIQUE`

```typescript
import { Entity, PrimaryGeneratedColumn, Column, Index } from "typeorm"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Index({ unique: true })
    @Column()
    firstName: string

    @Column()
    @Index({ unique: true })
    lastName: string
}
```

## Индексы по нескольким колонкам

Чтобы создать индекс по нескольким колонкам, поместите `@Index` на саму сущность
и укажите имена всех свойств-колонок, которые должны войти в индекс.
Пример:

```typescript
import { Entity, PrimaryGeneratedColumn, Column, Index } from "typeorm"

@Entity()
@Index(["firstName", "lastName"])
@Index(["firstName", "middleName", "lastName"], { unique: true })
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    firstName: string

    @Column()
    middleName: string

    @Column()
    lastName: string
}
```

## Пространственные индексы

MySQL, CockroachDB и PostgreSQL (при наличии PostGIS) поддерживают пространственные индексы.

Чтобы создать пространственный индекс по колонке в MySQL, добавьте `Index` с `spatial: true` к колонке пространственного типа (`geometry`, `point`, `linestring`,
`polygon`, `multipoint`, `multilinestring`, `multipolygon`,
`geometrycollection`):

```typescript
@Entity()
export class Thing {
    @Column("point")
    @Index({ spatial: true })
    point: string
}
```

Чтобы создать пространственный индекс по колонке, добавьте `Index` с `spatial: true` к колонке пространственного типа (`geometry`, `geography`):

```typescript
export interface Geometry {
    type: "Point"
    coordinates: [Number, Number]
}

@Entity()
export class Thing {
    @Column("geometry", {
        spatialFeatureType: "Point",
        srid: 4326,
    })
    @Index({ spatial: true })
    point: Geometry
}
```

## Конкурентное создание

Чтобы не захватывать блокировку ACCESS EXCLUSIVE при создании и удалении индексов в Postgres, их можно создавать с модификатором CONCURRENTLY.
Если вы хотите использовать опцию concurrent, необходимо задать `migrationsTransactionMode: none` в опциях источника данных.

TypeORM умеет генерировать SQL с этой опцией, если она указана для индекса.

```typescript
@Index(["firstName", "middleName", "lastName"], { concurrent: true })
```

Подробнее см. [документацию Postgres](https://www.postgresql.org/docs/current/sql-createindex.html).

## Тип индекса

Если нужно указать для индекса особый тип, воспользуйтесь свойством `type`. Если задано свойство `spatial`, это поле игнорируется.

```typescript
@Index({ type: 'hash' })
```

Эта возможность сейчас поддерживается только для PostgreSQL.

## Отключение синхронизации

TypeORM не поддерживает некоторые опции и определения индексов (например, `lower`, `pg_trgm`) из-за множества различий между базами данных и многочисленных
проблем с получением информации о существующих индексах и их автоматической синхронизацией. В таких случаях индекс следует создавать вручную
(например, в [миграциях](<06. Migrations/01. How migrations work.md>)) с любой нужной вам сигнатурой. Чтобы TypeORM игнорировал такие индексы при синхронизации, используйте опцию
`synchronize: false` у декоратора `@Index`.

Например, вы создаёте индекс со сравнением без учёта регистра:

```sql
CREATE INDEX "POST_NAME_INDEX" ON "post" (lower("name"))
```

после этого следует отключить синхронизацию для этого индекса, чтобы он не был удалён при следующей синхронизации схемы:

```ts
@Entity()
@Index("POST_NAME_INDEX", { synchronize: false })
export class Post {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string
}
```

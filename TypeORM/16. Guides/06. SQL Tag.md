# SQL-тег

TypeORM позволяет писать SQL-запросы с помощью шаблонных строк, автоматически подставляя параметры в формате, соответствующем вашей СУБД. Это помогает избежать SQL-инъекций и делает запросы читаемее. SQL-тег реализован как обёртка над методом `.query` — он даёт альтернативный интерфейс, сохраняя то же поведение «под капотом».

## Базовое использование

Тег `sql` доступен у экземпляров DataSource, EntityManager, Repository и QueryRunner:

```typescript
const users = await dataSource.sql`SELECT * FROM users WHERE name = ${"John"}`
```

## Обработка параметров

Параметры автоматически экранируются и форматируются в соответствии с типом вашей СУБД:

- **PostgreSQL**, **CockroachDB**, **Aurora PostgreSQL** используют `$1`, `$2` и т. д.:

```typescript
// Запрос превращается в: SELECT * FROM users WHERE name = $1
const users = await dataSource.sql`SELECT * FROM users WHERE name = ${"John"}`
```

- **MySQL**, **MariaDB**, **Aurora MySQL**, **SAP**, **SQLite** используют `?`:

```typescript
// Запрос превращается в: SELECT * FROM users WHERE name = ?
const users = await dataSource.sql`SELECT * FROM users WHERE name = ${"John"}`
```

- **Oracle** использует `:1`, `:2` и т. д.:

```typescript
// Запрос превращается в: SELECT * FROM users WHERE name = :1
const users = await dataSource.sql`SELECT * FROM users WHERE name = ${"John"}`
```

- **MSSQL** использует `@1`, `@2` и т. д.:

```typescript
// Запрос превращается в: SELECT * FROM users WHERE name = @1
const users = await dataSource.sql`SELECT * FROM users WHERE name = ${"John"}`
```

### Несколько параметров

Можно использовать несколько параметров и сложные выражения:

```typescript
const name = "John"
const age = 30
const active = true
const users = await dataSource.sql`
    SELECT * FROM users
    WHERE name LIKE ${name + "%"}
    AND age > ${age}
    AND is_active = ${active}
`
```

### Развёртывание списков параметров

Чтобы превратить массив значений в динамический список параметров внутри шаблонного выражения, оберните массив в функцию. Обычно это нужно для выражения `IN (...)` в SQL, где каждое значение списка должно передаваться отдельным параметром:

```typescript
// Запрос превращается в: SELECT * FROM users WHERE id IN (?, ?, ?)
const users = await dataSource.sql`
    SELECT * FROM users
    WHERE id IN (${() => [1, 2, 3]})
`
```

### Подстановка неэкранированных выражений

Если шаблонное выражение _не_ должно превращаться в параметр запроса, оберните строку в функцию. Так можно динамически задавать имена колонок, таблиц или схем, которые нельзя параметризовать, либо условно добавлять части SQL-выражения.

**Осторожно!** Вставленный таким образом сырой SQL никак не экранируется. Использовать этот приём со значениями, полученными от пользователя, небезопасно.

```typescript
// Запрос превращается в: SELECT * FROM dynamic_table_name
const rawData = await dataSource.sql`
    SELECT * FROM ${() => "dynamic_table_name"}
`
```

## Возможности

- **Защита от SQL-инъекций**: параметры корректно экранируются
- **Независимость от СУБД**: формат параметров подбирается по типу вашей базы данных
- **Читаемость запросов**: шаблонные строки нередко читаются лучше, чем массивы параметров

## Сравнение с методом query

Традиционный метод `query` требует вручную расставлять плейсхолдеры параметров:

```typescript
// Традиционный метод query
await dataSource.query("SELECT * FROM users WHERE name = $1 AND age > $2", [
    "John",
    30,
])

// Альтернатива через SQL-тег
await dataSource.sql`SELECT * FROM users WHERE name = ${"John"} AND age > ${30}`
```

SQL-тег форматирует параметры автоматически, что снижает вероятность ошибок.

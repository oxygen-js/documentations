# Переход с Sequelize на TypeORM

## Настройка источника данных

В sequelize источник данных создаётся так:

```javascript
const sequelize = new Sequelize("database", "username", "password", {
    host: "localhost",
    dialect: "mysql",
})

sequelize
    .authenticate()
    .then(() => {
        console.log("Data Source has been initialized successfully.")
    })
    .catch((err) => {
        console.error("Error during Data Source initialization:", err)
    })
```

В TypeORM источник данных создаётся следующим образом:

```typescript
import { DataSource } from "typeorm"

const dataSource = new DataSource({
    type: "mysql",
    host: "localhost",
    username: "username",
    password: "password",
})

dataSource
    .initialize()
    .then(() => {
        console.log("Data Source has been initialized successfully.")
    })
    .catch((err) => {
        console.error("Error during Data Source initialization:", err)
    })
```

После этого экземпляр `dataSource` можно использовать в любом месте приложения.

Подробнее об [источнике данных](<../02. Data Source/01. DataSource.md>)

## Синхронизация схемы

В sequelize синхронизация схемы выполняется так:

```javascript
Project.sync({ force: true })
Task.sync({ force: true })
```

В TypeORM достаточно добавить `synchronize: true` в опции источника данных:

```typescript
const dataSource = new DataSource({
    type: "mysql",
    host: "localhost",
    username: "username",
    password: "password",
    synchronize: true,
})
```

## Создание моделей

Так модели описываются в sequelize:

```javascript
module.exports = function (sequelize, DataTypes) {
    const Project = sequelize.define("project", {
        title: DataTypes.STRING,
        description: DataTypes.TEXT,
    })

    return Project
}
```

```javascript
module.exports = function (sequelize, DataTypes) {
    const Task = sequelize.define("task", {
        title: DataTypes.STRING,
        description: DataTypes.TEXT,
        deadline: DataTypes.DATE,
    })

    return Task
}
```

В TypeORM такие модели называются сущностями и описываются следующим образом:

```typescript
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm"

@Entity()
export class Project {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    title: string

    @Column()
    description: string
}
```

```typescript
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm"

@Entity()
export class Task {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    title: string

    @Column("text")
    description: string

    @Column()
    deadline: Date
}
```

Настоятельно рекомендуется описывать по одному классу сущности на файл.
TypeORM позволяет использовать ваши классы как модели базы данных
и даёт декларативный способ указать, какая часть модели
станет частью таблицы в базе данных.
Возможности TypeScript дают подсказки типов и другие полезные средства, которые вы можете применять в классах.

Подробнее о [сущностях и колонках](<../03. Entity/01. Entities.md>)

## Прочие настройки моделей

Следующее в sequelize:

```javascript
flag: { type: Sequelize.BOOLEAN, allowNull: true, defaultValue: true },
```

В TypeORM достигается так:

```typescript
@Column({ nullable: true, default: true })
flag: boolean;
```

Следующее в sequelize:

```javascript
flag: { type: Sequelize.DATE, defaultValue: Sequelize.NOW }
```

В TypeORM записывается так:

```typescript
@Column({ default: () => "NOW()" })
myDate: Date;
```

Следующее в sequelize:

```javascript
someUnique: { type: Sequelize.STRING, unique: true },
```

В TypeORM достигается так:

```typescript
@Column({ unique: true })
someUnique: string;
```

Следующее в sequelize:

```javascript
fieldWithUnderscores: { type: Sequelize.STRING, field: "field_with_underscores" },
```

Переводится в TypeORM в это:

```typescript
@Column({ name: "field_with_underscores" })
fieldWithUnderscores: string;
```

Следующее в sequelize:

```javascript
incrementMe: { type: Sequelize.INTEGER, autoIncrement: true },
```

В TypeORM достигается так:

```typescript
@Column()
@Generated()
incrementMe: number;
```

Следующее в sequelize:

```javascript
identifier: { type: Sequelize.STRING, primaryKey: true },
```

В TypeORM достигается так:

```typescript
@Column({ primary: true })
identifier: string;
```

Чтобы получить колонки вроде `createDate` и `updateDate`, нужно описать в сущности две колонки (назвать их можно как угодно):

```typescript
@CreateDateColumn();
createDate: Date;

@UpdateDateColumn();
updateDate: Date;
```

### Работа с моделями

Чтобы создать и сохранить новую модель, в sequelize пишут:

```javascript
const employee = await Employee.create({
    name: "John Doe",
    title: "senior engineer",
})
```

В TypeORM создать и сохранить новую модель можно несколькими способами:

```typescript
const employee = new Employee() // можно использовать и параметры конструктора
employee.name = "John Doe"
employee.title = "senior engineer"
await dataSource.getRepository(Employee).save(employee)
```

или через паттерн active record

```typescript
const employee = Employee.create({ name: "John Doe", title: "senior engineer" })
await employee.save()
```

если нужно загрузить существующую сущность из базы данных и заменить часть её свойств, можно воспользоваться таким методом:

```typescript
const employee = await Employee.preload({ id: 1, name: "John Doe" })
```

Подробнее об [Active Record vs Data Mapper](<01. Active Record vs Data Mapper.md>) и [Repository API](<../07. Working with Entity Manager/06. Repository APIs.md>).

Чтобы обратиться к свойствам, в sequelize делают так:

```typescript
console.log(employee.get("name"))
```

В TypeORM это делается просто:

```typescript
console.log(employee.name)
```

Чтобы создать индекс, в sequelize пишут:

```typescript
sequelize.define(
    "user",
    {},
    {
        indexes: [
            {
                unique: true,
                fields: ["firstName", "lastName"],
            },
        ],
    },
)
```

В TypeORM это выглядит так:

```typescript
@Entity()
@Index(["firstName", "lastName"], { unique: true })
export class User {}
```

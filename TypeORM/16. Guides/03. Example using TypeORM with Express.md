# Пример использования TypeORM с Express

## Начальная настройка

Создадим простое приложение «user», которое хранит пользователей в базе данных
и позволяет создавать, обновлять и удалять их, а также получать список всех пользователей и одного пользователя по id
через веб-API.

Сначала создайте каталог «user»:

```shell
mkdir user
```

Затем перейдите в него и создайте новый проект:

```shell
cd user
npm init
```

Завершите процесс инициализации, заполнив всю необходимую информацию о приложении.

Теперь нужно установить и настроить компилятор TypeScript. Сначала установим его:

```shell
npm i typescript --save-dev
```

Затем создадим файл `tsconfig.json` с конфигурацией, необходимой для компиляции и запуска
приложения. Создайте его в любимом редакторе и поместите туда следующую конфигурацию:

```json
{
    "compilerOptions": {
        "lib": ["es5", "es6", "dom"],
        "target": "es5",
        "module": "commonjs",
        "moduleResolution": "node",
        "emitDecoratorMetadata": true,
        "experimentalDecorators": true
    }
}
```

Теперь создадим главную точку входа приложения — `app.ts` внутри каталога `src`:

```shell
mkdir src
cd src
touch app.ts
```

Добавим внутрь простой `console.log`:

```typescript
console.log("Application is up and running")
```

Пришло время запустить наше приложение.
Чтобы его запустить, сначала нужно скомпилировать TypeScript-проект:

```shell
tsc
```

После компиляции у вас должен появиться сгенерированный файл `src/app.js`.
Запустить его можно так:

```shell
node src/app.js
```

Сразу после запуска приложения в консоли должно появиться сообщение «Application is up and running».

Файлы нужно перекомпилировать после каждого изменения.
Как вариант, можно настроить watcher или установить [ts-node](https://github.com/TypeStrong/ts-node), чтобы не компилировать вручную каждый раз.

## Добавление Express в приложение

Добавим в наше приложение Express. Сначала установим нужные пакеты:

```shell
npm install express
npm install @types/express --save-dev
```

- `express` — сам движок express. Он позволяет создать веб-API
- `@types/express` — нужен для получения информации о типах при работе с express

Отредактируем файл `src/app.ts` и добавим логику, связанную с express:

```typescript
import * as express from "express"
import { Request, Response } from "express"

// создаём и настраиваем express-приложение
const app = express()
app.use(express.json())

// регистрируем маршруты

app.get("/users", function (req: Request, res: Response) {
    // здесь будет логика возврата всех пользователей
})

app.get("/users/:id", function (req: Request, res: Response) {
    // здесь будет логика возврата пользователя по id
})

app.post("/users", function (req: Request, res: Response) {
    // здесь будет логика сохранения пользователя
})

app.put("/users/:id", function (req: Request, res: Response) {
    // здесь будет логика обновления пользователя по заданному id
})

app.delete("/users/:id", function (req: Request, res: Response) {
    // здесь будет логика удаления пользователя по заданному id
})

// запускаем express-сервер
app.listen(3000)
```

Теперь можно скомпилировать и запустить проект.
У вас должен работать express-сервер с рабочими маршрутами.
Правда, эти маршруты пока не возвращают никакого содержимого.

## Добавление TypeORM в приложение

Наконец, добавим в приложение TypeORM.
В этом примере мы будем использовать драйвер `mysql`.
Для остальных драйверов настройка выполняется аналогично.

Сначала установим необходимые пакеты:

```shell
npm install typeorm reflect-metadata mysql
```

- `typeorm` — сам пакет typeorm
- `reflect-metadata` — необходим для корректной работы декораторов. Не забудьте импортировать его до вашего кода TypeORM.
- `mysql` — драйвер базы данных нижнего уровня. Если вы используете другую СУБД, нужно установить соответствующий пакет

Создадим файл `app-data-source.ts`, в котором зададим начальные параметры подключения к базе данных:

```ts
import { DataSource } from "typeorm"

export const myDataSource = new DataSource({
    type: "mysql",
    host: "localhost",
    port: 3306,
    username: "test",
    password: "test",
    database: "test",
    entities: [__dirname + "/entities/**/*{.js,.ts}"],
    logging: true,
    synchronize: true,
})
```

Настройте каждую опцию под свои нужды.
Подробнее об опциях читайте [здесь](<../02. Data Source/02. Data Source Options.md>).

Создадим сущность `user.entity.ts` в папке `entities`:

```typescript
import { Entity, Column, PrimaryGeneratedColumn } from "typeorm"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    firstName: string

    @Column()
    lastName: string
}
```

Изменим `src/app.ts`, чтобы установить соединение с базой данных и начать использовать `myDataSource`:

```typescript
import "reflect-metadata"

import * as express from "express"
import { Request, Response } from "express"
import { User } from "./entities/User"
import { myDataSource } from "./app-data-source.ts"

// устанавливаем соединение с базой данных
try {
    await myDataSource.initialize()
    console.log("Data Source has been initialized!")
} catch (error) {
    console.error("Error during Data Source initialization:", error)
}

// создаём и настраиваем express-приложение
const app = express()
app.use(express.json())

// регистрируем маршруты
app.get("/users", async function (req: Request, res: Response) {
    const users = await myDataSource.getRepository(User).find()
    res.json(users)
})

app.get("/users/:id", async function (req: Request, res: Response) {
    const results = await myDataSource.getRepository(User).findOneBy({
        id: req.params.id,
    })
    return res.send(results)
})

app.post("/users", async function (req: Request, res: Response) {
    const user = await myDataSource.getRepository(User).create(req.body)
    const results = await myDataSource.getRepository(User).save(user)
    return res.send(results)
})

app.put("/users/:id", async function (req: Request, res: Response) {
    const user = await myDataSource.getRepository(User).findOneBy({
        id: req.params.id,
    })
    myDataSource.getRepository(User).merge(user, req.body)
    const results = await myDataSource.getRepository(User).save(user)
    return res.send(results)
})

app.delete("/users/:id", async function (req: Request, res: Response) {
    const results = await myDataSource.getRepository(User).delete(req.params.id)
    return res.send(results)
})

// запускаем express-сервер
app.listen(3000)
```

Теперь у вас должно быть работающее базовое express-приложение, подключённое к базе данных MySQL.

# Логирование

## Включение логирования

Включить логирование всех запросов и ошибок можно, просто указав `logging: true` в опциях источника данных:

```typescript
{
    name: "mysql",
    type: "mysql",
    host: "localhost",
    port: 3306,
    username: "test",
    password: "test",
    database: "test",
    ...
    logging: true
}
```

## Опции логирования

В опциях источника данных можно включить разные виды логирования:

```typescript
{
    host: "localhost",
    ...
    logging: ["query", "error"]
}
```

Если нужно логировать только неудачные запросы, укажите только `error`:

```typescript
{
    host: "localhost",
    ...
    logging: ["error"]
}
```

Доступны и другие значения:

- `query` — логирует все запросы.
- `error` — логирует все неудачные запросы и ошибки.
- `schema` — логирует процесс построения схемы.
- `warn` — логирует внутренние предупреждения ORM.
- `info` — логирует внутренние информационные сообщения ORM.
- `log` — логирует внутренние сообщения журнала ORM.

Указывать можно сколько угодно значений.
Если нужно включить всё логирование, достаточно указать `logging: "all"`:

```typescript
{
    host: "localhost",
    ...
    logging: "all"
}
```

## Логирование долгих запросов

Если у вас есть проблемы с производительностью, можно логировать запросы, выполняющиеся слишком долго,
задав `maxQueryExecutionTime` в опциях источника данных:

```typescript
{
    host: "localhost",
    ...
    maxQueryExecutionTime: 1000
}
```

Этот код будет логировать все запросы, выполняющиеся дольше `1 секунды`.

## Замена логгера по умолчанию

TypeORM поставляется с 5 встроенными логгерами:

- `advanced-console` — логгер по умолчанию, выводящий все сообщения в консоль с цветовой подсветкой и подсветкой синтаксиса SQL.
- `simple-console` — простой консольный логгер, полностью аналогичный advanced-логгеру, но без цветовой подсветки. Его можно использовать, если с цветным выводом возникают проблемы или он вам просто не нравится.
- `formatted-console` — почти то же самое, что advanced-логгер, но форматирует SQL-запросы для удобочитаемости (с помощью [@sqltools/formatter](https://github.com/mtxr/vscode-sqltools)).
- `file` — этот логгер пишет все логи в указанный файл (или в `ormlogs.log` в текущем рабочем каталоге, если путь не задан).
- `debug` — этот логгер использует [пакет debug](https://github.com/debug-js/debug); чтобы включить логирование, задайте переменную окружения `DEBUG=typeorm:*` (учтите: логгер передаёт все логи пакету `debug`, который и управляет уровнем логирования).

Любой из них можно включить в опциях источника данных:

```typescript
{
    host: "localhost",
    ...
    logging: true,
    logger: "file"
}
```

## Использование собственного логгера

Вы можете создать свой класс логгера, реализовав интерфейс `Logger`:

```typescript
import { Logger } from "typeorm"

export class MyCustomLogger implements Logger {
    // реализуйте все методы класса logger
}
```

Либо унаследоваться от класса `AbstractLogger`:

```typescript
import { AbstractLogger } from "typeorm"

export class MyCustomLogger extends AbstractLogger {
    /**
     * Записать лог в нужный вывод.
     */
    protected writeLog(
        level: LogLevel,
        logMessage: LogMessage | LogMessage[],
        queryRunner?: QueryRunner,
    ) {
        const messages = this.prepareLogMessages(
            logMessage,
            {
                highlightSql: false,
            },
            queryRunner,
        )

        for (let message of messages) {
            switch (message.type ?? level) {
                case "log":
                case "schema-build":
                case "migration":
                    console.log(message.message)
                    break

                case "info":
                case "query":
                    if (message.prefix) {
                        console.info(message.prefix, message.message)
                    } else {
                        console.info(message.message)
                    }
                    break

                case "warn":
                case "query-slow":
                    if (message.prefix) {
                        console.warn(message.prefix, message.message)
                    } else {
                        console.warn(message.message)
                    }
                    break

                case "error":
                case "query-error":
                    if (message.prefix) {
                        console.error(message.prefix, message.message)
                    } else {
                        console.error(message.message)
                    }
                    break
            }
        }
    }
}
```

И указать его в опциях источника данных:

```typescript
import { DataSource } from "typeorm"
import { MyCustomLogger } from "./logger/MyCustomLogger"

const dataSource = new DataSource({
    name: "mysql",
    type: "mysql",
    host: "localhost",
    port: 3306,
    username: "test",
    password: "test",
    database: "test",
    logger: new MyCustomLogger(),
})
```

Методы логгера могут принимать `QueryRunner`, если он доступен. Это удобно, когда нужно логировать дополнительные данные.
Кроме того, через query runner можно получить доступ к дополнительным данным, переданным при сохранении/удалении. Например:

```typescript
// пользователь отправляет запрос во время сохранения сущности
postRepository.save(post, { data: { request: request } });

// в логгере получить их можно так:
logQuery(query: string, parameters?: any[], queryRunner?: QueryRunner) {
    const requestUrl = queryRunner && queryRunner.data["request"] ? "(" + queryRunner.data["request"].url + ") " : "";
    console.log(requestUrl + "executing query: " + query);
}
```

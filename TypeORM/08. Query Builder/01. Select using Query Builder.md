# Выборка с помощью Query Builder

## Что такое QueryBuilder?

`QueryBuilder` — одна из самых мощных возможностей TypeORM:
он позволяет строить SQL-запросы с помощью элегантного и удобного синтаксиса,
выполнять их и получать автоматически преобразованные сущности.

Простой пример `QueryBuilder`:

```typescript
const firstUser = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .where("user.id = :id", { id: 1 })
    .getOne()
```

Он строит следующий SQL-запрос:

```sql
SELECT
    user.id as userId,
    user.firstName as userFirstName,
    user.lastName as userLastName
FROM users user
WHERE user.id = 1
```

и возвращает вам экземпляр `User`:

```javascript
User {
    id: 1,
    firstName: "Timber",
    lastName: "Saw"
}
```

## Важное замечание при использовании `QueryBuilder`

При использовании `QueryBuilder` нужно задавать уникальные имена параметров в выражениях `WHERE`. **Так работать не будет**:

```typescript
const result = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.linkedSheep", "linkedSheep")
    .leftJoinAndSelect("user.linkedCow", "linkedCow")
    .where("user.linkedSheep = :id", { id: sheepId })
    .andWhere("user.linkedCow = :id", { id: cowId })
```

...а так будет:

```typescript
const result = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.linkedSheep", "linkedSheep")
    .leftJoinAndSelect("user.linkedCow", "linkedCow")
    .where("user.linkedSheep = :sheepId", { sheepId })
    .andWhere("user.linkedCow = :cowId", { cowId })
```

Обратите внимание: мы дали параметрам уникальные имена `:sheepId` и `:cowId` вместо того, чтобы дважды использовать `:id` для разных значений.

## Как создать и использовать QueryBuilder?

Создать `Query Builder` можно несколькими способами:

- Через DataSource:

    ```typescript
    const user = await dataSource
        .createQueryBuilder()
        .select("user")
        .from(User, "user")
        .where("user.id = :id", { id: 1 })
        .getOne()
    ```

- Через entity manager:

    ```typescript
    const user = await dataSource.manager
        .createQueryBuilder(User, "user")
        .where("user.id = :id", { id: 1 })
        .getOne()
    ```

- Через репозиторий:

    ```typescript
    const user = await dataSource
        .getRepository(User)
        .createQueryBuilder("user")
        .where("user.id = :id", { id: 1 })
        .getOne()
    ```

Доступно 5 разных типов `QueryBuilder`:

- `SelectQueryBuilder` — для построения и выполнения запросов `SELECT`. Пример:

    ```typescript
    const user = await dataSource
        .createQueryBuilder()
        .select("user")
        .from(User, "user")
        .where("user.id = :id", { id: 1 })
        .getOne()
    ```

- `InsertQueryBuilder` — для построения и выполнения запросов `INSERT`. Пример:

    ```typescript
    await dataSource
        .createQueryBuilder()
        .insert()
        .into(User)
        .values([
            { firstName: "Timber", lastName: "Saw" },
            { firstName: "Phantom", lastName: "Lancer" },
        ])
        .execute()
    ```

- `UpdateQueryBuilder` — для построения и выполнения запросов `UPDATE`. Пример:

    ```typescript
    await dataSource
        .createQueryBuilder()
        .update(User)
        .set({ firstName: "Timber", lastName: "Saw" })
        .where("id = :id", { id: 1 })
        .execute()
    ```

- `DeleteQueryBuilder` — для построения и выполнения запросов `DELETE`. Пример:

    ```typescript
    await dataSource
        .createQueryBuilder()
        .delete()
        .from(User)
        .where("id = :id", { id: 1 })
        .execute()
    ```

- `RelationQueryBuilder` — для построения и выполнения операций, специфичных для связей [TBD].
  Пример:

    ```typescript
    await dataSource
        .createQueryBuilder()
        .relation(User, "photos")
        .of(id)
        .loadMany()
    ```

Вы можете переключаться между разными типами query builder внутри любого из них;
как только вы это сделаете, вы получите новый экземпляр query builder (в отличие от всех остальных методов).

## Получение значений с помощью `QueryBuilder`

Чтобы получить из базы данных единственный результат —
например, пользователя по id или имени, — используйте `getOne`:

```typescript
const timber = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .where("user.id = :id OR user.name = :name", { id: 1, name: "Timber" })
    .getOne()
```

`getOneOrFail` тоже вернёт из базы данных единственный результат, но если
результата нет — выбросит `EntityNotFoundError`:

```typescript
const timber = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .where("user.id = :id OR user.name = :name", { id: 1, name: "Timber" })
    .getOneOrFail()
```

Чтобы получить из базы данных несколько результатов —
например, всех пользователей, — используйте `getMany`:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .getMany()
```

С помощью select query builder можно получить результаты двух видов: **сущности** или **сырые результаты**.
Чаще всего вам нужно выбрать из базы данных настоящие сущности, например пользователей.
Для этого используются `getOne` и `getMany`.
Но иногда требуется выбрать какие-то конкретные данные — скажем, _сумму всех фотографий пользователя_.
Такие данные не являются сущностью, они называются сырыми данными.
Для получения сырых данных используются `getRawOne` и `getRawMany`.
Примеры:

```typescript
const { sum } = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .select("SUM(user.photosCount)", "sum")
    .where("user.id = :id", { id: 1 })
    .getRawOne()
```

```typescript
const photosSums = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .select("user.id")
    .addSelect("SUM(user.photosCount)", "sum")
    .groupBy("user.id")
    .getRawMany()

// результат будет примерно таким: [{ id: 1, sum: 25 }, { id: 2, sum: 13 }, ...]
```

## Получение количества

Узнать количество строк, которые вернёт запрос, можно с помощью `getCount()`. Он вернёт число, а не результат в виде сущностей.

```typescript
const count = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .where("user.name = :name", { name: "Timber" })
    .getCount()
```

Это даёт следующий SQL-запрос:

```sql
SELECT count(*) FROM users user WHERE user.name = 'Timber'
```

## Зачем нужны алиасы?

Мы использовали `createQueryBuilder("user")`. Но что такое «user»?
Это обычный SQL-алиас.
Мы используем алиасы повсюду, кроме работы с выбранными данными.

`createQueryBuilder("user")` эквивалентно:

```typescript
createQueryBuilder().select("user").from(User, "user")
```

и приведёт к следующему SQL-запросу:

```sql
SELECT ... FROM users user
```

В этом SQL-запросе `users` — имя таблицы, а `user` — алиас, который мы ей назначили.
Далее мы используем этот алиас для обращения к таблице:

```typescript
createQueryBuilder()
    .select("user")
    .from(User, "user")
    .where("user.name = :name", { name: "Timber" })
```

Это даёт следующий SQL-запрос:

```sql
SELECT ... FROM users user WHERE user.name = 'Timber'
```

Видите: мы обратились к таблице users через алиас `user`, назначенный при создании query builder.

Один query builder не ограничен одним алиасом — их может быть несколько.
У каждой выборки может быть свой алиас,
вы можете выбирать из нескольких таблиц, каждая со своим алиасом,
можете присоединять несколько таблиц, каждая со своим алиасом.
Эти алиасы используются для обращения к таблицам (или данным), которые вы выбираете.

## Использование параметров для экранирования данных

Мы использовали `where("user.name = :name", { name: "Timber" })`.
Что означает `{ name: "Timber" }`? Это параметр, который мы применили, чтобы предотвратить SQL-инъекцию.
Можно было бы написать `where("user.name = '" + name + "')`,
однако это небезопасно: код становится уязвим для SQL-инъекций.
Безопасный способ — использовать специальный синтаксис: `where("user.name = :name", { name: "Timber" })`,
где `:name` — имя параметра, а значение задаётся в объекте: `{ name: "Timber" }`.

```typescript
.where("user.name = :name", { name: "Timber" })
```

— это сокращение для:

```typescript
.where("user.name = :name")
.setParameter("name", "Timber")
```

Примечание: не используйте одно и то же имя параметра для разных значений в пределах query builder. Если задать его несколько раз, значения будут перезаписаны.

Также можно передать массив значений и превратить его в список значений в SQL-выражении
с помощью специального синтаксиса развёртывания:

```typescript
.where("user.name IN (:...names)", { names: [ "Timber", "Crystal", "Lina" ] })
```

Что превратится в:

```sql
WHERE user.name IN ('Timber', 'Crystal', 'Lina')
```

## Добавление выражения `WHERE`

Добавить выражение `WHERE` очень просто:

```typescript
createQueryBuilder("user").where("user.name = :name", { name: "Timber" })
```

Это даст:

```sql
SELECT ... FROM users user WHERE user.name = 'Timber'
```

К существующему выражению `WHERE` можно добавить `AND`:

```typescript
createQueryBuilder("user")
    .where("user.firstName = :firstName", { firstName: "Timber" })
    .andWhere("user.lastName = :lastName", { lastName: "Saw" })
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user WHERE user.firstName = 'Timber' AND user.lastName = 'Saw'
```

К существующему выражению `WHERE` можно добавить `OR`:

```typescript
createQueryBuilder("user")
    .where("user.firstName = :firstName", { firstName: "Timber" })
    .orWhere("user.lastName = :lastName", { lastName: "Saw" })
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user WHERE user.firstName = 'Timber' OR user.lastName = 'Saw'
```

В выражении `WHERE` можно выполнить запрос с `IN`:

```typescript
createQueryBuilder("user").where("user.id IN (:...ids)", { ids: [1, 2, 3, 4] })
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user WHERE user.id IN (1, 2, 3, 4)
```

Сложное выражение `WHERE` можно добавить к существующему `WHERE` с помощью `Brackets`

```typescript
createQueryBuilder("user")
    .where("user.registered = :registered", { registered: true })
    .andWhere(
        new Brackets((qb) => {
            qb.where("user.firstName = :firstName", {
                firstName: "Timber",
            }).orWhere("user.lastName = :lastName", { lastName: "Saw" })
        }),
    )
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user WHERE user.registered = true AND (user.firstName = 'Timber' OR user.lastName = 'Saw')
```

Отрицание сложного выражения `WHERE` можно добавить к существующему `WHERE` с помощью `NotBrackets`

```typescript
createQueryBuilder("user")
    .where("user.registered = :registered", { registered: true })
    .andWhere(
        new NotBrackets((qb) => {
            qb.where("user.firstName = :firstName", {
                firstName: "Timber",
            }).orWhere("user.lastName = :lastName", { lastName: "Saw" })
        }),
    )
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user WHERE user.registered = true AND NOT((user.firstName = 'Timber' OR user.lastName = 'Saw'))
```

Вы можете комбинировать сколько угодно выражений `AND` и `OR`.
Если использовать `.where` больше одного раза, все предыдущие выражения `WHERE` будут перезаписаны.

Примечание: будьте осторожны с `orWhere` — если вы строите сложные выражения одновременно с `AND` и `OR`,
помните, что они складываются без каких-либо ухищрений.
Иногда вместо `orWhere` лучше составить строку условия целиком.

## Добавление выражения `HAVING`

Добавить выражение `HAVING` очень просто:

```typescript
createQueryBuilder("user").having("user.name = :name", { name: "Timber" })
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user HAVING user.name = 'Timber'
```

К существующему выражению `HAVING` можно добавить `AND`:

```typescript
createQueryBuilder("user")
    .having("user.firstName = :firstName", { firstName: "Timber" })
    .andHaving("user.lastName = :lastName", { lastName: "Saw" })
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user HAVING user.firstName = 'Timber' AND user.lastName = 'Saw'
```

К существующему выражению `HAVING` можно добавить `OR`:

```typescript
createQueryBuilder("user")
    .having("user.firstName = :firstName", { firstName: "Timber" })
    .orHaving("user.lastName = :lastName", { lastName: "Saw" })
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user HAVING user.firstName = 'Timber' OR user.lastName = 'Saw'
```

Вы можете комбинировать сколько угодно выражений `AND` и `OR`.
Если использовать `.having` больше одного раза, все предыдущие выражения `HAVING` будут перезаписаны.

## Добавление выражения `ORDER BY`

Добавить выражение `ORDER BY` очень просто:

```typescript
createQueryBuilder("user").orderBy("user.id")
```

Это даст:

```sql
SELECT ... FROM users user ORDER BY user.id
```

Направление сортировки можно изменить с возрастающего на убывающее (и наоборот):

```typescript
createQueryBuilder("user").orderBy("user.id", "DESC")

createQueryBuilder("user").orderBy("user.id", "ASC")
```

Можно задать несколько критериев сортировки:

```typescript
createQueryBuilder("user").orderBy("user.name").addOrderBy("user.id")
```

Также можно использовать объект с полями сортировки:

```typescript
createQueryBuilder("user").orderBy({
    "user.name": "ASC",
    "user.id": "DESC",
})
```

Если использовать `.orderBy` больше одного раза, все предыдущие выражения `ORDER BY` будут перезаписаны.

## Добавление выражения `DISTINCT ON` (только Postgres)

Если distinct-on используется вместе с выражением order-by, выражение distinct-on должно совпадать с самым левым выражением order-by.
Выражения distinct-on интерпретируются по тем же правилам, что и order-by. Учтите: использование distinct-on без order-by означает, что первая строка каждого набора будет непредсказуемой.

Добавить выражение `DISTINCT ON` очень просто:

```typescript
createQueryBuilder("user").distinctOn(["user.id"]).orderBy("user.id")
```

Это даст:

```sql
SELECT DISTINCT ON (user.id) ... FROM users user ORDER BY user.id
```

## Добавление выражения `GROUP BY`

Добавить выражение `GROUP BY` очень просто:

```typescript
createQueryBuilder("user").groupBy("user.id")
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user GROUP BY user.id
```

Чтобы добавить дополнительные критерии группировки, используйте `addGroupBy`:

```typescript
createQueryBuilder("user").groupBy("user.name").addGroupBy("user.id")
```

Если использовать `.groupBy` больше одного раза, все предыдущие выражения `GROUP BY` будут перезаписаны.

## Добавление выражения `LIMIT`

Добавить выражение `LIMIT` очень просто:

```typescript
createQueryBuilder("user").limit(10)
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user LIMIT 10
```

Итоговый SQL-запрос зависит от типа базы данных (SQL, mySQL, Postgres и т. д.).
Примечание: LIMIT может работать не так, как вы ожидаете, если вы используете сложные запросы с join-ами или подзапросами.
Если вам нужна постраничная выборка, рекомендуется использовать `take`.

## Добавление выражения `OFFSET`

Добавить SQL-выражение `OFFSET` очень просто:

```typescript
createQueryBuilder("user").offset(10)
```

Это даст следующий SQL-запрос:

```sql
SELECT ... FROM users user OFFSET 10
```

Итоговый SQL-запрос зависит от типа базы данных (SQL, mySQL, Postgres и т. д.).
Примечание: OFFSET может работать не так, как вы ожидаете, если вы используете сложные запросы с join-ами или подзапросами.
Если вам нужна постраничная выборка, рекомендуется использовать `skip`.

## Присоединение связей

Допустим, у вас есть следующие сущности:

```typescript
import { Entity, PrimaryGeneratedColumn, Column, OneToMany } from "typeorm"
import { Photo } from "./Photo"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @OneToMany((type) => Photo, (photo) => photo.user)
    photos: Photo[]
}
```

```typescript
import { Entity, PrimaryGeneratedColumn, Column, ManyToOne } from "typeorm"
import { User } from "./User"

@Entity()
export class Photo {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    url: string

    @ManyToOne((type) => User, (user) => user.photos)
    user: User
}
```

Теперь допустим, вы хотите загрузить пользователя «Timber» вместе со всеми его фотографиями:

```typescript
const user = await dataSource
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.photos", "photo")
    .where("user.name = :name", { name: "Timber" })
    .getOne()
```

Вы получите следующий результат:

```typescript
{
    id: 1,
    name: "Timber",
    photos: [{
        id: 1,
        url: "me-with-chakram.jpg"
    }, {
        id: 2,
        url: "me-with-trees.jpg"
    }]
}
```

Как видите, `leftJoinAndSelect` автоматически загрузил все фотографии Timber.
Первый аргумент — связь, которую вы хотите загрузить, второй — алиас, назначаемый таблице этой связи.
Этот алиас можно использовать в любом месте query builder.
Например, возьмём все неудалённые фотографии Timber.

```typescript
const user = await dataSource
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.photos", "photo")
    .where("user.name = :name", { name: "Timber" })
    .andWhere("photo.isRemoved = :isRemoved", { isRemoved: false })
    .getOne()
```

Это сгенерирует следующий SQL-запрос:

```sql
SELECT user.*, photo.* FROM users user
    LEFT JOIN photos photo ON photo.user = user.id
    WHERE user.name = 'Timber' AND photo.isRemoved = FALSE
```

Условия можно добавить и в само выражение join вместо использования «where»:

```typescript
const user = await dataSource
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.photos", "photo", "photo.isRemoved = :isRemoved", {
        isRemoved: false,
    })
    .where("user.name = :name", { name: "Timber" })
    .getOne()
```

Это сгенерирует следующий SQL-запрос:

```sql
SELECT user.*, photo.* FROM users user
    LEFT JOIN photos photo ON photo.user = user.id AND photo.isRemoved = FALSE
    WHERE user.name = 'Timber'
```

## Inner и left join

Если вместо `LEFT JOIN` вам нужен `INNER JOIN`, просто используйте `innerJoinAndSelect`:

```typescript
const user = await dataSource
    .createQueryBuilder("user")
    .innerJoinAndSelect(
        "user.photos",
        "photo",
        "photo.isRemoved = :isRemoved",
        { isRemoved: false },
    )
    .where("user.name = :name", { name: "Timber" })
    .getOne()
```

Это сгенерирует:

```sql
SELECT user.*, photo.* FROM users user
    INNER JOIN photos photo ON photo.user = user.id AND photo.isRemoved = FALSE
    WHERE user.name = 'Timber'
```

Разница между `LEFT JOIN` и `INNER JOIN` в том, что `INNER JOIN` не вернёт пользователя, если у него нет ни одной фотографии.
`LEFT JOIN` вернёт пользователя даже без фотографий.
Подробнее о разных типах join читайте в [документации по SQL](https://msdn.microsoft.com/en-us/library/zt8wzxy4.aspx).

## Join без выборки

Данные можно присоединить, не выбирая их.
Для этого используйте `leftJoin` или `innerJoin`:

```typescript
const user = await dataSource
    .createQueryBuilder("user")
    .innerJoin("user.photos", "photo")
    .where("user.name = :name", { name: "Timber" })
    .getOne()
```

Это сгенерирует:

```sql
SELECT user.* FROM users user
    INNER JOIN photos photo ON photo.user = user.id
    WHERE user.name = 'Timber'
```

Такой запрос выберет Timber, если у него есть фотографии, но самих фотографий не вернёт.

## Присоединение любой сущности или таблицы

Присоединять можно не только связи, но и другие, никак не связанные сущности или таблицы.
Примеры:

```typescript
const user = await dataSource
    .createQueryBuilder("user")
    .leftJoinAndSelect(Photo, "photo", "photo.userId = user.id")
    .getMany()
```

```typescript
const user = await dataSource
    .createQueryBuilder("user")
    .leftJoinAndSelect("photos", "photo", "photo.userId = user.id")
    .getMany()
```

## Join с маппингом

Добавьте `profilePhoto` в сущность `User` — и с помощью `QueryBuilder` вы сможете отобразить в это свойство любые данные:

```typescript
export class User {
    /// ...
    profilePhoto: Photo
}
```

```typescript
const user = await dataSource
    .createQueryBuilder("user")
    .leftJoinAndMapOne(
        "user.profilePhoto",
        "user.photos",
        "photo",
        "photo.isForProfile = TRUE",
    )
    .where("user.name = :name", { name: "Timber" })
    .getOne()
```

Это загрузит фотографию профиля Timber и запишет её в `user.profilePhoto`.
Если нужно загрузить и отобразить одну сущность, используйте `leftJoinAndMapOne`.
Если нужно загрузить и отобразить несколько сущностей, используйте `leftJoinAndMapMany`.

## Получение сгенерированного запроса

Иногда бывает нужно получить SQL-запрос, сгенерированный `QueryBuilder`.
Для этого используйте `getSql`:

```typescript
const sql = dataSource
    .createQueryBuilder("user")
    .where("user.firstName = :firstName", { firstName: "Timber" })
    .orWhere("user.lastName = :lastName", { lastName: "Saw" })
    .getSql()
```

Для отладки сгенерированный SQL можно посмотреть через `getSql()` или `getQuery()`:

```typescript
const qb = dataSource
    .createQueryBuilder("user")
    .where("user.firstName = :firstName", { firstName: "Timber" })
    .orWhere("user.lastName = :lastName", { lastName: "Saw" })

console.log(qb.getSql())
// SELECT ... WHERE "user"."firstName" = $1 OR "user"."lastName" = $2

const users = await qb.getMany()
```

`getSql()` возвращает SQL с плейсхолдерами параметров. `getQuery()` возвращает ту же SQL-строку. Чтобы увидеть ещё и значения параметров, используйте `getQueryAndParameters()` — он возвращает кортеж `[sql, parameters]`.

Все выполненные запросы также автоматически записываются в настроенный логгер, если включить логирование запросов в опциях DataSource:

```typescript
new DataSource({
    // ...
    logging: ["query"],
})
```

## Получение сырых результатов

С помощью select query builder можно получить результаты двух видов: **сущности** и **сырые результаты**.
Чаще всего вам нужно выбрать из базы данных настоящие сущности, например пользователей.
Для этого используются `getOne` и `getMany`.
Однако иногда требуется выбрать конкретные данные — например, _сумму всех фотографий пользователя_.
Такие данные не являются сущностью, они называются сырыми данными.
Для получения сырых данных используются `getRawOne` и `getRawMany`.
Примеры:

```typescript
const { sum } = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .select("SUM(user.photosCount)", "sum")
    .where("user.id = :id", { id: 1 })
    .getRawOne()
```

```typescript
const photosSums = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .select("user.id")
    .addSelect("SUM(user.photosCount)", "sum")
    .groupBy("user.id")
    .getRawMany()

// результат будет примерно таким: [{ id: 1, sum: 25 }, { id: 2, sum: 13 }, ...]
```

## Потоковая передача результатов

Вы можете использовать `stream`, который возвращает поток.
Потоковая передача возвращает сырые данные, и преобразованием в сущности вам придётся заниматься вручную:

```typescript
const stream = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .where("user.id = :id", { id: 1 })
    .stream()
```

## Постраничная выборка

При разработке приложения постраничная выборка нужна почти всегда.
Она используется, если в приложении есть пагинация, слайдер страниц или бесконечная прокрутка.

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.photos", "photo")
    .take(10)
    .getMany()
```

Это даст первых 10 пользователей с их фотографиями.

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.photos", "photo")
    .skip(10)
    .getMany()
```

Это даст всех пользователей с их фотографиями, кроме первых 10.
Эти методы можно комбинировать:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.photos", "photo")
    .skip(5)
    .take(10)
    .getMany()
```

Это пропустит первых 5 пользователей и возьмёт следующих 10.

Может показаться, что `take` и `skip` — это то же самое, что `limit` и `offset`, но это не так.
`limit` и `offset` могут работать не так, как вы ожидаете, если запросы усложняются join-ами или подзапросами.
Использование `take` и `skip` избавит вас от этих проблем.

## Установка блокировок

QueryBuilder поддерживает как оптимистичные, так и пессимистичные блокировки.

### Режимы блокировки

Поддержка режимов блокировки и SQL-выражения, в которые они преобразуются, приведены в таблице ниже (пустая ячейка означает отсутствие поддержки). Если указанный режим блокировки не поддерживается, будет выброшена ошибка `LockNotSupportedOnGivenDriverError`.

|                       | pessimistic_read                             | pessimistic_write         | dirty_read      | for_no_key_update   | for_key_share   |
| --------------------- | -------------------------------------------- | ------------------------- | --------------- | ------------------- | --------------- |
| MySQL, MariaDB        | `FOR SHARE` (MySQL 8+), `LOCK IN SHARE MODE` | `FOR UPDATE`              | (ничего)        |                     |                 |
| Oracle                | `FOR UPDATE`                                 | `FOR UPDATE`              | (ничего)        |                     |                 |
| Postgres, CockroachDB | `FOR SHARE`                                  | `FOR UPDATE`              | (ничего)        | `FOR NO KEY UPDATE` | `FOR KEY SHARE` |
| SAP HANA              | `FOR SHARE LOCK`                             | `FOR UPDATE`              | (ничего)        |                     |                 |
| SQL Server            | `WITH (HOLDLOCK, ROWLOCK)`                   | `WITH (UPDLOCK, ROWLOCK)` | `WITH (NOLOCK)` |                     |                 |

Чтобы использовать пессимистичную блокировку на чтение, применяйте такой метод:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .setLock("pessimistic_read")
    .getMany()
```

Чтобы использовать пессимистичную блокировку на запись, применяйте такой метод:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .setLock("pessimistic_write")
    .getMany()
```

Чтобы использовать блокировку типа dirty read, применяйте такой метод:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .setLock("dirty_read")
    .getMany()
```

Чтобы использовать оптимистичную блокировку, применяйте такой метод:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .setLock("optimistic", existUser.version)
    .getMany()
```

Оптимистичная блокировка работает совместно с декораторами `@Version` и `@UpdatedDate`.

### Блокировка таблиц

Таблицы также можно блокировать следующим методом:

```typescript
const users = await dataSource
    .getRepository(Post)
    .createQueryBuilder("post")
    .leftJoin("post.author", "user")
    .setLock("pessimistic_write", undefined, ["post"])
    .getMany()
```

Если передан аргумент со списком блокируемых таблиц, в конструкции `FOR UPDATE OF` указывается только заблокированная таблица. Блокировка таблиц поддерживается только в PostgreSQL / CockroachDB.

### setOnLocked

Позволяет управлять поведением при заблокированной строке. По умолчанию база данных ждёт снятия блокировки.
Изменить это поведение можно с помощью `setOnLocked`

Чтобы не ждать:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .setLock("pessimistic_write")
    .setOnLocked("nowait")
    .getMany()
```

Чтобы пропустить строку:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .setLock("pessimistic_write")
    .setOnLocked("skip_locked")
    .getMany()
```

Поддержка `setOnLocked` базами данных в зависимости от [режима блокировки](#режимы-блокировки):

- MySQL < 8, Maria DB: `pessimistic_write`
- MySQL 8+: `pessimistic_read`, `pessimistic_write`
- Oracle: `pessimistic_write`
- Postgres, Cockroach: `pessimistic_read`, `pessimistic_write`, `for_no_key_update`, `for_key_share`
- SAP HANA: `pessimistic_read`, `pessimistic_write`

## Использование собственного индекса

В некоторых случаях вы можете указать серверу базы данных конкретный индекс. Эта возможность поддерживается только в MySQL.

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .useIndex("my_index") // имя индекса
    .getMany()
```

## Максимальное время выполнения

Медленный запрос можно прервать, чтобы не уронить сервер.

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .maxExecutionTime(1000) // миллисекунды.
    .getMany()
```

## Частичная выборка

Если нужно выбрать только некоторые свойства сущности, используйте такой синтаксис:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .select(["user.id", "user.name"])
    .getMany()
```

Так будут выбраны только `id` и `name` сущности `User`.

## Использование подзапросов

Подзапросы создаются легко. Они поддерживаются в выражениях `FROM`, `WHERE` и `JOIN`.
Пример:

```typescript
const qb = await dataSource.getRepository(Post).createQueryBuilder("post")

const posts = qb
    .where(
        "post.title IN " +
            qb
                .subQuery()
                .select("user.name")
                .from(User, "user")
                .where("user.registered = :registered")
                .getQuery(),
    )
    .setParameter("registered", true)
    .getMany()
```

Более элегантный способ сделать то же самое:

```typescript
const posts = await dataSource
    .getRepository(Post)
    .createQueryBuilder("post")
    .where((qb) => {
        const subQuery = qb
            .subQuery()
            .select("user.name")
            .from(User, "user")
            .where("user.registered = :registered")
            .getQuery()
        return "post.title IN " + subQuery
    })
    .setParameter("registered", true)
    .getMany()
```

Кроме того, можно создать отдельный query builder и использовать сгенерированный им SQL:

```typescript
const userQb = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .select("user.name")
    .where("user.registered = :registered", { registered: true })

const posts = await dataSource
    .getRepository(Post)
    .createQueryBuilder("post")
    .where("post.title IN (" + userQb.getQuery() + ")")
    .setParameters(userQb.getParameters())
    .getMany()
```

Подзапросы в `FROM` создаются так:

```typescript
const userQb = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .select("user.name", "name")
    .where("user.registered = :registered", { registered: true })

const posts = await dataSource
    .createQueryBuilder()
    .select("user.name", "name")
    .from("(" + userQb.getQuery() + ")", "user")
    .setParameters(userQb.getParameters())
    .getRawMany()
```

или с помощью более элегантного синтаксиса:

```typescript
const posts = await dataSource
    .createQueryBuilder()
    .select("user.name", "name")
    .from((subQuery) => {
        return subQuery
            .select("user.name", "name")
            .from(User, "user")
            .where("user.registered = :registered", { registered: true })
    }, "user")
    .getRawMany()
```

Если нужно добавить подзапрос как «второй from», используйте `addFrom`.

Подзапросы можно использовать и в выражениях `SELECT`:

```typescript
const posts = await dataSource
    .createQueryBuilder()
    .select("post.id", "id")
    .addSelect((subQuery) => {
        return subQuery.select("user.name", "name").from(User, "user").limit(1)
    }, "name")
    .from(Post, "post")
    .getRawMany()
```

## Скрытые колонки

Если у модели, к которой вы обращаетесь, есть колонка с параметром `select: false`, для получения данных из этой колонки нужно использовать функцию `addSelect`.

Допустим, у вас есть такая сущность:

```typescript
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @Column({ select: false })
    password: string
}
```

При обычном `find` или запросе свойство `password` для модели вы не получите. Однако если сделать так:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder()
    .select("user.id", "id")
    .addSelect("user.password")
    .getMany()
```

то свойство `password` в результате запроса будет.

`select: false` управляет только тем, что загружается из базы данных при чтении. На сущность в памяти, возвращаемую методом `repository.save()`, это не влияет: любое значение, которое вы передали для колонки с `select: false`, сохраняется и в возвращённой сущности, и в исходной ссылке, переданной в `save()`. Чтобы затем перечитать эти колонки из базы данных, выполните отдельный запрос с `addSelect`.

## Запрос удалённых строк

Если у модели, к которой вы обращаетесь, есть колонка с атрибутом `@DeleteDateColumn`, query builder автоматически будет запрашивать строки, помеченные как «мягко удалённые».

Допустим, у вас есть такая сущность:

```typescript
import {
    Entity,
    PrimaryGeneratedColumn,
    Column,
    DeleteDateColumn,
} from "typeorm"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column()
    name: string

    @DeleteDateColumn()
    deletedAt?: Date
}
```

При обычном `find` или запросе строки со значением в этой колонке вы не получите. Однако если сделать так:

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder()
    .select("user.id", "id")
    .withDeleted()
    .getMany()
```

вы получите все строки, включая удалённые.

## Обобщённые табличные выражения

Экземпляры `QueryBuilder`
поддерживают [обобщённые табличные выражения](https://en.wikipedia.org/wiki/Hierarchical_and_recursive_queries_in_SQL#Common_table_expression)
, если минимальная поддерживаемая версия вашей базы данных с ними работает. Для Oracle обобщённые табличные выражения пока не поддерживаются.

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .select("user.id", "id")
    .addCommonTableExpression(
        `
      SELECT "userId" FROM "post"
    `,
        "post_users_ids",
    )
    .where(`user.id IN (SELECT "userId" FROM 'post_users_ids')`)
    .getMany()
```

В Postgres можно использовать результирующие значения `InsertQueryBuilder` или `UpdateQueryBuilder`:

```typescript
const insertQueryBuilder = dataSource
    .getRepository(User)
    .createQueryBuilder()
    .insert({
        name: "John Smith",
    })
    .returning(["id"])

const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .addCommonTableExpression(insertQueryBuilder, "insert_results")
    .where(`user.id IN (SELECT "id" FROM 'insert_results')`)
    .getMany()
```

## Запросы с путешествием во времени

[Запросы с путешествием во времени](https://www.cockroachlabs.com/blog/time-travel-queries-select-witty_subtitle-the_future/)
в настоящее время поддерживаются только базой данных `CockroachDB`.

```typescript
const repository = dataSource.getRepository(Account)

// создаём новый аккаунт
const account = new Account()
account.name = "John Smith"
account.balance = 100
await repository.save(account)

// представим, что мы обновляем баланс аккаунта через 1 час после создания
account.balance = 200
await repository.save(account)

// выводит { name: "John Smith", balance: "200" }
console.log(account)

// загружаем состояние аккаунта на 1 час назад
account = await repository
    .createQueryBuilder("account")
    .timeTravelQuery(`'-1h'`)
    .getOneOrFail()

// выводит { name: "John Smith", balance: "100" }
console.log(account)
```

По умолчанию, если аргументы не переданы, `timeTravelQuery()` использует функцию `follower_read_timestamp()`.
Другие поддерживаемые аргументы с временными метками и дополнительные сведения см. в документации
[CockroachDB](https://www.cockroachlabs.com/docs/stable/as-of-system-time.html).

## Отладка

Сгенерированный SQL можно получить у query builder, вызвав `getQuery()` или `getQueryAndParameters()`.

Если вам нужен только запрос, используйте `getQuery()`

```typescript
const sql = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .where("user.id = :id", { id: 1 })
    .getQuery()
```

Результат:

```sql
SELECT `user`.`id` as `userId`, `user`.`firstName` as `userFirstName`, `user`.`lastName` as `userLastName` FROM `users` `user` WHERE `user`.`id` = ?
```

Если же нужны и запрос, и параметры, можно получить массив с помощью `getQueryAndParameters()`

```typescript
const queryAndParams = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .where("user.id = :id", { id: 1 })
    .getQueryAndParameters()
```

Результат:

```typescript
;[
    "SELECT `user`.`id` as `userId`, `user`.`firstName` as `userFirstName`, `user`.`lastName` as `userLastName` FROM `users` `user` WHERE `user`.`id` = ?",
    [1],
]
```

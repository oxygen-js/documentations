# Кэширование запросов

Вы можете кэшировать результаты, получаемые следующими методами `QueryBuilder`: `getMany`, `getOne`, `getRawMany`, `getRawOne` и `getCount`.

Также можно кэшировать результаты методов `find*` и `count*` у `Repository` и `EntityManager`.

Чтобы включить кэширование, его нужно явно активировать в опциях источника данных:

```typescript
{
    type: "mysql",
    host: "localhost",
    username: "test",
    ...
    cache: true
}
```

При первом включении кэша
необходимо синхронизировать схему базы данных (через CLI, миграции или опцию источника данных `synchronize`).

После этого в `QueryBuilder` можно включить кэширование для любого запроса:

```typescript
const users = await dataSource
    .createQueryBuilder(User, "user")
    .where("user.isAdmin = :isAdmin", { isAdmin: true })
    .cache(true)
    .getMany()
```

Эквивалентный запрос через `Repository`:

```typescript
const users = await dataSource.getRepository(User).find({
    where: { isAdmin: true },
    cache: true,
})
```

Этот код выполнит запрос, который выберет всех пользователей-администраторов, и закэширует результат.
При следующем выполнении того же кода все администраторы будут взяты из кэша.
Время жизни кэша по умолчанию — `1000 мс`, то есть 1 секунда.
Это значит, что кэш станет недействительным через 1 секунду после вызова кода query builder.
На практике это означает, что если пользователи откроют страницу пользователей 150 раз за 3 секунды, за это время выполнится всего три запроса.
Пользователи, добавленные в течение этого секундного окна кэширования, в выдачу не попадут.

Время кэширования можно задать вручную через `QueryBuilder`:

```typescript
const users = await dataSource
    .createQueryBuilder(User, "user")
    .where("user.isAdmin = :isAdmin", { isAdmin: true })
    .cache(60000) // 1 минута
    .getMany()
```

Или через `Repository`:

```typescript
const users = await dataSource.getRepository(User).find({
    where: { isAdmin: true },
    cache: 60000,
})
```

Или глобально в опциях источника данных:

```typescript
{
    type: "mysql",
    host: "localhost",
    username: "test",
    ...
    cache: {
        duration: 30000 // 30 секунд
    }
}
```

Кроме того, через `QueryBuilder` можно задать «идентификатор кэша»:

```typescript
const users = await dataSource
    .createQueryBuilder(User, "user")
    .where("user.isAdmin = :isAdmin", { isAdmin: true })
    .cache("users_admins", 25000)
    .getMany()
```

Или с помощью `Repository`:

```typescript
const users = await dataSource.getRepository(User).find({
    where: { isAdmin: true },
    cache: {
        id: "users_admins",
        milliseconds: 25000,
    },
})
```

Это даёт точный контроль над кэшем — например, позволяет сбрасывать закэшированные результаты при добавлении нового пользователя:

```typescript
await dataSource.queryResultCache.remove(["users_admins"])
```

По умолчанию TypeORM использует отдельную таблицу `query-result-cache` и хранит в ней все запросы и результаты.
Имя таблицы настраивается: его можно изменить, указав другое значение в свойстве tableName.
Пример:

```typescript
{
    type: "mysql",
    host: "localhost",
    username: "test",
    ...
    cache: {
        type: "database",
        tableName: "configurable-table-query-result-cache"
    }
}
```

Если хранить кэш в одной таблице базы данных вам неэффективно,
можно поменять тип кэша на "redis" или "ioredis", и TypeORM будет хранить все закэшированные записи в redis.
Пример:

```typescript
{
    type: "mysql",
    host: "localhost",
    username: "test",
    ...
    cache: {
        type: "redis",
        options: {
            socket: {
                host: "localhost",
                port: 6379
            }
        }
    }
}
```

В «options» можно передать [опции node_redis](https://github.com/redis/node-redis/blob/master/docs/client-configuration.md) или [опции ioredis](https://github.com/luin/ioredis/blob/master/API.md#new-redisport-host-options) — в зависимости от выбранного типа.

Если вы хотите подключиться к redis-кластеру через кластерную функциональность IORedis, это тоже возможно:

```typescript
{
    type: "mysql",
    host: "localhost",
    username: "test",
    cache: {
        type: "ioredis/cluster",
        options: {
            startupNodes: [
                {
                    host: 'localhost',
                    port: 7000,
                },
                {
                    host: 'localhost',
                    port: 7001,
                },
                {
                    host: 'localhost',
                    port: 7002,
                }
            ],
            options: {
                scaleReads: 'all',
                clusterRetryStrategy: function (times) { return null },
                redisOptions: {
                    maxRetriesPerRequest: 1
                }
            }
        }
    }
}
```

Обратите внимание: options по-прежнему можно передать первым аргументом кластерного конструктора IORedis.

```typescript
{
    ...
    cache: {
        type: "ioredis/cluster",
        options: [
            {
                host: 'localhost',
                port: 7000,
            },
            {
                host: 'localhost',
                port: 7001,
            },
            {
                host: 'localhost',
                port: 7002,
            }
        ]
    },
    ...
}
```

Если ни один из встроенных провайдеров кэша вам не подходит, вы можете указать собственный провайдер через фабричную функцию `provider`, которая должна возвращать новый объект, реализующий интерфейс `QueryResultCache`:

```typescript
class CustomQueryResultCache implements QueryResultCache {
    constructor(private dataSource: DataSource) {}
    ...
}
```

```typescript
{
    ...
    cache: {
        provider(dataSource) {
            return new CustomQueryResultCache(dataSource);
        }
    }
}
```

Если вы хотите игнорировать ошибки кэша и при таких ошибках направлять запросы напрямую в базу данных, воспользуйтесь опцией ignoreErrors.
Пример:

```typescript
{
    type: "mysql",
    host: "localhost",
    username: "test",
    ...
    cache: {
        type: "redis",
        options: {
            socket: {
                host: "localhost",
                port: 6379
            }
        },
        ignoreErrors: true
    }
}
```

Команда `typeorm cache:clear` очищает всё, что хранится в кэше.

# Работа со связями

`RelationQueryBuilder` — это особый вид `QueryBuilder`, позволяющий работать со связями.
С его помощью можно связывать сущности друг с другом прямо в базе данных, не загружая их,
а также легко загружать связанные сущности.

Например, у нас есть сущность `Post` со связью «многие ко многим» с `Category` под названием `categories`.
Добавим новую категорию в эту связь:

```typescript
await dataSource
    .createQueryBuilder()
    .relation(Post, "categories")
    .of(post)
    .add(category)
```

Этот код эквивалентен следующему:

```typescript
const postRepository = dataSource.manager.getRepository(Post)
const post = await postRepository.findOne({
    where: {
        id: 1,
    },
    relations: {
        categories: true,
    },
})
post.categories.push(category)
await postRepository.save(post)
```

Но он эффективнее, поскольку выполняет минимальное число операций и связывает сущности в базе данных,
в отличие от вызова тяжеловесного метода `save`.

Ещё одно преимущество такого подхода в том, что вам не нужно загружать все связанные сущности перед добавлением новой.
Например, если внутри одного поста десять тысяч категорий, добавление новых постов в этот список может стать проблемой,
поскольку стандартный способ — загрузить пост со всеми десятью тысячами категорий, добавить новую категорию
и сохранить его. Это влечёт очень высокие издержки производительности и по сути неприменимо в продакшене.
Однако `RelationQueryBuilder` решает эту проблему.

Кроме того, при «связывании» не обязательно использовать сами сущности — можно обойтись их идентификаторами.
Например, добавим категорию с id = 3 в пост с id = 1:

```typescript
await dataSource.createQueryBuilder().relation(Post, "categories").of(1).add(3)
```

Если вы используете составные первичные ключи, их нужно передавать в виде карты идентификаторов, например:

```typescript
await dataSource
    .createQueryBuilder()
    .relation(Post, "categories")
    .of({ firstPostId: 1, secondPostId: 3 })
    .add({ firstCategoryId: 2, secondCategoryId: 4 })
```

Удалять сущности можно так же, как и добавлять:

```typescript
// этот код удаляет категорию из указанного поста
await dataSource
    .createQueryBuilder()
    .relation(Post, "categories")
    .of(post) // можно передать и просто id поста
    .remove(category) // можно передать и просто id категории
```

Добавление и удаление связанных сущностей работает в связях `many-to-many` и `one-to-many`.
Для связей `one-to-one` и `many-to-one` используйте `set`:

```typescript
// этот код устанавливает категорию указанного поста
await dataSource
    .createQueryBuilder()
    .relation(Post, "categories")
    .of(post) // можно передать и просто id поста
    .set(category) // можно передать и просто id категории
```

Если нужно сбросить связь (установить её в null), просто передайте `null` в метод `set`:

```typescript
// этот код сбрасывает категорию указанного поста
await dataSource
    .createQueryBuilder()
    .relation(Post, "categories")
    .of(post) // можно передать и просто id поста
    .set(null)
```

Помимо обновления связей, relational query builder позволяет загружать связанные сущности.
Например, пусть внутри сущности `Post` есть связь «многие ко многим» `categories` и связь «многие к одному» `user`;
чтобы загрузить эти связи, можно использовать следующий код:

```typescript
const post = await dataSource.manager.findOneBy(Post, {
    id: 1,
})

post.categories = await dataSource
    .createQueryBuilder()
    .relation(Post, "categories")
    .of(post) // можно передать и просто id поста
    .loadMany()

post.author = await dataSource
    .createQueryBuilder()
    .relation(Post, "user")
    .of(post) // можно передать и просто id поста
    .loadOne()
```

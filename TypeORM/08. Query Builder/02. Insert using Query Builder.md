# Вставка данных через Query Builder

Запросы `INSERT` можно создавать с помощью `QueryBuilder`.
Примеры:

```typescript
await dataSource
    .createQueryBuilder()
    .insert()
    .into(User)
    .values([
        { firstName: "Timber", lastName: "Saw" },
        { firstName: "Phantom", lastName: "Lancer" },
    ])
    .execute()
```

С точки зрения производительности это самый эффективный способ вставки строк в базу данных.
Так же можно выполнять и массовую вставку.

## Поддержка сырого SQL

В некоторых случаях, когда нужно выполнить SQL-запросы, требуется задать значение в виде функции:

```typescript
await dataSource
    .createQueryBuilder()
    .insert()
    .into(User)
    .values({
        firstName: "Timber",
        lastName: () => "CONCAT('S', 'A', 'W')",
    })
    .execute()
```

> **Предупреждение** При использовании сырого SQL убедитесь, что значения должным образом экранированы, чтобы предотвратить SQL-инъекции.

## Обновление значений при ON CONFLICT

Если вставляемые значения конфликтуют с уже существующими данными, для обновления конкретных значений у конфликтующей цели можно использовать функцию `orUpdate`.

```typescript
await dataSource
    .createQueryBuilder()
    .insert()
    .into(User)
    .values({
        firstName: "Timber",
        lastName: "Saw",
        externalId: "abc123",
    })
    .orUpdate(["firstName", "lastName"], ["externalId"])
    .execute()
```

## Обновление значений при ON CONFLICT с условием (Postgres, Oracle, MSSQL, SAP HANA)

```typescript
await dataSource
    .createQueryBuilder()
    .insert()
    .into(User)
    .values({
        firstName: "Timber",
        lastName: "Saw",
        externalId: "abc123",
    })
    .orUpdate(["firstName", "lastName"], ["externalId"], {
        overwriteCondition: {
            where: {
                firstName: Equal("Phantom"),
            },
        },
    })
    .execute()
```

## Ошибка IGNORE (MySQL) или DO NOTHING (Postgres, Oracle, MSSQL, SAP HANA) при вставке

Если вставляемые значения конфликтуют с существующими данными или содержат некорректные данные, функцию `orIgnore` можно использовать для подавления ошибок и вставки только тех строк, которые содержат корректные данные.

```typescript
await dataSource
    .createQueryBuilder()
    .insert()
    .into(User)
    .values({
        firstName: "Timber",
        lastName: "Saw",
        externalId: "abc123",
    })
    .orIgnore()
    .execute()
```

## Пропуск обновления данных, если значения не изменились (Postgres, Oracle, MSSQL, SAP HANA)

```typescript
await dataSource
    .createQueryBuilder()
    .insert()
    .into(User)
    .values({
        firstName: "Timber",
        lastName: "Saw",
        externalId: "abc123",
    })
    .orUpdate(["firstName", "lastName"], ["externalId"], {
        skipUpdateIfNoValuesChanged: true,
    })
    .execute()
```

## Использование частичного индекса (Postgres)

```typescript
await dataSource
    .createQueryBuilder()
    .insert()
    .into(User)
    .values({
        firstName: "Timber",
        lastName: "Saw",
        externalId: "abc123",
    })
    .orUpdate(["firstName", "lastName"], ["externalId"], {
        skipUpdateIfNoValuesChanged: true,
        indexPredicate: "date > 2020-01-01",
    })
    .execute()
```

## Вставка из SELECT

Вы можете вставлять данные из одной таблицы в другую, используя запрос `SELECT` и метод `valuesFromSelect()`. Так формируется выражение `INSERT INTO ... SELECT FROM`, которое удобно для миграции данных, архивирования или копирования данных между таблицами.

### Использование SelectQueryBuilder напрямую

```typescript
// Create a select query to get the source data
const selectQuery = dataSource
    .createQueryBuilder()
    .select(["user.firstName", "user.lastName"])
    .from(User, "user")
    .where("user.isActive = :isActive", { isActive: true })

// Insert the selected data into another table
await dataSource
    .createQueryBuilder()
    .insert()
    .into(ArchivedUser, ["firstName", "lastName"])
    .valuesFromSelect(selectQuery)
    .execute()
```

### Использование функции обратного вызова

Для построения запроса SELECT можно также использовать функцию обратного вызова:

```typescript
await dataSource
    .createQueryBuilder()
    .insert()
    .into(ArchivedUser, ["firstName", "lastName", "archivedAt"])
    .valuesFromSelect((qb) =>
        qb
            .select(["user.firstName", "user.lastName", "NOW()"])
            .from(User, "user")
            .where("user.deletedAt IS NOT NULL"),
    )
    .execute()
```

### С join-ами

В запросе SELECT можно использовать join-ы, чтобы объединить данные из нескольких таблиц:

```typescript
await dataSource
    .createQueryBuilder()
    .insert()
    .into(UserReport, ["userName", "orderCount"])
    .valuesFromSelect((qb) =>
        qb
            .select(["user.name", "COUNT(order.id)"])
            .from(User, "user")
            .leftJoin("user.orders", "order")
            .groupBy("user.id"),
    )
    .execute()
```

> **Примечание** При использовании `valuesFromSelect()` слушатели и подписчики сущностей (`@BeforeInsert`, `@AfterInsert`) не вызываются, поскольку во время операции вставки экземпляры сущностей не создаются.

> **Примечание** Опция `updateEntity` не действует при использовании `valuesFromSelect()`, так как обновлять нечего — экземпляров сущностей нет.

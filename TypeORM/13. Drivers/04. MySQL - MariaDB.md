# MySQL / MariaDB

MySQL, MariaDB и Amazon Aurora MySQL поддерживаются в качестве драйверов TypeORM.

## Установка

```shell
npm install mysql2
```

## Опции источника данных

Общие опции источника данных описаны в разделе [Опции источника данных](<../02. Data Source/02. Data Source Options.md>). Для подключения к соответствующим базам данных используйте типы источника данных `mysql`, `mariadb` и `aurora-mysql`.

- `url` — URL подключения. Обратите внимание, что остальные опции источника данных переопределяют параметры, заданные в URL.

- `host` — хост базы данных.

- `port` — порт хоста базы данных. Порт mysql по умолчанию — `3306`.

- `username` — имя пользователя базы данных.

- `password` — пароль базы данных.

- `database` — имя базы данных.

- `socketPath` — путь к сокету базы данных.

- `poolSize` — максимальное количество клиентов, которое пул может содержать для каждого соединения.

- `charset` и `collation` — кодировка и правило сортировки для соединения. Если указана кодировка уровня SQL (например, utf8mb4), используется правило сортировки, заданное для этой кодировки по умолчанию.

- `timezone` — часовой пояс, настроенный на сервере MySQL. Используется для приведения серверных значений даты/времени
  к объекту JavaScript Date и обратно. Значением может быть `local`, `Z` или смещение в формате
  `+HH:MM` либо `-HH:MM`. (По умолчанию: `local`)

- `connectTimeout` — время в миллисекундах до истечения тайм-аута при первоначальном подключении к серверу MySQL.
  (По умолчанию: `10000`)

- `acquireTimeout` — время в миллисекундах до истечения тайм-аута при первоначальном подключении к серверу MySQL. Отличается от `connectTimeout` тем, что управляет тайм-аутом TCP-соединения, тогда как connectTimeout этого не делает. (по умолчанию: `10000`)

- `insecureAuth` — разрешить подключение к экземплярам MySQL, запрашивающим старый (небезопасный) метод аутентификации.
  (По умолчанию: `false`)

- `supportBigNumbers` — если в базе данных используются большие числа (колонки `BIGINT` и `DECIMAL`),
  эту опцию следует включить (по умолчанию: `true`)

- `bigNumberStrings` — при одновременном включении `supportBigNumbers` и `bigNumberStrings` большие числа
  (колонки `BIGINT` и `DECIMAL`) всегда возвращаются как объекты JavaScript String (по умолчанию: `true`).
  Если включён `supportBigNumbers`, но `bigNumberStrings` выключен, большие числа будут возвращаться как объекты String
  только тогда, когда их нельзя точно представить
  [объектами JavaScript Number](http://ecma262-5.com/ELS5_HTML.htm#Section_8.5)
  (это происходит при выходе за диапазон `[-2^53, +2^53]`), в остальных случаях они будут возвращаться как
  объекты Number. Эта опция игнорируется, если `supportBigNumbers` выключен.

- `dateStrings` — заставляет возвращать типы даты (`TIMESTAMP`, `DATETIME`, `DATE`) в виде строк, а не
  преобразовывать их в объекты JavaScript Date. Значением может быть true/false либо массив имён типов, которые нужно оставить строками.
  (По умолчанию: `false`)

- `debug` — выводит подробности протокола в stdout. Значением может быть true/false либо массив имён типов пакетов,
  которые следует выводить. (По умолчанию: `false`)

- `trace` — добавляет к ошибкам трассировку стека с местом вызова точки входа в библиотеку («длинные трассировки стека»).
  Незначительно снижает производительность большинства вызовов. (По умолчанию: `true`)

- `multipleStatements` — разрешает несколько выражений mysql в одном запросе. Пользуйтесь осторожно: это может расширить
  возможности для SQL-инъекций. (По умолчанию: `false`)

- `legacySpatialSupport` — использовать устаревшие пространственные функции вроде `GeomFromText` и `AsText`, которые в MySQL 8.0 заменены соответствующими стандарту `ST_GeomFromText` и `ST_AsText`. (По умолчанию: `false`)

- `flags` — список флагов соединения, отличных от заданных по умолчанию. Флаги по умолчанию также можно занести в чёрный список.
  Подробнее — в разделе [Connection Flags](https://github.com/mysqljs/mysql#connection-flags).

- `ssl` — объект с параметрами SSL либо строка с именем SSL-профиля.
  См. [SSL options](https://github.com/mysqljs/mysql#ssl-options).

- `enableQueryTimeout` — если задано значение maxQueryExecutionTime, то помимо записи предупреждения в лог при превышении запросом этого лимита времени указанное значение maxQueryExecutionTime используется ещё и как тайм-аут запроса. Подробнее — в разделе [mysql timeouts](https://github.com/mysqljs/mysql#timeouts).

Дополнительные опции можно добавить в объект `extra` — они будут переданы напрямую клиентской библиотеке. Подробнее — в [документации mysql2](https://sidorares.github.io/node-mysql2/docs).

## Типы колонок

`bit`, `int`, `integer`, `tinyint`, `smallint`, `mediumint`, `bigint`, `float`, `double`, `double precision`, `dec`, `decimal`, `numeric`, `fixed`, `bool`, `boolean`, `date`, `datetime`, `timestamp`, `time`, `year`, `char`, `nchar`, `national char`, `varchar`, `nvarchar`, `national varchar`, `text`, `tinytext`, `mediumtext`, `blob`, `longtext`, `tinyblob`, `mediumblob`, `longblob`, `enum`, `set`, `json`, `binary`, `varbinary`, `geometry`, `point`, `linestring`, `polygon`, `multipoint`, `multilinestring`, `multipolygon`, `geometrycollection`, `uuid`, `inet4`, `inet6`

> **Примечание** `uuid`, `inet4` и `inet6` доступны только в MariaDB и только в тех версиях, где они появились.

### Тип колонки `set`

Тип колонки `set` поддерживается в `mariadb` и `mysql`. Возможны разные варианты объявления колонки:

С использованием перечислений TypeScript:

```typescript
export enum UserRole {
    ADMIN = "admin",
    EDITOR = "editor",
    GHOST = "ghost",
}

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column({
        type: "set",
        enum: UserRole,
        default: [UserRole.GHOST, UserRole.EDITOR],
    })
    roles: UserRole[]
}
```

С использованием массива значений `set`:

```typescript
export type UserRoleType = "admin" | "editor" | "ghost"

@Entity()
export class User {
    @PrimaryGeneratedColumn()
    id: number

    @Column({
        type: "set",
        enum: ["admin", "editor", "ghost"],
        default: ["ghost", "editor"],
    })
    roles: UserRoleType[]
}
```

### Векторные типы

MySQL поддерживает [тип VECTOR](https://dev.mysql.com/doc/refman/en/vector.html) начиная с версии 9.0, а в MariaDB [векторы](https://mariadb.com/docs/server/reference/sql-structure/vectors/vector-overview) доступны начиная с 11.7.

# SAP HANA

## Установка

Для подключения к базе данных TypeORM использует `@sap/hana-client`:

```shell
npm install @sap/hana-client
```

Если вы используете TypeORM 0.3.25 или более раннюю версию, для управления пулом дополнительно потребуется `hdb-pool`.

## Опции источника данных

Общие опции источника данных описаны в разделе [Опции источника данных](<../02. Data Source/02. Data Source Options.md>).

- `host` — имя хоста сервера SAP HANA. Например, `"localhost"`.
- `port` — номер порта сервера SAP HANA. Например, `30015`.
- `username` — имя пользователя для подключения к серверу SAP HANA. Например, `"SYSTEM"`.
- `password` — пароль для подключения к серверу SAP HANA. Например, `"password"`.
- `database` — имя базы данных, к которой выполняется подключение. Например, `"HXE"`.
- `encrypt` — шифровать ли соединение. Например, `true`.
- `sslValidateCertificate` — проверять ли SSL-сертификат. Например, `true`.
- `key`, `cert` и `ca` — закрытый ключ, открытый сертификат и удостоверяющий центр для шифрованного соединения.
- `driver` — необязательный явно заданный экземпляр модуля `@sap/hana-client`. Если опция не указана, TypeORM загружает `@sap/hana-client` автоматически.
- `pool` — объект конфигурации пула соединений:
    - `maxConnectedOrPooled` (число) — максимальное количество активных или простаивающих соединений в пуле (по умолчанию: 10).
    - `maxPooledIdleTime` (секунды) — время, по истечении которого простаивающее соединение закрывается (по умолчанию: 30).
    - `maxWaitTimeoutIfPoolExhausted` (миллисекунды) — время ожидания освобождения соединения (по умолчанию: 0, без ожидания). Требуется `@sap/hana-client` версии `2.27` или новее.
    - `pingCheck` (логическое) — проверять ли соединения перед использованием (по умолчанию: false).
    - `poolCapacity` (число) — максимальное количество соединений, поддерживаемых доступными (по умолчанию: без ограничения).

Подробности, а также описание свойств `extra`, смотрите в официальной документации SAP HANA Client: [Node.js Connection Properties](https://help.sap.com/docs/SAP_HANA_CLIENT/f1b440ded6144a54ada97ff95dac7adf/4fe9978ebac44f35b9369ef5a4a26f4c.html).

## Типы колонок

SAP HANA 2.0 и SAP HANA Cloud поддерживают немного различающиеся типы данных. Подробности смотрите на страницах SAP Help:

- [SAP HANA 2.0 Data Types](https://help.sap.com/docs/SAP_HANA_PLATFORM/4fe29514fd584807ac9f2a04f6754767/20a1569875191014b507cf392724b7eb.html?locale=en-US)
- [SAP HANA Cloud Data Types](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-sql-reference-guide/data-types)

`SapDriver` из TypeORM поддерживает `tinyint`, `smallint`, `integer`, `bigint`, `smalldecimal`, `decimal`, `real`, `double`, `date`, `time`, `seconddate`, `timestamp`, `boolean`, `char`, `nchar`, `varchar`, `nvarchar`, `text`, `alphanum`, `shorttext`, `array`, `varbinary`, `blob`, `clob`, `nclob`, `st_geometry`, `st_point`, `real_vector` и `half_vector`. Часть этих типов данных объявлена устаревшей или удалена в SAP HANA Cloud — при подключении к базе Cloud они будут преобразованы в ближайшую доступную альтернативу.

### Векторные типы

Типы данных `real_vector` и `half_vector` появились в SAP HANA Cloud (в 2024Q1 и 2025Q2 соответственно) и требуют также поддерживающей их версии `@sap/hana-client`.

Для единообразия с поддержкой векторов в PostgreSQL TypeORM предоставляет и псевдонимы:

- `vector` (псевдоним для `real_vector`) — хранит векторы как 4-байтовые числа с плавающей точкой
- `halfvec` (псевдоним для `half_vector`) — хранит векторы как 2-байтовые числа с плавающей точкой ради экономии памяти

```typescript
@Entity()
export class Document {
    @PrimaryGeneratedColumn()
    id: number

    // Использование нативных имён типов SAP HANA
    @Column("real_vector", { length: 1536 })
    embedding: Buffer | number[]

    @Column("half_vector", { length: 768 })
    reduced_embedding: Buffer | number[]

    // Использование межбазовых псевдонимов (рекомендуется)
    @Column("vector", { length: 1536 })
    universal_embedding: Buffer | number[]

    @Column("halfvec", { length: 768 })
    universal_reduced_embedding: Buffer | number[]
}
```

По умолчанию клиент возвращает `Buffer` в формате `fvecs`/`hvecs`, что эффективнее. Можно заставить драйвер преобразовывать значения в `number[]`, добавив `{ extra: { vectorOutputType: "Array" } }` в опции подключения. Подробнее о [REAL_VECTOR](https://help.sap.com/docs/SAP_HANA_CLIENT/f1b440ded6144a54ada97ff95dac7adf/0d197e4389c64e6b9cf90f6f698f62fe.html) и [HALF_VECTOR](https://help.sap.com/docs/SAP_HANA_CLIENT/f1b440ded6144a54ada97ff95dac7adf/8bb854b4ce4a4299bed27c365b717e91.html) — в документации SAP HANA Client.

Для поиска по схожести используйте соответствующие [векторные функции](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-sql-reference-guide/vector-functions).

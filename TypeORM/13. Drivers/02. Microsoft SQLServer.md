# Microsoft SQLServer

## Установка

```shell
npm install mssql
```

## Опции источника данных

Общие опции источника данных описаны в разделе [Опции источника данных](<../02. Data Source/02. Data Source Options.md>).

Основано на реализации MSSQL [tedious](https://tediousjs.github.io/node-mssql/). Подробности о доступных атрибутах см. в [SqlServerConnectionOptions.ts](https://github.com/typeorm/typeorm/tree/master/src/driver/sqlserver/SqlServerConnectionOptions.ts).

- `url` — URL подключения, по которому выполняется соединение. Учтите, что остальные опции источника данных переопределяют параметры, заданные в url.

- `host` — хост базы данных.

- `port` — порт хоста базы данных. Порт mssql по умолчанию — `1433`.

- `username` — имя пользователя базы данных.

- `password` — пароль базы данных.

- `database` — имя базы данных.

- `schema` — имя схемы. По умолчанию «dbo».

- `domain` — если задать домен, драйвер будет подключаться к SQL Server, используя доменную учётную запись.

- `connectionTimeout` — тайм-аут подключения в мс (по умолчанию: `15000`).

- `requestTimeout` — тайм-аут запроса в мс (по умолчанию: `15000`). ПРИМЕЧАНИЕ: драйвер msnodesqlv8 не поддерживает
  тайм-ауты менее 1 секунды.

- `stream` — передавать наборы записей/строки потоком вместо возврата их все сразу в качестве аргумента коллбэка (по умолчанию: `false`).
  Потоковую передачу можно также включить для каждого запроса по отдельности (`request.stream = true`). Всегда указывайте `true`, если планируете
  работать с большим количеством строк.

- `pool.max` — максимальное количество соединений в пуле (по умолчанию: `10`).

- `pool.min` — минимальное количество соединений в пуле (по умолчанию: `0`).

- `pool.maxWaitingClients` — максимальное количество допустимых запросов в очереди; дополнительные вызовы acquire получат
  ошибку в одном из следующих циклов event loop.

- `pool.acquireTimeoutMillis` — максимальное количество миллисекунд, которое вызов `acquire` будет ждать ресурс до истечения тайм-аута.
  (по умолчанию без ограничения); если задаётся, должно быть ненулевым положительным целым числом.

- `pool.fifo` — если `true`, первыми выделяются самые старые ресурсы. Если `false`, первыми выделяются недавно освобождённые
  ресурсы. Фактически это превращает поведение пула из очереди в стек. Булево значение,
  (по умолчанию `true`).

- `pool.priorityRange` — целое число от 1 до x: если задано, потребители могут указывать свой относительный приоритет в очереди, когда
  свободных ресурсов нет. См. пример. (по умолчанию `1`).

- `pool.evictionRunIntervalMillis` — как часто выполнять проверки вытеснения. По умолчанию: `0` (не выполняются).

- `pool.numTestsPerRun` — количество ресурсов, проверяемых за один проход вытеснения. По умолчанию: `3`.

- `pool.softIdleTimeoutMillis` — время, в течение которого объект может простаивать в пуле, прежде чем станет кандидатом на вытеснение
  вытеснителем простаивающих объектов (если он есть), при дополнительном условии, что в пуле останется не менее «min idle» экземпляров объектов.
  По умолчанию `-1` (ничего не вытесняется).

- `pool.idleTimeoutMillis` — минимальное время, в течение которого объект может простаивать в пуле, прежде чем станет кандидатом на
  вытеснение по простою. Имеет приоритет над `softIdleTimeoutMillis`. По умолчанию: `30000`.

- `pool.errorHandler` — функция, вызываемая, когда нижележащий пул генерирует событие `'error'`. Принимает один параметр (экземпляр ошибки), по умолчанию логирует с уровнем `warn`.

- `options.fallbackToDefaultDb` — по умолчанию, если база данных, запрошенная через `options.database`, недоступна, подключение завершится ошибкой. Однако если `options.fallbackToDefaultDb` установлено в `true`, вместо неё будет использована база данных пользователя по умолчанию (по умолчанию: `false`).

- `options.instanceName` — имя экземпляра, к которому нужно подключиться. На сервере базы данных должна быть запущена служба SQL Server Browser, а UDP-порт 1434 на сервере базы данных должен быть доступен. Взаимоисключающая опция с `port`. (значения по умолчанию нет).

- `options.enableAnsiNullDefault` — если `true`, в начальном SQL будет выполнено `SET ANSI_NULL_DFLT_ON ON`. Это означает, что новые
  колонки по умолчанию будут допускать NULL. Подробности см. в [документации по T-SQL](https://msdn.microsoft.com/en-us/library/ms187375.aspx).
  (По умолчанию: `true`).

- `options.cancelTimeout` — количество миллисекунд, по истечении которых отмена (прерывание) запроса считается неудачной (по умолчанию: `5000`).

- `options.packetSize` — размер пакетов TDS (может быть скорректирован при согласовании с сервером). Должен быть степенью двойки. (по умолчанию: `4096`).

- `options.useUTC` — булево значение, определяющее, передавать ли значения времени в UTC или в локальном времени. (по умолчанию: `false`).

- `options.abortTransactionOnError` — булево значение, определяющее, откатывать ли транзакцию автоматически, если во время
  её выполнения возникнет ошибка. Задаёт значение `SET XACT_ABORT` во время
  начальной SQL-фазы соединения ([документация](http://msdn.microsoft.com/en-us/library/ms188792.aspx)).

- `options.localAddress` — строка, указывающая, какой сетевой интерфейс (ip-адрес) использовать при подключении к SQL Server.

- `options.useColumnNames` — булево значение, определяющее, возвращать ли строки как массивы или как коллекции «ключ-значение». (по умолчанию: `false`).

- `options.camelCaseColumns` — булево значение, управляющее тем, будет ли первая буква возвращаемых имён колонок
  переведена в нижний регистр (`true`) или нет. Это значение игнорируется, если вы задали `columnNameReplacer`. (по умолчанию: `false`).

- `options.isolationLevel` — уровень изоляции по умолчанию, с которым будут выполняться транзакции. См. [Известные проблемы > Пул соединений не сбрасывает уровень изоляции](#пул-соединений-не-сбрасывает-уровень-изоляции).
    - `READ UNCOMMITTED`
    - `READ COMMITTED`
    - `REPEATABLE READ`
    - `SERIALIZABLE`
    - `SNAPSHOT`

    (по умолчанию: `READ COMMITTED`)

- `options.connectionIsolationLevel` — уровень изоляции по умолчанию для новых соединений. С этой настройкой выполняются все
  запросы вне транзакций. См. [Известные проблемы > Пул соединений не сбрасывает уровень изоляции](#пул-соединений-не-сбрасывает-уровень-изоляции).
    - `READ UNCOMMITTED`
    - `READ COMMITTED`
    - `REPEATABLE READ`
    - `SERIALIZABLE`
    - `SNAPSHOT`

    (по умолчанию: `READ COMMITTED`)

- `options.readOnlyIntent` — булево значение, определяющее, будет ли соединение запрашивать доступ только на чтение у
  группы доступности SQL Server. Подробнее см. здесь. (по умолчанию: `false`).

- `options.encrypt` — булево значение, определяющее, будет ли соединение зашифровано. Установите в `true`, если вы работаете в Windows Azure. (по умолчанию: `true`).

- `options.cryptoCredentialsDetails` — при использовании шифрования можно передать объект, который будет использован в качестве
  первого аргумента при вызове [tls.createSecurePair](http://nodejs.org/docs/latest/api/tls.html#tls_tls_createsecurepair_credentials_isserver_requestcert_rejectunauthorized)
  (по умолчанию: `{}`).

- `options.rowCollectionOnDone` — булево значение; если `true`, полученные строки будут доступны в событиях `done*` объекта Request.
  См. done, [doneInProc](http://tediousjs.github.io/tedious/api-request.html#event_doneInProc)
  и [doneProc](http://tediousjs.github.io/tedious/api-request.html#event_doneProc). (по умолчанию: `false`)

    Осторожно: при получении большого количества строк включение этой опции может привести к чрезмерному расходу памяти.

- `options.rowCollectionOnRequestCompletion` — булево значение; если `true`, полученные строки будут доступны
  в коллбэке завершения объекта Request. См. [new Request](http://tediousjs.github.io/tedious/api-request.html#function_newRequest). (по умолчанию: `false`)

    Осторожно: при получении большого количества строк включение этой опции может привести к чрезмерному расходу памяти.

- `options.tdsVersion` — версия TDS, которую следует использовать. Если сервер не поддерживает указанную версию, вместо неё используется
  согласованная версия. Доступные версии находятся в `require('tedious').TDS_VERSION`.
    - `7_1`
    - `7_2`
    - `7_3_A`
    - `7_3_B`
    - `7_4`

    (по умолчанию: `7_4`)

- `options.appName` — имя приложения, используемое для идентификации конкретного приложения в инструментах профилирования, логирования или трассировки SQL Server. (по умолчанию: `node-mssql`)

- `options.trustServerCertificate` — булево значение, управляющее тем, выполняется ли шифрование при отсутствии проверяемого сертификата сервера. (по умолчанию: `false`)

- `options.multiSubnetFailover` — булево значение, управляющее тем, должен ли драйвер подключаться параллельно ко всем IP, полученным от DNS. (по умолчанию: `false`)

- `options.debug.packet` — булево значение, управляющее тем, будут ли генерироваться события `debug` с текстом, описывающим
  детали пакета (по умолчанию: `false`).

- `options.debug.data` — булево значение, управляющее тем, будут ли генерироваться события `debug` с текстом, описывающим
  детали данных пакета (по умолчанию: `false`).

- `options.debug.payload` — булево значение, управляющее тем, будут ли генерироваться события `debug` с текстом, описывающим
  детали полезной нагрузки пакета (по умолчанию: `false`).

- `options.debug.token` — булево значение, управляющее тем, будут ли генерироваться события `debug` с текстом, описывающим
  токены потока токенов (по умолчанию: `false`).

## Типы колонок

`int`, `bigint`, `bit`, `decimal`, `money`, `numeric`, `smallint`, `smallmoney`, `tinyint`, `float`, `real`, `date`, `datetime2`, `datetime`, `datetimeoffset`, `smalldatetime`, `time`, `char`, `varchar`, `text`, `nchar`, `nvarchar`, `ntext`, `binary`, `image`, `varbinary`, `hierarchyid`, `sql_variant`, `timestamp`, `uniqueidentifier`, `xml`, `geometry`, `geography`, `rowversion`, `vector`

### Тип vector

Тип данных `vector` доступен в SQL Server для хранения многомерных векторов и обычно применяется для:

- Семантического поиска по эмбеддингам
- Рекомендательных систем
- Поиска по схожести
- Задач машинного обучения

ПРИМЕЧАНИЕ: общая поддержка типа `halfvec` недоступна, поскольку эта возможность всё ещё находится в статусе preview. См. документацию Microsoft: [Vector data type](https://learn.microsoft.com/en-us/sql/t-sql/data-types/vector-data-type).

#### Использование

```typescript
@Entity()
export class DocumentChunk {
    @PrimaryGeneratedColumn()
    id: number

    @Column("varchar")
    content: string

    // Vector column with 1998 dimensions
    @Column("vector", { length: 1998 })
    embedding: number[]
}
```

#### Векторный поиск по схожести

SQL Server предоставляет функцию `VECTOR_DISTANCE` для вычисления расстояний между векторами:

```typescript
const queryEmbedding = [/* your query vector */]

const results = await dataSource.query(
    `
    DECLARE @question AS VECTOR (1998) = @0;
    SELECT TOP (10) dc.*,
           VECTOR_DISTANCE('cosine', @question, embedding) AS distance
    FROM document_chunk dc
    ORDER BY VECTOR_DISTANCE('cosine', @question, embedding)
`,
    [JSON.stringify(queryEmbedding)],
)
```

**Метрики расстояния:**

- `'cosine'` — косинусное расстояние (наиболее распространено для семантического поиска)
- `'euclidean'` — евклидово расстояние (L2)
- `'dot'` — отрицательное скалярное произведение

**Требования:**

- Версия SQL Server с включённой поддержкой векторов
- Размерность вектора должна быть задана через опцию `length`

## Известные проблемы

### Пул соединений не сбрасывает уровень изоляции

Специфичные для драйвера опции источника данных `options.isolationLevel` и `options.connectionIsolationLevel` корректно применяются при первоначальном создании соединения нижележащим драйвером [node-mssql](https://github.com/tediousjs/node-mssql). Однако `node-mssql` не вызывает `connection.reset()` при возврате соединений в пул. Это означает, что если какая-либо операция изменит уровень изоляции на соединении из пула (например, явная транзакция с другим уровнем), изменение сохранится и «протечёт» к следующему потребителю этого соединения.

На практике это делает `options.isolationLevel` и `options.connectionIsolationLevel` ненадёжными для приложений, которые к тому же используют уровни изоляции на уровне отдельных транзакций.

**Рекомендуемая альтернатива:** используйте вместо них опцию `isolationLevel` верхнего уровня у DataSource (доступна во всех драйверах). Она применяет уровень изоляции явно при старте каждой транзакции, полностью обходя ограничение пула. См. [Транзакции > Уровень изоляции по умолчанию](<../10. Transactions.md#уровень-изоляции-по-умолчанию>).

Это ограничение вышестоящей библиотеки, отслеживаемое в [tediousjs/node-mssql#1483](https://github.com/tediousjs/node-mssql/issues/1483).

# SQLite

## Установка

- для **Better SQLite**:

```shell
npm install better-sqlite3
```

- для **sql.js**:

```shell
npm install sql.js
```

- для **Capacitor**, **Cordova**, **Expo**, **NativeScript** и **React Native** смотрите [поддерживаемые платформы](<../17. Help/02. Supported platforms.md>).

## Опции источника данных

Общие опции источника данных описаны в разделе [Опции источника данных](<../02. Data Source/02. Data Source Options.md>).

### Опции источника данных `better-sqlite3`

- `database` — путь к базе данных. Например, `"mydb.sqlite"`.
- `enableWAL` — включает режим WAL (по умолчанию `false`). См. [SQLite WAL mode](https://www.sqlite.org/wal.html).
- `fileMustExist` — если база данных не существует, будет выброшена ошибка вместо создания нового файла. Не влияет на подключения в памяти и в режиме только для чтения (по умолчанию `false`).
- `key` — ключ шифрования для SQLCipher.
- `nativeBinding` — относительный или абсолютный путь к нативному аддону (`better_sqlite3.node`).
- `prepareDatabase` — функция, которая выполняется перед началом использования базы данных в typeorm. Здесь вы получаете доступ к исходному объекту Database из better-sqlite3.
- `readonly` — открыть подключение к базе данных в режиме только для чтения (по умолчанию `false`).
- `statementCacheSize` — размер кэша выражений SQLite для ускорения запросов (по умолчанию `100`).
- `timeout` — количество миллисекунд ожидания при выполнении запросов к заблокированной базе данных, после которого выбрасывается ошибка SQLITE_BUSY (по умолчанию `5000`).
- `verbose` — функция, вызываемая для каждой SQL-строки, выполненной подключением к базе данных.

### Опции источника данных `sql.js`

- `database` — сырая база данных типа UInt8Array, которую нужно импортировать.
- `sqlJsConfig` — необязательная конфигурация инициализации для sql.js.
- `autoSave` — включает автоматическое сохранение изменений базы данных, требует указания либо `location`, либо `autoSaveCallback`. Если задано `true`, каждое изменение сохраняется в файловую систему (Node.js) или в `localStorage`/`indexedDB` (браузер), если указан `location`, иначе вызывается `autoSaveCallback`.
- `autoSaveCallback` — функция, вызываемая при изменениях базы данных, когда включён `autoSave`. Функция получает `UInt8Array`, представляющий базу данных.
- `location` — путь к файлу, из которого загружается и в который сохраняется база данных.
- `useLocalForage` — включает использование [библиотеки localforage](https://github.com/localForage/localForage) для асинхронного сохранения и загрузки базы данных из indexedDB вместо синхронных методов локального хранилища в браузерной среде. Модуль localforage нужно добавить в проект, а localforage.js — подключить на странице.

### Опции источника данных `capacitor`

- `database` — имя базы данных (capacitor-sqlite добавит суффикс `SQLite.db`)
- `driver` — экземпляр capacitor-sqlite. Например, `new SQLiteConnection(CapacitorSQLite)`.
- `mode` — режим шифрования базы данных: "no-encryption" | "encryption" | "secret" | "newsecret"
- `version` — версия базы данных
- `journalMode` — режим журналирования SQLite (необязательно)

### Опции источника данных `cordova`

- `database` — имя базы данных
- `location` — где сохранять базу данных. Варианты см. в [cordova-sqlite-storage](https://github.com/litehelpers/Cordova-sqlite-storage#opening-a-database).

### Опции источника данных `expo`

- `database` — имя базы данных. Например, "mydb".
- `driver` — модуль Expo SQLite. Необязательный — TypeORM загружает `expo-sqlite` автоматически. Указывайте явно, только если нужен собственный экземпляр (например, пропатченный форк).

### Опции источника данных `nativescript`

- `database` — имя базы данных

### Опции источника данных `react-native`

- `database` — имя базы данных
- `location` — где сохранять базу данных. Варианты см. в [react-native-sqlite-storage](https://github.com/andpor/react-native-sqlite-storage#opening-a-database).

## Типы колонок

`int`, `int2`, `int8`, `integer`, `tinyint`, `smallint`, `mediumint`, `bigint`, `decimal`, `numeric`, `float`, `double`, `real`, `double precision`, `datetime`, `varying character`, `character`, `native character`, `varchar`, `nchar`, `nvarchar2`, `unsigned big int`, `boolean`, `blob`, `text`, `clob`, `date`, `json`, `jsonb`

TypeORM поддерживает в SQLite оба типа — `json` и `jsonb`:

- `json` хранится как `TEXT`.
- `jsonb` хранится в двоичном формате JSON, принятом в SQLite. При сохранении TypeORM автоматически оборачивает значения функцией `jsonb()`, а при чтении — функцией `json()`, что обеспечивает прозрачную поддержку и более высокую производительность.

Поддержка JSONB требует SQLite 3.45.0 или новее. При использовании типа колонки `jsonb` TypeORM применит в схеме базы данных тип `jsonb`, который SQLite внутренне обрабатывает как двоичный `BLOB`.

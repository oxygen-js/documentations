# MongoDB

## Поддержка MongoDB

TypeORM имеет базовую поддержку MongoDB (драйвер Node.js **v7 и выше**).

Большая часть возможностей TypeORM относится к реляционным СУБД, поэтому на этой странице собрано всё, что специфично для MongoDB.

## Установка

```shell
npm install mongodb
```

## Опции источника данных

- `appName` — имя приложения, создавшего этот экземпляр MongoClient. MongoDB выводит это значение в журнал сервера при установлении каждого соединения. Оно также записывается в журнал медленных запросов и в коллекции профилирования.

- `authMechanism` — задаёт механизм аутентификации, который MongoDB будет использовать для аутентификации соединения.

- `authSource` — имя базы данных, связанной с учётными данными пользователя.

- `autoEncryption` — необязательное включение автоматического шифрования in-use.

- `checkServerIdentity` — проверяет, что сертификат `cert` выдан для `hostname`.

- `compressors` — массив или строка со списком компрессоров через запятую для включения сжатия сетевого трафика между этим клиентом и экземпляром mongod/mongos.

- `connectTimeoutMS` — время в миллисекундах на попытку установить соединение до истечения тайм-аута. По умолчанию: `30000`.

- `database` — имя базы данных.

- `directConnection` — позволяет драйверу принудительно использовать топологию типа Single со строкой подключения, содержащей один хост.

- `driver` — объект драйвера. По умолчанию `require("mongodb")`.

- `family` — семейство IP.

- `forceServerObjectId` — заставляет сервер, а не драйвер, назначать значения \_id. По умолчанию: `false`.

- `host` — хост базы данных.

- `hostReplicaSet` — набор реплик хоста базы данных.

- `ignoreUndefined` — определяет, должен ли сериализатор BSON игнорировать поля со значением undefined. По умолчанию: `false`.

- `localThresholdMS` — размер (в миллисекундах) окна задержки для выбора между несколькими подходящими экземплярами MongoDB.

- `maxStalenessSeconds` — указывает в секундах, насколько сильно вторичный узел может отставать, прежде чем клиент перестанет использовать его для операций чтения. Минимум — 90 секунд.

- `minPoolSize` — минимальное количество соединений в пуле соединений.

- `monitorCommands` — включает мониторинг команд для этого клиента.

- `noDelay` — отключение задержки для TCP-соединения.

- `password` — пароль базы данных.

- `pkFactory` — объект-фабрика первичных ключей для генерации собственных ключей \_id.

- `poolSize` — максимальное количество соединений в пуле соединений. Отображается на опцию `maxPoolSize` драйвера MongoDB.

- `port` — порт хоста базы данных. Порт mongodb по умолчанию — `27017`.

- `promoteBuffers` — преобразует значения Binary BSON в нативные Buffer из Node. По умолчанию: `false`.

- `promoteLongs` — преобразует значения Long в number, если они умещаются в 53-битное представление. По умолчанию: `true`.

- `promoteValues` — по возможности преобразует значения BSON в нативные типы; установите false, чтобы получать только обёртки. По умолчанию: `true`.

- `proxyHost` — задаёт хост Socks5-прокси, используемый для создания TCP-соединений.

- `proxyPassword` — задаёт пароль Socks5-прокси, если прокси требует аутентификации по имени пользователя и паролю.

- `proxyPort` — задаёт порт Socks5-прокси, используемый для создания TCP-соединений.

- `proxyUsername` — задаёт имя пользователя Socks5-прокси, если прокси требует аутентификации по имени пользователя и паролю.

- `raw` — возвращать результаты в виде «сырых» BSON-буферов. По умолчанию: `false`.

- `readConcern` — задаёт read concern для коллекции.

- `readPreference` — предпочтительный режим чтения.
    - `ReadPreference.PRIMARY`
    - `ReadPreference.PRIMARY_PREFERRED`
    - `ReadPreference.SECONDARY`
    - `ReadPreference.SECONDARY_PREFERRED`
    - `ReadPreference.NEAREST`

- `readPreferenceTags` — задаёт документ с тегами в виде списка пар «ключ-значение», разделённых двоеточием, перечисленных через запятую.

- `replicaSet` — имя набора реплик, если mongod входит в набор реплик.

- `retryWrites` — включает повторяемые операции записи.

- `serializeFunctions` — сериализовать функции у любого объекта. По умолчанию: `false`.

- `socketTimeoutMS` — время в миллисекундах на попытку отправки или приёма данных через сокет до истечения тайм-аута. По умолчанию: `360000`.

- `tls` — включает или отключает TLS/SSL для соединения. По умолчанию: `false`.

- `tlsAllowInvalidCertificates` — пропускает проверку сертификатов, предоставленных экземпляром mongod/mongos. По умолчанию: `false`.

- `tlsCAFile` — путь к локальному .pem-файлу, содержащему корневую цепочку сертификатов удостоверяющего центра.

- `tlsCertificateKeyFile` — путь к локальному .pem-файлу, содержащему TLS/SSL-сертификат и ключ клиента.

- `tlsCertificateKeyFilePassword` — пароль для расшифровки `tlsCertificateKeyFile`.

- `url` — URL подключения, по которому выполняется соединение. Обратите внимание, что остальные опции источника данных переопределяют параметры, заданные в url.

- `username` — имя пользователя базы данных.

- `writeConcern` — WriteConcern для MongoDB, описывающий требуемый уровень подтверждения операций записи со стороны MongoDB.

Дополнительные опции можно добавить в объект `extra` — они будут переданы напрямую в клиентскую библиотеку. Подробнее — в документации `mongodb`, раздел [Connection Options](https://mongodb-node.netlify.app/docs/drivers/node/current/connect/connection-options/).

## Определение сущностей и колонок

Сущности и колонки определяются почти так же, как в реляционных базах данных;
главное отличие в том, что нужно использовать `@ObjectIdColumn`
вместо `@PrimaryColumn` или `@PrimaryGeneratedColumn`.

Простой пример сущности:

```typescript
import { ObjectId } from "mongodb"
import { Entity, ObjectIdColumn, Column } from "typeorm"

@Entity()
export class User {
    @ObjectIdColumn()
    _id: ObjectId

    @Column()
    firstName: string

    @Column()
    lastName: string
}
```

А так выполняется начальная настройка приложения:

```typescript
import { DataSource } from "typeorm"

const myDataSource = new DataSource({
    type: "mongodb",
    host: "localhost",
    port: 27017,
    database: "test",
})
```

## Определение вложенных документов (embed documents)

Поскольку MongoDB хранит объекты и объекты внутри объектов (или документы внутри документов), то же самое можно делать и в TypeORM:

```typescript
import { ObjectId } from "mongodb"
import { Entity, ObjectIdColumn, Column } from "typeorm"

export class Profile {
    @Column()
    about: string

    @Column()
    education: string

    @Column()
    career: string
}
```

```typescript
import { ObjectId } from "mongodb"
import { Entity, ObjectIdColumn, Column } from "typeorm"

export class Photo {
    @Column()
    url: string

    @Column()
    description: string

    @Column()
    size: number

    constructor(url: string, description: string, size: number) {
        this.url = url
        this.description = description
        this.size = size
    }
}
```

```typescript
import { Entity, ObjectId, ObjectIdColumn, Column } from "typeorm"

@Entity()
export class User {
    @ObjectIdColumn()
    id: ObjectId

    @Column()
    firstName: string

    @Column()
    lastName: string

    @Column((type) => Profile)
    profile: Profile

    @Column((type) => Photo)
    photos: Photo[]
}
```

Если сохранить такую сущность:

```typescript
const user = new User()
user.firstName = "Timber"
user.lastName = "Saw"
user.profile = new Profile()
user.profile.about = "About Trees and Me"
user.profile.education = "Tree School"
user.profile.career = "Lumberjack"
user.photos = [
    new Photo("me-and-trees.jpg", "Me and Trees", 100),
    new Photo("me-and-chakram.jpg", "Me and Chakram", 200),
]

await myDataSource.manager.save(user)
```

В базе данных будет сохранён следующий документ:

```json
{
    "firstName": "Timber",
    "lastName": "Saw",
    "profile": {
        "about": "About Trees and Me",
        "education": "Tree School",
        "career": "Lumberjack"
    },
    "photos": [
        {
            "url": "me-and-trees.jpg",
            "description": "Me and Trees",
            "size": 100
        },
        {
            "url": "me-and-chakram.jpg",
            "description": "Me and Chakram",
            "size": 200
        }
    ]
}
```

## Выборка отдельных полей через `select`

С помощью опции `select` в методах `find*` можно получать только нужные поля. Объект преобразуется в [проекцию](https://www.mongodb.com/docs/manual/tutorial/project-fields-from-query-results/) MongoDB; вложенные встроенные документы описываются объектным синтаксисом и разворачиваются в проекции с точечными путями.

```typescript
const products = await myDataSource.getMongoRepository(Product).find({
    select: { name: true, specs: { weight: true } },
})
```

В примере выше каждый продукт возвращается только с заполненными `name` и `specs.weight` (плюс идентификатор сущности). Неизвестные имена свойств — как на верхнем уровне, так и внутри встроенного документа — приводят к ошибке `EntityPropertyNotFoundError`, как и в SQL-драйверах.

## Использование `MongoEntityManager` и `MongoRepository`

Вам доступно большинство методов `EntityManager` (кроме специфичных для реляционных СУБД, таких как `query` и `transaction`).
Например:

```typescript
const timber = await myDataSource.manager.findOneBy(User, {
    firstName: "Timber",
    lastName: "Saw",
})
```

Для MongoDB также существует отдельный `MongoEntityManager`, расширяющий `EntityManager`.

```typescript
const timber = await myDataSource.manager.findOneBy(User, {
    firstName: "Timber",
    lastName: "Saw",
})
```

Так же, как есть отдельный `MongoEntityManager`, есть и `MongoRepository`, расширяющий `Repository`:

```typescript
const timber = await myDataSource.getMongoRepository(User).findOneBy({
    firstName: "Timber",
    lastName: "Saw",
})
```

Расширенные возможности find():

Equal:

```typescript
const timber = await myDataSource.getMongoRepository(User).find({
    where: {
        firstName: { $eq: "Timber" },
    },
})
```

LessThan:

```typescript
const timber = await myDataSource.getMongoRepository(User).find({
    where: {
        age: { $lt: 60 },
    },
})
```

In:

```typescript
const timber = await myDataSource.getMongoRepository(User).find({
    where: {
        firstName: { $in: ["Timber", "Zhang"] },
    },
})
```

Not in:

```typescript
const timber = await myDataSource.getMongoRepository(User).find({
    where: {
        firstName: { $not: { $in: ["Timber", "Zhang"] } },
    },
})
```

Or:

```typescript
const timber = await myDataSource.getMongoRepository(User).find({
    where: {
        $or: [{ firstName: "Timber" }, { firstName: "Zhang" }],
    },
})
```

Запросы к вложенным документам

```typescript
const users = await myDataSource.getMongoRepository(User).find({
    where: {
        "profile.education": { $eq: "Tree School" },
    },
})
```

Запросы к массиву вложенных документов

```typescript
// Выбрать пользователей, у которых есть фотографии размером меньше 500
const users = await myDataSource.getMongoRepository(User).find({
    where: {
        "photos.size": { $lt: 500 },
    },
})
```

И `MongoEntityManager`, и `MongoRepository` содержат множество полезных методов, специфичных для MongoDB:

### `createCursor`

Создаёт курсор для запроса, с помощью которого можно перебирать результаты из MongoDB.

### `createEntityCursor`

Создаёт курсор для запроса, с помощью которого можно перебирать результаты из MongoDB.
Возвращает изменённую версию курсора, которая преобразует каждый результат в модель сущности.

### `aggregate`

Выполняет конвейер агрегации над коллекцией.

### `bulkWrite`

Выполняет операцию bulkWrite без использования fluent-интерфейса.

### `count`

Подсчитывает количество документов в базе, соответствующих запросу.

### `countDocuments`

Подсчитывает количество документов в базе, соответствующих запросу.

### `createCollectionIndex`

Создаёт индекс в базе данных и коллекции.

### `createCollectionIndexes`

Создаёт несколько индексов в коллекции. Спецификации индексов описаны в [createIndexes](http://docs.mongodb.org/manual/reference/command/createIndexes/).

### `deleteMany`

Удаляет несколько документов в MongoDB.

### `deleteOne`

Удаляет документ в MongoDB.

### `distinct`

Команда distinct возвращает список уникальных значений для заданного ключа по всей коллекции.

### `dropCollectionIndex`

Удаляет индекс из этой коллекции.

### `dropCollectionIndexes`

Удаляет все индексы из коллекции.

### `findOneAndDelete`

Находит документ и удаляет его одной атомарной операцией; на время операции требуется блокировка на запись.

### `findOneAndReplace`

Находит документ и заменяет его одной атомарной операцией; на время операции требуется блокировка на запись.

### `findOneAndUpdate`

Находит документ и обновляет его одной атомарной операцией; на время операции требуется блокировка на запись.

### `geoHaystackSearch`

Выполняет геопоиск по коллекции с использованием индекса geo haystack.

### `geoNear`

Выполняет команду geoNear для поиска объектов в коллекции.

### `group`

Выполняет команду group по коллекции.

### `collectionIndexes`

Возвращает все индексы коллекции.

### `collectionIndexExists`

Проверяет, существует ли индекс в коллекции.

### `collectionIndexInformation`

Возвращает информацию об индексах этой коллекции.

### `initializeOrderedBulkOp`

Начинает упорядоченную операцию массовой записи: операции выполняются последовательно в порядке добавления, при каждой смене типа создаётся новая операция.

### `initializeUnorderedBulkOp`

Начинает неупорядоченную операцию пакетной записи. Все операции буферизуются в команды insert/update/remove, выполняемые в произвольном порядке.

### `insertMany`

Вставляет массив документов в MongoDB.

### `insertOne`

Вставляет один документ в MongoDB.

### `isCapped`

Возвращает признак того, является ли коллекция ограниченной (capped).

### `listCollectionIndexes`

Возвращает список информации обо всех индексах коллекции.

### `parallelCollectionScan`

Возвращает N параллельных курсоров для коллекции, позволяя читать всю коллекцию параллельно. Порядок возвращаемых результатов не гарантируется.

### `reIndex`

Перестраивает все индексы коллекции. Предупреждение: reIndex — блокирующая операция (индексы перестраиваются на переднем плане), и для больших коллекций она будет медленной.

### `rename`

Изменяет имя существующей коллекции.

### `replaceOne`

Заменяет документ в MongoDB.

### `updateMany`

Обновляет несколько документов в коллекции по заданному фильтру.

### `updateOne`

Обновляет один документ в коллекции по заданному фильтру.

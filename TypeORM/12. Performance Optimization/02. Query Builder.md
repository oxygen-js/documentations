# Эффективное использование Query Builder

## Как избежать проблемы N+1 запросов

Проблема N+1 запросов возникает, когда система выполняет слишком много подзапросов для каждой полученной строки данных.

Чтобы этого избежать, используйте `leftJoinAndSelect` или `innerJoinAndSelect` — они объединяют таблицы в одном запросе вместо выполнения нескольких запросов.

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .leftJoinAndSelect("user.posts", "post")
    .getMany()
```

Здесь `leftJoinAndSelect` позволяет получить все посты пользователей одним запросом, а не множеством мелких.

## Используйте `getRawMany()`, когда нужны только сырые данные

Если полные объекты не нужны, можно использовать `getRawMany()` — он возвращает сырые данные и избавляет TypeORM от обработки лишней информации.

```typescript
const rawPosts = await dataSource
    .getRepository(Post)
    .createQueryBuilder("post")
    .select("post.title, post.createdAt")
    .getRawMany()
```

## Ограничивайте набор полей через `select`

Чтобы оптимизировать расход памяти и не тянуть лишние данные, выбирайте через `select` только необходимые поля.

```typescript
const users = await dataSource
    .getRepository(User)
    .createQueryBuilder("user")
    .select(["user.name", "user.email"])
    .getMany()
```

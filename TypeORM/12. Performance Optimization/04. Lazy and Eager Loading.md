# Ленивая и жадная загрузка

TypeORM предлагает два основных способа загрузки данных по связям: ленивую загрузку (lazy loading) и жадную загрузку (eager loading). Каждый из них по-своему влияет на производительность приложения.

## Ленивая загрузка

Ленивая загрузка подгружает данные связи только тогда, когда они нужны, снижая нагрузку на базу данных в случаях, когда все связанные данные требуются не всегда.

```typescript
@Entity()
export class User {
    @OneToMany(() => Post, (post) => post.user, { lazy: true })
    posts: Promise<Post[]>
}
```

Когда данные понадобятся, просто вызовите

```typescript
const user = await userRepository.findOne(userId)
const posts = await user.posts
```

Преимущества:

- Экономия ресурсов: загружаются только те данные, которые действительно нужны, что снижает стоимость запросов и расход памяти.
- Подходит для выборочного использования данных: удобно в сценариях, где нужны не все связанные данные.

Недостатки:

- Усложнение работы с запросами: каждое обращение к связанным данным порождает дополнительный запрос к базе, что при небрежном подходе увеличивает задержки.
- Трудно отследить: при неаккуратном использовании легко получить проблему n+1 запросов.

## Жадная загрузка

Жадная загрузка автоматически подтягивает все связанные данные при выполнении основного запроса. Это удобно, но может привести к проблемам с производительностью, если связей слишком много и они сложные.

```typescript
@Entity()
export class User {
    @OneToMany(() => Post, (post) => post.user, { eager: true })
    posts: Post[]
}
```

В этом случае posts будут загружены сразу же, как только будут получены данные пользователя.

Преимущества:

- Связанные данные загружаются автоматически, поэтому обращаться к связям можно без дополнительных запросов.
- Исключает проблему n+1 запросов: поскольку все данные извлекаются одним запросом, нет риска породить лишние множественные запросы.

Недостатки:

- Одновременная загрузка всех связанных данных может приводить к большим запросам, даже если нужны не все данные.
- Не подходит для сценариев, где требуется лишь часть связанных данных, — это ведёт к неэффективному использованию данных.

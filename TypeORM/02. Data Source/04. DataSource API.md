# DataSource API

- `options` — опции, использованные для создания этого источника данных.
  Подробнее об [опциях источника данных](<02. Data Source Options.md>).

```typescript
const dataSourceOptions: DataSourceOptions = dataSource.options
```

- `isInitialized` — показывает, был ли `DataSource` инициализирован и установлено ли соединение / пул соединений с базой данных.

```typescript
const isInitialized: boolean = dataSource.isInitialized
```

- `driver` — низкоуровневый драйвер базы данных, используемый этим источником данных.

```typescript
const driver: Driver = dataSource.driver
```

- `manager` — `EntityManager`, используемый для работы с сущностями.
  Подробнее об [Entity Manager](<../07. Working with Entity Manager/01. EntityManager.md>) и [репозитории](<../07. Working with Entity Manager/02. Repository.md>).

```typescript
const manager: EntityManager = dataSource.manager
// вы можете вызывать методы менеджера, например find:
const users = await manager.find()
```

- `mongoManager` — `MongoEntityManager`, используемый для работы с сущностями в источнике данных mongodb.
  Подробнее о MongoEntityManager см. в документации по [MongoDB](<../13. Drivers/03. MongoDB.md>).

```typescript
const manager: MongoEntityManager = dataSource.mongoManager
// вы можете вызывать методы менеджера или специфичные для mongodb-менеджера, например find:
const users = await manager.find()
```

- `initialize` — инициализирует источник данных и открывает пул соединений с базой данных.

```typescript
await dataSource.initialize()
```

- `destroy` — уничтожает DataSource и закрывает все соединения с базой данных.
  Обычно этот метод вызывают при завершении работы приложения.

```typescript
await dataSource.destroy()
```

- `synchronize` — синхронизирует схему базы данных. Когда в опциях источника данных задано `synchronize: true`, вызывается именно этот метод.
  Обычно его вызывают при запуске приложения.

```typescript
await dataSource.synchronize()
```

- `dropDatabase` — удаляет базу данных вместе со всеми данными.
  Будьте осторожны с этим методом в продакшене: он сотрёт все таблицы базы данных и их данные.
  Использовать можно только после установки соединения с базой данных.

```typescript
await dataSource.dropDatabase()
```

- `runMigrations` — выполняет все ожидающие миграции.

```typescript
await dataSource.runMigrations()
```

- `undoLastMigration` — откатывает последнюю выполненную миграцию.

```typescript
await dataSource.undoLastMigration()
```

- `hasMetadata` — проверяет, зарегистрированы ли метаданные для указанной сущности.

```typescript
if (dataSource.hasMetadata(User))
    const userMetadata = dataSource.getMetadata(User)
```

- `getMetadata` — возвращает `EntityMetadata` указанной сущности.
  Можно также указать имя таблицы: если метаданные сущности с таким именем таблицы найдены, они будут возвращены.

```typescript
const userMetadata = dataSource.getMetadata(User)
// теперь вы можете получить любую информацию о сущности User
```

- `getRepository` — возвращает `Repository` указанной сущности.
  Можно также указать имя таблицы: если репозиторий для такой таблицы найден, он будет возвращён.
  Подробнее о [репозиториях](<../07. Working with Entity Manager/02. Repository.md>).

```typescript
const repository = dataSource.getRepository(User)
// теперь вы можете вызывать методы репозитория, например find:
const users = await repository.find()
```

- `getTreeRepository` — возвращает `TreeRepository` указанной сущности.
  Можно также указать имя таблицы: если репозиторий для такой таблицы найден, он будет возвращён.
  Подробнее о [репозиториях](<../07. Working with Entity Manager/02. Repository.md>).

```typescript
const repository = dataSource.getTreeRepository(Category)
// теперь вы можете вызывать методы древовидного репозитория, например findTrees:
const categories = await repository.findTrees()
```

- `getMongoRepository` — возвращает `MongoRepository` указанной сущности.
  Этот репозиторий используется для сущностей в источнике данных MongoDB.
  Подробнее о [поддержке MongoDB](<../13. Drivers/03. MongoDB.md>).

```typescript
const repository = dataSource.getMongoRepository(User)
// теперь вы можете вызывать специфичные для mongodb методы репозитория, например createEntityCursor:
const categoryCursor = repository.createEntityCursor()
const category1 = await categoryCursor.next()
const category2 = await categoryCursor.next()
```

- `transaction` — предоставляет единую транзакцию, в рамках которой несколько запросов к базе данных выполняются в одной транзакции.
  Подробнее о [транзакциях](<../10. Transactions.md>).

```typescript
await dataSource.transaction(async (manager) => {
    // ВНИМАНИЕ: все операции с базой данных нужно выполнять через переданный экземпляр manager
    // это специальный экземпляр EntityManager, работающий с этой транзакцией
    // и не забывайте использовать await
})
```

- `query` — выполняет сырой SQL-запрос.

```typescript
const rawData = await dataSource.query(`SELECT * FROM USERS`)

// Также можно использовать параметры, чтобы избежать SQL-инъекций
// Синтаксис различается в зависимости от драйвера

// aurora-mysql, better-sqlite3, capacitor, cordova,
// expo, mariadb, mysql, nativescript, react-native,
// sap, sqlite, sqljs
const rawData = await dataSource.query(
    "SELECT * FROM USERS WHERE name = ? and age = ?",
    ["John", 24],
)

// mysql2 дополнительно поддерживает именованные параметры,
// когда extra.namedPlaceholders равно true
const rawData = await dataSource.query(
    "SELECT * FROM USERS WHERE name = :name and age = :age",
    { name: "John", age: 24 },
)

// aurora-postgres, cockroachdb, postgres
const rawData = await dataSource.query(
    "SELECT * FROM USERS WHERE name = $1 and age = $2",
    ["John", 24],
)

// oracle
const rawData = await dataSource.query(
    "SELECT * FROM USERS WHERE name = :1 and age = :2",
    ["John", 24],
)

// spanner
const rawData = await dataSource.query(
    "SELECT * FROM USERS WHERE name = @param0 and age = @param1",
    ["John", 24],
)

// mssql
const rawData = await dataSource.query(
    "SELECT * FROM USERS WHERE name = @0 and age = @1",
    ["John", 24],
)
```

- `sql` — выполняет сырой SQL-запрос с использованием шаблонных литералов.

```typescript
const rawData =
    await dataSource.sql`SELECT * FROM USERS WHERE name = ${"John"} and age = ${24}`
```

Подробнее об использовании [синтаксиса SQL Tag](<../16. Guides/06. SQL Tag.md>).

- `createQueryBuilder` — создаёт query builder, с помощью которого можно строить запросы.
  Подробнее о [QueryBuilder](<../08. Query Builder/01. Select using Query Builder.md>).

```typescript
const users = await dataSource
    .createQueryBuilder()
    .select()
    .from(User, "user")
    .where("user.name = :name", { name: "John" })
    .getMany()
```

- `createQueryRunner` — создаёт query runner, используемый для управления и работы с одним реальным соединением источника данных.
  Подробнее о [QueryRunner](<../09. Query Runner.md>).

```typescript
const queryRunner = dataSource.createQueryRunner()

// вызывать его методы можно только после вызова connect,
// который выполняет реальное подключение к базе данных
await queryRunner.connect()

// .. теперь вы можете работать с query runner и вызывать его методы

// очень важно — не забудьте освободить query runner, когда закончите с ним работать
await queryRunner.release()
```

# DataSource

## Что такое DataSource?

Взаимодействие с базой данных возможно только после того, как вы настроите `DataSource`.
`DataSource` в TypeORM хранит настройки подключения к базе данных и
устанавливает первоначальное подключение или пул соединений — в зависимости от используемой СУБД.

Чтобы установить первоначальное подключение или пул соединений, нужно вызвать метод `initialize` вашего экземпляра `DataSource`.

Отключение (закрытие всех соединений в пуле) происходит при вызове метода `destroy`.

Обычно метод `initialize` экземпляра `DataSource` вызывают при запуске приложения,
а `destroy` — после завершения работы с базой данных.
На практике, если вы создаёте бэкенд для сайта и ваш сервер работает постоянно, вы никогда не вызываете `destroy` у DataSource.

## Создание нового DataSource

Чтобы создать новый экземпляр `DataSource`, нужно вызвать конструктор через `new DataSource`
и присвоить результат глобальной переменной, которую вы будете использовать во всём приложении:

```typescript
import { DataSource } from "typeorm"

const AppDataSource = new DataSource({
    type: "mysql",
    host: "localhost",
    port: 3306,
    username: "test",
    password: "test",
    database: "test",
})

try {
    await AppDataSource.initialize()
    console.log("Data Source has been initialized!")
} catch (error) {
    console.error("Error during Data Source initialization", error)
}
```

Хорошая идея — сделать `AppDataSource` доступным глобально через `export`, поскольку этот экземпляр будет использоваться во всём приложении.

`DataSource` принимает `DataSourceOptions`, и эти опции различаются в зависимости от используемого типа базы данных (`type`).
Для разных типов баз данных доступны разные опции.

В приложении можно определить сколько угодно источников данных, например:

```typescript
import { DataSource } from "typeorm"

const MysqlDataSource = new DataSource({
    type: "mysql",
    host: "localhost",
    port: 3306,
    username: "test",
    password: "test",
    database: "test",
    entities: [__dirname + "/entities/**/*{.js,.ts}"],
})

const PostgresDataSource = new DataSource({
    type: "postgres",
    host: "localhost",
    port: 5432,
    username: "test",
    password: "test",
    database: "test",
    entities: [__dirname + "/entities/**/*{.js,.ts}"],
})
```

## Как использовать DataSource?

После настройки `DataSource` его можно использовать в любом месте приложения, например:

```typescript
import { AppDataSource } from "./app-data-source"
import { User } from "../entity/User"

export class UserController {
    @Get("/users")
    getAll() {
        return AppDataSource.manager.find(User)
    }
}
```

С помощью экземпляра `DataSource` вы можете выполнять операции с базой данных над своими сущностями,
в первую очередь через свойства `.manager` и `.getRepository()`.
Подробнее о них см. в документации по [Entity Manager](<../07. Working with Entity Manager/01. EntityManager.md>) и [Репозиторию](<../07. Working with Entity Manager/02. Repository.md>).

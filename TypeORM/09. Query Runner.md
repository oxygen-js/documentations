# Query Runner

## Что такое QueryRunner?

Каждый новый экземпляр `QueryRunner` забирает одно соединение из пула соединений, если СУБД поддерживает пул соединений.
Для баз данных, не поддерживающих пулы соединений, используется одно и то же соединение на весь источник данных.

Полный API QueryRunner описан в [разделе о миграциях](<06. Migrations/09. Query Runner API.md>).

## Создание нового экземпляра `QueryRunner`

Для создания нового `QueryRunner` используйте метод `createQueryRunner`:

```typescript
const queryRunner = dataSource.createQueryRunner()
```

## Использование `QueryRunner`

После создания нового экземпляра `QueryRunner` соединение будет получено из пула при выполнении первого запроса:

```typescript
const queryRunner = dataSource.createQueryRunner()
await queryRunner.query("SELECT 1")
await queryRunner.release()
```

Также можно воспользоваться методом `connect`, чтобы получить соединение из пула напрямую:

```typescript
const queryRunner = dataSource.createQueryRunner()
const clientConnection = await queryRunner.connect()
await queryRunner.release()
```

**Важно**: обязательно освобождайте `QueryRunner`, когда он больше не нужен, чтобы вернуть соединение обратно в пул:

```typescript
await queryRunner.release()
```

После освобождения `QueryRunner` пользоваться методами query runner уже нельзя.

У `QueryRunner` есть и собственный экземпляр `EntityManager`, доступный через свойство `manager`: с его помощью можно выполнять запросы `EntityManager` на конкретном соединении с базой данных, которое использует экземпляр `QueryRunner`:

```typescript
let queryRunner: QueryRunner
try {
    queryRunner = dataSource.createQueryRunner()
    // используем одно соединение с базой данных для выполнения нескольких запросов
    await queryRunner.manager.update(
        Employee,
        { level: "junior" },
        { bonus: 0.2 },
    )
    await queryRunner.manager.update(
        Employee,
        { level: "senior" },
        { bonus: 0.1 },
    )
} catch (error) {
    console.error(error)
} finally {
    // не забудьте освободить соединение, когда закончите с ним работать
    await queryRunner.release()
}
```

## Явное управление ресурсами

`QueryRunner` также поддерживает явное управление ресурсами (explicit resource management):

```typescript
async function updateSalaries() {
    await using queryRunner = dataSource.createQueryRunner()
    await queryRunner.manager.update(
        Employee,
        { level: "junior" },
        { bonus: 0.2 },
    )
    await queryRunner.manager.update(
        Employee,
        { level: "senior" },
        { bonus: 0.1 },
    )
    // больше не нужно вручную освобождать QueryRunner
}

try {
    await updateSalaries()
} catch (error) {
    console.error(error)
}
```

При таком объявлении query runner будет освобождён автоматически после выполнения последней инструкции в содержащей его области видимости.

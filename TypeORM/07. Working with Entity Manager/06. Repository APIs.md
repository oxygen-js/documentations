# API репозиториев

## API `Repository`

- `manager` — `EntityManager`, используемый этим репозиторием.

```typescript
const manager = repository.manager
```

- `metadata` — `EntityMetadata` сущности, которой управляет этот репозиторий.

```typescript
const metadata = repository.metadata
```

- `queryRunner` — query runner, используемый `EntityManager`.
  Применяется только в транзакционных экземплярах EntityManager.

```typescript
const queryRunner = repository.queryRunner
```

- `target` — класс целевой сущности, которой управляет этот репозиторий.
  Применяется только в транзакционных экземплярах EntityManager.

```typescript
const target = repository.target
```

- `createQueryBuilder` — создаёт query builder для построения SQL-запросов.
  Подробнее о [QueryBuilder](<../08. Query Builder/01. Select using Query Builder.md>).

```typescript
const users = await repository
    .createQueryBuilder("user")
    .where("user.name = :name", { name: "John" })
    .getMany()
```

- `hasId` — проверяет, определено ли свойство первичной колонки у заданной сущности.

```typescript
if (repository.hasId(user)) {
    // ... делаем что-нибудь
}
```

- `getId` — возвращает значения свойств первичных колонок заданной сущности.
  Если у сущности составной первичный ключ, возвращается объект с именами и значениями первичных колонок.

```typescript
const userId = repository.getId(user) // userId === 1
```

- `create` — создаёт новый экземпляр `User`. Опционально принимает литерал объекта со свойствами пользователя,
  которые будут записаны во вновь созданный объект пользователя

```typescript
const user = repository.create() // то же самое, что const user = new User();
const user = repository.create({
    id: 1,
    firstName: "Timber",
    lastName: "Saw",
}) // то же самое, что const user = new User(); user.firstName = "Timber"; user.lastName = "Saw";
```

- `merge` — объединяет несколько сущностей в одну.

```typescript
const user = new User()
repository.merge(user, { firstName: "Timber" }, { lastName: "Saw" }) // то же самое, что user.firstName = "Timber"; user.lastName = "Saw";
```

- `preload` — создаёт новую сущность из заданного простого JavaScript-объекта. Если сущность уже есть в базе данных,
  она загружается (вместе со всем, что с ней связано), все её значения заменяются новыми из переданного объекта,
  и возвращается новая сущность. По сути, новая сущность — это загруженная из базы данных сущность, у которой все
  свойства заменены значениями из нового объекта.

    > Обратите внимание: переданный объект, похожий на сущность, должен содержать id сущности / первичный ключ, по которому её можно найти. Возвращает undefined, если сущность с указанным id не найдена.

```typescript
const partialUser = {
    id: 1,
    firstName: "Rizzrak",
    profile: {
        id: 1,
    },
}
const user = await repository.preload(partialUser)
// user будет содержать все недостающие в partialUser данные, а также значения свойств из partialUser:
// { id: 1, firstName: "Rizzrak", lastName: "Saw", profile: { id: 1, ... } }
```

- `save` — сохраняет заданную сущность или массив сущностей.
  Если сущность уже есть в базе данных, она обновляется.
  Если сущности в базе данных нет, она вставляется.
  Все переданные сущности сохраняются в одной транзакции (в случае, когда менеджер сущности не транзакционный).
  Также поддерживает частичное обновление, поскольку все свойства со значением undefined пропускаются.
  Возвращает сохранённую сущность или сущности.

```typescript
await repository.save(user)
await repository.save([category1, category2, category3])
```

- `remove` — удаляет заданную сущность или массив сущностей.
  Все переданные сущности удаляются в одной транзакции (в случае, когда менеджер сущности не транзакционный).
  Возвращает удалённую сущность или сущности.

```typescript
await repository.remove(user)
await repository.remove([category1, category2, category3])
```

- `insert` — вставляет новую сущность или массив сущностей.

```typescript
await repository.insert({
    firstName: "Timber",
    lastName: "Timber",
})

await repository.insert([
    {
        firstName: "Foo",
        lastName: "Bar",
    },
    {
        firstName: "Rizz",
        lastName: "Rak",
    },
])
```

- `update` — обновляет сущности по id сущности, по набору id или по заданным условиям. Устанавливает поля из переданной частичной сущности.

```typescript
await repository.update({ age: 18 }, { category: "ADULT" })
// выполняет UPDATE user SET category = ADULT WHERE age = 18

await repository.update(1, { firstName: "Rizzrak" })
// выполняет UPDATE user SET firstName = Rizzrak WHERE id = 1

// при желании можно запросить значения RETURNING / OUTPUT (только для поддерживающих драйверов)
const result = await repository.update(
    1,
    { firstName: "Rizzrak" },
    { returning: ["id", "firstName"] },
)
console.log(result.raw) // [{ id: 1, firstName: "Rizzrak" }]
```

Можно передать **массив объектов-условий**, чтобы за один вызов охватить несколько разных наборов строк (условия объединяются по OR):

```typescript
await repository.update([{ status: "expired" }, { flagged: true }], {
    active: false,
})
// выполняет UPDATE user SET active = false WHERE status = 'expired' OR flagged = true
```

- `updateAll` — обновляет _все_ сущности целевого типа (без условия WHERE). Устанавливает поля из переданной частичной сущности.

```typescript
await repository.updateAll({ category: "ADULT" })
// выполняет UPDATE user SET category = ADULT

await repository.updateAll(
    { category: "ADULT" },
    { returning: "*" }, // только для драйверов, поддерживающих returning-выражения
)
```

- `upsert` — вставляет новую сущность или массив сущностей, а если они уже существуют — обновляет их. Поддерживается драйверами баз данных AuroraDataApi, Cockroach, Mysql, Postgres и Sqlite.

Когда операция upsert приводит к обновлению (из-за конфликта), специальные колонки вроде `@UpdateDateColumn` и `@VersionColumn` автоматически обновляются до актуальных значений.

Колонки, помеченные `update: false`, а также вычисляемые генерируемые колонки (заданные через `asExpression`/`generatedType`) **никогда** не попадают в набор обновляемых значений при конфликте. Если по этим правилам исключены все неконфликтные колонки (то есть обновлять нечего), upsert вырождается в операцию «вставить или проигнорировать», и существующая строка остаётся полностью без изменений. В базах данных, поддерживающих цели конфликта (например, PostgreSQL, CockroachDB), это ограничивается указанными конфликтными колонками; в базах семейства MySQL используется `INSERT IGNORE`, который распространяется на все уникальные ограничения.

```typescript
await repository.upsert(
    [
        { externalId: "abc123", firstName: "Rizzrak" },
        { externalId: "bca321", firstName: "Karzzir" },
    ],
    ["externalId"],
)
/** выполняет
 *  INSERT INTO user
 *  VALUES
 *      (externalId = abc123, firstName = Rizzrak),
 *      (externalId = cba321, firstName = Karzzir),
 *  ON CONFLICT (externalId) DO UPDATE
 *  SET firstName = EXCLUDED.firstName,
 *      updatedDate = CURRENT_TIMESTAMP,
 *      version = version + 1
 **/
```

Можно также запросить возврат значений из upsert (поддерживается драйверами с поддержкой RETURNING / OUTPUT):

```typescript
const { raw } = await repository.upsert(
    { externalId: "abc123", firstName: "Rizzrak" },
    {
        conflictPaths: ["externalId"],
        returning: ["externalId", "firstName"],
    },
)
console.log(raw) // [{ externalId: "abc123", firstName: "Rizzrak" }]
```

```typescript
await repository.upsert(
    [
        { externalId: "abc123", firstName: "Rizzrak" },
        { externalId: "bca321", firstName: "Karzzir" },
    ],
    {
        conflictPaths: ["externalId"],
        skipUpdateIfNoValuesChanged: true, // поддерживается postgres, пропускает обновление, если оно не изменит значения строки
        upsertType: "upsert", //  "on-conflict-do-update" | "on-duplicate-key-update" | "upsert" — опционально задаёт UpsertType; 'upsert' сейчас поддерживается только CockroachDB
    },
)
/** выполняет
 *  INSERT INTO user
 *  VALUES
 *      (externalId = abc123, firstName = Rizzrak),
 *      (externalId = cba321, firstName = Karzzir),
 *  ON CONFLICT (externalId) DO UPDATE
 *  SET firstName = EXCLUDED.firstName
 *  WHERE user.firstName IS DISTINCT FROM EXCLUDED.firstName
 **/
```

```typescript
await repository.upsert(
    [
        { externalId: "abc123", firstName: "Rizzrak", dateAdded: "2020-01-01" },
        { externalId: "bca321", firstName: "Karzzir", dateAdded: "2022-01-01" },
    ],
    {
        conflictPaths: ["externalId"],
        skipUpdateIfNoValuesChanged: true, // поддерживается postgres, пропускает обновление, если оно не изменит значения строки
        indexPredicate: "dateAdded > 2020-01-01", // поддерживается postgres, позволяет использовать частичные индексы
    },
)
/** выполняет
 *  INSERT INTO user
 *  VALUES
 *      (externalId = abc123, firstName = Rizzrak, dateAdded = 2020-01-01),
 *      (externalId = cba321, firstName = Karzzir, dateAdded = 2022-01-01),
 *  ON CONFLICT (externalId) WHERE ( dateAdded > 2021-01-01 ) DO UPDATE
 *  SET firstName = EXCLUDED.firstName,
 *  SET dateAdded = EXCLUDED.dateAdded,
 *  WHERE user.firstName IS DISTINCT FROM EXCLUDED.firstName OR user.dateAdded IS DISTINCT FROM EXCLUDED.dateAdded
 **/
```

- `delete` — удаляет сущности по id сущности, по набору id или по заданным условиям:

```typescript
await repository.delete(1)
await repository.delete([1, 2, 3])
await repository.delete({ firstName: "Timber" })
```

- `deleteAll` — удаляет _все_ сущности целевого типа (без условия WHERE).

```typescript
await repository.deleteAll()
// выполняет DELETE FROM user
```

Обратите внимание также на метод `clear`, который вместо этого выполняет операцию `TRUNCATE TABLE`.

- `softDelete` и `restore` — мягкое удаление и восстановление строки по id, набору id, заданным условиям или массиву объектов-условий:

```typescript
const repository = dataSource.getRepository(Entity)
// мягко удаляем сущность
await repository.softDelete(1)
// а восстановить её можно с помощью restore;
await repository.restore(1)
// мягкое удаление нескольких сущностей
await repository.softDelete([1, 2, 3])
// или мягкое удаление по другому атрибуту
await repository.softDelete({ firstName: "Jake" })

// массовое мягкое удаление с разными условиями для каждой операции
await repository.softDelete([{ firstName: "Jake" }, { age: 25 }, { id: 42 }])
// выполняет три отдельных запроса UPDATE (проставляя метку времени deletedAt):
// UPDATE entity SET deletedAt = NOW() WHERE firstName = Jake
// UPDATE entity SET deletedAt = NOW() WHERE age = 25
// UPDATE entity SET deletedAt = NOW() WHERE id = 42
```

- `softRemove` и `recover` — альтернатива для `softDelete` и `restore`.

```typescript
// мягко удалить их можно с помощью softRemove
const entities = await repository.find()
const entitiesAfterSoftRemove = await repository.softRemove(entities)

// а восстановить — с помощью recover;
await repository.recover(entitiesAfterSoftRemove)
```

- `increment` — увеличивает значение некоторой колонки на заданную величину для сущностей, подходящих под заданные условия.

```typescript
await repository.increment({ firstName: "Timber" }, "age", 3)
```

- `decrement` — уменьшает значение некоторой колонки на заданную величину для сущностей, подходящих под заданные условия.

```typescript
await repository.decrement({ firstName: "Timber" }, "age", 3)
```

- `exists` — проверяет, существует ли хотя бы одна сущность, подходящая под `FindOptions`.

```typescript
const exists = await repository.exists({
    where: {
        firstName: "Timber",
    },
})
```

- `existsBy` — проверяет, существует ли хотя бы одна сущность, подходящая под `FindOptionsWhere`.

```typescript
const exists = await repository.existsBy({ firstName: "Timber" })
```

- `count` — подсчитывает сущности, подходящие под `FindOptions`. Удобно для постраничной выборки.

```typescript
const count = await repository.count({
    where: {
        firstName: "Timber",
    },
})

const distinctCount = await repository.count({
    select: {
        firstName: true,
    },
})
```

Если задан `select`, подсчитываются уникальные комбинации выбранных колонок, включая выбранные колонки связей, если `select` используется вместе с `relations`.
Если `select` не задан, поведение подсчёта остаётся стандартным.

- `countBy` — подсчитывает сущности, подходящие под `FindOptionsWhere`. Удобно для постраничной выборки.

```typescript
const count = await repository.countBy({ firstName: "Timber" })
```

- `sum` — возвращает сумму числового поля по всем сущностям, подходящим под `FindOptionsWhere`.

```typescript
const sum = await repository.sum("age", { firstName: "Timber" })
```

- `average` — возвращает среднее значение числового поля по всем сущностям, подходящим под `FindOptionsWhere`.

```typescript
const average = await repository.average("age", { firstName: "Timber" })
```

- `minimum` — возвращает минимальное значение числового поля по всем сущностям, подходящим под `FindOptionsWhere`.

```typescript
const minimum = await repository.minimum("age", { firstName: "Timber" })
```

- `maximum` — возвращает максимальное значение числового поля по всем сущностям, подходящим под `FindOptionsWhere`.

```typescript
const maximum = await repository.maximum("age", { firstName: "Timber" })
```

- `find` — находит сущности, подходящие под заданные `FindOptions`.

```typescript
const timbers = await repository.find({
    where: {
        firstName: "Timber",
    },
})
```

- `findBy` — находит сущности, подходящие под заданные `FindWhereOptions`.

```typescript
const timbers = await repository.findBy({
    firstName: "Timber",
})
```

- `findAndCount` — находит сущности, подходящие под заданные `FindOptions`.
  Дополнительно подсчитывает все сущности, подходящие под заданные условия,
  но игнорирует настройки постраничной выборки (опции from и take).

```typescript
const [timbers, timbersCount] = await repository.findAndCount({
    where: {
        firstName: "Timber",
    },
})
```

- `findAndCountBy` — находит сущности, подходящие под заданные `FindOptionsWhere`.
  Дополнительно подсчитывает все сущности, подходящие под заданные условия,
  но игнорирует настройки постраничной выборки (опции from и take).

```typescript
const [timbers, timbersCount] = await repository.findAndCountBy({
    firstName: "Timber",
})
```

- `findOne` — находит первую сущность, подходящую под заданные `FindOptions`.

```typescript
const timber = await repository.findOne({
    where: {
        firstName: "Timber",
    },
})
```

- `findOneBy` — находит первую сущность, подходящую под заданные `FindOptionsWhere`.

```typescript
const timber = await repository.findOneBy({ firstName: "Timber" })
```

- `findOneOrFail` — находит первую сущность, подходящую под заданный id или параметры поиска.
  Отклоняет возвращаемый промис, если ничего не найдено.

```typescript
const timber = await repository.findOneOrFail({
    where: {
        firstName: "Timber",
    },
})
```

- `findOneByOrFail` — находит первую сущность, подходящую под заданные `FindOptions`.
  Отклоняет возвращаемый промис, если ничего не найдено.

```typescript
const timber = await repository.findOneByOrFail({ firstName: "Timber" })
```

- `query` — выполняет сырой SQL-запрос.

```typescript
const rawData = await repository.query(`SELECT * FROM USERS`)

// также можно использовать параметры, чтобы избежать SQL-инъекций
// синтаксис различается в зависимости от драйвера

// aurora-mysql, better-sqlite3, capacitor, cordova,
// expo, mariadb, mysql, nativescript, react-native,
// sap, sqlite, sqljs
const rawData = await repository.query(
    "SELECT * FROM USERS WHERE name = ? and age = ?",
    ["John", 24],
)

// aurora-postgres, cockroachdb, postgres
const rawData = await repository.query(
    "SELECT * FROM USERS WHERE name = $1 and age = $2",
    ["John", 24],
)

// oracle
const rawData = await repository.query(
    "SELECT * FROM USERS WHERE name = :1 and age = :2",
    ["John", 24],
)

// spanner
const rawData = await repository.query(
    "SELECT * FROM USERS WHERE name = @param0 and age = @param1",
    ["John", 24],
)

// mssql
const rawData = await repository.query(
    "SELECT * FROM USERS WHERE name = @0 and age = @1",
    ["John", 24],
)

// mysql2 дополнительно поддерживает именованные плейсхолдеры,
// когда extra.namedPlaceholders равно true
const rawData = await repository.query(
    "SELECT * FROM USERS WHERE name = :name and age = :age",
    { name: "John", age: 24 },
)
```

- `clear` — очищает (усекает) все данные заданной таблицы. Поддерживает опцию cascade, позволяющую также очистить данные в таблицах, ссылающихся на эту таблицу внешними ключами (поддерживается только PostgreSQL/CockroachDB и Oracle; остальные базы данных выбросят ошибку, если опция cascade равна true).

```typescript
await repository.clear()

// с опцией cascade (только PostgreSQL/CockroachDB и Oracle)
await repository.clear({ cascade: true })
```

### Дополнительные опции

В `save` можно передать необязательные `SaveOptions`.

- `data` — дополнительные данные, передаваемые вместе с методом сохранения. Эти данные затем можно использовать в подписчиках.
- `listeners`: boolean — указывает, вызываются ли слушатели и подписчики для этой операции. По умолчанию они включены; отключить их можно, задав `{ listeners: false }` в опциях save/remove.
- `transaction`: boolean — по умолчанию транзакции включены, и все запросы в операции сохранения оборачиваются в транзакцию. Это поведение можно отключить, задав `{ transaction: false }` в опциях сохранения.
- `chunk`: number — разбивает выполнение сохранения на несколько групп-порций. Например, если нужно сохранить 100 000 объектов, но с этим возникают проблемы, их можно разбить на 10 групп по 10 000 объектов (задав `{ chunk: 10000 }`) и сохранять каждую группу отдельно. Эта опция нужна для очень больших вставок, когда вы упираетесь в ограничение нижележащего драйвера на количество параметров.
- `reload`: boolean — флаг, определяющий, нужно ли перезагружать сохраняемую сущность в ходе операции сохранения. Работает только в базах данных, не поддерживающих выражение RETURNING / OUTPUT. Включён по умолчанию.

Пример:

```typescript
userRepository.save(users, { chunk: 1000 })
```

В `remove` и `delete` можно передать необязательные `RemoveOptions`.

- `data` — дополнительные данные, передаваемые вместе с методом удаления. Эти данные затем можно использовать в подписчиках.
- `listeners`: boolean — указывает, вызываются ли слушатели и подписчики для этой операции. По умолчанию они включены; отключить их можно, задав `{ listeners: false }` в опциях save/remove.
- `transaction`: boolean — по умолчанию транзакции включены, и все запросы в операции сохранения оборачиваются в транзакцию. Это поведение можно отключить, задав `{ transaction: false }` в опциях сохранения.
- `chunk`: number — разбивает выполнение удаления на несколько групп-порций. Например, если нужно удалить 100 000 объектов, но с этим возникают проблемы, их можно разбить на 10 групп по 10 000 объектов, задав `{ chunk: 10000 }`, и удалять каждую группу отдельно. Эта опция нужна для очень больших удалений, когда вы упираетесь в ограничение нижележащего драйвера на количество параметров.

Пример:

```typescript
userRepository.remove(users, { chunk: 1000 })
```

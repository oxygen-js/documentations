# Пользовательские репозитории

Вы можете создать пользовательский репозиторий, в котором будут собраны методы для работы с вашей базой данных.
Например, допустим, нам нужен метод `findByName(firstName: string, lastName: string)`,
который будет искать пользователей по заданным имени и фамилии.
Лучшее место для такого метода — `Repository`,
чтобы вызывать его как `userRepository.findByName(...)`.
Этого можно добиться с помощью пользовательских репозиториев.

Создать пользовательский репозиторий можно несколькими способами.

- [Как создать пользовательский репозиторий](#как-создать-пользовательский-репозиторий)
- [Использование пользовательских репозиториев в транзакциях](#использование-пользовательских-репозиториев-в-транзакциях)

## Как создать пользовательский репозиторий?

Распространённая практика — присвоить экземпляр репозитория глобально экспортируемой переменной
и использовать эту переменную по всему приложению, например:

```ts
// user.repository.ts
export const UserRepository = dataSource.getRepository(User)

// user.controller.ts
export class UserController {
    users() {
        return UserRepository.find()
    }
}
```

Чтобы расширить возможности `UserRepository`, воспользуйтесь методом `.extend` класса `Repository`:

```typescript
// user.repository.ts
export const UserRepository = dataSource.getRepository(User).extend({
    findByName(firstName: string, lastName: string) {
        return this.createQueryBuilder("user")
            .where("user.firstName = :firstName", { firstName })
            .andWhere("user.lastName = :lastName", { lastName })
            .getMany()
    },
})

// user.controller.ts
export class UserController {
    users() {
        return UserRepository.findByName("Timber", "Saw")
    }
}
```

## Использование пользовательских репозиториев в транзакциях

У транзакций своя область выполнения: собственные экземпляры query runner, entity manager и репозиториев.
Поэтому глобальный entity manager (принадлежащий источнику данных) и глобальные репозитории в транзакциях работать не будут.
Чтобы запросы корректно выполнялись в рамках транзакции, вы **обязаны** использовать предоставленный entity manager
и его метод `getRepository`. А чтобы использовать в транзакции пользовательские репозитории,
воспользуйтесь методом `withRepository` предоставленного экземпляра entity manager:

```typescript
await dataSource.transaction(async (manager) => {
    // в транзакциях вы ОБЯЗАНЫ использовать экземпляр manager, предоставленный транзакцией,
    // нельзя использовать глобальные entity manager или репозитории,
    // потому что этот manager монопольный и транзакционный

    const userRepository = manager.withRepository(UserRepository)
    await userRepository.createAndSave("Timber", "Saw")
    const timber = await userRepository.findByName("Timber", "Saw")
})
```

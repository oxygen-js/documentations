# Опции find

## Базовые опции

Все методы `.find*` репозитория и менеджера принимают специальные опции, с помощью которых можно получить нужные данные без `QueryBuilder`:

- `select` — указывает, какие свойства основного объекта нужно выбрать

```typescript
userRepository.find({
    select: {
        firstName: true,
        lastName: true,
    },
})
```

выполнит следующий запрос:

```sql
SELECT "firstName", "lastName" FROM "user"
```

- `relations` — связи, которые нужно загрузить вместе с основной сущностью. Можно загружать и вложенные связи (сокращённая запись для `join` и `leftJoinAndSelect`)

```typescript
userRepository.find({
    relations: {
        profile: true,
        photos: true,
        videos: true,
    },
})
userRepository.find({
    relations: {
        profile: true,
        photos: true,
        videos: {
            videoAttributes: true,
        },
    },
})
```

выполнит следующие запросы:

```sql
SELECT * FROM "user"
LEFT JOIN "profile" ON "profile"."id" = "user"."profileId"
LEFT JOIN "photos" ON "photos"."id" = "user"."photoId"
LEFT JOIN "videos" ON "videos"."id" = "user"."videoId"

SELECT * FROM "user"
LEFT JOIN "profile" ON "profile"."id" = "user"."profileId"
LEFT JOIN "photos" ON "photos"."id" = "user"."photoId"
LEFT JOIN "videos" ON "videos"."id" = "user"."videoId"
LEFT JOIN "video_attributes" ON "video_attributes"."id" = "videos"."video_attributesId"
```

- `relationLoadStrategy` — управляет способом загрузки связей: `"join"` (по умолчанию) использует SQL-соединения JOIN, `"query"` — отдельные запросы. Подробнее см. [Жадные и ленивые связи](<../04. Relations/05. Eager and Lazy Relations.md#стратегия-загрузки-связей>).

- `loadEagerRelations` — управляет тем, загружаются ли автоматически жадные связи (помеченные `eager: true`). По умолчанию `true`. Подробнее см. [Жадные и ленивые связи](<../04. Relations/05. Eager and Lazy Relations.md#стратегия-загрузки-связей>).

- `where` — простые условия, по которым нужно искать сущность.

```typescript
userRepository.find({
    where: {
        firstName: "Timber",
        lastName: "Saw",
    },
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "user"
WHERE "firstName" = 'Timber' AND "lastName" = 'Saw'
```

Запрос к колонке встраиваемой сущности должен учитывать иерархию, в которой она объявлена. Пример:

```typescript
userRepository.find({
    relations: {
        project: true,
    },
    where: {
        project: {
            name: "TypeORM",
            initials: "TORM",
        },
    },
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "user"
LEFT JOIN "project" ON "project"."id" = "user"."projectId"
WHERE "project"."name" = 'TypeORM' AND "project"."initials" = 'TORM'
```

Запрос с оператором OR:

```typescript
userRepository.find({
    where: [
        { firstName: "Timber", lastName: "Saw" },
        { firstName: "Stan", lastName: "Lee" },
    ],
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "user" WHERE ("firstName" = 'Timber' AND "lastName" = 'Saw') OR ("firstName" = 'Stan' AND "lastName" = 'Lee')
```

- `order` — порядок выборки.

```typescript
userRepository.find({
    order: {
        name: "ASC",
        id: "DESC",
    },
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "user"
ORDER BY "name" ASC, "id" DESC
```

- `withDeleted` — включать в выборку сущности, мягко удалённые через `softDelete` или `softRemove`, то есть те, у которых заполнена колонка `@DeleteDateColumn`. По умолчанию мягко удалённые сущности не включаются.

```typescript
userRepository.find({
    withDeleted: true,
})
```

Методы `find*`, возвращающие несколько сущностей (`find`, `findBy`, `findAndCount`, `findAndCountBy`), принимают также следующие опции:

- `skip` — смещение (для постраничной выборки), с которого нужно брать сущности.

```typescript
userRepository.find({
    skip: 5,
})
```

```sql
SELECT * FROM "user"
OFFSET 5
```

- `take` — ограничение (для постраничной выборки) — максимальное количество выбираемых сущностей.

```typescript
userRepository.find({
    take: 10,
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "user"
LIMIT 10
```

\*\* `skip` и `take` следует использовать вместе

\*\* Если вы работаете с typeorm и MSSQL и хотите использовать `take` или `limit`, вам также придётся задать порядок сортировки, иначе вы получите ошибку: `'Invalid usage of the option NEXT in the FETCH statement.'`

```typescript
userRepository.find({
    order: {
        columnName: "ASC",
    },
    skip: 0,
    take: 10,
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "user"
ORDER BY "columnName" ASC
LIMIT 10 OFFSET 0
```

- `cache` — включает или отключает кэширование результатов запроса. Подробности и доступные опции см. в разделе [кэширование](<../08. Query Builder/06. Caching queries.md>).

```typescript
userRepository.find({
    cache: true,
})
```

- `lock` — включает механизм блокировок для запроса. Может использоваться только в методах `findOne` и `findOneBy`.
  `lock` — это объект, который можно задать так:

```ts
{ mode: "optimistic", version: number | Date }
```

или

```ts
{
    mode: "pessimistic_read" |
        "pessimistic_write" |
        "dirty_read" |
        "for_no_key_update" |
        "for_key_share",

    tables: string[],
    onLocked: "nowait" | "skip_locked"
}
```

например:

```typescript
userRepository.findOne({
    where: {
        id: 1,
    },
    lock: { mode: "optimistic", version: 1 },
})
```

Подробнее см. [режимы блокировок](<../08. Query Builder/01. Select using Query Builder.md#режимы-блокировки>).

## Пример

Полный пример опций find:

```typescript
userRepository.find({
    select: {
        firstName: true,
        lastName: true,
    },
    relations: {
        profile: true,
        photos: true,
        videos: true,
    },
    where: {
        firstName: "Timber",
        lastName: "Saw",
        profile: {
            userName: "tshaw",
        },
    },
    order: {
        name: "ASC",
        id: "DESC",
    },
    lock: {
        mode: "pessimistic_write",
        onLocked: "nowait",
    },
    skip: 5,
    take: 10,
    cache: true,
})
```

API подсчёта `count` тоже принимает эти опции find. Если использовать `select`, можно запросить подсчёт уникальных значений:

```typescript
const distinctCount = await userRepository.count({
    select: {
        firstName: true,
    },
})
```

Если в `select` входят поля связей, добавьте соответствующие записи в `relations` — тогда подсчёт уникальных значений будет выполнен по этим выбранным колонкам связей.

Вызов find без аргументов:

```ts
userRepository.find()
```

выполнит следующий запрос:

```sql
SELECT * FROM "user"
```

## Расширенные опции

TypeORM предоставляет множество встроенных операторов, с помощью которых можно строить более сложные сравнения:

- `Not`

```ts
import { Not } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    title: Not("About #1"),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "title" != 'About #1'
```

- `LessThan`

```ts
import { LessThan } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    likes: LessThan(10),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "likes" < 10
```

- `LessThanOrEqual`

```ts
import { LessThanOrEqual } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    likes: LessThanOrEqual(10),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "likes" <= 10
```

- `MoreThan`

```ts
import { MoreThan } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    likes: MoreThan(10),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "likes" > 10
```

- `MoreThanOrEqual`

```ts
import { MoreThanOrEqual } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    likes: MoreThanOrEqual(10),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "likes" >= 10
```

- `Equal`

```ts
import { Equal } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    title: Equal("About #2"),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "title" = 'About #2'
```

- `Like`

```ts
import { Like } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    title: Like("%out #%"),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "title" LIKE '%out #%'
```

- `ILike`

```ts
import { ILike } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    title: ILike("%out #%"),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "title" ILIKE '%out #%'
```

- `Between`

```ts
import { Between } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    likes: Between(1, 10),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "likes" BETWEEN 1 AND 10
```

- `In`

```ts
import { In } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    title: In(["About #2", "About #3"]),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "title" IN ('About #2','About #3')
```

- `Any`

```ts
import { Any } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    title: Any(["About #2", "About #3"]),
})
```

выполнит следующий запрос (в нотации Postgres):

```sql
SELECT * FROM "post" WHERE "title" = ANY(['About #2','About #3'])
```

- `IsNull`

```ts
import { IsNull } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    title: IsNull(),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "title" IS NULL
```

- `ArrayContains`

```ts
import { ArrayContains } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    categories: ArrayContains(["TypeScript"]),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "categories" @> '{TypeScript}'
```

- `ArrayContainedBy`

```ts
import { ArrayContainedBy } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    categories: ArrayContainedBy(["TypeScript"]),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "categories" <@ '{TypeScript}'
```

- `ArrayOverlap`

```ts
import { ArrayOverlap } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    categories: ArrayOverlap(["TypeScript"]),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "categories" && '{TypeScript}'
```

- `JsonContains` (только PostgreSQL/CockroachDB)

```ts
import { JsonContains } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    metadata: JsonContains({ author: { name: "John" } }),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "metadata" ::jsonb @> '{"author":{"name":"John"}}'
```

`JsonContains` принимает и массивы, поэтому к колонкам jsonb, хранящим JSON-массивы,
можно обращаться напрямую:

```ts
import { JsonContains } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    tags: JsonContains([{ name: "TypeScript" }]),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "tags" ::jsonb @> '[{"name":"TypeScript"}]'
```

- `Raw`

```ts
import { Raw } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    likes: Raw("dislikes - 4"),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "likes" = "dislikes" - 4
```

В простейшем случае сырой запрос подставляется сразу после знака равенства.
Но с помощью функции можно полностью переписать логику сравнения.

```ts
import { Raw } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    currentDate: Raw((alias) => `${alias} > NOW()`),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "currentDate" > NOW()
```

Если вам нужно подставить пользовательский ввод, не включайте его в запрос напрямую — так можно получить уязвимость к SQL-инъекциям. Вместо этого воспользуйтесь вторым аргументом функции `Raw`, чтобы передать список параметров для привязки к запросу.

```ts
import { Raw } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    currentDate: Raw((alias) => `${alias} > :date`, { date: "2020-10-06" }),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "currentDate" > '2020-10-06'
```

Если пользовательский ввод представляет собой массив, его можно привязать как список значений в SQL-выражении, используя специальный синтаксис:

```ts
import { Raw } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    title: Raw((alias) => `${alias} IN (:...titles)`, {
        titles: [
            "Go To Statement Considered Harmful",
            "Structured Programming",
        ],
    }),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "title" IN ('Go To Statement Considered Harmful', 'Structured Programming')
```

## Комбинирование расширенных опций

Эти операторы также можно комбинировать со следующими:

- `Not`

```ts
import { Not, MoreThan, Equal } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    likes: Not(MoreThan(10)),
    title: Not(Equal("About #2")),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE NOT("likes" > 10) AND NOT("title" = 'About #2')
```

- `Or`

```ts
import { Or, Equal, ILike } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    title: Or(Equal("About #2"), ILike("About%")),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE "title" = 'About #2' OR "title" ILIKE 'About%'
```

- `And`

```ts
import { And, Not, Equal, ILike } from "typeorm"

const loadedPosts = await dataSource.getRepository(Post).findBy({
    title: And(Not(Equal("About #2")), ILike("%About%")),
})
```

выполнит следующий запрос:

```sql
SELECT * FROM "post" WHERE NOT("title" = 'About #2') AND "title" ILIKE '%About%'
```

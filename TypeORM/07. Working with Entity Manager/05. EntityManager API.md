# `EntityManager` API

- `dataSource` — DataSource, используемый `EntityManager`.

```typescript
const dataSource = manager.dataSource
```

- `queryRunner` — query runner, используемый `EntityManager`.
  Применяется только в транзакционных экземплярах EntityManager.

```typescript
const queryRunner = manager.queryRunner
```

- `transaction` — предоставляет транзакцию, в рамках которой несколько запросов к базе данных выполняются в одной транзакции.
  Подробнее о [транзакциях](<../10. Transactions.md>).

```typescript
await manager.transaction(async (manager) => {
    // ВНИМАНИЕ: все операции с базой данных нужно выполнять через переданный экземпляр manager
    // это специальный экземпляр EntityManager, работающий с этой транзакцией
    // и не забывайте использовать await
})
```

- `query` — выполняет сырой SQL-запрос.

```typescript
const rawData = await manager.query(`SELECT * FROM USERS`)

// Также можно использовать параметры, чтобы избежать SQL-инъекций
// Синтаксис различается в зависимости от драйвера

// aurora-mysql, better-sqlite3, capacitor, cordova,
// expo, mariadb, mysql, nativescript, react-native,
// sap, sqlite, sqljs
const rawData = await manager.query(
    "SELECT * FROM USERS WHERE name = ? and age = ?",
    ["John", 24],
)

// aurora-postgres, cockroachdb, postgres
const rawData = await manager.query(
    "SELECT * FROM USERS WHERE name = $1 and age = $2",
    ["John", 24],
)

// oracle
const rawData = await manager.query(
    "SELECT * FROM USERS WHERE name = :1 and age = :2",
    ["John", 24],
)

// spanner
const rawData = await manager.query(
    "SELECT * FROM USERS WHERE name = @param0 and age = @param1",
    ["John", 24],
)

// mssql
const rawData = await manager.query(
    "SELECT * FROM USERS WHERE name = @0 and age = @1",
    ["John", 24],
)
```

- `sql` — выполняет сырой SQL-запрос с использованием шаблонных литералов.

```typescript
const rawData =
    await manager.sql`SELECT * FROM USERS WHERE name = ${"John"} and age = ${24}`
```

Подробнее об использовании [синтаксиса SQL Tag](<../16. Guides/06. SQL Tag.md>).

- `createQueryBuilder` — создаёт query builder для построения SQL-запросов.
  Подробнее о [QueryBuilder](<../08. Query Builder/01. Select using Query Builder.md>).

```typescript
const users = await manager
    .createQueryBuilder()
    .select()
    .from(User, "user")
    .where("user.name = :name", { name: "John" })
    .getMany()
```

- `hasId` — проверяет, задано ли у переданной сущности свойство первичной колонки.

```typescript
if (manager.hasId(user)) {
    // ... сделать что-нибудь
}
```

- `getId` — возвращает значение свойства первичной колонки переданной сущности.
  Если у сущности составной первичный ключ, возвращается объект с именами и значениями первичных колонок.

```typescript
const userId = manager.getId(user) // userId === 1
```

- `create` — создаёт новый экземпляр `User`. Опционально принимает объектный литерал со свойствами пользователя,
  которые будут записаны во вновь созданный объект.

```typescript
const user = manager.create(User) // то же, что const user = new User();
const user = manager.create(User, {
    id: 1,
    firstName: "Timber",
    lastName: "Saw",
}) // то же, что const user = new User(); user.firstName = "Timber"; user.lastName = "Saw";
```

- `merge` — объединяет несколько сущностей в одну.

```typescript
const user = new User()
manager.merge(User, user, { firstName: "Timber" }, { lastName: "Saw" }) // то же, что user.firstName = "Timber"; user.lastName = "Saw";
```

- `preload` — создаёт новую сущность из переданного простого JavaScript-объекта. Если сущность уже существует в базе данных,
  метод загружает её (и всё связанное с ней), заменяет все значения новыми из переданного объекта
  и возвращает новую сущность. По сути, новая сущность — это загруженная из базы данных сущность,
  все свойства которой заменены значениями из нового объекта.

```typescript
const partialUser = {
    id: 1,
    firstName: "Rizzrak",
    profile: {
        id: 1,
    },
}
const user = await manager.preload(User, partialUser)
// user будет содержать все недостающие в partialUser данные плюс значения свойств partialUser:
// { id: 1, firstName: "Rizzrak", lastName: "Saw", profile: { id: 1, ... } }
```

- `save` — сохраняет переданную сущность или массив сущностей.
  Если сущность уже существует в базе данных, она обновляется.
  Если сущности в базе данных ещё нет, она вставляется.
  Все переданные сущности сохраняются в одной транзакции (если entity manager не транзакционный).
  Также поддерживает частичное обновление, поскольку все свойства со значением undefined пропускаются. Чтобы записать значение `NULL`, нужно вручную присвоить свойству `null`.

```typescript
await manager.save(user)
await manager.save([category1, category2, category3])
```

- `remove` — удаляет переданную сущность или массив сущностей.
  Все переданные сущности удаляются в одной транзакции (если менеджер не транзакционный).

```typescript
await manager.remove(user)
await manager.remove([category1, category2, category3])
```

- `insert` — вставляет новую сущность или массив сущностей.

```typescript
await manager.insert(User, {
    firstName: "Timber",
    lastName: "Timber",
})

await manager.insert(User, [
    {
        firstName: "Foo",
        lastName: "Bar",
    },
    {
        firstName: "Rizz",
        lastName: "Rak",
    },
])
```

- `update` — обновляет сущности по id, набору id, заданным условиям или массиву объектов-условий. Устанавливает поля из переданной частичной сущности.

```typescript
await manager.update(User, { age: 18 }, { category: "ADULT" })
// выполняет UPDATE user SET category = ADULT WHERE age = 18

await manager.update(User, 1, { firstName: "Rizzrak" })
// выполняет UPDATE user SET firstName = Rizzrak WHERE id = 1

// Массовые обновления с разными условиями для каждой операции
await manager.update(User, [
    { criteria: { id: 1 }, data: { firstName: "Rizzrak" } },
    { criteria: { id: 2 }, data: { firstName: "Karzzir" } },
    { criteria: { age: 18 }, data: { category: "ADULT" } },
])
// выполняет три отдельных запроса UPDATE:
// UPDATE user SET firstName = Rizzrak WHERE id = 1
// UPDATE user SET firstName = Karzzir WHERE id = 2
// UPDATE user SET category = ADULT WHERE age = 18
```

Можно передать **массив объектов-условий**, чтобы за один вызов затронуть несколько разных наборов строк (условия объединяются через OR):

```typescript
await manager.update(User, [{ status: "expired" }, { flagged: true }], {
    active: false,
})
// выполняет UPDATE user SET active = false WHERE status = 'expired' OR flagged = true
```

- `updateAll` — обновляет _все_ сущности целевого типа (без условия WHERE). Устанавливает поля из переданной частичной сущности.

```typescript
await manager.updateAll(User, { category: "ADULT" })
// выполняет UPDATE user SET category = ADULT
```

- `upsert` — вставляет новую сущность или массив сущностей, а если они уже существуют — обновляет их. Поддерживается драйверами AuroraDataApi, Cockroach, Mysql, Postgres и Sqlite.

Когда операция upsert приводит к обновлению (из-за конфликта), специальные колонки вроде `@UpdateDateColumn` и `@VersionColumn` автоматически получают актуальные значения.

Колонки, помеченные `update: false`, а также вычисляемые генерируемые колонки (заданные через `asExpression`/`generatedType`) **никогда** не попадают в набор обновляемых полей при конфликте. Если по этим правилам исключены все неконфликтные колонки (то есть обновлять нечего), upsert вырождается в операцию «вставить или проигнорировать», и существующая строка остаётся полностью неизменной. В базах данных, поддерживающих указание конфликтных колонок (например, PostgreSQL, CockroachDB), это ограничено заданными конфликтными колонками; в базах семейства MySQL используется `INSERT IGNORE`, который применяется ко всем уникальным ограничениям.

```typescript
await manager.upsert(
    User,
    [
        { externalId: "abc123", firstName: "Rizzrak" },
        { externalId: "bca321", firstName: "Karzzir" },
    ],
    ["externalId"],
)
/** выполняет
 *  INSERT INTO user
 *  VALUES
 *      (externalId = abc123, firstName = Rizzrak),
 *      (externalId = cba321, firstName = Karzzir),
 *  ON CONFLICT (externalId) DO UPDATE firstName = EXCLUDED.firstName
 **/
```

- `delete` — удаляет сущности по id, набору id, заданным условиям или массиву объектов-условий.

```typescript
await manager.delete(User, 1)
await manager.delete(User, [1, 2, 3])
await manager.delete(User, { firstName: "Timber" })

// Массовые удаления с разными условиями для каждой операции
await manager.delete(User, [{ firstName: "Timber" }, { age: 18 }, { id: 42 }])
// выполняет три отдельных запроса DELETE:
// DELETE FROM user WHERE firstName = Timber
// DELETE FROM user WHERE age = 18
// DELETE FROM user WHERE id = 42
```

Можно передать **массив объектов-условий**, чтобы за один вызов затронуть несколько разных наборов строк (условия объединяются через OR). Это отличается от передачи массива примитивов, который трактуется как список идентификаторов:

```typescript
// Массив примитивов — интерпретируется как WHERE id IN (1, 2, 3)
await manager.delete(User, [1, 2, 3])

// Массив объектов — каждый элемент является отдельным условием (объединяются через OR)
await manager.delete(User, [{ id: 1 }, { email: "old@example.com" }])
// выполняет DELETE FROM user WHERE id = 1 OR email = 'old@example.com'
```

Обратите внимание: передача пустого объекта `{}` или `[{}]` вызывает `TypeORMError`, чтобы предотвратить случайное удаление всей таблицы.

- `deleteAll` — удаляет _все_ сущности целевого типа (без условия WHERE).

```typescript
await manager.deleteAll(User)
// выполняет DELETE FROM user
```

См. также метод `clear`, который вместо этого выполняет операцию `TRUNCATE TABLE`.

- `increment` — увеличивает значение указанной колонки на заданную величину для сущностей, подходящих под заданные условия.

```typescript
await manager.increment(User, { firstName: "Timber" }, "age", 3)
```

- `decrement` — уменьшает значение указанной колонки на заданную величину для сущностей, подходящих под заданные условия.

```typescript
await manager.decrement(User, { firstName: "Timber" }, "age", 3)
```

- `exists` — проверяет, существует ли хотя бы одна сущность, соответствующая `FindOptions`.

```typescript
const exists = await manager.exists(User, {
    where: {
        firstName: "Timber",
    },
})
```

- `existsBy` — проверяет, существует ли хотя бы одна сущность, соответствующая `FindOptionsWhere`.

```typescript
const exists = await manager.existsBy(User, { firstName: "Timber" })
```

- `count` — подсчитывает сущности, соответствующие `FindOptions`. Полезно для постраничной выборки.

```typescript
const count = await manager.count(User, {
    where: {
        firstName: "Timber",
    },
})

const distinctCount = await manager.count(User, {
    select: {
        firstName: true,
    },
})
```

Если задан `select`, подсчитываются уникальные комбинации выбранных колонок, включая колонки выбранных связей, когда `select` используется вместе с `relations`.
Если `select` не задан, поведение подсчёта остаётся стандартным.

- `countBy` — подсчитывает сущности, соответствующие `FindOptionsWhere`. Полезно для постраничной выборки.

```typescript
const count = await manager.countBy(User, { firstName: "Timber" })
```

- `find` — находит сущности, соответствующие заданным `FindOptions`.

```typescript
const timbers = await manager.find(User, {
    where: {
        firstName: "Timber",
    },
})
```

- `findBy` — находит сущности, соответствующие заданным `FindWhereOptions`.

```typescript
const timbers = await manager.findBy(User, {
    firstName: "Timber",
})
```

- `findAndCount` — находит сущности, соответствующие заданным `FindOptions`.
  Также подсчитывает все сущности, удовлетворяющие заданным условиям,
  но игнорирует настройки постраничной выборки (опции from и take).

```typescript
const [timbers, timbersCount] = await manager.findAndCount(User, {
    where: {
        firstName: "Timber",
    },
})
```

- `findAndCountBy` — находит сущности, соответствующие заданным `FindOptionsWhere`.
  Также подсчитывает все сущности, удовлетворяющие заданным условиям,
  но игнорирует настройки постраничной выборки (опции from и take).

```typescript
const [timbers, timbersCount] = await manager.findAndCountBy(User, {
    firstName: "Timber",
})
```

- `findOne` — находит первую сущность, соответствующую заданным `FindOptions`.

```typescript
const timber = await manager.findOne(User, {
    where: {
        firstName: "Timber",
    },
})
```

- `findOneBy` — находит первую сущность, соответствующую заданным `FindOptionsWhere`.

```typescript
const timber = await manager.findOneBy(User, { firstName: "Timber" })
```

- `findOneOrFail` — находит первую сущность, соответствующую какому-либо id или опциям поиска.
  Если ничего не найдено, возвращённый promise отклоняется.

```typescript
const timber = await manager.findOneOrFail(User, {
    where: {
        firstName: "Timber",
    },
})
```

- `findOneByOrFail` — находит первую сущность, соответствующую заданным `FindOptions`.
  Если ничего не найдено, возвращённый promise отклоняется.

```typescript
const timber = await manager.findOneByOrFail(User, { firstName: "Timber" })
```

- `clear` — очищает (усекает) указанную таблицу, удаляя из неё все данные. Поддерживает опцию cascade, чтобы также очистить таблицы, имеющие внешние ключи на эту таблицу (поддерживается только в PostgreSQL/CockroachDB и Oracle; остальные базы данных выбрасывают ошибку, если опция cascade установлена в true).

```typescript
await manager.clear(User)

// С опцией cascade (только PostgreSQL/CockroachDB и Oracle)
await manager.clear(User, { cascade: true })
```

- `getRepository` — возвращает `Repository` для выполнения операций над конкретной сущностью.
  Подробнее о [репозиториях](<02. Repository.md>).

```typescript
const userRepository = manager.getRepository(User)
```

- `getTreeRepository` — возвращает `TreeRepository` для выполнения операций над конкретной сущностью.
  Подробнее о [репозиториях](<02. Repository.md>).

```typescript
const categoryRepository = manager.getTreeRepository(Category)
```

- `getMongoRepository` — возвращает `MongoRepository` для выполнения операций над конкретной сущностью.
  Подробнее о [MongoDB](<../13. Drivers/03. MongoDB.md>).

```typescript
const userRepository = manager.getMongoRepository(User)
```

- `withRepository` — возвращает экземпляр пользовательского репозитория, используемый в транзакции.
  Подробнее о [пользовательских репозиториях](<04. Custom repositories.md>).

```typescript
const myUserRepository = manager.withRepository(UserRepository)
```

- `release` — освобождает query runner менеджера сущностей.
  Используется только тогда, когда query runner был создан и управляется вручную.

```typescript
await manager.release()
```

### Гибридное приложение

Гибридное приложение — это приложение, которое принимает запросы из двух или более разных источников. Оно может сочетать HTTP-сервер со слушателем микросервиса или же объединять несколько разных слушателей микросервисов. Стандартный метод `createMicroservice` не позволяет создать несколько серверов, поэтому в таком случае каждый микросервис приходится создавать и запускать вручную. Для этого экземпляр `INestApplication` можно связать с экземплярами `INestMicroservice` через метод `connectMicroservice()`.

```typescript
const app = await NestFactory.create(AppModule);
const microservice = app.connectMicroservice<MicroserviceOptions>({
  transport: Transport.TCP,
});

await app.startAllMicroservices();
await app.listen(3001);
```

> **Подсказка** метод `app.listen(port)` запускает HTTP-сервер по указанному адресу. Если ваше приложение не обрабатывает HTTP-запросы, вместо него следует использовать метод `app.init()`.

Чтобы подключить несколько экземпляров микросервисов, вызовите `connectMicroservice()` для каждого из них:

```typescript
const app = await NestFactory.create(AppModule);
// микросервис №1
const microserviceTcp = app.connectMicroservice<MicroserviceOptions>({
  transport: Transport.TCP,
  options: {
    port: 3001,
  },
});
// микросервис №2
const microserviceRedis = app.connectMicroservice<MicroserviceOptions>({
  transport: Transport.REDIS,
  options: {
    host: 'localhost',
    port: 6379,
  },
});

await app.startAllMicroservices();
await app.listen(3001);
```

Чтобы в гибридном приложении с несколькими микросервисами привязать `@MessagePattern()` только к одной транспортной стратегии (например, MQTT), можно передать второй аргумент типа `Transport` — перечисление, в котором определены все встроенные транспортные стратегии.

```typescript
@MessagePattern('time.us.*', Transport.NATS)
getDate(@Payload() data: number[], @Ctx() context: NatsContext) {
  console.log(`Subject: ${context.getSubject()}`); // например, "time.us.east"
  return new Date().toLocaleTimeString(...);
}
@MessagePattern({ cmd: 'time.us' }, Transport.TCP)
getTCPDate(@Payload() data: number[]) {
  return new Date().toLocaleTimeString(...);
}
```

> **Подсказка** `@Payload()`, `@Ctx()`, `Transport` и `NatsContext` импортируются из `@nestjs/microservices`.

#### Общая конфигурация

По умолчанию гибридное приложение не наследует глобальные pipe-ы, interceptor-ы, guard-ы и фильтры, настроенные для основного (HTTP) приложения.
Чтобы унаследовать эти настройки от основного приложения, задайте свойство `inheritAppConfig` во втором аргументе вызова `connectMicroservice()` (это необязательный объект параметров):

```typescript
const microservice = app.connectMicroservice<MicroserviceOptions>(
  {
    transport: Transport.TCP,
  },
  { inheritAppConfig: true },
);
```

### Сырое тело запроса

Один из самых частых сценариев, когда нужен доступ к сырому телу запроса, — проверка подписи вебхуков. Обычно для такой проверки требуется несериализованное тело запроса, чтобы вычислить HMAC-хэш.

> **Предупреждение** Эта возможность работает только при включённом встроенном глобальном middleware разбора тела запроса, то есть при создании приложения нельзя передавать `bodyParser: false`.

#### Использование с Express

Сначала включите соответствующую опцию при создании приложения Nest Express:

```typescript
import { NestFactory } from '@nestjs/core';
import type { NestExpressApplication } from '@nestjs/platform-express';
import { AppModule } from './app.module';

// в функции "bootstrap"
const app = await NestFactory.create<NestExpressApplication>(AppModule, {
  rawBody: true,
});
await app.listen(process.env.PORT ?? 3000);
```

Для доступа к сырому телу запроса в контроллере предусмотрен удобный интерфейс `RawBodyRequest`, который добавляет к запросу поле `rawBody`: используйте тип интерфейса `RawBodyRequest`:

```typescript
import { Controller, Post, RawBodyRequest, Req } from '@nestjs/common';
import { Request } from 'express';

@Controller('cats')
class CatsController {
  @Post()
  create(@Req() req: RawBodyRequest<Request>) {
    const raw = req.rawBody; // возвращает `Buffer`.
  }
}
```

#### Регистрация другого парсера

По умолчанию регистрируются только парсеры `json` и `urlencoded`. Если вам нужно на лету зарегистрировать другой парсер, это придётся сделать явно.

Например, чтобы зарегистрировать парсер `text`, можно использовать такой код:

```typescript
app.useBodyParser('text');
```

> **Предупреждение** Убедитесь, что в вызов `NestFactory.create` передан правильный тип приложения. Для приложений Express правильный тип — `NestExpressApplication`. Иначе метод `.useBodyParser` не будет найден.

#### Ограничение размера тела запроса

Если вашему приложению нужно разбирать тело больше стандартных для Express `100kb`, используйте следующее:

```typescript
app.useBodyParser('json', { limit: '10mb' });
```

Метод `.useBodyParser` учитывает опцию `rawBody`, переданную в опциях приложения.

#### Использование с Fastify

Сначала включите соответствующую опцию при создании приложения Nest Fastify:

```typescript
import { NestFactory } from '@nestjs/core';
import {
  FastifyAdapter,
  NestFastifyApplication,
} from '@nestjs/platform-fastify';
import { AppModule } from './app.module';

// в функции "bootstrap"
const app = await NestFactory.create<NestFastifyApplication>(
  AppModule,
  new FastifyAdapter(),
  {
    rawBody: true,
  },
);
await app.listen(process.env.PORT ?? 3000);
```

Для доступа к сырому телу запроса в контроллере предусмотрен удобный интерфейс `RawBodyRequest`, который добавляет к запросу поле `rawBody`: используйте тип интерфейса `RawBodyRequest`:

```typescript
import { Controller, Post, RawBodyRequest, Req } from '@nestjs/common';
import { FastifyRequest } from 'fastify';

@Controller('cats')
class CatsController {
  @Post()
  create(@Req() req: RawBodyRequest<FastifyRequest>) {
    const raw = req.rawBody; // возвращает `Buffer`.
  }
}
```

#### Регистрация другого парсера

По умолчанию регистрируются только парсеры `application/json` и `application/x-www-form-urlencoded`. Если вам нужно на лету зарегистрировать другой парсер, это придётся сделать явно.

Например, чтобы зарегистрировать парсер `text/plain`, можно использовать такой код:

```typescript
app.useBodyParser('text/plain');
```

> **Предупреждение** Убедитесь, что в вызов `NestFactory.create` передан правильный тип приложения. Для приложений Fastify правильный тип — `NestFastifyApplication`. Иначе метод `.useBodyParser` не будет найден.

#### Ограничение размера тела запроса

Если вашему приложению нужно разбирать тело больше стандартного для Fastify 1 МиБ, используйте следующее:

```typescript
const bodyLimit = 10_485_760; // 10MiB
app.useBodyParser('application/json', { bodyLimit });
```

Метод `.useBodyParser` учитывает опцию `rawBody`, переданную в опциях приложения.

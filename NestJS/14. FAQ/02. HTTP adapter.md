### HTTP-адаптер

Иногда может потребоваться доступ к нижележащему HTTP-серверу — как изнутри контекста приложения Nest, так и снаружи.

Каждый экземпляр нативного (платформозависимого) HTTP-сервера или библиотеки (например, Express и Fastify) обёрнут в **адаптер**. Адаптер регистрируется как глобально доступный провайдер, который можно получить из контекста приложения, а также внедрить в другие провайдеры.

#### Стратегия вне контекста приложения

Чтобы получить ссылку на `HttpAdapter` вне контекста приложения, вызовите метод `getHttpAdapter()`.

```typescript
const app = await NestFactory.create(AppModule);
const httpAdapter = app.getHttpAdapter();
```

#### Через внедрение зависимостей

Чтобы получить ссылку на `HttpAdapterHost` внутри контекста приложения, внедрите его тем же способом, что и любой другой существующий провайдер (например, через внедрение в конструктор).

```typescript
export class CatsService {
  constructor(private adapterHost: HttpAdapterHost) {}
}
```

> **Подсказка** `HttpAdapterHost` импортируется из пакета `@nestjs/core`.

`HttpAdapterHost` **не** является собственно `HttpAdapter`. Чтобы получить сам экземпляр `HttpAdapter`, просто обратитесь к свойству `httpAdapter`.

```typescript
const adapterHost = app.get(HttpAdapterHost);
const httpAdapter = adapterHost.httpAdapter;
```

`httpAdapter` — это фактический экземпляр HTTP-адаптера, используемого нижележащим фреймворком. Это экземпляр либо `ExpressAdapter`, либо `FastifyAdapter` (оба класса наследуются от `AbstractHttpAdapter`).

Объект адаптера предоставляет несколько полезных методов для взаимодействия с HTTP-сервером. Однако если вам нужен прямой доступ к экземпляру библиотеки (например, к экземпляру Express), вызовите метод `getInstance()`.

```typescript
const instance = httpAdapter.getInstance();
```

#### Событие начала прослушивания

Чтобы выполнить некоторое действие в момент, когда сервер начинает принимать входящие запросы, можно подписаться на поток `listen$`, как показано ниже:

```typescript
this.httpAdapterHost.listen$.subscribe(() =>
  console.log('HTTP server is listening'),
);
```

Кроме того, `HttpAdapterHost` предоставляет булево свойство `listening`, которое показывает, активен ли сервер и принимает ли он соединения в данный момент:

```typescript
if (this.httpAdapterHost.listening) {
  console.log('HTTP server is listening');
}
```

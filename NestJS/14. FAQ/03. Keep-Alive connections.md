### Keep-alive соединения

По умолчанию HTTP-адаптеры NestJS дожидаются завершения ответа и только затем закрывают приложение. Но иногда такое поведение нежелательно или оказывается неожиданным. Часть запросов может использовать заголовок `Connection: Keep-Alive` и жить очень долго.

Для случаев, когда приложение должно завершаться всегда, не дожидаясь окончания запросов, при создании приложения NestJS можно включить опцию `forceCloseConnections`.

> **Совет** Большинству пользователей эта опция не понадобится. Признак того, что она нужна, — приложение не завершается тогда, когда вы этого ожидаете. Обычно это проявляется при включённом `app.enableShutdownHooks()`, когда вы замечаете, что приложение не перезапускается и не завершается. Чаще всего — при запуске приложения NestJS в режиме разработки с флагом `--watch`.

#### Использование

В файле `main.ts` включите эту опцию при создании приложения NestJS:

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule, {
    forceCloseConnections: true,
  });
  await app.listen(process.env.PORT ?? 3000);
}

bootstrap();
```

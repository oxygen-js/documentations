### Адаптеры

Модуль WebSockets не привязан к конкретной платформе, поэтому вы можете подключить собственную библиотеку (или даже нативную реализацию), воспользовавшись интерфейсом `WebSocketAdapter`. Этот интерфейс требует реализовать несколько методов, описанных в таблице ниже:

<table>
  <tr>
    <td><code>create</code></td>
    <td>Создаёт экземпляр сокета на основе переданных аргументов</td>
  </tr>
  <tr>
    <td><code>bindClientConnect</code></td>
    <td>Привязывает событие подключения клиента</td>
  </tr>
  <tr>
    <td><code>bindClientDisconnect</code></td>
    <td>Привязывает событие отключения клиента (необязательно*)</td>
  </tr>
  <tr>
    <td><code>bindMessageHandlers</code></td>
    <td>Привязывает входящее сообщение к соответствующему обработчику сообщений</td>
  </tr>
  <tr>
    <td><code>close</code></td>
    <td>Завершает работу экземпляра сервера</td>
  </tr>
</table>

#### Расширение socket.io

Пакет [socket.io](https://github.com/socketio/socket.io) обёрнут в класс `IoAdapter`. Что делать, если вы хотите расширить базовую функциональность адаптера? Например, ваши технические требования предполагают возможность рассылать события между несколькими экземплярами веб-сервиса за балансировщиком нагрузки. Для этого вы можете унаследоваться от `IoAdapter` и переопределить единственный метод, отвечающий за создание новых серверов socket.io. Но прежде всего установим необходимый пакет.

> **Предупреждение** Чтобы использовать socket.io с несколькими экземплярами за балансировщиком нагрузки, вам нужно либо отключить polling, задав `transports: ['websocket']` в конфигурации socket.io на клиенте, либо включить маршрутизацию на основе cookie в балансировщике нагрузки. Одного Redis недостаточно. Подробнее смотрите [здесь](https://socket.io/docs/v4/using-multiple-nodes/#enabling-sticky-session).

```bash
$ npm i --save redis socket.io @socket.io/redis-adapter
```

После установки пакета мы можем создать класс `RedisIoAdapter`.

```typescript
import { IoAdapter } from '@nestjs/platform-socket.io';
import { ServerOptions } from 'socket.io';
import { createAdapter } from '@socket.io/redis-adapter';
import { createClient } from 'redis';

export class RedisIoAdapter extends IoAdapter {
  private adapterConstructor: ReturnType<typeof createAdapter>;

  async connectToRedis(): Promise<void> {
    const pubClient = createClient({ url: `redis://localhost:6379` });
    const subClient = pubClient.duplicate();

    await Promise.all([pubClient.connect(), subClient.connect()]);

    this.adapterConstructor = createAdapter(pubClient, subClient);
  }

  createIOServer(port: number, options?: ServerOptions): any {
    const server = super.createIOServer(port, options);
    server.adapter(this.adapterConstructor);
    return server;
  }
}
```

После этого просто переключитесь на только что созданный адаптер Redis.

```typescript
const app = await NestFactory.create(AppModule);
const redisIoAdapter = new RedisIoAdapter(app);
await redisIoAdapter.connectToRedis();

app.useWebSocketAdapter(redisIoAdapter);
```

#### Библиотека ws

Ещё один доступный адаптер — `WsAdapter`, который выступает в роли прослойки между фреймворком и молниеносно быстрой, тщательно протестированной библиотекой [ws](https://github.com/websockets/ws). Этот адаптер полностью совместим с нативными браузерными WebSocket и работает значительно быстрее пакета socket.io. К сожалению, «из коробки» он предоставляет заметно меньше возможностей. Впрочем, в ряде случаев они вам и не нужны.

> **Подсказка** Библиотека `ws` не поддерживает пространства имён (каналы связи, популяризированные `socket.io`). Однако, чтобы как-то воспроизвести эту возможность, вы можете разместить несколько серверов `ws` по разным путям (например: `@WebSocketGateway({ path: '/users' })`).

Чтобы использовать `ws`, сначала нужно установить необходимый пакет:

```bash
$ npm i --save @nestjs/platform-ws
```

После установки пакета мы можем переключить адаптер:

```typescript
const app = await NestFactory.create(AppModule);
app.useWebSocketAdapter(new WsAdapter(app));
```

> **Подсказка** `WsAdapter` импортируется из `@nestjs/platform-ws`.

`wsAdapter` рассчитан на обработку сообщений в формате `{ event: string, data: any }`. Если вам нужно принимать и обрабатывать сообщения в другом формате, потребуется настроить парсер сообщений, который приведёт их к требуемому формату.

```typescript
const wsAdapter = new WsAdapter(app, {
  // Для обработки сообщений в формате [event, data]
  messageParser: (data) => {
    const [event, payload] = JSON.parse(data.toString());
    return { event, data: payload };
  },
});
```

Кроме того, вы можете настроить парсер сообщений после создания адаптера с помощью метода `setMessageParser`.

#### Продвинутый вариант (пользовательский адаптер)

В демонстрационных целях мы интегрируем библиотеку [ws](https://github.com/websockets/ws) вручную. Как уже упоминалось, адаптер для этой библиотеки уже создан и доступен из пакета `@nestjs/platform-ws` в виде класса `WsAdapter`. Вот как могла бы выглядеть упрощённая реализация:

**`ws-adapter.ts`**

```typescript
import * as WebSocket from 'ws';
import { WebSocketAdapter, INestApplicationContext } from '@nestjs/common';
import { MessageMappingProperties } from '@nestjs/websockets';
import { Observable, fromEvent, EMPTY } from 'rxjs';
import { mergeMap, filter } from 'rxjs/operators';

export class WsAdapter implements WebSocketAdapter {
  constructor(private app: INestApplicationContext) {}

  create(port: number, options: any = {}): any {
    return new WebSocket.Server({ port, ...options });
  }

  bindClientConnect(server, callback: Function) {
    server.on('connection', callback);
  }

  bindMessageHandlers(
    client: WebSocket,
    handlers: MessageMappingProperties[],
    process: (data: any) => Observable<any>,
  ) {
    fromEvent(client, 'message')
      .pipe(
        mergeMap(data => this.bindMessageHandler(data, handlers, process)),
        filter(result => result),
      )
      .subscribe(response => client.send(JSON.stringify(response)));
  }

  bindMessageHandler(
    buffer,
    handlers: MessageMappingProperties[],
    process: (data: any) => Observable<any>,
  ): Observable<any> {
    const message = JSON.parse(buffer.data);
    const messageHandler = handlers.find(
      handler => handler.message === message.event,
    );
    if (!messageHandler) {
      return EMPTY;
    }
    return process(messageHandler.callback(message.data));
  }

  close(server) {
    server.close();
  }
}
```

> **Подсказка** Если вы хотите воспользоваться библиотекой [ws](https://github.com/websockets/ws), используйте встроенный `WsAdapter`, а не создавайте собственный.

Затем мы можем задать пользовательский адаптер с помощью метода `useWebSocketAdapter()`:

**`main.ts`**

```typescript
const app = await NestFactory.create(AppModule);
app.useWebSocketAdapter(new WsAdapter(app));
```

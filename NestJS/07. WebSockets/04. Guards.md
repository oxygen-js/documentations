### Guards

Принципиальной разницы между guard'ами для веб-сокетов и [обычными guard'ами HTTP-приложения](<../02. Overview/08. Guards.md>) нет. Единственное отличие в том, что вместо `HttpException` следует выбрасывать `WsException`.

> **Подсказка** Класс `WsException` экспортируется из пакета `@nestjs/websockets`.

#### Привязка guard'ов

В примере ниже используется guard уровня метода. Как и в HTTP-приложениях, можно применять guard'ы уровня шлюза (то есть пометить класс шлюза декоратором `@UseGuards()`).

```typescript
@UseGuards(AuthGuard)
@SubscribeMessage('events')
handleEvent(client: Client, data: unknown): WsResponse<unknown> {
  const event = 'events';
  return { event, data };
}
```

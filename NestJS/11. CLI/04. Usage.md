### Справочник команд CLI

#### nest new

Создаёт новый проект Nest (в стандартном режиме).

```bash
$ nest new <name> [options]
$ nest n <name> [options]
```

##### Описание

Создаёт и инициализирует новый проект Nest. Запрашивает менеджер пакетов.

- Создаёт папку с заданным именем `<name>`
- Наполняет папку файлами конфигурации
- Создаёт вложенные папки для исходного кода (`/src`) и сквозных (end-to-end) тестов (`/test`)
- Наполняет вложенные папки файлами по умолчанию для компонентов приложения и тестов

##### Аргументы

| Аргумент | Описание                    |
| -------- | --------------------------- |
| `<name>` | Имя нового проекта          |

##### Опции

| Опция                                 | Описание                                                                                                                                                                                             |
| ------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--dry-run`                           | Сообщает, какие изменения были бы внесены, но не изменяет файловую систему.<br/> Псевдоним: `-d`                                                                                                     |
| `--skip-git`                          | Пропустить инициализацию git-репозитория.<br/> Псевдоним: `-g`                                                                                                                                       |
| `--skip-install`                      | Пропустить установку пакетов.<br/> Псевдоним: `-s`                                                                                                                                                   |
| `--package-manager [package-manager]` | Указать менеджер пакетов. Используйте `npm`, `yarn` или `pnpm`. Менеджер пакетов должен быть установлен глобально.<br/> Псевдоним: `-p`                                                              |
| `--language [language]`               | Указать язык программирования (`TS` или `JS`).<br/> Псевдоним: `-l`                                                                                                                                  |
| `--collection [collectionName]`       | Указать коллекцию схематиков. Используйте имя установленного npm-пакета, содержащего схематик.<br/> Псевдоним: `-c`                                                                                  |
| `--strict`                            | Запустить проект со следующими включёнными флагами компилятора TypeScript: `strictNullChecks`, `noImplicitAny`, `strictBindCallApply`, `forceConsistentCasingInFileNames`, `noFallthroughCasesInSwitch` |

#### nest generate

Генерирует и/или изменяет файлы на основе схематика

```bash
$ nest generate <schematic> <name> [options]
$ nest g <schematic> <name> [options]
```

##### Аргументы

| Аргумент      | Описание                                                                                                    |
| ------------- | ------------------------------------------------------------------------------------------------------------ |
| `<schematic>` | Генерируемый `schematic` или `collection:schematic`. Доступные схематики перечислены в таблице ниже.        |
| `<name>`      | Имя генерируемого компонента.                                                                               |

##### Схематики

| Название      | Псевдоним | Описание                                                                                                           |
| ------------- | ----- | ---------------------------------------------------------------------------------------------------------------------- |
| `app`         |       | Сгенерировать новое приложение внутри монорепозитория (преобразовав проект в монорепозиторий, если структура стандартная). |
| `library`     | `lib` | Сгенерировать новую библиотеку внутри монорепозитория (преобразовав проект в монорепозиторий, если структура стандартная). |
| `class`       | `cl`  | Сгенерировать новый класс.                                                                                         |
| `controller`  | `co`  | Сгенерировать объявление контроллера.                                                                              |
| `decorator`   | `d`   | Сгенерировать пользовательский декоратор.                                                                          |
| `filter`      | `f`   | Сгенерировать объявление фильтра.                                                                                  |
| `gateway`     | `ga`  | Сгенерировать объявление шлюза.                                                                                    |
| `guard`       | `gu`  | Сгенерировать объявление guard.                                                                                    |
| `interface`   | `itf` | Сгенерировать интерфейс.                                                                                           |
| `interceptor` | `itc` | Сгенерировать объявление interceptor.                                                                              |
| `middleware`  | `mi`  | Сгенерировать объявление middleware.                                                                               |
| `module`      | `mo`  | Сгенерировать объявление модуля.                                                                                   |
| `pipe`        | `pi`  | Сгенерировать объявление pipe.                                                                                     |
| `provider`    | `pr`  | Сгенерировать объявление провайдера.                                                                               |
| `resolver`    | `r`   | Сгенерировать объявление резолвера.                                                                                |
| `resource`    | `res` | Сгенерировать новый CRUD-ресурс. Подробнее — в разделе [CRUD (resource) generator](<../13. Recipes/02. CRUD generator.md>). (только TS) |
| `service`     | `s`   | Сгенерировать объявление сервиса.                                                                                  |

##### Опции

| Опция                           | Описание                                                                                                        |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| `--dry-run`                     | Сообщает, какие изменения были бы внесены, но не изменяет файловую систему.<br/> Псевдоним: `-d`                |
| `--project [project]`           | Проект, в который следует добавить элемент.<br/> Псевдоним: `-p`                                                |
| `--flat`                        | Не создавать папку для элемента.                                                                                |
| `--collection [collectionName]` | Указать коллекцию схематиков. Используйте имя установленного npm-пакета, содержащего схематик.<br/> Псевдоним: `-c` |
| `--spec`                        | Принудительно генерировать spec-файлы (по умолчанию)                                                            |
| `--no-spec`                     | Отключить генерацию spec-файлов                                                                                 |

#### nest build

Компилирует приложение или рабочее пространство в выходную папку.

Кроме того, команда `build` отвечает за:

- сопоставление путей (если используются псевдонимы путей) через `tsconfig-paths`
- аннотирование DTO декораторами OpenAPI (если включён CLI-плагин `@nestjs/swagger`)
- аннотирование DTO декораторами GraphQL (если включён CLI-плагин `@nestjs/graphql`)

```bash
$ nest build <name> [options]
```

##### Аргументы

| Аргумент | Описание                          |
| -------- | --------------------------------- |
| `<name>` | Имя собираемого проекта.          |

##### Опции

| Опция                   | Описание                                                                                                                                                                                   |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--path [path]`         | Путь к файлу `tsconfig`. <br/>Псевдоним `-p`                                                                                                                                               |
| `--config [path]`       | Путь к файлу конфигурации `nest-cli`. <br/>Псевдоним `-c`                                                                                                                                  |
| `--watch`               | Запуск в режиме наблюдения (live-reload).<br /> Если для компиляции используется `tsc`, можно ввести `rs`, чтобы перезапустить приложение (когда опция `manualRestart` имеет значение `true`). <br/>Псевдоним `-w` |
| `--builder [name]`      | Указать сборщик для компиляции (`tsc`, `swc` или `webpack`). <br/>Псевдоним `-b`                                                                                                           |
| `--webpack`             | Использовать webpack для компиляции (устарело: используйте `--builder webpack`).                                                                                                           |
| `--webpackPath`         | Путь к конфигурации webpack.                                                                                                                                                               |
| `--tsc`                 | Принудительно использовать `tsc` для компиляции.                                                                                                                                           |
| `--watchAssets`         | Следить за файлами, не относящимися к TS (ресурсы вроде `.graphql` и т. п.). Подробнее — в разделе [Assets](<02. Workspaces.md#ресурсы>).                                                          |
| `--type-check`          | Включить проверку типов (когда используется SWC).                                                                                                                                          |
| `--all`                 | Собрать все проекты в монорепозитории.                                                                                                                                                     |
| `--preserveWatchOutput` | Сохранять устаревший вывод в консоли в режиме наблюдения вместо очистки экрана. (только для режима наблюдения `tsc`)                                                                       |

#### nest start

Компилирует и запускает приложение (или проект по умолчанию в рабочем пространстве).

```bash
$ nest start <name> [options]
```

##### Аргументы

| Аргумент | Описание                        |
| -------- | ------------------------------- |
| `<name>` | Имя запускаемого проекта.       |

##### Опции

| Опция                   | Описание                                                                                                                           |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| `--path [path]`         | Путь к файлу `tsconfig`. <br/>Псевдоним `-p`                                                                                       |
| `--config [path]`       | Путь к файлу конфигурации `nest-cli`. <br/>Псевдоним `-c`                                                                          |
| `--watch`               | Запуск в режиме наблюдения (live-reload) <br/>Псевдоним `-w`                                                                       |
| `--builder [name]`      | Указать сборщик для компиляции (`tsc`, `swc` или `webpack`). <br/>Псевдоним `-b`                                                   |
| `--preserveWatchOutput` | Сохранять устаревший вывод в консоли в режиме наблюдения вместо очистки экрана. (только для режима наблюдения `tsc`)               |
| `--watchAssets`         | Запуск в режиме наблюдения (live-reload) со слежением за файлами, не относящимися к TS (ресурсами). Подробнее — в разделе [Assets](<02. Workspaces.md#ресурсы>). |
| `--debug [hostport]`    | Запуск в режиме отладки (с флагом --inspect) <br/>Псевдоним `-d`                                                                   |
| `--webpack`             | Использовать webpack для компиляции. (устарело: используйте `--builder webpack`)                                                   |
| `--webpackPath`         | Путь к конфигурации webpack.                                                                                                       |
| `--tsc`                 | Принудительно использовать `tsc` для компиляции.                                                                                   |
| `--exec [binary]`       | Запускаемый исполняемый файл (по умолчанию `node`). <br/>Псевдоним `-e`                                                            |
| `--no-shell`            | Не порождать дочерние процессы внутри оболочки (см. документацию метода `child_process.spawn()` в Node).                           |
| `--env-file`            | Загружает переменные окружения из файла относительно текущего каталога, делая их доступными приложениям через `process.env`.       |
| `-- [key=value]`        | Аргументы командной строки, к которым можно обратиться через `process.argv`.                                                       |

#### nest add

Импортирует библиотеку, упакованную как **библиотека nest**, выполняя её установочный схематик.

```bash
$ nest add <name> [options]
```

##### Аргументы

| Аргумент | Описание                           |
| -------- | ---------------------------------- |
| `<name>` | Имя импортируемой библиотеки.      |

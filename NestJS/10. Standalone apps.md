### Автономные приложения

Приложение Nest можно запустить несколькими способами. Вы можете создать веб-приложение, микросервис или просто голое **автономное приложение** Nest (без каких-либо сетевых слушателей). Автономное приложение Nest — это обёртка над **IoC-контейнером** Nest, который хранит все созданные экземпляры классов. Через объект автономного приложения можно напрямую получить ссылку на любой существующий экземпляр из любого импортированного модуля. Таким образом, преимуществами фреймворка Nest можно пользоваться где угодно — например, в скриптовых задачах **CRON**. На его основе можно даже построить **CLI**.

#### Начало работы

Чтобы создать автономное приложение Nest, используйте следующую конструкцию:

```typescript
async function bootstrap() {
  const app = await NestFactory.createApplicationContext(AppModule);
  // здесь логика вашего приложения ...
}
bootstrap();
```

#### Получение провайдеров из статических модулей

Объект автономного приложения позволяет получить ссылку на любой экземпляр, зарегистрированный в приложении Nest. Представим, что у нас есть провайдер `TasksService` в модуле `TasksModule`, который импортирован нашим модулем `AppModule`. Этот класс предоставляет набор методов, которые мы хотим вызывать из CRON-задачи.

```typescript
const tasksService = app.get(TasksService);
```

Чтобы получить доступ к экземпляру `TasksService`, мы используем метод `get()`. Метод `get()` работает как **запрос**, который ищет экземпляр в каждом зарегистрированном модуле. Ему можно передать токен любого провайдера. Кроме того, для строгой проверки контекста можно передать объект настроек со свойством `strict: true`. При включённой опции придётся пройти через конкретные модули, чтобы получить нужный экземпляр из выбранного контекста.

```typescript
const tasksService = app.select(TasksModule).get(TasksService, { strict: true });
```

Ниже приведена сводка методов, доступных для получения ссылок на экземпляры через объект автономного приложения.

<table>
  <tr>
    <td>
      <code>get()</code>
    </td>
    <td>
      Возвращает экземпляр контроллера или провайдера (включая guard'ы, фильтры и т. д.), доступный в контексте приложения.
    </td>
  </tr>
  <tr>
    <td>
      <code>select()</code>
    </td>
    <td>
      Перемещается по графу модулей, чтобы извлечь конкретный экземпляр из выбранного модуля (используется вместе со строгим режимом, описанным выше).
    </td>
  </tr>
</table>

> **Подсказка** В нестрогом режиме по умолчанию выбирается корневой модуль. Чтобы выбрать любой другой модуль, нужно пройти по графу модулей вручную, шаг за шагом.

Учтите, что у автономного приложения нет сетевых слушателей, поэтому все возможности Nest, связанные с HTTP (например, middleware, interceptor'ы, pipe'ы, guard'ы и т. д.), в этом контексте недоступны.

Например, даже если вы зарегистрируете в приложении глобальный interceptor, а затем получите экземпляр контроллера методом `app.get()`, interceptor выполнен не будет.

#### Получение провайдеров из динамических модулей

При работе с [динамическими модулями](<03. Fundamentals/03. Dynamic modules.md>) в `app.select` нужно передавать тот же объект, который представляет зарегистрированный в приложении динамический модуль. Например:

```typescript
export const dynamicConfigModule = ConfigModule.register({ folder: './config' });

@Module({
  imports: [dynamicConfigModule],
})
export class AppModule {}
```

После этого вы сможете выбрать этот модуль:

```typescript
const configService = app.select(dynamicConfigModule).get(ConfigService, { strict: true });
```

#### Фаза завершения

Если вы хотите, чтобы Node-приложение завершалось после окончания работы скрипта (например, для скрипта, выполняющего CRON-задачи), в конце функции `bootstrap` нужно вызвать метод `app.close()`:

```typescript
async function bootstrap() {
  const app = await NestFactory.createApplicationContext(AppModule);
  // логика приложения...
  await app.close();
}
bootstrap();
```

И, как упомянуто в главе [События жизненного цикла](<03. Fundamentals/09. Lifecycle events.md>), это запустит хуки жизненного цикла.

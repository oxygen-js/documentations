### Первые шаги

В этой серии статей вы познакомитесь с **основами** Nest. Чтобы освоиться с ключевыми строительными блоками Nest-приложений, мы напишем простое CRUD-приложение, возможности которого покрывают довольно широкий круг тем на вводном уровне.

#### Язык

Мы влюблены в [TypeScript](https://www.typescriptlang.org/), но прежде всего — в [Node.js](https://nodejs.org/en/). Поэтому Nest совместим и с TypeScript, и с чистым JavaScript. Nest использует новейшие возможности языка, так что для работы с обычным JavaScript понадобится компилятор [Babel](https://babeljs.io/).

В примерах мы в основном используем TypeScript, но вы всегда можете **переключить фрагменты кода** на синтаксис обычного JavaScript (просто нажмите на переключатель языка в правом верхнем углу каждого фрагмента).

#### Предварительные требования

Убедитесь, что в вашей операционной системе установлен [Node.js](https://nodejs.org) (версии >= 20).

#### Настройка

Создать новый проект с помощью [Nest CLI](<../11. CLI/01. Overview.md>) совсем просто. Если у вас установлен [npm](https://www.npmjs.com/), выполните в терминале вашей ОС следующие команды:

```bash
$ npm i -g @nestjs/cli
$ nest new project-name
```

> **Подсказка** Чтобы создать проект с [более строгим](https://www.typescriptlang.org/tsconfig#strict) набором проверок TypeScript, передайте команде `nest new` флаг `--strict`.

Будет создан каталог `project-name`, установлены модули Node и несколько других файлов шаблонного кода, а также создан каталог `src/` с несколькими основными файлами.

```text
src
  app.controller.spec.ts
  app.controller.ts
  app.module.ts
  app.service.ts
  main.ts
```

Краткий обзор этих основных файлов:

|                          |                                                                                                                     |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------- |
| `app.controller.ts`      | Простой контроллер с одним маршрутом.                                                                               |
| `app.controller.spec.ts` | Модульные тесты для контроллера.                                                                                    |
| `app.module.ts`          | Корневой модуль приложения.                                                                                         |
| `app.service.ts`         | Простой сервис с одним методом.                                                                                     |
| `main.ts`                | Точка входа приложения: использует основную функцию `NestFactory` для создания экземпляра Nest-приложения. |

Файл `main.ts` содержит асинхронную функцию, которая выполняет **начальную загрузку** нашего приложения:

**`main.ts`**

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
```

Для создания экземпляра Nest-приложения используется базовый класс `NestFactory`. Он предоставляет несколько статических методов, позволяющих создать экземпляр приложения. Метод `create()` возвращает объект приложения, реализующий интерфейс `INestApplication`. Этот объект предоставляет набор методов, которые описаны в следующих главах. В приведённом выше примере `main.ts` мы просто запускаем HTTP-слушатель, благодаря чему приложение начинает ожидать входящие HTTP-запросы.

Обратите внимание: проект, сгенерированный с помощью Nest CLI, получает начальную структуру, которая побуждает разработчиков придерживаться соглашения — держать каждый модуль в отдельном каталоге.

> **Подсказка** По умолчанию, если при создании приложения произойдёт ошибка, приложение завершится с кодом `1`. Если вы хотите, чтобы вместо этого выбрасывалось исключение, отключите опцию `abortOnError` (например, `NestFactory.create(AppModule, { abortOnError: false })`).

#### Платформа

Nest стремится быть платформенно-независимым фреймворком. Независимость от платформы позволяет создавать переиспользуемые логические части, которые разработчики могут применять в разных типах приложений. Технически Nest способен работать с любым HTTP-фреймворком для Node, как только для него написан адаптер. «Из коробки» поддерживаются две HTTP-платформы: [express](https://expressjs.com/) и [fastify](https://www.fastify.io). Вы можете выбрать ту, которая лучше подходит для ваших задач.

|                    |                                                                                                                                                                                                                                                                                                                                    |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `platform-express` | [Express](https://expressjs.com/) — хорошо известный минималистичный веб-фреймворк для node. Это проверенная в боевых условиях, готовая к production библиотека с множеством ресурсов, созданных сообществом. Пакет `@nestjs/platform-express` используется по умолчанию. Многим пользователям Express полностью подходит, и никаких действий для его включения не требуется. |
| `platform-fastify` | [Fastify](https://www.fastify.io/) — высокопроизводительный фреймворк с низкими накладными расходами, ориентированный на максимальную эффективность и скорость. О том, как его использовать, читайте [здесь](<../04. Techniques/19. Performance (Fastify).md>).                                                                                                                                  |

Какая бы платформа ни использовалась, она предоставляет собственный интерфейс приложения. Это, соответственно, `NestExpressApplication` и `NestFastifyApplication`.

Когда вы передаёте тип в метод `NestFactory.create()`, как в примере ниже, объект `app` получает методы, доступные исключительно для этой платформы. Учтите, однако, что указывать тип **не требуется**, если только вы действительно не собираетесь обращаться к API нижележащей платформы.

```typescript
const app = await NestFactory.create<NestExpressApplication>(AppModule);
```

#### Запуск приложения

После завершения установки выполните в командной строке вашей ОС следующую команду, чтобы запустить приложение и начать приём входящих HTTP-запросов:

```bash
$ npm run start
```

> **Подсказка** Чтобы ускорить процесс разработки (сборка быстрее в 20 раз), вы можете использовать [сборщик SWC](<../13. Recipes/03. SWC (fast compiler).md>), передав скрипту `start` флаг `-b swc`, например: `npm run start -- -b swc`.

Эта команда запускает приложение с HTTP-сервером, слушающим порт, заданный в файле `src/main.ts`. После запуска приложения откройте браузер и перейдите по адресу `http://localhost:3000/`. Вы должны увидеть сообщение `Hello World!`.

Чтобы отслеживать изменения в файлах, запустите приложение следующей командой:

```bash
$ npm run start:dev
```

Эта команда следит за вашими файлами, автоматически перекомпилируя их и перезапуская сервер.

#### Линтинг и форматирование

[CLI](<../11. CLI/01. Overview.md>) делает всё возможное, чтобы с самого начала выстроить надёжный процесс разработки, масштабируемый на большие проекты. Поэтому в сгенерированном Nest-проекте уже установлены **линтер** и **форматтер** кода (соответственно [eslint](https://eslint.org/) и [prettier](https://prettier.io/)).

> **Подсказка** Не уверены, чем форматтеры отличаются от линтеров? О разнице читайте [здесь](https://prettier.io/docs/en/comparison.html).

Для максимальной стабильности и расширяемости мы используем базовые CLI-пакеты [`eslint`](https://www.npmjs.com/package/eslint) и [`prettier`](https://www.npmjs.com/package/prettier). Такая конфигурация по замыслу обеспечивает удобную интеграцию с IDE через официальные расширения.

Для окружений без графического интерфейса, где IDE неактуальна (непрерывная интеграция, git-хуки и т. п.), в Nest-проекте есть готовые `npm`-скрипты.

```bash
# Проверка и автоисправление с помощью eslint
$ npm run lint

# Форматирование с помощью prettier
$ npm run format
```

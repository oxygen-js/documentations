### Защита от CSRF

Межсайтовая подделка запроса (CSRF или XSRF) — это тип атаки, при которой в веб-приложение отправляются **неавторизованные** команды от имени доверенного пользователя. Чтобы защититься от неё, вы можете использовать пакет [csrf-csrf](https://github.com/Psifi-Solutions/csrf-csrf).

#### Использование с Express (по умолчанию)

Начните с установки необходимого пакета:

```bash
$ npm i csrf-csrf
```

> **Предупреждение** Как отмечено в [документации csrf-csrf](https://github.com/Psifi-Solutions/csrf-csrf?tab=readme-ov-file#getting-started), этому middleware требуется, чтобы session middleware или `cookie-parser` были инициализированы заранее. Подробности смотрите в документации.

После завершения установки зарегистрируйте middleware `csrf-csrf` как глобальный.

```typescript
import { doubleCsrf } from 'csrf-csrf';
// ...
// где-то в файле инициализации
const {
  invalidCsrfTokenError, // Предоставляется исключительно для удобства, если вы планируете создавать собственный middleware.
  generateToken, // Используйте это в маршрутах для генерации и выдачи CSRF-хэша вместе с cookie-токеном и самим токеном.
  validateRequest, // Также для удобства, если вы собираетесь писать свой middleware.
  doubleCsrfProtection, // Это middleware защиты от CSRF по умолчанию.
} = doubleCsrf(doubleCsrfOptions);
app.use(doubleCsrfProtection);
```

#### Использование с Fastify

Начните с установки необходимого пакета:

```bash
$ npm i --save @fastify/csrf-protection
```

После завершения установки зарегистрируйте плагин `@fastify/csrf-protection` следующим образом:

```typescript
import fastifyCsrf from '@fastify/csrf-protection';
// ...
// где-то в файле инициализации после регистрации плагина хранилища
await app.register(fastifyCsrf);
```

> **Предупреждение** Как поясняется в документации `@fastify/csrf-protection` [здесь](https://github.com/fastify/csrf-protection#usage), этому плагину требуется, чтобы сначала был инициализирован плагин хранилища. Дальнейшие указания смотрите в этой документации.

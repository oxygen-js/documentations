### CORS

Cross-origin resource sharing (CORS) — механизм, позволяющий запрашивать ресурсы с другого домена. «Под капотом» Nest использует пакет [cors](https://github.com/expressjs/cors) для Express или [@fastify/cors](https://github.com/fastify/fastify-cors) для Fastify — в зависимости от используемой платформы. Эти пакеты предоставляют различные параметры, которые вы можете настроить под свои требования.

#### Начало работы

Чтобы включить CORS, вызовите метод `enableCors()` у объекта приложения Nest.

```typescript
const app = await NestFactory.create(AppModule);
app.enableCors();
await app.listen(process.env.PORT ?? 3000);
```

Метод `enableCors()` принимает необязательный аргумент — объект конфигурации. Доступные свойства этого объекта описаны в официальной документации [CORS](https://github.com/expressjs/cors#configuration-options). Другой вариант — передать [функцию обратного вызова](https://github.com/expressjs/cors#configuring-cors-asynchronously), которая позволяет определять объект конфигурации асинхронно на основе запроса (на лету).

Кроме того, CORS можно включить через объект параметров метода `create()`. Задайте свойству `cors` значение `true`, чтобы включить CORS с настройками по умолчанию.
Либо передайте в свойство `cors` [объект конфигурации CORS](https://github.com/expressjs/cors#configuration-options) или [функцию обратного вызова](https://github.com/expressjs/cors#configuring-cors-asynchronously), чтобы настроить поведение.

```typescript
const app = await NestFactory.create(AppModule, { cors: true });
await app.listen(process.env.PORT ?? 3000);
```

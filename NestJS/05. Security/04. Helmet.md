### Helmet

[Helmet](https://github.com/helmetjs/helmet) помогает защитить приложение от ряда известных веб-уязвимостей за счёт корректной установки HTTP-заголовков. По сути Helmet — это просто набор небольших middleware-функций, устанавливающих связанные с безопасностью HTTP-заголовки (подробнее [здесь](https://github.com/helmetjs/helmet#how-it-works)).

> **Подсказка** Учтите, что глобальное применение или регистрация `helmet` должны идти раньше остальных вызовов `app.use()` и функций настройки, которые могут вызывать `app.use()`. Так устроена работа нижележащей платформы (Express или Fastify): порядок объявления middleware и маршрутов имеет значение. Если подключить middleware вроде `helmet` или `cors` после объявления маршрута, к этому маршруту оно применяться не будет — только к маршрутам, объявленным после middleware.

#### Использование с Express (по умолчанию)

Начните с установки нужного пакета.

```bash
$ npm i --save helmet
```

После установки примените его как глобальный middleware.

```typescript
import helmet from 'helmet';
// где-то в файле инициализации
app.use(helmet());
```

> **Предупреждение** При совместном использовании `helmet`, `@apollo/server` (4.x) и [Apollo Sandbox](<../06. GraphQL/01. Quick start.md#apollo-sandbox>) возможны проблемы с [CSP](https://developer.mozilla.org/en-US/docs/Web/HTTP/CSP) в Apollo Sandbox. Чтобы решить их, настройте CSP так:
>
> ```typescript
> app.use(helmet({
>   crossOriginEmbedderPolicy: false,
>   contentSecurityPolicy: {
>     directives: {
>       imgSrc: [`'self'`, 'data:', 'apollo-server-landing-page.cdn.apollographql.com'],
>       scriptSrc: [`'self'`, `https: 'unsafe-inline'`],
>       manifestSrc: [`'self'`, 'apollo-server-landing-page.cdn.apollographql.com'],
>       frameSrc: [`'self'`, 'sandbox.embed.apollographql.com'],
>     },
>   },
> }));
> ```

#### Использование с Fastify

Если вы используете `FastifyAdapter`, установите пакет [@fastify/helmet](https://github.com/fastify/fastify-helmet):

```bash
$ npm i --save @fastify/helmet
```

[fastify-helmet](https://github.com/fastify/fastify-helmet) следует подключать не как middleware, а как [плагин Fastify](https://www.fastify.io/docs/latest/Reference/Plugins/), то есть через `app.register()`:

```typescript
import helmet from '@fastify/helmet'
// где-то в файле инициализации
await app.register(helmet)
```

> **Предупреждение** При совместном использовании `apollo-server-fastify` и `@fastify/helmet` возможны проблемы с [CSP](https://developer.mozilla.org/en-US/docs/Web/HTTP/CSP) в GraphQL playground. Чтобы устранить этот конфликт, настройте CSP так:
>
> ```typescript
> await app.register(fastifyHelmet, {
>    contentSecurityPolicy: {
>      directives: {
>        defaultSrc: [`'self'`, 'unpkg.com'],
>        styleSrc: [
>          `'self'`,
>          `'unsafe-inline'`,
>          'cdn.jsdelivr.net',
>          'fonts.googleapis.com',
>          'unpkg.com',
>        ],
>        fontSrc: [`'self'`, 'fonts.gstatic.com', 'data:'],
>        imgSrc: [`'self'`, 'data:', 'cdn.jsdelivr.net'],
>        scriptSrc: [
>          `'self'`,
>          `https: 'unsafe-inline'`,
>          `cdn.jsdelivr.net`,
>          `'unsafe-eval'`,
>        ],
>      },
>    },
>  });
>
> // Если CSP вам не нужен вовсе, можно поступить так:
> await app.register(fastifyHelmet, {
>   contentSecurityPolicy: false,
> });
> ```

### HTTP-модуль

[Axios](https://github.com/axios/axios) — широко распространённый и богатый возможностями пакет HTTP-клиента. Nest оборачивает Axios и предоставляет его через встроенный `HttpModule`. `HttpModule` экспортирует класс `HttpService`, который даёт основанные на Axios методы для выполнения HTTP-запросов. Кроме того, библиотека преобразует полученные HTTP-ответы в `Observables`.

> **Подсказка** Вы также можете напрямую использовать любую универсальную библиотеку HTTP-клиента для Node.js, например [got](https://github.com/sindresorhus/got) или [undici](https://github.com/nodejs/undici).

#### Установка

Чтобы начать работу, сначала установим необходимые зависимости.

```bash
$ npm i --save @nestjs/axios axios
```

#### Начало работы

После завершения установки для использования `HttpService` сначала импортируйте `HttpModule`.

```typescript
@Module({
  imports: [HttpModule],
  providers: [CatsService],
})
export class CatsModule {}
```

Затем внедрите `HttpService` обычным способом — через конструктор.

> **Подсказка** `HttpModule` и `HttpService` импортируются из пакета `@nestjs/axios`.

```typescript
@Injectable()
export class CatsService {
  constructor(private readonly httpService: HttpService) {}

  findAll(): Observable<AxiosResponse<Cat[]>> {
    return this.httpService.get('http://localhost:3000/cats');
  }
}
```

> **Подсказка** `AxiosResponse` — это интерфейс, экспортируемый пакетом `axios` (`$ npm i axios`).

Все методы `HttpService` возвращают `AxiosResponse`, обёрнутый в объект `Observable`.

#### Конфигурация

Поведение `HttpService` можно настраивать с помощью разнообразных опций [Axios](https://github.com/axios/axios). Подробнее о них читайте [здесь](https://github.com/axios/axios#request-config). Чтобы сконфигурировать нижележащий экземпляр Axios, передайте необязательный объект опций в метод `register()` модуля `HttpModule` при его импорте. Этот объект будет передан напрямую в конструктор Axios.

```typescript
@Module({
  imports: [
    HttpModule.register({
      timeout: 5000,
      maxRedirects: 5,
    }),
  ],
  providers: [CatsService],
})
export class CatsModule {}
```

#### Асинхронная конфигурация

Когда опции модуля нужно передавать асинхронно, а не статически, используйте метод `registerAsync()`. Как и для большинства динамических модулей, Nest предлагает несколько способов работы с асинхронной конфигурацией.

Первый способ — фабричная функция:

```typescript
HttpModule.registerAsync({
  useFactory: () => ({
    timeout: 5000,
    maxRedirects: 5,
  }),
});
```

Как и в других фабричных провайдерах, наша фабричная функция может быть [асинхронной](<../03. Fundamentals/01. Custom providers.md#провайдеры-фабрики-usefactory>) и получать зависимости через `inject`.

```typescript
HttpModule.registerAsync({
  imports: [ConfigModule],
  useFactory: async (configService: ConfigService) => ({
    timeout: configService.get('HTTP_TIMEOUT'),
    maxRedirects: configService.get('HTTP_MAX_REDIRECTS'),
  }),
  inject: [ConfigService],
});
```

Кроме того, `HttpModule` можно настроить с помощью класса вместо фабрики, как показано ниже.

```typescript
HttpModule.registerAsync({
  useClass: HttpConfigService,
});
```

Приведённая конструкция создаёт экземпляр `HttpConfigService` внутри `HttpModule` и использует его для формирования объекта опций. Обратите внимание: в этом примере `HttpConfigService` должен реализовывать интерфейс `HttpModuleOptionsFactory`, как показано ниже. `HttpModule` вызовет метод `createHttpOptions()` у созданного экземпляра переданного класса.

```typescript
@Injectable()
class HttpConfigService implements HttpModuleOptionsFactory {
  createHttpOptions(): HttpModuleOptions {
    return {
      timeout: 5000,
      maxRedirects: 5,
    };
  }
}
```

Если вы хотите переиспользовать существующий провайдер опций вместо создания приватной копии внутри `HttpModule`, воспользуйтесь синтаксисом `useExisting`.

```typescript
HttpModule.registerAsync({
  imports: [ConfigModule],
  useExisting: HttpConfigService,
});
```

В метод `registerAsync()` можно также передать так называемые `extraProviders`. Эти провайдеры будут объединены с провайдерами модуля.

```typescript
HttpModule.registerAsync({
  imports: [ConfigModule],
  useClass: HttpConfigService,
  extraProviders: [MyAdditionalProvider],
});
```

Это удобно, когда нужно предоставить дополнительные зависимости фабричной функции или конструктору класса.

#### Прямое использование Axios

Если вам кажется, что опций `HttpModule.register` недостаточно, или вы просто хотите получить доступ к нижележащему экземпляру Axios, созданному пакетом `@nestjs/axios`, обратиться к нему можно через `HttpService#axiosRef`:

```typescript
@Injectable()
export class CatsService {
  constructor(private readonly httpService: HttpService) {}

  findAll(): Promise<AxiosResponse<Cat[]>> {
    return this.httpService.axiosRef.get('http://localhost:3000/cats');
    //                      ^ интерфейс AxiosInstance
  }
}
```

#### Полный пример

Поскольку методы `HttpService` возвращают Observable, для получения данных запроса в виде промиса можно воспользоваться функциями `rxjs` — `firstValueFrom` или `lastValueFrom`.

```typescript
import { catchError, firstValueFrom } from 'rxjs';

@Injectable()
export class CatsService {
  private readonly logger = new Logger(CatsService.name);
  constructor(private readonly httpService: HttpService) {}

  async findAll(): Promise<Cat[]> {
    const { data } = await firstValueFrom(
      this.httpService.get<Cat[]>('http://localhost:3000/cats').pipe(
        catchError((error: AxiosError) => {
          this.logger.error(error.response.data);
          throw 'An error happened!';
        }),
      ),
    );
    return data;
  }
}
```

> **Подсказка** О различиях между [`firstValueFrom`](https://rxjs.dev/api/index/function/firstValueFrom) и [`lastValueFrom`](https://rxjs.dev/api/index/function/lastValueFrom) читайте в документации RxJS.

### Сессии

**HTTP-сессии** позволяют хранить информацию о пользователе между несколькими запросами, что особенно полезно для [MVC](<18. Model-View-Controller.md>)-приложений.

#### Использование с Express (по умолчанию)

Сначала установите [необходимый пакет](https://github.com/expressjs/session) (и его типы, если вы пишете на TypeScript):

```shell
$ npm i express-session
$ npm i -D @types/express-session
```

После установки подключите middleware `express-session` как глобальный middleware (например, в файле `main.ts`).

```typescript
import * as session from 'express-session';
// где-нибудь в файле инициализации
app.use(
  session({
    secret: 'my-secret',
    resave: false,
    saveUninitialized: false,
  }),
);
```

> **Важно** Серверное хранилище сессий по умолчанию намеренно не рассчитано на production-окружение. В большинстве случаев оно приводит к утечкам памяти, не масштабируется за пределы одного процесса и предназначено для отладки и разработки. Подробнее — в [официальном репозитории](https://github.com/expressjs/session).

Значение `secret` используется для подписи cookie с идентификатором сессии. Это может быть строка (один секрет) либо массив из нескольких секретов. Если передан массив, для подписи cookie используется только первый элемент, но при проверке подписи в запросах учитываются все элементы. Сам секрет не должен легко угадываться человеком — лучше всего использовать случайный набор символов.

Включение опции `resave` заставляет сохранять сессию обратно в хранилище, даже если она ни разу не изменялась в ходе запроса. Значение по умолчанию — `true`, однако полагаться на него не рекомендуется: в будущем значение по умолчанию изменится.

Аналогично, включение опции `saveUninitialized` заставляет сохранять в хранилище «неинициализированную» сессию. Сессия считается неинициализированной, если она новая, но не изменялась. Значение `false` полезно при реализации сессий входа, для сокращения расхода хранилища на сервере и для соблюдения законов, требующих разрешения пользователя перед установкой cookie. Кроме того, `false` помогает избежать состояний гонки, когда клиент без сессии отправляет несколько параллельных запросов ([источник](https://github.com/expressjs/session#saveuninitialized)).

Middleware `session` принимает и ряд других опций — подробнее о них в [документации по API](https://github.com/expressjs/session#options).

> **Подсказка** Обратите внимание, что опцию `secure: true` рекомендуется включать. Однако она требует сайта с поддержкой https, то есть для защищённых cookie необходим HTTPS. Если `secure` включена, а вы обращаетесь к сайту по HTTP, cookie установлена не будет. Если ваше node.js-приложение находится за прокси и вы используете `secure: true`, необходимо задать `"trust proxy"` в express.

После этого вы можете записывать и читать значения сессии внутри обработчиков маршрутов:

```typescript
@Get()
findAll(@Req() request: Request) {
  request.session.visits = request.session.visits ? request.session.visits + 1 : 1;
}
```

> **Подсказка** Декоратор `@Req()` импортируется из `@nestjs/common`, а `Request` — из пакета `express`.

Кроме того, для извлечения объекта сессии из запроса можно использовать декоратор `@Session()`:

```typescript
@Get()
findAll(@Session() session: Record<string, any>) {
  session.visits = session.visits ? session.visits + 1 : 1;
}
```

> **Подсказка** Декоратор `@Session()` импортируется из пакета `@nestjs/common`.

#### Использование с Fastify

Сначала установите необходимый пакет:

```shell
$ npm i @fastify/secure-session
```

После установки зарегистрируйте плагин `fastify-secure-session`:

```typescript
import secureSession from '@fastify/secure-session';

// где-нибудь в файле инициализации
const app = await NestFactory.create<NestFastifyApplication>(
  AppModule,
  new FastifyAdapter(),
);
await app.register(secureSession, {
  secret: 'averylogphrasebiggerthanthirtytwochars',
  salt: 'mq9hDxBVDbspDR6n',
});
```

> **Подсказка** Вы также можете сгенерировать ключ заранее ([см. инструкции](https://github.com/fastify/fastify-secure-session)) или использовать [ротацию ключей](https://github.com/fastify/fastify-secure-session#using-keys-with-key-rotation).

Подробнее о доступных опциях — в [официальном репозитории](https://github.com/fastify/fastify-secure-session).

После этого вы можете записывать и читать значения сессии внутри обработчиков маршрутов:

```typescript
@Get()
findAll(@Req() request: FastifyRequest) {
  const visits = request.session.get('visits');
  request.session.set('visits', visits ? visits + 1 : 1);
}
```

Кроме того, для извлечения объекта сессии из запроса можно использовать декоратор `@Session()`:

```typescript
@Get()
findAll(@Session() session: secureSession.Session) {
  const visits = session.get('visits');
  session.set('visits', visits ? visits + 1 : 1);
}
```

> **Подсказка** Декоратор `@Session()` импортируется из `@nestjs/common`, а `secureSession.Session` — из пакета `@fastify/secure-session` (инструкция импорта: `import * as secureSession from '@fastify/secure-session'`).

### Cookies

**HTTP-cookie** — небольшой фрагмент данных, который хранится в браузере пользователя. Cookies были задуманы как надёжный механизм, позволяющий сайтам запоминать информацию о состоянии. Когда пользователь снова заходит на сайт, cookie автоматически отправляется вместе с запросом.

#### Использование с Express (по умолчанию)

Сначала установите [нужный пакет](https://github.com/expressjs/cookie-parser) (и типы к нему, если вы пишете на TypeScript):

```shell
$ npm i cookie-parser
$ npm i -D @types/cookie-parser
```

После установки подключите middleware `cookie-parser` как глобальный middleware (например, в файле `main.ts`).

```typescript
import * as cookieParser from 'cookie-parser';
// где-нибудь в файле инициализации
app.use(cookieParser());
```

В middleware `cookieParser` можно передать несколько параметров:

- `secret` — строка или массив, используемые для подписи cookies. Параметр необязательный; если он не указан, подписанные cookies разбираться не будут. Если передана строка, она используется как секрет. Если передан массив, попытка снять подпись с cookie будет выполнена каждым секретом по порядку.
- `options` — объект, который передаётся в `cookie.parse` вторым аргументом. Подробности смотрите в [cookie](https://www.npmjs.org/package/cookie).

Middleware разберёт заголовок `Cookie` в запросе и предоставит данные cookies в свойстве `req.cookies`, а если был задан секрет — то и в свойстве `req.signedCookies`. Эти свойства представляют собой пары «имя cookie — значение cookie».

Если секрет задан, модуль снимет подпись и проверит все подписанные значения cookies и перенесёт соответствующие пары «имя — значение» из `req.cookies` в `req.signedCookies`. Подписанная cookie — это cookie, значение которой начинается с префикса `s:`. Подписанные cookies, не прошедшие проверку подписи, получат значение `false` вместо изменённого значения.

Теперь вы можете читать cookies внутри обработчиков маршрутов:

```typescript
@Get()
findAll(@Req() request: Request) {
  console.log(request.cookies); // или "request.cookies['cookieKey']"
  // или console.log(request.signedCookies);
}
```

> **Подсказка** Декоратор `@Req()` импортируется из `@nestjs/common`, а `Request` — из пакета `express`.

Чтобы прикрепить cookie к исходящему ответу, используйте метод `Response#cookie()`:

```typescript
@Get()
findAll(@Res({ passthrough: true }) response: Response) {
  response.cookie('key', 'value')
}
```

> **Предупреждение** Если вы хотите оставить логику обработки ответа фреймворку, не забудьте установить параметр `passthrough` в `true`, как показано выше. Подробнее читайте [здесь](<../02. Overview/02. Controllers.md#подход-специфичный-для-библиотеки>).

> **Подсказка** Декоратор `@Res()` импортируется из `@nestjs/common`, а `Response` — из пакета `express`.

#### Использование с Fastify

Сначала установите нужный пакет:

```shell
$ npm i @fastify/cookie
```

После установки зарегистрируйте плагин `@fastify/cookie`:

```typescript
import fastifyCookie from '@fastify/cookie';

// где-нибудь в файле инициализации
const app = await NestFactory.create<NestFastifyApplication>(AppModule, new FastifyAdapter());
await app.register(fastifyCookie, {
  secret: 'my-secret', // для подписи cookies
});
```

Теперь вы можете читать cookies внутри обработчиков маршрутов:

```typescript
@Get()
findAll(@Req() request: FastifyRequest) {
  console.log(request.cookies); // или "request.cookies['cookieKey']"
}
```

> **Подсказка** Декоратор `@Req()` импортируется из `@nestjs/common`, а `FastifyRequest` — из пакета `fastify`.

Чтобы прикрепить cookie к исходящему ответу, используйте метод `FastifyReply#setCookie()`:

```typescript
@Get()
findAll(@Res({ passthrough: true }) response: FastifyReply) {
  response.setCookie('key', 'value')
}
```

Подробнее о методе `FastifyReply#setCookie()` читайте на этой [странице](https://github.com/fastify/fastify-cookie#sending).

> **Предупреждение** Если вы хотите оставить логику обработки ответа фреймворку, не забудьте установить параметр `passthrough` в `true`, как показано выше. Подробнее читайте [здесь](<../02. Overview/02. Controllers.md#подход-специфичный-для-библиотеки>).

> **Подсказка** Декоратор `@Res()` импортируется из `@nestjs/common`, а `FastifyReply` — из пакета `fastify`.

#### Создание пользовательского декоратора (кроссплатформенно)

Чтобы получить удобный декларативный способ доступа к входящим cookies, можно создать [пользовательский декоратор](<../02. Overview/10. Custom decorators.md>).

```typescript
import { createParamDecorator, ExecutionContext } from '@nestjs/common';

export const Cookies = createParamDecorator((data: string, ctx: ExecutionContext) => {
  const request = ctx.switchToHttp().getRequest();
  return data ? request.cookies?.[data] : request.cookies;
});
```

Декоратор `@Cookies()` извлечёт все cookies или указанную по имени cookie из объекта `req.cookies` и подставит это значение в декорируемый параметр.

Теперь мы можем использовать декоратор в сигнатуре обработчика маршрута:

```typescript
@Get()
findAll(@Cookies('name') name: string) {}
```

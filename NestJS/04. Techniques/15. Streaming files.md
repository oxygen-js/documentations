### Потоковая передача файлов

> **Примечание** В этой главе показано, как передавать файлы потоком из **HTTP-приложения**. Приведённые ниже примеры не применимы к GraphQL- и микросервисным приложениям.

Иногда требуется вернуть клиенту файл из REST API. В Nest обычно это делают так:

```ts
@Controller('file')
export class FileController {
  @Get()
  getFile(@Res() res: Response) {
    const file = createReadStream(join(process.cwd(), 'package.json'));
    file.pipe(res);
  }
}
```

Но при таком подходе вы теряете доступ к логике interceptor'ов, работающих после контроллера. Чтобы этого избежать, можно вернуть экземпляр `StreamableFile` — «под капотом» фреймворк сам позаботится о передаче потока в ответ.

#### Класс StreamableFile

`StreamableFile` — это класс, который хранит поток, подлежащий возврату. Чтобы создать новый `StreamableFile`, передайте в его конструктор либо `Buffer`, либо `Stream`.

> **Подсказка** Класс `StreamableFile` импортируется из `@nestjs/common`.

#### Кроссплатформенная поддержка

Fastify по умолчанию умеет отправлять файлы без вызова `stream.pipe(res)`, так что класс `StreamableFile` там вообще не нужен. Тем не менее Nest поддерживает `StreamableFile` на обеих платформах, поэтому при переходе с Express на Fastify (и обратно) о совместимости движков можно не беспокоиться.

#### Пример

Ниже приведён простой пример возврата `package.json` в виде файла, а не JSON, но эта идея естественным образом распространяется на изображения, документы и любые другие типы файлов.

```ts
import { Controller, Get, StreamableFile } from '@nestjs/common';
import { createReadStream } from 'node:fs';
import { join } from 'node:path';

@Controller('file')
export class FileController {
  @Get()
  getFile(): StreamableFile {
    const file = createReadStream(join(process.cwd(), 'package.json'));
    return new StreamableFile(file);
  }
}
```

Тип содержимого по умолчанию (значение HTTP-заголовка ответа `Content-Type`) — `application/octet-stream`. Если нужно задать другое значение, используйте опцию `type` класса `StreamableFile`, метод `res.set` или декоратор [`@Header()`](<../02. Overview/02. Controllers.md#заголовки-ответа>), например:

```ts
import { Controller, Get, StreamableFile, Res } from '@nestjs/common';
import { createReadStream } from 'node:fs';
import { join } from 'node:path';
import type { Response } from 'express'; // Предполагаем, что используется HTTP-адаптер ExpressJS

@Controller('file')
export class FileController {
  @Get()
  getFile(): StreamableFile {
    const file = createReadStream(join(process.cwd(), 'package.json'));
    return new StreamableFile(file, {
      type: 'application/json',
      disposition: 'attachment; filename="package.json"',
      // Если нужно задать для Content-Length другое значение вместо длины файла:
      // length: 123,
    });
  }

  // Или даже так:
  @Get()
  getFileChangingResponseObjDirectly(@Res({ passthrough: true }) res: Response): StreamableFile {
    const file = createReadStream(join(process.cwd(), 'package.json'));
    res.set({
      'Content-Type': 'application/json',
      'Content-Disposition': 'attachment; filename="package.json"',
    });
    return new StreamableFile(file);
  }

  // Или даже так:
  @Get()
  @Header('Content-Type', 'application/json')
  @Header('Content-Disposition', 'attachment; filename="package.json"')
  getFileUsingStaticValues(): StreamableFile {
    const file = createReadStream(join(process.cwd(), 'package.json'));
    return new StreamableFile(file);
  }  
}
```

### Планирование задач

Планирование задач позволяет запускать произвольный код (методы/функции) в заданную дату/время, через регулярные интервалы или один раз по истечении указанного интервала. В мире Linux это обычно решается на уровне ОС пакетами вроде [cron](https://en.wikipedia.org/wiki/Cron). Для приложений на Node.js есть несколько пакетов, эмулирующих функциональность cron. Nest предоставляет пакет `@nestjs/schedule`, который интегрируется с популярным пакетом Node.js [cron](https://github.com/kelektiv/node-cron). Этот пакет мы и рассмотрим в текущей главе.

#### Установка

Чтобы начать работу, сначала установим необходимые зависимости.

```bash
$ npm install --save @nestjs/schedule
```

Чтобы активировать планирование задач, импортируйте `ScheduleModule` в корневой модуль `AppModule` и вызовите статический метод `forRoot()`, как показано ниже:

**`app.module.ts`**

```typescript
import { Module } from '@nestjs/common';
import { ScheduleModule } from '@nestjs/schedule';

@Module({
  imports: [
    ScheduleModule.forRoot()
  ],
})
export class AppModule {}
```

Вызов `.forRoot()` инициализирует планировщик и регистрирует все декларативные <a href="08.%20Task%20scheduling.md#декларативные-задачи-cron">задачи cron</a>, <a href="08.%20Task%20scheduling.md#декларативные-таймауты">таймауты</a> и <a href="08.%20Task%20scheduling.md#декларативные-интервалы">интервалы</a>, объявленные в приложении. Регистрация происходит при срабатывании хука жизненного цикла `onApplicationBootstrap`, что гарантирует, что все модули загружены и объявили свои запланированные задачи.

#### Декларативные задачи cron

Задача cron планирует автоматический запуск произвольной функции (вызова метода). Задачи cron могут выполняться:

- Один раз в заданную дату/время.
- Регулярно; повторяющиеся задачи могут выполняться в заданный момент внутри заданного интервала (например, раз в час, раз в неделю, раз в 5 минут).

Объявите задачу cron с помощью декоратора `@Cron()` перед определением метода, содержащего исполняемый код:

```typescript
import { Injectable, Logger } from '@nestjs/common';
import { Cron } from '@nestjs/schedule';

@Injectable()
export class TasksService {
  private readonly logger = new Logger(TasksService.name);

  @Cron('45 * * * * *')
  handleCron() {
    this.logger.debug('Called when the current second is 45');
  }
}
```

В этом примере метод `handleCron()` будет вызываться каждый раз, когда текущая секунда равна `45`. Иначе говоря, метод будет выполняться раз в минуту, на отметке 45 секунд.

Декоратор `@Cron()` поддерживает следующие стандартные [шаблоны cron](http://crontab.org/):

- Звёздочка (например, `*`)
- Диапазоны (например, `1-3,5`)
- Шаги (например, `*/2`)

В примере выше мы передали декоратору `45 * * * * *`. Следующая схема показывает, как интерпретируется каждая позиция в строке шаблона cron:

<pre class="language-javascript"><code class="language-javascript">

| | | | | |
| | | | | день недели
| | | | месяцы
| | | день месяца
| | часы
| минуты
секунды (необязательно)
</code></pre>

Пакет `@nestjs/schedule` предоставляет удобное перечисление с часто используемыми шаблонами cron. Использовать его можно так:

```typescript
import { Injectable, Logger } from '@nestjs/common';
import { Cron, CronExpression } from '@nestjs/schedule';

@Injectable()
export class TasksService {
  private readonly logger = new Logger(TasksService.name);

  @Cron(CronExpression.EVERY_30_SECONDS)
  handleCron() {
    this.logger.debug('Called every 30 seconds');
  }
}
```

В этом примере метод `handleCron()` будет вызываться каждые `30` секунд. Если возникнет исключение, оно будет записано в консоль, поскольку каждый метод, помеченный `@Cron()`, автоматически оборачивается в блок `try-catch`.

Кроме того, декоратору `@Cron()` можно передать объект JavaScript `Date`. В этом случае задача выполнится ровно один раз в указанную дату.

> **Подсказка** Используйте арифметику дат JavaScript, чтобы планировать задачи относительно текущей даты. Например, `@Cron(new Date(Date.now() + 10 * 1000))` запланирует задачу на 10 секунд после запуска приложения.

Также вторым параметром декоратора `@Cron()` можно передать дополнительные опции.

```typescript
import { Injectable } from '@nestjs/common';
import { Cron, CronExpression } from '@nestjs/schedule';

@Injectable()
export class NotificationService {
  @Cron('* * 0 * * *', {
    name: 'notifications',
    timeZone: 'Europe/Paris',
  })
  triggerNotifications() {}
}
```

Обращаться к задаче cron и управлять ею после объявления, а также динамически создавать задачу cron (когда её шаблон определяется во время выполнения) можно с помощью <a href="08.%20Task%20scheduling.md#динамический-api-модуля-планирования">динамического API</a>. Чтобы обратиться к декларативной задаче cron через API, задаче нужно назначить имя, передав свойство `name` в необязательном объекте опций вторым аргументом декоратора.

#### Декларативные интервалы

Чтобы объявить, что метод должен выполняться через заданный (повторяющийся) интервал, поставьте перед определением метода декоратор `@Interval()`. Передайте декоратору значение интервала в миллисекундах, как показано ниже:

```typescript
@Interval(10000)
handleInterval() {
  this.logger.debug('Called every 10 seconds');
}
```

> **Подсказка** Этот механизм «под капотом» использует функцию JavaScript `setInterval()`. Для планирования повторяющихся задач можно также использовать задачу cron.

Если вы хотите управлять декларативным интервалом извне объявляющего класса через <a href="08.%20Task%20scheduling.md#динамический-api-модуля-планирования">динамический API</a>, свяжите интервал с именем следующим образом:

```typescript
@Interval('notifications', 2500)
handleInterval() {}
```

Если возникнет исключение, оно будет записано в консоль, поскольку каждый метод, помеченный `@Interval()`, автоматически оборачивается в блок `try-catch`.

<a href="08.%20Task%20scheduling.md#динамические-интервалы">Динамический API</a> также позволяет **создавать** динамические интервалы, свойства которых определяются во время выполнения, а также **выводить их список и удалять**.

#### Декларативные таймауты

Чтобы объявить, что метод должен выполниться (один раз) по истечении заданного таймаута, поставьте перед определением метода декоратор `@Timeout()`. Передайте декоратору относительное смещение времени (в миллисекундах) от момента запуска приложения, как показано ниже:

```typescript
@Timeout(5000)
handleTimeout() {
  this.logger.debug('Called once after 5 seconds');
}
```

> **Подсказка** Этот механизм «под капотом» использует функцию JavaScript `setTimeout()`.

Если возникнет исключение, оно будет записано в консоль, поскольку каждый метод, помеченный `@Timeout()`, автоматически оборачивается в блок `try-catch`.

Если вы хотите управлять декларативным таймаутом извне объявляющего класса через <a href="08.%20Task%20scheduling.md#динамический-api-модуля-планирования">динамический API</a>, свяжите таймаут с именем следующим образом:

```typescript
@Timeout('notifications', 2500)
handleTimeout() {}
```

<a href="08.%20Task%20scheduling.md#динамические-таймауты">Динамический API</a> также позволяет **создавать** динамические таймауты, свойства которых определяются во время выполнения, а также **выводить их список и удалять**.

#### Динамический API модуля планирования

Модуль `@nestjs/schedule` предоставляет динамический API, позволяющий управлять декларативными <a href="08.%20Task%20scheduling.md#декларативные-задачи-cron">задачами cron</a>, <a href="08.%20Task%20scheduling.md#декларативные-таймауты">таймаутами</a> и <a href="08.%20Task%20scheduling.md#декларативные-интервалы">интервалами</a>. Этот API также позволяет создавать **динамические** задачи cron, таймауты и интервалы, свойства которых определяются во время выполнения, и управлять ими.

#### Динамические задачи cron

Получить ссылку на экземпляр `CronJob` по имени из любого места в коде можно с помощью API `SchedulerRegistry`. Сначала внедрите `SchedulerRegistry` стандартным способом через конструктор:

```typescript
constructor(private schedulerRegistry: SchedulerRegistry) {}
```

> **Подсказка** Импортируйте `SchedulerRegistry` из пакета `@nestjs/schedule`.

Затем используйте его в классе следующим образом. Допустим, задача cron была создана таким объявлением:

```typescript
@Cron('* * 8 * * *', {
  name: 'notifications',
})
triggerNotifications() {}
```

Обратиться к этой задаче можно так:

```typescript
const job = this.schedulerRegistry.getCronJob('notifications');

job.stop();
console.log(job.lastDate());
```

Метод `getCronJob()` возвращает именованную задачу cron. У возвращённого объекта `CronJob` есть следующие методы:

- `stop()` — останавливает запланированную задачу.
- `start()` — перезапускает остановленную задачу.
- `setTime(time: CronTime)` — останавливает задачу, задаёт ей новое время и снова запускает её.
- `lastDate()` — возвращает представление `DateTime` даты последнего выполнения задачи.
- `nextDate()` — возвращает представление `DateTime` даты следующего запланированного выполнения задачи.
- `nextDates(count: number)` — возвращает массив (размером `count`) представлений `DateTime` для следующих дат, когда задача будет запущена. По умолчанию `count` равен 0, и возвращается пустой массив.

> **Подсказка** Используйте `toJSDate()` на объектах `DateTime`, чтобы получить эквивалентный объект JavaScript Date.

**Создать** новую задачу cron динамически можно методом `SchedulerRegistry#addCronJob`:

```typescript
addCronJob(name: string, seconds: string) {
  const job = new CronJob(`${seconds} * * * * *`, () => {
    this.logger.warn(`time (${seconds}) for job ${name} to run!`);
  });

  this.schedulerRegistry.addCronJob(name, job);
  job.start();

  this.logger.warn(
    `job ${name} added for each minute at ${seconds} seconds!`,
  );
}
```

В этом коде мы используем объект `CronJob` из пакета `cron` для создания задачи cron. Конструктор `CronJob` принимает первым аргументом шаблон cron (так же, как <a href="08.%20Task%20scheduling.md#декларативные-задачи-cron">декоратор</a> `@Cron()`), а вторым — обратный вызов, выполняемый при срабатывании таймера cron. Метод `SchedulerRegistry#addCronJob` принимает два аргумента: имя для `CronJob` и сам объект `CronJob`.

> **Предупреждение** Не забудьте внедрить `SchedulerRegistry` перед обращением к нему. Импортируйте `CronJob` из пакета `cron`.

**Удалить** именованную задачу cron можно методом `SchedulerRegistry#deleteCronJob`:

```typescript
deleteCron(name: string) {
  this.schedulerRegistry.deleteCronJob(name);
  this.logger.warn(`job ${name} deleted!`);
}
```

**Вывести список** всех задач cron можно методом `SchedulerRegistry#getCronJobs`:

```typescript
getCrons() {
  const jobs = this.schedulerRegistry.getCronJobs();
  jobs.forEach((value, key, map) => {
    let next;
    try {
      next = value.nextDate().toJSDate();
    } catch (e) {
      next = 'error: next fire date is in the past!';
    }
    this.logger.log(`job: ${key} -> next: ${next}`);
  });
}
```

Метод `getCronJobs()` возвращает `map`. В этом коде мы перебираем map и пытаемся вызвать метод `nextDate()` у каждой `CronJob`. В API `CronJob`, если задача уже сработала и у неё нет будущих дат запуска, выбрасывается исключение.

#### Динамические интервалы

Получить ссылку на интервал можно методом `SchedulerRegistry#getInterval`. Как и выше, внедрите `SchedulerRegistry` стандартным способом через конструктор:

```typescript
constructor(private schedulerRegistry: SchedulerRegistry) {}
```

И используйте его так:

```typescript
const interval = this.schedulerRegistry.getInterval('notifications');
clearInterval(interval);
```

**Создать** новый интервал динамически можно методом `SchedulerRegistry#addInterval`:

```typescript
addInterval(name: string, milliseconds: number) {
  const callback = () => {
    this.logger.warn(`Interval ${name} executing at time (${milliseconds})!`);
  };

  const interval = setInterval(callback, milliseconds);
  this.schedulerRegistry.addInterval(name, interval);
}
```

В этом коде мы создаём обычный интервал JavaScript, а затем передаём его методу `SchedulerRegistry#addInterval`.
Этот метод принимает два аргумента: имя интервала и сам интервал.

**Удалить** именованный интервал можно методом `SchedulerRegistry#deleteInterval`:

```typescript
deleteInterval(name: string) {
  this.schedulerRegistry.deleteInterval(name);
  this.logger.warn(`Interval ${name} deleted!`);
}
```

**Вывести список** всех интервалов можно методом `SchedulerRegistry#getIntervals`:

```typescript
getIntervals() {
  const intervals = this.schedulerRegistry.getIntervals();
  intervals.forEach(key => this.logger.log(`Interval: ${key}`));
}
```

#### Динамические таймауты

Получить ссылку на таймаут можно методом `SchedulerRegistry#getTimeout`. Как и выше, внедрите `SchedulerRegistry` стандартным способом через конструктор:

```typescript
constructor(private readonly schedulerRegistry: SchedulerRegistry) {}
```

И используйте его так:

```typescript
const timeout = this.schedulerRegistry.getTimeout('notifications');
clearTimeout(timeout);
```

**Создать** новый таймаут динамически можно методом `SchedulerRegistry#addTimeout`:

```typescript
addTimeout(name: string, milliseconds: number) {
  const callback = () => {
    this.logger.warn(`Timeout ${name} executing after (${milliseconds})!`);
  };

  const timeout = setTimeout(callback, milliseconds);
  this.schedulerRegistry.addTimeout(name, timeout);
}
```

В этом коде мы создаём обычный таймаут JavaScript, а затем передаём его методу `SchedulerRegistry#addTimeout`.
Этот метод принимает два аргумента: имя таймаута и сам таймаут.

**Удалить** именованный таймаут можно методом `SchedulerRegistry#deleteTimeout`:

```typescript
deleteTimeout(name: string) {
  this.schedulerRegistry.deleteTimeout(name);
  this.logger.warn(`Timeout ${name} deleted!`);
}
```

**Вывести список** всех таймаутов можно методом `SchedulerRegistry#getTimeouts`:

```typescript
getTimeouts() {
  const timeouts = this.schedulerRegistry.getTimeouts();
  timeouts.forEach(key => this.logger.log(`Timeout: ${key}`));
}
```

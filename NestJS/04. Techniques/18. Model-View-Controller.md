### Model-View-Controller

По умолчанию Nest использует «под капотом» библиотеку [Express](https://github.com/expressjs/express). Поэтому любой приём применения паттерна MVC (Model-View-Controller) в Express подходит и для Nest.

Сначала сгенерируем каркас простого приложения Nest с помощью [CLI](https://github.com/nestjs/nest-cli):

```bash
$ npm i -g @nestjs/cli
$ nest new project
```

Чтобы создать MVC-приложение, нам также понадобится [шаблонизатор](https://expressjs.com/en/guide/using-template-engines.html) для рендеринга HTML-представлений:

```bash
$ npm install --save hbs
```

Мы взяли движок `hbs` ([Handlebars](https://github.com/pillarjs/hbs#readme)), но вы можете использовать любой подходящий под ваши требования. После установки нужно настроить экземпляр express следующим кодом:

**`main.ts`**

```typescript
import { NestFactory } from '@nestjs/core';
import { NestExpressApplication } from '@nestjs/platform-express';
import { join } from 'node:path';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create<NestExpressApplication>(
    AppModule,
  );

  app.useStaticAssets(join(__dirname, '..', 'public'));
  app.setBaseViewsDir(join(__dirname, '..', 'views'));
  app.setViewEngine('hbs');

  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
```

Мы сообщили [Express](https://github.com/expressjs/express), что каталог `public` будет использоваться для хранения статических ресурсов, `views` будет содержать шаблоны, а для формирования HTML-вывода следует использовать шаблонизатор `hbs`.

#### Рендеринг шаблона

Теперь создадим каталог `views`, а внутри него — шаблон `index.hbs`. В шаблоне выведем `message`, переданное из контроллера:

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>App</title>
  </head>
  <body>
    {{ message }\}
  </body>
</html>
```

Далее откройте файл `app.controller` и замените метод `root()` следующим кодом:

**`app.controller.ts`**

```typescript
import { Get, Controller, Render } from '@nestjs/common';

@Controller()
export class AppController {
  @Get()
  @Render('index')
  root() {
    return { message: 'Hello world!' };
  }
}
```

В этом коде мы указываем используемый шаблон в декораторе `@Render()`, а возвращаемое значение метода-обработчика маршрута передаётся в шаблон для рендеринга. Обратите внимание, что возвращается объект со свойством `message`, соответствующим плейсхолдеру `message`, который мы создали в шаблоне.

Пока приложение работает, откройте браузер и перейдите по адресу `http://localhost:3000`. Вы должны увидеть сообщение `Hello world!`.

#### Динамический рендеринг шаблона

Если логика приложения должна динамически решать, какой шаблон рендерить, следует использовать декоратор `@Res()` и указывать имя представления в обработчике маршрута, а не в декораторе `@Render()`:

> **Подсказка** Обнаружив декоратор `@Res()`, Nest внедряет специфичный для библиотеки объект `response`. С его помощью можно динамически рендерить шаблон. Подробнее об API объекта `response` — [здесь](https://expressjs.com/en/api.html).

**`app.controller.ts`**

```typescript
import { Get, Controller, Res, Render } from '@nestjs/common';
import { Response } from 'express';
import { AppService } from './app.service';

@Controller()
export class AppController {
  constructor(private appService: AppService) {}

  @Get()
  root(@Res() res: Response) {
    return res.render(
      this.appService.getViewName(),
      { message: 'Hello world!' },
    );
  }
}
```

#### Fastify

Как упоминалось в этой [главе](<19. Performance (Fastify).md>), вместе с Nest можно использовать любой совместимый HTTP-провайдер. Одна из таких библиотек — [Fastify](https://github.com/fastify/fastify). Чтобы создать MVC-приложение на Fastify, нужно установить следующие пакеты:

```bash
$ npm i --save @fastify/static @fastify/view handlebars
```

Дальнейшие шаги практически повторяют процесс для Express, с небольшими отличиями, специфичными для платформы. После установки откройте файл `main.ts` и обновите его содержимое:

**`main.ts`**

```typescript
import { NestFactory } from '@nestjs/core';
import { NestFastifyApplication, FastifyAdapter } from '@nestjs/platform-fastify';
import { AppModule } from './app.module';
import { join } from 'node:path';

async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter(),
  );
  app.useStaticAssets({
    root: join(__dirname, '..', 'public'),
    prefix: '/public/',
  });
  app.setViewEngine({
    engine: {
      handlebars: require('handlebars'),
    },
    templates: join(__dirname, '..', 'views'),
  });
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
```

В API Fastify есть несколько отличий, но итоговый результат этих вызовов тот же. Одно заметное отличие: при использовании Fastify имя шаблона, передаваемое в декоратор `@Render()`, должно включать расширение файла.

Вот как это настроить:

**`app.controller.ts`**

```typescript
import { Get, Controller, Render } from '@nestjs/common';

@Controller()
export class AppController {
  @Get()
  @Render('index.hbs')
  root() {
    return { message: 'Hello world!' };
  }
}
```

Как вариант, можно с помощью декоратора `@Res()` напрямую внедрить объект ответа и указать нужное представление, как показано ниже:

```typescript
import { Res } from '@nestjs/common';
import { FastifyReply } from 'fastify';

@Get()
root(@Res() res: FastifyReply) {
  return res.view('index.hbs', { title: 'Hello world!' });
}
```

Пока приложение работает, откройте браузер и перейдите по адресу `http://localhost:3000`. Вы должны увидеть сообщение `Hello world!`.

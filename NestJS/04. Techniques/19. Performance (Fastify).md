### Производительность (Fastify)

По умолчанию Nest использует фреймворк [Express](https://expressjs.com/). Как уже упоминалось, Nest также совместим с другими библиотеками — например, с [Fastify](https://github.com/fastify/fastify). Независимость от фреймворка достигается за счёт адаптера, основная задача которого — проксировать middleware и обработчики в соответствующие реализации конкретной библиотеки.

> **Подсказка** Обратите внимание: чтобы адаптер фреймворка можно было реализовать, целевая библиотека должна обрабатывать конвейер запроса/ответа схожим с Express образом.

[Fastify](https://github.com/fastify/fastify) — хорошая альтернатива для Nest, поскольку решает вопросы архитектуры похожим на Express способом. При этом fastify значительно **быстрее** Express: в бенчмарках он показывает почти вдвое лучшие результаты. Резонный вопрос: почему же Nest использует Express в качестве HTTP-провайдера по умолчанию? Причина в том, что Express широко распространён, хорошо известен и обладает огромным набором совместимого middleware, доступного пользователям Nest «из коробки».

Но поскольку Nest не привязан к конкретному фреймворку, вы легко можете переходить с одного на другой. Fastify может оказаться предпочтительнее, если для вас особенно важна высокая производительность. Чтобы задействовать Fastify, просто выберите встроенный `FastifyAdapter`, как показано в этой главе.

#### Установка

Сначала нужно установить необходимый пакет:

```bash
$ npm i --save @nestjs/platform-fastify
```

#### Адаптер

После установки платформы Fastify мы можем использовать `FastifyAdapter`.

**`main.ts`**

```typescript
import { NestFactory } from '@nestjs/core';
import {
  FastifyAdapter,
  NestFastifyApplication,
} from '@nestjs/platform-fastify';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter()
  );
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
```

По умолчанию Fastify слушает только интерфейс `localhost 127.0.0.1` ([подробнее](https://fastify.dev/docs/latest/Guides/Getting-Started/#your-first-server)). Если вы хотите принимать соединения с других хостов, укажите `'0.0.0.0'` в вызове `listen()`:

```typescript
async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter(),
  );
  await app.listen(3000, '0.0.0.0');
}
```

#### Пакеты, специфичные для платформы

Имейте в виду: при использовании `FastifyAdapter` Nest применяет Fastify в качестве **HTTP-провайдера**. Это означает, что рецепты, опирающиеся на Express, могут перестать работать. Вместо них следует использовать эквивалентные пакеты для Fastify.

#### Ответ с редиректом

Fastify обрабатывает ответы с перенаправлением немного иначе, чем Express. Чтобы выполнить корректный редирект в Fastify, верните и код состояния, и URL:

```typescript
@Get()
index(@Res() res) {
  res.status(302).redirect('/login');
}
```

#### Параметры Fastify

Вы можете передать параметры в конструктор Fastify через конструктор `FastifyAdapter`. Например:

```typescript
new FastifyAdapter({ logger: true });
```

#### Middleware

Функции middleware получают исходные объекты `req` и `res`, а не обёртки Fastify. Именно так работают пакет `middie` (используемый «под капотом») и `fastify` — подробнее см. на этой [странице](https://fastify.dev/docs/latest/Reference/Middleware/),

**`logger.middleware.ts`**

```typescript
import { Injectable, NestMiddleware } from '@nestjs/common';
import { FastifyRequest, FastifyReply } from 'fastify';

@Injectable()
export class LoggerMiddleware implements NestMiddleware {
  use(req: FastifyRequest['raw'], res: FastifyReply['raw'], next: () => void) {
    console.log('Request...');
    next();
  }
}
```

#### Конфигурация маршрута

Вы можете использовать возможность Fastify [route config](https://fastify.dev/docs/latest/Reference/Routes/#config) с помощью декоратора `@RouteConfig()`.

```typescript
@RouteConfig({ output: 'hello world' })
@Get()
index(@Req() req) {
  return req.routeConfig.output;
}
```

#### Ограничения маршрутов

Начиная с версии 10.3.0 `@nestjs/platform-fastify` поддерживает возможность Fastify [route constraints](https://fastify.dev/docs/latest/Reference/Routes/#constraints) через декоратор `@RouteConstraints`.

```typescript
@RouteConstraints({ version: '1.2.x' })
newFeature() {
  return 'This works only for version >= 1.2.x';
}
```

> **Подсказка** `@RouteConfig()` и `@RouteConstraints` импортируются из `@nestjs/platform-fastify`.

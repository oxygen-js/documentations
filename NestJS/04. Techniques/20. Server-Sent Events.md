### Server-Sent Events

Server-Sent Events (SSE) — технология server push, позволяющая клиенту получать автоматические обновления от сервера по HTTP-соединению. Каждое уведомление отправляется как блок текста, завершающийся парой переводов строки (подробнее [здесь](https://developer.mozilla.org/en-US/docs/Web/API/Server-sent_events)).

#### Использование

Чтобы включить Server-Sent Events для маршрута (маршрута, зарегистрированного в **классе контроллера**), пометьте метод-обработчик декоратором `@Sse()`.

```typescript
@Sse('sse')
sse(): Observable<MessageEvent> {
  return interval(1000).pipe(map((_) => ({ data: { hello: 'world' } })));
}
```

> **Подсказка** Декоратор `@Sse()` и интерфейс `MessageEvent` импортируются из `@nestjs/common`, а `Observable`, `interval` и `map` — из пакета `rxjs`.

> **Предупреждение** Маршруты Server-Sent Events должны возвращать поток `Observable`.

В примере выше мы определили маршрут с именем `sse`, который позволит транслировать обновления в реальном времени. Слушать эти события можно с помощью [EventSource API](https://developer.mozilla.org/en-US/docs/Web/API/EventSource).

Метод `sse` возвращает `Observable`, который выдаёт несколько `MessageEvent` (в этом примере — новый `MessageEvent` каждую секунду). Чтобы соответствовать спецификации, объект `MessageEvent` должен удовлетворять следующему интерфейсу:

```typescript
export interface MessageEvent {
  data: string | object;
  id?: string;
  type?: string;
  retry?: number;
}
```

Теперь мы можем создать в клиентском приложении экземпляр класса `EventSource`, передав аргументом конструктора маршрут `/sse` (он совпадает с эндпоинтом, который мы передали в декоратор `@Sse()` выше).

Экземпляр `EventSource` открывает постоянное соединение с HTTP-сервером, который отправляет события в формате `text/event-stream`. Соединение остаётся открытым, пока не будет закрыто вызовом `EventSource.close()`.

После открытия соединения входящие сообщения от сервера доставляются в ваш код в виде событий. Если во входящем сообщении есть поле event, то генерируется событие с именем из значения этого поля. Если поля event нет, генерируется обобщённое событие `message` ([источник](https://developer.mozilla.org/en-US/docs/Web/API/EventSource)).

```javascript
const eventSource = new EventSource('/sse');
eventSource.onmessage = ({ data }) => {
  console.log('New message', JSON.parse(data));
};
```

#### Отключение клиента

Когда клиент закрывает SSE-соединение (например, вызовом `eventSource.close()`), NestJS автоматически отписывается от возвращённого Observable, что останавливает поток событий и освобождает связанные ресурсы — включая таймер интервала из примера выше.

Чтобы выполнить собственную логику очистки при отключении клиента, используйте оператор `finalize`:

```typescript
@Sse('sse')
sse(): Observable<MessageEvent> {
  return interval(1000).pipe(
    map((_) => ({ data: { hello: 'world' } })),
    finalize(() => console.log('Client disconnected')),
  );
}
```

> **Подсказка** Оператор `finalize` (импортируется из `rxjs`) выполняет свой коллбэк при любом завершении Observable — по завершении, по ошибке или при отписке (в том числе при отключении клиента). Именно здесь уместно освобождать внешние ресурсы: курсоры БД, файловые дескрипторы и прочее, связанное с потоком.

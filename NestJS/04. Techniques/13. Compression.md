### Сжатие

Сжатие может значительно уменьшить размер тела ответа и тем самым повысить скорость веб-приложения.

Для сайтов с **высокой нагрузкой** в продакшене настоятельно рекомендуется вынести сжатие с сервера приложения — как правило, на обратный прокси (например, Nginx). В этом случае middleware для сжатия использовать не следует.

#### Использование с Express (по умолчанию)

Чтобы включить gzip-сжатие, используйте пакет middleware [compression](https://github.com/expressjs/compression).

Сначала установите нужный пакет:

```bash
$ npm i --save compression
$ npm i --save-dev @types/compression
```

После завершения установки примените middleware сжатия как глобальный middleware.

```typescript
import * as compression from 'compression';
// где-нибудь в файле инициализации
app.use(compression());
```

#### Использование с Fastify

Если вы используете `FastifyAdapter`, вам понадобится [fastify-compress](https://github.com/fastify/fastify-compress):

```bash
$ npm i --save @fastify/compress
```

После завершения установки примените middleware `@fastify/compress` как глобальный middleware.

> **Предупреждение** Убедитесь, что при создании приложения вы используете тип `NestFastifyApplication`. Иначе вы не сможете применить middleware сжатия через `register`.

```typescript
import { FastifyAdapter, NestFastifyApplication } from '@nestjs/platform-fastify';

import compression from '@fastify/compress';

// внутри bootstrap()
const app = await NestFactory.create<NestFastifyApplication>(AppModule, new FastifyAdapter());
await app.register(compression);
```

По умолчанию `@fastify/compress` использует сжатие Brotli (на Node >= 11.7.0), когда браузеры сообщают о поддержке этой кодировки. Brotli может быть весьма эффективен по степени сжатия, но при этом довольно медленным. По умолчанию Brotli использует максимальное качество сжатия 11, однако его можно уменьшить, пожертвовав качеством сжатия ради скорости, — параметр `BROTLI_PARAM_QUALITY` принимает значения от 0 (минимум) до 11 (максимум). Здесь потребуется тонкая настройка для оптимального баланса «место/время». Пример с качеством 4:

```typescript
import { constants } from 'node:zlib';
// где-нибудь в файле инициализации
await app.register(compression, { brotliOptions: { params: { [constants.BROTLI_PARAM_QUALITY]: 4 } } });
```

Для упрощения вы можете указать `fastify-compress` использовать для сжатия ответов только deflate и gzip; ответы будут потенциально крупнее, но доставляться значительно быстрее.

Чтобы задать кодировки, передайте второй аргумент в `app.register`:

```typescript
await app.register(compression, { encodings: ['gzip', 'deflate'] });
```

Приведённый код указывает `fastify-compress` использовать только кодировки gzip и deflate, отдавая предпочтение gzip, если клиент поддерживает обе.

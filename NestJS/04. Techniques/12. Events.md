### Events

Пакет [Event Emitter](https://www.npmjs.com/package/@nestjs/event-emitter) (`@nestjs/event-emitter`) предоставляет простую реализацию паттерна «наблюдатель», позволяя подписываться на различные события, происходящие в приложении, и слушать их. События — отличный способ развязать разные части приложения, поскольку у одного события может быть несколько независимых друг от друга слушателей.

Внутри `EventEmitterModule` использует пакет [eventemitter2](https://github.com/EventEmitter2/EventEmitter2).

#### Начало работы

Сначала установите нужный пакет:

```shell
$ npm i --save @nestjs/event-emitter
```

После установки импортируйте `EventEmitterModule` в корневой `AppModule` и вызовите статический метод `forRoot()`, как показано ниже:

**`app.module.ts`**

```typescript
import { Module } from '@nestjs/common';
import { EventEmitterModule } from '@nestjs/event-emitter';

@Module({
  imports: [
    EventEmitterModule.forRoot()
  ],
})
export class AppModule {}
```

Вызов `.forRoot()` инициализирует event emitter и регистрирует все декларативные слушатели событий, объявленные в приложении. Регистрация происходит при срабатывании хука жизненного цикла `onApplicationBootstrap`, что гарантирует, что все модули загружены и все запланированные задачи объявлены.

Чтобы настроить используемый экземпляр `EventEmitter`, передайте объект конфигурации в метод `.forRoot()`:

```typescript
EventEmitterModule.forRoot({
  // задайте `true`, чтобы использовать подстановочные знаки
  wildcard: false,
  // разделитель, используемый для сегментации пространств имён
  delimiter: '.',
  // задайте `true`, если хотите генерировать событие newListener
  newListener: false,
  // задайте `true`, если хотите генерировать событие removeListener
  removeListener: false,
  // максимальное число слушателей, которое можно назначить событию
  maxListeners: 10,
  // показывать имя события в сообщении об утечке памяти, когда назначено больше слушателей, чем максимум
  verboseMemoryLeak: false,
  // отключить выброс uncaughtException, если сгенерировано событие error и у него нет слушателей
  ignoreErrors: false,
});
```

#### Отправка событий

Чтобы отправить (то есть сгенерировать) событие, сначала внедрите `EventEmitter2` через обычное внедрение в конструктор:

```typescript
constructor(private eventEmitter: EventEmitter2) {}
```

> **Подсказка** Импортируйте `EventEmitter2` из пакета `@nestjs/event-emitter`.

Затем используйте его в классе так:

```typescript
this.eventEmitter.emit(
  'order.created',
  new OrderCreatedEvent({
    orderId: 1,
    payload: {},
  }),
);
```

#### Прослушивание событий

Чтобы объявить слушатель события, пометьте декоратором `@OnEvent()` метод, содержащий выполняемый код:

```typescript
@OnEvent('order.created')
handleOrderCreatedEvent(payload: OrderCreatedEvent) {
  // обрабатываем событие "OrderCreatedEvent"
}
```

> **Предупреждение** Подписчики на события не могут иметь область запроса (request-scoped).

Первым аргументом может быть `string` или `symbol` для простого event emitter-а и `string | symbol | Array<string | symbol>` в случае emitter-а с подстановочными знаками.

Второй аргумент (необязательный) — объект параметров слушателя:

```typescript
export type OnEventOptions = OnOptions & {
  /**
   * Если "true", добавляет указанный слушатель в начало массива слушателей (вместо конца).
   *
   * @see https://github.com/EventEmitter2/EventEmitter2#emitterprependlistenerevent-listener-options
   *
   * @default false
   */
  prependListener?: boolean;

  /**
   * Если "true", коллбэк onEvent не будет выбрасывать ошибку при обработке события. Иначе, при "false", он выбросит ошибку.
   *
   * @default true
   */
  suppressErrors?: boolean;
};
```

> **Подсказка** Подробнее об объекте параметров `OnOptions` читайте в документации [`eventemitter2`](https://github.com/EventEmitter2/EventEmitter2#emitteronevent-listener-options-objectboolean).

```typescript
@OnEvent('order.created', { async: true })
handleOrderCreatedEvent(payload: OrderCreatedEvent) {
  // обрабатываем событие "OrderCreatedEvent"
}
```

Чтобы использовать пространства имён и подстановочные знаки, передайте параметр `wildcard` в метод `EventEmitterModule#forRoot()`. Когда они включены, события могут быть либо строками (`foo.bar`), разделёнными разделителем, либо массивами (`['foo', 'bar']`). Сам разделитель тоже настраивается свойством конфигурации (`delimiter`). При включённых пространствах имён можно подписываться на события с подстановочным знаком:

```typescript
@OnEvent('order.*')
handleOrderEvents(payload: OrderCreatedEvent | OrderRemovedEvent | OrderUpdatedEvent) {
  // обрабатываем событие
}
```

Учтите, что такой подстановочный знак действует только на один сегмент. Аргумент `order.*` совпадёт, например, с событиями `order.created` и `order.shipped`, но не с `order.delayed.out_of_stock`. Чтобы слушать такие события,
используйте шаблон `multilevel wildcard` (то есть `**`), описанный в [документации](https://github.com/EventEmitter2/EventEmitter2#multi-level-wildcards) `EventEmitter2`.

С помощью этого шаблона можно, например, создать слушатель, перехватывающий все события.

```typescript
@OnEvent('**')
handleEverything(payload: any) {
  // обрабатываем событие
}
```

> **Подсказка** Класс `EventEmitter2` предоставляет несколько полезных методов для работы с событиями, например `waitFor` и `onAny`. Подробнее о них читайте [здесь](https://github.com/EventEmitter2/EventEmitter2).

#### Как не потерять события

События, сгенерированные до или во время хука жизненного цикла `onApplicationBootstrap` — например, из конструкторов модулей или метода `onModuleInit`, — могут быть пропущены, поскольку `EventSubscribersLoader` мог ещё не завершить настройку слушателей.

Чтобы избежать этой проблемы, используйте метод `waitUntilReady` у `EventEmitterReadinessWatcher`: он возвращает промис, который разрешается после регистрации всех слушателей. Этот метод можно вызвать в хуке жизненного цикла `onApplicationBootstrap` модуля, чтобы гарантировать корректный перехват всех событий.

```typescript
await this.eventEmitterReadinessWatcher.waitUntilReady();
this.eventEmitter.emit(
  'order.created',
  new OrderCreatedEvent({ orderId: 1, payload: {} }),
);
```

> **Примечание** Это нужно только для событий, генерируемых до завершения хука жизненного цикла `onApplicationBootstrap`.

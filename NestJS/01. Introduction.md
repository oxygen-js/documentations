### Введение

Nest (NestJS) — это фреймворк для построения эффективных и масштабируемых серверных приложений на [Node.js](https://nodejs.org/). Он использует прогрессивный JavaScript, написан на [TypeScript](http://www.typescriptlang.org/) и полностью его поддерживает (при этом позволяя разработчикам писать на чистом JavaScript), а также сочетает элементы ООП (объектно-ориентированного программирования), ФП (функционального программирования) и ФРП (функционального реактивного программирования).

«Под капотом» Nest использует надёжные фреймворки HTTP-серверов — [Express](https://expressjs.com/) (по умолчанию), а при желании его можно настроить и на работу с [Fastify](https://github.com/fastify/fastify)!

Nest даёт уровень абстракции над этими распространёнными фреймворками Node.js (Express/Fastify), но при этом открывает разработчику прямой доступ к их API. Благодаря этому вы можете свободно пользоваться множеством сторонних модулей, доступных для нижележащей платформы.

#### Философия

За последние годы благодаря Node.js язык JavaScript стал «лингва франка» веба — и для фронтенда, и для бэкенда. Это привело к появлению таких замечательных проектов, как [Angular](https://angular.dev/), [React](https://github.com/facebook/react) и [Vue](https://github.com/vuejs/core), которые повышают продуктивность разработчиков и позволяют создавать быстрые, тестируемые и расширяемые фронтенд-приложения. Однако, несмотря на обилие прекрасных библиотек, вспомогательных инструментов и утилит для Node (и серверного JavaScript в целом), ни одно из этих решений не отвечает по-настоящему на главный вопрос — вопрос **архитектуры**.

Nest предоставляет архитектуру приложения «из коробки», позволяя разработчикам и командам создавать хорошо тестируемые, масштабируемые, слабо связанные и легко сопровождаемые приложения. Эта архитектура во многом вдохновлена Angular.

#### Установка

Для начала вы можете сгенерировать каркас проекта с помощью [Nest CLI](<11. CLI/01. Overview.md>) либо [клонировать стартовый проект](#альтернативы) (результат будет одинаковым).

Чтобы сгенерировать каркас проекта с помощью Nest CLI, выполните следующие команды. Они создадут новый каталог проекта и наполнят его начальными основными файлами Nest и вспомогательными модулями, сформировав привычную базовую структуру. Новичкам рекомендуется создавать проект именно через **Nest CLI**. Далее мы будем придерживаться этого подхода в разделе [Первые шаги](<02. Overview/01. First steps.md>).

```bash
$ npm i -g @nestjs/cli
$ nest new project-name
```

> **Подсказка** Чтобы создать новый проект на TypeScript с более строгим набором правил, передайте команде `nest new` флаг `--strict`.

#### Альтернативы

Кроме того, стартовый проект на TypeScript можно установить через **Git**:

```bash
$ git clone https://github.com/nestjs/typescript-starter.git project
$ cd project
$ npm install
$ npm run start
```

> **Подсказка** Если вы хотите склонировать репозиторий без истории git, воспользуйтесь [degit](https://github.com/Rich-Harris/degit).

Откройте браузер и перейдите по адресу [`http://localhost:3000/`](http://localhost:3000/).

Чтобы установить вариант стартового проекта на JavaScript, используйте в приведённой выше последовательности команд `javascript-starter.git`.

Также можно начать новый проект с нуля, установив основные и вспомогательные пакеты. Учтите, что шаблонные файлы проекта в этом случае придётся настраивать самостоятельно. Как минимум вам понадобятся зависимости: `@nestjs/core`, `@nestjs/common`, `rxjs` и `reflect-metadata`. О том, как собрать полноценный проект, рассказывает короткая статья: [5 steps to create a bare minimum NestJS app from scratch!](https://dev.to/micalevisk/5-steps-to-create-a-bare-minimum-nestjs-app-from-scratch-5c3b).

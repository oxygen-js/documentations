### Фильтры исключений

Единственное отличие слоя [фильтров исключений](<../02. Overview/06. Exception filters.md>) для HTTP от соответствующего слоя микросервисов состоит в том, что вместо `HttpException` следует использовать `RpcException`.

```typescript
throw new RpcException('Invalid credentials.');
```

> **Подсказка** Класс `RpcException` импортируется из пакета `@nestjs/microservices`.

В приведённом примере Nest обработает выброшенное исключение и вернёт объект `error` следующей структуры:

```json
{
  "status": "error",
  "message": "Invalid credentials."
}
```

#### Фильтры

Фильтры исключений микросервисов работают почти так же, как фильтры исключений HTTP, с одним небольшим отличием: метод `catch()` должен возвращать `Observable`.

**`rpc-exception.filter.ts`**

```typescript
import { Catch, RpcExceptionFilter, ArgumentsHost } from '@nestjs/common';
import { Observable, throwError } from 'rxjs';
import { RpcException } from '@nestjs/microservices';

@Catch(RpcException)
export class ExceptionFilter implements RpcExceptionFilter<RpcException> {
  catch(exception: RpcException, host: ArgumentsHost): Observable<any> {
    return throwError(() => exception.getError());
  }
}
```

> **Предупреждение** Глобальные фильтры исключений микросервисов не включены по умолчанию при использовании [гибридного приложения](<../14. FAQ/06. Hybrid application.md>).

В следующем примере используется фильтр уровня метода, созданный вручную. Как и в приложениях на основе HTTP, вы можете использовать фильтры уровня контроллера (то есть указать декоратор `@UseFilters()` перед классом контроллера).

```typescript
@UseFilters(new ExceptionFilter())
@MessagePattern({ cmd: 'sum' })
accumulate(data: number[]): number {
  return (data || []).reduce((a, b) => a + b);
}
```

#### Наследование

Как правило, вы создаёте полностью настроенные фильтры исключений под требования своего приложения. Однако возможны случаи, когда достаточно просто расширить **базовый фильтр исключений** и переопределить поведение при определённых условиях.

Чтобы делегировать обработку исключения базовому фильтру, нужно унаследоваться от `BaseExceptionFilter` и вызвать унаследованный метод `catch()`.

```typescript
import { Catch, ArgumentsHost } from '@nestjs/common';
import { BaseRpcExceptionFilter } from '@nestjs/microservices';

@Catch()
export class AllExceptionsFilter extends BaseRpcExceptionFilter {
  catch(exception: any, host: ArgumentsHost) {
    return super.catch(exception, host);
  }
}
```

Приведённая реализация — лишь заготовка, демонстрирующая подход. В вашей реализации расширенного фильтра исключений будет собственная **бизнес-логика** (например, обработка различных условий).

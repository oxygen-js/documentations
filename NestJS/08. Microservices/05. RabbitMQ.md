### RabbitMQ

[RabbitMQ](https://www.rabbitmq.com/) — это лёгкий брокер сообщений с открытым исходным кодом, поддерживающий несколько протоколов обмена сообщениями. Он может разворачиваться в распределённых и федеративных конфигурациях, отвечая требованиям высокой масштабируемости и высокой доступности. Кроме того, это самый распространённый брокер сообщений, который используют по всему миру как небольшие стартапы, так и крупные предприятия.

#### Установка

Чтобы начать разрабатывать микросервисы на основе RabbitMQ, сначала установите необходимые пакеты:

```bash
$ npm i --save amqplib amqp-connection-manager
```

#### Обзор

Чтобы использовать транспорт RabbitMQ, передайте в метод `createMicroservice()` следующий объект настроек:

**`main.ts`**

```typescript
const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
  transport: Transport.RMQ,
  options: {
    urls: ['amqp://localhost:5672'],
    queue: 'cats_queue',
    queueOptions: {
      durable: false
    },
  },
});
```

> **Подсказка** Перечисление `Transport` импортируется из пакета `@nestjs/microservices`.

#### Клиент

Как и в случае других транспортов микросервисов, для создания экземпляра `ClientProxy` для RabbitMQ у вас есть <a href="01.%20Overview.md#клиент-класс-производитель">несколько вариантов</a>.

Один из способов — использовать `ClientsModule`. Чтобы создать экземпляр клиента через `ClientsModule`, импортируйте его и вызовите метод `register()`, передав объект настроек с теми же свойствами, что показаны выше для метода `createMicroservice()`, а также свойство `name`, которое будет использоваться как токен внедрения. Подробнее о `ClientsModule` читайте <a href="01.%20Overview.md#клиент-класс-производитель">здесь</a>.

```typescript
@Module({
  imports: [
    ClientsModule.register([
      {
        name: 'MATH_SERVICE',
        transport: Transport.RMQ,
        options: {
          urls: ['amqp://localhost:5672'],
          queue: 'cats_queue',
          queueOptions: {
            durable: false
          },
        },
      },
    ]),
  ]
  ...
})
```

Можно использовать и другие способы создания клиента (`ClientProxyFactory` или `@Client()`). Прочитать о них можно <a href="01.%20Overview.md#клиент-класс-производитель">здесь</a>.

#### Контекст

В более сложных сценариях может потребоваться доступ к дополнительной информации о входящем запросе. При использовании транспорта RabbitMQ вы можете обратиться к объекту `RmqContext`.

```typescript
@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: RmqContext) {
  console.log(`Pattern: ${context.getPattern()}`);
}
```

> **Подсказка** `@Payload()`, `@Ctx()` и `RmqContext` импортируются из пакета `@nestjs/microservices`.

Чтобы получить доступ к исходному сообщению RabbitMQ (со свойствами `properties`, `fields` и `content`), используйте метод `getMessage()` объекта `RmqContext`:

```typescript
@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: RmqContext) {
  console.log(context.getMessage());
}
```

Чтобы получить ссылку на [канал](https://www.rabbitmq.com/channels.html) RabbitMQ, используйте метод `getChannelRef` объекта `RmqContext`:

```typescript
@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: RmqContext) {
  console.log(context.getChannelRef());
}
```

#### Подтверждение сообщений

Чтобы сообщение никогда не терялось, RabbitMQ поддерживает [подтверждения сообщений](https://www.rabbitmq.com/confirms.html). Подтверждение отправляется потребителем обратно, чтобы сообщить RabbitMQ, что конкретное сообщение получено, обработано и брокер может его удалить. Если потребитель завершится аварийно (его канал закрыт, соединение закрыто или потеряно TCP-соединение), не отправив подтверждение, RabbitMQ поймёт, что сообщение не было обработано полностью, и вернёт его в очередь.

Чтобы включить режим ручного подтверждения, задайте свойству `noAck` значение `false`:

```typescript
options: {
  urls: ['amqp://localhost:5672'],
  queue: 'cats_queue',
  noAck: false,
  queueOptions: {
    durable: false
  },
},
```

Когда ручное подтверждение со стороны потребителя включено, мы обязаны отправлять корректное подтверждение из воркера, сигнализируя о завершении работы над задачей.

```typescript
@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: RmqContext) {
  const channel = context.getChannelRef();
  const originalMsg = context.getMessage();

  channel.ack(originalMsg);
}
```

#### Построители записей

Для настройки параметров сообщения можно использовать класс `RmqRecordBuilder` (примечание: это применимо и к сценариям на основе событий). Например, чтобы задать свойства `headers` и `priority`, используйте метод `setOptions`:

```typescript
const message = ':cat:';
const record = new RmqRecordBuilder(message)
  .setOptions({
    headers: {
      ['x-version']: '1.0.0',
    },
    priority: 3,
  })
  .build();

this.client.send('replace-emoji', record).subscribe(...);
```

> **Подсказка** Класс `RmqRecordBuilder` экспортируется из пакета `@nestjs/microservices`.

Прочитать эти значения можно и на стороне сервера, обратившись к `RmqContext`:

```typescript
@MessagePattern('replace-emoji')
replaceEmoji(@Payload() data: string, @Ctx() context: RmqContext): string {
  const { properties: { headers } } = context.getMessage();
  return headers['x-version'] === '1.0.0' ? '🐱' : '🐈';
}
```

#### Обновления состояния экземпляра

Чтобы получать обновления о состоянии подключения и нижележащего экземпляра драйвера в реальном времени, можно подписаться на поток `status`. Этот поток предоставляет обновления статуса, специфичные для выбранного драйвера. Для драйвера RMQ поток `status` испускает события `connected` и `disconnected`.

```typescript
this.client.status.subscribe((status: RmqStatus) => {
  console.log(status);
});
```

> **Подсказка** Тип `RmqStatus` импортируется из пакета `@nestjs/microservices`.

Аналогично можно подписаться на поток `status` сервера, чтобы получать уведомления о его состоянии.

```typescript
const server = app.connectMicroservice<MicroserviceOptions>(...);
server.status.subscribe((status: RmqStatus) => {
  console.log(status);
});
```

#### Прослушивание событий RabbitMQ

В некоторых случаях может понадобиться слушать внутренние события, испускаемые микросервисом. Например, вы можете слушать событие `error`, чтобы выполнять дополнительные действия при возникновении ошибки. Для этого используйте метод `on()`:

```typescript
this.client.on('error', (err) => {
  console.error(err);
});
```

Точно так же можно слушать внутренние события сервера:

```typescript
server.on<RmqEvents>('error', (err) => {
  console.error(err);
});
```

> **Подсказка** Тип `RmqEvents` импортируется из пакета `@nestjs/microservices`.

#### Доступ к нижележащему драйверу

В более продвинутых сценариях может потребоваться доступ к экземпляру нижележащего драйвера. Это бывает полезно, например, для ручного закрытия соединения или использования специфичных для драйвера методов. Однако имейте в виду, что в большинстве случаев обращаться к драйверу напрямую **не требуется**.

Для этого используйте метод `unwrap()`, который возвращает экземпляр нижележащего драйвера. В параметре обобщённого типа следует указать ожидаемый тип экземпляра драйвера.

```typescript
const managerRef =
  this.client.unwrap<import('amqp-connection-manager').AmqpConnectionManager>();
```

Аналогично можно получить доступ к нижележащему драйверу сервера:

```typescript
const managerRef =
  server.unwrap<import('amqp-connection-manager').AmqpConnectionManager>();
```

#### Подстановочные символы

RabbitMQ поддерживает подстановочные символы в ключах маршрутизации, что делает маршрутизацию сообщений гибкой. Символ `#` соответствует нулю или более словам, а символ `*` — ровно одному слову.

Например, ключ маршрутизации `cats.#` соответствует `cats`, `cats.meow` и `cats.meow.purr`. Ключ маршрутизации `cats.*` соответствует `cats.meow`, но не `cats.meow.purr`.

Чтобы включить поддержку подстановочных символов в микросервисе RabbitMQ, задайте настройке `wildcards` значение `true` в объекте настроек:

```typescript
const app = await NestFactory.createMicroservice<MicroserviceOptions>(
  AppModule,
  {
    transport: Transport.RMQ,
    options: {
      urls: ['amqp://localhost:5672'],
      queue: 'cats_queue',
      wildcards: true,
    },
  },
);
```

С такой конфигурацией вы можете использовать подстановочные символы в ключах маршрутизации при подписке на события и сообщения. Например, чтобы слушать сообщения с ключом маршрутизации `cats.#`, используйте следующий код:

```typescript
@MessagePattern('cats.#')
getCats(@Payload() data: { message: string }, @Ctx() context: RmqContext) {
  console.log(`Received message with routing key: ${context.getPattern()}`);

  return {
    message: 'Hello from the cats service!',
  }
}
```

Чтобы отправить сообщение с конкретным ключом маршрутизации, используйте метод `send()` экземпляра `ClientProxy`:

```typescript
this.client.send('cats.meow', { message: 'Meow!' }).subscribe((response) => {
  console.log(response);
});
```

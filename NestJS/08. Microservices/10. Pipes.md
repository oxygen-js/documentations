### Pipes

Между [обычными pipes](<../02. Overview/07. Pipes.md>) и pipes для микросервисов нет принципиальной разницы. Единственное отличие: вместо `HttpException` следует выбрасывать `RpcException`.

> **Подсказка** Класс `RpcException` экспортируется из пакета `@nestjs/microservices`.

#### Привязка pipes

В следующем примере используется вручную созданный pipe уровня метода. Как и в приложениях на базе HTTP, вы можете применять pipes уровня контроллера (то есть указать декоратор `@UsePipes()` перед классом контроллера).

```typescript
@UsePipes(new ValidationPipe({ exceptionFactory: (errors) => new RpcException(errors) }))
@MessagePattern({ cmd: 'sum' })
accumulate(data: number[]): number {
  return (data || []).reduce((a, b) => a + b);
}
```

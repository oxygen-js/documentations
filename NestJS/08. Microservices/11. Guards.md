### Guards

Между guard'ами микросервисов и [guard'ами обычных HTTP-приложений](<../02. Overview/08. Guards.md>) нет принципиальной разницы.
Единственное отличие — вместо `HttpException` следует использовать `RpcException`.

> **Подсказка** Класс `RpcException` экспортируется из пакета `@nestjs/microservices`.

#### Привязка guard'ов

В следующем примере используется guard уровня метода. Как и в HTTP-приложениях, можно использовать guard'ы уровня контроллера (то есть указать декоратор `@UseGuards()` перед классом контроллера).

```typescript
@UseGuards(AuthGuard)
@MessagePattern({ cmd: 'sum' })
accumulate(data: number[]): number {
  return (data || []).reduce((a, b) => a + b);
}
```

### Плагины в Apollo

Плагины позволяют расширять базовую функциональность Apollo Server, выполняя произвольные действия в ответ на определённые события. Сейчас эти события соответствуют отдельным фазам жизненного цикла GraphQL-запроса, а также запуску самого Apollo Server (подробнее [здесь](https://www.apollographql.com/docs/apollo-server/integrations/plugins/)). Например, простой плагин логирования может записывать строку GraphQL-запроса для каждого обращения к Apollo Server.

#### Пользовательские плагины

Чтобы создать плагин, объявите класс с декоратором `@Plugin`, экспортируемым из пакета `@nestjs/apollo`. Кроме того, для более удобного автодополнения кода реализуйте интерфейс `ApolloServerPlugin` из пакета `@apollo/server`.

```typescript
import { ApolloServerPlugin, GraphQLRequestListener } from '@apollo/server';
import { Plugin } from '@nestjs/apollo';

@Plugin()
export class LoggingPlugin implements ApolloServerPlugin {
  async requestDidStart(): Promise<GraphQLRequestListener<any>> {
    console.log('Request started');
    return {
      async willSendResponse() {
        console.log('Will send response');
      },
    };
  }
}
```

После этого мы можем зарегистрировать `LoggingPlugin` как провайдер.

```typescript
@Module({
  providers: [LoggingPlugin],
})
export class CommonModule {}
```

Nest автоматически создаст экземпляр плагина и применит его к Apollo Server.

#### Использование внешних плагинов

Ряд плагинов доступен «из коробки». Чтобы воспользоваться готовым плагином, просто импортируйте его и добавьте в массив `plugins`:

```typescript
GraphQLModule.forRoot({
  // ...
  plugins: [ApolloServerOperationRegistry({ /* options */})]
}),
```

> **Подсказка** Плагин `ApolloServerOperationRegistry` экспортируется из пакета `@apollo/server-plugin-operation-registry`.

#### Плагины в Mercurius

Некоторые из существующих Fastify-плагинов, специфичных для mercurius, должны загружаться в дереве плагинов после самого плагина mercurius (подробнее [здесь](https://mercurius.dev/#/docs/plugins)).

> **Предупреждение** [mercurius-upload](https://github.com/mercurius-js/mercurius-upload) — исключение, его следует регистрировать в главном файле.

Для этого `MercuriusDriver` предоставляет необязательную опцию конфигурации `plugins`. Она представляет собой массив объектов с двумя атрибутами: `plugin` и его `options`. Таким образом, регистрация [плагина кэширования](https://github.com/mercurius-js/cache) будет выглядеть так:

```typescript
GraphQLModule.forRoot({
  driver: MercuriusDriver,
  // ...
  plugins: [
    {
      plugin: cache,
      options: {
        ttl: 10,
        policy: {
          Query: {
            add: true
          }
        }
      },
    }
  ]
}),
```

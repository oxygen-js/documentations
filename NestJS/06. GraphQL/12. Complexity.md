### Сложность запросов

> **Предупреждение** Эта глава относится только к подходу code first.

Сложность запросов (query complexity) позволяет определить, насколько «дорого» обходятся те или иные поля, и ограничить запросы **максимальной сложностью**. Идея в том, чтобы выразить сложность каждого поля простым числом. Распространённое значение по умолчанию — сложность `1` для каждого поля. Кроме того, расчёт сложности GraphQL-запроса можно настроить с помощью так называемых оценщиков сложности (complexity estimators). Оценщик сложности — это обычная функция, вычисляющая сложность поля. К правилу можно добавить любое количество оценщиков, они будут выполняться один за другим. Сложность поля определяет первый оценщик, вернувший числовое значение.

Пакет `@nestjs/graphql` прекрасно интегрируется с такими инструментами, как [graphql-query-complexity](https://github.com/slicknode/graphql-query-complexity), который предлагает решение на основе анализа стоимости запросов. С помощью этой библиотеки вы можете отклонять запросы к вашему GraphQL-серверу, признанные слишком дорогими для выполнения.

#### Установка

Чтобы начать работу, сначала установим необходимую зависимость.

```bash
$ npm install --save graphql-query-complexity
```

#### Начало работы

После завершения установки мы можем определить класс `ComplexityPlugin`:

```typescript
import { GraphQLSchemaHost } from '@nestjs/graphql';
import { Plugin } from '@nestjs/apollo';
import {
  ApolloServerPlugin,
  BaseContext,
  GraphQLRequestListener,
} from '@apollo/server';
import { GraphQLError } from 'graphql';
import {
  fieldExtensionsEstimator,
  getComplexity,
  simpleEstimator,
} from 'graphql-query-complexity';

@Plugin()
export class ComplexityPlugin implements ApolloServerPlugin {
  constructor(private gqlSchemaHost: GraphQLSchemaHost) {}

  async requestDidStart(): Promise<GraphQLRequestListener<BaseContext>> {
    const maxComplexity = 20;
    const { schema } = this.gqlSchemaHost;

    return {
      async didResolveOperation({ request, document }) {
        const complexity = getComplexity({
          schema,
          operationName: request.operationName,
          query: document,
          variables: request.variables,
          estimators: [
            fieldExtensionsEstimator(),
            simpleEstimator({ defaultComplexity: 1 }),
          ],
        });
        if (complexity > maxComplexity) {
          throw new GraphQLError(
            `Query is too complex: ${complexity}. Maximum allowed complexity: ${maxComplexity}`,
          );
        }
        console.log('Query Complexity:', complexity);
      },
    };
  }
}
```

Для наглядности мы задали максимально допустимую сложность равной `20`. В примере выше использованы 2 оценщика: `simpleEstimator` и `fieldExtensionsEstimator`.

- `simpleEstimator`: простой оценщик возвращает фиксированную сложность для каждого поля
- `fieldExtensionsEstimator`: оценщик расширений полей извлекает значение сложности для каждого поля вашей схемы

> **Подсказка** Не забудьте добавить этот класс в массив providers какого-либо модуля.

#### Сложность на уровне поля

С подключённым плагином мы можем задать сложность любого поля, указав свойство `complexity` в объекте опций, передаваемом в декоратор `@Field()`, например так:

```typescript
@Field({ complexity: 3 })
title: string;
```

Либо можно определить функцию-оценщик:

```typescript
@Field({ complexity: (options: ComplexityEstimatorArgs) => ... })
title: string;
```

#### Сложность на уровне Query/Mutation

Кроме того, свойство `complexity` можно указать и в декораторах `@Query()` и `@Mutation()`:

```typescript
@Query({ complexity: (options: ComplexityEstimatorArgs) => options.args.count * options.childComplexity })
items(@Args('count') count: number) {
  return this.itemsService.getItems({ count });
}
```

### Генерация SDL

> **Предупреждение** Эта глава применима только к подходу code first.

Чтобы сгенерировать SDL-схему GraphQL вручную (то есть без запуска приложения, подключения к базе данных, подключения резолверов и т. д.), используйте `GraphQLSchemaBuilderModule`.

```typescript
async function generateSchema() {
  const app = await NestFactory.create(GraphQLSchemaBuilderModule);
  await app.init();

  const gqlSchemaFactory = app.get(GraphQLSchemaFactory);
  const schema = await gqlSchemaFactory.create([RecipesResolver]);
  console.log(printSchema(schema));
}
```

> **Подсказка** `GraphQLSchemaBuilderModule` и `GraphQLSchemaFactory` импортируются из пакета `@nestjs/graphql`. Функция `printSchema` импортируется из пакета `graphql`.

#### Использование

Метод `gqlSchemaFactory.create()` принимает массив ссылок на классы резолверов. Например:

```typescript
const schema = await gqlSchemaFactory.create([
  RecipesResolver,
  AuthorsResolver,
  PostsResolver,
]);
```

Он также принимает второй необязательный аргумент — массив классов скаляров:

```typescript
const schema = await gqlSchemaFactory.create(
  [RecipesResolver, AuthorsResolver, PostsResolver],
  [DurationScalar, DateScalar],
);
```

Наконец, вы можете передать объект с настройками:

```typescript
const schema = await gqlSchemaFactory.create([RecipesResolver], {
  skipCheck: true,
  orphanedTypes: [],
});
```

- `skipCheck`: пропустить валидацию схемы; логическое значение, по умолчанию `false`
- `orphanedTypes`: список классов, которые не упоминаются явно (не входят в граф объектов), но должны быть сгенерированы. Обычно, если класс объявлен, но больше нигде в графе не используется, он пропускается. Значение свойства — массив ссылок на классы.

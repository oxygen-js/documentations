### Интерфейсы

Как и многие системы типов, GraphQL поддерживает интерфейсы. **Интерфейс** — это абстрактный тип, включающий определённый набор полей, которые тип обязан содержать, чтобы реализовать этот интерфейс (подробнее читайте [здесь](https://graphql.org/learn/schema/#interfaces)).

#### Code first

При подходе code first интерфейс GraphQL определяется созданием абстрактного класса, помеченного декоратором `@InterfaceType()`, который экспортируется из `@nestjs/graphql`.

```typescript
import { Field, ID, InterfaceType } from '@nestjs/graphql';

@InterfaceType()
export abstract class Character {
  @Field(() => ID)
  id: string;

  @Field()
  name: string;
}
```

> **Предупреждение** Интерфейсы TypeScript нельзя использовать для определения интерфейсов GraphQL.

В результате в схеме GraphQL на SDL будет сгенерирован следующий фрагмент:

```graphql
interface Character {
  id: ID!
  name: String!
}
```

Теперь, чтобы реализовать интерфейс `Character`, используйте ключ `implements`:

```typescript
@ObjectType({
  implements: () => [Character],
})
export class Human implements Character {
  id: string;
  name: string;
}
```

> **Подсказка** Декоратор `@ObjectType()` экспортируется из пакета `@nestjs/graphql`.

Функция `resolveType()`, которую библиотека генерирует по умолчанию, определяет тип по значению, возвращённому методом резолвера. Это значит, что вы должны возвращать экземпляры классов (возвращать литеральные объекты JavaScript нельзя).

Чтобы задать собственную функцию `resolveType()`, передайте свойство `resolveType` в объект настроек декоратора `@InterfaceType()`:

```typescript
@InterfaceType({
  resolveType(book) {
    if (book.colors) {
      return ColoringBook;
    }
    return TextBook;
  },
})
export abstract class Book {
  @Field(() => ID)
  id: string;

  @Field()
  title: string;
}
```

#### Резолверы интерфейсов

До сих пор интерфейсы позволяли вам делиться с объектами только определениями полей. Если вы хотите использовать совместно и саму реализацию резолверов полей, создайте отдельный резолвер интерфейса:

```typescript
import { Resolver, ResolveField, Parent, Info } from '@nestjs/graphql';

@Resolver((type) => Character) // Напоминание: Character — это интерфейс
export class CharacterInterfaceResolver {
  @ResolveField(() => [Character])
  friends(
    @Parent() character, // Разрешённый объект, реализующий Character
    @Info() { parentType }, // Тип объекта, реализующего Character
    @Args('search', { type: () => String }) searchTerm: string,
  ) {
    // Получаем друзей персонажа
    return [];
  }
}
```

Теперь резолвер поля `friends` автоматически регистрируется для всех объектных типов, реализующих интерфейс `Character`.

> **Предупреждение** Для этого в конфигурации `GraphQLModule` свойство `inheritResolversFromInterfaces` должно быть равно true.

#### Schema first

Чтобы определить интерфейс при подходе schema first, просто создайте интерфейс GraphQL на SDL.

```graphql
interface Character {
  id: ID!
  name: String!
}
```

Затем вы можете воспользоваться генерацией типов (как показано в главе [быстрый старт](<01. Quick start.md>)), чтобы получить соответствующие определения TypeScript:

```typescript
export interface Character {
  id: string;
  name: string;
}
```

Интерфейсам требуется дополнительное поле `__resolveType` в карте резолверов — оно определяет, в какой тип должен разрешаться интерфейс. Создадим класс `CharactersResolver` и определим метод `__resolveType`:

```typescript
@Resolver('Character')
export class CharactersResolver {
  @ResolveField()
  __resolveType(value) {
    if ('age' in value) {
      return Person;
    }
    return null;
  }
}
```

> **Подсказка** Все декораторы экспортируются из пакета `@nestjs/graphql`.

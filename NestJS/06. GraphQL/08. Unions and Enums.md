### Union-типы

Union-типы очень похожи на интерфейсы, но они не задают никаких общих полей между типами (подробнее [здесь](https://graphql.org/learn/schema/#union-types)). Union-типы удобны, когда из одного поля нужно возвращать несовместимые между собой типы данных.

#### Code first

Чтобы определить union-тип GraphQL, нужно определить классы, из которых он будет состоять. Следуя [примеру](https://www.apollographql.com/docs/apollo-server/schema/unions-interfaces/#union-type) из документации Apollo, создадим два класса. Сначала `Book`:

```typescript
import { Field, ObjectType } from '@nestjs/graphql';

@ObjectType()
export class Book {
  @Field()
  title: string;
}
```

А затем `Author`:

```typescript
import { Field, ObjectType } from '@nestjs/graphql';

@ObjectType()
export class Author {
  @Field()
  name: string;
}
```

После этого зарегистрируйте union `ResultUnion` с помощью функции `createUnionType`, экспортируемой из пакета `@nestjs/graphql`:

```typescript
export const ResultUnion = createUnionType({
  name: 'ResultUnion',
  types: () => [Author, Book] as const,
});
```

> **Предупреждение** Массив, возвращаемый свойством `types` функции `createUnionType`, должен сопровождаться const-утверждением. Если const-утверждение не указано, во время компиляции будет сгенерирован неверный файл деклараций, и при использовании из другого проекта возникнет ошибка.

Теперь мы можем сослаться на `ResultUnion` в нашем запросе:

```typescript
@Query(() => [ResultUnion])
search(): Array<typeof ResultUnion> {
  return [new Author(), new Book()];
}
```

В результате будет сгенерирована следующая часть схемы GraphQL в формате SDL:

```graphql
type Author {
  name: String!
}

type Book {
  title: String!
}

union ResultUnion = Author | Book

type Query {
  search: [ResultUnion!]!
}
```

Функция `resolveType()`, генерируемая библиотекой по умолчанию, определяет тип на основе значения, возвращённого методом резолвера. Это означает, что возвращать необходимо экземпляры классов, а не литеральные объекты JavaScript.

Чтобы задать собственную функцию `resolveType()`, передайте свойство `resolveType` в объект настроек, передаваемый в функцию `createUnionType()`:

```typescript
export const ResultUnion = createUnionType({
  name: 'ResultUnion',
  types: () => [Author, Book] as const,
  resolveType(value) {
    if (value.name) {
      return Author;
    }
    if (value.title) {
      return Book;
    }
    return null;
  },
});
```

#### Schema first

Чтобы определить union в подходе schema first, просто создайте union GraphQL с помощью SDL.

```graphql
type Author {
  name: String!
}

type Book {
  title: String!
}

union ResultUnion = Author | Book
```

Затем вы можете воспользоваться генерацией типов (как показано в главе [Быстрый старт](<01. Quick start.md>)), чтобы получить соответствующие определения TypeScript:

```typescript
export class Author {
  name: string;
}

export class Book {
  title: string;
}

export type ResultUnion = Author | Book;
```

Для union-типов в карте резолверов требуется дополнительное поле `__resolveType`, определяющее, в какой тип должен разрешаться union. Обратите также внимание, что класс `ResultUnionResolver` необходимо зарегистрировать как провайдер в каком-либо модуле. Создадим класс `ResultUnionResolver` и определим метод `__resolveType`.

```typescript
@Resolver('ResultUnion')
export class ResultUnionResolver {
  @ResolveField()
  __resolveType(value) {
    if (value.name) {
      return 'Author';
    }
    if (value.title) {
      return 'Book';
    }
    return null;
  }
}
```

> **Подсказка** Все декораторы экспортируются из пакета `@nestjs/graphql`.

### Перечисления (enums)

Перечисления — это особая разновидность скалярных типов, ограниченная определённым набором допустимых значений (подробнее [здесь](https://graphql.org/learn/schema/#enumeration-types)). Это позволяет:

- проверять, что любые аргументы этого типа имеют одно из допустимых значений
- сообщать средствами системы типов, что поле всегда будет иметь одно из конечного набора значений

#### Code first

При подходе code first тип-перечисление GraphQL определяется простым созданием перечисления TypeScript.

```typescript
export enum AllowedColor {
  RED,
  GREEN,
  BLUE,
}
```

После этого зарегистрируйте перечисление `AllowedColor` с помощью функции `registerEnumType`, экспортируемой из пакета `@nestjs/graphql`:

```typescript
registerEnumType(AllowedColor, {
  name: 'AllowedColor',
});
```

Теперь вы можете ссылаться на `AllowedColor` в наших типах:

```typescript
@Field(type => AllowedColor)
favoriteColor: AllowedColor;
```

В результате будет сгенерирована следующая часть схемы GraphQL в формате SDL:

```graphql
enum AllowedColor {
  RED
  GREEN
  BLUE
}
```

Чтобы добавить описание перечисления, передайте свойство `description` в функцию `registerEnumType()`.

```typescript
registerEnumType(AllowedColor, {
  name: 'AllowedColor',
  description: 'The supported colors.',
});
```

Чтобы добавить описания значений перечисления или пометить значение как устаревшее, передайте свойство `valuesMap`:

```typescript
registerEnumType(AllowedColor, {
  name: 'AllowedColor',
  description: 'The supported colors.',
  valuesMap: {
    RED: {
      description: 'The default color.',
    },
    BLUE: {
      deprecationReason: 'Too blue.',
    },
  },
});
```

Это сгенерирует следующую схему GraphQL в формате SDL:

```graphql
"""
The supported colors.
"""
enum AllowedColor {
  """
  The default color.
  """
  RED
  GREEN
  BLUE @deprecated(reason: "Too blue.")
}
```

#### Schema first

Чтобы определить перечисление в подходе schema first, просто создайте перечисление GraphQL с помощью SDL.

```graphql
enum AllowedColor {
  RED
  GREEN
  BLUE
}
```

Затем вы можете воспользоваться генерацией типов (как показано в главе [Быстрый старт](<01. Quick start.md>)), чтобы получить соответствующие определения TypeScript:

```typescript
export enum AllowedColor {
  RED
  GREEN
  BLUE
}
```

Иногда во внутренней реализации бэкенд использует для перечисления значения, отличные от публичного API. В этом примере API содержит `RED`, однако в резолверах мы можем использовать вместо него `#f00` (подробнее [здесь](https://www.apollographql.com/docs/apollo-server/schema/scalars-enums/#internal-values)). Чтобы этого добиться, объявите объект-резолвер для перечисления `AllowedColor`:

```typescript
export const allowedColorResolver: Record<keyof typeof AllowedColor, any> = {
  RED: '#f00',
};
```

> **Подсказка** Все декораторы экспортируются из пакета `@nestjs/graphql`.

Затем используйте этот объект-резолвер вместе со свойством `resolvers` метода `GraphQLModule#forRoot()`:

```typescript
GraphQLModule.forRoot({
  resolvers: {
    AllowedColor: allowedColorResolver,
  },
});
```

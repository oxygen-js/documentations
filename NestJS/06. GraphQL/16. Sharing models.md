### Совместное использование моделей

> **Предупреждение** Эта глава относится только к подходу code first.

Одно из главных преимуществ использования TypeScript на бэкенде проекта — возможность переиспользовать те же самые модели во фронтенд-приложении на TypeScript через общий пакет.

Но есть проблема: модели, созданные в подходе code first, густо увешаны декораторами, связанными с GraphQL. На фронтенде эти декораторы не нужны и негативно сказываются на производительности.

#### Использование shim для моделей

Чтобы решить эту проблему, NestJS предоставляет «заглушку» (shim), которая позволяет заменить исходные декораторы на пустой код с помощью конфигурации `webpack` (или аналогичного сборщика).
Чтобы задействовать эту заглушку, настройте алиас между пакетом `@nestjs/graphql` и заглушкой.

Например, для webpack это делается так:

```typescript
resolve: { // see: https://webpack.js.org/configuration/resolve/
  alias: {
      "@nestjs/graphql": path.resolve(__dirname, "../node_modules/@nestjs/graphql/dist/extra/graphql-model-shim")
  }
}
```

> **Подсказка** У пакета [TypeORM](<../04. Techniques/02. Database.md>) есть похожая заглушка, её можно найти [здесь](https://github.com/typeorm/typeorm/blob/master/extra/typeorm-model-shim.js).

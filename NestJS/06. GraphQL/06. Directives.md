### Директивы

Директива может быть привязана к полю или к включению фрагмента и способна влиять на выполнение запроса любым образом, который нужен серверу (подробнее читайте [здесь](https://graphql.org/learn/queries/#directives)). Спецификация GraphQL предоставляет несколько директив по умолчанию:

- `@include(if: Boolean)` — включать это поле в результат, только если аргумент равен true
- `@skip(if: Boolean)` — пропустить это поле, если аргумент равен true
- `@deprecated(reason: String)` — помечает поле как устаревшее с указанием сообщения

Директива — это идентификатор с символом `@` впереди, за которым необязательно следует список именованных аргументов; она может располагаться практически после любого элемента языка запросов и языка схем GraphQL.

#### Пользовательские директивы

Чтобы задать, что должно происходить, когда Apollo/Mercurius встречает вашу директиву, создайте функцию-трансформер. Такая функция использует `mapSchema` для обхода мест в схеме (определения полей, определения типов и т. д.) и выполнения соответствующих преобразований.

```typescript
import { getDirective, MapperKind, mapSchema } from '@graphql-tools/utils';
import { defaultFieldResolver, GraphQLSchema } from 'graphql';

export function upperDirectiveTransformer(
  schema: GraphQLSchema,
  directiveName: string,
) {
  return mapSchema(schema, {
    [MapperKind.OBJECT_FIELD]: (fieldConfig) => {
      const upperDirective = getDirective(
        schema,
        fieldConfig,
        directiveName,
      )?.[0];

      if (upperDirective) {
        const { resolve = defaultFieldResolver } = fieldConfig;

        // Заменяем исходный резолвер функцией, которая *сначала* вызывает
        // исходный резолвер, а затем переводит его результат в верхний регистр
        fieldConfig.resolve = async function (source, args, context, info) {
          const result = await resolve(source, args, context, info);
          if (typeof result === 'string') {
            return result.toUpperCase();
          }
          return result;
        };
        return fieldConfig;
      }
    },
  });
}
```

Теперь примените функцию преобразования `upperDirectiveTransformer` в методе `GraphQLModule#forRoot` через параметр `transformSchema`:

```typescript
GraphQLModule.forRoot({
  // ...
  transformSchema: (schema) => upperDirectiveTransformer(schema, 'upper'),
});
```

После регистрации директиву `@upper` можно использовать в нашей схеме. Однако способ её применения зависит от выбранного подхода (code first или schema first).

#### Code first

В подходе code first для применения директивы используется декоратор `@Directive()`.

```typescript
@Directive('@upper')
@Field()
title: string;
```

> **Подсказка** Декоратор `@Directive()` экспортируется из пакета `@nestjs/graphql`.

Директивы можно применять к полям, резолверам полей, входным и объектным типам, а также к запросам, мутациям и подпискам. Вот пример директивы, применённой на уровне обработчика запроса:

```typescript
@Directive('@deprecated(reason: "This query will be removed in the next version")')
@Query(() => Author, { name: 'author' })
async getAuthor(@Args({ name: 'id', type: () => Int }) id: number) {
  return this.authorsService.findOneById(id);
}
```

> **Предупреждение** Директивы, применённые через декоратор `@Directive()`, не попадают в сгенерированный файл определения схемы.

Наконец, не забудьте объявить директивы в `GraphQLModule`:

```typescript
GraphQLModule.forRoot({
  // ...,
  transformSchema: schema => upperDirectiveTransformer(schema, 'upper'),
  buildSchemaOptions: {
    directives: [
      new GraphQLDirective({
        name: 'upper',
        locations: [DirectiveLocation.FIELD_DEFINITION],
      }),
    ],
  },
}),
```

> **Подсказка** И `GraphQLDirective`, и `DirectiveLocation` экспортируются из пакета `graphql`.

#### Schema first

В подходе schema first директивы применяются напрямую в SDL.

```graphql
directive @upper on FIELD_DEFINITION

type Post {
  id: Int!
  title: String! @upper
  votes: Int
}
```

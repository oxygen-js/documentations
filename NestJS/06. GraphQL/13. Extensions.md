### Расширения

> **Предупреждение** Эта глава относится только к подходу code first.

Расширения — это **продвинутая низкоуровневая возможность**, позволяющая определять произвольные данные в конфигурации типов. Прикрепление пользовательских метаданных к определённым полям даёт возможность строить более сложные и универсальные решения. Например, с помощью расширений вы можете задать роли, необходимые для доступа к конкретным полям. Такие роли можно считать во время выполнения, чтобы определить, достаточно ли у вызывающей стороны прав на получение того или иного поля.

#### Добавление пользовательских метаданных

Чтобы прикрепить к полю пользовательские метаданные, используйте декоратор `@Extensions()`, экспортируемый из пакета `@nestjs/graphql`.

```typescript
@Field()
@Extensions({ role: Role.ADMIN })
password: string;
```

В примере выше мы присвоили свойству метаданных `role` значение `Role.ADMIN`. `Role` — это простой TypeScript-enum, объединяющий все роли пользователей, доступные в нашей системе.

Обратите внимание: помимо установки метаданных на поля, вы можете применять декоратор `@Extensions()` на уровне класса и на уровне метода (например, на обработчике запроса).

#### Использование пользовательских метаданных

Логика, использующая пользовательские метаданные, может быть сколь угодно сложной. Например, вы можете создать простой interceptor, который сохраняет или логирует события при каждом вызове метода, либо [field middleware](<09. Field middleware.md>), сопоставляющий роли, необходимые для получения поля, с правами вызывающей стороны (система прав на уровне полей).

Для наглядности определим `checkRoleMiddleware`, который сравнивает роль пользователя (здесь она захардкожена) с ролью, требуемой для доступа к целевому полю:

```typescript
export const checkRoleMiddleware: FieldMiddleware = async (
  ctx: MiddlewareContext,
  next: NextFn,
) => {
  const { info } = ctx;
  const { extensions } = info.parentType.getFields()[info.fieldName];

  /**
   * В реальном приложении переменная "userRole"
   * должна отражать роль вызывающего пользователя (например, "ctx.user.role").
   */
  const userRole = Role.USER;
  if (userRole === extensions.role) {
    // или просто "return null", чтобы проигнорировать
    throw new ForbiddenException(
      `User does not have sufficient permissions to access "${info.fieldName}" field.`,
    );
  }
  return next();
};
```

После этого мы можем зарегистрировать middleware для поля `password` следующим образом:

```typescript
@Field({ middleware: [checkRoleMiddleware] })
@Extensions({ role: Role.ADMIN })
password: string;
```

### Field middleware

> **Предупреждение** Эта глава применима только к подходу code first.

Field middleware позволяет выполнять произвольный код **до или после** разрешения поля. С его помощью можно преобразовать результат поля, проверить его аргументы или даже проверить роли на уровне поля (например, необходимые для доступа к тому полю, для которого выполняется функция middleware).

К одному полю можно подключить несколько функций middleware. В этом случае они будут вызываться последовательно по цепочке, где каждая предыдущая функция решает, вызывать ли следующую. Порядок функций в массиве `middleware` важен. Первый резолвер — это «самый внешний» слой, поэтому он выполняется первым и последним (аналогично пакету `graphql-middleware`). Второй резолвер — «второй снаружи» слой, поэтому он выполняется вторым и предпоследним.

#### Начало работы

Начнём с простого middleware, который будет логировать значение поля перед его отправкой клиенту:

```typescript
import { FieldMiddleware, MiddlewareContext, NextFn } from '@nestjs/graphql';

const loggerMiddleware: FieldMiddleware = async (
  ctx: MiddlewareContext,
  next: NextFn,
) => {
  const value = await next();
  console.log(value);
  return value;
};
```

> **Подсказка** `MiddlewareContext` — объект, состоящий из тех же аргументов, которые обычно получает функция-резолвер GraphQL (`{ source, args, context, info }`), а `NextFn` — функция, позволяющая выполнить следующий middleware в стеке (привязанном к этому полю) либо сам резолвер поля.

> **Предупреждение** Функции field middleware не могут внедрять зависимости и не имеют доступа к DI-контейнеру Nest, поскольку задуманы очень лёгкими и не должны выполнять потенциально долгие операции (например, получение данных из базы). Если вам нужно обратиться к внешним сервисам или запросить данные из источника, сделайте это в guard или interceptor, привязанном к корневому обработчику query/mutation, и поместите результат в объект `context`, к которому можно обратиться изнутри field middleware (а именно из объекта `MiddlewareContext`).

Обратите внимание, что field middleware должен соответствовать интерфейсу `FieldMiddleware`. В примере выше мы сначала вызываем функцию `next()` (которая выполняет сам резолвер поля и возвращает его значение), а затем выводим это значение в терминал. Кроме того, значение, возвращённое из функции middleware, полностью перекрывает предыдущее, и поскольку никаких изменений мы вносить не хотим, то просто возвращаем исходное значение.

Теперь можно зарегистрировать наш middleware прямо в декораторе `@Field()`:

```typescript
@ObjectType()
export class Recipe {
  @Field({ middleware: [loggerMiddleware] })
  title: string;
}
```

Отныне при каждом запросе поля `title` объектного типа `Recipe` исходное значение поля будет выводиться в консоль.

> **Подсказка** Чтобы узнать, как реализовать систему прав на уровне полей с помощью возможности [extensions](<13. Extensions.md>), посмотрите этот [раздел](<13. Extensions.md#использование-пользовательских-метаданных>).

> **Предупреждение** Field middleware можно применять только к классам `ObjectType`. Подробности — в этом [issue](https://github.com/nestjs/graphql/issues/2446).

Кроме того, как упоминалось выше, значением поля можно управлять изнутри функции middleware. Для демонстрации переведём название рецепта в верхний регистр (если оно есть):

```typescript
const value = await next();
return value?.toUpperCase();
```

В этом случае каждое название при запросе будет автоматически приводиться к верхнему регистру.

Точно так же field middleware можно привязать к пользовательскому резолверу поля (методу, помеченному декоратором `@ResolveField()`):

```typescript
@ResolveField(() => String, { middleware: [loggerMiddleware] })
title() {
  return 'Placeholder';
}
```

> **Предупреждение** Если enhancer'ы включены на уровне резолвера поля ([подробнее](<17. Other features.md#запуск-enhancers-на-уровне-резолверов-полей>)), функции field middleware будут выполняться раньше любых interceptor'ов, guard'ов и т. п., **привязанных к методу** (но после enhancer'ов корневого уровня, зарегистрированных для обработчиков query или mutation).

#### Глобальный field middleware

Помимо привязки middleware непосредственно к конкретному полю, можно зарегистрировать одну или несколько функций middleware глобально. В этом случае они будут автоматически подключены ко всем полям ваших объектных типов.

```typescript
GraphQLModule.forRoot({
  autoSchemaFile: 'schema.gql',
  buildSchemaOptions: {
    fieldMiddleware: [loggerMiddleware],
  },
}),
```

> **Подсказка** Глобально зарегистрированные функции field middleware выполняются **раньше** зарегистрированных локально (то есть привязанных непосредственно к конкретным полям).

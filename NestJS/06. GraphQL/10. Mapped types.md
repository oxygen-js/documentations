### Mapped-типы

> **Предупреждение** Эта глава относится только к подходу code first.

По мере разработки возможностей вроде CRUD (Create/Read/Update/Delete) часто оказывается полезно строить варианты базового типа сущности. Nest предоставляет несколько служебных функций, выполняющих преобразования типов, чтобы упростить эту задачу.

#### Partial

При создании типов для валидации входных данных (их также называют Data Transfer Object, или DTO) часто удобно построить варианты **create** и **update** на основе одного и того же типа. Например, вариант **create** может требовать все поля, а вариант **update** — делать все поля необязательными.

Чтобы упростить эту задачу и уменьшить объём шаблонного кода, Nest предоставляет служебную функцию `PartialType()`.

Функция `PartialType()` возвращает тип (класс), в котором все свойства исходного типа помечены как необязательные. Например, пусть у нас есть такой тип **create**:

```typescript
@InputType()
class CreateUserInput {
  @Field()
  email: string;

  @Field()
  password: string;

  @Field()
  firstName: string;
}
```

По умолчанию все эти поля обязательны. Чтобы получить тип с теми же полями, но необязательными, используйте `PartialType()`, передав ссылку на класс (`CreateUserInput`) в качестве аргумента:

```typescript
@InputType()
export class UpdateUserInput extends PartialType(CreateUserInput) {}
```

> **Подсказка** Функция `PartialType()` импортируется из пакета `@nestjs/graphql`.

Функция `PartialType()` принимает необязательный второй аргумент — ссылку на фабрику декоратора. С его помощью можно изменить декоратор, применяемый к получившемуся (дочернему) классу. Если аргумент не указан, дочерний класс фактически использует тот же декоратор, что и **родительский** класс (класс, указанный в первом аргументе). В примере выше мы наследуемся от `CreateUserInput`, помеченного декоратором `@InputType()`. Поскольку мы хотим, чтобы `UpdateUserInput` тоже воспринимался как помеченный `@InputType()`, передавать `InputType` вторым аргументом не понадобилось. Если же родительский и дочерний типы различаются (например, родитель помечен `@ObjectType`), вторым аргументом нужно передать `InputType`. Например:

```typescript
@InputType()
export class UpdateUserInput extends PartialType(User, InputType) {}
```

#### Pick

Функция `PickType()` создаёт новый тип (класс), выбирая набор свойств из исходного типа. Например, пусть мы начинаем с такого типа:

```typescript
@InputType()
class CreateUserInput {
  @Field()
  email: string;

  @Field()
  password: string;

  @Field()
  firstName: string;
}
```

С помощью служебной функции `PickType()` мы можем выбрать из этого класса набор свойств:

```typescript
@InputType()
export class UpdateEmailInput extends PickType(CreateUserInput, [
  'email',
] as const) {}
```

> **Подсказка** Функция `PickType()` импортируется из пакета `@nestjs/graphql`.

#### Omit

Функция `OmitType()` создаёт тип, беря все свойства исходного типа и удаляя из них определённый набор ключей. Например, пусть мы начинаем с такого типа:

```typescript
@InputType()
class CreateUserInput {
  @Field()
  email: string;

  @Field()
  password: string;

  @Field()
  firstName: string;
}
```

Мы можем получить производный тип, содержащий все свойства **кроме** `email`, как показано ниже. В этой конструкции второй аргумент `OmitType` — массив имён свойств.

```typescript
@InputType()
export class UpdateUserInput extends OmitType(CreateUserInput, [
  'email',
] as const) {}
```

> **Подсказка** Функция `OmitType()` импортируется из пакета `@nestjs/graphql`.

#### Intersection

Функция `IntersectionType()` объединяет два типа в один новый тип (класс). Например, пусть мы начинаем с двух типов:

```typescript
@InputType()
class CreateUserInput {
  @Field()
  email: string;

  @Field()
  password: string;
}

@ObjectType()
export class AdditionalUserInfo {
  @Field()
  firstName: string;

  @Field()
  lastName: string;
}
```

Мы можем создать новый тип, объединяющий все свойства обоих типов.

```typescript
@InputType()
export class UpdateUserInput extends IntersectionType(
  CreateUserInput,
  AdditionalUserInfo,
) {}
```

> **Подсказка** Функция `IntersectionType()` импортируется из пакета `@nestjs/graphql`.

#### Композиция

Служебные функции преобразования типов можно комбинировать. Например, следующий код создаст тип (класс) со всеми свойствами типа `CreateUserInput`, кроме `email`, причём все эти свойства будут необязательными:

```typescript
@InputType()
export class UpdateUserInput extends PartialType(
  OmitType(CreateUserInput, ['email'] as const),
) {}
```

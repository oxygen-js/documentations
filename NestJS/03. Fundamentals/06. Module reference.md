### Ссылка на модуль

Nest предоставляет класс `ModuleRef` для навигации по внутреннему списку провайдеров и получения ссылки на любой провайдер по его токену внедрения, который служит ключом поиска. Кроме того, `ModuleRef` позволяет динамически создавать экземпляры как статических, так и scoped-провайдеров. `ModuleRef` внедряется в класс обычным способом:

**`cats.service.ts`**

```typescript
@Injectable()
export class CatsService {
  constructor(private moduleRef: ModuleRef) {}
}
```

> **Подсказка** Класс `ModuleRef` импортируется из пакета `@nestjs/core`.

#### Получение экземпляров

У экземпляра `ModuleRef` (далее будем называть его **ссылкой на модуль**) есть метод `get()`. По умолчанию этот метод возвращает провайдер, контроллер или injectable-объект (например, guard, interceptor и т. д.), который был зарегистрирован и создан в *текущем модуле*, по его токену внедрения или имени класса. Если экземпляр не найден, будет выброшено исключение.

**`cats.service.ts`**

```typescript
@Injectable()
export class CatsService implements OnModuleInit {
  private service: Service;
  constructor(private moduleRef: ModuleRef) {}

  onModuleInit() {
    this.service = this.moduleRef.get(Service);
  }
}
```

> **Предупреждение** Методом `get()` нельзя получить scoped-провайдеры (transient или request-scoped). Вместо этого используйте приём, описанный <a href="06.%20Module%20reference.md#разрешение-scoped-провайдеров">ниже</a>. О том, как управлять областями внедрения, читайте [здесь](<04. Injection scopes.md>).

Чтобы получить провайдер из глобального контекста (например, если провайдер был внедрён в другом модуле), передайте вторым аргументом `get()` параметр `{ strict: false }`.

```typescript
this.moduleRef.get(Service, { strict: false });
```

#### Разрешение scoped-провайдеров

Чтобы динамически разрешить scoped-провайдер (transient или request-scoped), используйте метод `resolve()`, передав ему токен внедрения провайдера.

**`cats.service.ts`**

```typescript
@Injectable()
export class CatsService implements OnModuleInit {
  private transientService: TransientService;
  constructor(private moduleRef: ModuleRef) {}

  async onModuleInit() {
    this.transientService = await this.moduleRef.resolve(TransientService);
  }
}
```

Метод `resolve()` возвращает уникальный экземпляр провайдера из отдельного **поддерева DI-контейнера**. У каждого поддерева свой уникальный **идентификатор контекста**. Поэтому, если вызвать этот метод несколько раз и сравнить ссылки на экземпляры, окажется, что они не равны.

**`cats.service.ts`**

```typescript
@Injectable()
export class CatsService implements OnModuleInit {
  constructor(private moduleRef: ModuleRef) {}

  async onModuleInit() {
    const transientServices = await Promise.all([
      this.moduleRef.resolve(TransientService),
      this.moduleRef.resolve(TransientService),
    ]);
    console.log(transientServices[0] === transientServices[1]); // false
  }
}
```

Чтобы получить один и тот же экземпляр при нескольких вызовах `resolve()` и обеспечить общее поддерево DI-контейнера, передайте методу `resolve()` идентификатор контекста. Для его создания служит класс `ContextIdFactory`: его метод `create()` возвращает подходящий уникальный идентификатор.

**`cats.service.ts`**

```typescript
@Injectable()
export class CatsService implements OnModuleInit {
  constructor(private moduleRef: ModuleRef) {}

  async onModuleInit() {
    const contextId = ContextIdFactory.create();
    const transientServices = await Promise.all([
      this.moduleRef.resolve(TransientService, contextId),
      this.moduleRef.resolve(TransientService, contextId),
    ]);
    console.log(transientServices[0] === transientServices[1]); // true
  }
}
```

> **Подсказка** Класс `ContextIdFactory` импортируется из пакета `@nestjs/core`.

#### Регистрация провайдера `REQUEST`

Идентификаторы контекста, созданные вручную (через `ContextIdFactory.create()`), соответствуют поддеревьям DI, в которых провайдер `REQUEST` равен `undefined`, поскольку они не создаются и не управляются системой внедрения зависимостей Nest.

Чтобы зарегистрировать собственный объект `REQUEST` для созданного вручную поддерева DI, воспользуйтесь методом `ModuleRef#registerRequestByContextId()`:

```typescript
const contextId = ContextIdFactory.create();
this.moduleRef.registerRequestByContextId(/* ВАШ_ОБЪЕКТ_ЗАПРОСА */, contextId);
```

#### Получение текущего поддерева

Иногда требуется разрешить экземпляр request-scoped провайдера внутри **контекста запроса**. Допустим, `CatsService` является request-scoped, и вы хотите разрешить экземпляр `CatsRepository`, который также помечен как request-scoped провайдер. Чтобы использовать одно и то же поддерево DI-контейнера, нужно получить текущий идентификатор контекста, а не создавать новый (например, функцией `ContextIdFactory.create()`, как показано выше). Чтобы получить текущий идентификатор контекста, сначала внедрите объект запроса с помощью декоратора `@Inject()`.

**`cats.service.ts`**

```typescript
@Injectable()
export class CatsService {
  constructor(
    @Inject(REQUEST) private request: Record<string, unknown>,
  ) {}
}
```

> **Подсказка** Подробнее о провайдере запроса читайте [здесь](<04. Injection scopes.md#провайдер-запроса>).

Теперь с помощью метода `getByRequest()` класса `ContextIdFactory` создайте идентификатор контекста на основе объекта запроса и передайте его в вызов `resolve()`:

```typescript
const contextId = ContextIdFactory.getByRequest(this.request);
const catsRepository = await this.moduleRef.resolve(CatsRepository, contextId);
```

#### Динамическое создание экземпляров пользовательских классов

Чтобы динамически создать экземпляр класса, который **не был ранее зарегистрирован** как **провайдер**, используйте метод `create()` ссылки на модуль.

**`cats.service.ts`**

```typescript
@Injectable()
export class CatsService implements OnModuleInit {
  private catsFactory: CatsFactory;
  constructor(private moduleRef: ModuleRef) {}

  async onModuleInit() {
    this.catsFactory = await this.moduleRef.create(CatsFactory);
  }
}
```

Этот приём позволяет условно создавать экземпляры разных классов вне контейнера фреймворка.

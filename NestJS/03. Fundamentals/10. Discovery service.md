### Discovery service

`DiscoveryService` из пакета `@nestjs/core` — мощный инструмент, позволяющий динамически исследовать и получать провайдеры, контроллеры и прочие метаданные внутри приложения NestJS. Это особенно полезно при создании плагинов, декораторов или продвинутых возможностей, опирающихся на интроспекцию во время выполнения. Опираясь на `DiscoveryService`, можно строить более гибкие и модульные архитектуры, добавляя в приложение автоматизацию и динамическое поведение.

#### Начало работы

Прежде чем использовать `DiscoveryService`, нужно импортировать `DiscoveryModule` в тот модуль, где вы планируете его применять. Это сделает сервис доступным для внедрения зависимостей. Ниже пример настройки внутри модуля NestJS:

```typescript
import { Module } from '@nestjs/common';
import { DiscoveryModule } from '@nestjs/core';
import { ExampleService } from './example.service';

@Module({
  imports: [DiscoveryModule],
  providers: [ExampleService],
})
export class ExampleModule {}
```

После настройки модуля `DiscoveryService` можно внедрить в любой провайдер или сервис, где требуется динамическое обнаружение.

**`example.service.ts`**

```typescript
@Injectable()
export class ExampleService {
  constructor(private readonly discoveryService: DiscoveryService) {}
}
```

#### Обнаружение провайдеров и контроллеров

Одна из ключевых возможностей `DiscoveryService` — получение всех зарегистрированных в приложении провайдеров. Это удобно, когда провайдеры нужно динамически обрабатывать по определённым условиям. Следующий фрагмент показывает, как получить доступ ко всем провайдерам:

```typescript
const providers = this.discoveryService.getProviders();
console.log(providers);
```

Каждый объект провайдера содержит такую информацию, как его экземпляр, токен и метаданные. Аналогично, если нужно получить все зарегистрированные в приложении контроллеры, это делается так:

```typescript
const controllers = this.discoveryService.getControllers();
console.log(controllers);
```

Эта возможность особенно полезна в сценариях, где контроллеры требуется обрабатывать динамически, — например, для сбора аналитики или механизмов автоматической регистрации.

#### Извлечение метаданных

Помимо обнаружения провайдеров и контроллеров, `DiscoveryService` позволяет получать метаданные, привязанные к этим компонентам. Это особенно ценно при работе с пользовательскими декораторами, которые сохраняют метаданные во время выполнения.

Рассмотрим, например, случай, когда пользовательский декоратор помечает провайдеры определёнными метаданными:

```typescript
import { DiscoveryService } from '@nestjs/core';

export const FeatureFlag = DiscoveryService.createDecorator();
```

Применение этого декоратора к сервису позволяет сохранить метаданные, которые впоследствии можно запросить:

```typescript
import { Injectable } from '@nestjs/common';
import { FeatureFlag } from './custom-metadata.decorator';

@Injectable()
@FeatureFlag('experimental')
export class CustomService {}
```

После того как метаданные привязаны к провайдерам таким образом, `DiscoveryService` позволяет легко отфильтровать провайдеры по заданным метаданным. Следующий фрагмент кода демонстрирует, как получить провайдеры, помеченные конкретным значением метаданных:

```typescript
const providers = this.discoveryService.getProviders();

const [provider] = providers.filter(
  (item) =>
    this.discoveryService.getMetadataByDecorator(FeatureFlag, item) ===
    'experimental',
);

console.log(
  'Providers with the "experimental" feature flag metadata:',
  provider,
);
```

#### Заключение

`DiscoveryService` — универсальный и мощный инструмент, обеспечивающий интроспекцию во время выполнения в приложениях NestJS. Позволяя динамически обнаруживать провайдеры, контроллеры и метаданные, он играет ключевую роль в построении расширяемых фреймворков, плагинов и возможностей, основанных на автоматизации. Нужно ли вам просканировать и обработать провайдеры, извлечь метаданные для сложной обработки или создать модульную и масштабируемую архитектуру — `DiscoveryService` даёт эффективный и структурированный способ добиться этих целей.

### Асинхронные провайдеры

Иногда запуск приложения нужно отложить до завершения одной или нескольких **асинхронных задач**. Например, вы можете не захотеть принимать запросы, пока не установлено соединение с базой данных. Добиться этого помогают асинхронные провайдеры.

Синтаксис здесь такой: `async/await` в сочетании с `useFactory`. Фабрика возвращает `Promise`, а внутри фабричной функции можно использовать `await` для асинхронных задач. Nest дождётся разрешения промиса, прежде чем создавать экземпляр любого класса, который зависит от такого провайдера (внедряет его).

```typescript
{
  provide: 'ASYNC_CONNECTION',
  useFactory: async () => {
    const connection = await createConnection(options);
    return connection;
  },
}
```

> **Подсказка** Подробнее о синтаксисе пользовательских провайдеров читайте [здесь](<01. Custom providers.md>).

#### Внедрение

Асинхронные провайдеры внедряются в другие компоненты по своим токенам, как и любые другие провайдеры. В примере выше вы использовали бы конструкцию `@Inject('ASYNC_CONNECTION')`.

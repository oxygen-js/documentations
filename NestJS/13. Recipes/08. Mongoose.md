### MongoDB (Mongoose)

> **Предупреждение** В этой статье вы узнаете, как с нуля создать `DatabaseModule` на основе пакета **Mongoose** с помощью пользовательских компонентов. Как следствие, это решение содержит много лишнего кода, которого можно избежать, взяв готовый и доступный «из коробки» специализированный пакет `@nestjs/mongoose`. Подробнее об этом — [здесь](<../04. Techniques/03. Mongo.md>).

[Mongoose](https://mongoosejs.com) — самый популярный инструмент объектного моделирования для [MongoDB](https://www.mongodb.org/).

#### Начало работы

Чтобы начать работу с этой библиотекой, нужно установить все требуемые зависимости:

```typescript
$ npm install --save mongoose
```

Первым делом необходимо установить соединение с базой данных с помощью функции `connect()`. Функция `connect()` возвращает `Promise`, поэтому нам понадобится [асинхронный провайдер](<../03. Fundamentals/02. Asynchronous providers.md>).

**`database.providers.ts`**

```typescript
import * as mongoose from 'mongoose';

export const databaseProviders = [
  {
    provide: 'DATABASE_CONNECTION',
    useFactory: (): Promise<typeof mongoose> =>
      mongoose.connect('mongodb://localhost/nest'),
  },
];
```

> **Подсказка** Следуя лучшим практикам, мы объявили пользовательский провайдер в отдельном файле с суффиксом `*.providers.ts`.

Затем эти провайдеры нужно экспортировать, чтобы они стали **доступны** остальной части приложения.

**`database.module.ts`**

```typescript
import { Module } from '@nestjs/common';
import { databaseProviders } from './database.providers';

@Module({
  providers: [...databaseProviders],
  exports: [...databaseProviders],
})
export class DatabaseModule {}
```

Теперь мы можем внедрять объект `Connection` с помощью декоратора `@Inject()`. Каждый класс, зависящий от асинхронного провайдера `Connection`, будет ждать разрешения `Promise`.

#### Внедрение модели

В Mongoose всё строится на [схеме](https://mongoosejs.com/docs/guide.html). Определим `CatSchema`:

**`schemas/cat.schema.ts`**

```typescript
import * as mongoose from 'mongoose';

export const CatSchema = new mongoose.Schema({
  name: String,
  age: Number,
  breed: String,
});
```

`CatSchema` находится в каталоге `cats`. Этот каталог соответствует модулю `CatsModule`.

Теперь пора создать провайдер **Model**:

**`cats.providers.ts`**

```typescript
import { Connection } from 'mongoose';
import { CatSchema } from './schemas/cat.schema';

export const catsProviders = [
  {
    provide: 'CAT_MODEL',
    useFactory: (connection: Connection) => connection.model('Cat', CatSchema),
    inject: ['DATABASE_CONNECTION'],
  },
];
```

> **Предупреждение** В реальных приложениях следует избегать **магических строк**. И `CAT_MODEL`, и `DATABASE_CONNECTION` стоит вынести в отдельный файл `constants.ts`.

Теперь мы можем внедрить `CAT_MODEL` в `CatsService` с помощью декоратора `@Inject()`:

**`cats.service.ts`**

```typescript
import { Model } from 'mongoose';
import { Injectable, Inject } from '@nestjs/common';
import { Cat } from './interfaces/cat.interface';
import { CreateCatDto } from './dto/create-cat.dto';

@Injectable()
export class CatsService {
  constructor(
    @Inject('CAT_MODEL')
    private catModel: Model<Cat>,
  ) {}

  async create(createCatDto: CreateCatDto): Promise<Cat> {
    const createdCat = new this.catModel(createCatDto);
    return createdCat.save();
  }

  async findAll(): Promise<Cat[]> {
    return this.catModel.find().exec();
  }
}
```

В примере выше мы использовали интерфейс `Cat`. Этот интерфейс расширяет `Document` из пакета mongoose:

```typescript
import { Document } from 'mongoose';

export interface Cat extends Document {
  readonly name: string;
  readonly age: number;
  readonly breed: string;
}
```

Соединение с базой данных **асинхронное**, но Nest делает этот процесс полностью незаметным для конечного пользователя. Класс `CatModel` ждёт соединения с БД, а `CatsService` откладывается до тех пор, пока модель не будет готова к использованию. Всё приложение может запуститься, когда каждый класс создан.

Вот итоговый `CatsModule`:

**`cats.module.ts`**

```typescript
import { Module } from '@nestjs/common';
import { CatsController } from './cats.controller';
import { CatsService } from './cats.service';
import { catsProviders } from './cats.providers';
import { DatabaseModule } from '../database/database.module';

@Module({
  imports: [DatabaseModule],
  controllers: [CatsController],
  providers: [
    CatsService,
    ...catsProviders,
  ],
})
export class CatsModule {}
```

> **Подсказка** Не забудьте импортировать `CatsModule` в корневой модуль `AppModule`.

### Раздача статики

Чтобы раздавать статический контент, например одностраничное приложение (SPA), можно использовать `ServeStaticModule` из пакета [`@nestjs/serve-static`](https://www.npmjs.com/package/@nestjs/serve-static).

#### Установка

Сначала нужно установить необходимый пакет:

```bash
$ npm install --save @nestjs/serve-static
```

#### Начальная загрузка

После установки можно импортировать `ServeStaticModule` в корневой `AppModule` и настроить его, передав объект конфигурации в метод `forRoot()`.

```typescript
import { Module } from '@nestjs/common';
import { AppController } from './app.controller';
import { AppService } from './app.service';
import { ServeStaticModule } from '@nestjs/serve-static';
import { join } from 'path';

@Module({
  imports: [
    ServeStaticModule.forRoot({
      rootPath: join(__dirname, '..', 'client'),
    }),
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

После этого соберите статический сайт и поместите его содержимое в каталог, указанный в свойстве `rootPath`.

#### Настройка

[ServeStaticModule](https://github.com/nestjs/serve-static) можно настроить множеством параметров, меняющих его поведение.
Вы можете задать путь, по которому отдаётся статическое приложение, указать исключаемые пути, включить или отключить установку заголовка ответа Cache-Control и т. д. Полный список параметров смотрите [здесь](https://github.com/nestjs/serve-static/blob/master/lib/interfaces/serve-static-options.interface.ts).

> **Важно** Значение `renderPath` для статического приложения по умолчанию — `*` (все пути), и модуль будет отдавать в ответ файлы «index.html».
> Это позволяет реализовать клиентскую маршрутизацию для вашего SPA. Пути, объявленные в контроллерах, будут обрабатываться сервером.
> Изменить это поведение можно, задав `serveRoot` и `renderPath` в сочетании с другими параметрами.
> Кроме того, в адаптере Fastify реализован параметр `serveStaticOptions.fallthrough`, повторяющий поведение fallthrough из Express: его нужно установить в `true`, чтобы для несуществующего маршрута отдавался `index.html`, а не ошибка 404.

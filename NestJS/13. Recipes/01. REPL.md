### Read-Eval-Print-Loop (REPL)

REPL — это простая интерактивная среда, которая принимает отдельные команды пользователя, выполняет их и возвращает результат.
Режим REPL позволяет исследовать граф зависимостей и вызывать методы ваших провайдеров (и контроллеров) прямо из терминала.

#### Использование

Чтобы запустить приложение NestJS в режиме REPL, создайте новый файл `repl.ts` (рядом с существующим `main.ts`) и добавьте в него следующий код:

**`repl.ts`**

```typescript
import { repl } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  await repl(AppModule);
}
bootstrap();
```

Теперь запустите REPL в терминале следующей командой:

```bash
$ npm run start -- --entryFile repl
```

> **Подсказка** `repl` возвращает объект [REPL-сервера Node.js](https://nodejs.org/api/repl.html).

После запуска в консоли должно появиться такое сообщение:

```bash
LOG [NestFactory] Starting Nest application...
LOG [InstanceLoader] AppModule dependencies initialized
LOG REPL initialized
```

Теперь можно начать работу с графом зависимостей. Например, можно получить `AppService` (здесь в качестве примера используется стартовый проект) и вызвать метод `getHello()`:

```typescript
> get(AppService).getHello()
'Hello World!'
```

Из терминала можно выполнять любой JavaScript-код: например, присвоить экземпляр `AppController` локальной переменной и вызвать асинхронный метод через `await`:

```typescript
> appController = get(AppController)
AppController { appService: AppService {} }
> await appController.getHello()
'Hello World!'
```

Чтобы вывести все публичные методы, доступные у конкретного провайдера или контроллера, используйте функцию `methods()`:

```typescript
> methods(AppController)

Methods:
 ◻ getHello
```

Чтобы вывести список всех зарегистрированных модулей вместе с их контроллерами и провайдерами, используйте `debug()`.

```typescript
> debug()

AppModule:
 - controllers:
  ◻ AppController
 - providers:
  ◻ AppService
```

#### Нативные функции

Встроенный REPL NestJS содержит несколько нативных функций, которые становятся глобально доступны при его запуске. Вызовите `help()`, чтобы получить их список.

Если вы не помните сигнатуру функции (то есть ожидаемые параметры и тип возвращаемого значения), вызовите `<function_name>.help`.
Например:

```text
> $.help
Retrieves an instance of either injectable or controller, otherwise, throws exception.
Interface: $(token: InjectionToken) => any
```

> **Подсказка** Интерфейсы этих функций записаны в [синтаксисе типов функций TypeScript](https://www.typescriptlang.org/docs/handbook/2/functions.html#function-type-expressions).

| Функция      | Описание                                                                                                           | Сигнатура                                                             |
| ------------ | ------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------- |
| `debug`      | Выводит список всех зарегистрированных модулей вместе с их контроллерами и провайдерами.                            | `debug(moduleCls?: ClassRef \| string) => void`                       |
| `get` или `$` | Получает экземпляр injectable-класса или контроллера, иначе выбрасывает исключение.                                | `get(token: InjectionToken) => any`                                   |
| `methods`    | Выводит все публичные методы, доступные у указанного провайдера или контроллера.                                    | `methods(token: ClassRef \| string) => void`                          |
| `resolve`    | Разрешает transient- или request-scoped-экземпляр injectable-класса или контроллера, иначе выбрасывает исключение. | `resolve(token: InjectionToken, contextId: any) => Promise<any>`      |
| `select`     | Позволяет перемещаться по дереву модулей, например, чтобы извлечь конкретный экземпляр из выбранного модуля.        | `select(token: DynamicModule \| ClassRef) => INestApplicationContext` |

#### Режим наблюдения

В процессе разработки удобно запускать REPL в режиме наблюдения, чтобы все изменения кода подхватывались автоматически:

```bash
$ npm run start -- --watch --entryFile repl
```

У этого подхода есть один недостаток: история команд REPL сбрасывается после каждой перезагрузки, что может раздражать.
К счастью, решение очень простое. Измените функцию `bootstrap` следующим образом:

```typescript
async function bootstrap() {
  const replServer = await repl(AppModule);
  replServer.setupHistory(".nestjs_repl_history", (err) => {
    if (err) {
      console.error(err);
    }
  });
}
```

Теперь история сохраняется между запусками и перезагрузками.

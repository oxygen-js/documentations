### SQL (Sequelize)

##### Эта глава применима только к TypeScript

> **Предупреждение** В этой статье вы узнаете, как с нуля создать `DatabaseModule` на основе пакета **Sequelize**, используя пользовательские компоненты. Как следствие, этот подход влечёт немало накладных расходов, которых можно избежать, воспользовавшись готовым пакетом `@nestjs/sequelize`. Подробнее — [здесь](<../04. Techniques/02. Database.md#интеграция-с-sequelize>).

[Sequelize](https://github.com/sequelize/sequelize) — популярный объектно-реляционный маппер (ORM), написанный на чистом JavaScript, но существует и TypeScript-обёртка [sequelize-typescript](https://github.com/RobinBuschmann/sequelize-typescript), которая добавляет к базовому sequelize набор декораторов и другие возможности.

#### Начало работы

Чтобы начать работу с этой библиотекой, нужно установить следующие зависимости:

```bash
$ npm install --save sequelize sequelize-typescript mysql2
$ npm install --save-dev @types/sequelize
```

Первым делом нужно создать экземпляр **Sequelize**, передав в конструктор объект параметров. Также необходимо добавить все модели (альтернатива — использовать свойство `modelPaths`) и синхронизировать таблицы базы данных вызовом `sync()`.

**`database.providers.ts`**

```typescript
import { Sequelize } from 'sequelize-typescript';
import { Cat } from '../cats/cat.entity';

export const databaseProviders = [
  {
    provide: 'SEQUELIZE',
    useFactory: async () => {
      const sequelize = new Sequelize({
        dialect: 'mysql',
        host: 'localhost',
        port: 3306,
        username: 'root',
        password: 'password',
        database: 'nest',
      });
      sequelize.addModels([Cat]);
      await sequelize.sync();
      return sequelize;
    },
  },
];
```

> **Подсказка** Следуя лучшим практикам, мы объявили пользовательский провайдер в отдельном файле с суффиксом `*.providers.ts`.

Далее нужно экспортировать эти провайдеры, чтобы сделать их **доступными** остальной части приложения.

```typescript
import { Module } from '@nestjs/common';
import { databaseProviders } from './database.providers';

@Module({
  providers: [...databaseProviders],
  exports: [...databaseProviders],
})
export class DatabaseModule {}
```

Теперь мы можем внедрить объект `Sequelize` с помощью декоратора `@Inject()`. Каждый класс, зависящий от асинхронного провайдера `Sequelize`, будет ждать, пока разрешится `Promise`.

#### Внедрение моделей

В [Sequelize](https://github.com/sequelize/sequelize) **модель** описывает таблицу в базе данных. Экземпляры этого класса представляют строки таблицы. Прежде всего нам нужна хотя бы одна сущность:

**`cat.entity.ts`**

```typescript
import { Table, Column, Model } from 'sequelize-typescript';

@Table
export class Cat extends Model {
  @Column
  name: string;

  @Column
  age: number;

  @Column
  breed: string;
}
```

Сущность `Cat` относится к каталогу `cats`. Этот каталог представляет `CatsModule`. Теперь пора создать провайдер **репозитория**:

**`cats.providers.ts`**

```typescript
import { Cat } from './cat.entity';

export const catsProviders = [
  {
    provide: 'CATS_REPOSITORY',
    useValue: Cat,
  },
];
```

> **Предупреждение** В реальных приложениях следует избегать **магических строк**. И `CATS_REPOSITORY`, и `SEQUELIZE` нужно хранить в отдельном файле `constants.ts`.

В Sequelize для работы с данными используются статические методы, поэтому здесь мы создали **псевдоним**.

Теперь можно внедрить `CATS_REPOSITORY` в `CatsService` с помощью декоратора `@Inject()`:

**`cats.service.ts`**

```typescript
import { Injectable, Inject } from '@nestjs/common';
import { CreateCatDto } from './dto/create-cat.dto';
import { Cat } from './cat.entity';

@Injectable()
export class CatsService {
  constructor(
    @Inject('CATS_REPOSITORY')
    private catsRepository: typeof Cat
  ) {}

  async findAll(): Promise<Cat[]> {
    return this.catsRepository.findAll<Cat>();
  }
}
```

Подключение к базе данных **асинхронное**, но Nest делает этот процесс полностью незаметным для конечного пользователя. Провайдер `CATS_REPOSITORY` ждёт подключения к БД, а `CatsService` откладывается до момента, когда репозиторий будет готов к использованию. Всё приложение может стартовать, когда создан экземпляр каждого класса.

Вот итоговый `CatsModule`:

**`cats.module.ts`**

```typescript
import { Module } from '@nestjs/common';
import { CatsController } from './cats.controller';
import { CatsService } from './cats.service';
import { catsProviders } from './cats.providers';
import { DatabaseModule } from '../database/database.module';

@Module({
  imports: [DatabaseModule],
  controllers: [CatsController],
  providers: [
    CatsService,
    ...catsProviders,
  ],
})
export class CatsModule {}
```

> **Подсказка** Не забудьте импортировать `CatsModule` в корневой `AppModule`.

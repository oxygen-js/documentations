### Генератор CRUD (только TypeScript)

На протяжении жизни проекта, создавая новую функциональность, мы часто добавляем в приложение новые ресурсы. Обычно такие ресурсы требуют множества однотипных операций, которые приходится повторять при каждом объявлении нового ресурса.

#### Введение

Представим реальную ситуацию: нам нужно предоставить CRUD-эндпоинты для двух сущностей, скажем, **User** и **Product**.
Следуя лучшим практикам, для каждой сущности пришлось бы выполнить несколько операций:

- Сгенерировать модуль (`nest g mo`), чтобы организовать код и обозначить чёткие границы (сгруппировать связанные компоненты)
- Сгенерировать контроллер (`nest g co`), чтобы описать CRUD-маршруты (или запросы/мутации для GraphQL-приложений)
- Сгенерировать сервис (`nest g s`), чтобы реализовать и изолировать бизнес-логику
- Сгенерировать класс или интерфейс сущности, описывающий структуру данных ресурса
- Сгенерировать Data Transfer Objects (или input-типы для GraphQL-приложений), чтобы определить, как данные будут передаваться по сети

Многовато шагов!

Чтобы ускорить этот однообразный процесс, [Nest CLI](<../11. CLI/01. Overview.md>) предоставляет генератор (схематик), который автоматически создаёт весь шаблонный код и заметно упрощает работу разработчика.

> **Примечание** Схематик умеет генерировать **HTTP**-контроллеры, контроллеры **микросервисов**, **GraphQL**-резолверы (как code first, так и schema first) и **WebSocket**-шлюзы.

#### Генерация нового ресурса

Чтобы создать новый ресурс, просто выполните в корневом каталоге проекта следующую команду:

```shell
$ nest g resource
```

Команда `nest g resource` генерирует не только все строительные блоки NestJS (классы модуля, сервиса, контроллера), но и класс сущности, классы DTO, а также файлы тестов (`.spec`).

Ниже показан сгенерированный файл контроллера (для REST API):

```typescript
@Controller('users')
export class UsersController {
  constructor(private readonly usersService: UsersService) {}

  @Post()
  create(@Body() createUserDto: CreateUserDto) {
    return this.usersService.create(createUserDto);
  }

  @Get()
  findAll() {
    return this.usersService.findAll();
  }

  @Get(':id')
  findOne(@Param('id') id: string) {
    return this.usersService.findOne(+id);
  }

  @Patch(':id')
  update(@Param('id') id: string, @Body() updateUserDto: UpdateUserDto) {
    return this.usersService.update(+id, updateUserDto);
  }

  @Delete(':id')
  remove(@Param('id') id: string) {
    return this.usersService.remove(+id);
  }
}
```

Кроме того, автоматически создаются заготовки для всех CRUD-эндпоинтов (маршруты для REST API, запросы и мутации для GraphQL, подписки на сообщения для микросервисов и WebSocket-шлюзов) — и всё это без единого лишнего действия с вашей стороны.

> **Примечание** Сгенерированные классы сервисов **не** привязаны к какой-либо конкретной **ORM (или источнику данных)**. Благодаря этому генератор достаточно универсален для нужд любого проекта. По умолчанию все методы содержат заглушки, которые вы наполняете работой с источниками данных, специфичными для вашего проекта.

Аналогично, если вы хотите сгенерировать резолверы для GraphQL-приложения, просто выберите в качестве транспортного слоя `GraphQL (code first)` (или `GraphQL (schema first)`).

В этом случае NestJS сгенерирует класс резолвера вместо контроллера REST API:


Ниже видно, что созданы не только шаблонные мутации и запросы, но и все связи между ними. Здесь используются `UsersService`, сущность `User` и наши DTO.

```typescript
import { Resolver, Query, Mutation, Args, Int } from '@nestjs/graphql';
import { UsersService } from './users.service';
import { User } from './entities/user.entity';
import { CreateUserInput } from './dto/create-user.input';
import { UpdateUserInput } from './dto/update-user.input';

@Resolver(() => User)
export class UsersResolver {
  constructor(private readonly usersService: UsersService) {}

  @Mutation(() => User)
  createUser(@Args('createUserInput') createUserInput: CreateUserInput) {
    return this.usersService.create(createUserInput);
  }

  @Query(() => [User], { name: 'users' })
  findAll() {
    return this.usersService.findAll();
  }

  @Query(() => User, { name: 'user' })
  findOne(@Args('id', { type: () => Int }) id: number) {
    return this.usersService.findOne(id);
  }

  @Mutation(() => User)
  updateUser(@Args('updateUserInput') updateUserInput: UpdateUserInput) {
    return this.usersService.update(updateUserInput.id, updateUserInput);
  }

  @Mutation(() => User)
  removeUser(@Args('id', { type: () => Int }) id: number) {
    return this.usersService.remove(id);
  }
}
```

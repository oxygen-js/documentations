### Горячая перезагрузка

Больше всего на процесс начальной загрузки вашего приложения влияет **компиляция TypeScript**. К счастью, благодаря HMR (Hot-Module Replacement) в [webpack](https://github.com/webpack/webpack) не нужно перекомпилировать весь проект при каждом изменении. Это существенно сокращает время создания экземпляра приложения и заметно упрощает итеративную разработку.

> **Предупреждение** Учтите, что `webpack` не копирует ваши ресурсы (например, файлы `graphql`) в папку `dist` автоматически. Кроме того, `webpack` несовместим со статическими путями с glob-шаблонами (например, со свойством `entities` в `TypeOrmModule`).

### С использованием CLI

Если вы используете [Nest CLI](<../11. CLI/01. Overview.md>), настройка довольно проста. CLI является обёрткой над `webpack`, что позволяет применять `HotModuleReplacementPlugin`.

#### Установка

Сначала установите нужные пакеты:

```bash
$ npm i --save-dev webpack-node-externals run-script-webpack-plugin webpack
```

> **Подсказка** Если вы используете **Yarn Berry** (не классический Yarn), установите пакет `webpack-pnp-externals` вместо `webpack-node-externals`.

#### Настройка

После установки создайте файл `webpack-hmr.config.js` в корневом каталоге приложения.

```typescript
const nodeExternals = require('webpack-node-externals');
const { RunScriptWebpackPlugin } = require('run-script-webpack-plugin');

module.exports = function (options, webpack) {
  return {
    ...options,
    entry: ['webpack/hot/poll?100', options.entry],
    externals: [
      nodeExternals({
        allowlist: ['webpack/hot/poll?100'],
      }),
    ],
    plugins: [
      ...options.plugins,
      new webpack.HotModuleReplacementPlugin(),
      new webpack.WatchIgnorePlugin({
        paths: [/\.js$/, /\.d\.ts$/],
      }),
      new RunScriptWebpackPlugin({ name: options.output.filename, autoRestart: false }),
    ],
  };
};
```

> **Подсказка** В случае **Yarn Berry** (не классического Yarn) вместо `nodeExternals` в свойстве конфигурации `externals` используйте `WebpackPnpExternals` из пакета `webpack-pnp-externals`: `WebpackPnpExternals({ exclude: ['webpack/hot/poll?100'] })`.

Эта функция принимает первым аргументом исходный объект с конфигурацией webpack по умолчанию, а вторым — ссылку на пакет `webpack`, используемый Nest CLI. Возвращает она изменённую конфигурацию webpack с плагинами `HotModuleReplacementPlugin`, `WatchIgnorePlugin` и `RunScriptWebpackPlugin`.

#### Hot-Module Replacement

Чтобы включить **HMR**, откройте входной файл приложения (`main.ts`) и добавьте следующие инструкции, относящиеся к webpack:

```typescript
declare const module: any;

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(process.env.PORT ?? 3000);

  if (module.hot) {
    module.hot.accept();
    module.hot.dispose(() => app.close());
  }
}
bootstrap();
```

Чтобы упростить запуск, добавьте скрипт в файл `package.json`.

```json
"start:dev": "nest build --webpack --webpackPath webpack-hmr.config.js --watch"
```

Теперь просто откройте командную строку и выполните команду:

```bash
$ npm run start:dev
```

### Без CLI

Если вы не используете [Nest CLI](<../11. CLI/01. Overview.md>), настройка будет несколько сложнее (потребуется больше ручных действий).

#### Установка

Сначала установите нужные пакеты:

```bash
$ npm i --save-dev webpack webpack-cli webpack-node-externals ts-loader run-script-webpack-plugin
```

> **Подсказка** Если вы используете **Yarn Berry** (не классический Yarn), установите пакет `webpack-pnp-externals` вместо `webpack-node-externals`.

#### Настройка

После установки создайте файл `webpack.config.js` в корневом каталоге приложения.

```typescript
const webpack = require('webpack');
const path = require('path');
const nodeExternals = require('webpack-node-externals');
const { RunScriptWebpackPlugin } = require('run-script-webpack-plugin');

module.exports = {
  entry: ['webpack/hot/poll?100', './src/main.ts'],
  target: 'node',
  externals: [
    nodeExternals({
      allowlist: ['webpack/hot/poll?100'],
    }),
  ],
  module: {
    rules: [
      {
        test: /.tsx?$/,
        use: 'ts-loader',
        exclude: /node_modules/,
      },
    ],
  },
  mode: 'development',
  resolve: {
    extensions: ['.tsx', '.ts', '.js'],
  },
  plugins: [new webpack.HotModuleReplacementPlugin(), new RunScriptWebpackPlugin({ name: 'server.js', autoRestart: false })],
  output: {
    path: path.join(__dirname, 'dist'),
    filename: 'server.js',
  },
};
```

> **Подсказка** В случае **Yarn Berry** (не классического Yarn) вместо `nodeExternals` в свойстве конфигурации `externals` используйте `WebpackPnpExternals` из пакета `webpack-pnp-externals`: `WebpackPnpExternals({ exclude: ['webpack/hot/poll?100'] })`.

Эта конфигурация сообщает webpack несколько важных вещей о вашем приложении: расположение входного файла, каталог для **скомпилированных** файлов и какой загрузчик применять для компиляции исходников. Как правило, вы можете использовать этот файл как есть, даже если не до конца понимаете все параметры.

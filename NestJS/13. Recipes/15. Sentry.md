### Sentry

[Sentry](https://sentry.io) — платформа для отслеживания ошибок и мониторинга производительности, помогающая разработчикам находить и устранять проблемы в реальном времени. В этом рецепте показано, как интегрировать [NestJS SDK](https://docs.sentry.io/platforms/javascript/guides/nestjs/) от Sentry в ваше приложение на NestJS.

#### Установка

Сначала установите необходимые зависимости:

```bash
$ npm install --save @sentry/nestjs @sentry/profiling-node
```

> **Подсказка** Пакет `@sentry/profiling-node` необязателен, но рекомендуется для профилирования производительности.

#### Базовая настройка

Чтобы начать работу с Sentry, создайте файл `instrument.ts`, который должен импортироваться раньше всех остальных модулей вашего приложения:

**`instrument.ts`**

```typescript
const Sentry = require("@sentry/nestjs");
const { nodeProfilingIntegration } = require("@sentry/profiling-node");

// Обязательно вызовите это до подключения любых других модулей!
Sentry.init({
  dsn: SENTRY_DSN,
  integrations: [
    // Добавляем интеграцию профилирования
    nodeProfilingIntegration(),
  ],

  // Включаем трассировку, задав tracesSampleRate
  // В production это значение рекомендуется скорректировать
  tracesSampleRate: 1.0,

  // Задаём долю выборки для профилирования
  // Значение задаётся относительно tracesSampleRate
  profilesSampleRate: 1.0,
});
```

Измените файл `main.ts` так, чтобы `instrument.ts` импортировался раньше остальных импортов:

**`main.ts`**

```typescript
// Импортируйте это первым!
import "./instrument";

// Теперь импортируем остальные модули
import { NestFactory } from "@nestjs/core";
import { AppModule } from "./app.module";

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(3000);
}

bootstrap();
```

Затем добавьте `SentryModule` в качестве корневого модуля в ваш главный модуль:

**`app.module.ts`**

```typescript
import { Module } from "@nestjs/common";
import { SentryModule } from "@sentry/nestjs/setup";
import { AppController } from "./app.controller";
import { AppService } from "./app.service";

@Module({
  imports: [
    SentryModule.forRoot(),
    // ...другие модули
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

#### Обработка исключений

Если вы используете глобальный универсальный фильтр исключений (то есть фильтр, зарегистрированный через `app.useGlobalFilters()`, либо фильтр, зарегистрированный в провайдерах модуля приложения и помеченный декоратором `@Catch()` без аргументов), добавьте декоратор `@SentryExceptionCaptured()` к методу `catch()` этого фильтра. Декоратор будет отправлять в Sentry все непредвиденные ошибки, попадающие в ваш глобальный фильтр:

```typescript
import { Catch, ExceptionFilter } from '@nestjs/common';
import { SentryExceptionCaptured } from '@sentry/nestjs';

@Catch()
export class YourCatchAllExceptionFilter implements ExceptionFilter {
  @SentryExceptionCaptured()
  catch(exception, host): void {
    // здесь ваша реализация
  }
}
```

По умолчанию в Sentry попадают только необработанные исключения, не перехваченные фильтром ошибок. Исключения `HttpExceptions` (включая [производные от них](<../02. Overview/06. Exception filters.md#встроенные-http-исключения>)) по умолчанию тоже не фиксируются, поскольку в основном служат средством управления потоком выполнения.

Если глобального универсального фильтра исключений у вас нет, добавьте `SentryGlobalFilter` в провайдеры главного модуля. Этот фильтр отправит в Sentry все необработанные ошибки, не перехваченные другими фильтрами.

> **Предупреждение** `SentryGlobalFilter` должен быть зарегистрирован раньше любых других фильтров исключений.

**`app.module.ts`**

```typescript
import { Module } from "@nestjs/common";
import { APP_FILTER } from "@nestjs/core";
import { SentryGlobalFilter } from "@sentry/nestjs/setup";

@Module({
  providers: [
    {
      provide: APP_FILTER,
      useClass: SentryGlobalFilter,
    },
    // ..другие провайдеры
  ],
})
export class AppModule {}
```

#### Читаемые стек-трейсы

В зависимости от того, как настроен ваш проект, стек-трейсы в ошибках Sentry, скорее всего, будут мало похожи на ваш реальный код.

Чтобы это исправить, загрузите в Sentry ваши source map'ы. Проще всего сделать это с помощью Sentry Wizard:

```bash
npx @sentry/wizard@latest -i sourcemaps
```

#### Проверка интеграции

Чтобы убедиться, что интеграция с Sentry работает, добавьте тестовый эндпоинт, выбрасывающий ошибку:

```typescript
@Get("debug-sentry")
getError() {
  throw new Error("My first Sentry error!");
}
```

Откройте `/debug-sentry` в своём приложении — ошибка должна появиться в панели управления Sentry.

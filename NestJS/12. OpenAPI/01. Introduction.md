### Введение

Спецификация [OpenAPI](https://swagger.io/specification/) — это независимый от языка формат описания RESTful API. Nest предоставляет специальный [модуль](https://github.com/nestjs/swagger), который позволяет генерировать такую спецификацию на основе декораторов.

#### Установка

Чтобы начать работу, сначала установим необходимую зависимость.

```bash
$ npm install --save @nestjs/swagger
```

#### Начальная загрузка

После установки откройте файл `main.ts` и инициализируйте Swagger с помощью класса `SwaggerModule`:

**`main.ts`**

```typescript
import { NestFactory } from '@nestjs/core';
import { SwaggerModule, DocumentBuilder } from '@nestjs/swagger';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  const config = new DocumentBuilder()
    .setTitle('Cats example')
    .setDescription('The cats API description')
    .setVersion('1.0')
    .addTag('cats')
    .build();
  const documentFactory = () => SwaggerModule.createDocument(app, config);
  SwaggerModule.setup('api', app, documentFactory);

  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
```

> **Подсказка** Фабричный метод `SwaggerModule.createDocument()` предназначен именно для того, чтобы сгенерировать документ Swagger в момент запроса. Такой подход помогает сэкономить время инициализации, а полученный документ представляет собой сериализуемый объект, соответствующий спецификации [OpenAPI Document](https://swagger.io/specification/#openapi-document). Вместо того чтобы отдавать документ по HTTP, вы можете сохранить его в файл JSON или YAML и использовать по-разному.

`DocumentBuilder` помогает выстроить базовый документ, соответствующий спецификации OpenAPI. Он предоставляет несколько методов, позволяющих задать такие свойства, как заголовок, описание, версия и т. д. Чтобы создать полный документ (со всеми объявленными HTTP-маршрутами), мы используем метод `createDocument()` класса `SwaggerModule`. Этот метод принимает два аргумента: экземпляр приложения и объект параметров Swagger. Дополнительно можно передать третий аргумент типа `SwaggerDocumentOptions`. Подробнее об этом — в разделе [Параметры документа](<01. Introduction.md#параметры-документа>).

После создания документа можно вызвать метод `setup()`. Он принимает:

1. Путь, по которому будет доступен Swagger UI
2. Экземпляр приложения
3. Созданный выше объект документа
4. Необязательный параметр конфигурации (подробнее [здесь](<01. Introduction.md#параметры-настройки>))

Теперь можно запустить HTTP-сервер следующей командой:

```bash
$ npm run start
```

Пока приложение работает, откройте браузер и перейдите по адресу `http://localhost:3000/api`. Вы увидите Swagger UI.

<figure><img src="https://docs.nestjs.com/assets/swagger1.png" /></figure>

Как видите, `SwaggerModule` автоматически отражает все ваши эндпоинты.

> **Подсказка** Чтобы сгенерировать и скачать файл Swagger JSON, перейдите по адресу `http://localhost:3000/api-json` (при условии, что ваша документация Swagger доступна по адресу `http://localhost:3000/api`).
> Его также можно опубликовать по любому выбранному вами маршруту, используя только метод setup из `@nestjs/swagger`, например так:
>
> ```typescript
> SwaggerModule.setup('swagger', app, documentFactory, {
>   jsonDocumentUrl: 'swagger/json',
> });
> ```
>
> В этом случае он будет доступен по адресу `http://localhost:3000/swagger/json`

> **Предупреждение** При использовании `fastify` и `helmet` возможна проблема с [CSP](https://developer.mozilla.org/en-US/docs/Web/HTTP/CSP). Чтобы устранить этот конфликт, настройте CSP так:
>
> ```typescript
> app.register(helmet, {
>   contentSecurityPolicy: {
>     directives: {
>       defaultSrc: [`'self'`],
>       styleSrc: [`'self'`, `'unsafe-inline'`],
>       imgSrc: [`'self'`, 'data:', 'validator.swagger.io'],
>       scriptSrc: [`'self'`, `https:`, `'unsafe-inline'`],
>     },
>   },
> });
>
> // Если вы вообще не собираетесь использовать CSP, можно поступить так:
> app.register(helmet, {
>   contentSecurityPolicy: false,
> });
> ```

#### Параметры документа

При создании документа можно передать дополнительные параметры для тонкой настройки поведения библиотеки. Эти параметры должны иметь тип `SwaggerDocumentOptions`, который выглядит так:

```TypeScript
export interface SwaggerDocumentOptions {
  /**
   * Список модулей, которые нужно включить в спецификацию
   */
  include?: Function[];

  /**
   * Дополнительные модели, которые нужно проанализировать и включить в спецификацию
   */
  extraModels?: Function[];

  /**
   * Если `true`, swagger будет игнорировать глобальный префикс, заданный методом `setGlobalPrefix()`
   */
  ignoreGlobalPrefix?: boolean;

  /**
   * Если `true`, swagger также загрузит маршруты из модулей, импортированных модулями из `include`
   */
  deepScanRoutes?: boolean;

  /**
   * Собственная функция operationIdFactory, которая будет использоваться для генерации `operationId`
   * на основе `controllerKey`, `methodKey` и версии.
   * @default () => controllerKey_methodKey_version
   */
  operationIdFactory?: OperationIdFactory;

  /**
   * Собственная функция linkNameFactory, которая будет использоваться для генерации имён ссылок
   * в поле `links` ответов
   *
   * @see [Link objects](https://swagger.io/docs/specification/links/)
   *
   * @default () => `${controllerKey}_${methodKey}_from_${fieldKey}`
   */
  linkNameFactory?: (
    controllerKey: string,
    methodKey: string,
    fieldKey: string
  ) => string;

  /*
   * Автоматически генерировать теги на основе имени контроллера.
   * Если `false`, теги нужно задавать декоратором `@ApiTags()`.
   * Иначе будет использовано имя контроллера без суффикса `Controller`.
   * @default true
   */
  autoTagControllers?: boolean;
}
```

Например, если вы хотите, чтобы библиотека генерировала имена операций вида `createUser`, а не `UsersController_createUser`, задайте следующее:

```TypeScript
const options: SwaggerDocumentOptions =  {
  operationIdFactory: (
    controllerKey: string,
    methodKey: string
  ) => methodKey
};
const documentFactory = () => SwaggerModule.createDocument(app, config, options);
```

#### Параметры настройки

Swagger UI можно настроить, передав четвёртым аргументом метода `SwaggerModule#setup` объект параметров, соответствующий интерфейсу `SwaggerCustomOptions`.

```TypeScript
export interface SwaggerCustomOptions {
  /**
   * Если `true`, пути к ресурсам Swagger будут снабжены глобальным префиксом, заданным через `setGlobalPrefix()`.
   * По умолчанию: `false`.
   * @see https://docs.nestjs.com/faq/global-prefix
   */
  useGlobalPrefix?: boolean;

  /**
   * Если `false`, Swagger UI не будет отдаваться. Доступны будут только определения API (JSON и YAML)
   * (по путям `/{path}-json` и `/{path}-yaml`). Чтобы полностью отключить и Swagger UI, и определения API, используйте `raw: false`.
   * По умолчанию: `true`.
   * @deprecated Используйте `ui`.
   */
  swaggerUiEnabled?: boolean;

  /**
   * Если `false`, Swagger UI не будет отдаваться. Доступны будут только определения API (JSON и YAML)
   * (по путям `/{path}-json` и `/{path}-yaml`). Чтобы полностью отключить и Swagger UI, и определения API, используйте `raw: false`.
   * По умолчанию: `true`.
   */
  ui?: boolean;

  /**
   * Если `true`, будут отдаваться исходные определения во всех форматах.
   * Также можно передать массив, чтобы указать нужные форматы, например `raw: ['json']` — отдавать только определения JSON.
   * Если параметр опущен или задан пустым массивом, определения (JSON и YAML) отдаваться не будут.
   * Используйте этот параметр, чтобы управлять доступностью связанных со Swagger эндпоинтов.
   * По умолчанию: `true`.
   */
  raw?: boolean | Array<'json' | 'yaml'>;

  /**
   * URL определения API, которое нужно загрузить в Swagger UI.
   */
  swaggerUrl?: string;

  /**
   * Путь, по которому отдаётся определение API в формате JSON.
   * По умолчанию: `<path>-json`.
   */
  jsonDocumentUrl?: string;

  /**
   * Путь, по которому отдаётся определение API в формате YAML.
   * По умолчанию: `<path>-yaml`.
   */
  yamlDocumentUrl?: string;

  /**
   * Хук, позволяющий изменить документ OpenAPI перед его отдачей.
   * Вызывается после генерации документа и до его отдачи в формате JSON и YAML.
   */
  patchDocumentOnRequest?: <TRequest = any, TResponse = any>(
    req: TRequest,
    res: TResponse,
    document: OpenAPIObject
  ) => OpenAPIObject;

  /**
   * Если `true`, в интерфейсе Swagger UI отображается селектор определений OpenAPI.
   * По умолчанию: `false`.
   */
  explorer?: boolean;

  /**
   * Дополнительные параметры Swagger UI
   */
  swaggerOptions?: SwaggerUiOptions;

  /**
   * Собственные CSS-стили, внедряемые на страницу Swagger UI.
   */
  customCss?: string;

  /**
   * URL (или несколько) собственной таблицы стилей CSS для загрузки на странице Swagger UI.
   */
  customCssUrl?: string | string[];

  /**
   * URL (или несколько) собственных файлов JavaScript для загрузки на странице Swagger UI.
   */
  customJs?: string | string[];

  /**
   * Собственные скрипты JavaScript для загрузки на странице Swagger UI.
   */
  customJsStr?: string | string[];

  /**
   * Собственный favicon для страницы Swagger UI.
   */
  customfavIcon?: string;

  /**
   * Собственный заголовок страницы Swagger UI.
   */
  customSiteTitle?: string;

  /**
   * Путь в файловой системе (например, ./node_modules/swagger-ui-dist) со статическими ресурсами Swagger UI.
   */
  customSwaggerUiPath?: string;

  /**
   * @deprecated Это свойство ни на что не влияет.
   */
  validatorUrl?: string;

  /**
   * @deprecated Это свойство ни на что не влияет.
   */
  url?: string;

  /**
   * @deprecated Это свойство ни на что не влияет.
   */
  urls?: Record<'url' | 'name', string>[];
}
```

> **Подсказка** `ui` и `raw` — независимые параметры. Отключение Swagger UI (`ui: false`) не отключает определения API (JSON/YAML). И наоборот, отключение определений API (`raw: []`) не отключает Swagger UI.
>
> Например, следующая конфигурация отключит Swagger UI, но оставит доступ к определениям API:
>
> ```typescript
> const options: SwaggerCustomOptions = {
>   ui: false, // Swagger UI отключён
>   raw: ['json'], // определение API в формате JSON по-прежнему доступно (YAML отключён)
> };
> SwaggerModule.setup('api', app, options);
> ```
>
> В этом случае http://localhost:3000/api-json останется доступным, а http://localhost:3000/api (Swagger UI) — нет.

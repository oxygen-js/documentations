### Mapped types

По мере разработки таких возможностей, как **CRUD** (Create/Read/Update/Delete), часто оказывается полезно строить варианты базового типа сущности. Nest предоставляет несколько вспомогательных функций, которые выполняют преобразования типов и упрощают эту задачу.

#### Partial

При создании типов для валидации входных данных (их также называют DTO) часто удобно иметь варианты **create** и **update** одного и того же типа. Например, вариант **create** может требовать все поля, тогда как в варианте **update** все поля могут быть необязательными.

Nest предоставляет вспомогательную функцию `PartialType()`, которая упрощает эту задачу и сводит к минимуму шаблонный код.

Функция `PartialType()` возвращает тип (класс), в котором все свойства входного типа сделаны необязательными. Например, пусть у нас есть такой тип **create**:

```typescript
import { ApiProperty } from '@nestjs/swagger';

export class CreateCatDto {
  @ApiProperty()
  name: string;

  @ApiProperty()
  age: number;

  @ApiProperty()
  breed: string;
}
```

По умолчанию все эти поля обязательны. Чтобы создать тип с теми же полями, но с каждым полем необязательным, используйте `PartialType()`, передав ссылку на класс (`CreateCatDto`) в качестве аргумента:

```typescript
export class UpdateCatDto extends PartialType(CreateCatDto) {}
```

> **Подсказка** Функция `PartialType()` импортируется из пакета `@nestjs/swagger`.

#### Pick

Функция `PickType()` конструирует новый тип (класс), выбирая набор свойств из входного типа. Например, пусть мы начинаем с такого типа:

```typescript
import { ApiProperty } from '@nestjs/swagger';

export class CreateCatDto {
  @ApiProperty()
  name: string;

  @ApiProperty()
  age: number;

  @ApiProperty()
  breed: string;
}
```

Выбрать набор свойств из этого класса можно с помощью вспомогательной функции `PickType()`:

```typescript
export class UpdateCatAgeDto extends PickType(CreateCatDto, ['age'] as const) {}
```

> **Подсказка** Функция `PickType()` импортируется из пакета `@nestjs/swagger`.

#### Omit

Функция `OmitType()` конструирует тип, беря все свойства входного типа и удаляя из них заданный набор ключей. Например, пусть мы начинаем с такого типа:

```typescript
import { ApiProperty } from '@nestjs/swagger';

export class CreateCatDto {
  @ApiProperty()
  name: string;

  @ApiProperty()
  age: number;

  @ApiProperty()
  breed: string;
}
```

Мы можем получить производный тип, содержащий все свойства **кроме** `name`, как показано ниже. В этой конструкции вторым аргументом `OmitType` передаётся массив имён свойств.

```typescript
export class UpdateCatDto extends OmitType(CreateCatDto, ['name'] as const) {}
```

> **Подсказка** Функция `OmitType()` импортируется из пакета `@nestjs/swagger`.

#### Intersection

Функция `IntersectionType()` объединяет два типа в один новый тип (класс). Например, пусть мы начинаем с двух таких типов:

```typescript
import { ApiProperty } from '@nestjs/swagger';

export class CreateCatDto {
  @ApiProperty()
  name: string;

  @ApiProperty()
  breed: string;
}

export class AdditionalCatInfo {
  @ApiProperty()
  color: string;
}
```

Мы можем получить новый тип, объединяющий все свойства обоих типов.

```typescript
export class UpdateCatDto extends IntersectionType(
  CreateCatDto,
  AdditionalCatInfo,
) {}
```

> **Подсказка** Функция `IntersectionType()` импортируется из пакета `@nestjs/swagger`.

#### Композиция

Функции преобразования типов можно комбинировать. Например, следующий код порождает тип (класс), содержащий все свойства типа `CreateCatDto`, кроме `name`, причём все эти свойства будут необязательными:

```typescript
export class UpdateCatDto extends PartialType(
  OmitType(CreateCatDto, ['name'] as const),
) {}
```

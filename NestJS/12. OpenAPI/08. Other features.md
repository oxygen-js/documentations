### Прочие возможности

На этой странице перечислены остальные доступные возможности, которые могут вам пригодиться.

#### Глобальный префикс

Чтобы игнорировать глобальный префикс маршрутов, заданный через `setGlobalPrefix()`, используйте `ignoreGlobalPrefix`:

```typescript
const document = SwaggerModule.createDocument(app, options, {
  ignoreGlobalPrefix: true,
});
```

#### Глобальные параметры

Вы можете задать параметры для всех маршрутов с помощью `DocumentBuilder`, как показано ниже:

```typescript
const config = new DocumentBuilder()
  .addGlobalParameters({
    name: 'tenantId',
    in: 'header',
  })
  // прочие настройки
  .build();
```

#### Глобальные ответы

С помощью `DocumentBuilder` вы можете задать глобальные ответы для всех маршрутов. Это удобно, когда нужно единообразно описать ответы для всех эндпоинтов приложения — например, коды ошибок вроде `401 Unauthorized` или `500 Internal Server Error`.

```typescript
const config = new DocumentBuilder()
  .addGlobalResponse({
    status: 500,
    description: 'Internal server error',
  })
  // прочие настройки
  .build();
```

#### Несколько спецификаций

`SwaggerModule` позволяет работать с несколькими спецификациями. Иными словами, вы можете отдавать разную документацию с разными интерфейсами на разных эндпоинтах.

Чтобы это было возможно, приложение должно быть написано модульно. Метод `createDocument()` принимает третий аргумент `extraOptions` — объект со свойством `include`. В свойство `include` передаётся массив модулей.

Настроить поддержку нескольких спецификаций можно так:

```typescript
import { NestFactory } from '@nestjs/core';
import { SwaggerModule, DocumentBuilder } from '@nestjs/swagger';
import { AppModule } from './app.module';
import { CatsModule } from './cats/cats.module';
import { DogsModule } from './dogs/dogs.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  /**
   * createDocument(application, configurationOptions, extraOptions);
   *
   * Метод createDocument принимает необязательный третий аргумент "extraOptions" —
   * объект со свойством "include", в котором можно передать массив модулей,
   * включаемых в данную спецификацию Swagger.
   * Например: CatsModule и DogsModule получат две отдельные спецификации Swagger,
   * которые будут доступны в двух разных SwaggerUI на двух разных эндпоинтах.
   */

  const options = new DocumentBuilder()
    .setTitle('Cats example')
    .setDescription('The cats API description')
    .setVersion('1.0')
    .addTag('cats')
    .build();

  const catDocumentFactory = () =>
    SwaggerModule.createDocument(app, options, {
      include: [CatsModule],
    });
  SwaggerModule.setup('api/cats', app, catDocumentFactory);

  const secondOptions = new DocumentBuilder()
    .setTitle('Dogs example')
    .setDescription('The dogs API description')
    .setVersion('1.0')
    .addTag('dogs')
    .build();

  const dogDocumentFactory = () =>
    SwaggerModule.createDocument(app, secondOptions, {
      include: [DogsModule],
    });
  SwaggerModule.setup('api/dogs', app, dogDocumentFactory);

  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
```

Теперь можно запустить сервер такой командой:

```bash
$ npm run start
```

Перейдите по адресу `http://localhost:3000/api/cats`, чтобы увидеть Swagger UI для cats:

#### Выпадающий список в панели обозревателя

Чтобы включить поддержку нескольких спецификаций в выпадающем меню панели обозревателя, нужно задать `explorer: true` и настроить `swaggerOptions.urls` в `SwaggerCustomOptions`.

> **Подсказка** Убедитесь, что `swaggerOptions.urls` указывает на документы Swagger в формате JSON! Чтобы задать JSON-документ, используйте `jsonDocumentUrl` в `SwaggerCustomOptions`. Остальные настройки описаны [здесь](<01. Introduction.md#параметры-настройки>).

Вот как настроить несколько спецификаций в выпадающем списке панели обозревателя:

```typescript
import { NestFactory } from '@nestjs/core';
import { SwaggerModule, DocumentBuilder } from '@nestjs/swagger';
import { AppModule } from './app.module';
import { CatsModule } from './cats/cats.module';
import { DogsModule } from './dogs/dogs.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  // Опции основного API
  const options = new DocumentBuilder()
    .setTitle('Multiple Specifications Example')
    .setDescription('Description for multiple specifications')
    .setVersion('1.0')
    .build();

  // Создаём документ основного API
  const document = SwaggerModule.createDocument(app, options);

  // Настраиваем Swagger UI основного API с поддержкой выпадающего списка
  SwaggerModule.setup('api', app, document, {
    explorer: true,
    swaggerOptions: {
      urls: [
        {
          name: '1. API',
          url: 'api/swagger.json',
        },
        {
          name: '2. Cats API',
          url: 'api/cats/swagger.json',
        },
        {
          name: '3. Dogs API',
          url: 'api/dogs/swagger.json',
        },
      ],
    },
    jsonDocumentUrl: '/api/swagger.json',
  });

  // Опции Cats API
  const catOptions = new DocumentBuilder()
    .setTitle('Cats Example')
    .setDescription('Description for the Cats API')
    .setVersion('1.0')
    .addTag('cats')
    .build();

  // Создаём документ Cats API
  const catDocument = SwaggerModule.createDocument(app, catOptions, {
    include: [CatsModule],
  });

  // Настраиваем Swagger UI для Cats API
  SwaggerModule.setup('api/cats', app, catDocument, {
    jsonDocumentUrl: '/api/cats/swagger.json',
  });

  // Опции Dogs API
  const dogOptions = new DocumentBuilder()
    .setTitle('Dogs Example')
    .setDescription('Description for the Dogs API')
    .setVersion('1.0')
    .addTag('dogs')
    .build();

  // Создаём документ Dogs API
  const dogDocument = SwaggerModule.createDocument(app, dogOptions, {
    include: [DogsModule],
  });

  // Настраиваем Swagger UI для Dogs API
  SwaggerModule.setup('api/dogs', app, dogDocument, {
    jsonDocumentUrl: '/api/dogs/swagger.json',
  });

  await app.listen(3000);
}

bootstrap();
```

В этом примере мы настроили основной API вместе с отдельными спецификациями для Cats и Dogs, каждая из которых доступна из выпадающего списка в панели обозревателя.

### Операции

В терминах OpenAPI пути (paths) — это эндпоинты (ресурсы), которые предоставляет ваш API, например `/users` или `/reports/summary`, а операции (operations) — это HTTP-методы, используемые для работы с этими путями, такие как `GET`, `POST` или `DELETE`.

#### Теги

Чтобы привязать контроллер к определённому тегу, используйте декоратор `@ApiTags(...tags)`.

```typescript
@ApiTags('cats')
@Controller('cats')
export class CatsController {}
```

OpenAPI 3.2 расширяет объект Tag: теги можно выстраивать в иерархию и снабжать подсказкой о том, как их следует отображать. Чтобы объявить такие связи, определите теги заранее через `DocumentBuilder` и передайте в `addTag()` опции `parent` и `kind`:

```typescript
const config = new DocumentBuilder()
  .setOpenAPIVersion('3.2.0')
  .addTag('Animals', 'Everything about animals', undefined, { kind: 'nav' })
  .addTag('Cats', 'Cat operations', undefined, { parent: 'Animals' })
  .addTag('Dogs', 'Dog operations', undefined, { parent: 'Animals' })
  .build();
```

Опция `parent` ссылается на другой тег по имени, а `kind` — это произвольная машиночитаемая строка, подсказывающая, как использовать тег; чаще всего это `nav`, `badge` или `audience`.

> **Предупреждение** Поля `parent` и `kind` относятся к объекту Tag из OpenAPI 3.2. Вы обязаны вызвать `setOpenAPIVersion('3.2.0')`, иначе в сгенерированном документе останется `openapi: 3.0.0` и строгие валидаторы отклонят эти поля. Поля иерархии можно задать только через `DocumentBuilder.addTag()`; установка их в декораторе `@ApiTags()` ни на что не влияет.

#### Заголовки

Чтобы описать пользовательские заголовки, ожидаемые в составе запроса, используйте `@ApiHeader()`.

```typescript
@ApiHeader({
  name: 'X-MyHeader',
  description: 'Custom header',
})
@Controller('cats')
export class CatsController {}
```

#### Ответы

Чтобы описать пользовательский HTTP-ответ, используйте декоратор `@ApiResponse()`.

```typescript
@Post()
@ApiResponse({ status: 201, description: 'The record has been successfully created.'})
@ApiResponse({ status: 403, description: 'Forbidden.'})
async create(@Body() createCatDto: CreateCatDto) {
  this.catsService.create(createCatDto);
}
```

Nest предоставляет набор сокращённых декораторов **ответов API**, унаследованных от декоратора `@ApiResponse`:

- `@ApiOkResponse()`
- `@ApiCreatedResponse()`
- `@ApiAcceptedResponse()`
- `@ApiNoContentResponse()`
- `@ApiMovedPermanentlyResponse()`
- `@ApiFoundResponse()`
- `@ApiBadRequestResponse()`
- `@ApiUnauthorizedResponse()`
- `@ApiNotFoundResponse()`
- `@ApiForbiddenResponse()`
- `@ApiMethodNotAllowedResponse()`
- `@ApiNotAcceptableResponse()`
- `@ApiRequestTimeoutResponse()`
- `@ApiConflictResponse()`
- `@ApiPreconditionFailedResponse()`
- `@ApiTooManyRequestsResponse()`
- `@ApiGoneResponse()`
- `@ApiPayloadTooLargeResponse()`
- `@ApiUnsupportedMediaTypeResponse()`
- `@ApiUnprocessableEntityResponse()`
- `@ApiInternalServerErrorResponse()`
- `@ApiNotImplementedResponse()`
- `@ApiBadGatewayResponse()`
- `@ApiServiceUnavailableResponse()`
- `@ApiGatewayTimeoutResponse()`
- `@ApiDefaultResponse()`

```typescript
@Post()
@ApiCreatedResponse({ description: 'The record has been successfully created.'})
@ApiForbiddenResponse({ description: 'Forbidden.'})
async create(@Body() createCatDto: CreateCatDto) {
  this.catsService.create(createCatDto);
}
```

Чтобы указать модель возвращаемого значения для запроса, нужно создать класс и пометить все его свойства декоратором `@ApiProperty()`.

```typescript
export class Cat {
  @ApiProperty()
  id: number;

  @ApiProperty()
  name: string;

  @ApiProperty()
  age: number;

  @ApiProperty()
  breed: string;
}
```

После этого модель `Cat` можно использовать вместе со свойством `type` декоратора ответа.

```typescript
@ApiTags('cats')
@Controller('cats')
export class CatsController {
  @Post()
  @ApiCreatedResponse({
    description: 'The record has been successfully created.',
    type: Cat,
  })
  async create(@Body() createCatDto: CreateCatDto): Promise<Cat> {
    return this.catsService.create(createCatDto);
  }
}
```

<figure><img src="https://docs.nestjs.com/assets/swagger-response-type.png" /></figure>

Вместо того чтобы описывать ответы для каждого эндпоинта или контроллера по отдельности, вы можете задать глобальный ответ для всех эндпоинтов с помощью класса `DocumentBuilder`. Такой подход удобен, когда нужно определить общий ответ для всех эндпоинтов приложения (например, для ошибок вроде `401 Unauthorized` или `500 Internal Server Error`).

```typescript
const config = new DocumentBuilder()
  .addGlobalResponse({
    status: 500,
    description: 'Internal server error',
  })
  // другие настройки
  .build();
```

#### Загрузка файлов

Вы можете включить загрузку файлов для конкретного метода с помощью декоратора `@ApiBody` вместе с `@ApiConsumes()`. Вот полный пример с использованием техники [загрузки файлов](<../04. Techniques/14. File upload.md>):

```typescript
@UseInterceptors(FileInterceptor('file'))
@ApiConsumes('multipart/form-data')
@ApiBody({
  description: 'List of cats',
  type: FileUploadDto,
})
uploadFile(@UploadedFile() file: Express.Multer.File) {}
```

Где `FileUploadDto` определяется так:

```typescript
class FileUploadDto {
  @ApiProperty({ type: 'string', format: 'binary' })
  file: any;
}
```

Чтобы обрабатывать загрузку нескольких файлов, вы можете определить `FilesUploadDto` следующим образом:

```typescript
class FilesUploadDto {
  @ApiProperty({ type: 'array', items: { type: 'string', format: 'binary' } })
  files: any[];
}
```

#### Расширения

Чтобы добавить к запросу расширение, используйте декоратор `@ApiExtension()`. Имя расширения должно начинаться с префикса `x-`.

```typescript
@ApiExtension('x-foo', { hello: 'world' })
```

#### Продвинутое: обобщённый `ApiResponse`

Благодаря возможности задавать [сырые определения](<02. Types and Parameters.md#сырые-определения>) мы можем описать обобщённую схему для Swagger UI. Предположим, у нас есть следующий DTO:

```ts
export class PaginatedDto<TData> {
  @ApiProperty()
  total: number;

  @ApiProperty()
  limit: number;

  @ApiProperty()
  offset: number;

  results: TData[];
}
```

Мы не помечаем `results` декоратором, поскольку позже зададим для него сырое определение. Теперь определим ещё один DTO и назовём его, например, `CatDto`:

```ts
export class CatDto {
  @ApiProperty()
  name: string;

  @ApiProperty()
  age: number;

  @ApiProperty()
  breed: string;
}
```

Имея всё это, мы можем описать ответ `PaginatedDto<CatDto>` следующим образом:

```ts
@ApiOkResponse({
  schema: {
    allOf: [
      { $ref: getSchemaPath(PaginatedDto) },
      {
        properties: {
          results: {
            type: 'array',
            items: { $ref: getSchemaPath(CatDto) },
          },
        },
      },
    ],
  },
})
async findAll(): Promise<PaginatedDto<CatDto>> {}
```

В этом примере мы указываем, что ответ будет содержать allOf `PaginatedDto`, а свойство `results` будет иметь тип `Array<CatDto>`.

- `getSchemaPath()` — функция, возвращающая путь к схеме OpenAPI внутри файла спецификации OpenAPI для заданной модели.
- `allOf` — концепция OAS 3, покрывающая различные сценарии, связанные с наследованием.

Наконец, поскольку ни один контроллер не ссылается на `PaginatedDto` напрямую, `SwaggerModule` пока не сможет сгенерировать соответствующее определение модели. В этом случае нужно добавить её как [дополнительную модель](<02. Types and Parameters.md#дополнительные-модели>). Например, мы можем применить декоратор `@ApiExtraModels()` на уровне контроллера:

```ts
@Controller('cats')
@ApiExtraModels(PaginatedDto)
export class CatsController {}
```

Если запустить Swagger сейчас, в сгенерированном `swagger.json` для этого эндпоинта будет описан такой ответ:

```json
"responses": {
  "200": {
    "description": "",
    "content": {
      "application/json": {
        "schema": {
          "allOf": [
            {
              "$ref": "#/components/schemas/PaginatedDto"
            },
            {
              "properties": {
                "results": {
                  "$ref": "#/components/schemas/CatDto"
                }
              }
            }
          ]
        }
      }
    }
  }
}
```

Чтобы это можно было переиспользовать, создадим для `PaginatedDto` собственный декоратор:

```ts
export const ApiPaginatedResponse = <TModel extends Type<any>>(
  model: TModel,
) => {
  return applyDecorators(
    ApiExtraModels(PaginatedDto, model),
    ApiOkResponse({
      schema: {
        allOf: [
          { $ref: getSchemaPath(PaginatedDto) },
          {
            properties: {
              results: {
                type: 'array',
                items: { $ref: getSchemaPath(model) },
              },
            },
          },
        ],
      },
    }),
  );
};
```

> **Подсказка** Интерфейс `Type<any>` и функция `applyDecorators` импортируются из пакета `@nestjs/common`.

Чтобы `SwaggerModule` сгенерировал определение для нашей модели, её нужно добавить как дополнительную модель — так же, как мы ранее поступили с `PaginatedDto` в контроллере.

После этого мы можем применить собственный декоратор `@ApiPaginatedResponse()` к нашему эндпоинту:

```ts
@ApiPaginatedResponse(CatDto)
async findAll(): Promise<PaginatedDto<CatDto>> {}
```

Для инструментов генерации клиентского кода такой подход создаёт неоднозначность в том, как для клиента будет сгенерирован `PaginatedResponse<TModel>`. Ниже приведён пример результата работы генератора клиента для приведённого выше эндпоинта `GET /`.

```typescript
// Angular
findAll(): Observable<{ total: number, limit: number, offset: number, results: CatDto[] }>
```

Как видите, **тип возвращаемого значения** здесь неоднозначен. Чтобы обойти эту проблему, вы можете добавить в `schema` для `ApiPaginatedResponse` свойство `title`:

```typescript
export const ApiPaginatedResponse = <TModel extends Type<any>>(
  model: TModel,
) => {
  return applyDecorators(
    ApiOkResponse({
      schema: {
        title: `PaginatedResponseOf${model.name}`,
        allOf: [
          // ...
        ],
      },
    }),
  );
};
```

Теперь результат работы генератора клиента будет таким:

```ts
// Angular
findAll(): Observable<PaginatedResponseOfCatDto>
```

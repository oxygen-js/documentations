# Задание: последовательный запуск анимаций

В этом задании вам предстоит доработать страницу так, чтобы она проигрывала серию анимаций в определённой последовательности. Для этого вы воспользуетесь некоторыми приёмами, которые мы изучили в статье [«Как использовать промисы»](<02. How to use promises.md>).

| | |
|---|---|
| **Предварительные требования:** | Разумное понимание основ JavaScript и того, как использовать API на основе промисов. |
| **Цель:** | Проверить понимание того, как использовать API на основе промисов. |

Единственный файл, который вам нужно будет редактировать, — это «main.js».

## Описание проекта

Мы хотим доработать эту страницу так, чтобы применить анимацию ко всем трём изображениям — одно за другим. То есть, когда первое закончится, мы анимируем второе, а когда закончится второе — анимируем третье.

Анимация уже определена в «main.js»: она просто вращает изображение и уменьшает его, пока оно не исчезнет.

Чтобы вам было понятнее, как мы хотим, чтобы работала страница, [взгляните на готовый пример](https://mdn.github.io/learning-area/javascript/asynchronous/sequencing-animations/finished/). Обратите внимание, что анимации проигрываются только один раз: чтобы увидеть их снова, перезагрузите страницу.

## Шаги для выполнения

### Анимация первого изображения

Мы используем [Web Animations API](https://developer.mozilla.org/en-US/docs/Web/API/Web_Animations_API) для анимации изображений — в частности, метод [`element.animate()`](https://developer.mozilla.org/en-US/docs/Web/API/Element/animate).

Обновите «main.js», добавив вызов `alice1.animate()`, вот так:

```js
const aliceTumbling = [
  { transform: "rotate(0) scale(1)" },
  { transform: "rotate(360deg) scale(0)" },
];

const aliceTiming = {
  duration: 2000,
  iterations: 1,
  fill: "forwards",
};

const alice1 = document.querySelector("#alice1");
const alice2 = document.querySelector("#alice2");
const alice3 = document.querySelector("#alice3");

alice1.animate(aliceTumbling, aliceTiming);
```

Перезагрузите страницу — и вы должны увидеть, как первое изображение вращается и уменьшается.

### Анимация всех изображений

Далее мы хотим анимировать `alice2`, когда закончится `alice1`, и `alice3`, когда закончится `alice2`.

Метод `animate()` возвращает объект [`Animation`](https://developer.mozilla.org/en-US/docs/Web/API/Animation). У этого объекта есть свойство `finished`, которое представляет собой `Promise`, выполняемый, когда анимация закончила проигрываться. Так что мы можем использовать этот промис, чтобы понять, когда запускать следующую анимацию.

Мы хотели бы, чтобы вы попробовали несколько разных способов это реализовать, чтобы закрепить разные способы использования промисов.

1. Сначала реализуйте что-то, что работает, но обладает промис-версией проблемы «ада колбэков», которую мы видели в нашем [обсуждении использования колбэков](<01. Introducing asynchronous JavaScript.md>) (см. раздел «Колбэки»).

2. Затем реализуйте это в виде [цепочки промисов](<02. How to use promises.md>) (см. раздел «Цепочки промисов»). Обратите внимание, что написать это можно несколькими разными способами из-за разных форм, которые можно использовать для [стрелочной функции](https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/Scripting/Functions#arrow_functions). Попробуйте разные формы. Какая из них самая лаконичная? Какую вы находите наиболее читаемой?

3. Наконец, реализуйте это с помощью [`async` и `await`](<02. How to use promises.md>) (см. раздел «async и await»).

Помните, что `element.animate()` _не_ возвращает `Promise`: он возвращает объект `Animation`, у которого есть свойство `finished`, являющееся `Promise`.

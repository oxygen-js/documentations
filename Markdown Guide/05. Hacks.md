# Приёмы

## Обзор

Большинству тех, кто пользуется Markdown, вполне хватает элементов [базового](<03. Basic Syntax.md>) и [расширенного синтаксиса](<04. Extended Syntax.md>). Но если вы работаете с Markdown достаточно долго, то рано или поздно наверняка обнаружите, что он не поддерживает что-то нужное вам. На этой странице собраны советы и приёмы, которые помогут обойти ограничения Markdown.

<div class="alert alert-success">
  <i class="fas fa-lightbulb"></i> <strong>Совет:</strong> Эти приёмы не гарантированно работают в вашем приложении Markdown. Если такие приёмы нужны вам часто, стоит подумать о том, чтобы писать не в Markdown, а в чём-то другом. 
</div>

## Подчёркивание

Подчёркнутый текст нечасто встречается в вебе — вероятно, потому что подчёркивание практически синонимично ссылкам. Однако, если вы пишете статью или отчёт, вам может понадобиться возможность подчёркивать слова и фразы. Некоторые приложения, например [Bear](https://www.markdownguide.org/tools/bear/) и [Simplenote](https://www.markdownguide.org/tools/simplenote/), поддерживают подчёркивание текста, но сам Markdown его изначально не поддерживает. Если ваш обработчик Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), для подчёркивания текста в документе можно использовать HTML-тег `<ins>`.

```html
Some of these words <ins>will be underlined</ins>.
```

Результат выглядит так:

Some of these words <ins>will be underlined</ins>.

## Отступ (табуляция)

Табуляция и пробельные символы имеют в Markdown особое значение. С помощью пробелов в конце строки можно создавать [переносы строк](<03. Basic Syntax.md#переносы-строк>), а с помощью табуляции — [блоки кода](<03. Basic Syntax.md#блоки-кода>). Но что делать, если нужно сделать отступ у абзаца по старинке, клавишей Tab? Markdown не предлагает для этого простого способа. 

Лучший вариант — воспользоваться редактором Markdown, который поддерживает отступы. Это часто встречается в приложениях, ориентированных больше на вёрстку и издательскую подготовку. Например, [iA Writer](https://www.markdownguide.org/tools/ia-writer/) позволяет настроить параметры отступов для редактора в настройках приложения. Кроме того, он предоставляет возможности настройки шаблонов, так что вы можете добиться, чтобы итоговый документ выглядел именно так, как вы ожидаете, — вместе со всеми отступами.

Ещё один вариант, если ваш обработчик Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), — использовать HTML-сущность неразрывного пробела (`&nbsp;`). Скорее всего, это должен быть ваш крайний вариант, поскольку он получается неуклюжим. По сути, каждый `&nbsp;` в исходном тексте Markdown будет заменён пробелом в итоговом результате. Так что если поставить перед абзацем четыре `&nbsp;`, абзац будет выглядеть так, будто он отступлен на четыре пробела.

```html
&nbsp;&nbsp;&nbsp;&nbsp;This is the first sentence of my indented paragraph.
```

Результат выглядит так:

&nbsp;&nbsp;&nbsp;&nbsp;This is the first sentence of my indented paragraph.

## Выравнивание по центру

Возможность выравнивать текст по центру необходима при написании статьи или отчёта. К сожалению, в Markdown нет самого понятия выравнивания текста (единственное возможное исключение — использование [таблиц](<04. Extended Syntax.md#выравнивание>)). Хорошая новость в том, что есть HTML-тег, который можно применить: `<center>`. Если ваш обработчик Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), вы можете обернуть этими тегами любой текст, который хотите выровнять по центру.

```html
<center>This text is centered.</center>
```

Результат выглядит так:

<p style="text-align:center">This text is centered.</p>

HTML-тег `<center>` технически поддерживается, но официально <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/center">объявлен устаревшим</a>, то есть пока он работает, но использовать его не рекомендуется. К сожалению, другой альтернативы на чистом HTML нет. Можно попробовать один из вариантов на CSS. Не все приложения Markdown поддерживают CSS, но если то, которым вы пользуетесь, поддерживает, вот альтернатива тегу `<center>`: 

```html
<p style="text-align:center">Center this text</p>
```

Если ваше приложение Markdown это поддерживает, результат выглядит так:

<p style="text-align:center">Center this text</p>

## Цвет

Markdown не позволяет менять цвет текста, но если ваш обработчик Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), можно использовать HTML-тег `<font>`. Атрибут `color` позволяет задать цвет шрифта с помощью названия цвета или шестнадцатеричного кода `#RRGGBB`.

```html
<font color="red">This text is red!</font>
```

Результат выглядит так:

<p style="color:red">This text is red!</p>

HTML-тег `<font>` технически поддерживается, но официально <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/font">объявлен устаревшим</a>, то есть пока он работает, но использовать его не рекомендуется. К сожалению, другой альтернативы на чистом HTML нет. Можно попробовать один из вариантов на CSS. Не все приложения Markdown поддерживают CSS, но если то, которым вы пользуетесь, поддерживает, вот альтернатива тегу `<font>`: 

```html
<p style="color:blue">Make this text blue.</p>
```

Если ваше приложение Markdown это поддерживает, результат выглядит так:

<p style="color:blue">Make this text blue.</p>

## Комментарии

Некоторым нужна возможность писать в файлах Markdown предложения, которые *не* появятся в итоговом результате. Такие комментарии — это по сути скрытый текст. Автор документа его видит, но на веб-странице или в PDF он не печатается. Markdown изначально не поддерживает комментарии, но несколько предприимчивых людей придумали решение. 

Чтобы добавить комментарий, поместите текст в квадратные скобки, а после них поставьте двоеточие, пробел и знак решётки (например, `[comment]: #`). До и после комментария следует оставить пустые строки.

```text
Here's a paragraph that will be visible.

[This is a comment that will be hidden.]: # 

And here's another paragraph that's visible.
```

Результат выглядит так:

Here's a paragraph that will be visible.

[This is a comment that will be hidden.]: # 

And here's another paragraph that's visible.

## Врезки

Врезки часто используются в документации, чтобы привлечь внимание к предупреждениям, примечаниям и советам. Markdown не предоставляет специального синтаксиса для врезок, и большинство приложений Markdown их не поддерживают (одно из исключений — [MkDocs](https://www.markdownguide.org/tools/mkdocs/)). 

Тем не менее, если вам нужны врезки, вы можете попробовать использовать [цитаты](<03. Basic Syntax.md#цитаты-1>) вместе с [эмодзи](<04. Extended Syntax.md#эмодзи>) и [выделением](<03. Basic Syntax.md#выделение>), чтобы создать нечто похожее на врезки, которые вы видите на других сайтах.

```text
> :warning: **Warning:** Do not push the big red button.

> :memo: **Note:** Sunrises are beautiful.

> :bulb: **Tip:** Remember to appreciate the little things in life.
```

Результат выглядит так:

> ⚠️ **Warning:** Do not push the big red button.

> 📝 **Note:** Sunrises are beautiful.

> 💡 **Tip:** Remember to appreciate the little things in life.

## Размер изображения

Синтаксис Markdown для [изображений](<03. Basic Syntax.md#изображения-1>) не позволяет задавать ширину и высоту изображений. Если вам нужно изменить размер изображения, а ваш обработчик Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), можно использовать HTML-тег `img` с атрибутами `width` и `height`, чтобы задать размеры изображения в пикселях.

```html
<img src="image.png" width="200" height="100">
```

В итоговом результате изображение будет масштабировано до указанных вами размеров. 

## Подписи к изображениям

Markdown изначально не поддерживает подписи к изображениям, но есть два возможных обходных пути. Если ваше приложение Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), для добавления подписи к изображению можно использовать HTML-теги `figure` и `figcaption`.

```html
<figure>
    <img src="https://www.markdownguide.org/assets/images/albuquerque.jpg"
         alt="Albuquerque, New Mexico">
    <figcaption>A single track trail outside of Albuquerque, New Mexico.</figcaption>
</figure>
```

Результат выглядит так:

<figure>
    ![Albuquerque, New Mexico](https://www.markdownguide.org/assets/images/albuquerque.jpg)
    <figcaption>A single track trail outside of Albuquerque, New Mexico.</figcaption>
</figure>

<div class="alert alert-success">
  <i class="fas fa-lightbulb"></i> <strong>Совет:</strong> Если ваше приложение Markdown поддерживает CSS, вы можете стилизовать внешний вид подписи с помощью CSS.
</div>

Если ваше приложение Markdown не поддерживает HTML, можно попробовать поместить подпись прямо под изображением и использовать [выделение](<03. Basic Syntax.md#выделение>).

```text
![Albuquerque, New Mexico](/assets/images/albuquerque.jpg)
*A single track trail outside of Albuquerque, New Mexico.*
```

Результат выглядит так:

![Albuquerque, New Mexico](https://www.markdownguide.org/assets/images/albuquerque.jpg)
*A single track trail outside of Albuquerque, New Mexico.*

## Цели ссылок

Некоторым нравится создавать ссылки, которые открываются в новых вкладках или окнах. Синтаксис Markdown для [ссылок](<03. Basic Syntax.md#ссылки>) не позволяет указать атрибут `target`, но если ваш обработчик Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), для создания таких ссылок можно использовать HTML.

```html
<a href="https://www.markdownguide.org" target="_blank">Learn Markdown!</a>
```

Результат выглядит так:

<a href="https://www.markdownguide.org" target="_blank">Learn Markdown!</a>

## Символы

Markdown не предоставляет специального синтаксиса для символов. Однако в большинстве случаев нужный символ можно просто скопировать и вставить в документ Markdown. Например, если вам нужно отобразить число пи (π), найдите этот символ на любой веб-странице, скопируйте и вставьте его в документ. В итоговом результате символ должен отобразиться как ожидается.

Как вариант, если ваше приложение Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), можно использовать HTML-сущность нужного символа. Например, если вы хотите отобразить знак авторского права (&copy;), скопируйте и вставьте в документ Markdown HTML-сущность для этого знака (`&copy;`).

Вот неполный список HTML-сущностей для символов:

- Знак авторского права (&copy;) — `&copy;`
- Зарегистрированный товарный знак (&reg;) — `&reg;`
- Товарный знак (&trade;) — `&trade;`
- Евро (&euro;) — `&euro;`
- Стрелка влево (&larr;) — `&larr;`
- Стрелка вверх (&uarr;) — `&uarr;`
- Стрелка вправо (&rarr;) — `&rarr;`
- Стрелка вниз (&darr;) — `&darr;`
- Градус (&#176;) — `&#176;`
- Пи (&#960;) — `&#960;`

Полный список доступных HTML-сущностей можно найти на странице Википедии, посвящённой [HTML-сущностям](https://en.wikipedia.org/wiki/List_of_XML_and_HTML_character_entity_references). 

## Форматирование таблиц

[Таблицы](<04. Extended Syntax.md#таблицы>) в Markdown известны своей капризностью. Многие элементы синтаксиса Markdown нельзя использовать для форматирования текста в ячейках таблиц. Но по крайней мере для двух распространённых проблем с таблицами есть обходные пути: переносы строк и списки.

### Переносы строк внутри ячеек таблицы

Разделять абзацы внутри ячейки таблицы можно с помощью одного или нескольких HTML-тегов `<br>`. 

```
| Syntax      | Description |
| ----------- | ----------- |
| Header      | Title |
| Paragraph   | First paragraph. <br><br> Second paragraph. |
```

Результат выглядит так:

<table class="table table-bordered">
  <thead>
    <tr>
      <th>Syntax</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Header</td>
      <td>Title</td>
    </tr>
    <tr>
      <td>Paragraph</td>
      <td>First paragraph. <br><br> Second paragraph.</td>
    </tr>
  </tbody>
</table>

### Списки внутри ячеек таблицы

Добавить список внутри ячейки таблицы можно с помощью HTML-тегов. 

```
| Syntax      | Description |
| ----------- | ----------- |
| Header      | Title |
| List        | Here's a list! <ul><li>Item one.</li><li>Item two.</li></ul> |
```

Результат выглядит так:

<table class="table table-bordered">
  <thead>
    <tr>
      <th>Syntax</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Header</td>
      <td>Title</td>
    </tr>
    <tr>
      <td>List</td>
      <td>Here's a list! <ul><li>Item one.</li><li>Item two.</li></ul></td>
    </tr>
  </tbody>
</table>

## Оглавление

Некоторые приложения Markdown, например [Markdeep](https://www.markdownguide.org/tools/markdeep/), могут автоматически создавать оглавление (его также называют *toc*, от table of contents) на основе ваших [заголовков](<03. Basic Syntax.md#заголовки>), но эта возможность есть не во всех приложениях Markdown. Тем не менее, если ваше приложение Markdown поддерживает [идентификаторы заголовков](<04. Extended Syntax.md#идентификаторы-заголовков>), вы можете создать оглавление для файла Markdown с помощью [списка](<03. Basic Syntax.md#списки-1>) и нескольких [ссылок](<03. Basic Syntax.md#ссылки>).

```html
#### Table of Contents

- [Underline](#подчёркивание)
- [Indent](#отступ-табуляция)
- [Center](#выравнивание-по-центру)
- [Color](#цвет)
```

Результат выглядит так:

<h4 class="no-anchor" data-toc-skip>Table of Contents</h4>

- [Underline](#подчёркивание)
- [Indent](#отступ-табуляция)
- [Center](#выравнивание-по-центру)
- [Color](#цвет)

## Видео

Если ваше приложение Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), вы сможете встроить видео в файл Markdown, скопировав и вставив HTML-код, который предоставляет видеосайт вроде YouTube или Vimeo. Если ваше приложение Markdown не поддерживает HTML, встроить видео не получится, но можно приблизиться к этому, добавив [изображение](<03. Basic Syntax.md#изображения-1>) и ссылку на видео. Так можно поступить практически с любым видео на любом видеосервисе. 

Поскольку с YouTube это делать удобно, возьмём его в качестве примера. Рассмотрим, к примеру, вот это видео: `https://www.youtube.com/watch?v=8q2IjQOzVpE`. Последняя часть URL (`8q2IjQOzVpE`) — это идентификатор видео. Мы можем взять этот идентификатор и подставить его в следующий шаблон:

```test
[![Image alt text](https://img.youtube.com/vi/YOUTUBE-ID/0.jpg)](https://www.youtube.com/watch?v=YOUTUBE-ID)
```

YouTube автоматически создаёт изображение для каждого видео (`https://img.youtube.com/vi/YOUTUBE-ID/0.jpg`), так что мы можем использовать его и [сделать изображение ссылкой](<03. Basic Syntax.md#ссылки-на-изображениях>) на видео на YouTube. После того как мы заменим alt-текст изображения и добавим идентификатор видео, наш пример будет выглядеть так:

```test
[![Less Than Jake — Scott Farcas Takes It On The Chin](https://img.youtube.com/vi/PYCxct2e0zI/0.jpg)](https://www.youtube.com/watch?v=PYCxct2e0zI)
```

Результат выглядит так:

<a href="https://www.youtube.com/watch?v=PYCxct2e0zI" rel="nofollow"><img src="https://img.youtube.com/vi/PYCxct2e0zI/0.jpg" width="480" height="360" alt="Less Than Jake — Scott Farcas Takes It On The Chin"></a>

# Расширенный синтаксис

## Обзор

[Базовый синтаксис](<03. Basic Syntax.md>), описанный в исходном документе о структуре Markdown, добавил многие элементы, необходимые в повседневной работе, но некоторым этого оказалось недостаточно. Здесь на помощь приходит расширенный синтаксис.

Несколько человек и организаций взяли на себя задачу расширить базовый синтаксис, добавив дополнительные элементы, такие как таблицы, блоки кода, подсветка синтаксиса, автоматическое создание ссылок из URL и сноски. Эти элементы можно включить с помощью лёгкого языка разметки, который надстраивается над базовым синтаксисом Markdown, либо добавив расширение в совместимый обработчик Markdown.

## Доступность

Не все приложения Markdown поддерживают элементы расширенного синтаксиса. Вам нужно будет проверить, поддерживает ли лёгкий язык разметки, который использует ваше приложение, те элементы расширенного синтаксиса, которые вы хотите применить. Если нет, то, возможно, всё же удастся включить расширения в вашем обработчике Markdown.

### Лёгкие языки разметки

Существует несколько лёгких языков разметки, которые являются *надмножествами* Markdown. Они включают базовый синтаксис и надстраиваются над ним, добавляя дополнительные элементы, такие как таблицы, блоки кода, подсветка синтаксиса, автоматическое создание ссылок из URL и сноски. Многие из наиболее популярных приложений Markdown используют один из следующих лёгких языков разметки:

- [CommonMark](https://commonmark.org)
- [GitHub Flavored Markdown (GFM)](https://github.github.com/gfm/)
- [Markdown Extra](https://michelf.ca/projects/php-markdown/extra/)
- [MultiMarkdown](https://fletcherpenney.net/multimarkdown/)
- [R Markdown](https://rmarkdown.rstudio.com/)

### Обработчики Markdown

Доступны [десятки обработчиков Markdown](https://github.com/markdown/markdown.github.com/wiki/Implementations). Многие из них позволяют добавлять расширения, которые включают элементы расширенного синтаксиса. За дополнительной информацией обращайтесь к документации вашего обработчика.

## Таблицы

Чтобы добавить таблицу, используйте три или более дефиса (`---`) для создания заголовка каждого столбца и вертикальные черты (`|`) для разделения столбцов. Для совместимости также следует добавлять вертикальную черту с обоих концов строки.

```
| Syntax      | Description |
| ----------- | ----------- |
| Header      | Title       |
| Paragraph   | Text        |
```

Результат выглядит так:

<table class="table table-bordered">
  <thead>
    <tr>
      <th>Syntax</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Header</td>
      <td>Title</td>
    </tr>
    <tr>
      <td>Paragraph</td>
      <td>Text</td>
    </tr>
  </tbody>
</table>

Ширина ячеек может различаться, как показано ниже. Результат будет выглядеть одинаково.

```
| Syntax | Description |
| --- | ----------- |
| Header | Title |
| Paragraph | Text |
```

<div class="alert alert-success">
  <i class="fas fa-lightbulb"></i> <strong>Совет:</strong> Создавать таблицы с помощью дефисов и вертикальных черт бывает утомительно. Чтобы ускорить процесс, попробуйте воспользоваться <a href="https://www.tablesgenerator.com/markdown_tables">генератором таблиц Markdown</a> или <a href="https://anywaydata.com">экспортом в Markdown от AnyWayData</a>. Постройте таблицу через графический интерфейс, а затем скопируйте сгенерированный текст в формате Markdown в свой файл.
</div>

### Выравнивание

Вы можете выровнять текст в столбцах по левому краю, правому краю или по центру, добавив двоеточие (`:`) слева, справа или с обеих сторон от дефисов внутри строки заголовка.

```
| Syntax      | Description | Test Text     |
| :---        |    :----:   |          ---: |
| Header      | Title       | Here's this   |
| Paragraph   | Text        | And more      |
```

Результат выглядит так:

<table class="table table-bordered">
  <thead>
    <tr>
      <th style="text-align: left">Syntax</th>
      <th style="text-align: center">Description</th>
      <th style="text-align: right">Test Text</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td style="text-align: left">Header</td>
      <td style="text-align: center">Title</td>
      <td style="text-align: right">Here’s this</td>
    </tr>
    <tr>
      <td style="text-align: left">Paragraph</td>
      <td style="text-align: center">Text</td>
      <td style="text-align: right">And more</td>
    </tr>
  </tbody>
</table>

### Форматирование текста в таблицах

Текст внутри таблиц можно форматировать. Например, вы можете добавлять [ссылки](<03. Basic Syntax.md#ссылки>), [код](<03. Basic Syntax.md#код>) (только слова или фразы в обратных кавычках (`` ` ``), но не [блоки кода](<03. Basic Syntax.md#блоки-кода>)) и [выделение](<03. Basic Syntax.md#выделение>).

Нельзя использовать заголовки, цитаты, списки, горизонтальные линии, изображения и большинство HTML-тегов.

  <i class="fas fa-lightbulb"></i> <strong>Совет:</strong> С помощью HTML вы можете создавать <a href="05.%20Hacks.md#переносы-строк-внутри-ячеек-таблицы">переносы строк</a> и добавлять <a href="05.%20Hacks.md#списки-внутри-ячеек-таблицы">списки</a> внутри ячеек таблицы.
</div>

### Экранирование символов вертикальной черты в таблицах

Вы можете отобразить символ вертикальной черты (`|`) в таблице, используя его HTML-код (`&#124;`).

## Огороженные блоки кода

Базовый синтаксис Markdown позволяет создавать [блоки кода](<03. Basic Syntax.md#блоки-кода>) путём отступа строк на четыре пробела или одну табуляцию. Если это неудобно, попробуйте использовать огороженные блоки кода. В зависимости от вашего обработчика или редактора Markdown вы будете использовать три обратные кавычки (<code>```</code>) или три тильды (`~~~`) на строках до и после блока кода. И самое приятное — вам не нужно делать отступ ни для одной строки!

~~~~~~~~~
```
{
  "firstName": "John",
  "lastName": "Smith",
  "age": 25
}
```
~~~~~~~~~

Результат выглядит так:

```text
{
  "firstName": "John",
  "lastName": "Smith",
  "age": 25
}
```

<div class="alert alert-success">
  <i class="fas fa-lightbulb"></i> <strong>Совет:</strong> Нужно отобразить обратные кавычки внутри блока кода? Смотрите <a href="03.%20Basic%20Syntax.md#экранирование-обратных-кавычек">этот раздел</a>, чтобы узнать, как их экранировать.
</div>

### Подсветка синтаксиса

Многие обработчики Markdown поддерживают подсветку синтаксиса для огороженных блоков кода. Эта возможность позволяет добавить цветовую подсветку в зависимости от языка, на котором написан ваш код. Чтобы добавить подсветку синтаксиса, укажите язык рядом с обратными кавычками перед огороженным блоком кода.

~~~~~~~~~
```json
{
  "firstName": "John",
  "lastName": "Smith",
  "age": 25
}
```
~~~~~~~~~

Результат выглядит так:

```json
{
  "firstName": "John",
  "lastName": "Smith",
  "age": 25
}
```

## Сноски

Сноски позволяют добавлять примечания и ссылки, не загромождая основной текст документа. Когда вы создаёте сноску, в месте, где вы добавили ссылку на сноску, появляется надстрочный номер со ссылкой. Читатели могут нажать на ссылку, чтобы перейти к содержимому сноски внизу страницы.

Чтобы создать ссылку на сноску, добавьте знак вставки и идентификатор внутри квадратных скобок (`[^1]`). Идентификаторы могут быть числами или словами, но они не могут содержать пробелы или табуляции. Идентификаторы лишь связывают ссылку на сноску с самой сноской — в выводе сноски нумеруются последовательно.

Добавьте сноску с помощью ещё одного знака вставки и номера внутри квадратных скобок с двоеточием и текстом (`[^1]: My footnote.`). Необязательно размещать сноски в конце документа. Вы можете разместить их где угодно, кроме как внутри других элементов, таких как списки, цитаты и таблицы.

```
Here's a simple footnote,[^1] and here's a longer one.[^bignote]

[^1]: This is the first footnote.

[^bignote]: Here's one with multiple paragraphs and code.

    Indent paragraphs to include them in the footnote.

    `{ my code }`

    Add as many paragraphs as you like.
```

Результат выглядит так:

Here's a simple footnote,[^1] and here's a longer one.[^bignote]

[^1]: This is the first footnote.

[^bignote]: Here's one with multiple paragraphs and code.

    Indent paragraphs to include them in the footnote.

    `{ my code }`

    Add as many paragraphs as you like.

## Идентификаторы заголовков

Многие обработчики Markdown поддерживают пользовательские идентификаторы для [заголовков](<03. Basic Syntax.md#заголовки>) — некоторые обработчики Markdown добавляют их автоматически. Добавление пользовательских идентификаторов позволяет ссылаться непосредственно на заголовки и изменять их с помощью CSS. Чтобы добавить пользовательский идентификатор заголовка, заключите его в фигурные скобки на той же строке, что и заголовок.

```text
### My Great Heading {#custom-id}
```

HTML выглядит так:

```html
<h3 id="custom-id">My Great Heading</h3>
```

### Ссылки на идентификаторы заголовков

Вы можете ссылаться на заголовки с пользовательскими идентификаторами внутри файла, создав [обычную ссылку](<03. Basic Syntax.md#ссылки>) со знаком решётки (`#`), за которым следует пользовательский идентификатор заголовка. Такие ссылки обычно называют *якорными ссылками*.

<table class="table table-bordered">
  <thead class="thead-light">
    <tr>
      <th>Markdown</th>
      <th>HTML</th>
      <th>Результат</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code class="highlighter-rouge">[Heading IDs](#идентификаторы-заголовков)</code></td>
      <td><code class="highlighter-rouge"> &lt;a href="#идентификаторы-заголовков"&gt;Heading IDs&lt;/a&gt;</code></td>
      <td><a href="#идентификаторы-заголовков">Heading IDs</a></td>
    </tr>
  </tbody>
</table>

Другие сайты могут ссылаться на заголовок, добавив пользовательский идентификатор заголовка к полному URL веб-страницы (например, `[Heading IDs](https://www.markdownguide.org/extended-syntax#heading-ids)`).

## Списки определений

Некоторые обработчики Markdown позволяют создавать *списки определений* из терминов и соответствующих им определений. Чтобы создать список определений, введите термин на первой строке. На следующей строке введите двоеточие, за которым следует пробел и определение.  

```
First Term
: This is the definition of the first term.

Second Term
: This is one definition of the second term.
: This is another definition of the second term.
```

HTML выглядит так:

```html
<dl>
  <dt>First Term</dt>
  <dd>This is the definition of the first term.</dd>
  <dt>Second Term</dt>
  <dd>This is one definition of the second term. </dd>
  <dd>This is another definition of the second term.</dd>
</dl>
```

Результат выглядит так:

First Term
: This is the definition of the first term.

Second Term
: This is one definition of the second term.
: This is another definition of the second term.

## Зачёркивание

Вы можете зачёркивать слова, проводя горизонтальную линию через их центр. Результат выглядит ~~вот так~~. Эта возможность позволяет указать, что определённые слова являются ошибкой и не предназначены для включения в документ. Чтобы зачеркнуть слова, используйте два символа тильды (`~~`) до и после слов.

```
~~The world is flat.~~ We now know that the world is round.
```

Результат выглядит так:

~~The world is flat.~~ We now know that the world is round.

## Списки задач

Списки задач (также называемые *контрольными списками* и списками *дел*) позволяют создавать список пунктов с флажками. В приложениях Markdown, которые поддерживают списки задач, рядом с содержимым будут отображаться флажки. Чтобы создать список задач, добавьте перед пунктами списка задач дефисы (`-`) и квадратные скобки с пробелом (`[ ]`). Чтобы установить флажок, добавьте `x` между скобками (`[x]`).

```
- [x] Write the press release
- [ ] Update the website
- [ ] Contact the media
```

## Эмодзи

Есть два способа добавить эмодзи в файлы Markdown: скопировать и вставить эмодзи в текст в формате Markdown или ввести *короткие коды эмодзи*.

### Копирование и вставка эмодзи

В большинстве случаев вы можете просто скопировать эмодзи из источника вроде [Emojipedia](https://emojipedia.org/) и вставить его в свой документ. Многие приложения Markdown автоматически отобразят эмодзи в тексте в формате Markdown. Файлы HTML и PDF, которые вы экспортируете из своего приложения Markdown, должны отображать эмодзи.

<div class="alert alert-success">
  <i class="fas fa-lightbulb"></i> <strong>Совет:</strong> Если вы используете генератор статических сайтов, убедитесь, что вы <a href="https://www.w3.org/International/tutorials/tutorial-char-enc/">кодируете HTML-страницы в UTF-8</a>.
</div>

### Использование коротких кодов эмодзи

Некоторые приложения Markdown позволяют вставлять эмодзи с помощью коротких кодов эмодзи. Они начинаются и заканчиваются двоеточием и содержат название эмодзи.

```text
Gone camping! :tent: Be back soon.

That is so funny! :joy:
```

Результат выглядит так:

Gone camping! ⛺ Be back soon.

That is so funny! 😂

<div class="alert alert-info">
  <i class="fas fa-info-circle"></i> <strong>Примечание:</strong> Вы можете использовать этот <a href="https://gist.github.com/rxaviers/7360908">список коротких кодов эмодзи</a>, но имейте в виду, что короткие коды эмодзи различаются от приложения к приложению. За дополнительной информацией обращайтесь к документации вашего приложения Markdown.
</div>

## Выделение цветом

Это встречается нечасто, но некоторые обработчики Markdown позволяют выделять текст цветом. Результат выглядит <mark>вот так</mark>. Чтобы выделить слова цветом, используйте два знака равенства (`==`) до и после слов.

```text
I need to highlight these ==very important words==.
```

Результат выглядит так:

I need to highlight these <mark>very important words</mark>.

Кроме того, если ваше приложение Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), вы можете использовать HTML-тег `mark`.

```html
I need to highlight these <mark>very important words</mark>.
```

## Нижний индекс

Это встречается нечасто, но некоторые обработчики Markdown позволяют использовать *нижний индекс*, чтобы разместить один или несколько символов чуть ниже обычной линии текста. Чтобы создать нижний индекс, используйте один символ тильды (`~`) до и после символов.

```text
H~2~O
```

Результат выглядит так:

H<sub>2</sub>O

<div class="alert alert-success">
  <i class="fas fa-lightbulb"></i> <strong>Совет:</strong> Обязательно проверьте это в своём приложении Markdown, прежде чем использовать. Некоторые приложения Markdown используют один символ тильды до и после слов не для нижнего индекса, а для <a href="04.%20Extended%20Syntax.md#зачёркивание">зачёркивания</a>. 
</div>

Кроме того, если ваше приложение Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), вы можете использовать HTML-тег `sub`.

```html
H<sub>2</sub>O
```

## Верхний индекс

Это встречается нечасто, но некоторые обработчики Markdown позволяют использовать *верхний индекс*, чтобы разместить один или несколько символов чуть выше обычной линии текста. Чтобы создать верхний индекс, используйте один символ вставки (`^`) до и после символов.

```text
X^2^
```

Результат выглядит так:

X<sup>2</sup>

Кроме того, если ваше приложение Markdown поддерживает [HTML](<03. Basic Syntax.md#html>), вы можете использовать HTML-тег `sup`.

```html
X<sup>2</sup>
```

## Автоматическое создание ссылок из URL

Многие обработчики Markdown автоматически превращают URL-адреса в ссылки. Это означает, что если вы наберёте http://www.example.com, ваш обработчик Markdown автоматически превратит его в ссылку, даже если вы не [использовали квадратные скобки](<03. Basic Syntax.md#ссылки>).

```
http://www.example.com
```

Результат выглядит так:

[http://www.example.com](http://www.example.com)

## Отключение автоматического создания ссылок из URL

Если вы не хотите, чтобы URL автоматически превращался в ссылку, вы можете убрать её, [обозначив URL как код](<03. Basic Syntax.md#код>) с помощью обратных кавычек.

```
`http://www.example.com`
```

Результат выглядит так:

`http://www.example.com`

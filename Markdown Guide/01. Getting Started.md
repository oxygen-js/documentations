# Начало работы

## Что такое Markdown?

Markdown — это лёгкий язык разметки, с помощью которого можно добавлять элементы форматирования в текстовые документы в формате простого текста. Созданный [Джоном Грубером (John Gruber)](https://daringfireball.net/projects/markdown/) в 2004 году, Markdown сегодня стал одним из самых популярных языков разметки в мире.

Работа с Markdown отличается от работы в [WYSIWYG](https://en.wikipedia.org/wiki/WYSIWYG)-редакторе. В приложении вроде Microsoft Word вы нажимаете кнопки, чтобы отформатировать слова и фразы, и изменения сразу становятся видны. С Markdown всё иначе. Когда вы создаёте файл в формате Markdown, вы добавляете к тексту синтаксис Markdown, чтобы указать, какие слова и фразы должны выглядеть по-другому.

Например, чтобы обозначить заголовок, перед ним ставят знак решётки (например, `# Heading One`). А чтобы сделать фразу полужирной, её обрамляют двумя звёздочками с обеих сторон (например, `**this text is bold**`). Возможно, потребуется время, чтобы привыкнуть видеть синтаксис Markdown прямо в тексте, особенно если вы привыкли к WYSIWYG-приложениям. На скриншоте ниже показан файл Markdown, открытый в [текстовом редакторе Visual Studio Code](https://www.markdownguide.org/tools/vscode/).

![Файл Markdown в текстовом редакторе Visual Studio Code](https://www.markdownguide.org/assets/images/vscode.png)

Добавлять элементы форматирования Markdown в файл простого текста можно с помощью текстового редактора. Или же можно воспользоваться одним из множества приложений для Markdown под операционные системы macOS, Windows, Linux, iOS и Android. Есть и несколько веб-приложений, специально разработанных для письма на Markdown.

В зависимости от используемого приложения предпросмотр отформатированного документа в реальном времени может быть недоступен. Но это не страшно. [По словам Грубера](https://daringfireball.net/projects/markdown/), синтаксис Markdown задуман читаемым и ненавязчивым, поэтому текст в файлах Markdown можно читать, даже если он не отрендерен.

> Главная цель при разработке синтаксиса форматирования Markdown — сделать его как можно более читаемым. Идея в том, что документ в формате Markdown должен быть пригоден к публикации как есть, в виде простого текста, не выглядя размеченным тегами или инструкциями форматирования.

## Зачем использовать Markdown?

Возможно, вы задаётесь вопросом, почему люди используют Markdown вместо WYSIWYG-редактора. Зачем писать на Markdown, когда можно нажимать кнопки в интерфейсе, чтобы форматировать текст? Как выясняется, есть несколько причин, по которым люди предпочитают Markdown WYSIWYG-редакторам.

- Markdown можно использовать для чего угодно. С его помощью создают [веб-сайты](#веб-сайты), [документы](#документы), [заметки](#заметки), [книги](#книги), [презентации](#презентации), [электронные письма](#электронная-почта) и [техническую документацию](#документация).

- Markdown переносим. Файлы с текстом в формате Markdown можно открыть практически в любом приложении. Если вам разонравится приложение для Markdown, которым вы сейчас пользуетесь, вы сможете импортировать свои файлы Markdown в другое приложение. Это резко контрастирует с текстовыми процессорами вроде Microsoft Word, которые запирают ваш контент в проприетарном формате файла.

- Markdown не зависит от платформы. Текст в формате Markdown можно создавать на любом устройстве под управлением любой операционной системы.

- Markdown защищён от устаревания. Даже если приложение, которым вы пользуетесь, когда-нибудь в будущем перестанет работать, вы всё равно сможете прочитать свой текст в формате Markdown с помощью любого текстового редактора. Это важное соображение, когда речь идёт о книгах, университетских диссертациях и других значимых документах, которые нужно сохранить на неопределённо долгий срок.

- Markdown повсюду. Такие сайты, как [Reddit](https://www.markdownguide.org/tools/reddit/) и GitHub, поддерживают Markdown, и его поддерживает множество настольных и веб-приложений.

## Пробуем в деле

Лучший способ освоить Markdown — начать им пользоваться. Сделать это проще, чем когда-либо, благодаря множеству бесплатных инструментов.

Вам даже не нужно ничего скачивать. Есть несколько онлайн-редакторов Markdown, в которых можно попробовать писать на Markdown. [JotBird](https://www.jotbird.com) (который создал я) — один из лучших. Просто откройте сайт и начните печатать в левой панели. Предпросмотр отрендеренного документа появляется в правой панели.

![Редактор Markdown JotBird](https://www.markdownguide.org/assets/images/jotbird.png)

JotBird умеет ещё кое-что, о чём стоит знать. Когда вы написали что-то, чем хотите поделиться, — заметки, документ, README, — вы можете опубликовать это как настоящую веб-страницу с собственной ссылкой в один клик. Так что это не просто место для тренировки в Markdown, а место, где с Markdown можно что-то сделать.

Скорее всего, вам захочется держать сайт JotBird открытым, пока вы читаете это руководство. Так вы сможете пробовать синтаксис по мере его изучения. Освоившись с Markdown, вы, возможно, захотите использовать приложение для Markdown, которое можно установить на настольный компьютер или мобильное устройство.

## Как это работает?

JotBird упрощает письмо на Markdown, скрывая то, что происходит за кулисами, но всё же стоит разобраться, как этот процесс устроен в целом.

Когда вы пишете на Markdown, текст сохраняется в файле простого текста с расширением `.md` или `.markdown`. Но что дальше? Как ваш файл в формате Markdown превращается в HTML или готовый к печати документ?

Короткий ответ таков: вам нужно *приложение Markdown*, способное обработать файл Markdown. Доступно множество приложений — от простых скриптов до настольных программ, похожих на Microsoft Word. Несмотря на внешние различия, все эти приложения делают одно и то же. Как и JotBird, все они преобразуют текст в формате Markdown в HTML, чтобы его можно было отобразить в веб-браузерах.

Приложения Markdown используют так называемый *обработчик Markdown* (его также часто называют «парсером» или «реализацией»), чтобы взять текст в формате Markdown и вывести его в формате HTML. После этого ваш документ можно просмотреть в веб-браузере или объединить с таблицей стилей и распечатать. Наглядное представление этого процесса приведено ниже.

![Процесс Markdown](https://www.markdownguide.org/assets/images/markdown-flowchart.png)

Если подытожить, это процесс из четырёх частей:

1. Создайте файл Markdown с помощью текстового редактора или специализированного приложения Markdown. Файл должен иметь расширение `.md` или `.markdown`.
2. Откройте файл Markdown в приложении Markdown.
3. С помощью приложения Markdown преобразуйте файл Markdown в документ HTML.
4. Просмотрите файл HTML в веб-браузере или используйте приложение Markdown, чтобы преобразовать его в другой формат файла, например PDF.

С вашей точки зрения процесс будет несколько различаться в зависимости от используемого приложения. Например, JotBird по сути объединяет шаги 1–3 в едином, бесшовном интерфейсе — всё, что нужно, — это печатать в левой панели, а отрендеренный результат волшебным образом появляется в правой. Но если вы используете другие инструменты, например текстовый редактор вместе с генератором статических сайтов, процесс окажется куда более наглядным.

## Для чего хорош Markdown?

Markdown — быстрый и простой способ делать заметки, создавать контент для сайта и готовить документы к печати.

Освоить синтаксис Markdown недолго, и, разобравшись, как им пользоваться, вы сможете писать на Markdown практически везде. Большинство людей используют Markdown для создания контента для веба, но Markdown хорош для форматирования чего угодно — от электронных писем до списков покупок.

Вот несколько примеров того, что можно делать с помощью Markdown.

### Веб-сайты

Markdown был разработан для веба, так что неудивительно, что существует множество приложений, специально предназначенных для создания контента сайтов.

Если вы ищете самый простой из возможных способов создать сайт из файлов Markdown, обратите внимание на [blot.im](https://blot.im). После регистрации в Blot он создаёт на вашем компьютере папку Dropbox. Просто перетащите свои файлы Markdown в эту папку — и вуаля! — они уже на вашем сайте. Проще не бывает.

Если вы знакомы с HTML, CSS и системами контроля версий, посмотрите на [Jekyll](https://www.markdownguide.org/tools/jekyll/) — популярный генератор статических сайтов, который берёт файлы Markdown и собирает из них HTML-сайт. Одно из преимуществ такого подхода в том, что [GitHub Pages](https://www.markdownguide.org/tools/github-pages/) предоставляет бесплатный хостинг для сайтов, сгенерированных Jekyll. Если Jekyll вам не по душе, просто выберите один из [множества других доступных генераторов статических сайтов](https://jamstack.org/generators/).

  <i class="fas fa-info-circle"></i> <strong>Примечание:</strong> Немного саморекламы! Если хотите научиться создавать статические сайты с нуля, обратите внимание на <a href="https://www.staticguide.org/"><em>The Static Site Guide</em></a> — ещё одну написанную мной книгу.
</div>

Если вы хотите использовать для сайта систему управления контентом (CMS), взгляните на [Ghost](https://www.markdownguide.org/tools/ghost/). Это бесплатная платформа для блогов с открытым исходным кодом и удобным редактором Markdown. А если вы пользователь WordPress, вам будет приятно узнать, что есть [поддержка Markdown](https://wordpress.com/support/wordpress-editor/blocks/markdown-block/) для сайтов, размещённых на WordPress.com. Для самостоятельно размещённых сайтов WordPress можно использовать [плагин Jetpack](https://jetpack.com/support/jetpack-blocks/markdown/).

### Документы

В Markdown нет всех наворотов текстовых процессоров вроде Microsoft Word, но его вполне достаточно для создания простых документов — например, заданий и писем. Вы можете использовать приложение для создания документов на Markdown, чтобы создавать и экспортировать документы в формате Markdown в PDF или HTML. Часть с PDF здесь ключевая, потому что, получив документ PDF, вы можете делать с ним что угодно — распечатать, отправить по электронной почте или загрузить на сайт.

Вот несколько приложений для создания документов на Markdown, которые я рекомендую:

- **Mac:** [MacDown](https://www.markdownguide.org/tools/macdown/), [iA Writer](https://www.markdownguide.org/tools/ia-writer/) или [Marked 2](https://www.markdownguide.org/tools/marked-2/)
- **iOS / Android:** [iA Writer](https://www.markdownguide.org/tools/ia-writer/)
- **Windows:** [ghostwriter](https://kde.github.io/ghostwriter/) или [Markdown Monster](https://markdownmonster.west-wind.com/)
- **Linux:** [ReText](https://github.com/retext-project/retext) или [ghostwriter](https://kde.github.io/ghostwriter/)
- **Web:** [JotBird](https://www.markdownguide.org/tools/jotbird/), [Dillinger](https://www.markdownguide.org/tools/dillinger/) или [StackEdit](https://www.markdownguide.org/tools/stackedit/)

### Заметки

Практически по всем параметрам Markdown — идеальный синтаксис для ведения заметок. К сожалению, [Evernote](https://evernote.com/) и [OneNote](https://www.onenote.com/), два самых популярных приложения для заметок, в настоящее время не поддерживают Markdown. Хорошая новость в том, что несколько других приложений для заметок *всё же* поддерживают Markdown:

- [Obsidian](https://www.markdownguide.org/tools/obsidian/) — популярное приложение для заметок на Markdown, набитое возможностями.
- [Simplenote](https://www.markdownguide.org/tools/simplenote/) — бесплатное минималистичное приложение для заметок, доступное на всех платформах.
- [Notable](https://www.markdownguide.org/tools/notable/) — приложение для заметок, работающее на самых разных платформах.
- [Bear](https://www.markdownguide.org/tools/bear/) — приложение в духе Evernote, доступное для устройств Mac и iOS. По умолчанию оно использует не только Markdown, но вы можете включить режим совместимости с Markdown.
- [Joplin](https://www.markdownguide.org/tools/joplin/) — приложение для заметок, которое уважает вашу конфиденциальность. Оно доступно на всех платформах.

Если вы не можете расстаться с Evernote, обратите внимание на [Marxico](https://marxi.co/) — редактор Markdown для Evernote по подписке — или используйте [Markdown Here](https://www.markdownguide.org/tools/markdown-here/) на сайте Evernote.

### Книги

Хотите самостоятельно издать роман? Попробуйте [Leanpub](https://leanpub.com/) — сервис, который берёт ваши файлы в формате Markdown и превращает их в электронную книгу. Leanpub выдаёт вашу книгу в форматах PDF, EPUB и MOBI. Если вы хотите создать бумажные экземпляры своей книги, можно загрузить файл PDF в другой сервис, например [Kindle Direct Publishing](https://kdp.amazon.com). Чтобы узнать больше о написании и самостоятельном издании книги с помощью Markdown, прочтите [эту запись в блоге](https://medium.com/techspiration-ideas-making-it-happen/how-i-wrote-and-published-my-novel-using-only-open-source-tools-5cdfbd7c00ca).

### Презентации

Хотите верьте, хотите нет, но из файлов в формате Markdown можно создавать презентации. Создание презентаций на Markdown требует некоторой привычки, но, набив руку, вы обнаружите, что это гораздо быстрее и проще, чем в приложении вроде PowerPoint или Keynote. [Remark](https://remarkjs.com) ([проект на GitHub](https://github.com/gnab/remark)) — популярный браузерный инструмент для слайд-шоу на Markdown, как и [Cleaver](https://jdan.github.io/cleaver/) ([проект на GitHub](https://github.com/jdan/cleaver)) и [Marp](https://marp.app/) ([проект на GitHub](https://github.com/marp-team/marp)). Если вы пользуетесь Mac и предпочли бы отдельное приложение, обратите внимание на [Deckset](https://www.decksetapp.com/) или [Hyperdeck](https://hyperdeck.io/).

### Электронная почта

Если вы отправляете много писем и устали от средств форматирования, доступных на сайтах большинства почтовых провайдеров, вам будет приятно узнать, что есть простой способ писать электронные письма с помощью Markdown. [Markdown Here](https://www.markdownguide.org/tools/markdown-here/) — бесплатное расширение для браузера с открытым исходным кодом, которое преобразует текст в формате Markdown в готовый к отправке HTML.

### Совместная работа

Приложения для совместной работы и командных сообщений — популярный способ общения с коллегами и друзьями на работе и дома. Эти приложения используют не все возможности Markdown, но те, что они предоставляют, довольно полезны. Например, возможность делать текст полужирным и курсивным без WYSIWYG-интерфейса весьма удобна. [Slack](https://www.markdownguide.org/tools/slack/), [Discord](https://www.markdownguide.org/tools/discord/), [Wiki.js](https://www.markdownguide.org/tools/wiki-js/) и [Mattermost](https://www.markdownguide.org/tools/mattermost/) — все это хорошие приложения для совместной работы.

### Документация

Markdown естественно подходит для технической документации. Такие компании, как GitHub, всё чаще переходят на Markdown для своей документации — посмотрите их [запись в блоге](https://github.com/blog/1939-how-github-uses-github-to-document-github) о том, как они перенесли свою документацию в формате Markdown на [Jekyll](https://www.markdownguide.org/tools/jekyll/). Если вы пишете документацию для продукта или сервиса, взгляните на эти удобные инструменты:

- [Read the Docs](https://readthedocs.org) может сгенерировать сайт документации из ваших файлов Markdown с открытым исходным кодом. Просто подключите свой репозиторий GitHub к их сервису и сделайте push — Read the Docs сделает всё остальное. У них также есть [сервис для коммерческих организаций](https://readthedocs.com/).
- [MkDocs](https://www.markdownguide.org/tools/mkdocs/) — быстрый и простой генератор статических сайтов, ориентированный на создание проектной документации. Исходные файлы документации пишутся на Markdown и настраиваются одним файлом конфигурации YAML. У MkDocs есть несколько [встроенных тем](https://www.mkdocs.org/user-guide/styling-your-docs/), включая порт темы документации [Read the Docs](https://readthedocs.org/) для использования с MkDocs. Одна из новейших тем — [MkDocs Material](https://squidfunk.github.io/mkdocs-material/).
- [Docusaurus](https://www.markdownguide.org/tools/docusaurus/) — генератор статических сайтов, предназначенный исключительно для создания сайтов документации. Он поддерживает переводы, поиск и версионирование.
- [VuePress](https://vuepress.vuejs.org/) — генератор статических сайтов на основе [Vue](https://vuejs.org/), оптимизированный для написания технической документации.
- [Jekyll](https://www.markdownguide.org/tools/jekyll/) уже упоминался ранее в разделе о веб-сайтах, но это также хороший вариант для генерации сайта документации из файлов Markdown. Если вы пойдёте этим путём, обязательно ознакомьтесь с [темой документации Jekyll](https://idratherbewriting.com/documentation-theme-jekyll/).

## Разновидности Markdown

Один из самых запутанных аспектов работы с Markdown в том, что практически каждое приложение Markdown реализует немного другую версию Markdown. Эти варианты Markdown принято называть *разновидностями* (flavors). Ваша задача — освоить ту разновидность Markdown, которую реализовало ваше приложение.

Чтобы уложить в голове понятие разновидностей Markdown, может помочь мысль о них как о диалектах языка. Жители Нью-Йорка говорят по-английски так же, как и жители Лондона, но между диалектами этих двух городов есть существенные различия. То же верно и для людей, использующих разные приложения Markdown. Писать на Markdown в [JotBird](https://www.jotbird.com) — это совершенно иной опыт, чем в [Ulysses](https://www.markdownguide.org/tools/ulysses/).

На практике это означает, что вы никогда не знаете точно, что имеет в виду компания, говоря, что поддерживает «Markdown». Идёт ли речь только о [базовых элементах синтаксиса](<03. Basic Syntax.md>), или обо всех базовых и [расширенных элементах синтаксиса](<04. Extended Syntax.md>) вместе, или о каком-то произвольном сочетании элементов синтаксиса? Вы этого не узнаете, пока не прочитаете документацию или не начнёте пользоваться приложением.

Если вы только начинаете, лучший совет, который я могу дать, — выбрать приложение Markdown с хорошей поддержкой Markdown. Это сильно поспособствует сохранению переносимости ваших файлов Markdown. Возможно, вам захочется хранить и использовать свои файлы Markdown в других приложениях, а для этого нужно начать с приложения, обеспечивающего хорошую поддержку. Найти подходящее приложение можно с помощью [каталога инструментов](https://www.markdownguide.org/tools/).

## Дополнительные ресурсы

Существует множество ресурсов, по которым можно изучить Markdown. Вот некоторые другие вводные материалы:

- [Документация Markdown Джона Грубера](https://daringfireball.net/projects/markdown/). Оригинальное руководство, написанное создателем Markdown.
- [Markdown Tutorial](https://www.markdowntutorial.com/). Сайт с открытым исходным кодом, где можно попробовать Markdown прямо в браузере.
- [Awesome Markdown](https://github.com/mundimark/awesome-markdown). Список инструментов и обучающих ресурсов по Markdown.
- [Typesetting Markdown](https://dave.autonoma.ca/blog/2019/05/22/typesetting-markdown-part-1). Серия из нескольких частей, описывающая экосистему для вёрстки документов Markdown с помощью [pandoc](https://pandoc.org/) и [ConTeXt](https://www.contextgarden.net/).

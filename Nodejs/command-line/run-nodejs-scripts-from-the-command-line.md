# Запуск скриптов Node.js из командной строки

Обычный способ запустить программу Node.js — выполнить глобально доступную команду `node` (после установки Node.js) и передать ей имя файла, который вы хотите выполнить.

Если ваш главный файл приложения Node.js называется `app.js`, вы можете вызвать его, набрав:
```
node app.js
```

Выше вы явно указываете оболочке запустить ваш скрипт с помощью `node`. Вы также можете встроить эту информацию в сам файл JavaScript с помощью строки [«shebang»](https://en.wikipedia.org/wiki/Shebang_\(Unix\)). «Shebang» — это первая строка в файле, которая сообщает ОС, какой интерпретатор использовать для запуска скрипта. Ниже приведена первая строка JavaScript:
```
#!/usr/bin/node
```

Выше мы явно указываем абсолютный путь к интерпретатору. Не во всех операционных системах `node` находится в папке bin, но во всех должен быть `env`. Вы можете указать ОС запустить `env` с node в качестве параметра:
```js
#!/usr/bin/env node

// ваш код на javascript
```

Чтобы использовать shebang, у вашего файла должны быть права на выполнение. Вы можете дать `app.js` права на выполнение, запустив:
```
chmod u+x app.js
```

При выполнении команды убедитесь, что вы находитесь в том же каталоге, где содержится файл `app.js`.

## Передача строки в качестве аргумента `node` вместо пути к файлу

Чтобы выполнить строку в качестве аргумента, вы можете использовать `-e`, `--eval "script"`. Следующий аргумент вычисляется как JavaScript. Модули, предопределённые в REPL, также можно использовать в скрипте.

В Windows при использовании cmd.exe одинарные кавычки работать не будут, потому что она распознаёт для кавычек только двойные `"`. В PowerShell или Git bash можно использовать и `'`, и `"`.
```
node -e "console.log(123)"
```

## Автоматический перезапуск приложения

Начиная с Node.js v16 есть встроенная опция для автоматического перезапуска приложения при изменении файла. Это удобно для целей разработки. Чтобы воспользоваться этой возможностью, нужно передать Node.js флаг `--watch`.
```
node --watch app.js
```

Таким образом, при изменении файла приложение будет перезапускаться автоматически. См. [документацию по флагу `--watch`](https://nodejs.org/docs/latest-v22.x/api/cli.html#--watch).

## Запуск задачи с помощью Node.js

Node.js предоставляет встроенное средство запуска задач, которое позволяет выполнять определённые команды, заданные в вашем файле `package.json`. Это может быть особенно полезно для автоматизации повторяющихся задач, таких как запуск тестов, сборка проекта или проверка кода линтером.

### Использование флага `--run`

Флаг [`--run`](https://nodejs.org/docs/latest-v22.x/api/cli.html#--run) позволяет запустить указанную команду из раздела `scripts` вашего файла `package.json`. Например, если у вас есть следующий `package.json`:
```json
{
  "type": "module",
  "scripts": {
    "start": "node app.js",
    "test": "node --test"
  }
}
```

Вы можете запустить скрипт `test` с помощью флага `--run`:
```
node --run test
```

### Передача аргументов команде

Вы можете использовать синтаксис `-- --another-argument`, чтобы передать аргументы нижележащему скрипту. Например, если вы хотите передать аргумент `--port` скрипту `start`:
```
node --run start -- --port 8080
```

Это запустит скрипт `start` и добавит `--port 8080` к выполняемой команде, что эквивалентно запуску `node app.js --port 8080`.

> Примечание: аргументы, переданные после `--`, направляются скрипту и не интерпретируются как флаги CLI Node.js. Например, `--watch` не будет вести себя как `node --watch app.js`, если передать его таким образом.

### Переменные окружения

Флаг `--run` устанавливает определённые переменные окружения, которые могут быть полезны для ваших скриптов:

  * `NODE_RUN_SCRIPT_NAME`: имя запускаемого скрипта.
  * `NODE_RUN_PACKAGE_JSON_PATH`: путь к обрабатываемому файлу `package.json`.

### Намеренные ограничения

Средство запуска задач в Node.js намеренно более ограничено по сравнению с другими средствами запуска задач, такими как `npm run` или `yarn run`. Оно сосредоточено на производительности и простоте, опуская такие возможности, как запуск скриптов `pre` или `post`. Это делает его пригодным для простых задач, но оно может подойти не для всех сценариев использования.

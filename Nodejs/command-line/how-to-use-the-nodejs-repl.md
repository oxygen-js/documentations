# Как пользоваться REPL в Node.js

## Что такое REPL в Node.js?

Node.js поставляется со встроенной средой REPL (Read-Eval-Print Loop — цикл «чтение — вычисление — вывод»), которая позволяет выполнять код JavaScript в интерактивном режиме. REPL доступен через терминал и отлично подходит для проверки небольших фрагментов кода.

## Как пользоваться REPL в Node.js

Команда `node` — это та, которую мы используем для запуска наших скриптов Node.js:
```
node script.js
```

Если запустить команду `node` без скрипта для выполнения или без каких-либо аргументов, мы запускаем сеанс REPL:
```
node
```

> **Примечание:** REPL расшифровывается как Read Evaluate Print Loop (цикл «чтение — вычисление — вывод») и представляет собой среду языка программирования (по сути, окно консоли), которая принимает от пользователя одно выражение и возвращает результат обратно в консоль после выполнения. Сеанс REPL — это удобный способ быстро проверить простой код JavaScript.

Если вы попробуете сделать это прямо сейчас в своём терминале, произойдёт следующее:
```bash
❯ node
>
```

Команда остаётся в режиме ожидания и ждёт, пока мы что-нибудь введём.

> **Совет:** если вы не знаете, как открыть терминал, поищите в Google «How to open terminal on your-operating-system».

Если точнее, REPL ждёт, пока мы введём какой-нибудь код JavaScript.

Начнём с простого и введём
```
> console.log('test')
test
undefined
>
```

Сеанс оболочки

Первое значение, `test`, — это вывод, который мы попросили консоль напечатать, затем мы получаем `undefined` — возвращаемое значение вызова `console.log()`. Node прочитал эту строку кода, вычислил её, напечатал результат, а затем вернулся к ожиданию новых строк кода. Node будет проходить по этим трём шагам для каждого фрагмента кода, который мы выполняем в REPL, пока мы не выйдем из сеанса. Именно отсюда REPL и получил своё название.

Node автоматически печатает результат любой строки кода JavaScript без необходимости специально указывать ему это делать. Например, наберите следующую строку и нажмите Enter:
```
> 5 === '5'
false
>
```

Сеанс оболочки

Обратите внимание на разницу в выводах двух приведённых выше строк. Node REPL напечатал `undefined` после выполнения `console.log()`, тогда как для `5 === '5'` он просто напечатал результат. Нужно помнить, что первое — это всего лишь инструкция (statement) в JavaScript, а второе — выражение (expression).

В некоторых случаях код, который вы хотите проверить, может занимать несколько строк. Например, предположим, вы хотите определить функцию, генерирующую случайное число; в сеансе REPL наберите следующую строку и нажмите Enter:
```
function generateRandom() {
...
```

Сеанс оболочки

Node REPL достаточно умён, чтобы определить, что вы ещё не закончили писать код, и переходит в многострочный режим, чтобы вы могли ввести больше кода. Теперь завершите определение функции и нажмите Enter:
```
function generateRandom() {
...return Math.random()
}
undefined
```

Сеанс оболочки

### Специальная переменная `_`

Если после некоторого кода вы наберёте `_`, будет напечатан результат последней операции.

### Клавиша «стрелка вверх»

Если вы нажмёте клавишу «стрелка вверх», вы получите доступ к истории предыдущих строк кода, выполненных в текущем и даже в предыдущих сеансах REPL.

### Точечные команды

В REPL есть несколько специальных команд, все они начинаются с точки `.`. Вот они:

  * `.help`: показывает справку по точечным командам.
  * `.editor`: входит в режим редактора для написания многострочного кода JavaScript.
  * `.break` / `.clear`: выходит из многострочного кода, например из функций. То же самое, что нажатие CTRL-C.
  * `.load`: загружает файл JavaScript относительно текущего рабочего каталога.
  * `.save`: сохраняет все команды, введённые в сеансе, в файл.
  * `.exit`: выходит из REPL (то же, что и двойное нажатие CTRL-C).

REPL понимает, когда вы вводите многострочную инструкцию, без необходимости вызывать `.editor`.

Например, если вы начнёте набирать итерацию вот так:
```
[1, 2, 3].forEach(num => {
```

и нажмёте `enter`, REPL перейдёт на новую строку, которая начинается с трёх точек, показывая, что теперь вы можете продолжить работу над этим блоком.
```
... console.log(num)
... })
```

Сеанс оболочки

Если вы наберёте `.break` в конце строки, многострочный режим будет остановлен, а инструкция не будет выполнена.

### Запуск REPL из файла JavaScript

Мы можем импортировать REPL в файл JavaScript с помощью `repl`.
```
const repl = require('node:repl');
```

Используя переменную repl, мы можем выполнять различные операции. Чтобы запустить командную строку REPL, наберите следующую строку
```
repl.start();
```

Запустите файл в командной строке.
```
node repl.js
```

Вы можете передать строку, которая отображается при запуске REPL. По умолчанию это '> ' (с завершающим пробелом), но мы можем задать собственное приглашение.
```js
// приглашение в стиле Unix
const local = repl.start('$ ');
```

Вы можете отобразить сообщение при выходе из REPL
```js
local.on('exit', () => {
  console.log('exiting repl');
  process.exit();
});
```

Подробнее о модуле REPL можно прочитать в [документации по repl](https://nodejs.org/api/repl.html).

# Как читать переменные окружения из Node.js

Основной модуль `process` в Node.js предоставляет свойство `env`, в котором хранятся все переменные окружения, заданные в момент запуска процесса.

Приведённый ниже код запускает `app.js` и задаёт `USER_ID` и `USER_KEY`.
```
USER_ID=239482 USER_KEY=foobar node app.js
```

Это передаст пользовательский `USER_ID` как **239482**, а `USER_KEY` как **foobar**. Такой способ подходит для тестирования, однако в продакшене вы, скорее всего, будете настраивать bash-скрипты для экспорта переменных.

> Примечание: `process` не нужно импортировать, это глобальный объект в Node.js.

Вот пример, обращающийся к переменным окружения `USER_ID` и `USER_KEY`, которые мы задали в коде выше.
```js
console.log(process.env.USER_ID); // "239482"
console.log(process.env.USER_KEY); // "foobar"
```

Точно так же вы можете обращаться к любой заданной вами пользовательской переменной окружения.

В Node.js 20 появилась **экспериментальная** [поддержка файлов .env](https://nodejs.org/docs/v24.5.0/api/environment_variables.html#env-files).

Теперь вы можете использовать флаг `--env-file`, чтобы указать файл с окружением при запуске приложения Node.js. Вот пример файла `.env` и того, как обращаться к его переменным через `process.env`.
```bash
# файл .env
PORT=3000
```

В вашем js-файле
```
console.log(process.env.PORT); // "3000"
```

Запустите файл `app.js` с переменными окружения, заданными в файле `.env`.
```
node --env-file=.env app.js
```

Эта команда загружает все переменные окружения из файла `.env`, делая их доступными приложению через `process.env`.

Кроме того, вы можете передать несколько аргументов `--env-file`. Последующие файлы переопределяют уже существующие переменные, определённые в предыдущих файлах.
```
node --env-file=.env --env-file=.development.env app.js
```

> Примечание: если одна и та же переменная определена и в окружении, и в файле, значение из окружения имеет приоритет.

Если вы хотите читать из файла `.env` опционально, можно избежать выброса ошибки при отсутствии файла с помощью флага `--env-file-if-exists`.
```
node --env-file-if-exists=.env app.js
```

## Программная загрузка файлов `.env` с помощью `process.loadEnvFile(path)`

Node.js предоставляет встроенный API для загрузки файлов `.env` прямо из вашего кода: [`process.loadEnvFile(path)`](https://nodejs.org/api/process.html#processloadenvfilepath).

Этот метод загружает переменные из файла `.env` в `process.env` — подобно тому, как работает флаг `--env-file`, — но может быть вызван программно.

Поскольку этот метод вызывается после инициализации, задание переменных окружения, связанных со стартом (то есть `NODE_OPTIONS`), не влияет на процесс (однако к этим переменным по-прежнему можно обращаться через `process.env`).

### Пример
```text
# файл .env
PORT=1234
```

Вы также можете указать пользовательский путь:
```js
const { loadEnvFile } = require('node:process');
loadEnvFile('./config/.env');
```

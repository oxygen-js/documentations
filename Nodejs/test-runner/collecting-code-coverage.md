# Сбор покрытия кода в Node.js

Node.js предоставляет встроенную поддержку покрытия кода через своё средство запуска тестов, которую можно включить с помощью флага [`--experimental-test-coverage`](https://nodejs.org/api/cli.html#--experimental-test-coverage).

Если вы используете API `run()`, опция `coverage` должна быть установлена в `true`. Дополнительную информацию об API `run()` см. в [документации `node:test`](https://nodejs.org/docs/latest/api/test.html#runoptions).

## Что такое покрытие кода?

Покрытие кода — это метрика для средств запуска тестов, которая оценивает, какая часть исходного кода программы выполняется во время тестирования. Она показывает, какие части кодовой базы протестированы, а какие нет, помогая выявить пробелы в наборе тестов. Это обеспечивает более полное тестирование программного обеспечения и минимизирует риск необнаруженных ошибок. Обычно покрытие выражается в процентах: более высокие проценты покрытия кода указывают на более тщательное покрытие тестами. Более подробное объяснение покрытия кода можно найти в [статье «Code coverage» в Википедии](https://en.wikipedia.org/wiki/Code_coverage).

## Базовый отчёт о покрытии

Давайте разберём простой пример, чтобы продемонстрировать, как покрытие кода работает в Node.js.

> **Примечание:** Этот пример, как и все остальные в этом файле, написан с использованием CommonJS. Если вы не знакомы с этой концепцией, прочитайте документацию по [модулям CommonJS](https://nodejs.org/docs/latest/api/modules.html).
```js
function add(a, b) {
  return a + b;
}

function isEven(num) {
  return num % 2 === 0;
}

function multiply(a, b) {
  return a * b;
}

module.exports = { add, isEven, multiply };
```

В модуле у нас есть три функции: `add`, `isEven` и `multiply`.

В тестовом файле мы тестируем функции `add()` и `isEven()`. Обратите внимание, что функция `multiply()` не покрыта никакими тестами.

Чтобы собрать покрытие кода во время выполнения ваших тестов, см. следующие фрагменты:
```
node --experimental-test-coverage --test main.test.js
```

После выполнения тестов вы получите отчёт, который выглядит примерно так:
```text
✔ add() should add two numbers (1.505987ms)
✔ isEven() should report whether a number is even (0.175859ms)
ℹ tests 2
ℹ suites 0
ℹ pass 2
ℹ fail 0
ℹ cancelled 0
ℹ skipped 0
ℹ todo 0
ℹ duration_ms 59.480373
ℹ start of coverage report
ℹ -------------------------------------------------------------
ℹ file         | line % | branch % | funcs % | uncovered lines
ℹ -------------------------------------------------------------
ℹ main.js      |  76.92 |   100.00 |   66.67 | 9-11
ℹ main.test.js | 100.00 |   100.00 |  100.00 |
ℹ -------------------------------------------------------------
ℹ all files    |  86.96 |   100.00 |   80.00 |
ℹ -------------------------------------------------------------
ℹ end of coverage report
```

Отчёт о покрытии предоставляет разбивку того, какая часть вашего кода покрыта тестами:

  * **Покрытие строк (Line Coverage)**: Процент строк, выполненных во время тестов.
  * **Покрытие ветвей (Branch Coverage)**: Процент ветвей кода (таких как конструкции if-else), которые протестированы.
  * **Покрытие функций (Function Coverage)**: Процент функций, которые были вызваны во время тестирования.

В этом примере:

  * `main.js` показывает 76,92% покрытия строк и 66,67% покрытия функций, потому что функция `multiply()` не была протестирована. Непокрытые строки (9-11) соответствуют этой функции.
  * `main.test.js` показывает 100% покрытия по всем метрикам, что указывает на то, что сами тесты были полностью выполнены.

## Включение и исключение

При работе над приложениями вы можете столкнуться с ситуациями, когда определённые файлы или строки кода нужно исключить.

Node.js предоставляет механизмы для этого, включая использование комментариев для игнорирования конкретных участков кода и CLI для исключения целых шаблонов.

### Использование комментариев
```js
function add(a, b) {
  return a + b;
}

function isEven(num) {
  return num % 2 === 0;
}

/* node:coverage ignore next 3 */
function multiply(a, b) {
  return a * b;
}

module.exports = { add, isEven, multiply };
```

При формировании отчёта о покрытии с этим изменённым файлом `main.js` отчёт теперь покажет 100% покрытия по всем метрикам. Это происходит потому, что непокрытые строки (9-11) были проигнорированы.

Существует несколько способов игнорировать участки кода с помощью комментариев.
```js
function add(a, b) {
  return a + b;
}

function isEven(num) {
  return num % 2 === 0;
}

/* node:coverage ignore next 3 */
function multiply(a, b) {
  return a * b;
}

module.exports = { add, isEven, multiply };
```

Каждый из этих разных методов даст один и тот же отчёт со 100% покрытием кода по всем метрикам.

### Использование CLI

Node.js предлагает два аргумента CLI для управления включением или исключением конкретных файлов в отчёте о покрытии.

Флаг [`--test-coverage-include`](https://nodejs.org/api/cli.html#--test-coverage-include) (`coverageIncludeGlobs` в API `run()`) ограничивает покрытие файлами, которые соответствуют предоставленному glob-шаблону. По умолчанию файлы в директории `/node_modules/` исключаются, но этот флаг позволяет явно включить их.

Флаг [`--test-coverage-exclude`](https://nodejs.org/api/cli.html#--test-coverage-exclude) (`coverageExcludeGlobs` в API `run()`) исключает из отчёта о покрытии файлы, соответствующие заданному glob-шаблону.

Эти флаги можно использовать несколько раз, и когда оба используются вместе, файлы должны соответствовать правилам включения, одновременно избегая правил исключения.
```text
.
├── main.test.js
├── src
│   ├── age.js
│   └── name.js
```

`src/age.js` имеет не самое оптимальное покрытие в приведённом выше отчёте, но с флагом `--test-coverage-exclude` (`coverageExcludeGlobs` в API `run()`) его можно полностью исключить из отчёта.
```
node --experimental-test-coverage --test-coverage-exclude=src/age.js --test main.test.js
```

Наш тестовый файл также включён в этот отчёт о покрытии, но нам нужны только файлы JavaScript в директории `src/`. В этом случае можно использовать флаг `--test-coverage-include` (`coverageIncludeGlobs` в API `run()`).
```
node --experimental-test-coverage --test-coverage-include=src/*.js --test main.test.js
```

## Пороговые значения

По умолчанию, когда все тесты проходят, Node.js завершается с кодом `0`, что указывает на успешное выполнение. Однако отчёт о покрытии можно настроить так, чтобы он завершался с кодом `1`, когда покрытие не соответствует требованиям.

В настоящее время Node.js поддерживает пороговые значения для всех трёх поддерживаемых видов покрытия:

  * [`--test-coverage-lines`](https://nodejs.org/api/cli.html#--test-coverage-linesthreshold) (`lineCoverage` в API `run()`) для покрытия строк.
  * [`--test-coverage-branches`](https://nodejs.org/api/cli.html#--test-coverage-branchesthreshold) (`branchCoverage` в API `run()`) для покрытия ветвей.
  * [`--test-coverage-functions`](https://nodejs.org/api/cli.html#--test-coverage-functionsthreshold) (`functionCoverage` в API `run()`) для покрытия функций.

Если бы вы хотели потребовать, чтобы в предыдущем примере покрытие строк было >= 90%, вы могли бы использовать флаг `--test-coverage-lines=90` (`lineCoverage: 90` в API `run()`).
```
node --experimental-test-coverage --test-coverage-lines=90 --test main.test.js
```

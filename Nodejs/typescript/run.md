# Запуск TypeScript с помощью средства запуска

Если вам нужна более продвинутая обработка TypeScript, чем встроенная поддержка (или вы используете Node.js до версии v22.7.0), у вас есть 2 варианта: использовать средство запуска (runner), которое берёт на себя большую часть сложностей, либо всё сделать самостоятельно через [транспиляцию](./transpile).

## Запуск кода на TypeScript с помощью `ts-node`

[ts-node](https://typestrong.org/ts-node/) — это среда выполнения TypeScript для Node.js. Она позволяет запускать код на TypeScript непосредственно в Node.js без необходимости предварительной компиляции. По умолчанию `ts-node` выполняет проверку типов, если не включён режим `transpileOnly`. Хотя `ts-node` может выявлять ошибки типов во время выполнения, мы всё же рекомендуем сначала проверить типы вашего кода с помощью `tsc`, прежде чем выпускать его.

Чтобы использовать `ts-node`, вам сначала нужно его установить:
```
npm i -D ts-node
```

Затем вы можете запустить свой код на TypeScript так:
```
npx ts-node example.ts
```

## Запуск кода на TypeScript с помощью `tsx`

[tsx](https://tsx.hirok.io/) — ещё одна среда выполнения TypeScript для Node.js. Она позволяет запускать код на TypeScript непосредственно в Node.js без необходимости предварительной компиляции. Обратите внимание, однако, что она не проверяет типы вашего кода. Поэтому мы рекомендуем сначала проверить типы вашего кода с помощью `tsc`, а затем запускать его с помощью `tsx`, прежде чем выпускать его.

Чтобы использовать `tsx`, вам сначала нужно его установить:
```
npm i -D tsx
```

Затем вы можете запустить свой код на TypeScript так:
```
npx tsx example.ts
```

### Регистрация `tsx` через `node`

Если вы хотите использовать `tsx` через `node`, вы можете зарегистрировать `tsx` с помощью `--import`:
```
node --import=tsx example.ts
```

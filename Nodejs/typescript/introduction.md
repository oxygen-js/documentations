# Введение в TypeScript

## Что такое TypeScript

**[TypeScript](https://www.typescriptlang.org)** — это язык с открытым исходным кодом, который сопровождается и разрабатывается компанией Microsoft.

По сути, TypeScript добавляет к JavaScript дополнительный синтаксис для поддержки более тесной интеграции с вашим редактором. Он позволяет выявлять ошибки на ранних этапах — прямо в редакторе или в конвейере CI/CD — и писать более удобный для сопровождения код.

О других преимуществах TypeScript мы поговорим позже, а сейчас давайте посмотрим на несколько примеров!

## Первый код на TypeScript

Взгляните на этот фрагмент кода, а затем мы вместе разберём его:
```ts
type User = {
  name: string;
  age: number;
};

function isAdult(user: User): boolean {
  return user.age >= 18;
}

const justine = {
  name: 'Justine',
  age: 23,
} satisfies User;

const isJustineAnAdult = isAdult(justine);
```

Первая часть (с ключевым словом `type`) отвечает за объявление нашего собственного типа объекта, представляющего пользователей. Позднее мы используем этот вновь созданный тип для создания функции `isAdult`, которая принимает один аргумент типа `User` и возвращает `boolean`. После этого мы создаём `justine` — наши примерные данные, которые можно использовать для вызова ранее определённой функции. Наконец, мы создаём новую переменную с информацией о том, является ли `justine` взрослой.

Есть ещё дополнительные вещи, касающиеся этого примера, которые вам следует знать. Во-первых, если мы не соблюдаем объявленные типы, TypeScript сообщит нам, что что-то не так, и предотвратит неправильное использование. Во-вторых, не всё обязательно должно быть типизировано явно — TypeScript выводит типы за нас. Например, переменная `isJustineAnAdult` имеет тип `boolean`, даже если мы не типизировали её явно, а `justine` была бы допустимым аргументом для нашей функции, хотя мы и не объявляли эту переменную как имеющую тип `User`.

## Из чего состоит TypeScript?

TypeScript состоит из двух основных компонентов: самого кода и определений типов.

### Код TypeScript

Код — это обычный JavaScript с дополнительным специфичным для TypeScript синтаксисом для аннотаций типов. Когда код TypeScript компилируется, все специфичные для TypeScript части удаляются, в результате чего получается чистый JavaScript, который может работать в любой среде. Например:
```ts
function greet(name: string) {
  console.log(`Hello, ${name}!`);
}
```

### Определения типов

Определения типов описывают структуру существующего кода JavaScript. Обычно они хранятся в файлах `.d.ts` и не содержат никакой реальной реализации — они лишь описывают типы. Эти определения необходимы для взаимодействия с JavaScript: код обычно распространяется не как TypeScript, а транспилируется в JavaScript, к которому прилагаются сопутствующие файлы определений типов.

Например, когда вы используете Node.js вместе с TypeScript, вам понадобятся определения типов для API Node.js. Они доступны через `@types/node`. Установите их с помощью:
```
npm add --save-dev @types/node
```

Эти определения типов позволяют TypeScript понимать API Node.js и обеспечивают надлежащую проверку типов и автодополнение, когда вы используете такие функции, как `fs.readFile` или `http.createServer`. Например:
```ts
// @errors: 2345
import { resolve } from 'node:path';

resolve(123, 456);
```

Многие популярные библиотеки JavaScript имеют определения типов, доступные в пространстве имён `@types`, которое сопровождается сообществом DefinitelyTyped. Это обеспечивает бесшовную интеграцию существующих библиотек JavaScript с проектами на TypeScript.

### Возможности преобразования

TypeScript также включает мощные возможности преобразования, особенно для JSX (используется в React и подобных фреймворках). Компилятор TypeScript может преобразовывать синтаксис JSX в обычный JavaScript, подобно тому, как это делает Babel. Хотя мы не будем рассматривать эти возможности преобразования в этих статьях, стоит отметить, что TypeScript — это не только инструмент для проверки типов, но и инструмент сборки для преобразования современного синтаксиса JavaScript в совместимые версии для различных сред.

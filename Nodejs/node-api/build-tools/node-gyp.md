# node-gyp

[node-gyp](https://github.com/nodejs/node-gyp) — это стандартное средство сборки для нативных аддонов Node.js, и оно используется подавляющим большинством пакетов в экосистеме npm. Оно активно сопровождается командой Node.js. Большинство примеров на этом сайте используют node-gyp для сборки бинарных файлов.

node-gyp основан на средстве сборки [GYP](https://gyp.gsrc.io/) от Google. GYP предоставляет единый кроссплатформенный формат конфигурации для сборок C/C++. Хотя Google заархивировал вышестоящий репозиторий GYP, node-gyp продолжает активно развиваться и сопровождаться через [gyp-next](https://github.com/nodejs/gyp-next).

> node-gyp требует **Python 3.6 или новее**. Python 2 не поддерживается. Полный список требований для каждой платформы можно найти в [документации по установке node-gyp](https://github.com/nodejs/node-gyp#installation).

node-gyp входит в состав npm; когда npm видит `"gypfile": true` в `package.json`, он автоматически вызывает node-gyp во время `npm install`. Вы также можете установить и использовать его напрямую:
```
npm install -g node-gyp
```

Разработчикам, которые находят node-gyp слишком ограничивающим, [CMake.js](/learn/node-api/build-tools/cmake-js.html) является хорошей альтернативой.

### Плюсы

  * Входит в состав npm — для потребителей не требуется отдельная глобальная установка.
  * Используется практически повсеместно в экосистеме Node.js, с обширной документацией и знаниями сообщества.
  * Поддерживает Windows, macOS и Linux из единого файла конфигурации `binding.gyp`.

### Минусы

  * Нижележащий формат GYP больше активно не развивается Google.
  * Некоторые разработчики находят синтаксис конфигурации GYP многословным или трудным для отладки.

# Инструменты сборки

Сборка нативного аддона Node.js требует компиляции исходного кода на C/C++ в бинарный файл `.node`, который Node.js может загрузить во время выполнения. В этом разделе рассматриваются доступные вам инструменты сборки и компромиссы между ними.

## Компиляция при установке vs. распространение предсобранных бинарников

Есть две широкие стратегии поставки нативного аддона:

**Компиляция на машине пользователя** \- когда пользователь запускает `npm install`, инструмент сборки компилирует ваш исходный код на C/C++ в его системе. Это просто настроить, но требует, чтобы у каждого пользователя был установлен работающий инструментарий C/C++.

**Распространение предсобранных бинарников** \- вы заранее компилируете бинарники для каждой поддерживаемой платформы и архитектуры и загружаете их куда-то, откуда пользователи могут их скачать. Пользователи, скачавшие подходящий бинарник, полностью пропускают этап компиляции; остальные откатываются к локальной компиляции.

## Инструменты сборки, рассматриваемые в этом разделе

  * [node-gyp](<01. node-gyp.md>) \- инструмент сборки по умолчанию, поставляемый вместе с npm; использует формат GYP от Google и поддерживается практически повсеместно в экосистеме Node
  * [CMake.js](<02. CMake.js.md>) \- альтернатива на основе CMake, хорошо подходящая для проектов, которые уже используют CMake
  * [node-pre-gyp](<03. node-pre-gyp.md>) \- слой поверх node-gyp для распространения предсобранных бинарников через Amazon S3

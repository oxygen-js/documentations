# Node-API

[Node-API](https://nodejs.org/api/n-api.html) — это стабильный C API, встроенный в Node.js, который позволяет коду на C/C++ создавать, читать и обрабатывать значения JavaScript так, как если бы они были созданы самим JavaScript. Он был представлен экспериментально в Node.js 8.0.0 и стал стабильным (больше не за флагом) в период Node.js 8/10.

Поскольку Node-API является частью самого Node.js, он не требует никакой дополнительной установки.

## Почему Node-API вместо NAN?

До появления Node-API доминирующим подходом к разработке нативных аддонов был [Native Abstractions for Node.js (NAN)](https://github.com/nodejs/nan). NAN работает путём прямых вызовов [V8](https://developers.google.com/v8/) API, который меняется с каждым мажорным выпуском V8. Это означает, что аддоны на основе NAN часто нуждаются в перекомпиляции — а иногда и в обновлении — когда Node.js выпускается с новой версией V8.

Node-API абстрагирует движок JavaScript за стабильным интерфейсом. Это даёт вам две важные гарантии:

  1. **Обратная совместимость** — модуль, собранный сегодня, будет работать во всех будущих версиях Node.js без какой-либо перекомпиляции или изменений кода.
  2. **Стабильность ABI** — поскольку API является ABI-стабильным, сам скомпилированный бинарный файл `.node` продолжает работать в разных выпусках Node.js.

## Распространённые сценарии использования

### Обёртывание существующей библиотеки C/C++

Наиболее распространённая причина написания нативного аддона — предоставить существующую библиотеку C/C++ разработчикам на JavaScript. Node-API позволяет вам поддерживать ваш код на C/C++ независимо и синхронизировать связки JavaScript с минимальными усилиями.

### Доступ к ресурсам ОС

Некоторым приложениям — например, построенным на [Electron](https://electronjs.org) или [NW.js](https://nwjs.io) — нужен доступ к системным API, которые Node.js не предоставляет. Node-API обеспечивает механизмы для доступа к этим ресурсам.

### Вычислительно интенсивная работа

Для задач с большим объёмом вычислений написание горячего участка (hot path) на C или C++ и вызов его из JavaScript через Node-API может дать значительный прирост производительности. В отличие от JIT-компилируемого JavaScript, скомпилированный бинарный файл C/C++ доступен Node.js сразу же, без фазы прогрева.

## `node-addon-api`

Важным дополнением к Node-API является npm-пакет [`node-addon-api`](https://www.npmjs.com/package/node-addon-api). Он оборачивает C API в идиоматичную объектную модель C++, уменьшая объём шаблонного кода и делая распространённые паттерны — такие как обёртывание объектов C++ в объекты JavaScript — гораздо более эргономичными. Он сохраняет полную гарантию ABI-стабильности Node-API.

Большинство примеров на этом сайте используют `node-addon-api`.

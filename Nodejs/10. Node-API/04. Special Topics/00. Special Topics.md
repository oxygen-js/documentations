# Специальные темы

Этот раздел охватывает продвинутые паттерны Node-API, с которыми вы столкнётесь, как только ваш нативный аддон вырастет за пределы простого синхронного вызова функции.

## Рассматриваемые темы

  * [Ссылки на объекты и функции](/learn/node-api/special-topics/object-function-refs.html) \- использование `ObjectReference` и `FunctionReference` для поддержания JavaScript-объектов «живыми» через границы вызовов, чтобы сборщик мусора не освободил их преждевременно
  * [AsyncWorker](/learn/node-api/special-topics/asyncworker.html) \- выполнение долго работающих операций на C/C++ в фоновом потоке выполнения с помощью класса `AsyncWorker` из `node-addon-api`, что не блокирует цикл событий Node
  * [Потокобезопасные функции](/learn/node-api/special-topics/thread-safe-functions.html) \- обратные вызовы в JavaScript из нативных потоков выполнения, не являющихся основным потоком Node.js, с помощью API потокобезопасных функций
  * [Осведомлённость о контексте](/learn/node-api/special-topics/context-awareness.html) \- написание аддонов, которые корректно загружаются и выгружаются при наличии [рабочих потоков (Worker Threads)](https://nodejs.org/api/worker_threads.html), включая данные экземпляра и хуки очистки

## Когда применяются эти темы?

Тема| Когда это нужно| Ссылки на объекты/функции| Хранение JS-колбэка или объекта в члене данных C++, который переживёт текущий вызов| AsyncWorker| Вынос CPU-нагруженной или блокирующей работы с I/O за пределы основного потока выполнения| Потокобезопасные функции| Вызов JS-колбэков из нативных потоков выполнения, которыми вы управляете сами| Осведомлённость о контексте| Ваш аддон используется в окружениях с рабочими потоками (Worker Thread) или загружается/выгружается несколько раз  
---|---

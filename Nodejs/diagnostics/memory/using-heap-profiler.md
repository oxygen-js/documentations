# Использование профилировщика кучи

Профилировщик кучи работает поверх V8 и захватывает выделения памяти во времени. В этом документе мы рассмотрим профилирование памяти с помощью:

  1. Временной шкалы выделений (Allocation Timeline)
  2. Выборочного профилировщика кучи (Sampling Heap Profiler)

В отличие от снимков кучи (heap dumps), которые рассматривались в руководстве [Использование снимка кучи](/diagnostics/memory/using-heap-snapshot/), идея использования профилирования в реальном времени состоит в том, чтобы понять выделения памяти на протяжении некоторого периода времени.

## Профилировщик кучи — временная шкала выделений

Профилировщик кучи похож на выборочный профилировщик кучи, за исключением того, что он отслеживает каждое выделение памяти. У него более высокие накладные расходы, чем у выборочного профилировщика кучи, поэтому его не рекомендуется использовать в продакшене.

> Вы можете использовать [@mmarchini/observe](https://www.npmjs.com/package/@mmarchini/observe), чтобы запускать и останавливать профилировщик программно.

### Как это сделать

Запустите приложение:
```
node --inspect index.js
```

> Для скриптов лучше подойдёт `--inspect-brk`.

Подключитесь к экземпляру dev-tools в Chrome, а затем:

  * Выберите вкладку `Memory`.
  * Выберите `Allocation instrumentation timeline`.
  * Запустите профилирование.

![шаг 1 руководства по профилировщику кучи][heap profiler tutorial 1]

Как только профилирование кучи запущено, настоятельно рекомендуется прогнать образцы нагрузки, чтобы выявить проблемы с памятью. Например, если бы мы профилировали кучу веб-приложения, мы могли бы использовать `Apache Benchmark` для создания нагрузки:
```
$ ab -n 1000 -c 5 http://localhost:3000
```

Затем, когда нагрузка завершится, нажмите кнопку остановки:

![шаг 2 руководства по профилировщику кучи][heap profiler tutorial 2]

Наконец, изучите данные снимка:

![шаг 3 руководства по профилировщику кучи][heap profiler tutorial 3]

Загляните в раздел с полезными ссылками, чтобы получить дополнительную информацию о терминологии, связанной с памятью.

## Выборочный профилировщик кучи

Выборочный профилировщик кучи отслеживает паттерн выделения памяти и зарезервированное пространство во времени. Поскольку он основан на выборке (sampling), его накладные расходы достаточно низки, чтобы использовать его в продакшн-системах.

> Вы можете использовать модуль [`heap-profiler`](https://www.npmjs.com/package/heap-profiler), чтобы запускать и останавливать профилировщик кучи программно.

### Как это сделать

Запустите приложение:
```
$ node --inspect index.js
```

> Для скриптов лучше подойдёт `--inspect-brk`.

Подключитесь к экземпляру dev-tools, а затем:

  1. Выберите вкладку `Memory`.
  2. Выберите `Allocation sampling`.
  3. Запустите профилирование.

![шаг 4 руководства по профилировщику кучи][heap profiler tutorial 4]

Создайте некоторую нагрузку и остановите профилировщик. Он сгенерирует сводку с выделениями памяти на основе их стектрейсов. Вы можете сосредоточиться на функциях с наибольшим количеством выделений в куче, см. пример ниже:

![шаг 5 руководства по профилировщику кучи][heap profiler tutorial 5]

# Использование снимка кучи

Вы можете снять снимок кучи (Heap Snapshot) с работающего приложения и загрузить его в [Chrome Developer Tools](https://developer.chrome.com/docs/devtools/), чтобы изучить отдельные переменные или проверить размер удерживаемой памяти (retainer size). Также можно сравнивать несколько снимков, чтобы увидеть изменения с течением времени.

## Предупреждение

При создании снимка вся остальная работа в основном потоке выполнения останавливается. В зависимости от содержимого кучи это может занять даже больше минуты. Снимок строится в памяти, поэтому он может удвоить размер кучи, что приведёт к заполнению всей памяти и последующему аварийному завершению приложения.

Если вы собираетесь снимать снимок кучи в продакшене, убедитесь, что процесс, с которого вы его снимаете, может аварийно завершиться без влияния на доступность вашего приложения.

## Как это сделать

### Получение снимка кучи

Существует несколько способов получить снимок кучи:

  1. через инспектор,
  2. через внешний сигнал и флаг командной строки,
  3. через вызов `writeHeapSnapshot` внутри процесса,
  4. через протокол инспектора.

#### 1\. Профилирование памяти в инспекторе

> Работает во всех активно поддерживаемых версиях Node.js

Запустите node с флагом `--inspect` и откройте инспектор. ![открытие инспектора][open inspector image]

Простейший способ получить снимок кучи — подключить инспектор к процессу, работающему локально. Затем перейдите на вкладку Memory и сделайте снимок кучи.

![создание снимка кучи][take a heap snapshot image]

#### 2\. Использование флага `--heapsnapshot-signal`

> Работает в v12.0.0 и новее

Вы можете запустить node с флагом командной строки, включающим реакцию на сигнал для создания снимка кучи.
```
$ node --heapsnapshot-signal=SIGUSR2 index.js
```

Подробности смотрите в актуальной документации по [флагу heapsnapshot-signal](https://nodejs.org/api/cli.html#--heapsnapshot-signalsignal).

#### 3\. Использование функции `writeHeapSnapshot`

> Работает в v11.13.0 и новее. В более старых версиях может работать с [пакетом heapdump](https://www.npmjs.com/package/heapdump)

Если вам нужен снимок с работающего процесса, например приложения, запущенного на сервере, вы можете реализовать его получение так:
```
require('node:v8').writeHeapSnapshot();
```

Ознакомьтесь с [документацией `writeHeapSnapshot`](https://nodejs.org/api/v8.html#v8writeheapsnapshotfilenameoptions), чтобы узнать о параметрах имени файла.

Вам нужен способ вызвать её без остановки процесса, поэтому рекомендуется вызывать её в HTTP-обработчике или в ответ на сигнал от операционной системы. Будьте осторожны и не открывайте наружу HTTP-эндпоинт, запускающий создание снимка. Никто посторонний не должен иметь к нему доступа.

Для версий Node.js до v11.13.0 можно использовать [пакет heapdump](https://www.npmjs.com/package/heapdump).

#### 4\. Запуск создания снимка кучи через протокол инспектора

Протокол инспектора можно использовать для запуска создания снимка кучи извне процесса.

Для использования этого API не обязательно запускать сам инспектор из Chromium.

Вот пример запуска создания снимка на bash с использованием `websocat` и `jq`:
```bash
#!/bin/bash
set -e

kill -USR1 "$1"
rm -f fifo out
mkfifo ./fifo
websocat -B 10000000000 "$(curl -s http://localhost:9229/json | jq -r '.[0].webSocketDebuggerUrl')" < ./fifo > ./out &
exec 3>./fifo
echo '{"method": "HeapProfiler.enable", "id": 1}' > ./fifo
echo '{"method": "HeapProfiler.takeHeapSnapshot", "id": 2}' > ./fifo
while jq -e "[.id != 2, .result != {}] | all" < <(tail -n 1 ./out); do
  sleep 1s
  echo "Capturing Heap Snapshot..."
done

echo -n "" > ./out.heapsnapshot
while read -r line; do
  f="$(echo "$line" | jq -r '.params.chunk')"
  echo -n "$f" >> out.heapsnapshot
  i=$((i+1))
done < <(cat out | tail -n +2 | head -n -1)

exec 3>&-
```

Ниже приведён неполный список инструментов профилирования памяти, которые можно использовать с протоколом инспектора:

  * [OpenProfiling for Node.js](https://github.com/vmarchaud/openprofiling-node)

## Как найти утечку памяти с помощью снимков кучи

Утечку памяти можно найти, сравнив два снимка. Важно убедиться, что разница между снимками не содержит лишней информации. Следующие шаги должны дать чистую разницу между снимками.

  1. Дайте процессу загрузить все исходники и завершить начальную инициализацию (bootstrapping). Это должно занять не более нескольких секунд.
  2. Начните использовать функциональность, которую вы подозреваете в утечке памяти. Вероятно, она делает какие-то первоначальные выделения памяти, не относящиеся к утечке.
  3. Сделайте один снимок кучи.
  4. Продолжайте использовать эту функциональность некоторое время, желательно не запуская между этим ничего другого.
  5. Сделайте ещё один снимок кучи. Разница между двумя снимками в основном будет содержать то, что утекало.
  6. Откройте инструменты разработчика Chromium/Chrome и перейдите на вкладку _Memory_
  7. Сначала загрузите файл более старого снимка, затем более нового. ![кнопка загрузки в инструментах][load button image]
  8. Выберите более новый снимок и переключите режим в выпадающем списке сверху с _Summary_ на _Comparison_. ![выпадающий список Comparison][comparison image]
  9. Ищите большие положительные дельты и исследуйте вызвавшие их ссылки на нижней панели.

Попрактиковаться в создании снимков кучи и поиске утечек памяти можно с помощью [этого упражнения по снимкам кучи](https://github.com/naugtur/node-example-heapdump).

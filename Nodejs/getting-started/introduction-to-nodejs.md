# Введение в Node.js

Node.js — это среда выполнения JavaScript с открытым исходным кодом, работающая на разных платформах. Это популярный инструмент почти для любого рода проектов!

Node.js запускает JavaScript-движок V8 — ядро Google Chrome — вне браузера. Это позволяет Node.js быть очень производительным.

Приложение Node.js работает в едином процессе, не создавая новый поток выполнения на каждый запрос. Node.js предоставляет набор асинхронных примитивов ввода-вывода в своей стандартной библиотеке, которые не дают JavaScript-коду блокироваться. Кроме того, библиотеки в Node.js обычно пишутся с использованием неблокирующих парадигм. Соответственно, блокирующее поведение в Node.js — скорее исключение, чем норма.

Когда Node.js выполняет операцию ввода-вывода, такую как чтение из сети, обращение к базе данных или файловой системе, вместо блокировки потока выполнения и напрасной траты циклов CPU на ожидание, Node.js возобновит операции, когда придёт ответ.

Это позволяет Node.js обрабатывать тысячи одновременных соединений с помощью одного сервера, не создавая при этом бремени управления конкурентностью потоков, которое могло бы стать значительным источником ошибок.

У Node.js есть уникальное преимущество, ведь миллионы фронтенд-разработчиков, пишущих JavaScript для браузера, теперь могут писать серверный код в дополнение к клиентскому без необходимости изучать совершенно другой язык.

В Node.js новые стандарты ECMAScript можно использовать без проблем, так как вам не нужно ждать, пока все ваши пользователи обновят свои браузеры — вы сами решаете, какую версию ECMAScript использовать, меняя версию Node.js, и можете также включать отдельные экспериментальные возможности, запуская Node.js с флагами.

## Пример приложения Node.js

Самый распространённый пример Hello World для Node.js — это веб-сервер:
```js
const { createServer } = require('node:http');

const hostname = '127.0.0.1';
const port = 3000;

const server = createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

Чтобы запустить этот фрагмент, сохраните его как файл `server.js` и выполните `node server.js` в терминале. Если вы используете mjs-версию кода, сохраните её как файл `server.mjs` и выполните `node server.mjs` в терминале.

Этот код сначала подключает [модуль `http`](https://nodejs.org/api/http.html) Node.js.

У Node.js есть замечательная [стандартная библиотека](https://nodejs.org/api/), включая первоклассную поддержку работы с сетью.

Метод `createServer()` модуля `http` создаёт новый HTTP-сервер и возвращает его.

Сервер настроен на прослушивание указанного порта и имени хоста. Когда сервер готов, вызывается колбэк-функция, в данном случае информирующая нас о том, что сервер работает.

Всякий раз, когда поступает новый запрос, вызывается [событие `request`](https://nodejs.org/api/http.html#event-request), предоставляющее два объекта: запрос (объект [`http.IncomingMessage`](https://nodejs.org/api/http.html#class-httpincomingmessage)) и ответ (объект [`http.ServerResponse`](https://nodejs.org/api/http.html#class-httpserverresponse)).

Эти два объекта необходимы для обработки HTTP-вызова.

Первый предоставляет детали запроса. В этом простом примере он не используется, но вы могли бы получить доступ к заголовкам запроса и данным запроса.

Второй используется для возврата данных вызывающей стороне.

В данном случае с помощью:
```
res.statusCode = 200;
```

мы устанавливаем свойство `statusCode` в `200`, чтобы обозначить успешный ответ.

Мы устанавливаем заголовок `Content-Type`:
```
res.setHeader('Content-Type', 'text/plain');
```

и закрываем ответ, добавляя содержимое в качестве аргумента к `end()`:
```
res.end('Hello World');
```

Если вы ещё этого не сделали, [загрузите](https://nodejs.org/en/download) Node.js.

# ECMAScript 2015 (ES6) и далее

Node.js собирается на основе современных версий [V8](https://v8.dev/). Поддерживая актуальность с последними выпусками этого движка, мы обеспечиваем своевременное появление у разработчиков Node.js новых возможностей из [спецификации JavaScript ECMA-262](http://www.ecma-international.org/publications/standards/Ecma-262.htm), а также постоянные улучшения производительности и стабильности.

Все возможности ECMAScript 2015 (ES6) разделены на три группы — **поставляемые** (shipping), **подготавливаемые** (staged) и **в разработке** (in progress):

  * Все **поставляемые** возможности, которые V8 считает стабильными, **включены по умолчанию в Node.js** и **НЕ** требуют никаких флагов среды выполнения.
  * **Подготавливаемые** возможности, которые почти завершены, но не считаются стабильными командой V8, требуют флага среды выполнения: `--harmony`.
  * Возможности **в разработке** можно активировать по отдельности с помощью соответствующего им harmony-флага, хотя это крайне не рекомендуется, кроме как для целей тестирования. Примечание: эти флаги предоставляются V8 и потенциально могут измениться без какого-либо уведомления об устаревании.

### Какие возможности поставляются с какой версией Node.js по умолчанию?

Веб-сайт [node.green](https://node.green/) предоставляет отличный обзор поддерживаемых возможностей ECMAScript в различных версиях Node.js, основанный на compat-table от kangax.

### Какие возможности находятся в разработке?

В движок V8 постоянно добавляются новые возможности. В общем случае, ожидайте, что они появятся в будущем выпуске Node.js, хотя сроки неизвестны.

Вы можете вывести список всех возможностей _в разработке_, доступных в каждом выпуске Node.js, отфильтровав grep'ом вывод аргумента `--v8-options`. Пожалуйста, учтите, что это неполные и, возможно, неработающие возможности V8, поэтому используйте их на свой страх и риск:
```
node --v8-options | grep "in progress"
```

### У меня инфраструктура настроена на использование флага --harmony. Стоит ли мне его убрать?

Текущее поведение флага `--harmony` в Node.js — включать только **подготавливаемые** возможности. В конце концов, теперь это синоним `--es_staging`. Как упоминалось выше, это завершённые возможности, которые пока не считаются стабильными. Если вы хотите играть наверняка, особенно в продакшн-окружениях, рассмотрите возможность удаления этого флага среды выполнения, пока эти возможности не начнут поставляться по умолчанию в V8 и, следовательно, в Node.js. Если вы оставите его включённым, вам следует быть готовыми к тому, что последующие обновления Node.js сломают ваш код, если V8 изменит их семантику для более точного соответствия стандарту.

### Как узнать, какая версия V8 поставляется с конкретной версией Node.js?

Node.js предоставляет простой способ вывести список всех зависимостей и соответствующих версий, которые поставляются с конкретным бинарным файлом, через глобальный объект `process`. В случае движка V8 наберите в терминале следующее, чтобы получить его версию:
```
node -p process.versions.v8
```

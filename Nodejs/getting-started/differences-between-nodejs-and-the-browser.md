# Различия между Node.js и браузером

И браузер, и Node.js используют JavaScript в качестве языка программирования. Создание приложений, работающих в браузере, полностью отличается от создания приложения на Node.js. Несмотря на то, что это всегда JavaScript, есть некоторые ключевые различия, которые делают этот опыт радикально разным.

С точки зрения фронтенд-разработчика, который активно использует JavaScript, приложения на Node.js несут с собой огромное преимущество: удобство программирования всего — и фронтенда, и бэкенда — на одном языке.

У вас есть огромная возможность, потому что мы знаем, как трудно полностью и глубоко изучить язык программирования, и, используя один и тот же язык для выполнения всей вашей работы в вебе — как на клиенте, так и на сервере, — вы оказываетесь в уникально выгодном положении.

> **Что меняется, так это экосистема.**

В браузере большую часть времени вы взаимодействуете с DOM или другими API веб-платформы, такими как Cookies. Их, конечно же, не существует в Node.js. У вас нет `document`, `window` и всех остальных объектов, которые предоставляются браузером.

А в браузере у нас нет всех тех замечательных API, которые Node.js предоставляет через свои модули, например, функциональности доступа к файловой системе.

Ещё одно большое различие в том, что в Node.js вы контролируете окружение. Если только вы не создаёте приложение с открытым исходным кодом, которое кто угодно может развернуть где угодно, вы знаете, на какой версии Node.js будете запускать приложение. По сравнению с окружением браузера, где у вас нет роскоши выбирать, каким браузером будут пользоваться ваши посетители, это очень удобно.

Это означает, что вы можете писать весь современный JavaScript уровня ES2015+, который поддерживает ваша версия Node.js. Поскольку JavaScript развивается очень быстро, но браузеры бывают несколько медлительны в обновлении, иногда в вебе вы вынуждены использовать более старые выпуски JavaScript / ECMAScript. Вы можете использовать Babel, чтобы преобразовать ваш код в совместимый с ES5 перед отправкой его в браузер, но в Node.js вам это не понадобится.

Ещё одно различие в том, что Node.js поддерживает как систему модулей CommonJS, так и ES-модули (начиная с Node.js v12), тогда как в браузере мы только начинаем видеть внедрение стандарта ES-модулей.

На практике это означает, что в Node.js вы можете использовать как `require()`, так и `import`, тогда как в браузере вы ограничены только `import`.

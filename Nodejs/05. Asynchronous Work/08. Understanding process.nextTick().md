# Понимание process.nextTick()

Разбираясь в цикле событий Node.js, важно обратить внимание на одну из его составляющих — `process.nextTick()`. Каждый раз, когда среда выполнения возвращает управление в JavaScript для обработки события, мы называем это тиком (tick).

Когда мы передаём функцию в `process.nextTick()`, мы планируем её выполнение сразу после того, как завершится текущий стек вызовов, но до того, как цикл событий продолжит работу, и до того, как будут обработаны любые другие поставленные в очередь задачи или фазы:
```js
process.nextTick(() => {
  // сделать что-нибудь
});
```

Цикл событий занят выполнением кода текущей функции. Когда эта операция завершается, движок JS выполняет все функции, переданные в вызовы `nextTick` во время этой операции.

Это способ сказать движку JS обработать функцию асинхронно (после текущей функции), но как можно скорее, а не ставить её в общую очередь.

Вызов `setTimeout(() => {}, 0)` выполнит функцию в конце следующего тика — значительно позже, чем при использовании `nextTick()`, который отдаёт вызову приоритет и выполняет его непосредственно перед началом следующего тика.

Используйте `nextTick()`, когда хотите гарантировать, что к следующей итерации цикла событий этот код уже будет выполнен.

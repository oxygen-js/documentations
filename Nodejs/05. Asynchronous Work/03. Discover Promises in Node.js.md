# Знакомство с промисами в Node.js

**Promise** (промис) — это специальный объект в JavaScript, который представляет собой будущее завершение (или неудачу) асинхронной операции и её результирующее значение. По сути, промис — это заполнитель для значения, которое пока недоступно, но станет доступным в будущем.

Представьте промис как заказ пиццы: вы не получаете её сразу, но курьер обещает привезти её вам позже. Вы не знаете _точно_, когда, но знаете, что итог будет либо «пицца доставлена», либо «что-то пошло не так».

## Состояния промиса

Промис может находиться в одном из трёх состояний:

  * **Pending (ожидание)** : начальное состояние, когда асинхронная операция всё ещё выполняется.
  * **Fulfilled (выполнен)** : операция завершилась успешно, и промис теперь разрешён с некоторым значением.
  * **Rejected (отклонён)** : операция завершилась неудачей, и промис улажен с причиной (обычно это ошибка).

Когда вы заказываете пиццу, вы находитесь в состоянии ожидания — голодные и полные надежд. Если пицца приезжает горячей и сырной, вы перешли в состояние «выполнен». Но если из ресторана звонят и сообщают, что уронили вашу пиццу на пол, вы оказываетесь в состоянии «отклонён».

Независимо от того, заканчивается ли ваш ужин радостью или разочарованием, как только появляется окончательный итог, промис считается **улаженным (settled)**.

## Базовый синтаксис промиса

Один из самых распространённых способов создать промис — использовать конструктор `new Promise()`. Конструктор принимает функцию с двумя параметрами: `resolve` и `reject`. Эти функции используются, чтобы перевести промис из состояния **pending** в **fulfilled** или **rejected**.

Если внутри функции-исполнителя выбрасывается ошибка, промис будет отклонён с этой ошибкой. Возвращаемое значение функции-исполнителя игнорируется: для улаживания промиса следует использовать только `resolve` или `reject`.
```js
const myPromise = new Promise((resolve, reject) => {
  const success = true;

  if (success) {
    resolve('Operation was successful!');
  } else {
    reject('Something went wrong.');
  }
});
```

В приведённом выше примере:

  * Если условие `success` равно `true`, промис выполняется, и значение `'Operation was successful!'` передаётся в функцию `resolve`.
  * Если условие `success` равно `false`, промис отклоняется, и ошибка `'Something went wrong.'` передаётся в функцию `reject`.

## Обработка промисов с помощью `.then()`, `.catch()` и `.finally()`

После того как промис создан, вы можете обработать его исход с помощью методов `.then()`, `.catch()` и `.finally()`.

  * `.then()` используется для обработки выполненного промиса и доступа к его результату.
  * `.catch()` используется для обработки отклонённого промиса и перехвата любых возникших ошибок.
  * `.finally()` используется для обработки улаженного промиса, независимо от того, был ли он разрешён или отклонён.

```js
const myPromise = new Promise((resolve, reject) => {
  const success = true;

  if (success) {
    resolve('Operation was successful!');
  } else {
    reject('Something went wrong.');
  }
});

myPromise
  .then(result => {
    console.log(result); // выполнится, если промис разрешён
  })
  .catch(error => {
    console.error(error); // выполнится, если промис отклонён
  })
  .finally(() => {
    console.log('The promise has completed'); // выполнится, когда промис будет улажен
  });
```

## Цепочки промисов

Одна из замечательных возможностей промисов состоит в том, что они позволяют объединять несколько асинхронных операций в цепочку. Когда вы связываете промисы в цепочку, каждый блок `.then()` ждёт завершения предыдущего, прежде чем выполниться.
```js
const { setTimeout: delay } = require('node:timers/promises');

const promise = delay(1000).then(() => 'First task completed');

promise
  .then(result => {
    console.log(result); // 'First task completed'
    return delay(1000).then(() => 'Second task completed'); // Возвращаем второй промис
  })
  .then(result => {
    console.log(result); // 'Second task completed'
  })
  .catch(error => {
    console.error(error); // Если какой-либо промис отклонён, перехватываем ошибку
  });
```

## Использование async/await с промисами

Один из лучших способов работы с промисами в современном JavaScript — использование **async/await**. Это позволяет писать асинхронный код, который выглядит как синхронный, что значительно облегчает его чтение и сопровождение.

  * `async` используется для определения функции, которая возвращает промис.
  * `await` используется внутри `async`-функции, чтобы приостановить выполнение до тех пор, пока промис не будет улажен.

```js
async function performTasks() {
  try {
    const result1 = await promise1;
    console.log(result1); // 'First task completed'

    const result2 = await promise2;
    console.log(result2); // 'Second task completed'
  } catch (error) {
    console.error(error); // Перехватывает любое отклонение или ошибку
  }
}

performTasks();
```

В функции `performTasks` ключевое слово `await` гарантирует, что каждый промис будет улажен, прежде чем перейти к следующему оператору. Это приводит к более линейному и читаемому потоку асинхронного кода.

По сути, приведённый выше код выполнится так же, как если бы пользователь написал:
```js
promise1
  .then(function (result1) {
    console.log(result1);
    return promise2;
  })
  .then(function (result2) {
    console.log(result2);
  })
  .catch(function (error) {
    console.log(error);
  });
```

### Await на верхнем уровне (Top-Level Await)

При использовании [модулей ECMAScript](https://nodejs.org/api/esm.html) сам модуль рассматривается как область видимости верхнего уровня, которая нативно поддерживает асинхронные операции. Это означает, что вы можете использовать [`await` на верхнем уровне](https://nodejs.org/api/esm.html#top-level-await) без необходимости в `async`-функции.
```js
import { setTimeout as delay } from 'node:timers/promises';

await delay(1000);
```

Async/await может быть гораздо более запутанным, чем приведённые простые примеры. Джеймс Снелл (James Snell), член Технического руководящего комитета Node.js, подготовил [подробную презентацию](https://www.youtube.com/watch?v=XV-u_Ow47s0), в которой исследуются тонкости промисов и async/await.

## API Node.js на основе промисов

Node.js предоставляет **версии на основе промисов** для многих своих встроенных API, особенно в тех случаях, где асинхронные операции традиционно обрабатывались с помощью колбэков. Это облегчает работу с API Node.js и промисами и снижает риск «ада колбэков» (callback hell).

Например, у модуля `fs` (файловая система) есть API на основе промисов в `fs.promises`:
```js
const fs = require('node:fs').promises;
// Или вы можете импортировать промисифицированную версию напрямую:
// const fs = require('node:fs/promises');

async function readFile() {
  try {
    const data = await fs.readFile('example.txt', 'utf8');
    console.log(data);
  } catch (err) {
    console.error('Error reading file:', err);
  }
}

readFile();
```

В этом примере `fs.readFile()` возвращает промис, который мы обрабатываем с помощью синтаксиса `async/await`, чтобы асинхронно прочитать содержимое файла.

## Продвинутые методы промисов

Глобальный объект `Promise` в JavaScript предоставляет несколько мощных методов, которые помогают эффективнее управлять множеством асинхронных задач:

M

### Promise.all
```
Promise.all(): void
```

Этот метод принимает массив промисов и возвращает новый промис, который разрешается после того, как все промисы будут выполнены. Если хотя бы один промис отклоняется, `Promise.all()` немедленно отклонится. Однако, даже если произошло отклонение, промисы продолжают выполняться. При обработке большого числа промисов, особенно в пакетной обработке, использование этой функции может нагрузить память системы.
```js
const { setTimeout: delay } = require('node:timers/promises');

const fetchData1 = delay(1000).then(() => 'Data from API 1');
const fetchData2 = delay(2000).then(() => 'Data from API 2');

Promise.all([fetchData1, fetchData2])
  .then(results => {
    console.log(results); // ['Data from API 1', 'Data from API 2']
  })
  .catch(error => {
    console.error('Error:', error);
  });
```

M

### Promise.allSettled
```
Promise.allSettled(): void
```

Этот метод ждёт, пока все промисы разрешатся или отклонятся, и возвращает массив объектов, описывающих исход каждого промиса.
```js
const promise1 = Promise.resolve('Success');
const promise2 = Promise.reject('Failed');

Promise.allSettled([promise1, promise2]).then(results => {
  console.log(results);
  // [ { status: 'fulfilled', value: 'Success' }, { status: 'rejected', reason: 'Failed' } ]
});
```

В отличие от `Promise.all()`, метод `Promise.allSettled()` не прерывается досрочно при неудаче. Он дожидается улаживания всех промисов, даже если некоторые из них отклоняются. Это обеспечивает более качественную обработку ошибок для пакетных операций, где вам может понадобиться узнать статус всех задач независимо от неудач.

M

### Promise.race
```
Promise.race(): void
```

Этот метод разрешается или отклоняется, как только уладится первый промис — независимо от того, разрешается он или отклоняется. Вне зависимости от того, какой промис уладится первым, все промисы выполняются полностью.
```js
const { setTimeout: delay } = require('node:timers/promises');

const task1 = delay(2000).then(() => 'Task 1 done');
const task2 = delay(1000).then(() => 'Task 2 done');

Promise.race([task1, task2]).then(result => {
  console.log(result); // 'Task 2 done' (поскольку task2 завершается первым)
});
```

M

### Promise.any
```
Promise.any(): void
```

Этот метод разрешается, как только разрешается один из промисов. Если все промисы отклоняются, он отклонится с ошибкой `AggregateError`.
```js
const { setTimeout: delay } = require('node:timers/promises');

const api1 = delay(2000).then(() => 'API 1 success');
const api2 = delay(1000).then(() => 'API 2 success');
const api3 = delay(1500).then(() => 'API 3 success');

Promise.any([api1, api2, api3])
  .then(result => {
    console.log(result); // 'API 2 success' (поскольку он разрешается первым)
  })
  .catch(error => {
    console.error('All promises rejected:', error);
  });
```

### `Promise.reject()` и `Promise.resolve()`

Эти методы создают отклонённый или разрешённый промис напрямую.
```js
Promise.resolve('Resolved immediately').then(result => {
  console.log(result); // 'Resolved immediately'
});
```

M

### Promise.try
```
Promise.try(): void
```

`Promise.try()` — это метод, который выполняет заданную функцию, синхронную или асинхронную, и оборачивает результат в промис. Если функция выбрасывает ошибку или возвращает отклонённый промис, `Promise.try()` вернёт отклонённый промис. Если функция завершается успешно, возвращаемый промис будет выполнен с её значением.

Это может быть особенно полезно для единообразного начала цепочек промисов, особенно при работе с кодом, который может синхронно выбрасывать ошибки.
```js
function mightThrow() {
  if (Math.random() > 0.5) {
    throw new Error('Oops, something went wrong!');
  }
  return 'Success!';
}

Promise.try(mightThrow)
  .then(result => {
    console.log('Result:', result);
  })
  .catch(err => {
    console.error('Caught error:', err.message);
  });
```

В этом примере `Promise.try()` гарантирует, что если `mightThrow()` выбросит ошибку, она будет перехвачена в блоке `.catch()`, что упрощает обработку как синхронных, так и асинхронных ошибок в одном месте.

M

### Promise.withResolvers
```
Promise.withResolvers(): void
```

Этот метод создаёт новый промис вместе со связанными с ним функциями resolve и reject и возвращает их в удобном объекте. Это используется, например, когда вам нужно создать промис, но разрешить или отклонить его позже извне функции-исполнителя.
```js
const { promise, resolve, reject } = Promise.withResolvers();

setTimeout(() => {
  resolve('Resolved successfully!');
}, 1000);

promise.then(value => {
  console.log('Success:', value);
});
```

В этом примере `Promise.withResolvers()` даёт вам полный контроль над тем, когда и как промис будет разрешён или отклонён, без необходимости определять функцию-исполнитель встроенно. Этот паттерн часто используется в событийно-ориентированном программировании, при работе с таймаутами или при интеграции с API, не основанными на промисах.

## Обработка ошибок в промисах

Обработка ошибок в промисах гарантирует, что ваше приложение будет вести себя корректно в непредвиденных ситуациях.

  * Вы можете использовать `.catch()`, чтобы обработать любые ошибки или отклонения, возникающие во время выполнения промисов.

```js
myPromise
  .then(result => console.log(result))
  .catch(error => console.error(error)) // Обрабатывает отклонение
  .finally(() => console.log('Promise completed')); // Выполняется независимо от разрешения промиса
```

  * Как вариант, при использовании `async/await` вы можете применить блок `try/catch`, чтобы перехватывать и обрабатывать ошибки.

```js
async function performTask() {
  try {
    const result = await myPromise;
    console.log(result);
  } catch (error) {
    console.error(error); // Обрабатывает любые ошибки
  } finally {
    // Этот код выполняется независимо от неудачи
    console.log('performTask() completed');
  }
}

performTask();
```

## Планирование задач в цикле событий

Помимо промисов, Node.js предоставляет несколько других механизмов планирования задач в цикле событий.

M

### queueMicrotask
```
queueMicrotask(): void
```

`queueMicrotask()` используется для планирования микрозадачи (microtask) — лёгкой задачи, которая выполняется после текущего исполняемого скрипта, но до любых других событий I/O или таймеров. Микрозадачи включают такие задачи, как разрешение промисов и другие асинхронные операции, которым отдаётся приоритет над обычными задачами.
```js
queueMicrotask(() => {
  console.log('Microtask is executed');
});

console.log('Synchronous task is executed');
```

В приведённом выше примере сообщение «Microtask is executed» будет выведено после «Synchronous task is executed», но до любых операций I/O, таких как таймеры.

M

### process.nextTick
```
process.nextTick(): void
```

`process.nextTick()` используется для планирования колбэка, который должен выполниться сразу после завершения текущей операции. Это полезно в ситуациях, когда вы хотите гарантировать, что колбэк выполнится как можно скорее, но всё же после текущего контекста выполнения.
```js
process.nextTick(() => {
  console.log('Next tick callback');
});

console.log('Synchronous task executed');
```

M

### setImmediate
```
setImmediate(): void
```

`setImmediate()` планирует колбэк для выполнения на фазе проверки (check) [цикла событий](<06. The Node.js Event Loop.md>) Node.js, которая идёт после фазы опроса (poll), где обрабатывается большинство колбэков I/O.
```js
setImmediate(() => {
  console.log('Immediate callback');
});

console.log('Synchronous task executed');
```

### Когда что использовать

  * Используйте `queueMicrotask()` для задач, которые должны выполниться сразу после текущего скрипта и до любых колбэков I/O или таймеров, — как правило, для разрешения промисов.
  * Используйте `process.nextTick()` для задач, которые должны выполниться до любых событий I/O; это часто полезно для отсрочки операций или синхронной обработки ошибок.
  * Используйте `setImmediate()` для задач, которые должны выполниться после фазы опроса (poll), когда большинство колбэков I/O уже обработано.

Поскольку эти задачи выполняются вне текущего синхронного потока, необработанные исключения внутри этих колбэков не будут перехвачены окружающими блоками `try/catch` и могут привести к аварийному завершению приложения, если ими не управлять надлежащим образом (например, прикрепляя `.catch()` к промисам или используя глобальные обработчики ошибок вроде `process.on('uncaughtException')`).

Больше информации о цикле событий и порядке выполнения различных фаз смотрите в связанной статье [Цикл событий Node.js](<06. The Node.js Event Loop.md>).

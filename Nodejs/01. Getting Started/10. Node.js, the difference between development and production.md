# Node.js: разница между разработкой и продакшеном

**В Node.js нет разницы между разработкой и продакшеном**, то есть нет каких-то особых настроек, которые нужно применить, чтобы Node.js работала в продакшн-конфигурации. Однако несколько библиотек в реестре npm распознают переменную `NODE_ENV` и по умолчанию выставляют ей значение `development`. Всегда запускайте вашу Node.js с установленным `NODE_ENV=production`.

Популярный способ конфигурирования вашего приложения — использование [методологии двенадцати факторов](https://12factor.net/).

## Почему NODE_ENV считается антипаттерном?

Окружение (environment) — это цифровая платформа или система, где инженеры могут создавать, тестировать, _развёртывать_ и сопровождать программные продукты. По соглашению существует четыре стадии, или типа, окружений, в которых запускается наше приложение:

  * Разработка (Development)
  * Тестирование (Testing)
  * Предпродакшн (Staging)
  * Продакшн (Production)

Фундаментальная проблема `NODE_ENV` проистекает из того, что разработчики связывают оптимизации и поведение ПО с окружением, в котором оно работает. Результат — код вроде такого:
```js
if (process.env.NODE_ENV === 'development') {
  // ...
}

if (process.env.NODE_ENV === 'production') {
  // ...
}

if (['production', 'staging'].includes(process.env.NODE_ENV)) {
  // ...
}
```

Хотя это может выглядеть безобидно, из-за этого продакшн- и предпродакшн-окружения (staging) становятся разными, что делает надёжное тестирование невозможным. Например, тест — а значит, и функциональность вашего продукта — может пройти, когда `NODE_ENV` установлен в `development`, но упасть, когда `NODE_ENV` установлен в `production`. Поэтому установка `NODE_ENV` в любое значение, кроме `production`, считается _антипаттерном_.

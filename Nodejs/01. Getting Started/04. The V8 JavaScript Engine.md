# Движок JavaScript V8

V8 — это название движка JavaScript, который используется в Google Chrome. Это то, что берёт наш JavaScript и выполняет его во время работы в Chrome.

V8 — это движок JavaScript, то есть он разбирает и выполняет код JavaScript. DOM и другие API веб-платформы (все вместе они составляют среду выполнения) предоставляются браузером.

Замечательно то, что движок JavaScript независим от браузера, в котором он размещён. Именно эта ключевая особенность сделала возможным появление Node.js. V8 был выбран в качестве движка, на котором работает Node.js, ещё в 2009 году, и по мере того как популярность Node.js взрывного росла, V8 стал движком, который теперь обеспечивает работу невероятного объёма серверного кода, написанного на JavaScript.

Экосистема Node.js огромна, и благодаря V8 он также обеспечивает работу настольных приложений — с такими проектами, как Electron.

## Другие движки JS

У других браузеров есть свои собственные движки JavaScript:

  * У Firefox — [**SpiderMonkey**](https://spidermonkey.dev)
  * У Safari — [**JavaScriptCore**](https://developer.apple.com/documentation/javascriptcore) (также называемый Nitro)
  * Edge изначально был основан на [**Chakra**](https://github.com/Microsoft/ChakraCore), но не так давно был [пересобран на основе Chromium](https://support.microsoft.com/en-us/help/4501095/download-the-new-microsoft-edge-based-on-chromium) и движка V8.

и существует множество других.

Все эти движки реализуют [стандарт ECMA ES-262](https://www.ecma-international.org/publications/standards/Ecma-262.htm), также называемый ECMAScript, — стандарт, используемый JavaScript.

## Погоня за производительностью

V8 написан на C++ и непрерывно совершенствуется. Он переносим и работает на Mac, Windows, Linux и нескольких других системах.

В этом введении в V8 мы опустим детали реализации V8: их можно найти на более авторитетных ресурсах (например, на [официальном сайте V8](https://v8.dev/)), и со временем они меняются, часто радикально.

V8 постоянно развивается, как и другие движки JavaScript, чтобы ускорить веб и экосистему Node.js.

В вебе уже много лет идёт гонка за производительность, и мы (как пользователи и разработчики) многое выигрываем от этой конкуренции, потому что год за годом получаем более быстрые и более оптимизированные машины.

## Компиляция

JavaScript обычно считается интерпретируемым языком, но современные движки JavaScript больше не просто интерпретируют JavaScript — они его компилируют.

Это происходит с 2009 года, когда компилятор JavaScript SpiderMonkey был добавлен в Firefox 3.5, и все подхватили эту идею.

Внутри V8 компилирует JavaScript с помощью **компиляции точно в срок (just-in-time, JIT)**, чтобы ускорить выполнение.

Это может показаться нелогичным, но с момента появления Google Maps в 2004 году JavaScript эволюционировал из языка, который обычно выполнял несколько десятков строк кода, до полноценных приложений с тысячами и сотнями тысяч строк, работающих в браузере.

Наши приложения теперь могут работать часами внутри браузера, а не быть лишь несколькими правилами валидации форм или простыми скриптами.

В этом _новом мире_ компиляция JavaScript имеет полный смысл, потому что, хотя подготовка JavaScript к работе может занять чуть больше времени, после её завершения он будет гораздо производительнее, чем чисто интерпретируемый код.

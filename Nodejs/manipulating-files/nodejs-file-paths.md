# Пути к файлам в Node.js

У каждого файла в системе есть путь. В Linux и macOS путь может выглядеть так: `/users/joe/file.txt`, тогда как компьютеры на Windows устроены иначе и имеют структуру вроде: `C:\users\joe\file.txt`

При использовании путей в своих приложениях нужно быть внимательным, так как это различие необходимо учитывать.

Этот модуль подключается в файлах с помощью `const path = require('node:path');`, после чего можно начинать пользоваться его методами.

## Извлечение информации из пути

Имея путь, вы можете извлечь из него информацию с помощью следующих методов:

  * `dirname`: получает родительскую папку файла
  * `basename`: получает часть с именем файла
  * `extname`: получает расширение файла

### Пример
```js
const path = require('node:path');

const notes = '/users/joe/notes.txt';

path.dirname(notes); // /users/joe
path.basename(notes); // notes.txt
path.extname(notes); // .txt
```

Получить имя файла без расширения можно, указав второй аргумент для `basename`:
```
path.basename(notes, path.extname(notes)); // notes
```

## Работа с путями

Соединить две или более частей пути можно с помощью `path.join()`:
```js
const name = 'joe';
path.join('/', 'users', name, 'notes.txt'); // '/users/joe/notes.txt'
```

Получить абсолютный путь, вычисленный из относительного, можно с помощью `path.resolve()`:
```
path.resolve('joe.txt'); // '/Users/joe/joe.txt', если запущено из моей домашней папки
```

В этом случае Node.js просто добавит `/joe.txt` к текущему рабочему каталогу. Если указать вторым параметром папку, `resolve` использует первый аргумент как основу для второго:
```
path.resolve('tmp', 'joe.txt'); // '/Users/joe/tmp/joe.txt', если запущено из моей домашней папки
```

Если первый параметр начинается со слэша, это означает, что это абсолютный путь:
```
path.resolve('/etc', 'joe.txt'); // '/etc/joe.txt'
```

`path.normalize()` — ещё одна полезная функция, которая попытается вычислить фактический путь, когда он содержит относительные спецификаторы вроде `.` или `..` либо двойные слэши:
```
path.normalize('/users/joe/..//test.txt'); // '/users/test.txt'
```

**Ни `resolve`, ни `normalize` не проверяют, существует ли путь**. Они лишь вычисляют путь на основе полученной информации.

# Как опубликовать Node-API-версию пакета вместе с версией без Node-API

Следующие шаги проиллюстрированы на примере пакета `iotivity-node`:

  * Сначала опубликуйте версию без Node-API: 
    * Обновите версию в `package.json`. Для `iotivity-node` версия становится `1.2.0-2`.
    * Пройдите по чек-листу релиза (убедитесь, что тесты/демо/документация в порядке)
    * `npm publish`
  * Затем опубликуйте Node-API-версию: 
    * Обновите версию в `package.json`. В случае `iotivity-node` версия становится `1.2.0-3`. Для версионирования мы рекомендуем следовать схеме предварительных версий (pre-release), описанной на [semver.org](https://semver.org/#spec-item-9), например `1.2.0-napi`.
    * Пройдите по чек-листу релиза (убедитесь, что тесты/демо/документация в порядке)
    * `npm publish --tag n-api`

В этом примере пометка релиза тегом `n-api` гарантировала, что, хотя версия 1.2.0-3 более поздняя, чем опубликованная версия без Node-API (1.2.0-2), она не будет установлена, если кто-то решит установить `iotivity-node`, просто запустив `npm install iotivity-node`. Это по умолчанию установит версию без Node-API. Пользователю придётся выполнить `npm install iotivity-node@n-api`, чтобы получить Node-API-версию. Дополнительную информацию об использовании тегов с npm смотрите в разделе [«Using dist-tags»](https://docs.npmjs.com/getting-started/using-tags).

## Как ввести зависимость от Node-API-версии пакета

Чтобы добавить Node-API-версию `iotivity-node` в качестве зависимости, `package.json` будет выглядеть так:
```json
"dependencies": {
  "iotivity-node": "n-api"
}
```

> Как объясняется в разделе [«Using dist-tags»](https://docs.npmjs.com/getting-started/using-tags), в отличие от обычных версий, версии с тегами нельзя адресовать диапазонами версий, такими как `"^2.0.0"`, внутри `package.json`. Причина в том, что тег ссылается ровно на одну версию. Поэтому, если сопровождающий пакета решит пометить более позднюю версию пакета тем же тегом, `npm update` получит эту более позднюю версию. В большинстве случаев это должно быть приемлемо. Если вам нужна версия, отличная от последней опубликованной, зависимость в `package.json` должна будет ссылаться на точную версию, как в примере ниже:
```json
"dependencies": {
  "iotivity-node": "1.2.0-3"
}
```

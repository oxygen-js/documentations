# Знакомство с таймерами JavaScript

M

## setTimeout
```
setTimeout(): void
```

При написании кода на JavaScript вам может понадобиться отложить выполнение функции.

Именно за это отвечает `setTimeout`. Вы указываете функцию обратного вызова (колбэк), которая должна выполниться позже, и значение, выражающее, насколько позже её нужно запустить, в миллисекундах:
```js
setTimeout(() => {
  // выполнится через 2 секунды
}, 2000);

setTimeout(() => {
  // выполнится через 50 миллисекунд
}, 50);
```

Такой синтаксис задаёт новую функцию. Внутри неё вы можете вызвать любую другую функцию, либо передать имя уже существующей функции и набор параметров:
```js
const myFunction = (firstParam, secondParam) => {
  // сделать что-нибудь
};

// выполнится через 2 секунды
setTimeout(myFunction, 2000, firstParam, secondParam);
```

В Node.js `setTimeout` возвращает экземпляр [`Timeout`](https://nodejs.org/api/timers.html#class-timeout), тогда как в браузерах возвращается числовой идентификатор таймера. Этот объект или идентификатор можно использовать, чтобы отменить запланированное выполнение функции:
```js
const timeout = setTimeout(() => {
  // должно выполниться через 2 секунды
}, 2000);

// я передумал
clearTimeout(timeout);
```

### Нулевая задержка

Если задать задержку таймаута равной `0`, функция обратного вызова выполнится как можно скорее, но уже после завершения текущей функции:
```js
setTimeout(() => {
  console.log('after ');
}, 0);

console.log(' before ');
```

Этот код выведет
```bash
before
after
```

Это особенно полезно, чтобы не блокировать CPU при интенсивных задачах и позволить другим функциям выполняться во время тяжёлых вычислений, ставя функции в очередь планировщика.

> Некоторые браузеры (IE и Edge) реализуют метод `setImmediate()`, который делает ровно то же самое, но он не является стандартом и [недоступен в других браузерах](https://caniuse.com/#feat=setimmediate). Однако в Node.js это стандартная функция.

## setInterval
```
setInterval(): void
```

`setInterval` — функция, похожая на `setTimeout`, с одним отличием: вместо однократного запуска функции обратного вызова она будет выполнять её бесконечно, через заданный вами интервал времени (в миллисекундах):
```js
setInterval(() => {
  // выполняется каждые 2 секунды
}, 2000);
```

Функция выше выполняется каждые 2 секунды, пока вы не прикажете ей остановиться с помощью `clearInterval`, передав идентификатор интервала, который вернул `setInterval`:
```js
const timeout = setInterval(() => {
  // выполняется каждые 2 секунды
}, 2000);

clearInterval(timeout);
```

Часто `clearInterval` вызывают внутри самой функции обратного вызова setInterval, чтобы она сама определяла, стоит ли ей выполниться снова или остановиться. Например, этот код выполняет что-то до тех пор, пока `App.somethingIWait` не примет значение `arrived`:
```js
const interval = setInterval(() => {
  if (App.somethingIWait === 'arrived') {
    clearInterval(interval);
  }
  // иначе делать что-нибудь
}, 100);
```

## Рекурсивный setTimeout

`setInterval` запускает функцию каждые n миллисекунд, никак не учитывая, когда функция завершила своё выполнение.

Если функция всегда занимает одинаковое время, всё в порядке:
```
0         1000        2000        3000
|          |           |           |
|          |           |           |
|       [██████████]   |           |
|       [exe script]   |           |
|       [██████████]   |           |
|          |  [██████████]         |
|          |  [exe script]         |
|          |  [██████████]         |
|          |           |  [██████████]
|          |           |  [exe script]
|          |           |  [██████████]
|__________|___________|___________|____
```

Но, возможно, функция выполняется за разное время, например в зависимости от условий сети:
```
0         1000        2000        3000
|          |           |           |
|          |           |           |
|       [██████████]   |           |
|       [exe script]   |           |
|       [██████████]   |           |
|          |  [████████████████████]
|          |  [  execute script    ]
|          |  [████████████████████]
|          |           |     [██████████]
|          |           |     [exe script]
|          |           |     [██████████]
|__________|___________|___________|____
```

А может быть, одно длительное выполнение перекрывает следующее:
```
0         1000        2000        3000        4000
|          |           |           |           |
|     [████████████████████]       |           |
|     [    execute script  ]       |           |
|     [████████████████████]       |           |
|          [███████████████████████████]       |
|          [      execute script       ]       |
|          [███████████████████████████]       |
|          |           |           [██████████████]
|          |           |           [execute script]
|          |           |           [██████████████]
|__________|___________|___________|___________|__
```

Чтобы этого избежать, вы можете запланировать рекурсивный setTimeout, который будет вызываться после завершения функции обратного вызова:
```js
const myFunction = () => {
  // сделать что-нибудь

  setTimeout(myFunction, 1000);
};

setTimeout(myFunction, 1000);
```

чтобы получить такой сценарий:
```
0         1000         2000        3000
|          |            |           |
|     [█████████████]   |    [█████████████]
|     [ exec script ]>──1s──<[ exec script ]
|     [█████████████]   |    [█████████████]
|          |            |           |
|__________|____________|___________|__________
```

`setTimeout` и `setInterval` доступны в Node.js через [модуль Timers](https://nodejs.org/api/timers.html).

Node.js также предоставляет `setImmediate()`, который планирует выполнение колбэка на фазе проверки (check) цикла событий. В отличие от `setTimeout(() => {}, 0)`, он, как правило, не эквивалентен ему, поскольку их колбэки выполняются на разных фазах цикла событий, а порядок их выполнения зависит от контекста выполнения.

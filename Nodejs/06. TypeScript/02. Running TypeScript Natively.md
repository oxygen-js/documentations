# Нативный запуск TypeScript

Вы можете писать код, являющийся корректным TypeScript, непосредственно в Node.js без необходимости предварительной транспиляции.

Node.js выполняет TypeScript с помощью лёгкого процесса, называемого удалением типов (type stripping). Он убирает стираемый синтаксис TypeScript, такой как аннотации типов и интерфейсы, а затем выполняет оставшийся JavaScript.

Если вы используете v22.18.0 или новее и ваш исходный код содержит только [стираемый синтаксис TypeScript](https://devblogs.microsoft.com/typescript/announcing-typescript-5-8-beta/#the---erasablesyntaxonly-option), вы можете выполнять код TypeScript без каких-либо флагов.
```
node example.ts
```

Если вы используете версию младше v22.18.0, вы можете использовать флаг `--experimental-strip-types`, чтобы запускать файлы TypeScript напрямую в Node.js.
```
node --experimental-strip-types example.ts
```

Вот и всё! Теперь вы можете запускать код TypeScript напрямую в Node.js без необходимости предварительной транспиляции.

Node.js не проверяет типы вашего кода при выполнении файлов TypeScript. Если вы хотите отлавливать ошибки, связанные с типами, используйте компилятор TypeScript отдельно:
```
npx tsc --noEmit
```

При необходимости вы можете отключить это с помощью флага [`--no-experimental-strip-types`](https://nodejs.org/docs/latest-v22.x/api/cli.html#--no-experimental-strip-types).
```
node --no-experimental-strip-types example.ts
```

### Удаление типов

Удаление типов работает только для того синтаксиса TypeScript, который можно убрать, не меняя JavaScript времени выполнения. Сюда входит распространённый чисто типовой синтаксис, такой как аннотации типов, интерфейсы, псевдонимы типов и `import type`.

Синтаксис, требующий генерации кода JavaScript, не обрабатывается одним лишь удалением типов. Примеры включают `enum`, свойства параметров (parameter properties), пространства имён с кодом времени выполнения и псевдонимы импортов (import aliases). Если вашему проекту нужны эти возможности, используйте раннер или отдельный шаг транспиляции.

### Проверка типов

Запуск файла `.ts` с помощью `node` — это не то же самое, что запуск `tsc`. Node.js выполняет файл после удаления поддерживаемого типового синтаксиса, но не сообщает об ошибках типов.

Для разработки распространённая конфигурация — запускать Node.js напрямую для быстрой обратной связи и запускать `tsc --noEmit` в отдельной команде или в CI-задании для проверки типов.

### Конфигурация

Загрузчик TypeScript в Node.js ([Amaro](https://github.com/nodejs/amaro)) не требует и не использует `tsconfig.json` для запуска кода TypeScript.

Мы рекомендуем настроить ваш редактор и `tsc` так, чтобы они отражали поведение Node.js, создав `tsconfig.json` с параметрами `compilerOptions`, перечисленными [здесь](https://nodejs.org/api/typescript.html#type-stripping), а также использовать TypeScript версии **5.7 или выше**.

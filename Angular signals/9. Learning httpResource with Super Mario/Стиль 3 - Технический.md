## Изучаем httpResource на примере Super Mario

В Angular 19.2 добавлен новый `httpResource`, выполняющий загрузку данных в рамках реактивного потока (reactive flow). В статье рассматривается применение этой — на данный момент экспериментальной — возможности на примере реализации простого приложения, прокручивающего уровни в стиле классической игры Super Mario.

Каждый уровень состоит из тайлов (плиток), доступных в 4 различных стилях оформления — Overworld, Underground, Underwater и Castle, — которые можно произвольно выбирать:

📂 [Исходный код](https://github.com/manfredsteyer/mario.git) предоставляет рассматриваемый компонент, который через `httpResource` обеспечивает загрузку файлов уровней (JSON) и тайлов для их отрисовки. Для рендеринга и анимации уровней код также **содержит** очень простой **«движок» (engine)**, рассматриваемый в статье как **чёрный ящик (black box)**.

## Файлы уровней

Различные уровни представлены JSON-файлами, определяющими, какие тайлы (например, пол или облако) отображаются в каких координатах:

```json
{
  "levelId": 1,
  "backgroundColor": "#9494ff",
  "items": [
    { "tileKey": "floor", "col": 0, "row": 13, [...] },
    { "tileKey": "cloud", "col": 12, "row": 1, [...] },
    [...]
  ]
}
```

Координаты задают позицию внутри матрицы блоков размером 16x16 пикселей. Помимо этих файлов уровней существует `overview.json`, содержащий информацию о доступных уровнях:

```json
{
  "levels": [
 {
      "levelKey": "01",
      "title": "Level 1"
 },
 {
      "levelKey": "02",
      "title": "Level 2"
 },
 [...]
 ]
}
```

## Загрузка JSON с помощью ngResource

Для абстрагирования доступа к данным целесообразно скрыть создание `httpResource` внутри сервиса:

```typescript
@Injectable({ providedIn: 'root' })
export class LevelLoader {
  getLevelOverviewResource(): HttpResourceRef<LevelOverview> {
    return httpResource<LevelOverview>(() => '/levels/overview.json', {
      defaultValue: initLevelOverview,
    });
  }

  getLevelResource(levelKey: () => string | undefined): HttpResourceRef<Level> {
    return httpResource<Level>(() => !levelKey() ? undefined : /levels/${levelKey()}.json, {
      defaultValue: initLevel,
    });
  }

 [...]
}
```

Объект опций, передаваемый вторым параметром, позволяет задать значение по умолчанию (`defaultValue`), которое используется до завершения загрузки ресурса (resource).

Для указания загружаемого уровня `getLevelResource` ожидает сигнал (Signal), возвращающий ключ уровня, на основании которого сервис формирует имя файла. Этот сигнал доступен только для чтения и описан обобщённой абстракцией `() => string | undefined` (то есть функцией без аргументов, возвращающей строку или `undefined`).

Обратите внимание, что URL, передаваемый в `getLevelResource`, представлен **лямбда-выражением.** Это требуется для того, чтобы ресурс **автоматически пересчитывал** URL при изменении входящего в выражение сигнала `levelKey`. Внутренне `httpResource` использует это лямбда-выражение для построения вычисляемого сигнала (Computed Signal), выполняющего роль триггера: при каждом изменении триггера ресурс выполняет загрузку URL.

Чтобы предотвратить срабатывание `httpResource`, выражение должно вернуть `undefined`. Это необходимо в случаях, когда загрузку требуется отложить до момента появления конкретных параметров, таких как `levelKey` (то есть пока зависимости запроса не определены).

## Дополнительные опции через явный HttpResourceRequest

Для получения более детального контроля над формируемым HTTP-запросом (request) вызывающая сторона может передать объект `HttpResourceRequest` вместо URL:

```typescript
getLevelResource(levelKey: () => string) {
  return httpResource<Level>(
    () => ({
      url: /levels/${levelKey()}.json,
      method: "GET",
      headers: {
        accept: "application/json",
      },
      params: {
        levelId: levelKey(),
      },
      reportProgress: false,
      body: null,
      transferCache: false,
      withCredentials: false,
    }),
    { defaultValue: initLevel }
  );
}
```

Этот `HttpResourceRequest` также может быть представлен лямбда-выражением, на основе которого `httpResource` внутренне строит вычисляемый сигнал (Computed Signal).

Важно отметить, что `httpResource` предназначен **исключительно для получения данных,** несмотря на то что допускает задание метода (HTTP-глагола) помимо `GET` и тела запроса `body`, передаваемого как полезная нагрузка. Эта опция позволяет работать с веб-API, не согласующимися с семантикой HTTP-глаголов. По умолчанию переданное `body` сериализуется в JSON. Однако данная опция не предназначена для записи данных обратно на сервер — эта задача `httpResource` не покрывается. Кроме того, хотя автоматическое применение семантики `switchMap` (отмена предыдущего запроса при появлении нового) подходит для чтения данных, при записи на сервер оно с высокой вероятностью является некорректным поведением.

С помощью опции `reportProgress` вызывающая сторона может запросить информацию о ходе выполнения текущей операции. Это удобно при скачивании файлов большого размера. Пример приведён в разделе ниже.

## Разбор и валидация полученных данных

По умолчанию `httpResource` ожидает JSON, соответствующий заданному параметру типа. Поэтому после разбора (parsing) полученного JSON он применяет лишь приведение типа (type assertion), благодаря чему TypeScript считает данные значением указанного типа (без рантайм-проверки).

Тем не менее в этот процесс можно встроиться, предоставив собственную логику валидации полученного сырого значения и преобразования его к типу, заданному параметром типа. Для этого в объекте опций задаётся функция `parse`:

```typescript
getLevelResourceAlternative(levelKey: () => string) {
  return httpResource<Level>(() => /levels/${levelKey()}.json, {
    defaultValue: initLevel,
    parse: (raw) => {
      return toLevel(raw);
    },
  });
}
```

`httpResource` преобразует полученный JSON в объект типа `unknown` и передаёт его в `parse`. В рассматриваемом примере используется простая написанная вручную функция `toLevel`. Однако `parse` можно также применить для интеграции ресурса с библиотекой валидации схем, например [Zod](https://zod.dev/).

## Загрузка данных в форматах, отличных от JSON

Хотя по умолчанию `httpResource` ожидает JSON, преобразованный в JavaScript-объект, вызывающая сторона может запросить и другие представления данных:

* `httpResource.text` возвращает простой текст
* `httpResource.blob` возвращает полученные данные в виде Blob
* `httpResource.arrayBuffer` возвращает полученные данные в виде ArrayBuffer

Для демонстрации в примере выполняется запрос изображения со всеми возможными тайлами в виде Blob. Из этого Blob можно извлечь нужные тайлы для выбранного стиля уровня.

На следующем изображении показан фрагмент карты тайлов; видно, что пример легко переключается между различными стилями за счёт горизонтального или вертикального смещения (offset):

![Фрагмент карты тайлов, используемой в примере](https://www.angulararchitects.io/wp-content/uploads/2025/02/tile-map.png)

Эта карта тайлов взята [отсюда](https://www.spriters-resource.com/nes/supermariobros/sheet/52571). Для загрузки карты тайлов сервис `TilesMapLoader` делегирует работу `httpResource.blob`:

```typescript
@Injectable({ providedIn: "root" })
export class TilesMapLoader {
  getTilesMapResource(): HttpResourceRef<Blob | undefined> {
    return httpResource.blob(() => {
      url: "/tiles.png",
      reportProgress: true,
    });
  }
}
```

Данный ресурс также запрашивает информацию о прогрессе, которую пример использует для отображения хода загрузки слева от выпадающих списков.

## Замечание на полях: используемый под капотом HttpClient допускает применение интерсепторов

Под капотом новый `httpResource` в настоящее время использует `HttpClient`. Следовательно, необходимо предоставить HttpClient, что обычно выполняется через `provideHttpClient` на этапе начальной загрузки (bootstrapping). Как следствие, `httpResource` также автоматически задействует зарегистрированные `HttpInterceptor`-ы.

При этом использование `HttpClient` является деталью реализации, которая со временем может быть заменена другой реализацией.

## Собираем всё воедино: реактивный поток

Теперь, когда у нас имеются фабрики для всех `httpResource`-ов, можно описать реактивный граф (reactive graph):

![Реактивный поток ngMario](https://www.angulararchitects.io/wp-content/uploads/2025/06/graph-scaled.png)

Сигналы `levelKey`, `style` и `animation` представляют пользовательский ввод. Первые два соответствуют выпадающим спискам в верхней части. Сигнал `animation` содержит значение типа `boolean`, указывающее, запущена ли анимация нажатием кнопки `Toggle Animation` (см. скриншот выше).

`tilesResource` в верхней центральной части — это классический ресурс, формирующий отдельные тайлы для выбранного стиля. Для этого он, по сути, делегирует работу функции, предоставляемой игровым «движком», рассматриваемым здесь как чёрный ящик.

Рендеринг инициируется эффектом (effect), поскольку отрисовать уровень напрямую через привязку данных (data binding) невозможно. Эффект выполняет отрисовку или анимацию уровня на холсте (canvas), полученном как сигнальный `viewChild`. Эффект вызывается при каждом изменении `level` (предоставляемого `levelResource`), `style`, флага `animation` или `canvas`.

Сигнал `tilesMapProgress` использует информацию о прогрессе из `tilesMapResource`, чтобы отобразить долю уже скачанной карты тайлов. Для загрузки списка доступных уровней пример использует `levelOverviewResource`, не имеющий прямой связи с рассмотренным выше реактивным графом.

Следующий листинг показывает реализацию данного реактивного потока в виде членов класса `LevelComponent`:

```typescript
export class LevelComponent implements OnDestroy {
  private tilesMapLoader = inject(TilesMapLoader);
  private levelLoader = inject(LevelLoader);

  canvas = viewChild<ElementRef<HTMLCanvasElement>>("canvas");

  levelKey = linkedSignal<string | undefined>(() => this.getFirstLevelKey());
  style = signal<Style>("overworld");
  animation = signal(false);

  tilesMapResource = this.tilesMapLoader.getTilesMapResource();
  levelResource = this.levelLoader.getLevelResource(this.levelKey);
  levelOverviewResource = this.levelLoader.getLevelOverviewResource();

  tilesResource = createTilesResource(this.tilesMapResource, this.style);

  tilesMapProgress = computed(() =>
    calcProgress(this.tilesMapResource.progress())
  );

  constructor() {
    [...]
    effect(() => {
      this.render();
    });
  }

  reload() {
    this.tilesMapResource.reload();
    this.levelResource.reload();
  }

  private getFirstLevelKey(): string | undefined {
    return this.levelOverviewResource.value()?.levels?.[0]?.levelKey;
  }

  [...]
}
```

Применение `linkedSignal` для `levelKey` позволяет использовать первый уровень в качестве значения по умолчанию после загрузки списка уровней. Вспомогательная функция `getFirstLevelKey` возвращает этот уровень из `levelOverviewResource`.

Эффект рендеринга, по сути, считывает упомянутые значения и делегирует работу функциям «движка» `animateLevel` или `rederLevel`:

```typescript
private render() {
  const tiles = this.tilesResource.value();
  const level = this.levelResource.value();
  const canvas = this.canvas()?.nativeElement;
  const animation = this.animation();

  if (!tiles || !canvas) {
    return;
  }

  if (animation) {
    animateLevel({
      canvas,
      level,
      tiles,
    });
  } else {
    renderLevel({
      canvas,
      level,
      tiles,
    });
  }
}
```

## Ресурсы и отсутствующие параметры

`tilesResource`, изображённый на диаграмме выше, по сути делегирует работу асинхронной функции `extractTiles`, также предоставляемой игровым «движком»:

```typescript
function createTilesResource(
  tilesMapResource: HttpResourceRef<Blob | undefined>,
  style: () => Style
) {
  const tilesMap = tilesMapResource.value();

  // undefined предотвращает срабатывание ресурса
  const params = computed(() =>
    !tilesMap
      ? undefined
      : {
          tilesMap: tilesMap,
          style: style(),
        }
  );

  return resource({
    params,
    loader: (loaderParams) => {
      const { tilesMap, style } = loaderParams.params;
      return extractTiles(tilesMap, style);
    },
  });
}
```

Этот простой ресурс содержит заслуживающую внимания деталь: пока карта тайлов не загружена, `value` у `tilesMapResource` равно `undefined`. Однако без `tilesMap` невозможно вызвать `extractTiles` и, соответственно, запустить `tilesResource`. Это учитывается сигналом `params`: в данном случае он также возвращает `undefined`, в результате чего **загрузчик (loader) не вызывается** (то есть отсутствие параметров блокирует запуск ресурса).

## Отображение прогресса

Выше `tilesMapResource` был сконфигурирован на предоставление информации о ходе загрузки через сигнал `progress`. С помощью вычисляемого сигнала в `LevelComponent` эта информация проецируется в строку:

```typescript
function calcProgress(progress: HttpProgressEvent | undefined): string {
  if (!progress) {
    return "-";
  }

  if (progress.total) {
    const percent = Math.round((progress.loaded / progress.total) * 100);
    return percent + "%";
  }

  const kb = Math.round(progress.loaded / 1024);
  return kb + " KB";
}
```

Если сервер передаёт общий размер файла, функция вычисляет процент загрузки. В противном случае она отображает количество скачанных килобайт. До начала загрузки информация о прогрессе отсутствует — в этом случае возвращается прочерк.

Чтобы протестировать эту возможность, можно искусственно ограничить пропускную способность сети в инструментах разработчика браузера и нажать кнопку `reload`, вызывающую метод `reload` ресурса.

## Статус, заголовки, ошибка и не только

На случай, если приложению требуется код статуса или полученные заголовки (headers), `httpResource` предоставляет соответствующие сигналы:

```typescript
console.log("status", this.levelOverviewResource.status());
console.log("statusCode", this.levelOverviewResource.statusCode());
console.log("headers", this.levelOverviewResource.headers()?.keys());
```

Помимо этого, `httpResource` предоставляет всё то, что уже известно по обычным ресурсам: сигнал `error`, информирующий о возможной ошибке, и возможность обновления `value`, трактуемого как локальная рабочая копия. Это также означает, что значение не записывается на сервер автоматически — данную задачу по-прежнему необходимо реализовывать классическим способом.

## Заключение

Новый `httpResource` — это ещё один строительный блок, дополняющий новую модель реактивности в Angular. Он позволяет загружать данные в рамках реактивного графа. В настоящее время он использует `HttpClient` как деталь реализации, которая со временем может быть заменена другим решением.

Хотя `HTTP resource` допускает получение данных с использованием HTTP-глаголов помимо GET (например, посредством POST-вызовов), он не предназначен для записи данных обратно на сервер. Эту задачу по-прежнему необходимо решать традиционным способом.
